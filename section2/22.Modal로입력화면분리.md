# 22. Modal로 입력 화면 분리

## 핵심 요약

- React Native는 화면 위에 오버레이를 띄우기 위한 내장 `Modal` 컴포넌트를 제공한다.
- 목표 입력 UI를 `Modal` 안으로 옮기면 메인 화면과 입력 화면을 분리할 수 있다.
- 모달 표시 여부는 부모 컴포넌트인 `App`의 상태로 관리한다.
- `Button`의 `onPress`로 모달을 여는 핸들러를 실행한다.
- `Modal`의 `visible` prop으로 모달 표시 여부를 제어하고, `animationType`으로 표시 애니메이션을 설정할 수 있다.

## 학습 목표

- React Native의 `Modal` 컴포넌트 역할을 이해한다.
- 부모 상태로 자식 컴포넌트 안의 모달 표시 여부를 제어하는 방법을 익힌다.
- `visible`과 `animationType` prop의 기본 사용법을 배운다.
- 입력 화면을 메인 화면에서 분리하는 UI 구조를 이해한다.

## 주요 내용

### Modal의 역할

모달은 모바일 앱에서 메인 화면 위에 뜨는 팝업이나 오버레이 화면이다. 사용자는 모달에서 특정 작업을 수행한 뒤 다시 메인 화면으로 돌아온다.

목표 관리 앱에서는 새 목표를 입력하는 UI를 모달로 옮긴다. 이렇게 하면 메인 화면에는 목표 목록과 모달을 여는 버튼만 남기고, 입력 화면은 필요할 때만 보여줄 수 있다.

### GoalInput에 Modal 추가하기

목표 입력 UI는 이미 `GoalInput` 컴포넌트에 분리되어 있다. 따라서 `GoalInput`에서 React Native의 `Modal`을 import하고, 기존 입력 UI를 `Modal`로 감싼다.

```jsx
import { Button, Modal, StyleSheet, TextInput, View } from 'react-native';
```

```jsx
function GoalInput(props) {
  return (
    <Modal>
      <View style={styles.inputContainer}>
        <TextInput
          style={styles.textInput}
          placeholder="Your course goal!"
          onChangeText={goalInputHandler}
          value={enteredGoalText}
        />
        <Button title="Add Goal" onPress={addGoalHandler} />
      </View>
    </Modal>
  );
}
```

이 상태에서는 모달이 항상 화면을 덮기 때문에 메인 화면의 목표 목록이나 버튼이 보이지 않을 수 있다.

### 모달을 여는 버튼 추가

모달은 처음부터 항상 떠 있으면 안 된다. 메인 화면의 버튼을 눌렀을 때만 열리도록 만든다.

`App.js`에서 `GoalInput` 위에 버튼을 추가한다.

```jsx
<Button
  title="Add New Goal"
  color="#5e0acc"
  onPress={startAddGoalHandler}
/>
```

React Native의 기본 `Button`은 `style` prop을 지원하지 않는다. 대신 `color` prop으로 버튼 색상을 어느 정도 바꿀 수 있다.

더 자유롭게 스타일링하려면 나중에 `Pressable`, `View`, `Text`를 조합해 커스텀 버튼을 만들 수 있다.

### 모달 표시 상태 만들기

모달을 열고 닫으려면 표시 여부를 상태로 관리한다. 이 상태는 모달을 여는 버튼과 `GoalInput` 컴포넌트 모두에 영향을 주므로 부모인 `App`에 둔다.

```jsx
const [modalIsVisible, setModalIsVisible] = useState(false);
```

처음에는 모달이 보이지 않아야 하므로 초기값은 `false`다.

모달을 여는 핸들러도 만든다.

```jsx
function startAddGoalHandler() {
  setModalIsVisible(true);
}
```

버튼의 `onPress`에 이 함수를 연결하면 버튼을 눌렀을 때 상태가 `true`로 바뀐다.

### 조건부 렌더링 방식

가장 단순한 방법은 `modalIsVisible`이 `true`일 때만 `GoalInput`을 렌더링하는 것이다.

```jsx
{modalIsVisible && <GoalInput onAddGoal={addGoalHandler} />}
```

이 방식도 동작하지만, React Native의 `Modal` 컴포넌트가 제공하는 `visible` prop을 사용하는 편이 더 적합하다.

### Modal의 visible prop

`Modal`에는 표시 여부를 제어하는 `visible` prop이 있다.

`GoalInput`은 부모로부터 `visible` prop을 받아 `Modal`에 전달한다.

```jsx
function GoalInput(props) {
  return (
    <Modal visible={props.visible}>
      {/* 입력 UI */}
    </Modal>
  );
}
```

`App`에서는 `modalIsVisible` 상태를 `GoalInput`의 `visible` prop으로 전달한다.

```jsx
<GoalInput
  visible={modalIsVisible}
  onAddGoal={addGoalHandler}
/>
```

이렇게 하면 `App`의 상태가 모달 표시 여부를 제어하게 된다.

### animationType prop

`Modal`은 표시될 때의 애니메이션도 설정할 수 있다.

```jsx
<Modal visible={props.visible} animationType="slide">
  {/* 입력 UI */}
</Modal>
```

사용할 수 있는 값은 대표적으로 다음과 같다.

```text
none: 애니메이션 없음
fade: 서서히 나타남
slide: 아래에서 위로 올라옴
```

강의에서는 모달이 아래에서 올라오는 느낌을 주기 위해 `slide`를 사용한다.

### 아직 남은 작업

이 단계에서는 모달을 열 수는 있지만 아직 닫을 수 없다. 또한 모달 내부 스타일도 아직 완성되지 않았다.

다음 단계에서는 모달 화면을 더 보기 좋게 꾸미고, 모달을 닫는 기능을 추가한다.

## 정리

이번 강의에서는 목표 입력 UI를 `Modal`로 옮기고, `App` 컴포넌트의 상태로 모달 표시 여부를 제어했다. 메인 화면에는 모달을 여는 버튼을 추가하고, `GoalInput`에는 `visible` prop을 전달해 `Modal`의 표시 상태와 연결했다.

`Modal`은 `animationType`을 통해 표시 애니메이션을 설정할 수 있으며, 입력 화면처럼 메인 화면 위에 잠시 떠야 하는 UI를 만들 때 유용하다.
