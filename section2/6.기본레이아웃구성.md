# 6. 기본 레이아웃 구성

## 핵심 요약

- 단순히 중앙에 텍스트와 버튼을 두는 화면에서 벗어나 실제 앱 구조에 가까운 레이아웃을 만든다.
- 목표 입력 영역과 목표 목록 영역을 각각 별도의 `View`로 분리한다.
- 사용자가 목표를 입력할 수 있도록 `TextInput` 컴포넌트를 사용한다.
- 목표를 추가하는 버튼은 `Button` 컴포넌트로 배치한다.
- 화면 상단의 상태 표시줄과 겹치지 않도록 외부 컨테이너에 `padding`을 적용한다.
- 입력 영역과 목록 영역의 비율 배치를 위해 다음 단계에서 Flexbox를 사용하게 된다.

## 학습 목표

- `View`를 사용해 화면 영역을 나누는 기본 방식을 이해한다.
- `TextInput`과 `Button`을 조합해 입력 UI를 구성한다.
- 전체 화면 컨테이너에 스타일을 적용하는 방법을 복습한다.
- 왜 Flexbox가 React Native 레이아웃에서 중요한지 이해한다.

## 주요 내용

### 기존 화면 정리

이번 단계에서는 기존 JSX와 스타일을 정리하고, 목표 관리 앱의 기본 화면 구조를 새로 만든다.

기존에 있던 텍스트와 버튼, 그리고 외부 `View`에 적용된 스타일을 제거한다. `StyleSheet` 객체는 이후 스타일을 다시 정의할 예정이므로 남겨둔다.

### 두 개의 영역 만들기

목표 관리 앱의 화면은 크게 두 영역으로 나눌 수 있다.

```text
목표 입력 영역
목표 목록 영역
```

이를 위해 외부 `View` 안에 중첩된 `View` 두 개를 만든다. 첫 번째 `View`는 목표를 입력하고 추가하는 영역이고, 두 번째 `View`는 추가된 목표 목록을 보여주는 영역이다.

```jsx
<View style={styles.appContainer}>
  <View>
    {/* 목표 입력 영역 */}
  </View>

  <View>
    {/* 목표 목록 영역 */}
  </View>
</View>
```

이처럼 영역을 분리해두면 이후 스타일링과 레이아웃 조정이 쉬워진다.

### TextInput 추가

사용자가 목표를 입력할 수 있도록 `TextInput` 컴포넌트를 사용한다. `TextInput`도 React Native의 코어 컴포넌트이므로 `react-native`에서 import해야 한다.

```jsx
import { Button, StyleSheet, Text, TextInput, View } from 'react-native';
```

`TextInput`은 셀프 클로징 컴포넌트로 사용할 수 있으며, 웹의 input처럼 `placeholder`를 지정할 수 있다.

```jsx
<TextInput placeholder="Your course goal!" />
```

### Button과 목록 자리 추가

입력 영역에는 목표를 추가하기 위한 버튼도 배치한다.

```jsx
<Button title="Add Goal" />
```

목록 영역에는 이후 실제 목표 목록을 렌더링할 예정이므로, 우선 임시 텍스트를 넣어둔다.

```jsx
<View>
  <Text>List of goals...</Text>
</View>
```

### 외부 컨테이너에 padding 적용

처음 JSX를 구성하면 요소들이 화면 맨 위에 붙어 상태 표시줄과 겹칠 수 있다. 이를 피하기 위해 가장 바깥쪽 `View`에 스타일을 적용한다.

```jsx
const styles = StyleSheet.create({
  appContainer: {
    padding: 50,
  },
});
```

그리고 외부 `View`에 이 스타일을 전달한다.

```jsx
<View style={styles.appContainer}>
  {/* 입력 영역과 목록 영역 */}
</View>
```

이렇게 하면 화면 전체에 안쪽 여백이 생기고, 상태 표시줄과 콘텐츠가 겹치는 문제를 피할 수 있다.

### 아직 부족한 레이아웃

`padding`을 적용하면 콘텐츠가 보기 좋아지지만, 아직 원하는 앱 레이아웃은 아니다.

현재 상태에서는 입력창과 버튼이 위아래로 배치되고, 목표 목록과의 간격도 충분하지 않다. 원하는 구조는 입력 영역이 화면의 일부만 차지하고, 나머지 공간에 목표 목록이 표시되는 형태다.

```text
상단 일부: 목표 입력 + 추가 버튼
나머지 영역: 목표 목록
```

이런 비율과 배치를 만들기 위해 다음 단계에서 Flexbox를 사용한다.

## 정리

이번 강의에서는 목표 관리 앱의 기본 화면 구조를 만들었다. 외부 컨테이너 안에 목표 입력 영역과 목록 영역을 분리하고, `TextInput`, `Button`, `Text`를 배치했다.

아직 배치는 단순하지만, 다음 단계에서 Flexbox를 사용해 각 영역의 크기와 컴포넌트 위치를 더 세밀하게 조정하게 된다.
