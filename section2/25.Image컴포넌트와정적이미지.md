# 25. Image 컴포넌트와 정적 이미지

## 핵심 요약

- React Native에서는 이미지를 표시할 때 `Image` 컴포넌트를 사용한다.
- 프로젝트에 포함된 정적 이미지는 보통 `assets/images` 같은 폴더에 보관한다.
- 로컬 이미지는 웹처럼 문자열 경로를 그대로 넣는 대신 `require()`로 불러와야 한다.
- `Image` 컴포넌트는 `style` prop을 지원하므로 너비, 높이, 여백 등을 지정할 수 있다.
- 흰색 이미지처럼 배경과 구분되지 않는 경우, 모달의 루트 `View`에 배경색을 지정해 이미지를 보이게 만들 수 있다.

## 학습 목표

- React Native에서 정적 이미지를 추가하는 기본 흐름을 이해한다.
- `Image` 컴포넌트의 `source` prop에 로컬 이미지를 연결하는 방법을 배운다.
- `require()` 경로가 현재 파일 기준 상대 경로라는 점을 정리한다.
- 이미지와 모달 배경을 함께 스타일링하는 방법을 익힌다.
- 기본 앱 스타일을 정리하면서 불필요한 경계선과 여백을 제거하는 흐름을 이해한다.

## 주요 내용

### Image 컴포넌트 추가

React Native에는 이미지를 표시하기 위한 내장 컴포넌트인 `Image`가 있다. 웹의 `<img>` 요소와 비슷한 역할을 하지만, 사용법에는 React Native만의 차이가 있다.

먼저 `GoalInput` 컴포넌트에서 `Image`를 import한다.

```jsx
import { Button, Image, Modal, StyleSheet, TextInput, View } from 'react-native';
```

강의에서는 목표 입력창 위에 이미지를 표시한다.

```jsx
<Image source={require('../assets/images/goal.png')} />
<TextInput
  style={styles.textInput}
  placeholder="Your course goal!"
  onChangeText={goalInputHandler}
  value={enteredGoalText}
/>
```

### 이미지 파일 위치

정적 이미지는 보통 프로젝트의 `assets` 폴더에 보관한다. 이미지가 여러 개라면 `assets/images` 폴더를 만들어 관리하면 구조가 더 명확해진다.

```text
assets/
  images/
    goal.png
```

강의에서는 과녁 이미지를 다운로드한 뒤 `assets/images/goal.png` 위치에 둔다.

### 로컬 이미지는 require로 불러오기

웹에서는 보통 이미지 경로를 문자열로 작성한다.

```jsx
<img src="assets/images/goal.png" />
```

하지만 React Native에서 로컬 이미지를 `Image`에 연결할 때는 단순 문자열 경로를 그대로 쓰지 않는다.

```jsx
<Image source="../assets/images/goal.png" />
```

위와 같은 방식은 정상적으로 동작하지 않는다. 대신 `require()`를 사용한다.

```jsx
<Image source={require('../assets/images/goal.png')} />
```

이때 경로는 `require()`가 작성된 파일을 기준으로 계산한다. 예를 들어 `GoalInput.js`가 `components` 폴더 안에 있다면, 상위 폴더로 한 번 올라간 뒤 `assets/images/goal.png`로 접근해야 한다.

```text
components/GoalInput.js
../assets/images/goal.png
```

### 이미지 스타일링

`Image` 컴포넌트는 `style` prop을 지원한다. 따라서 `StyleSheet`에 이미지 전용 스타일 객체를 만들 수 있다.

```jsx
<Image
  style={styles.image}
  source={require('../assets/images/goal.png')}
/>
```

```jsx
image: {
  width: 100,
  height: 100,
  margin: 20,
},
```

이미지는 보통 너비와 높이를 명시해 주는 편이 좋다. 여백을 함께 주면 입력창과 버튼 사이에서 이미지가 답답하게 붙지 않는다.

### 배경색 바꾸기

강의에서 사용하는 과녁 이미지는 흰색이라서, 흰색 배경 위에서는 잘 보이지 않는다. 이럴 때는 모달 내부의 루트 `View`에 배경색을 지정한다.

```jsx
inputContainer: {
  flex: 1,
  justifyContent: 'center',
  alignItems: 'center',
  padding: 16,
  backgroundColor: '#311b6b',
},
```

`Modal` 자체도 일부 prop을 제공하지만, 전체 화면의 실제 배경과 레이아웃은 내부에 있는 `View`를 통해 제어하는 방식이 더 자연스럽다.

### 불필요한 스타일 정리

입력 화면이 모달로 분리되면 이전 레이아웃에서 필요했던 일부 스타일은 어색해질 수 있다.

예를 들어 입력 영역 아래에 목록이 있던 시절에는 아래쪽 경계선이나 `marginBottom`이 자연스러웠지만, 이제 모달 안에는 입력 UI만 있으므로 제거하는 편이 낫다.

```jsx
inputContainer: {
  flex: 1,
  justifyContent: 'center',
  alignItems: 'center',
  padding: 16,
  backgroundColor: '#311b6b',
},
```

이렇게 하면 모달의 배경이 짙은 보라색으로 바뀌고, 흰색 이미지도 명확하게 보인다.

### 남은 스타일링 작업

이미지를 추가하고 배경을 바꾸면 입력창과 버튼이 기존 색상과 잘 어울리지 않을 수 있다. 예를 들어 어두운 배경 위에서는 입력창의 글자나 버튼 색상을 조정해야 더 보기 좋다.

이 강의의 핵심은 이미지 추가였고, 다음 강의에서는 모달 안의 다른 요소들을 더 보기 좋게 다듬는다.

## 정리

이번 강의에서는 React Native의 `Image` 컴포넌트로 로컬 이미지를 추가하는 방법을 배웠다. 프로젝트 내부의 정적 이미지는 `assets/images`에 두고, `source` prop에는 문자열 경로가 아니라 `require()`를 사용해 연결한다.

또한 이미지가 배경과 구분되지 않을 때는 모달 내부의 루트 `View`에 `backgroundColor`를 적용해 전체 분위기를 조정했다. 이미지 자체는 `style` prop으로 너비, 높이, 여백을 지정할 수 있다.
