# 20. filter로 아이템 삭제하기

## 핵심 요약

- 실제로 아이템을 삭제하려면 삭제 대상 아이템의 `id`를 알아야 한다.
- 상태를 이전 상태에 기반해 갱신할 때는 `setCourseGoals`에 함수를 전달하는 방식이 안전하다.
- JavaScript 배열 메서드 `filter`를 사용하면 특정 조건을 만족하는 아이템만 남긴 새 배열을 만들 수 있다.
- `GoalItem`에서 삭제 핸들러를 호출할 때 `bind`를 사용해 삭제 대상 `id`를 미리 전달할 수 있다.
- 아이템 삭제는 원본 배열을 직접 수정하는 것이 아니라, 삭제 대상이 제외된 새 배열로 상태를 갱신하는 방식으로 처리한다.

## 학습 목표

- React 상태를 이전 상태 기반으로 갱신하는 방법을 이해한다.
- `filter`를 사용해 배열에서 특정 아이템을 제거하는 패턴을 익힌다.
- 자식 컴포넌트에서 부모 컴포넌트의 핸들러에 필요한 값을 전달하는 방법을 이해한다.

## 주요 내용

### 삭제 대상 id가 필요한 이유

목록에서 특정 아이템을 삭제하려면 어떤 아이템을 삭제할지 식별할 수 있어야 한다. 이 앱에서는 각 목표 아이템이 `id`를 가지고 있으므로, 삭제 핸들러가 해당 `id`를 받을 수 있으면 정확한 아이템을 제거할 수 있다.

삭제 로직은 목록 상태를 관리하는 `App.js`의 `deleteGoalHandler` 안에 작성한다.

### 이전 상태를 기반으로 상태 갱신하기

삭제 후의 새 목록은 기존 목록을 기준으로 만들어진다. 이처럼 이전 상태에 의존해 새 상태를 만들 때는 상태 갱신 함수에 콜백을 전달한다.

```jsx
function deleteGoalHandler(id) {
  setCourseGoals((currentCourseGoals) => {
    return currentCourseGoals.filter((goal) => goal.id !== id);
  });
}
```

`currentCourseGoals`는 현재 목표 목록 배열이고, 이 배열에서 삭제 대상이 아닌 아이템만 남겨 새 배열을 만든다.

### filter 메서드

`filter`는 JavaScript 배열에 사용할 수 있는 메서드다. 각 아이템에 대해 콜백 함수를 실행하고, 콜백이 `true`를 반환한 아이템만 새 배열에 포함한다.

```jsx
currentCourseGoals.filter((goal) => goal.id !== id)
```

위 조건은 각 목표의 `id`가 삭제하려는 `id`와 다를 때만 `true`가 된다. 따라서 삭제 대상과 같은 `id`를 가진 아이템은 새 배열에서 제외된다.

### GoalItem에 id 전달하기

삭제 핸들러가 올바른 `id`를 받을 수 있으려면 `GoalItem` 컴포넌트에 해당 아이템의 `id`를 prop으로 전달해야 한다.

```jsx
<GoalItem
  id={itemData.item.id}
  text={itemData.item.text}
  onDeleteItem={deleteGoalHandler}
/>
```

이렇게 하면 `GoalItem` 내부에서 `props.id`를 사용할 수 있다.

### bind로 매개변수 미리 전달하기

`Pressable`의 `onPress`는 눌렸을 때 실행할 함수를 받는다. 이때 삭제 핸들러에 `id`를 전달하려면 `bind`를 사용할 수 있다.

```jsx
<Pressable onPress={props.onDeleteItem.bind(this, props.id)}>
  <View>
    <Text>{props.text}</Text>
  </View>
</Pressable>
```

`bind`의 첫 번째 인수는 함수 내부의 `this` 값을 지정한다. 이 예제에서는 `this`가 중요하지 않다. 두 번째 인수인 `props.id`는 나중에 호출될 `onDeleteItem` 함수의 첫 번째 매개변수로 전달된다.

이렇게 연결하면 사용자가 특정 `GoalItem`을 누를 때 해당 아이템의 `id`가 `deleteGoalHandler`로 전달된다.

### 삭제 동작 확인

이제 여러 아이템을 추가한 뒤 특정 아이템을 누르면, 해당 아이템만 목록에서 제거된다.

다만 이 단계에서는 터치했을 때 별도의 시각적 피드백이 없다. 아이템이 바로 사라지기만 하므로, 다음 단계에서는 스타일링과 피드백을 다룬다.

## 정리

아이템 삭제는 삭제 대상의 `id`를 기준으로 기존 목록에서 해당 아이템을 제외한 새 배열을 만드는 방식으로 구현한다. `filter`를 사용하면 상태를 직접 수정하지 않고도 원하는 아이템만 남긴 새 상태를 만들 수 있다.
