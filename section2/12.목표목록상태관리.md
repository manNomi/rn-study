# 12. 목표 목록 상태 관리

## 핵심 요약

- 목표 목록은 동적으로 변하는 데이터이므로 React 상태(State)로 관리한다.
- 목표가 처음에는 없으므로 `courseGoals` 상태의 초기값은 빈 배열로 설정한다.
- 새 목표를 추가할 때는 기존 배열을 직접 수정하지 않고 새 배열을 만들어 상태를 갱신한다.
- 이전 상태를 기반으로 새 상태를 만들 때는 상태 업데이트 함수에 콜백을 전달하는 방식이 권장된다.
- 배열 데이터는 `map`을 사용해 JSX 요소 목록으로 변환해 렌더링할 수 있다.
- React에서 목록을 렌더링할 때는 각 항목에 고유한 `key` prop이 필요하다.

## 학습 목표

- 배열 형태의 상태를 `useState`로 관리하는 방법을 이해한다.
- 이전 상태를 기반으로 상태를 안전하게 업데이트하는 패턴을 익힌다.
- `map`을 사용해 데이터 배열을 화면에 출력하는 방법을 복습한다.
- 목록 렌더링에서 `key` prop이 필요한 이유를 이해한다.

## 주요 내용

### 목표 목록 상태 만들기

목표 목록은 사용자가 목표를 추가할 때마다 바뀐다. 목록이 바뀌면 화면도 다시 렌더링되어야 하므로 상태로 관리해야 한다.

처음에는 목표가 없기 때문에 빈 배열을 초기값으로 사용한다.

```jsx
const [courseGoals, setCourseGoals] = useState([]);
```

`courseGoals`는 현재 목표 목록이고, `setCourseGoals`는 목표 목록을 업데이트하는 함수다.

### 새 목표 추가하기

버튼을 누르면 `addGoalHandler`가 실행된다. 이 함수 안에서 기존 목표 목록에 현재 입력값인 `enteredGoalText`를 추가한다.

단순히 기존 배열을 수정하는 것이 아니라, 기존 목표를 포함한 새 배열을 만들어 상태를 갱신한다.

```jsx
function addGoalHandler() {
  setCourseGoals((currentCourseGoals) => [
    ...currentCourseGoals,
    enteredGoalText,
  ]);
}
```

여기서 `...currentCourseGoals`는 기존 목표 배열의 모든 항목을 새 배열에 펼쳐 넣는다. 그 뒤에 `enteredGoalText`를 추가해 새 목표를 덧붙인다.

### 이전 상태 기반 업데이트

새로운 목표 목록은 기존 목표 목록에 의존한다. 이런 경우에는 상태 업데이트 함수에 직접 새 값을 전달하기보다, 콜백 함수를 전달하는 방식이 더 안전하다.

```jsx
setCourseGoals((currentCourseGoals) => {
  return [...currentCourseGoals, enteredGoalText];
});
```

React는 이 콜백을 호출할 때 현재 상태를 자동으로 매개변수로 전달한다. 위 예시에서는 `currentCourseGoals`가 현재 목표 목록이다.

이 방식은 상태 업데이트가 이전 상태에 의존할 때 권장되는 패턴이다.

### 목표 목록 출력하기

목표 목록을 화면에 출력하려면 `courseGoals` 배열을 JSX 요소 배열로 변환해야 한다. 웹 React에서처럼 `map`을 사용한다.

```jsx
{courseGoals.map((goal) => (
  <Text>{goal}</Text>
))}
```

`map`은 배열의 각 항목에 대해 함수를 실행하고, 그 결과로 새 배열을 만든다. 여기서는 목표 문자열 하나를 `Text` 컴포넌트 하나로 변환한다.

### key prop 경고

위 코드만으로도 목록은 화면에 표시되지만, React는 다음과 같은 경고를 띄운다.

```text
Each child in a list should have a unique key prop.
```

React는 목록을 효율적으로 업데이트하기 위해 각 항목을 식별할 수 있는 `key`가 필요하다. 목록 항목을 렌더링할 때는 각 요소에 고유한 `key` prop을 추가해야 한다.

이번 단계에서는 목표 텍스트 자체를 key로 사용한다.

```jsx
{courseGoals.map((goal) => (
  <Text key={goal}>{goal}</Text>
))}
```

다만 이 방식은 같은 목표 텍스트를 두 번 입력하면 key가 중복될 수 있다. 지금은 임시로 충분하지만, 이후에는 고유한 ID를 사용하도록 개선한다.

### 동작 확인

입력창에 목표를 입력하고 `Add Goal` 버튼을 누르면, 입력한 텍스트가 `courseGoals` 상태에 추가된다.

상태가 바뀌면 컴포넌트가 다시 렌더링되고, `map`으로 변환된 `Text` 목록이 화면에 표시된다.

이제 앱은 사용자 입력을 받아 상태에 저장하고, 그 상태를 화면에 목록으로 출력할 수 있다.

## 정리

이번 강의에서는 목표 목록을 배열 상태로 관리하고, 버튼을 누를 때 새 목표를 추가했다. 이전 상태를 기반으로 배열을 갱신하기 위해 상태 업데이트 함수에 콜백을 전달하고, 전개 연산자로 기존 목표를 유지한 채 새 목표를 추가했다.

또한 `map`을 사용해 목표 배열을 `Text` 컴포넌트 목록으로 렌더링하고, React 목록 렌더링에서 `key` prop이 필요한 이유를 확인했다.
