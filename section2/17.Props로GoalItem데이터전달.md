# 17. Props로 GoalItem 데이터 전달

## 핵심 요약

- 분리한 `GoalItem` 컴포넌트가 실제 목표 텍스트를 렌더링하려면 부모인 `App`에서 데이터를 전달해야 한다.
- React Native의 커스텀 컴포넌트도 웹 React와 동일하게 props를 사용할 수 있다.
- `FlatList`의 `renderItem`에서 `itemData.item.text` 값을 `GoalItem`의 `text` prop으로 전달한다.
- `GoalItem` 내부에서는 props를 받아 `Text` 컴포넌트에 출력한다.
- React Native 코어 컴포넌트인 `View`, `Text`는 사용하는 파일마다 `react-native`에서 import해야 한다.

## 학습 목표

- 커스텀 컴포넌트에 props로 데이터를 전달하는 방법을 복습한다.
- `FlatList`의 `itemData`에서 실제 목표 텍스트를 꺼내 전달하는 흐름을 이해한다.
- React Native에서는 코어 컴포넌트를 직접 import해야 한다는 점을 확인한다.

## 주요 내용

### GoalItem에 데이터가 필요한 이유

이전 단계에서 목표 아이템 UI를 `GoalItem` 컴포넌트로 분리했다. 하지만 `GoalItem`은 목표 항목 하나를 렌더링하는 컴포넌트이므로, 어떤 텍스트를 보여줄지 부모 컴포넌트로부터 받아야 한다.

React Native도 React 앱이기 때문에 일반적인 React props를 그대로 사용할 수 있다.

### App에서 text prop 전달하기

`FlatList`의 `renderItem` 함수 안에서는 현재 렌더링 중인 항목 데이터에 접근할 수 있다.

목표 데이터는 객체 형태이며, 실제 텍스트는 `itemData.item.text`에 들어 있다.

```jsx
<FlatList
  data={courseGoals}
  renderItem={(itemData) => {
    return <GoalItem text={itemData.item.text} />;
  }}
  keyExtractor={(item) => item.id}
/>
```

여기서 `text`는 직접 만든 prop 이름이다. 커스텀 컴포넌트의 prop 이름은 원하는 대로 정할 수 있지만, 의미가 분명한 이름을 사용하는 것이 좋다.

### GoalItem에서 props 사용하기

`GoalItem` 컴포넌트는 props를 매개변수로 받아 전달된 값을 사용할 수 있다.

```jsx
function GoalItem(props) {
  return (
    <View style={styles.goalItem}>
      <Text style={styles.goalText}>{props.text}</Text>
    </View>
  );
}
```

이제 `App`에서 전달한 목표 텍스트가 `GoalItem` 내부의 `Text` 컴포넌트에 출력된다.

### View와 Text import

`GoalItem.js`에서 `View`와 `Text`를 사용한다면 반드시 `react-native`에서 import해야 한다.

```jsx
import { StyleSheet, Text, View } from 'react-native';
```

웹 React에서는 `div`, `h2` 같은 HTML 요소를 import하지 않고 사용할 수 있다. 하지만 React Native의 `View`, `Text`는 HTML 요소가 아니라 React Native가 제공하는 코어 컴포넌트다.

따라서 사용하는 파일마다 명시적으로 import해야 한다. import하지 않으면 `View 변수를 찾을 수 없습니다` 같은 오류가 발생한다.

### 기존 UI 유지

`GoalItem`을 사용하도록 바꿔도 UI는 이전과 동일하게 보인다. 차이는 내부 구조에 있다.

이제 `App.js`는 목록 데이터를 관리하고, `GoalItem`은 목표 아이템 하나의 UI를 담당한다. 같은 화면을 더 작은 컴포넌트들의 조합으로 만든 것이다.

## 정리

이번 강의에서는 분리한 `GoalItem` 컴포넌트에 props로 목표 텍스트를 전달했다. `App`의 `FlatList`에서 `itemData.item.text`를 `text` prop으로 넘기고, `GoalItem`에서는 `props.text`를 출력한다.

React Native에서도 props 사용 방식은 웹 React와 동일하다. 다만 `View`, `Text` 같은 코어 컴포넌트는 HTML 태그가 아니므로 사용하는 파일에서 반드시 import해야 한다.
