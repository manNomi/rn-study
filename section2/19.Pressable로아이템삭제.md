# 19. Pressable로 아이템 삭제 준비하기

## 핵심 요약

- React Native에서는 웹처럼 요소에 `onClick`을 직접 붙여 터치 이벤트를 처리하지 않는다.
- 터치 가능한 UI를 만들려면 `Pressable` 같은 React Native 전용 컴포넌트로 감싸야 한다.
- `Pressable`은 내부 요소가 눌렸을 때 `onPress`에 전달된 함수를 실행한다.
- 삭제 로직은 목록 상태를 관리하는 `App.js`에 두고, `GoalItem`에는 prop으로 전달한다.
- 이번 단계에서는 실제 삭제 전, 아이템을 눌렀을 때 삭제 핸들러가 호출되는지 확인한다.

## 학습 목표

- React Native에서 터치 이벤트를 처리하는 기본 방식을 이해한다.
- `Pressable` 컴포넌트의 역할을 이해한다.
- 부모 컴포넌트의 함수를 자식 컴포넌트에 prop으로 전달하는 흐름을 복습한다.

## 주요 내용

### 웹의 onClick과 React Native의 차이

웹 앱에서는 요소에 `onClick` prop을 추가해 클릭 이벤트를 처리할 수 있다. 하지만 React Native에서는 같은 방식으로 동작하지 않는다.

React Native에서 어떤 UI를 터치 가능하게 만들려면, 해당 요소를 터치 이벤트를 인식할 수 있는 컴포넌트로 감싸야 한다.

### Pressable 컴포넌트

React Native는 터치 가능한 UI를 만들기 위한 여러 컴포넌트를 제공한다. 예전에는 `TouchableHighlight`, `TouchableNativeFeedback`, `TouchableOpacity` 같은 컴포넌트를 많이 사용했지만, 최신 React Native에서는 `Pressable` 사용이 권장된다.

`Pressable`은 내부에 있는 요소가 눌렸을 때 `onPress` prop에 전달된 함수를 실행한다.

```jsx
<Pressable onPress={props.onDeleteItem}>
  <View>
    <Text>{props.text}</Text>
  </View>
</Pressable>
```

이렇게 하면 `GoalItem` 내부의 `View`와 `Text` 영역을 터치했을 때 삭제 관련 함수가 호출될 수 있다.

### 삭제 핸들러의 위치

목록 데이터인 `courseGoals` 상태는 `App.js`에서 관리한다. 따라서 실제 삭제 로직도 `App.js`에 두는 것이 자연스럽다.

이번 단계에서는 먼저 `deleteGoalHandler` 함수를 만들고, 실제 삭제 대신 함수 호출 여부를 확인하기 위해 로그를 출력한다.

```jsx
function deleteGoalHandler() {
  console.log('DELETE');
}
```

### GoalItem에 함수 전달하기

`App.js`에서 정의한 `deleteGoalHandler`는 `GoalItem` 컴포넌트에 prop으로 전달한다. prop 이름은 직접 만든 컴포넌트에서 정하는 것이므로 자유롭게 붙일 수 있다.

```jsx
<GoalItem
  text={itemData.item.text}
  onDeleteItem={deleteGoalHandler}
/>
```

`GoalItem`에서는 전달받은 `onDeleteItem`을 `Pressable`의 `onPress`에 연결한다.

### 동작 확인

이 단계에서는 아이템을 눌러도 화면상 변화는 없다. 하지만 터미널에 `DELETE` 로그가 출력된다면 `Pressable`과 `onPress` 연결은 정상적으로 동작한 것이다.

실제 삭제 로직과 터치 시각 피드백은 이후 단계에서 추가한다.

## 정리

React Native에서 터치 가능한 목록 아이템을 만들려면 `Pressable`로 아이템 UI를 감싸고 `onPress`에 실행할 함수를 전달해야 한다. 이번 강의에서는 삭제 기능의 첫 단계로, 아이템을 눌렀을 때 부모 컴포넌트의 삭제 핸들러가 호출되도록 연결했다.
