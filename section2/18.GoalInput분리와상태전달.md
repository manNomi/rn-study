# 18. GoalInput 분리와 상태 전달

## 핵심 요약

- 입력 영역의 JSX와 관련 로직을 `GoalInput` 컴포넌트로 분리한다.
- `GoalInput`은 `TextInput`, `Button`, 입력 컨테이너 스타일을 담당한다.
- 사용자가 입력 중인 텍스트 상태는 `GoalInput` 내부로 옮긴다.
- 실제 목표 목록 상태인 `courseGoals`는 부모인 `App` 컴포넌트에 남긴다.
- 자식 컴포넌트에서 부모의 상태를 업데이트하려면 함수 prop을 전달한다.
- 입력값을 초기화하려면 `TextInput`의 `value` prop을 상태와 연결해야 한다.

## 학습 목표

- 입력 UI를 별도 컴포넌트로 분리하는 방법을 이해한다.
- 부모 컴포넌트의 함수를 자식 컴포넌트에 prop으로 전달하는 흐름을 익힌다.
- 자식 컴포넌트의 로컬 상태와 부모 컴포넌트의 목록 상태를 구분한다.
- `TextInput`을 상태와 연결해 제어 컴포넌트처럼 사용하는 방법을 이해한다.

## 주요 내용

### GoalInput 컴포넌트 만들기

입력 영역도 `App.js`에서 분리한다. `components/GoalInput.js` 파일을 만들고, 함수형 컴포넌트를 정의한 뒤 기본 export 한다.

```jsx
function GoalInput() {
  return (
    // 입력 영역 JSX
  );
}

export default GoalInput;
```

이 컴포넌트는 목표를 입력하는 `TextInput`과 목표를 추가하는 `Button`, 그리고 이 둘을 감싸는 `View`를 담당한다.

### 입력 관련 JSX 옮기기

`App.js`에 있던 입력 영역 JSX를 `GoalInput.js`로 옮긴다.

```jsx
<View style={styles.inputContainer}>
  <TextInput
    style={styles.textInput}
    placeholder="Your course goal!"
    onChangeText={goalInputHandler}
  />
  <Button title="Add Goal" onPress={addGoalHandler} />
</View>
```

이 JSX에서 사용하는 `View`, `TextInput`, `Button`, `StyleSheet`는 `GoalInput.js`에서 직접 import해야 한다.

```jsx
import { Button, StyleSheet, TextInput, View } from 'react-native';
```

### 입력 스타일 옮기기

입력 영역에만 필요한 스타일도 `GoalInput.js`로 옮긴다.

```jsx
const styles = StyleSheet.create({
  inputContainer: {
    flex: 1,
    flexDirection: 'row',
    justifyContent: 'space-between',
    alignItems: 'center',
    marginBottom: 24,
    borderBottomWidth: 1,
    borderBottomColor: '#cccccc',
  },
  textInput: {
    borderWidth: 1,
    borderColor: '#cccccc',
    width: '70%',
    marginRight: 8,
    padding: 8,
  },
});
```

이렇게 하면 `App.js`의 스타일 정의가 더 간결해지고, 입력 컴포넌트에 속한 스타일이 해당 파일 가까이에 위치한다.

### 입력 상태는 GoalInput으로 이동

사용자가 현재 입력 중인 텍스트는 `TextInput`과 직접 연결된 상태다. 따라서 이 상태는 `GoalInput` 컴포넌트 내부로 옮길 수 있다.

```jsx
import { useState } from 'react';
```

```jsx
function GoalInput(props) {
  const [enteredGoalText, setEnteredGoalText] = useState('');

  function goalInputHandler(enteredText) {
    setEnteredGoalText(enteredText);
  }

  return (
    // JSX
  );
}
```

반면 전체 목표 목록인 `courseGoals`는 `App` 컴포넌트에 남긴다. 여러 목표를 관리하고 `FlatList`에 전달하는 책임은 부모 컴포넌트가 계속 가진다.

### 부모 함수 전달하기

`GoalInput` 안에서 버튼을 누르면 부모인 `App`의 목표 목록 상태를 업데이트해야 한다.

이를 위해 `App`에서 `GoalInput`을 사용할 때 함수 prop을 전달한다.

```jsx
<GoalInput onAddGoal={addGoalHandler} />
```

`onAddGoal`은 직접 만든 prop 이름이다. 값으로는 `App`에 있는 `addGoalHandler` 함수를 전달한다.

### 자식에서 부모 함수 호출하기

`GoalInput` 내부에서는 버튼을 누를 때 실행할 로컬 함수를 만든다. 이 함수 안에서 props로 받은 `onAddGoal`을 직접 호출하면서 현재 입력값을 넘긴다.

```jsx
function addGoalHandler() {
  props.onAddGoal(enteredGoalText);
  setEnteredGoalText('');
}
```

그리고 버튼의 `onPress`에는 이 로컬 함수를 연결한다.

```jsx
<Button title="Add Goal" onPress={addGoalHandler} />
```

이 구조에서는 `GoalInput`에도 `addGoalHandler`가 있고 `App`에도 `addGoalHandler`가 있을 수 있다. 서로 다른 컴포넌트 스코프 안에 있으므로 이름 충돌은 발생하지 않는다.

### App의 addGoalHandler 수정

이제 `App`의 `addGoalHandler`는 입력 상태를 직접 읽는 것이 아니라, 자식 컴포넌트가 넘겨준 텍스트를 매개변수로 받는다.

```jsx
function addGoalHandler(enteredGoalText) {
  setCourseGoals((currentCourseGoals) => [
    ...currentCourseGoals,
    {
      text: enteredGoalText,
      id: Math.random().toString(),
    },
  ]);
}
```

즉 입력값은 `GoalInput`이 관리하고, 목표 목록에 추가하는 작업은 `App`이 담당한다.

### 입력값 초기화와 value prop

목표를 추가한 뒤 입력 상태를 빈 문자열로 초기화할 수 있다.

```jsx
setEnteredGoalText('');
```

하지만 상태만 초기화하면 실제 입력창에 보이는 텍스트가 사라지지 않을 수 있다. `TextInput`의 표시 값을 상태와 연결하지 않았기 때문이다.

이를 해결하려면 `TextInput`에 `value` prop을 추가한다.

```jsx
<TextInput
  style={styles.textInput}
  placeholder="Your course goal!"
  onChangeText={goalInputHandler}
  value={enteredGoalText}
/>
```

이제 `enteredGoalText`가 빈 문자열로 바뀌면 `TextInput`에 표시된 값도 함께 초기화된다.

### 구조 정리

최종 역할 분리는 다음과 같다.

```text
App
- courseGoals 상태 관리
- 목표 추가 함수 관리
- GoalInput에 onAddGoal 전달
- FlatList로 GoalItem 목록 렌더링

GoalInput
- enteredGoalText 상태 관리
- TextInput 입력 처리
- Button press 처리
- 입력값을 부모 함수에 전달

GoalItem
- 목표 항목 하나의 UI 렌더링
```

## 정리

이번 강의에서는 입력 영역을 `GoalInput` 컴포넌트로 분리했다. 입력값 상태와 입력 핸들러는 `GoalInput`이 관리하고, 목표 목록 상태는 `App`이 계속 관리한다.

자식 컴포넌트에서 부모 상태를 바꿔야 할 때는 부모가 함수를 prop으로 전달하고, 자식이 그 함수를 호출하면서 필요한 값을 인수로 넘긴다. 이 방식은 React Native만의 특수한 방법이 아니라 일반적인 React 패턴이다.
