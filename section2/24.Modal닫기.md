# 24. Modal 닫기

## 핵심 요약

- `Modal`을 닫으려면 모달 표시 여부를 관리하는 `modalIsVisible` 상태를 `false`로 바꿔야 한다.
- 모달을 여는 `startAddGoalHandler`와 짝을 이루는 `endAddGoalHandler`를 `App` 컴포넌트에 만든다.
- `Cancel` 버튼은 `GoalInput`에서 눌리지만, 실제 상태는 `App`에 있으므로 닫기 함수를 prop으로 전달한다.
- `Add Goal` 버튼을 눌렀을 때는 목표를 추가한 뒤 모달도 함께 닫는다.
- 부모 컴포넌트의 상태를 자식 컴포넌트의 버튼에서 바꾸려면 함수 prop을 사용한다.

## 학습 목표

- 부모 컴포넌트가 관리하는 상태를 자식 컴포넌트의 이벤트로 변경하는 방법을 익힌다.
- 모달을 여는 핸들러와 닫는 핸들러를 분리해 UI 흐름을 명확히 만든다.
- `Cancel` 버튼과 `Add Goal` 버튼이 각각 모달 닫기 동작을 수행하도록 연결하는 법을 배운다.
- React Native에서도 일반 React와 같은 prop 기반 이벤트 전달 패턴을 사용한다는 점을 이해한다.

## 주요 내용

### 모달 표시 상태의 위치

모달 표시 여부는 `App` 컴포넌트의 `modalIsVisible` 상태가 제어한다.

```jsx
const [modalIsVisible, setModalIsVisible] = useState(false);
```

따라서 모달을 닫는 작업도 결국 `App`에서 `setModalIsVisible(false)`를 실행해야 한다.

### 닫기 핸들러 만들기

이미 모달을 여는 함수는 있다.

```jsx
function startAddGoalHandler() {
  setModalIsVisible(true);
}
```

이와 짝을 이루는 닫기 함수를 추가한다.

```jsx
function endAddGoalHandler() {
  setModalIsVisible(false);
}
```

이 함수가 실행되면 `visible` prop에 전달되는 값이 `false`가 되고, `Modal`은 화면에서 사라진다.

### Add Goal 후 모달 닫기

사용자가 목표를 추가하면 목표 목록 상태를 갱신한 뒤 모달도 닫아야 한다. `addGoalHandler` 안에서 직접 `setModalIsVisible(false)`를 호출하거나, 만들어 둔 `endAddGoalHandler`를 호출하면 된다.

```jsx
function addGoalHandler(enteredGoalText) {
  setCourseGoals((currentCourseGoals) => [
    ...currentCourseGoals,
    { text: enteredGoalText, id: Math.random().toString() },
  ]);

  endAddGoalHandler();
}
```

이렇게 하면 `Add Goal` 버튼을 눌렀을 때 새 목표가 추가되고, 입력 모달도 자동으로 닫힌다.

### Cancel 버튼에 닫기 함수 연결하기

`Cancel` 버튼은 `GoalInput` 컴포넌트 안에 있다. 하지만 모달 표시 상태는 `App`이 가지고 있으므로, `App`에서 닫기 함수를 `GoalInput`으로 전달해야 한다.

```jsx
<GoalInput
  visible={modalIsVisible}
  onAddGoal={addGoalHandler}
  onCancel={endAddGoalHandler}
/>
```

`onCancel`이라는 이름은 직접 만든 커스텀 prop이다. 컴포넌트 사용자가 정한 이름이므로 `onClose`, `onCancelGoal`, `onCancelAddGoal`처럼 다른 이름을 써도 된다.

### GoalInput에서 onCancel 사용하기

`GoalInput`은 부모에게서 받은 `onCancel` prop을 `Cancel` 버튼의 `onPress`에 연결한다.

```jsx
<View style={styles.button}>
  <Button title="Cancel" onPress={props.onCancel} />
</View>
```

이제 사용자가 `Cancel` 버튼을 누르면 `App`에 정의된 `endAddGoalHandler`가 실행되고, 모달이 닫힌다.

### 전체 흐름

```text
Add New Goal 버튼 클릭
-> startAddGoalHandler 실행
-> modalIsVisible = true
-> Modal 열림

Cancel 버튼 클릭
-> props.onCancel 실행
-> endAddGoalHandler 실행
-> modalIsVisible = false
-> Modal 닫힘

Add Goal 버튼 클릭
-> GoalInput의 addGoalHandler 실행
-> props.onAddGoal(enteredGoalText) 호출
-> App의 addGoalHandler 실행
-> 목표 추가
-> endAddGoalHandler 실행
-> Modal 닫힘
```

### 일반 React 패턴과 동일함

이번 작업은 React Native만의 특별한 방식이 아니다. 부모가 상태를 가지고 있고, 자식에서 발생한 이벤트가 부모 상태를 바꿔야 할 때 함수를 prop으로 내려보내는 일반적인 React 패턴이다.

React Native에서는 버튼 이벤트 이름이 `onClick`이 아니라 `onPress`라는 점만 다르고, 상태와 prop을 다루는 방식은 React와 같다.

## 정리

이번 강의에서는 `Modal`을 닫는 흐름을 구현했다. `App`에 `endAddGoalHandler`를 만들고, 이 함수를 `GoalInput`의 `onCancel` prop으로 전달해 `Cancel` 버튼과 연결했다.

또한 목표를 추가한 뒤에도 모달이 닫히도록 `addGoalHandler`에서 닫기 핸들러를 함께 실행했다. 이로써 목표 입력 모달은 열기, 추가 후 닫기, 취소로 닫기까지 자연스럽게 동작하게 된다.
