# 26. 앱 전체 스타일 마무리

## 핵심 요약

- 기본 `Button`은 `color` prop으로 색상을 바꿀 수 있다.
- 플랫폼마다 기본 버튼 스타일이 다르게 보일 수 있으므로, iOS와 Android 양쪽에서 읽기 쉬운 색상을 고르는 것이 중요하다.
- `TextInput`은 `borderColor`, `backgroundColor`, `color`, `borderRadius`, `padding` 등을 통해 입력창답게 다듬을 수 있다.
- Expo 프로젝트에서는 `app.json`의 `backgroundColor`로 앱 전체 기본 배경색을 설정할 수 있다.
- 어두운 배경에서는 상태 표시줄이 잘 보이지 않을 수 있으므로 `expo-status-bar`의 `StatusBar` 컴포넌트로 표시 스타일을 조정한다.

## 학습 목표

- 기본 버튼의 색상을 조정하는 방법을 익힌다.
- 입력창의 배경, 테두리, 글자색, 모서리, 여백을 스타일링하는 법을 배운다.
- 앱 전체 배경색을 개별 화면이 아니라 Expo 설정에서 관리하는 방법을 이해한다.
- 어두운 배경에서 상태 표시줄을 읽기 쉽게 만드는 방법을 정리한다.
- React Native 앱의 마무리 스타일링 흐름을 전체적으로 복습한다.

## 주요 내용

### 버튼 색상 변경

React Native의 기본 `Button`은 자유로운 스타일링은 어렵지만 `color` prop으로 색상을 바꿀 수 있다.

```jsx
<Button title="Cancel" color="#f31282" onPress={props.onCancel} />
<Button title="Add Goal" color="#5e0acc" onPress={addGoalHandler} />
```

다만 같은 색상이라도 iOS와 Android에서 다르게 보일 수 있다. 강의에서는 iOS에서도 글자가 잘 보이도록 `Add Goal` 버튼 색상을 더 밝은 보라색으로 조정한다.

```jsx
<Button title="Add Goal" color="#b180f0" onPress={addGoalHandler} />
```

플랫폼별로 더 정교하게 다르게 꾸미는 방법은 나중에 다룬다. 지금은 같은 코드가 두 플랫폼에서 모두 무난하게 보이도록 색상을 선택한다.

### 입력창 스타일 다듬기

모달 배경이 어두워졌기 때문에 입력창도 함께 조정해야 한다. `textInput` 스타일에서 테두리 색상, 배경색, 입력 텍스트 색상, 모서리, 안쪽 여백을 변경한다.

```jsx
textInput: {
  borderWidth: 1,
  borderColor: '#e4d0ff',
  backgroundColor: '#e4d0ff',
  color: '#120438',
  borderRadius: 6,
  width: '100%',
  padding: 16,
},
```

`borderColor`와 `backgroundColor`를 같은 밝은 보라색 계열로 맞추면 어두운 모달 배경 위에서 입력창이 잘 드러난다. `color`는 사용자가 입력하는 텍스트 색상이다.

### 버튼 순서 조정

강의에서는 `Add Goal` 버튼이 오른쪽에 있는 편이 더 직관적이라고 보고 버튼 순서를 바꾼다.

```jsx
<View style={styles.buttonContainer}>
  <View style={styles.button}>
    <Button title="Cancel" color="#f31282" onPress={props.onCancel} />
  </View>
  <View style={styles.button}>
    <Button title="Add Goal" color="#b180f0" onPress={addGoalHandler} />
  </View>
</View>
```

이런 배치는 정답이 정해져 있다기보다 앱의 흐름과 개인 선호에 따라 달라질 수 있다.

### 메인 화면 배경색 변경

모달뿐만 아니라 메인 화면도 흰색 배경 대신 어두운 보라색 계열로 바꾼다. 단일 화면만 있다면 `App.js`의 루트 `View`에 `backgroundColor`를 줄 수도 있다.

```jsx
appContainer: {
  flex: 1,
  paddingTop: 50,
  paddingHorizontal: 16,
  backgroundColor: '#1e085a',
},
```

하지만 화면이 여러 개라면 모든 화면마다 배경색을 반복해서 지정하는 것은 번거롭다.

### app.json으로 전체 배경색 설정

Expo 프로젝트에서는 `app.json`에 `backgroundColor`를 설정해 앱의 기본 배경색을 관리할 수 있다.

```json
{
  "expo": {
    "backgroundColor": "#1e085a"
  }
}
```

이 설정은 Expo가 앱의 기본 배경에 적용한다. 나중에 내비게이션을 추가해 화면이 여러 개가 되면 각 화면마다 배경색을 반복 설정하지 않아도 되어 더 유용하다.

모달은 별도의 오버레이 성격이 있으므로, 모달 내부 배경은 여전히 모달 안의 루트 `View`에서 따로 조정한다.

### Add New Goal 버튼 색상

메인 화면 배경이 어두워지면 기존 `Add New Goal` 버튼이 잘 보이지 않을 수 있다. 그래서 `App.js`에서 버튼 색상도 더 밝은 보라색으로 바꾼다.

```jsx
<Button
  title="Add New Goal"
  color="#b180f0"
  onPress={startAddGoalHandler}
/>
```

배경색을 바꿀 때는 주변 UI 요소도 함께 확인해야 한다. 특정 요소만 수정하면 다른 요소가 갑자기 덜 읽히는 문제가 생길 수 있다.

### StatusBar로 상태 표시줄 조정

어두운 배경을 적용하면 상태 표시줄의 시간, 배터리, 네트워크 정보가 잘 보이지 않을 수 있다. Expo 프로젝트에서는 `expo-status-bar` 패키지의 `StatusBar` 컴포넌트를 사용할 수 있다.

```jsx
import { StatusBar } from 'expo-status-bar';
```

`App` 컴포넌트에서 루트 `View` 옆에 `StatusBar`를 추가해야 하므로, JSX 루트는 fragment로 감싼다.

```jsx
return (
  <>
    <StatusBar style="light" />
    <View style={styles.appContainer}>
      {/* 앱 UI */}
    </View>
  </>
);
```

`style="light"`를 설정하면 상태 표시줄 텍스트가 밝게 표시되어 어두운 배경 위에서도 잘 보인다.

### StatusBar 스타일 옵션

`StatusBar`의 `style` prop은 상태 표시줄의 표시 방식을 제어한다.

```text
auto: 플랫폼과 환경에 맞춰 자동 결정
light: 밝은 색상으로 표시
dark: 어두운 색상으로 표시
```

이번 앱처럼 전체 배경이 어두운 경우에는 `light`가 더 적합하다.

## 정리

이번 강의에서는 목표 관리 앱의 전체적인 모양을 마무리했다. 버튼 색상, 입력창 스타일, 모달과 메인 화면 배경색을 조정해 앱의 분위기를 통일했다.

또한 Expo의 `app.json`을 통해 앱 전체 기본 배경색을 설정하는 방법을 배웠고, `StatusBar` 컴포넌트로 어두운 배경에서도 상태 표시줄이 잘 보이도록 수정했다. 이로써 섹션에서 만든 목표 관리 앱은 기능뿐 아니라 기본적인 시각적 완성도까지 갖추게 되었다.
