# 4. 스타일링 기본

## 핵심 요약

- React Native는 CSS 파일이나 CSS 언어를 직접 사용하지 않는다.
- 스타일은 JavaScript 객체로 작성하고 컴포넌트의 `style` prop에 전달한다.
- `View`, `Text` 같은 일부 코어 컴포넌트는 `style` prop을 지원한다.
- React Native 스타일 속성은 CSS의 영향을 받았지만, 이름과 지원 범위가 완전히 같지는 않다.
- 인라인 스타일도 가능하지만, 일반적으로는 `StyleSheet.create`로 스타일을 분리해 사용하는 편이 좋다.

## 학습 목표

- React Native에서 스타일을 적용하는 기본 방식을 이해한다.
- CSS와 React Native 스타일 속성의 차이를 구분한다.
- 인라인 스타일과 `StyleSheet` 방식의 장단점을 이해한다.
- `margin`, `padding`, `borderWidth`, `borderColor` 같은 기본 스타일 속성 사용법을 익힌다.

## 주요 내용

### React Native에는 CSS가 없다

React Native 앱은 브라우저에서 실행되는 웹 앱이 아니므로 CSS 파일을 사용하지 않는다. 대신 JavaScript 객체를 통해 스타일을 정의한다.

스타일 속성은 CSS와 비슷한 이름을 가진 경우가 많지만, CSS 전체 기능이 지원되는 것은 아니다. 또한 속성 이름이 CSS와 정확히 일치하지 않는 경우도 있다.

### 인라인 스타일

컴포넌트에 직접 스타일을 적용하려면 `style` prop에 JavaScript 객체를 전달한다.

```jsx
<Text style={{ margin: 16 }}>
  Hello World!
</Text>
```

`margin`처럼 숫자를 값으로 받는 스타일 속성은 기기 화면에 맞는 픽셀 값으로 처리된다. React Native는 디바이스 픽셀 밀도에 맞춰 값을 조정한다.

### CSS와 다른 스타일 속성

React Native에서는 CSS처럼 `border` 속성을 한 번에 지정할 수 없다.

```jsx
<Text style={{ border: '2px solid red' }}>
  Hello World!
</Text>
```

이런 방식은 React Native에서 유효하지 않다. 대신 테두리 너비와 색상을 별도의 속성으로 나누어 설정해야 한다.

```jsx
<Text
  style={{
    borderWidth: 2,
    borderColor: 'red',
  }}
>
  Hello World!
</Text>
```

`borderWidth`는 숫자를 값으로 받고, `borderColor`는 문자열을 값으로 받는다. 색상은 `red`, `blue`, `black` 같은 이름이나 헥스 코드로 지정할 수 있다.

### margin과 padding

`margin`은 컴포넌트 바깥쪽 여백이고, `padding`은 컴포넌트 안쪽 여백이다. 웹 CSS와 개념은 비슷하다.

```jsx
<Text
  style={{
    margin: 16,
    padding: 16,
    borderWidth: 2,
    borderColor: 'red',
  }}
>
  Hello World!
</Text>
```

위 스타일은 텍스트 주변에 바깥쪽 여백과 안쪽 여백을 만들고, 2픽셀 너비의 빨간 테두리를 추가한다.

### 인라인 스타일의 한계

인라인 스타일은 빠르게 테스트하기에는 편리하지만, 여러 컴포넌트에 같은 스타일을 적용하려면 같은 객체를 반복해서 복사해야 한다.

```jsx
<Text style={{ margin: 16, padding: 16 }}>
  First text
</Text>

<Text style={{ margin: 16, padding: 16 }}>
  Second text
</Text>
```

이 방식은 스타일을 수정할 때 여러 곳을 함께 변경해야 하므로 유지보수가 불편하다.

### StyleSheet 객체 사용

보통은 `StyleSheet.create`를 사용해 스타일을 별도로 정의한다.

```jsx
const styles = StyleSheet.create({
  dummyText: {
    margin: 16,
    padding: 16,
    borderWidth: 2,
    borderColor: 'blue',
  },
});
```

정의한 스타일은 JSX에서 `styles.dummyText`처럼 참조해 사용할 수 있다.

```jsx
<Text style={styles.dummyText}>
  Hello World!
</Text>
```

스타일을 한 곳에서 관리하므로 같은 스타일을 여러 컴포넌트에 쉽게 재사용할 수 있고, 색상이나 여백을 바꿀 때도 한 곳만 수정하면 된다.

### StyleSheet를 사용하는 이유

`StyleSheet.create`를 사용하면 JSX 코드와 스타일 코드를 분리할 수 있어 가독성이 좋아진다. 또한 VS Code 자동 완성이나 잘못된 스타일 값에 대한 경고를 통해 개발 경험도 좋아진다.

React Native는 `StyleSheet` 객체를 인식하고, 스타일 정의와 관리에 대한 내부 최적화도 수행할 수 있다.

## 정리

React Native의 스타일링은 CSS가 아니라 JavaScript 객체 기반으로 이루어진다. 인라인 스타일도 가능하지만, 재사용성과 가독성을 위해 `StyleSheet.create`로 스타일을 분리하는 방식이 더 일반적이다.

스타일 속성은 CSS와 비슷하지만 완전히 같지는 않으므로, 실제로 사용할 때는 React Native가 지원하는 속성 이름과 값 형식을 확인해야 한다.
