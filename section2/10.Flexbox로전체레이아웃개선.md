# 10. Flexbox로 전체 레이아웃 개선

## 핵심 요약

- `paddingTop`과 `paddingHorizontal`을 사용해 앱 전체 여백을 더 자연스럽게 조정한다.
- `TextInput`의 너비를 줄여 버튼이 화면 밖으로 밀려나지 않게 한다.
- `Button`은 `style` prop을 지원하지 않으므로, 부모 컨테이너의 `alignItems`로 버튼이 늘어나는 문제를 해결한다.
- 입력 영역에는 아래쪽 여백과 구분선을 추가해 목록 영역과 시각적으로 분리한다.
- 외부 컨테이너에 `flex: 1`을 설정해야 내부 영역들이 화면 높이를 기준으로 공간을 나눌 수 있다.
- 입력 영역과 목록 영역은 각각 `flex` 값을 통해 원하는 비율로 배치한다.

## 학습 목표

- 전체 화면 컨테이너와 내부 컨테이너의 `flex` 관계를 이해한다.
- 부모 컨테이너의 크기가 내부 `flex` 배분에 왜 중요한지 파악한다.
- `alignItems`로 자식 요소가 교차축 방향으로 늘어나는 문제를 해결하는 방법을 익힌다.
- 입력 영역과 목록 영역의 공간 비율을 Flexbox로 조정하는 방법을 배운다.

## 주요 내용

### 전체 컨테이너 여백 조정

먼저 앱 전체를 감싸는 `appContainer`의 여백을 조정한다. 위쪽은 상태 표시줄과 겹치지 않도록 넉넉하게 두고, 좌우는 너무 넓지 않게 설정한다.

```jsx
const styles = StyleSheet.create({
  appContainer: {
    paddingTop: 50,
    paddingHorizontal: 16,
  },
});
```

`paddingHorizontal`은 왼쪽과 오른쪽 안쪽 여백을 동시에 설정하는 React Native 스타일 속성이다.

### TextInput 너비 조정

입력창의 너비가 너무 넓으면 버튼이 화면 밖으로 밀려날 수 있다. 이전에 `width: '80%'`로 설정했다면, 이를 `70%` 정도로 줄여 입력창과 버튼이 함께 들어갈 공간을 만든다.

```jsx
const styles = StyleSheet.create({
  textInput: {
    width: '70%',
  },
});
```

퍼센트 값은 문자열로 작성해야 하며, 부모 컨테이너의 너비를 기준으로 계산된다.

### Button은 직접 스타일링하기 어렵다

React Native의 기본 `Button` 컴포넌트는 `style` prop을 지원하지 않는다. 따라서 버튼 자체에 스타일을 직접 적용해 높이나 정렬을 수정할 수 없다.

이 문제는 부모 컨테이너인 `inputContainer`의 정렬 방식을 바꿔 해결한다.

기본 `alignItems` 값은 `stretch`이므로 버튼이 입력창 높이에 맞춰 늘어날 수 있다. 이를 `center`로 바꾸면 자식 요소들이 교차축 중앙에 배치되고, 버튼이 불필요하게 늘어나지 않는다.

```jsx
const styles = StyleSheet.create({
  inputContainer: {
    flexDirection: 'row',
    justifyContent: 'space-between',
    alignItems: 'center',
  },
});
```

### 입력 영역과 목록 영역 분리

입력 영역과 목표 목록 영역 사이에 간격을 만들고, 구분선을 추가해 시각적으로 나눌 수 있다.

```jsx
const styles = StyleSheet.create({
  inputContainer: {
    marginBottom: 24,
    borderBottomWidth: 1,
    borderBottomColor: '#cccccc',
  },
});
```

처음에는 `paddingBottom`을 사용할 수도 있지만, 이후 레이아웃 조정 과정에서는 `marginBottom`이 더 자연스러운 간격을 만들 수 있다.

### 내부 영역에 flex 적용하기

입력 영역과 목록 영역이 화면 높이를 나눠 가지도록 하려면 각 영역에 `flex` 값을 설정한다.

입력 영역은 상대적으로 작은 공간을 차지하고, 목록 영역은 더 넓은 공간을 차지해야 한다.

```jsx
const styles = StyleSheet.create({
  inputContainer: {
    flex: 1,
  },
  goalsContainer: {
    flex: 4,
  },
});
```

위 설정은 두 영역의 사용 가능한 공간을 `1 + 4 = 5`로 나누고, 입력 영역은 5분의 1, 목록 영역은 5분의 4를 차지하게 한다.

목록 영역에는 새 스타일을 연결한다.

```jsx
<View style={styles.goalsContainer}>
  <Text>List of goals...</Text>
</View>
```

### 외부 컨테이너의 flex가 필요한 이유

내부 컨테이너에 `flex`를 설정했는데 화면이 이상하게 보인다면, 외부 컨테이너가 충분한 높이를 가지고 있지 않기 때문이다.

기본적으로 `View`는 자식 콘텐츠가 필요한 최소 공간만 차지한다. 따라서 내부 `View`들이 화면 높이를 비율로 나누려면, 그 부모인 외부 컨테이너가 먼저 사용 가능한 전체 높이를 차지해야 한다.

이를 위해 `appContainer`에 `flex: 1`을 추가한다.

```jsx
const styles = StyleSheet.create({
  appContainer: {
    flex: 1,
    paddingTop: 50,
    paddingHorizontal: 16,
  },
});
```

이제 외부 컨테이너가 화면 전체 높이를 차지하고, 내부의 입력 영역과 목록 영역이 그 공간을 `flex` 값에 따라 나눠 갖는다.

### 최종 레이아웃 방향

이번 단계의 목표는 다음과 같은 기본 구조를 만드는 것이다.

```text
상단 영역: 목표 입력창 + Add Goal 버튼
하단 영역: 목표 목록
```

아직 최종 디자인은 아니지만, 전체 화면 안에서 각 영역이 적절한 비율로 배치되는 기본 레이아웃을 만들었다.

## 정리

이번 강의에서는 Flexbox를 사용해 목표 관리 앱의 전체 레이아웃을 개선했다. 외부 컨테이너에는 `flex: 1`을 적용해 화면 전체 높이를 차지하게 하고, 내부 입력 영역과 목록 영역에는 각각 `flex` 값을 주어 공간 비율을 조정했다.

또한 `Button`이 직접 스타일링되지 않는 한계를 부모 컨테이너의 `alignItems`로 해결하고, 입력 영역과 목록 영역 사이에 여백과 구분선을 추가해 화면 구조를 더 명확하게 만들었다.
