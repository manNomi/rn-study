# 14. ScrollView로 목록 스크롤

## 핵심 요약

- React Native의 기본 `View`는 콘텐츠가 화면을 넘어가도 자동으로 스크롤되지 않는다.
- 목록이나 긴 콘텐츠를 스크롤 가능하게 만들려면 `ScrollView`를 명시적으로 사용해야 한다.
- `ScrollView`는 `react-native`에서 import해서 사용한다.
- 스크롤 가능한 영역의 높이는 `ScrollView` 자체보다 바깥 부모 컨테이너가 제어하는 경우가 많다.
- `ScrollView`는 플랫폼별 옵션을 포함해 다양한 설정 prop을 제공한다.

## 학습 목표

- React Native에서 스크롤이 자동으로 생기지 않는 이유를 이해한다.
- `ScrollView`로 목록을 스크롤 가능하게 만드는 방법을 익힌다.
- `ScrollView`와 부모 `View`의 역할 차이를 이해한다.
- `alwaysBounceVertical` 같은 설정 prop으로 스크롤 동작을 조정할 수 있음을 안다.

## 주요 내용

### 기본 View는 스크롤되지 않는다

목표를 계속 추가하면 목록이 화면의 사용 가능한 공간을 넘어갈 수 있다. 웹에서는 콘텐츠가 브라우저 화면을 넘으면 기본적으로 스크롤이 생기지만, React Native에서는 그렇지 않다.

모바일 앱에서는 스크롤 가능한 영역을 명시적으로 만들어야 한다. 일반 `View`는 콘텐츠를 담을 수는 있지만, 스크롤 동작을 자동으로 제공하지 않는다.

### ScrollView 사용하기

React Native는 스크롤 가능한 컨테이너를 만들기 위해 `ScrollView` 컴포넌트를 제공한다.

먼저 `react-native`에서 import한다.

```jsx
import { ScrollView } from 'react-native';
```

그리고 목표 목록을 감싸는 영역에 `ScrollView`를 사용한다.

```jsx
<ScrollView>
  {courseGoals.map((goal) => (
    <View key={goal} style={styles.goalItem}>
      <Text style={styles.goalText}>{goal}</Text>
    </View>
  ))}
</ScrollView>
```

이렇게 하면 목표 목록이 사용 가능한 공간을 넘어갈 때 스크롤할 수 있다.

### 부모 View와 ScrollView의 역할

`ScrollView`를 기존 목록 컨테이너 `View`와 그대로 바꿔버리면 레이아웃 비율이 예상과 달라질 수 있다.

스크롤 가능한 동작은 `ScrollView`가 제공하지만, 스크롤 가능한 영역의 크기는 보통 바깥 컨테이너가 제어한다.

따라서 바깥에는 일반 `View`를 두고, 그 안에 `ScrollView`를 넣는 구조가 더 적절하다.

```jsx
<View style={styles.goalsContainer}>
  <ScrollView>
    {courseGoals.map((goal) => (
      <View key={goal} style={styles.goalItem}>
        <Text style={styles.goalText}>{goal}</Text>
      </View>
    ))}
  </ScrollView>
</View>
```

여기서 `goalsContainer`는 화면에서 목록 영역이 차지할 높이를 제어하고, `ScrollView`는 그 영역 안에서 항목을 스크롤 가능하게 만든다.

### 스크롤이 필요한 경우에만 동작한다

목록 항목이 적어서 콘텐츠가 사용 가능한 높이를 넘지 않으면 실제로 스크롤할 필요가 없다. 이 경우 손으로 움직여도 다시 제자리로 돌아올 수 있다.

항목이 많아져서 목록이 영역을 넘어서면 그때 스크롤이 가능해진다.

### ScrollView 설정 prop

`ScrollView`는 다양한 설정 prop을 제공한다. 일부는 iOS 전용이고, 일부는 Android 전용이며, 양쪽 플랫폼 모두에서 동작하는 prop도 있다.

예를 들어 iOS에서 콘텐츠가 부족해도 위아래로 튀어 오르는 효과가 보일 수 있다. 이 동작은 `alwaysBounceVertical` prop으로 조정할 수 있다.

```jsx
<ScrollView alwaysBounceVertical={false}>
  {/* 목표 목록 */}
</ScrollView>
```

`alwaysBounceVertical={false}`를 설정하면 콘텐츠가 충분하지 않을 때 불필요한 세로 바운스 효과를 줄일 수 있다.

### 공식 문서 활용

`ScrollView`는 설정할 수 있는 prop이 많다. 앱마다 필요한 스크롤 동작이 다르므로, 세부 설정이 필요할 때는 React Native 공식 문서에서 `ScrollView`가 지원하는 prop을 확인하는 것이 좋다.

플랫폼별로 동작이 다를 수 있으므로 iOS와 Android에서 모두 확인하는 습관도 중요하다.

## 정리

React Native에서는 일반 `View`가 자동으로 스크롤을 제공하지 않는다. 긴 목록이나 화면을 넘어가는 콘텐츠를 다룰 때는 `ScrollView`를 사용해야 한다.

목록 영역의 크기는 부모 `View`가 담당하고, 그 안의 `ScrollView`가 실제 스크롤 동작을 담당하도록 구성하면 기존 레이아웃 비율을 유지하면서 스크롤 가능한 목록을 만들 수 있다.
