# 섹션 2. 프론트엔드 개발자 관점 정리

## 한 줄 요약

React Native의 핵심 컴포넌트, 스타일링, 이벤트, 상태, 목록 렌더링을 사용해 첫 모바일 앱을 만들면서 웹 React와 네이티브 앱 개발의 차이를 몸에 익히는 섹션이다.

## 이미 알고 있다고 보는 것

- React 함수 컴포넌트
- JSX
- props 전달
- `useState` 기반 상태 관리
- 이벤트 핸들러 연결
- 배열 상태 업데이트
- `map()`을 활용한 목록 렌더링
- 컴포넌트 분리
- CSS Flexbox의 기본 개념

## 새로 잡아야 하는 관점

섹션 2의 핵심은 "React 문법"이 아니라 "React Native 환경에서 UI를 구성하는 방법"이다.

웹에서는 HTML 요소와 CSS로 UI를 만든다.

```jsx
<div>
  <h1>Goals</h1>
  <input />
  <button>Add</button>
</div>
```

React Native에서는 HTML 요소를 쓸 수 없다.

```jsx
<View>
  <Text>Goals</Text>
  <TextInput />
  <Button title="Add" />
</View>
```

프론트엔드 개발자에게 가장 중요한 전환은 다음이다.

```text
HTML 요소
-> React Native 핵심 컴포넌트

CSS 파일
-> JavaScript 스타일 객체

브라우저 클릭
-> 모바일 터치 / press

DOM 스크롤
-> ScrollView / FlatList
```

## 핵심 컴포넌트

React Native UI는 내장 핵심 컴포넌트를 조합해 만든다.

| 웹에서 익숙한 개념 | React Native 컴포넌트 | 설명 |
| --- | --- | --- |
| `div` | `View` | 레이아웃과 컨테이너 |
| 텍스트 태그 | `Text` | 텍스트 출력 |
| `input` | `TextInput` | 텍스트 입력 |
| `button` | `Button`, `Pressable` | 버튼과 터치 영역 |
| `img` | `Image` | 이미지 표시 |
| 스크롤 영역 | `ScrollView` | 단순 스크롤 컨텐츠 |
| 긴 목록 | `FlatList` | 성능 최적화 목록 |
| 팝업 / 오버레이 | `Modal` | 모달 화면 |

특히 중요한 규칙은 텍스트는 반드시 `Text` 안에 있어야 한다는 점이다.

```jsx
// 잘못된 방식
<View>Hello</View>

// 올바른 방식
<View>
  <Text>Hello</Text>
</View>
```

웹의 `div` 안에는 문자열을 바로 넣을 수 있지만, React Native의 `View`는 텍스트를 직접 렌더링하지 못한다.

## 스타일링

React Native에는 CSS 파일이 없다. 스타일은 JavaScript 객체로 작성한다.

```jsx
const styles = StyleSheet.create({
  container: {
    flex: 1,
    padding: 16,
    backgroundColor: '#1e085a',
  },
});
```

그리고 컴포넌트의 `style` prop에 연결한다.

```jsx
<View style={styles.container}>
  <Text style={styles.title}>Hello</Text>
</View>
```

CSS와 비슷하지만 다음 차이를 반드시 기억해야 한다.

- `background-color`가 아니라 `backgroundColor`처럼 카멜 케이스를 쓴다.
- 숫자 값은 보통 디바이스 밀도를 고려한 크기로 해석된다.
- 스타일 상속이 없다.
- 부모 `View`의 글자색이 자식 `Text`에 자동 상속되지 않는다.
- 모든 컴포넌트가 모든 스타일 prop을 지원하지는 않는다.
- 같은 스타일도 iOS와 Android에서 다르게 보일 수 있다.

## Flexbox 레이아웃

React Native 레이아웃의 중심은 Flexbox다.

웹과 다른 기본값이 있다.

```text
웹 CSS Flexbox 기본 방향: row
React Native Flexbox 기본 방향: column
```

React Native의 `View`는 기본적으로 Flexbox 컨테이너처럼 동작한다.

자주 쓰는 속성은 다음과 같다.

```jsx
container: {
  flex: 1,
  flexDirection: 'row',
  justifyContent: 'center',
  alignItems: 'center',
}
```

프론트엔드 개발자라면 CSS Flexbox 개념은 그대로 가져가되, 기본 방향이 세로라는 점과 모바일 화면 크기 기준으로 레이아웃을 잡는 감각에 익숙해져야 한다.

## 이벤트와 상태

React Native도 React 앱이므로 상태 관리는 익숙한 방식 그대로다.

```jsx
const [enteredGoalText, setEnteredGoalText] = useState('');
```

입력 이벤트는 웹의 `onChange`가 아니라 `onChangeText`를 사용한다.

```jsx
<TextInput
  onChangeText={setEnteredGoalText}
  value={enteredGoalText}
/>
```

버튼 이벤트는 `onClick`이 아니라 `onPress`다.

```jsx
<Button title="Add Goal" onPress={addGoalHandler} />
```

이름은 다르지만 패턴은 웹 React와 같다.

```text
사용자 이벤트 발생
-> 핸들러 실행
-> 상태 업데이트
-> UI 리렌더링
```

## 목록 렌더링

처음에는 웹 React처럼 `map()`으로 목표 목록을 렌더링할 수 있다.

하지만 모바일에서는 목록이 길어질 때 성능을 더 신경 써야 한다.

`ScrollView`는 내부 항목을 모두 렌더링한다.

```text
ScrollView
-> 단순하고 길이가 제한된 컨텐츠에 적합
-> 긴 동적 목록에는 비효율적일 수 있음
```

긴 목록이나 동적으로 늘어나는 목록은 `FlatList`를 사용한다.

```jsx
<FlatList
  data={courseGoals}
  renderItem={(itemData) => (
    <GoalItem text={itemData.item.text} />
  )}
  keyExtractor={(item) => item.id}
/>
```

`FlatList`는 화면에 필요한 항목 중심으로 렌더링하기 때문에 성능 면에서 더 적합하다.

## 컴포넌트 분리와 props

앱이 커지면 웹 React와 똑같이 컴포넌트를 분리한다.

이번 섹션에서는 다음처럼 나눴다.

```text
App
-> 전체 상태와 목표 목록 관리

GoalInput
-> 목표 입력 UI와 입력 상태 관리

GoalItem
-> 개별 목표 항목 렌더링과 삭제 이벤트 연결
```

부모에서 자식으로 데이터와 함수를 전달하는 패턴도 동일하다.

```jsx
<GoalInput
  visible={modalIsVisible}
  onAddGoal={addGoalHandler}
  onCancel={endAddGoalHandler}
/>
```

자식 컴포넌트는 받은 함수를 이벤트에 연결한다.

```jsx
<Button title="Cancel" onPress={props.onCancel} />
```

## 모바일 앱다운 요소

섹션 후반에는 웹 프론트엔드보다 모바일 앱 쪽에 가까운 요소들을 다뤘다.

### Pressable

`Pressable`은 특정 영역을 터치 가능하게 만들고, 누르는 상태에 따라 시각적 피드백을 줄 수 있다.

```jsx
<Pressable onPress={deleteGoalHandler}>
  <Text>Learn React Native</Text>
</Pressable>
```

Android에서는 `android_ripple`을 사용해 물결 효과를 줄 수 있고, iOS에서는 `style` prop에 함수를 전달해 pressed 상태에 따라 스타일을 바꿀 수 있다.

### Modal

`Modal`은 화면 위에 입력 UI 같은 오버레이를 띄울 때 사용한다.

```jsx
<Modal visible={props.visible} animationType="slide">
  <View>{/* 입력 UI */}</View>
</Modal>
```

모달 표시 여부는 부모의 state로 관리하고, 열기와 닫기 함수는 props로 연결한다.

### Image

로컬 이미지는 문자열 경로가 아니라 `require()`로 불러온다.

```jsx
<Image
  style={styles.image}
  source={require('../assets/images/goal.png')}
/>
```

### StatusBar

어두운 배경에서는 상태 표시줄이 안 보일 수 있으므로 Expo의 `StatusBar`를 사용해 표시 스타일을 조정한다.

```jsx
<StatusBar style="light" />
```

