# 11. 이벤트와 입력 상태

## 핵심 요약

- React Native 앱도 웹 React 앱처럼 이벤트 리스너와 이벤트 핸들러로 상호작용을 처리한다.
- `TextInput`의 입력 변화는 `onChangeText` prop으로 처리할 수 있다.
- `Button`의 누름 이벤트는 웹의 `onClick`이 아니라 `onPress` prop으로 처리한다.
- 입력값처럼 컴포넌트 안에서 유지해야 하는 데이터는 React의 `useState` 훅으로 관리한다.
- React Native는 렌더링 대상만 다를 뿐, 이벤트 처리와 상태 관리 같은 React 핵심 개념은 웹 React와 동일하게 동작한다.

## 학습 목표

- React Native에서 입력 이벤트와 버튼 이벤트를 연결하는 방법을 이해한다.
- 이벤트 핸들러 함수를 JSX에 전달할 때 함수 호출이 아니라 함수 참조를 넘겨야 하는 이유를 안다.
- `useState`로 입력값을 저장하고 다른 핸들러에서 사용하는 흐름을 익힌다.

## 주요 내용

### 앱에 상호작용 추가하기

현재 앱에는 텍스트 입력란과 버튼이 있지만, 아직 이벤트를 처리하지 않기 때문에 아무 동작도 하지 않는다.

React Native에서도 웹 React와 마찬가지로 사용자의 입력이나 버튼 누름에 반응하려면 이벤트 리스너를 연결하고, 해당 이벤트가 발생했을 때 실행할 핸들러 함수를 만들어야 한다.

이번 단계에서는 두 가지 이벤트를 처리한다.

```text
TextInput의 텍스트 입력
Button의 누름 이벤트
```

### 입력 이벤트 처리

사용자가 입력한 값을 가져오기 위해 `goalInputHandler` 함수를 만든다.

```jsx
function goalInputHandler(enteredText) {
  console.log(enteredText);
}
```

`TextInput`에는 `onChangeText` prop을 사용해 이 함수를 연결한다.

```jsx
<TextInput
  style={styles.textInput}
  placeholder="Your course goal!"
  onChangeText={goalInputHandler}
/>
```

`onChangeText`는 React Native가 제공하는 특수 prop이다. 텍스트가 변경될 때마다 전달한 함수를 실행하고, 현재 입력값을 매개변수로 넘겨준다.

### 함수 참조를 전달하기

이벤트 핸들러를 연결할 때는 함수를 직접 실행하면 안 된다.

```jsx
onChangeText={goalInputHandler}
```

위처럼 함수 이름만 전달해야 한다.

다음처럼 괄호를 붙이면 컴포넌트가 렌더링되기 전에 함수가 즉시 실행된다.

```jsx
onChangeText={goalInputHandler()}
```

우리가 원하는 것은 텍스트가 변경될 때 React Native가 나중에 함수를 호출하는 것이므로, 함수 참조를 전달해야 한다.

### 버튼 이벤트 처리

웹에서는 버튼 클릭을 처리할 때 `onClick`을 사용하지만, React Native의 `Button`에는 `onClick`이 없다.

모바일 앱에서는 버튼을 클릭한다기보다 탭하거나 누르는 개념이므로, React Native에서는 `onPress`를 사용한다.

```jsx
function addGoalHandler() {
  console.log('Add goal');
}
```

```jsx
<Button
  title="Add Goal"
  onPress={addGoalHandler}
/>
```

이렇게 하면 사용자가 버튼을 누를 때마다 `addGoalHandler`가 실행된다.

### useState로 입력값 관리

입력값은 `goalInputHandler`에서 받을 수 있지만, 실제로 목표를 추가하는 시점은 버튼을 누르는 순간이다. 따라서 입력값을 컴포넌트 상태로 저장해두어야 `addGoalHandler`에서도 사용할 수 있다.

상태 관리는 React 핵심 라이브러리의 `useState` 훅을 사용한다.

```jsx
import { useState } from 'react';
```

`App` 컴포넌트 안에서 입력 상태를 만든다.

```jsx
const [enteredGoalText, setEnteredGoalText] = useState('');
```

초기값은 빈 문자열이다. 사용자가 텍스트를 입력할 때마다 상태를 갱신한다.

```jsx
function goalInputHandler(enteredText) {
  setEnteredGoalText(enteredText);
}
```

이제 버튼 핸들러에서 최신 입력값을 사용할 수 있다.

```jsx
function addGoalHandler() {
  console.log(enteredGoalText);
}
```

### 동작 확인

입력창에 텍스트를 입력하면 `goalInputHandler`가 실행되고, 입력값이 상태에 저장된다.

이후 `Add Goal` 버튼을 누르면 `addGoalHandler`가 실행되며, 현재 상태에 저장된 입력값을 확인할 수 있다.

아직 화면의 목표 목록을 업데이트하지는 않지만, 입력값을 받아 상태로 관리하고 버튼 이벤트에서 사용할 수 있는 기본 흐름을 완성했다.

## 정리

React Native의 이벤트 처리와 상태 관리는 웹 React와 같은 방식으로 동작한다. `TextInput`은 `onChangeText`로 입력 변화를 처리하고, `Button`은 `onPress`로 누름 이벤트를 처리한다.

입력값은 `useState`로 저장해두면 다른 이벤트 핸들러에서도 사용할 수 있다. 다음 단계에서는 이 상태를 활용해 실제 목표 목록을 관리하고 화면에 출력한다.
