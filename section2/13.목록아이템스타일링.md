# 13. 목록 아이템 스타일링

## 핵심 요약

- 목표 목록을 단순 텍스트로 출력하는 것에서 나아가 개별 아이템에 스타일을 적용한다.
- `margin`, `padding`, `borderRadius`, `backgroundColor` 등을 사용해 목록 아이템을 카드처럼 보이게 만들 수 있다.
- `Text`에 직접 `borderRadius`를 적용하면 Android에서는 보이지만 iOS에서는 적용되지 않을 수 있다.
- 플랫폼 차이를 줄이려면 `Text`를 `View`로 감싸고, 배경색과 모서리 스타일은 `View`에 적용한다.
- React Native 스타일은 CSS처럼 상속되지 않으므로, 텍스트 색상은 `Text` 컴포넌트에 직접 적용해야 한다.

## 학습 목표

- 목록 아이템에 기본적인 시각 스타일을 적용하는 방법을 익힌다.
- iOS와 Android에서 같은 스타일이 다르게 동작할 수 있다는 점을 이해한다.
- `View`와 `Text`의 스타일 역할을 분리하는 방법을 배운다.
- React Native 스타일링에는 CSS의 상속 개념이 없다는 점을 이해한다.

## 주요 내용

### 목록 아이템 스타일 추가

목표 목록의 각 항목에 스타일을 적용하기 위해 `StyleSheet`에 `goalItem` 스타일 객체를 추가한다.

```jsx
const styles = StyleSheet.create({
  goalItem: {
    margin: 8,
    padding: 8,
    borderRadius: 6,
    backgroundColor: '#5e0acc',
  },
});
```

`margin`은 아이템 사이의 간격을 만들고, `padding`은 아이템 내부 여백을 만든다. `borderRadius`는 모서리를 둥글게 만들며, `backgroundColor`는 아이템 배경색을 지정한다.

### Text에 직접 스타일을 적용했을 때의 문제

처음에는 `Text` 컴포넌트에 바로 `goalItem` 스타일을 적용할 수 있다.

```jsx
<Text key={goal} style={styles.goalItem}>
  {goal}
</Text>
```

하지만 이렇게 하면 Android에서는 둥근 모서리가 보이더라도 iOS에서는 `borderRadius`가 적용되지 않을 수 있다.

React Native는 `Text`를 각 플랫폼의 네이티브 텍스트 요소로 변환한다. Android의 텍스트 요소는 이런 스타일을 처리할 수 있지만, iOS의 네이티브 텍스트 요소는 같은 방식으로 처리하지 못할 수 있다.

### View로 감싸서 해결하기

이 문제를 해결하려면 `Text`를 `View`로 감싸고, 배경색과 둥근 모서리 같은 컨테이너 스타일은 `View`에 적용한다.

```jsx
{courseGoals.map((goal) => (
  <View key={goal} style={styles.goalItem}>
    <Text>{goal}</Text>
  </View>
))}
```

목록에서 반복 렌더링되는 가장 바깥 요소가 이제 `View`이므로, `key` prop도 `View`에 추가한다.

이렇게 하면 iOS와 Android 모두에서 둥근 모서리가 더 안정적으로 적용된다.

### 스타일 상속은 없다

`goalItem` 스타일을 `View`에 옮기면 배경색과 둥근 모서리는 제대로 적용된다. 하지만 텍스트 색상을 `goalItem`에 남겨두면 `Text` 안의 글자 색상은 바뀌지 않는다.

React Native 스타일은 CSS처럼 부모에서 자식으로 상속되지 않는다.

웹 CSS에서는 부모 요소에 `color: white`를 설정하면 자식 텍스트에도 영향을 줄 수 있다. 하지만 React Native에서는 `View`에 설정한 텍스트 색상이 내부 `Text`로 전달되지 않는다.

따라서 텍스트 색상은 `Text` 컴포넌트에 직접 적용해야 한다.

### goalText 스타일 추가

텍스트 색상을 별도로 관리하기 위해 `goalText` 스타일을 만든다.

```jsx
const styles = StyleSheet.create({
  goalItem: {
    margin: 8,
    padding: 8,
    borderRadius: 6,
    backgroundColor: '#5e0acc',
  },
  goalText: {
    color: 'white',
  },
});
```

그리고 `Text` 컴포넌트에 적용한다.

```jsx
{courseGoals.map((goal) => (
  <View key={goal} style={styles.goalItem}>
    <Text style={styles.goalText}>{goal}</Text>
  </View>
))}
```

이렇게 하면 컨테이너 스타일은 `View`가 담당하고, 텍스트 스타일은 `Text`가 담당한다.

### 플랫폼 차이를 다루는 자세

React Native는 하나의 코드베이스로 iOS와 Android 앱을 만들 수 있게 해주지만, 내부적으로는 각 플랫폼의 네이티브 UI 요소로 변환된다.

따라서 대부분의 코드는 동일하게 동작하지만, 스타일이나 일부 동작에서 플랫폼별 차이가 생길 수 있다. 이런 경우에는 더 적절한 컴포넌트로 감싸거나, 나중에 배우게 될 플랫폼별 코드를 사용해 해결할 수 있다.

## 정리

이번 강의에서는 목표 목록 아이템에 여백, 배경색, 둥근 모서리, 텍스트 색상을 적용했다. iOS와 Android의 네이티브 UI 차이 때문에 `Text`에 직접 `borderRadius`를 적용하는 방식은 안정적이지 않을 수 있으며, `View`로 감싸는 방식이 더 적합하다.

또한 React Native 스타일은 CSS처럼 상속되지 않으므로, 텍스트 색상처럼 특정 컴포넌트에 필요한 스타일은 해당 컴포넌트에 직접 적용해야 한다.
