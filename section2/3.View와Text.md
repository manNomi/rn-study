# 3. View와 Text

## 핵심 요약

- React Native의 코어 컴포넌트는 HTML 요소처럼 보일 수 있지만 역할과 제약이 다르다.
- 문자열 텍스트는 반드시 `Text` 컴포넌트 안에서 렌더링해야 한다.
- `View`는 다른 컴포넌트를 담고 배치하는 컨테이너 역할을 한다.
- React Native에서는 사용할 코어 컴포넌트를 `react-native`에서 직접 import해야 한다.
- `Button`은 여는 태그와 닫는 태그 사이에 텍스트를 넣는 방식이 아니라 `title` prop으로 표시할 문구를 설정한다.

## 학습 목표

- `View`와 `Text`의 역할 차이를 이해한다.
- React Native가 웹 React보다 텍스트 렌더링에 더 엄격한 이유를 안다.
- 코어 컴포넌트를 조합해 UI를 구성하는 기본 방식을 익힌다.
- `Button` 컴포넌트를 추가하는 기본 문법을 이해한다.

## 주요 내용

### Text 없이 문자열을 렌더링할 수 없다

React Native에서는 `View` 안에 문자열을 직접 넣을 수 없다.

```jsx
<View>
  Hello World!
</View>
```

이렇게 작성하면 다음과 같은 오류가 발생한다.

```text
텍스트 문자열은 Text 컴포넌트 안에서 렌더링되어야 한다.
```

웹 React에서는 `div` 안에 문자열을 직접 넣을 수 있다. 하지만 React Native의 `View`는 웹의 `div`와 비슷한 컨테이너 역할을 할 뿐, 텍스트를 직접 표시하는 컴포넌트가 아니다.

텍스트를 화면에 표시하려면 반드시 `Text` 컴포넌트로 감싸야 한다.

```jsx
<View>
  <Text>Hello World!</Text>
</View>
```

### View의 역할

`View`는 다른 컴포넌트를 담고 배치하기 위한 컨테이너 컴포넌트다. 웹에서 `div`로 여러 요소를 묶는 것처럼, React Native에서는 `View`를 사용해 여러 컴포넌트를 그룹화한다.

```jsx
<View>
  <Text>Hello World!</Text>
  <Text>Another piece of text!</Text>
</View>
```

`View` 안에는 여러 자식 컴포넌트를 넣을 수 있고, 필요하다면 `View`를 중첩해서 더 복잡한 UI 구조를 만들 수 있다.

React Native에서는 웹의 `section`, `article`, `div`처럼 다양한 컨테이너 태그를 사용하는 대신, 주로 `View`를 컨테이너로 사용한다.

### Text의 역할

`Text`는 화면에 텍스트를 표시하기 위한 컴포넌트다. 텍스트를 보여줘야 하는 모든 곳에서는 `Text`를 사용해야 한다.

이 구분은 React Native에서 매우 중요하다. 각 코어 컴포넌트는 명확한 용도를 가지고 있으며, 그 용도에 맞게 사용해야 한다.

```text
View: 컴포넌트를 담고 배치하는 컨테이너
Text: 텍스트를 표시하는 컴포넌트
```

### 코어 컴포넌트 import

웹 React에서는 `div`, `button`, `h1` 같은 HTML 요소를 import하지 않고 사용할 수 있다. 하지만 React Native에서는 사용할 코어 컴포넌트를 `react-native`에서 가져와야 한다.

```jsx
import { View, Text, Button } from 'react-native';
```

React Native가 제공하는 코어 컴포넌트에는 `View`, `Text`, `TextInput`, `Button`, `Image`, 스크롤 관련 컴포넌트 등이 있다.

### Button 컴포넌트

버튼을 추가하려면 `Button` 컴포넌트를 import한 뒤 JSX에 사용한다.

웹의 `button`처럼 여는 태그와 닫는 태그 사이에 문구를 넣는 방식이 아니라, `title` prop으로 버튼 문구를 지정한다.

```jsx
<Button title="Tap me" />
```

이렇게 추가한 버튼은 각 플랫폼의 기본 스타일과 터치 효과를 사용한다. iOS에서는 iOS 버튼처럼 보이고, Android에서는 Android 버튼처럼 보인다.

아직 이벤트 리스너를 연결하지 않았다면 버튼을 눌러도 별도 동작은 하지 않는다. 버튼을 눌렀을 때 특정 작업을 실행하는 방법은 이후 상호작용을 다룰 때 배운다.

### 코어 컴포넌트 조합

React Native UI는 코어 컴포넌트를 조합해서 만든다. 각 컴포넌트의 역할을 이해하고, 필요한 곳에 적절한 컴포넌트를 배치하는 것이 중요하다.

복잡한 사용자 인터페이스도 결국 `View`, `Text`, `Button`, `TextInput`, `Image` 같은 기본 컴포넌트를 조합하고 스타일링을 적용해 만든다.

## 정리

React Native에서 `View`는 컨테이너이고 `Text`는 텍스트를 표시하는 컴포넌트다. 문자열은 반드시 `Text` 안에 있어야 하며, `View` 안에 직접 넣을 수 없다.

코어 컴포넌트는 각각 목적이 분명하므로, 역할에 맞게 조합해야 한다. 다음 단계에서는 이렇게 만든 UI에 스타일을 적용하는 방법을 살펴본다.
