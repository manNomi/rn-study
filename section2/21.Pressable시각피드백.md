# 21. Pressable 시각 피드백

## 핵심 요약

- `Pressable`을 사용하면 아이템을 눌렀을 때 실행할 동작뿐 아니라 시각적 피드백도 줄 수 있다.
- Android에서는 `android_ripple` prop으로 기본 물결 효과를 쉽게 설정할 수 있다.
- 물결 효과가 적용되는 범위는 `Pressable`이 감싸는 영역과 padding 위치에 따라 달라진다.
- iOS에서는 `android_ripple`이 동작하지 않으므로 `Pressable`의 `style` 함수 형식을 사용해 직접 피드백을 만든다.
- `style` 함수는 현재 눌림 상태를 받아 조건부 스타일을 반환할 수 있다.

## 학습 목표

- `Pressable`에 Android 물결 효과를 적용하는 방법을 익힌다.
- `Pressable`의 감싸는 범위와 padding 위치가 터치 피드백에 미치는 영향을 이해한다.
- iOS에서 `pressed` 상태를 이용해 터치 피드백을 구현하는 방법을 배운다.
- 플랫폼별 터치 피드백 처리 차이를 이해한다.

## 주요 내용

### Android 물결 효과

Android에서는 `Pressable`에 `android_ripple` prop을 추가해 터치 시 물결 효과를 만들 수 있다.

```jsx
<Pressable
  android_ripple={{ color: '#dddddd' }}
  onPress={props.onDeleteItem.bind(this, props.id)}
>
  <View style={styles.goalItem}>
    <Text style={styles.goalText}>{props.text}</Text>
  </View>
</Pressable>
```

`android_ripple`은 객체를 값으로 받으며, `color`를 지정해 물결 효과의 색상을 설정할 수 있다.

### 피드백 범위 조절하기

`Pressable`이 전체 `View`를 감싸고 있으면 물결 효과가 아이템 컨테이너 전체에 적용된다. 반대로 `Pressable`을 `View` 내부로 옮겨 `Text`만 감싸면 물결 효과가 텍스트 영역에만 나타난다.

```jsx
<View style={styles.goalItem}>
  <Pressable
    android_ripple={{ color: '#210644' }}
    onPress={props.onDeleteItem.bind(this, props.id)}
  >
    <Text style={styles.goalText}>{props.text}</Text>
  </Pressable>
</View>
```

이때 `padding`이 바깥 `View`에 남아 있으면 물결 효과가 텍스트 자체에만 좁게 적용될 수 있다.

물결 효과가 안쪽 여백까지 포함되도록 하려면 `padding`을 `goalItem`에서 `goalText`로 옮긴다.

```jsx
const styles = StyleSheet.create({
  goalItem: {
    margin: 8,
    borderRadius: 6,
    backgroundColor: '#5e0acc',
  },
  goalText: {
    color: 'white',
    padding: 8,
  },
});
```

이렇게 하면 텍스트와 그 안쪽 여백 영역까지 `Pressable`의 터치 피드백 범위에 포함된다.

### iOS에서는 android_ripple이 동작하지 않는다

`android_ripple`은 이름 그대로 Android 전용 prop이다. iOS에서는 물결 효과가 표시되지 않는다.

iOS에서도 피드백을 주려면 `Pressable`의 `style` prop을 함수 형태로 사용한다.

```jsx
<Pressable
  android_ripple={{ color: '#210644' }}
  onPress={props.onDeleteItem.bind(this, props.id)}
  style={({ pressed }) => pressed && styles.pressedItem}
>
  <Text style={styles.goalText}>{props.text}</Text>
</Pressable>
```

`Pressable`은 눌림 상태가 바뀔 때마다 이 함수를 호출한다. 함수는 현재 상태 정보를 담은 객체를 받고, 그중 `pressed` 값으로 지금 눌려 있는지 확인할 수 있다.

### pressed 상태로 조건부 스타일 적용하기

눌렸을 때 적용할 스타일을 별도로 만든다.

```jsx
const styles = StyleSheet.create({
  pressedItem: {
    opacity: 0.5,
  },
});
```

그리고 `pressed`가 `true`일 때만 이 스타일을 반환한다.

```jsx
style={({ pressed }) => {
  return pressed ? styles.pressedItem : null;
}}
```

또는 짧게 작성할 수도 있다.

```jsx
style={({ pressed }) => pressed && styles.pressedItem}
```

이제 iOS에서도 아이템을 누르는 동안 투명도가 낮아지는 시각적 피드백이 표시된다.

### 플랫폼별 피드백 조합

Android에서는 `android_ripple`로 네이티브에 가까운 물결 효과를 쉽게 만들 수 있고, iOS에서는 `style` 함수와 `pressed` 상태를 이용해 직접 피드백을 만들 수 있다.

필요하다면 `View`와 `Pressable`의 중첩 구조를 바꾸거나, 추가 `View`로 감싸 더 세밀하게 배경색, 투명도, 피드백 범위를 조정할 수 있다.

## 정리

이번 강의에서는 목표 아이템을 눌렀을 때 시각적 피드백을 추가했다. Android에서는 `android_ripple` prop으로 물결 효과를 설정하고, iOS에서는 `Pressable`의 `style` 함수가 제공하는 `pressed` 상태를 사용해 투명도 변화를 적용했다.

`Pressable`이 어떤 요소를 감싸는지와 padding이 어디에 있는지에 따라 피드백 범위가 달라진다. 플랫폼별 차이를 이해하고 필요한 경우 구조와 스타일을 조정하는 것이 중요하다.
