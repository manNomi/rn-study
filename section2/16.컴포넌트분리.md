# 16. 컴포넌트 분리

## 핵심 요약

- 기능이 늘어나면 `App.js`가 커지므로 작은 컴포넌트로 분리하는 것이 좋다.
- React Native에서도 웹 React와 마찬가지로 여러 컴포넌트를 만들어 조합한다.
- 일반적으로 `components` 폴더를 만들어 재사용하거나 분리할 컴포넌트 파일을 관리한다.
- 목표 앱에서는 `GoalInput.js`와 `GoalItem.js`로 입력 영역과 목록 아이템을 분리한다.
- 컴포넌트에 가까운 스타일은 해당 컴포넌트 파일에 함께 두면 관리하기 쉽다.
- 커스텀 컴포넌트는 JSX에서 사용할 때 대문자로 시작해야 한다.

## 학습 목표

- 컴포넌트를 분리해야 하는 이유를 이해한다.
- React Native 프로젝트에서 컴포넌트 파일을 만드는 기본 흐름을 익힌다.
- `GoalItem` 컴포넌트로 목록 아이템 JSX와 스타일을 옮기는 방법을 이해한다.
- 커스텀 컴포넌트를 import해서 `App.js`에서 사용하는 방법을 복습한다.

## 주요 내용

### App.js가 커지는 문제

기능을 계속 추가하면 `App.js` 파일이 점점 커진다. 이는 React Native뿐 아니라 웹 React 앱에서도 흔히 발생하는 일이다.

컴포넌트가 너무 커지면 읽기 어렵고 유지보수하기도 힘들어진다. 따라서 기능이나 UI 단위로 더 작은 컴포넌트로 나누는 것이 좋은 습관이다.

### components 폴더 만들기

컴포넌트를 분리하기 위해 프로젝트에 `components` 폴더를 만든다.

```text
components/
```

이 이름을 반드시 사용해야 하는 것은 아니지만, React 프로젝트에서는 컴포넌트를 모아두는 폴더 이름으로 흔히 사용된다.

이번 앱에서는 두 개의 컴포넌트 파일을 만들 계획이다.

```text
components/GoalInput.js
components/GoalItem.js
```

`GoalInput.js`에는 목표 입력 영역과 관련된 JSX와 기능을 넣고, `GoalItem.js`에는 목표 항목 하나를 출력하는 JSX와 스타일을 넣는다.

### GoalItem 컴포넌트 만들기

먼저 `GoalItem.js`에서 함수형 컴포넌트를 만든다.

```jsx
function GoalItem() {
  return (
    // 목표 아이템 하나를 렌더링할 JSX
  );
}

export default GoalItem;
```

예전 React 프로젝트에서는 `import React from 'react'`를 자주 작성했지만, 최신 React와 React Native에서는 JSX를 사용하기 위해 꼭 작성할 필요는 없다.

### JSX 옮기기

`App.js`의 `FlatList`에서 목표 아이템 하나를 렌더링하던 JSX를 잘라내어 `GoalItem` 컴포넌트의 반환값으로 옮긴다.

기존에는 `renderItem` 안에서 `View`와 `Text`를 직접 반환했다.

```jsx
renderItem={(itemData) => {
  return (
    <View style={styles.goalItem}>
      <Text style={styles.goalText}>{itemData.item.text}</Text>
    </View>
  );
}}
```

이 JSX를 `GoalItem` 컴포넌트로 옮기면 `App.js`는 목록 구조에 집중하고, 개별 아이템의 UI는 `GoalItem`이 담당하게 된다.

### 스타일도 함께 옮기기

`GoalItem`에만 필요한 스타일은 `GoalItem.js`로 함께 옮긴다.

```jsx
import { StyleSheet, Text, View } from 'react-native';

function GoalItem() {
  return (
    <View style={styles.goalItem}>
      <Text style={styles.goalText}>...</Text>
    </View>
  );
}

const styles = StyleSheet.create({
  goalItem: {
    margin: 8,
    padding: 8,
    borderRadius: 6,
    backgroundColor: '#5e0acc',
  },
  goalText: {
    color: 'white',
  },
});

export default GoalItem;
```

스타일을 반드시 컴포넌트 파일 안에 둬야 하는 것은 아니다. 필요하다면 prop으로 전달할 수도 있다. 하지만 특정 컴포넌트에만 속한 스타일은 해당 컴포넌트 가까이에 두는 편이 관리하기 좋다.

### App.js에서 GoalItem 사용하기

`GoalItem`을 만들었다면 `App.js`에서 import해서 사용한다.

```jsx
import GoalItem from './components/GoalItem';
```

그리고 `FlatList`의 `renderItem`에서 `GoalItem`을 반환한다.

```jsx
<FlatList
  data={courseGoals}
  renderItem={(itemData) => {
    return <GoalItem />;
  }}
  keyExtractor={(item) => item.id}
/>
```

이후에는 `GoalItem`에 실제 목표 텍스트를 prop으로 전달해야 한다. 이 부분은 다음 단계에서 다루게 된다.

### 커스텀 컴포넌트 이름

React에서 직접 만든 컴포넌트는 대문자로 시작해야 한다.

```jsx
<GoalItem />
```

소문자로 시작하면 React는 이를 커스텀 컴포넌트가 아니라 네이티브 태그나 기본 요소처럼 해석하려고 할 수 있다. 따라서 컴포넌트 함수 이름과 JSX 태그 이름은 대문자로 시작하는 것이 규칙이다.

## 정리

이번 강의에서는 커지는 `App.js`를 정리하기 위해 컴포넌트를 분리하는 흐름을 살펴봤다. `components` 폴더를 만들고, 목표 아이템 하나를 담당하는 `GoalItem` 컴포넌트로 JSX와 관련 스타일을 옮겼다.

React Native에서도 컴포넌트를 만들고 import해서 사용하는 방식은 웹 React와 동일하다. 앞으로 앱이 커질수록 기능과 UI 단위로 컴포넌트를 나누는 방식이 중요해진다.
