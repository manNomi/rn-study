# 15. FlatList로 목록 최적화

## 핵심 요약

- `ScrollView`는 자식 항목을 모두 렌더링하므로 긴 동적 목록에는 성능 문제가 생길 수 있다.
- 긴 목록이나 항목 수가 늘어날 수 있는 목록에는 `FlatList`가 더 적합하다.
- `FlatList`는 화면에 보이는 항목과 곧 보일 항목만 렌더링해 성능을 최적화한다.
- `FlatList`는 `data`와 `renderItem` prop을 사용해 목록 데이터를 렌더링한다.
- 목록 항목에는 여전히 고유한 key가 필요하며, `key` prop 또는 `keyExtractor`로 지정할 수 있다.

## 학습 목표

- `ScrollView`와 `FlatList`의 차이를 이해한다.
- 긴 동적 목록에서 `FlatList`를 사용해야 하는 이유를 안다.
- `data`, `renderItem`, `itemData.item`의 역할을 익힌다.
- `key`와 `keyExtractor`로 목록 항목을 식별하는 방법을 이해한다.

## 주요 내용

### ScrollView의 한계

`ScrollView`는 콘텐츠를 스크롤 가능하게 만들 때 유용하다. 예를 들어 긴 기사나 분량이 어느 정도 정해진 콘텐츠를 보여줄 때는 좋은 선택이다.

하지만 동적으로 길어질 수 있는 목록에는 적합하지 않을 수 있다. `ScrollView`는 안에 있는 모든 자식 항목을 한 번에 렌더링하기 때문이다.

목록에 항목이 10개나 20개 정도라면 큰 문제가 없을 수 있다. 하지만 수백 개, 수천 개 항목이 있다면 화면에 보이지 않는 항목까지 모두 렌더링되므로 앱 성능이 나빠질 수 있다.

### FlatList가 필요한 이유

`FlatList`는 React Native가 제공하는 스크롤 가능한 목록 전용 컴포넌트다.

`FlatList`는 모든 항목을 처음부터 렌더링하지 않는다. 현재 화면에 보이는 항목과, 사용자가 곧 스크롤해서 볼 가능성이 있는 항목만 렌더링한다.

이 덕분에 목록에 많은 데이터가 있어도 초기 렌더링 부담이 줄어들고, 긴 목록에서도 더 나은 성능을 기대할 수 있다.

### ScrollView를 FlatList로 교체하기

먼저 `ScrollView` 대신 `FlatList`를 import한다.

```jsx
import { FlatList } from 'react-native';
```

기존에 `ScrollView`로 감싸고 직접 `map`을 사용하던 코드는 제거한다. `FlatList`는 데이터를 직접 받고, 각 항목을 어떻게 렌더링할지도 prop으로 전달받는다.

### data prop

`data` prop은 목록으로 출력할 배열 데이터를 지정한다.

```jsx
<FlatList data={courseGoals} />
```

여기서는 `courseGoals` 배열이 목표 목록 데이터이므로 `data`에 전달한다.

### renderItem prop

`renderItem`은 각 데이터 항목을 어떤 JSX로 렌더링할지 알려주는 함수다.

```jsx
<FlatList
  data={courseGoals}
  renderItem={(itemData) => {
    return (
      <View style={styles.goalItem}>
        <Text style={styles.goalText}>{itemData.item}</Text>
      </View>
    );
  }}
/>
```

`FlatList`는 각 항목을 렌더링해야 할 때마다 `renderItem` 함수를 호출한다. 이 함수는 `itemData` 객체를 매개변수로 받는다.

`itemData`에는 여러 정보가 들어 있으며, 실제 데이터 항목은 `itemData.item`으로 접근한다.

```text
itemData.item: 현재 렌더링 중인 실제 데이터
itemData.index: 현재 항목의 인덱스
```

### key가 필요한 이유

React에서 목록을 렌더링할 때는 각 항목을 식별할 고유한 key가 필요하다. `FlatList`를 사용해도 이 요구사항은 그대로 유지된다.

초기에는 목표를 문자열 배열로 관리했기 때문에 key가 없다는 경고가 발생한다.

이를 해결하는 첫 번째 방법은 데이터 항목을 객체로 만들고, 각 객체에 `key` 프로퍼티를 추가하는 것이다.

```jsx
function addGoalHandler() {
  setCourseGoals((currentCourseGoals) => [
    ...currentCourseGoals,
    {
      text: enteredGoalText,
      key: Math.random().toString(),
    },
  ]);
}
```

이제 `courseGoals` 배열의 각 항목은 문자열이 아니라 `text`와 `key`를 가진 객체다.

```text
{ text: 'Learn React Native', key: '0.12345' }
```

`FlatList`는 데이터 객체에 `key` 프로퍼티가 있으면 이를 자동으로 항목의 key로 사용한다.

렌더링할 텍스트도 이제 객체의 `text` 프로퍼티에서 가져와야 한다.

```jsx
<FlatList
  data={courseGoals}
  renderItem={(itemData) => {
    return (
      <View style={styles.goalItem}>
        <Text style={styles.goalText}>{itemData.item.text}</Text>
      </View>
    );
  }}
/>
```

### keyExtractor 사용하기

항상 데이터에 `key`라는 이름의 프로퍼티가 있는 것은 아니다. API에서 받은 데이터가 `id`를 사용하거나, 데이터를 직접 바꾸고 싶지 않을 수도 있다.

이럴 때는 `keyExtractor` prop을 사용한다.

```jsx
function addGoalHandler() {
  setCourseGoals((currentCourseGoals) => [
    ...currentCourseGoals,
    {
      text: enteredGoalText,
      id: Math.random().toString(),
    },
  ]);
}
```

데이터에 `id`를 사용한다면, `FlatList`에 각 항목의 key를 어떻게 추출할지 알려준다.

```jsx
<FlatList
  data={courseGoals}
  renderItem={(itemData) => {
    return (
      <View style={styles.goalItem}>
        <Text style={styles.goalText}>{itemData.item.text}</Text>
      </View>
    );
  }}
  keyExtractor={(item, index) => {
    return item.id;
  }}
/>
```

`keyExtractor`는 각 항목마다 호출되며, 고유한 key 문자열을 반환해야 한다. 위 예시에서는 각 항목의 `id`를 key로 사용한다.

### ScrollView와 비슷한 설정

`FlatList`는 스크롤 가능한 목록 컴포넌트이므로 `ScrollView`에서 사용하던 일부 prop을 그대로 지원한다.

예를 들어 세로 바운스 동작을 조정할 수 있다.

```jsx
<FlatList
  alwaysBounceVertical={false}
  data={courseGoals}
  renderItem={...}
/>
```

다만 `FlatList`의 핵심 장점은 단순 스크롤이 아니라, 필요한 항목만 렌더링하는 목록 최적화에 있다.

## 정리

`ScrollView`는 긴 콘텐츠를 스크롤할 때 유용하지만, 매우 길어질 수 있는 동적 목록에는 성능상 적합하지 않을 수 있다. 모든 항목을 한 번에 렌더링하기 때문이다.

`FlatList`는 `data`와 `renderItem`을 통해 목록을 렌더링하고, 화면에 필요한 항목만 렌더링해 성능을 개선한다. 데이터에 `key`가 없거나 다른 이름의 식별자를 사용한다면 `keyExtractor`로 고유 key를 지정할 수 있다.
