# 2. 핵심 컴포넌트와 스타일링

## 핵심 요약

- 기존 Expo 프로젝트를 열고 `npm install`, `npm start`로 개발 환경을 다시 준비한다.
- React Native에서는 함수 컴포넌트와 React 훅을 중심으로 앱을 구성한다.
- `View`, `Text` 같은 핵심 컴포넌트는 `react-native`에서 제공되며 JSX에서 사용된다.
- React Native 앱은 HTML 요소와 DOM을 사용하지 않으므로 `div`, `h1` 같은 태그를 사용할 수 없다.
- CSS 파일 대신 인라인 스타일이나 `StyleSheet` 객체를 사용해 JavaScript로 스타일을 작성한다.
- `App.js`의 `App` 컴포넌트는 Expo가 렌더링하는 루트 컴포넌트다.

## 학습 목표

- React Native 프로젝트를 다시 실행하는 기본 흐름을 확인한다.
- React Native 핵심 컴포넌트가 필요한 이유를 이해한다.
- 웹 CSS와 React Native 스타일링 방식의 차이를 구분한다.
- `App` 컴포넌트가 앱의 루트 컴포넌트라는 점을 이해한다.

## 주요 내용

### 프로젝트 다시 실행하기

이전 섹션에서 만든 프로젝트를 다시 열고 작업을 시작한다. 프로젝트를 새로 받았거나 의존성이 설치되어 있지 않다면 먼저 다음 명령어를 실행한다.

```bash
npm install
```

의존성 설치가 끝나면 Expo 개발 서버를 시작한다.

```bash
npm start
```

개발 서버가 실행되면 터미널에서 `a` 또는 `i`를 눌러 Android 에뮬레이터나 iOS 시뮬레이터에서 앱을 실행할 수 있다.

### App.js의 기본 구조

`App.js`에는 앱의 시작점이 되는 `App` 컴포넌트가 있다. 이 강의에서는 함수 컴포넌트와 React 훅을 사용하며, 클래스 기반 컴포넌트는 사용하지 않는다.

파일 상단에는 필요한 패키지와 컴포넌트를 가져오는 `import` 구문이 있다. Expo 관련 패키지에서 가져오는 항목도 있고, `View`, `Text`, `StyleSheet`처럼 `react-native`에서 가져오는 핵심 요소도 있다.

### View와 Text

`View`와 `Text`는 React Native에서 가장 기본적이고 중요한 내장 컴포넌트다. React Native 앱에서는 웹의 `div`, `h1`, `p` 같은 HTML 요소를 사용할 수 없다.

그 이유는 React Native 앱이 브라우저에서 실행되는 웹 앱이 아니기 때문이다. 네이티브 기기에는 브라우저 DOM이 없고, HTML 요소도 지원되지 않는다.

React Native는 대신 `View`, `Text`, `TextInput`, `Button`, `Image` 같은 핵심 컴포넌트를 제공한다. 이 컴포넌트들은 Android와 iOS에서 각각 알맞은 네이티브 UI 요소로 변환된다.

### 핵심 컴포넌트로 UI 만들기

React Native의 사용자 인터페이스는 핵심 컴포넌트를 조합해 만든다. 직접 만드는 커스텀 컴포넌트도 결국 내부적으로는 React Native가 제공하는 핵심 컴포넌트들을 합쳐서 구성된다.

웹에서 HTML 요소를 조합해 UI를 만드는 것과 비슷하다. 다만 웹에는 더 많은 HTML 태그가 있고, React Native에서는 비교적 적은 수의 핵심 컴포넌트를 조합해 앱 화면을 구성한다.

React Native 공식 문서의 Components 탭에서 제공되는 핵심 컴포넌트 목록을 확인할 수 있다. 앞으로 강의에서는 이 컴포넌트들을 반복해서 사용하게 된다.

### React Native의 스타일링

React Native에는 CSS가 없다. 브라우저가 아니기 때문에 별도의 CSS 파일을 작성하거나 CSS 언어를 그대로 사용할 수 없다.

대신 JavaScript 객체를 사용해 스타일을 작성한다. 스타일은 핵심 컴포넌트의 `style` prop에 인라인으로 전달하거나, `StyleSheet` 객체를 사용해 정의할 수 있다.

```jsx
const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#fff',
    alignItems: 'center',
    justifyContent: 'center',
  },
});
```

프로퍼티 이름은 CSS와 비슷하지만 완전히 같은 것은 아니다. React Native는 CSS의 모든 기능을 지원하지는 않지만, 앱 UI를 만들 때 필요한 주요 스타일 속성은 대부분 제공한다.

### App 컴포넌트의 역할

`App.js`의 `App` 컴포넌트는 Expo가 자동으로 앱의 루트 컴포넌트로 렌더링하는 특별한 컴포넌트다.

따라서 화면에 표시되는 모든 UI 요소와 직접 만든 컴포넌트는 `App` 컴포넌트의 자식이나 자손으로 연결되어야 한다.

앞으로 앱이 커지면 여러 커스텀 컴포넌트를 만들게 되지만, 최종적으로는 모두 `App` 컴포넌트 아래에서 렌더링된다.

## 정리

React Native 앱은 웹의 HTML 요소와 CSS가 아니라, `View`, `Text` 같은 핵심 컴포넌트와 JavaScript 기반 스타일링으로 구성된다.

`App.js`의 `App` 컴포넌트는 앱의 루트 컴포넌트이며, 앞으로 만들 모든 화면과 컴포넌트는 이 구조 아래에서 조합된다.
