# 23. Modal 입력 화면 스타일링

## 핵심 요약

- `GoalInput` 컴포넌트 안의 `Modal` 화면을 입력 전용 화면처럼 보이도록 정리한다.
- `TextInput` 아래에 `Add Goal` 버튼과 `Cancel` 버튼을 나란히 배치한다.
- React Native의 기본 `Button`은 `style` prop을 지원하지 않으므로, 버튼을 감싸는 `View`에 스타일을 적용한다.
- `inputContainer`에 `flex: 1`, `justifyContent: 'center'`, `padding`을 적용해 모달 콘텐츠를 화면 중앙에 배치한다.
- 버튼 간격과 너비는 `buttonContainer`와 개별 버튼 래퍼 스타일로 조정한다.

## 학습 목표

- `Modal` 내부 UI를 정돈하는 기본 스타일링 흐름을 익힌다.
- `View`를 중첩해 버튼 영역과 입력 영역을 분리하는 방법을 이해한다.
- 기본 `Button`의 스타일링 한계를 알고, 래퍼 `View`로 레이아웃을 제어하는 방법을 배운다.
- `TextInput`과 버튼 사이의 여백, 버튼 간 간격, 입력창 너비를 조정하는 법을 정리한다.

## 주요 내용

### Modal 내부 구조 변경

기존에는 `TextInput`과 `Add Goal` 버튼이 같은 `View` 안에 바로 들어 있었다. 이번 단계에서는 입력창 아래에 버튼 영역을 따로 만들고, 그 안에 `Add Goal` 버튼과 `Cancel` 버튼을 넣는다.

```jsx
<Modal visible={props.visible} animationType="slide">
  <View style={styles.inputContainer}>
    <TextInput
      style={styles.textInput}
      placeholder="Your course goal!"
      onChangeText={goalInputHandler}
      value={enteredGoalText}
    />
    <View style={styles.buttonContainer}>
      <View style={styles.button}>
        <Button title="Cancel" />
      </View>
      <View style={styles.button}>
        <Button title="Add Goal" onPress={addGoalHandler} />
      </View>
    </View>
  </View>
</Modal>
```

이 시점의 `Cancel` 버튼은 아직 모달을 닫지 않는다. 다음 강의에서 버튼을 눌렀을 때 모달이 닫히도록 핸들러를 연결한다.

### inputContainer 스타일 조정

모달 내부의 최상위 `View`는 입력 화면 전체의 배치를 담당한다. `flex: 1`을 주면 모달에서 사용할 수 있는 전체 공간을 차지하고, `justifyContent: 'center'`로 입력창과 버튼 묶음을 화면 중앙에 모을 수 있다.

```jsx
inputContainer: {
  flex: 1,
  justifyContent: 'center',
  padding: 16,
},
```

`flexDirection`은 기본값이 `column`이므로 별도로 설정하지 않아도 입력창과 버튼 영역이 위아래로 배치된다.

### 버튼을 가로로 배치하기

두 버튼을 나란히 보이게 하려면 버튼들을 감싼 `buttonContainer`에 `flexDirection: 'row'`를 적용한다.

```jsx
buttonContainer: {
  flexDirection: 'row',
  marginTop: 16,
},
```

`marginTop`은 `TextInput`과 버튼 영역 사이에 간격을 만든다.

### Button은 래퍼 View로 스타일링하기

React Native의 기본 `Button`은 `style` prop으로 자유롭게 꾸밀 수 없다. 그래서 버튼 자체가 아니라 버튼을 감싸는 `View`에 너비와 여백을 준다.

```jsx
button: {
  width: 100,
  marginHorizontal: 8,
},
```

이 방식으로 버튼 사이의 간격과 버튼이 차지하는 공간을 조정할 수 있다. 다만 버튼 글자 색상, 배경, 모양을 세밀하게 바꾸고 싶다면 `Pressable`, `View`, `Text`를 조합해 직접 커스텀 버튼을 만들어야 한다.

### TextInput 너비와 여백 정리

모달 안에서는 입력창이 화면 폭을 충분히 쓰도록 `width: '100%'`를 적용한다. 대신 바깥 컨테이너인 `inputContainer`에 `padding: 16`을 주어 화면 가장자리와 입력창 사이의 여백을 만든다.

```jsx
textInput: {
  borderWidth: 1,
  borderColor: '#cccccc',
  width: '100%',
  padding: 8,
},
```

이전 레이아웃에서 사용하던 `marginRight`는 더 이상 필요하지 않다. 입력창과 버튼이 가로로 붙어 있는 구조가 아니기 때문이다.

### 전체 스타일 예시

```jsx
const styles = StyleSheet.create({
  inputContainer: {
    flex: 1,
    justifyContent: 'center',
    padding: 16,
  },
  textInput: {
    borderWidth: 1,
    borderColor: '#cccccc',
    width: '100%',
    padding: 8,
  },
  buttonContainer: {
    flexDirection: 'row',
    marginTop: 16,
  },
  button: {
    width: 100,
    marginHorizontal: 8,
  },
});
```

## 정리

이번 강의에서는 `Modal` 안의 입력 UI를 더 자연스러운 화면 구조로 다듬었다. 입력창은 화면 중앙에 배치하고, 그 아래에 `Add Goal`과 `Cancel` 버튼을 가로로 나란히 놓았다.

기본 `Button`은 직접 스타일링할 수 있는 범위가 제한적이므로, 레이아웃 조정은 버튼을 감싸는 `View`에서 처리했다. 다음 단계에서는 `Cancel`이나 `Add Goal` 버튼을 눌렀을 때 모달을 닫는 동작을 연결한다.
