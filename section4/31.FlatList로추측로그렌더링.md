# 31. FlatList로 추측 로그 렌더링

## 핵심 요약

- 이전 단계에서 `map()`으로 출력하던 추측 로그를 `FlatList`로 바꾼다.
- `FlatList`는 `data` prop으로 출력할 배열을 받고, `renderItem` prop으로 각 항목을 어떻게 렌더링할지 정한다.
- 이번 데이터는 객체 배열이 아니라 숫자 배열이므로 `itemData.item` 자체가 추측 숫자다.
- 숫자 배열에는 기본 `key` 프로퍼티가 없기 때문에 `keyExtractor`를 직접 제공해야 한다.
- 이 게임에서는 같은 숫자를 다시 추측하지 않으므로 각 숫자 값을 key로 사용할 수 있다.
- `FlatList`로 변경해도 화면에 보이는 결과는 비슷하지만, React Native의 표준 목록 렌더링 방식에 더 가까워진다.
- 이번 단계에서는 목록 렌더링만 바꾸고, 항목 스타일링은 다음 단계에서 개선한다.

## 학습 목표

- `FlatList`의 기본 사용 흐름을 복습한다.
- `data`, `renderItem`, `keyExtractor` prop의 역할을 이해한다.
- `FlatList`가 `renderItem`에 전달하는 `itemData` 구조를 이해한다.
- 원시값 배열을 `FlatList`로 렌더링할 때 key를 어떻게 지정하는지 배운다.
- `map()`과 `FlatList`의 차이를 다시 정리한다.

## 주요 내용

### map에서 FlatList로 변경하기

이전 단계에서는 `guessRounds` 배열을 `map()`으로 렌더링했다.

```jsx
<View>
  {guessRounds.map((guessRound) => (
    <Text key={guessRound}>{guessRound}</Text>
  ))}
</View>
```

이 방식도 짧은 목록에는 충분하다.

하지만 React Native에서는 목록 렌더링을 위해 `FlatList`를 자주 사용한다.

```text
map
-> JavaScript 배열을 직접 JSX로 변환

FlatList
-> React Native의 목록 렌더링 컴포넌트
-> 긴 목록에 더 적합
-> 스크롤과 최적화에 유리
```

이번 강의에서는 연습을 위해 라운드 로그를 `FlatList`로 바꾼다.

### FlatList import하기

먼저 React Native에서 `FlatList`를 가져온다.

```jsx
import { FlatList } from 'react-native';
```

기존에 `Text`, `View` 등을 import하고 있었다면 그 목록에 추가하면 된다.

```jsx
import { Alert, FlatList, StyleSheet, Text, View } from 'react-native';
```

### FlatList의 data prop

`FlatList`는 어떤 데이터를 출력할지 `data` prop으로 받는다.

이번에 출력할 데이터는 `guessRounds` 상태다.

```jsx
<FlatList data={guessRounds} />
```

`guessRounds`는 장치가 추측한 숫자들이 들어 있는 배열이다.

```text
guessRounds
-> [63, 14]
-> [31, 42, 63, 14]
```

따라서 `FlatList`는 이 배열의 각 숫자를 항목으로 렌더링한다.

### renderItem prop

`FlatList`에는 각 항목을 어떻게 렌더링할지 알려줘야 한다.

이를 위해 `renderItem` prop을 사용한다.

```jsx
<FlatList
  data={guessRounds}
  renderItem={(itemData) => {
    return <Text>{itemData.item}</Text>;
  }}
/>
```

`renderItem`은 함수다.

이 함수는 `FlatList`가 각 데이터 항목마다 호출한다.

```text
guessRounds = [63, 14]

첫 번째 호출
-> itemData.item = 63

두 번째 호출
-> itemData.item = 14
```

함수의 반환값은 화면에 렌더링할 JSX다.

### itemData.item 이해하기

`FlatList`가 `renderItem`에 넘기는 값은 단순히 데이터 항목 자체가 아니라 객체다.

보통 이 인수를 `itemData`라고 부를 수 있다.

```jsx
renderItem={(itemData) => {
  console.log(itemData);
}}
```

이 객체 안에는 실제 데이터 항목이 `item` 프로퍼티로 들어 있다.

```text
itemData
-> item
-> index
-> separators 등 메타데이터
```

이번 데이터는 숫자 배열이므로 실제 값은 `itemData.item`이다.

```jsx
<Text>{itemData.item}</Text>
```

만약 데이터가 객체 배열이었다면 다음처럼 접근했을 수 있다.

```jsx
<Text>{itemData.item.value}</Text>
```

하지만 이번에는 항목 자체가 숫자이므로 `.value` 같은 프로퍼티가 없다.

```text
객체 배열
-> itemData.item.value

숫자 배열
-> itemData.item
```

### keyExtractor가 필요한 이유

`FlatList`도 React 목록이므로 각 항목을 고유하게 식별할 key가 필요하다.

객체 배열이라면 보통 `key` 또는 `id` 프로퍼티를 사용한다.

```jsx
[
  { id: 'g1', value: 63 },
  { id: 'g2', value: 14 },
]
```

하지만 이번 `guessRounds`는 숫자 배열이다.

```jsx
[63, 14]
```

각 항목이 객체가 아니므로 `key` 프로퍼티가 없다.

따라서 `FlatList`에 직접 key를 추출하는 방법을 알려줘야 한다.

```jsx
keyExtractor={(item) => item}
```

다만 key는 문자열이어야 하므로 더 안전하게 문자열로 변환할 수 있다.

```jsx
keyExtractor={(item) => item.toString()}
```

강의 흐름에서는 숫자 자체가 고유하므로 key로 사용할 수 있다고 설명한다.

실제 코드에서는 문자열 변환까지 해두면 더 명확하다.

### 왜 추측 숫자를 key로 쓸 수 있는가

이 게임은 같은 숫자를 다시 추측하지 않도록 로직을 구성했다.

```text
이미 추측한 숫자
-> 다음 난수 생성에서 제외

결과
-> guessRounds 안의 숫자는 중복되지 않음
```

따라서 각 숫자는 목록 안에서 고유하다.

```text
guessRounds = [63, 14, 42]
-> 63, 14, 42 모두 고유
```

이런 조건이 있기 때문에 숫자 값을 key로 사용할 수 있다.

단, 일반적인 목록에서는 값 자체가 중복될 수 있으므로 id를 쓰는 편이 더 안전하다.

### FlatList 전체 예시

라운드 로그 출력은 다음처럼 작성할 수 있다.

```jsx
<FlatList
  data={guessRounds}
  renderItem={(itemData) => (
    <Text>{itemData.item}</Text>
  )}
  keyExtractor={(item) => item.toString()}
/>
```

이 코드는 `guessRounds`의 모든 숫자를 `Text`로 출력한다.

```text
data
-> 출력할 배열

renderItem
-> 각 항목을 렌더링하는 함수

keyExtractor
-> 각 항목의 고유 key를 만드는 함수
```

### map과 비교하기

같은 결과를 `map()`으로 작성하면 다음과 같다.

```jsx
{guessRounds.map((guessRound) => (
  <Text key={guessRound}>{guessRound}</Text>
))}
```

`FlatList`로 작성하면 다음과 같다.

```jsx
<FlatList
  data={guessRounds}
  renderItem={(itemData) => (
    <Text>{itemData.item}</Text>
  )}
  keyExtractor={(item) => item.toString()}
/>
```

둘 다 화면에 숫자 목록을 출력한다.

하지만 `FlatList`는 React Native에서 긴 목록을 더 효율적으로 처리하기 위한 컴포넌트다.

```text
map
-> 단순하고 직관적
-> 짧은 목록에 적합

FlatList
-> 목록 전용 컴포넌트
-> 긴 목록, 스크롤 목록에 적합
```

이번 목록은 짧지만, 연습을 위해 `FlatList`를 사용한다.

### 아직 스타일은 부족하다

`FlatList`로 바꿔도 출력 결과가 바로 예뻐지는 것은 아니다.

현재는 숫자가 단순한 텍스트 목록처럼 보인다.

```text
현재 상태
-> 숫자들이 출력됨
-> 스타일은 거의 없음
```

다음 단계에서는 각 로그 항목을 보기 좋게 스타일링한다.

```text
다음 작업
-> 로그 항목 박스 만들기
-> 라운드 번호 표시
-> 추측 숫자 표시
-> 목록 스타일 개선
```

### 프론트엔드 개발자 관점에서의 핵심 차이

웹 React에서는 보통 `map()`으로 목록을 렌더링한다.

```jsx
items.map((item) => (
  <li key={item.id}>{item.value}</li>
));
```

React Native에서도 `map()`은 사용할 수 있다.

하지만 모바일 앱에서는 화면에 표시되는 항목만 효율적으로 렌더링하는 목록 컴포넌트가 중요하다.

그래서 React Native에서는 `FlatList`가 자주 등장한다.

```jsx
<FlatList
  data={items}
  renderItem={({ item }) => <Text>{item.value}</Text>}
  keyExtractor={(item) => item.id}
/>
```

웹에서 `react-window`나 `react-virtualized` 같은 가상화 목록을 쓰는 상황과 비슷하게 이해할 수 있다.

```text
웹
-> 기본은 map
-> 큰 목록은 virtualization 라이브러리 고려

React Native
-> FlatList가 목록 최적화 역할
```

### renderItem 구조 분해도 가능하다

강의에서는 `itemData`를 인수로 받아 `itemData.item`에 접근했다.

더 짧게 쓰려면 객체 구조 분해를 사용할 수도 있다.

```jsx
renderItem={({ item }) => (
  <Text>{item}</Text>
)}
```

두 방식은 같다.

```text
itemData.item
-> 명시적으로 구조 이해하기 좋음

{ item }
-> 간결하게 작성 가능
```

학습 초기에는 `itemData.item` 형태가 `FlatList`가 어떤 데이터를 넘기는지 이해하기 좋다.

## 정리

이번 강의에서는 이전에 `map()`으로 출력하던 추측 로그를 `FlatList`로 변경했다. `FlatList`에는 출력할 배열인 `guessRounds`를 `data`로 전달하고, 각 항목을 렌더링하는 함수를 `renderItem`으로 제공한다.

이번 데이터는 객체가 아니라 숫자 배열이므로 `itemData.item` 자체가 화면에 출력할 숫자다. 또한 숫자 항목에는 `key` 프로퍼티가 없으므로 `keyExtractor`를 사용해 각 숫자를 key로 변환해준다.

현재는 로그 항목이 단순 텍스트로 출력되지만, 다음 단계에서는 이 목록 항목을 더 보기 좋게 스타일링하게 된다.
