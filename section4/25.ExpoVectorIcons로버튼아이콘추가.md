# 25. Expo Vector Icons로 버튼 아이콘 추가

## 핵심 요약

- 게임 화면의 `+`, `-` 텍스트 버튼을 실제 아이콘으로 교체한다.
- Expo 프로젝트에는 `@expo/vector-icons`를 기본적으로 사용할 수 있는 환경이 포함되어 있다.
- `@expo/vector-icons`는 여러 아이콘 세트를 제공하며, 강의에서는 `Ionicons`를 사용한다.
- 아이콘은 React 컴포넌트처럼 JSX에서 렌더링한다.
- `name`, `size`, `color` prop으로 어떤 아이콘을 어떤 크기와 색상으로 보여줄지 설정한다.
- 이번 강의에서는 두 플랫폼에 같은 아이콘을 사용하지만, 플랫폼별 아이콘 분기는 이후에 다룬다.

## 학습 목표

- Expo 프로젝트에서 아이콘 라이브러리를 사용하는 방법을 익힌다.
- `@expo/vector-icons`와 `Ionicons`의 역할을 이해한다.
- 텍스트 기호를 아이콘 컴포넌트로 교체하는 흐름을 배운다.
- 아이콘 컴포넌트의 `name`, `size`, `color` prop 사용법을 익힌다.
- React Native에서도 아이콘을 JSX 컴포넌트처럼 조합할 수 있음을 이해한다.

## 주요 내용

### 게임 화면에서 개선할 부분

게임 화면의 피드백 버튼은 현재 일반 문자로 표시된다.

```jsx
<PrimaryButton>+</PrimaryButton>
<PrimaryButton>-</PrimaryButton>
```

기능적으로는 문제없지만, 모바일 앱 UI에서는 실제 아이콘을 사용하는 편이 더 자연스럽다.

```text
현재
-> + 문자
-> - 문자

개선
-> plus 아이콘
-> minus 아이콘
```

이번 강의에서는 이 텍스트 기호를 아이콘으로 교체한다.

### Expo에서 아이콘 사용하기

Expo로 생성한 프로젝트에서는 `@expo/vector-icons` 패키지를 사용할 수 있다.

```jsx
import { Ionicons } from '@expo/vector-icons';
```

별도의 설치 없이 사용할 수 있는 이유는 Expo 프로젝트에서 이 패키지가 기본적으로 연결되어 있기 때문이다.

```text
Expo 프로젝트
-> @expo/vector-icons 사용 가능
-> 다양한 아이콘 세트 제공
```

아이콘을 직접 이미지 파일로 추가하거나 SVG를 따로 관리하지 않아도 된다.

### @expo/vector-icons란?

`@expo/vector-icons`는 여러 아이콘 세트를 React Native 컴포넌트로 사용할 수 있게 해주는 패키지다.

```text
@expo/vector-icons
-> 여러 아이콘 세트 제공
-> 아이콘을 JSX 컴포넌트로 렌더링
-> size, color 등 prop으로 제어
```

사용 가능한 아이콘 세트는 다양하다.

```text
Ionicons
MaterialIcons
FontAwesome
Entypo
AntDesign
```

강의에서는 그중 `Ionicons`를 사용한다.

### Ionicons 사용하기

먼저 `GameScreen`에서 `Ionicons`를 import한다.

```jsx
import { Ionicons } from '@expo/vector-icons';
```

그 다음 `PrimaryButton` 내부의 `+`, `-` 텍스트를 아이콘 컴포넌트로 교체한다.

```jsx
<PrimaryButton onPress={nextGuessHandler.bind(this, 'greater')}>
  <Ionicons name="md-add" size={24} color="white" />
</PrimaryButton>
```

```jsx
<PrimaryButton onPress={nextGuessHandler.bind(this, 'lower')}>
  <Ionicons name="md-remove" size={24} color="white" />
</PrimaryButton>
```

`Ionicons`는 셀프 클로징 컴포넌트로 사용할 수 있다.

```jsx
<Ionicons />
```

여기에 필요한 prop을 전달해 구체적인 아이콘을 지정한다.

### name prop

`name` prop은 어떤 아이콘을 사용할지 정한다.

```jsx
name="md-add"
```

```jsx
name="md-remove"
```

이번 강의에서는 다음 아이콘을 사용한다.

```text
md-add
-> 플러스 아이콘

md-remove
-> 마이너스 아이콘
```

아이콘 이름은 아이콘 세트마다 다르다.

따라서 `Ionicons`를 import했다면 `Ionicons`에 속한 아이콘 이름을 사용해야 한다.

```text
Ionicons import
-> Ionicons 아이콘 이름 사용
```

다른 아이콘 세트를 import했다면 해당 세트의 이름 체계를 따라야 한다.

### size prop

`size` prop은 아이콘 크기를 정한다.

```jsx
size={24}
```

React Native의 많은 크기 값처럼 숫자를 전달한다.

이 값은 화면에서 아이콘이 표시될 크기를 결정한다.

```text
size={16}
-> 작은 아이콘

size={24}
-> 일반 버튼에 적당한 아이콘

size={32}
-> 더 큰 아이콘
```

강의에서는 버튼 안에서 충분히 잘 보이도록 `24`를 사용한다.

### color prop

`color` prop은 아이콘 색상을 정한다.

```jsx
color="white"
```

문자열 색상 이름이나 헥스 코드를 사용할 수 있다.

```jsx
color="#ffffff"
```

이번 버튼은 자주색 배경 위에 올라가므로 흰색 아이콘이 잘 어울린다.

```text
버튼 배경
-> primary 계열 자주색

아이콘 색상
-> white
```

### 텍스트와 아이콘의 차이

기존에는 버튼 내부에 문자열을 넣었다.

```jsx
<PrimaryButton>+</PrimaryButton>
```

이제는 버튼 내부에 아이콘 컴포넌트를 넣는다.

```jsx
<PrimaryButton>
  <Ionicons name="md-add" size={24} color="white" />
</PrimaryButton>
```

`PrimaryButton`은 `children`을 렌더링하므로 문자열뿐 아니라 컴포넌트도 받을 수 있다.

```text
children
-> 문자열 가능
-> React 컴포넌트 가능
-> 여러 요소 가능
```

이것이 `children`을 활용한 재사용 컴포넌트의 장점이다.

`PrimaryButton` 내부 구현을 바꾸지 않고도 버튼 내용만 교체할 수 있다.

### 버튼 내용 교체 예시

기존 코드:

```jsx
<PrimaryButton onPress={nextGuessHandler.bind(this, 'greater')}>
  +
</PrimaryButton>

<PrimaryButton onPress={nextGuessHandler.bind(this, 'lower')}>
  -
</PrimaryButton>
```

변경 후 코드:

```jsx
<PrimaryButton onPress={nextGuessHandler.bind(this, 'greater')}>
  <Ionicons name="md-add" size={24} color="white" />
</PrimaryButton>

<PrimaryButton onPress={nextGuessHandler.bind(this, 'lower')}>
  <Ionicons name="md-remove" size={24} color="white" />
</PrimaryButton>
```

동작은 그대로 유지된다.

달라지는 것은 버튼 안에 표시되는 콘텐츠뿐이다.

```text
onPress 로직
-> 그대로 유지

버튼 레이아웃
-> 그대로 유지

버튼 내부 표시
-> 문자에서 아이콘으로 변경
```

### 아이콘 검색 방법

원하는 아이콘 이름은 Expo Vector Icons 문서나 아이콘 목록에서 찾을 수 있다.

예를 들어 마이너스 아이콘을 찾을 때는 다음 키워드를 검색할 수 있다.

```text
minus
remove
subtract
```

Ionicons에서는 `md-remove`를 사용할 수 있다.

플러스 아이콘은 `md-add`를 사용할 수 있다.

```text
Ionicons
-> md-add
-> md-remove
```

아이콘 이름은 자동 완성이 도움을 줄 때도 있다.

하지만 최종적으로는 사용하는 아이콘 세트의 문서를 확인하는 것이 가장 정확하다.

### 플랫폼별 아이콘에 대한 예고

이번 강의에서는 Android와 iOS 모두 같은 아이콘을 사용한다.

```text
iOS
-> md-add
-> md-remove

Android
-> md-add
-> md-remove
```

하지만 실제 앱에서는 플랫폼별로 다른 아이콘이나 스타일을 쓰고 싶을 수 있다.

```text
iOS
-> iOS 스타일 아이콘

Android
-> Material Design 느낌의 아이콘
```

플랫폼별 코드는 이후 섹션에서 다룬다.

이번 단계에서는 하나의 아이콘 세트를 양쪽 플랫폼에 동일하게 적용한다.

### 커스텀 폰트는 다음 단계

강의 앞부분에서는 두 가지 개선 사항을 예고했다.

```text
1. 버튼 아이콘 교체
2. 커스텀 글꼴 적용
```

이번 강의에서 실제로 구현한 것은 아이콘 교체다.

커스텀 폰트는 다음 흐름에서 이어서 다룰 내용이다.

```text
이번 강의
-> Expo Vector Icons
-> Ionicons
-> + / - 버튼 아이콘화

다음 흐름
-> 커스텀 폰트 로딩
-> 앱 전체 텍스트 스타일 개선
```

### 프론트엔드 개발자 관점에서의 핵심 차이

웹 프론트엔드에서는 아이콘을 보통 다음 방식으로 사용한다.

```text
SVG 파일 import
icon font
react-icons
Material Icons
CSS background-image
```

React Native + Expo에서는 `@expo/vector-icons`를 사용하면 비슷한 경험을 얻을 수 있다.

```jsx
import { Ionicons } from '@expo/vector-icons';
```

그리고 JSX에서 컴포넌트처럼 렌더링한다.

```jsx
<Ionicons name="md-add" size={24} color="white" />
```

웹의 `react-icons`를 써본 개발자라면 매우 비슷하게 느껴질 수 있다.

```text
웹 React
-> <FaPlus />
-> <MdRemove />

React Native Expo
-> <Ionicons name="md-add" />
-> <Ionicons name="md-remove" />
```

차이는 아이콘 세트와 prop 이름 정도다.

컴포넌트를 조합해 UI를 만드는 감각은 동일하다.

### 실무 관점에서의 장점

아이콘을 텍스트 문자로 처리하는 것보다 아이콘 컴포넌트를 사용하는 편이 더 안정적이다.

```text
문자 + / -
-> 플랫폼 글꼴에 따라 모양 차이
-> 시각적 완성도 낮음

아이콘 컴포넌트
-> 일관된 형태
-> size와 color 제어 쉬움
-> 버튼 의미가 더 명확함
```

특히 모바일 앱에서는 아이콘 버튼이 자주 사용된다.

`@expo/vector-icons`를 익혀두면 탭, 헤더, 버튼, 목록 아이템 등 다양한 UI에서 활용할 수 있다.

## 정리

이번 강의에서는 게임 화면의 `+`, `-` 문자 버튼을 Expo의 `@expo/vector-icons`를 사용해 실제 아이콘으로 교체했다. `Ionicons`를 import하고, `md-add`, `md-remove` 아이콘을 `PrimaryButton` 내부에 렌더링했다.

아이콘 컴포넌트는 `name`, `size`, `color` prop으로 어떤 아이콘을 어떤 크기와 색상으로 보여줄지 제어한다. `PrimaryButton`은 `children`을 렌더링하므로 문자열 대신 아이콘 컴포넌트를 넣어도 기존 버튼 구조와 이벤트 로직은 그대로 유지된다.

커스텀 폰트 적용은 이번 강의에서 예고만 되었고, 실제 구현은 다음 흐름에서 이어진다.
