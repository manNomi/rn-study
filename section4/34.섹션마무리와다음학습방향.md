# 34. 섹션 마무리와 다음 학습 방향

## 핵심 요약

- 숫자 맞히기 게임 앱이 완성되었다.
- 이번 섹션에서는 React Native의 기본기를 실제 앱 하나에 반복 적용했다.
- 내장 핵심 컴포넌트, 스타일링, 커스텀 컴포넌트, 상태 관리, 화면 전환, `useEffect`, 목록 렌더링 등을 종합적으로 연습했다.
- `State`와 조건부 렌더링만으로도 간단한 앱에서는 여러 화면을 전환할 수 있다.
- 커스텀 버튼, 카드, 제목, 안내 문구, 숫자 컨테이너, 로그 항목 등 재사용 가능한 컴포넌트를 직접 만들었다.
- 아직 개선 여지는 많으며, 더 많은 컴포넌트 추출과 스타일 병합 패턴을 적용할 수 있다.
- 다음 섹션에서는 다양한 화면 크기, 플랫폼 차이, 기기 방향 전환에 대응하는 방법을 배운다.

## 학습 목표

- 이번 섹션에서 다룬 React Native 핵심 개념을 정리한다.
- 게임 앱을 만들며 어떤 React 지식이 그대로 사용되었는지 확인한다.
- 웹 프론트엔드 개발자가 React Native에서 특히 익숙해져야 할 부분을 구분한다.
- 완성된 앱에서 추가로 개선할 수 있는 지점을 파악한다.
- 다음 섹션의 주제인 반응형 레이아웃, 플랫폼 대응, 가로/세로 모드 대응을 예고한다.

## 주요 내용

### 완성된 앱의 의미

이번 섹션에서는 단순한 예제 조각이 아니라 작은 게임 앱 하나를 끝까지 완성했다.

앱의 흐름은 다음과 같다.

```text
1. 사용자가 1부터 99 사이의 숫자를 입력한다.
2. 앱이 임의의 숫자를 추측한다.
3. 사용자는 추측값이 더 커야 하는지 작아야 하는지 힌트를 준다.
4. 앱은 힌트를 바탕으로 다음 추측값을 계산한다.
5. 추측 기록을 목록으로 보여준다.
6. 정답을 맞히면 게임 오버 화면으로 이동한다.
7. 게임 결과와 라운드 수를 보여주고 새 게임을 시작할 수 있다.
```

앱 자체는 간단하지만, React Native 앱을 만들 때 필요한 핵심 흐름이 꽤 많이 들어 있다.

### 핵심 컴포넌트 복습

이번 섹션에서도 React Native의 내장 핵심 컴포넌트를 계속 사용했다.

```text
View
-> 레이아웃과 컨테이너 구성

Text
-> 텍스트 출력

TextInput
-> 사용자 입력 처리

Pressable
-> 터치 가능한 커스텀 버튼 구현

Image / ImageBackground
-> 이미지와 배경 이미지 출력

FlatList
-> 추측 로그 목록 렌더링

SafeAreaView
-> 노치와 상태바를 피하는 안전 영역 처리
```

웹에서 `div`, `button`, `input`, `img`를 조합하듯이, React Native에서는 이런 핵심 컴포넌트를 조합해 UI를 만든다.

차이는 HTML 태그를 사용할 수 없고, 반드시 React Native가 제공하는 컴포넌트를 사용해야 한다는 점이다.

### 스타일링과 레이아웃 연습

이번 앱에서는 다양한 스타일링을 다뤘다.

```text
StyleSheet.create
-> 스타일 객체 정의

Flexbox
-> 화면 배치와 버튼 행 구성

borderRadius / overflow
-> 둥근 이미지와 버튼 리플 영역 처리

elevation
-> Android 그림자

shadow*
-> iOS 그림자

fontFamily
-> 커스텀 폰트 적용

colors.js
-> 색상 상수 관리
```

React Native 스타일은 CSS처럼 보이지만 CSS는 아니다.

따라서 웹과 다른 점을 계속 의식해야 한다.

```text
CSS와 비슷한 점
-> margin, padding, color, fontSize 등 익숙한 속성이 많음

CSS와 다른 점
-> 계단식 상속이 거의 없음
-> 일부 속성은 플랫폼별로 다름
-> 스타일 값은 JavaScript 객체로 작성
-> kebab-case가 아니라 camelCase 사용
```

### 커스텀 컴포넌트 구축

이번 섹션에서는 앱 안에서 반복되는 UI를 직접 컴포넌트로 분리했다.

```text
PrimaryButton
-> Pressable 기반 커스텀 버튼

Title
-> 화면 제목

Card
-> 공통 카드 컨테이너

InstructionText
-> 안내 문구

NumberContainer
-> 추측 숫자 표시

GuessLogItem
-> 추측 로그 항목
```

이 패턴은 웹 React와 거의 같다.

```jsx
function Title({ children }) {
  return <Text style={styles.title}>{children}</Text>;
}
```

```jsx
<Title>Guess My Number</Title>
```

다른 점은 컴포넌트 내부에서 반환하는 요소가 HTML이 아니라 `View`, `Text`, `Pressable` 같은 React Native 컴포넌트라는 것이다.

### 일반 UI 컴포넌트와 앱 전용 컴포넌트

이번 섹션에서는 두 종류의 컴포넌트를 구분해서 만들었다.

```text
일반 UI 컴포넌트
-> PrimaryButton
-> Title
-> Card
-> InstructionText

앱 전용 컴포넌트
-> NumberContainer
-> GuessLogItem
```

일반 UI 컴포넌트는 다른 앱이나 다른 화면에서도 재사용하기 쉽다.

반면 앱 전용 컴포넌트는 이번 게임의 도메인에 더 밀접하다.

```text
NumberContainer
-> 게임에서 추측 숫자를 보여주는 역할

GuessLogItem
-> 특정 라운드의 추측 기록을 보여주는 역할
```

프론트엔드 실무에서도 이런 구분은 중요하다.

```text
공통 UI
-> 디자인 시스템에 가까움

도메인 컴포넌트
-> 특정 기능과 데이터 흐름에 가까움
```

### 상태 기반 화면 전환

이번 앱에서는 별도의 내비게이션 라이브러리를 사용하지 않았다.

대신 `App` 컴포넌트의 상태와 조건문으로 화면을 전환했다.

```text
userNumber 없음
-> StartGameScreen

userNumber 있음 + gameIsOver false
-> GameScreen

userNumber 있음 + gameIsOver true
-> GameOverScreen
```

코드 구조는 다음과 비슷하다.

```jsx
let screen = <StartGameScreen onPickNumber={pickedNumberHandler} />;

if (userNumber) {
  screen = <GameScreen userNumber={userNumber} onGameOver={gameOverHandler} />;
}

if (gameIsOver && userNumber) {
  screen = (
    <GameOverScreen
      roundsNumber={guessRounds}
      userNumber={userNumber}
      onStartNewGame={startNewGameHandler}
    />
  );
}
```

간단한 앱에서는 이 방식으로도 충분하다.

하지만 화면이 많아지고 뒤로 가기, 탭, 스택, 딥링크 등이 필요해지면 나중에 배울 내비게이션 라이브러리를 사용하는 것이 좋다.

### React 핵심 기능은 그대로 사용된다

이번 섹션은 React Native 앱이지만, 많은 부분은 일반 React와 동일했다.

```text
useState
-> 입력값, 게임 상태, 추측값, 라운드 기록 관리

useEffect
-> currentGuess가 userNumber와 같아졌는지 감지

props
-> 하위 컴포넌트에 데이터와 콜백 전달

children
-> Title, Card, PrimaryButton 같은 래퍼 컴포넌트 구현

조건부 렌더링
-> 화면 전환

배열 상태 업데이트
-> 추측 로그 추가
```

즉 React Native는 React의 정신 모델을 유지한다.

다만 렌더링 대상이 브라우저 DOM이 아니라 iOS와 Android의 네이티브 UI라는 점이 다르다.

### InstructionText의 스타일 병합 패턴

이번 섹션에서 중요한 재사용 패턴 중 하나는 `InstructionText`에서 사용한 스타일 병합이다.

기본 스타일을 컴포넌트 내부에 두고, 외부에서 전달한 스타일을 배열로 합쳤다.

```jsx
function InstructionText({ children, style }) {
  return (
    <Text style={[styles.instructionText, style]}>
      {children}
    </Text>
  );
}
```

이렇게 하면 기본 스타일을 유지하면서 특정 화면에서만 여백이나 크기를 추가할 수 있다.

```jsx
<InstructionText style={styles.instructionText}>
  Higher or lower?
</InstructionText>
```

웹 CSS의 cascade와 완전히 같지는 않지만, React Native에서 비슷한 유연성을 얻는 실용적인 방법이다.

### 아직 개선할 수 있는 부분

강의에서는 모든 코드를 최대한 최적화하지는 않았다.

그 이유는 두 가지다.

```text
1. 핵심 개념과 스타일을 명확히 보여주기 위해
2. 섹션에서 소개할 React Native 기능이 많았기 때문에
```

완성된 앱에도 개선 여지는 많다.

예를 들어 다음과 같은 작업을 해볼 수 있다.

```text
1. 더 많은 컴포넌트 추출
2. 중복 스타일 정리
3. 버튼 행 컴포넌트 분리
4. 게임 로직 유틸 함수 정리
5. 색상뿐 아니라 spacing, radius 같은 디자인 토큰 추가
6. 플랫폼별 스타일 분리
7. 작은 화면과 큰 화면에서 레이아웃 보정
```

이런 개선은 학습자가 직접 연습하기 좋은 지점이다.

## 프론트엔드 개발자 관점에서의 핵심 정리

웹 React를 아는 프론트엔드 개발자라면 이번 섹션에서 다음 감각을 잡는 것이 중요하다.

### React 지식은 그대로 가져간다

상태, props, children, 조건부 렌더링, effect, 리스트 렌더링은 기존 React 지식 그대로 사용할 수 있다.

```text
React 지식
-> 거의 그대로 유효

DOM 지식
-> 직접적으로는 사용할 수 없음
```

### UI 구성 요소가 다르다

웹에서는 HTML 태그를 쓰지만, React Native에서는 핵심 컴포넌트를 쓴다.

```text
div
-> View

p / span / h1
-> Text

input
-> TextInput

button
-> Button 또는 Pressable 기반 커스텀 버튼

img
-> Image
```

### 레이아웃은 더 명시적으로 잡아야 한다

웹 브라우저의 기본 문서 흐름에 기대던 부분을 React Native에서는 더 직접적으로 제어해야 한다.

```text
스크롤
-> FlatList / ScrollView 명시

안전 영역
-> SafeAreaView 고려

공간 분배
-> Flexbox 적극 사용

화면 크기 차이
-> 다음 섹션에서 더 자세히 다룸
```

### 플랫폼 차이를 받아들여야 한다

React Native는 하나의 코드베이스로 iOS와 Android 앱을 만들 수 있게 해주지만, 두 플랫폼이 완전히 같지는 않다.

```text
Android
-> elevation으로 그림자
-> android_ripple 지원

iOS
-> shadow* 속성으로 그림자
-> Pressable style 함수로 눌림 피드백 구성
```

이런 차이를 자연스럽게 다루는 것이 React Native 실력의 일부다.

## 다음 섹션 예고

다음 핵심 섹션에서는 이번 게임 앱을 기반으로 더 현실적인 모바일 대응을 배운다.

주요 주제는 다음과 같다.

```text
1. 다양한 화면 크기에 맞는 레이아웃
2. iOS와 Android 플랫폼별 스타일 또는 코드 분기
3. 세로 모드와 가로 모드 대응
4. 기기 회전 시 UI가 깨지지 않도록 조정
5. 더 실전적인 반응형 React Native 레이아웃
```

이번 섹션에서 만든 앱은 다음 섹션의 출발점이 된다.

즉 지금까지 만든 구조를 바탕으로 “모든 기기에서 보기 좋은 앱”으로 발전시키는 과정이 이어진다.

## 정리

이번 섹션에서는 숫자 맞히기 게임 앱을 완성하면서 React Native의 핵심 기능을 폭넓게 연습했다.

핵심 컴포넌트, 스타일링, 커스텀 컴포넌트, 공통 색상 상수, 커스텀 폰트, 아이콘, 이미지, 배경 이미지, 목록 렌더링, 상태 기반 화면 전환, `useEffect` 기반 게임 종료 처리까지 작은 앱 하나 안에 다양한 개념을 담았다.

이제 중요한 것은 직접 변형해 보는 것이다.

스타일을 바꾸고, 컴포넌트를 더 나누고, 다른 레이아웃을 시도하고, 공식 문서를 보며 핵심 컴포넌트의 옵션을 살펴보면 React Native에 훨씬 빨리 익숙해질 수 있다.
