# 26. Expo Font로 커스텀 글꼴 적용

## 핵심 요약

- React Native 앱에서도 기본 시스템 글꼴 대신 커스텀 글꼴을 사용할 수 있다.
- Expo에서는 `expo-font` 패키지의 `useFonts` 훅으로 글꼴 파일을 로딩한다.
- 커스텀 글꼴은 앱 시작 시점에 로딩하는 것이 일반적이므로 루트 컴포넌트인 `App`에서 처리한다.
- 로컬 글꼴 파일은 보통 `assets/fonts` 폴더에 넣고 `require()`로 불러온다.
- 글꼴 로딩이 끝나기 전에는 `AppLoading` 같은 로딩 컴포넌트를 렌더링해 앱 화면이 먼저 뜨지 않도록 한다.
- 로딩된 글꼴은 `fontFamily` 스타일 프로퍼티에 등록한 식별자 이름으로 적용한다.
- 굵은 글꼴 파일을 별도로 로딩했다면 `fontWeight` 대신 해당 `fontFamily`를 사용하는 방식이 자연스럽다.

## 학습 목표

- Expo 프로젝트에서 커스텀 폰트를 로딩하는 전체 흐름을 이해한다.
- `expo-font`의 `useFonts` 훅 사용법을 익힌다.
- 로컬 폰트 파일을 `assets/fonts`에 저장하고 `require()`로 연결하는 방법을 배운다.
- 폰트 로딩 완료 전 앱 화면을 지연시키는 이유를 이해한다.
- `fontFamily`로 로딩된 폰트를 실제 컴포넌트 스타일에 적용하는 방법을 익힌다.
- 웹의 `@font-face`와 React Native의 폰트 로딩 방식 차이를 비교한다.

## 주요 내용

### 왜 커스텀 글꼴을 사용하는가

React Native 앱은 기본적으로 각 플랫폼의 시스템 글꼴을 사용한다.

기본 글꼴도 나쁘지는 않지만, 앱의 분위기나 브랜드를 더 잘 표현하려면 커스텀 글꼴이 필요할 수 있다.

```text
기본 시스템 글꼴
-> 빠르게 사용 가능
-> 플랫폼별로 모양이 다를 수 있음

커스텀 글꼴
-> 앱 전체의 시각적 톤을 통일
-> 브랜드에 맞는 typography 적용 가능
```

이번 강의에서는 OpenSans 글꼴을 앱에 추가한다.

### Expo에서 폰트 로딩하기

Expo에서는 `expo-font` 패키지를 사용해 글꼴을 로딩한다.

먼저 패키지를 설치한다.

```bash
expo install expo-font
```

강의에서는 별도 터미널 탭에서 설치하고, 기존 `npm start` 프로세스는 그대로 실행해 둔다.

설치 후 `App.js`에서 `useFonts` 훅을 가져온다.

```jsx
import { useFonts } from 'expo-font';
```

`useFonts`는 로딩할 글꼴 목록을 받고, 글꼴 로딩 상태를 반환한다.

### 왜 App 컴포넌트에서 로딩하는가

글꼴은 앱 전반에서 사용할 가능성이 높다.

따라서 특정 화면에서 따로 로딩하기보다 루트 컴포넌트인 `App`에서 로딩하는 것이 좋다.

```text
App
-> 앱 시작 시 폰트 로딩
-> 모든 화면에서 같은 폰트 사용 가능
```

만약 개별 컴포넌트마다 폰트를 로딩하면 중복이 생기고, 화면마다 글꼴 적용 시점이 달라질 수 있다.

```text
권장 흐름
-> 앱 시작
-> 폰트 로딩
-> 로딩 완료 후 실제 화면 렌더링
```

### 로컬 폰트 파일 추가하기

강의에서는 로컬 폰트 파일을 사용한다.

프로젝트의 `assets` 폴더 아래에 `fonts` 폴더를 만든다.

```text
assets/
  fonts/
    OpenSans-Regular.ttf
    OpenSans-Bold.ttf
```

강의에 첨부된 두 파일을 이 위치에 넣는다.

```text
OpenSans-Regular.ttf
-> 일반 두께 글꼴

OpenSans-Bold.ttf
-> 굵은 두께 글꼴
```

파일명은 코드에서 그대로 참조하므로 오타가 없어야 한다.

### useFonts로 글꼴 등록하기

`useFonts`에 객체를 전달해 사용할 글꼴을 등록한다.

```jsx
const [fontsLoaded] = useFonts({
  'open-sans': require('./assets/fonts/OpenSans-Regular.ttf'),
  'open-sans-bold': require('./assets/fonts/OpenSans-Bold.ttf'),
});
```

객체의 key는 앱에서 사용할 글꼴 식별자다.

```text
'open-sans'
-> OpenSans-Regular.ttf를 가리키는 이름

'open-sans-bold'
-> OpenSans-Bold.ttf를 가리키는 이름
```

식별자에 대시가 들어가므로 문자열 key로 작성한다.

```jsx
{
  'open-sans': ...
}
```

그냥 `open-sans`처럼 쓰면 유효한 JavaScript 프로퍼티 이름이 아니므로 오류가 난다.

### require로 폰트 파일 연결하기

로컬 이미지 파일을 불러올 때와 비슷하게, 폰트 파일도 `require()`로 연결한다.

```jsx
require('./assets/fonts/OpenSans-Regular.ttf')
```

```jsx
require('./assets/fonts/OpenSans-Bold.ttf')
```

경로는 `App.js` 파일 기준의 상대 경로다.

```text
App.js
-> assets
-> fonts
-> OpenSans-Regular.ttf
```

React Native에서는 웹처럼 문자열 URL만 적는 방식보다 `require()`를 통해 정적 자산을 연결하는 경우가 많다.

### 폰트 로딩 상태 확인하기

`useFonts`는 배열을 반환한다.

첫 번째 값은 폰트가 모두 로딩되었는지를 나타내는 boolean이다.

```jsx
const [fontsLoaded] = useFonts({
  'open-sans': require('./assets/fonts/OpenSans-Regular.ttf'),
  'open-sans-bold': require('./assets/fonts/OpenSans-Bold.ttf'),
});
```

```text
fontsLoaded = false
-> 아직 폰트 로딩 중

fontsLoaded = true
-> 폰트 로딩 완료
```

폰트가 로딩되기 전에 실제 앱 화면을 렌더링하면 기본 글꼴이 잠깐 보이거나 스타일이 예상과 다르게 보일 수 있다.

따라서 로딩이 끝날 때까지 앱 화면 렌더링을 지연시킨다.

### AppLoading으로 스플래시 화면 유지하기

강의에서는 `expo-app-loading` 패키지를 사용한다.

```bash
expo install expo-app-loading
```

설치 후 `AppLoading` 컴포넌트를 가져온다.

```jsx
import AppLoading from 'expo-app-loading';
```

폰트가 아직 로딩되지 않았다면 `AppLoading`을 반환한다.

```jsx
if (!fontsLoaded) {
  return <AppLoading />;
}
```

이렇게 하면 폰트 로딩이 끝날 때까지 앱의 스플래시 화면이 유지된다.

```text
fontsLoaded false
-> AppLoading 렌더링
-> 실제 앱 화면 표시 지연

fontsLoaded true
-> 기존 앱 UI 렌더링
```

결과적으로 사용자는 폰트가 준비된 화면을 바로 보게 된다.

### App.js 전체 흐름 예시

이번 단계의 `App.js` 폰트 로딩 흐름은 다음처럼 정리할 수 있다.

```jsx
import AppLoading from 'expo-app-loading';
import { useFonts } from 'expo-font';

export default function App() {
  const [fontsLoaded] = useFonts({
    'open-sans': require('./assets/fonts/OpenSans-Regular.ttf'),
    'open-sans-bold': require('./assets/fonts/OpenSans-Bold.ttf'),
  });

  if (!fontsLoaded) {
    return <AppLoading />;
  }

  return (
    // 기존 앱 화면
  );
}
```

이제 앱 UI가 렌더링되는 시점에는 커스텀 글꼴이 준비되어 있다.

### fontFamily로 글꼴 적용하기

글꼴을 로딩했다고 자동으로 모든 텍스트에 적용되는 것은 아니다.

사용하려는 텍스트 스타일에 `fontFamily`를 지정해야 한다.

```jsx
fontFamily: 'open-sans'
```

```jsx
fontFamily: 'open-sans-bold'
```

이 이름은 `useFonts`에 등록한 key와 정확히 일치해야 한다.

```text
useFonts에 등록한 이름
-> 'open-sans'
-> 'open-sans-bold'

스타일에서 사용하는 이름
-> fontFamily: 'open-sans'
-> fontFamily: 'open-sans-bold'
```

오타가 있으면 폰트가 적용되지 않는다.

### Title에 굵은 글꼴 적용하기

`Title` 컴포넌트에는 굵은 글꼴을 적용한다.

```jsx
const styles = StyleSheet.create({
  title: {
    fontFamily: 'open-sans-bold',
    fontSize: 24,
    color: 'white',
    textAlign: 'center',
    borderWidth: 2,
    borderColor: Colors.accent500,
    padding: 12,
  },
});
```

굵은 폰트 파일 자체를 로딩했으므로 `fontWeight: 'bold'`는 제거할 수 있다.

```text
기존
-> fontWeight: 'bold'

변경
-> fontFamily: 'open-sans-bold'
```

두께가 다른 폰트 파일을 각각 등록해 사용하는 방식이다.

### InstructionText에 일반 글꼴 적용하기

`InstructionText`에는 일반 글꼴을 적용한다.

```jsx
const styles = StyleSheet.create({
  instructionText: {
    fontFamily: 'open-sans',
    color: Colors.accent500,
    fontSize: 24,
  },
});
```

안내 문구는 굵게 강조하기보다 읽기 좋게 일반 글꼴로 두는 흐름이다.

```text
Title
-> open-sans-bold

InstructionText
-> open-sans
```

### NumberContainer에 굵은 글꼴 적용하기

`NumberContainer`의 숫자 텍스트는 크게 강조되는 요소다.

따라서 굵은 글꼴을 적용한다.

```jsx
const styles = StyleSheet.create({
  numberText: {
    color: Colors.accent500,
    fontSize: 36,
    fontFamily: 'open-sans-bold',
  },
});
```

기존 `fontWeight: 'bold'`는 제거하거나 주석 처리할 수 있다.

```text
numberText
-> 큰 숫자
-> 굵은 OpenSans 적용
```

### fontWeight와 fontFamily의 관계

웹에서는 `font-family`와 `font-weight`를 함께 쓰는 일이 흔하다.

```css
font-family: 'Open Sans';
font-weight: 700;
```

React Native에서도 `fontWeight`를 사용할 수 있지만, 로컬 폰트 파일을 직접 등록할 때는 두께별 파일을 각각 등록해 사용하는 방식이 명확하다.

```text
OpenSans-Regular.ttf
-> fontFamily: 'open-sans'

OpenSans-Bold.ttf
-> fontFamily: 'open-sans-bold'
```

즉 굵은 글씨가 필요하면 `fontWeight`로 조정하기보다 굵은 폰트 식별자를 사용한다.

### Google Fonts를 사용하는 방법도 있다

Expo 공식 문서에서는 Google Fonts를 쉽게 사용하는 방법도 제공한다.

```text
@expo-google-fonts
```

이 패키지를 사용하면 Google Fonts의 특정 글꼴을 더 편하게 불러올 수 있다.

하지만 강의에서는 더 일반적인 상황을 보여주기 위해 로컬 폰트 파일을 직접 추가하는 방식을 사용한다.

```text
Google Fonts 패키지
-> 특정 Google 폰트를 쉽게 사용

로컬 폰트 파일
-> 직접 보유한 .ttf 파일 사용 가능
-> 브랜드 폰트나 커스텀 폰트에 적합
```

로컬 파일 방식을 이해하면 어떤 글꼴 파일이든 비슷하게 적용할 수 있다.

### 프론트엔드 개발자 관점에서의 핵심 차이

웹에서는 보통 `@font-face`나 외부 CSS 링크로 글꼴을 로딩한다.

```css
@font-face {
  font-family: 'Open Sans';
  src: url('/fonts/OpenSans-Regular.ttf');
}
```

React Native에서는 JavaScript에서 폰트 자산을 로딩하고, 로딩 완료 여부를 상태처럼 확인한다.

```jsx
const [fontsLoaded] = useFonts({
  'open-sans': require('./assets/fonts/OpenSans-Regular.ttf'),
});
```

그리고 스타일 객체에서 `fontFamily`를 사용한다.

```jsx
fontFamily: 'open-sans'
```

핵심 차이는 다음과 같다.

```text
웹
-> CSS가 폰트 로딩 선언을 담당
-> 브라우저가 렌더링 과정에서 폰트 적용

React Native
-> JS에서 폰트 파일을 로딩
-> 로딩 완료 전 화면 표시를 제어
-> StyleSheet에서 fontFamily 지정
```

React Native에서는 폰트 로딩이 비동기 작업이라는 점을 더 명시적으로 다룬다.

### 로딩 완료 전 화면을 막는 이유

폰트가 로딩되기 전에 텍스트를 렌더링하면 다음 문제가 생길 수 있다.

```text
기본 폰트가 잠깐 보임
커스텀 폰트 적용 후 레이아웃이 미세하게 바뀜
초기 화면이 덜 완성된 상태로 노출됨
```

그래서 앱 시작 시점에는 다음 순서를 유지한다.

```text
1. 앱 실행
2. 폰트 로딩 시작
3. 로딩 중에는 스플래시/로딩 화면 유지
4. 폰트 로딩 완료
5. 실제 앱 UI 렌더링
```

이런 흐름은 사용자에게 더 안정적인 첫 화면을 제공한다.

## 정리

이번 강의에서는 Expo 프로젝트에서 커스텀 글꼴을 사용하는 방법을 배웠다. `expo-font`의 `useFonts` 훅으로 `OpenSans-Regular.ttf`, `OpenSans-Bold.ttf` 파일을 로딩하고, 로딩이 끝나기 전에는 `AppLoading`을 렌더링해 스플래시 화면을 유지했다.

로딩된 글꼴은 `fontFamily` 스타일 프로퍼티로 사용한다. `Title`과 `NumberContainer`에는 `open-sans-bold`를, `InstructionText`에는 `open-sans`를 적용했다. 굵은 폰트 파일을 별도로 등록했으므로 `fontWeight` 대신 해당 폰트 식별자를 사용하는 방식으로 정리했다.

웹의 `@font-face`와 비슷한 목적이지만, React Native에서는 JavaScript에서 폰트 파일을 직접 로딩하고 로딩 완료 여부에 따라 화면 렌더링을 제어한다는 점이 다르다.
