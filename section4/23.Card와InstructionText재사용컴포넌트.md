# 23. Card와 InstructionText 재사용 컴포넌트

## 핵심 요약

- 시작 화면에는 앱 제목과 입력 영역 안내 문구를 추가해 화면 구성을 더 명확하게 만든다.
- `Title` 컴포넌트를 재사용해 `Guess My Number` 제목을 시작 화면에도 표시한다.
- 시작 화면의 입력 박스 스타일은 게임 화면의 버튼 영역에도 재사용할 수 있으므로 `Card` 컴포넌트로 분리한다.
- 반복되는 안내 문구 스타일은 `InstructionText` 컴포넌트로 분리한다.
- 재사용 컴포넌트를 만들 때는 내부 콘텐츠를 `children`으로 받아 유연하게 렌더링한다.
- 파일을 `components/ui`로 옮기면 import 경로와 누락된 React Native 컴포넌트 import를 함께 점검해야 한다.

## 학습 목표

- 기존 화면을 개선하면서 반복되는 UI 패턴을 찾아내는 방법을 익힌다.
- `Title`, `Card`, `InstructionText`처럼 재사용 가능한 UI 컴포넌트를 구성하는 기준을 이해한다.
- 컨테이너 스타일과 콘텐츠를 분리해 컴포넌트를 더 유연하게 만드는 방법을 배운다.
- `children` prop을 활용해 래퍼 컴포넌트를 만드는 패턴을 익힌다.
- 컴포넌트 이동 후 import 경로와 누락된 import 오류를 해결하는 흐름을 이해한다.

## 주요 내용

### 시작 화면에 제목 추가하기

게임 시작 화면은 숫자를 입력하는 기능은 갖췄지만, 화면의 제목이 없으면 무엇을 하는 앱인지 한눈에 들어오지 않는다.

이미 이전 강의에서 만든 `Title` 컴포넌트를 재사용할 수 있다.

```jsx
import Title from '../components/ui/Title';
```

`StartGameScreen`의 입력 카드 위에 제목을 추가한다.

```jsx
<Title>Guess My Number</Title>
```

이렇게 하면 기존에 만든 제목 스타일을 그대로 사용하면서 시작 화면에도 앱 제목을 표시할 수 있다.

```text
Title 재사용
-> GameScreen: Opponent's Guess
-> StartGameScreen: Guess My Number
```

### rootContainer로 시작 화면 레이아웃 조정

제목을 추가하면 화면 맨 위에 붙어 보일 수 있다.

이를 조정하기 위해 `StartGameScreen`의 전체 내용을 감싸는 루트 `View`를 추가한다.

```jsx
<View style={styles.rootContainer}>
  <Title>Guess My Number</Title>

  {/* 입력 영역 */}
</View>
```

그리고 `rootContainer` 스타일을 정의한다.

```jsx
rootContainer: {
  flex: 1,
  marginTop: 100,
  alignItems: 'center',
}
```

각 스타일의 역할은 다음과 같다.

```text
flex: 1
-> 가능한 공간 전체를 사용

marginTop: 100
-> 상단에서 충분히 떨어뜨림

alignItems: 'center'
-> 교차축 기준 중앙 정렬
```

기본적으로 `View`의 `alignItems`는 `stretch`처럼 동작해 자식 요소가 가로로 늘어날 수 있다.

`center`로 바꾸면 제목이 화면 너비 전체로 늘어나지 않고 콘텐츠 크기에 맞춰 중앙에 배치된다.

### inputContainer 위치 조정

루트 컨테이너에 상단 여백을 주었으므로, 기존 입력 박스의 `marginTop`은 줄인다.

```jsx
inputContainer: {
  marginTop: 36,
}
```

이렇게 하면 제목과 입력 영역 사이에 적당한 간격이 생긴다.

```text
rootContainer marginTop
-> 화면 전체 상단 위치 조정

inputContainer marginTop
-> 제목과 입력 박스 사이 간격 조정
```

화면 전체 위치와 내부 요소 간격을 나누어 관리하는 것이 좋다.

### 입력 영역 안내 문구 추가

숫자 입력 필드 위에 안내 문구를 추가한다.

```jsx
<Text>Enter a Number</Text>
```

이 텍스트는 사용자가 무엇을 입력해야 하는지 알려준다.

하지만 기본 텍스트 상태로는 잘 보이지 않으므로 스타일을 추가한다.

```jsx
instructionText: {
  color: Colors.accent500,
  fontSize: 24,
}
```

처음에는 `primary500` 같은 어두운 색상을 사용할 수도 있지만, 배경과 대비가 약하면 잘 보이지 않는다.

입력 필드의 강조색과 맞춰 `Colors.accent500`을 사용하면 화면의 일관성이 좋아진다.

```text
Instruction text
-> 노란색 강조 색상
-> 입력 필드 색상과 연결
```

### 반복되는 박스 UI 발견하기

시작 화면의 입력 영역은 카드 형태의 박스다.

```text
배경색
모서리 둥글게
padding
margin
shadow/elevation
```

게임 화면에서도 `Higher or lower?` 텍스트와 버튼 두 개를 같은 박스 안에 넣고 싶다.

이 경우 시작 화면의 입력 박스 스타일을 그대로 복사하는 것보다 재사용 컴포넌트로 분리하는 편이 좋다.

```text
StartGameScreen
-> 숫자 입력 카드

GameScreen
-> 피드백 버튼 카드

공통점
-> 같은 박스 스타일
```

이 공통 박스를 `Card` 컴포넌트로 만든다.

### Card 컴포넌트 만들기

`components/ui/Card.js` 파일을 만든다.

```text
components/
  ui/
    Card.js
```

`Card`는 특정 화면의 콘텐츠를 알 필요가 없다.

그저 자식 요소를 감싸고 카드 스타일을 적용하는 역할만 한다.

```jsx
import { StyleSheet, View } from 'react-native';

import Colors from '../../constants/colors';

function Card({ children }) {
  return (
    <View style={styles.card}>
      {children}
    </View>
  );
}

export default Card;
```

`children`을 사용하면 어떤 내용이든 카드 안에 넣을 수 있다.

```jsx
<Card>
  <TextInput />
  <PrimaryButton>Confirm</PrimaryButton>
</Card>
```

```jsx
<Card>
  <InstructionText>Higher or lower?</InstructionText>
  <PrimaryButton>+</PrimaryButton>
  <PrimaryButton>-</PrimaryButton>
</Card>
```

이런 형태가 래퍼 컴포넌트의 전형적인 패턴이다.

### Card 스타일 이동하기

기존 `StartGameScreen`의 `inputContainer` 스타일 중 카드의 외형을 담당하던 부분을 `Card`로 옮긴다.

```jsx
const styles = StyleSheet.create({
  card: {
    justifyContent: 'center',
    alignItems: 'center',
    marginTop: 36,
    marginHorizontal: 24,
    padding: 16,
    backgroundColor: Colors.primary800,
    borderRadius: 8,
    elevation: 4,
    shadowColor: 'black',
    shadowOffset: { width: 0, height: 2 },
    shadowRadius: 6,
    shadowOpacity: 0.25,
  },
});
```

정확한 스타일 값은 프로젝트 흐름에 따라 조정할 수 있다.

핵심은 카드의 외형과 관련된 스타일이 `Card.js` 안으로 이동한다는 점이다.

```text
Card가 책임지는 것
-> 배경색
-> 안쪽 여백
-> 둥근 모서리
-> 그림자
-> 자식 중앙 배치
```

반대로 `StartGameScreen`에만 필요한 레이아웃 값은 화면 컴포넌트에 남겨도 된다.

### StartGameScreen에서 Card 사용하기

기존에는 입력 영역을 `View`로 감쌌다.

```jsx
<View style={styles.inputContainer}>
  ...
</View>
```

이제 `Card`로 교체한다.

```jsx
import Card from '../components/ui/Card';
```

```jsx
<Card>
  <InstructionText>Enter a Number</InstructionText>
  <TextInput />
  {/* 버튼들 */}
</Card>
```

이렇게 하면 `StartGameScreen`은 카드 스타일의 세부사항을 몰라도 된다.

```text
StartGameScreen
-> 화면 구조 담당

Card
-> 카드 외형 담당
```

### GameScreen에서도 Card 사용하기

게임 화면의 피드백 영역도 같은 카드로 감싼다.

```jsx
import Card from '../components/ui/Card';
```

```jsx
<Card>
  <InstructionText>Higher or lower?</InstructionText>
  <View>
    <PrimaryButton onPress={...}>+</PrimaryButton>
    <PrimaryButton onPress={...}>-</PrimaryButton>
  </View>
</Card>
```

이제 시작 화면과 게임 화면의 박스 스타일이 일관된다.

```text
하나의 Card 컴포넌트
-> StartGameScreen 입력 영역
-> GameScreen 피드백 영역
```

스타일을 수정해야 할 때도 `Card.js`만 보면 된다.

### 반복되는 안내 문구 발견하기

시작 화면의 `Enter a Number`와 게임 화면의 `Higher or lower?`는 같은 성격의 텍스트다.

```text
Enter a Number
-> 입력 안내

Higher or lower?
-> 피드백 안내
```

두 텍스트 모두 같은 스타일을 사용하면 화면 전체의 톤이 통일된다.

따라서 이 안내 텍스트도 `InstructionText` 컴포넌트로 분리한다.

### InstructionText 컴포넌트 만들기

`components/ui/InstructionText.js` 파일을 만든다.

```jsx
import { StyleSheet, Text } from 'react-native';

import Colors from '../../constants/colors';

function InstructionText({ children }) {
  return (
    <Text style={styles.instructionText}>
      {children}
    </Text>
  );
}

export default InstructionText;
```

이 컴포넌트도 `children`을 사용한다.

```jsx
<InstructionText>Enter a Number</InstructionText>
```

```jsx
<InstructionText>Higher or lower?</InstructionText>
```

문구는 사용하는 쪽에서 전달하고, 스타일은 컴포넌트 내부에서 관리한다.

### InstructionText 스타일 이동하기

기존 `StartGameScreen`에 있던 `instructionText` 스타일을 `InstructionText.js`로 옮긴다.

```jsx
const styles = StyleSheet.create({
  instructionText: {
    color: Colors.accent500,
    fontSize: 24,
  },
});
```

이제 `StartGameScreen`과 `GameScreen`은 일반 `Text` 컴포넌트 대신 `InstructionText`를 사용한다.

```jsx
<InstructionText>Enter a Number</InstructionText>
```

```jsx
<InstructionText>Higher or lower?</InstructionText>
```

화면 컴포넌트에서 `Text`를 직접 사용하지 않게 되면 `Text` import도 제거할 수 있다.

### import 오류 해결하기

컴포넌트를 분리하거나 파일 위치를 옮기면 import 오류가 자주 발생한다.

이번 흐름에서도 몇 가지 오류가 날 수 있다.

```text
Colors is not defined
-> Colors를 import하지 않음

View is not defined
-> Card에서 View를 import하지 않음

Text is not defined
-> InstructionText에서 Text를 import하지 않음

잘못된 상대 경로
-> components/ui에서 constants까지 올라가는 경로가 틀림
```

예를 들어 `Card.js`와 `InstructionText.js`는 `components/ui` 폴더 안에 있다.

따라서 색상 상수 import 경로는 다음처럼 작성한다.

```jsx
import Colors from '../../constants/colors';
```

`../constants/colors`가 아니라 `../../constants/colors`다.

```text
components/ui/Card.js
-> components/ui
-> components
-> 프로젝트 루트
-> constants/colors
```

상대 경로를 잘못 쓰면 컴포넌트 구조는 맞아도 앱이 실행되지 않는다.

### 컴포넌트 분리 후 화면 구조

리팩터링 후 시작 화면은 더 읽기 쉬워진다.

```jsx
<View style={styles.rootContainer}>
  <Title>Guess My Number</Title>

  <Card>
    <InstructionText>Enter a Number</InstructionText>
    <TextInput />
    {/* buttons */}
  </Card>
</View>
```

게임 화면도 비슷하게 정리된다.

```jsx
<View style={styles.screen}>
  <Title>Opponent's Guess</Title>
  <NumberContainer>{currentGuess}</NumberContainer>

  <Card>
    <InstructionText>Higher or lower?</InstructionText>
    {/* + / - buttons */}
  </Card>
</View>
```

이제 각 컴포넌트의 역할이 더 선명하다.

```text
Title
-> 화면 제목

Card
-> 박스형 컨테이너

InstructionText
-> 안내 문구

NumberContainer
-> 추측 숫자 표시

PrimaryButton
-> 버튼
```

### 왜 이 리팩터링이 중요한가

이번 작업은 앱 기능을 크게 바꾸지는 않는다.

하지만 화면 컴포넌트의 복잡도를 줄이고 UI 일관성을 높인다.

```text
이전 구조
-> 화면마다 View, Text, 스타일 직접 작성
-> 중복 스타일 증가

개선 구조
-> 공통 UI 컴포넌트로 추출
-> 화면 컴포넌트는 조립에 집중
```

React 앱에서 화면 컴포넌트가 모든 UI 세부사항을 직접 가지면 금방 복잡해진다.

재사용 가능한 UI 단위를 분리하면 이후 스타일 변경과 화면 확장이 쉬워진다.

### 프론트엔드 개발자 관점에서의 핵심 차이

웹 React에서는 이런 구조가 익숙하다.

```jsx
<Card>
  <Heading>Guess My Number</Heading>
  <InputLabel>Enter a Number</InputLabel>
  <input />
</Card>
```

React Native에서도 개념은 같다.

다만 기반 요소가 HTML이 아니라 `View`, `Text`, `TextInput`이다.

```jsx
<Card>
  <InstructionText>Enter a Number</InstructionText>
  <TextInput />
</Card>
```

핵심은 “공통 스타일과 구조를 가진 UI를 컴포넌트로 추출한다”는 점이다.

```text
웹
-> div 기반 Card
-> CSS class 또는 styled component

React Native
-> View 기반 Card
-> StyleSheet 객체
```

컴포넌트 설계 감각은 동일하다.

### 실무 관점에서의 폴더 분리

이번 강의에서는 `components/ui`에 공통 UI 컴포넌트를 둔다.

```text
components/ui
-> Card
-> Title
-> InstructionText
-> PrimaryButton
```

게임 로직에 특화된 컴포넌트는 `components/game`에 둔다.

```text
components/game
-> NumberContainer
```

이런 분리는 작은 앱에서는 필수는 아니지만, 앱이 커질수록 도움이 된다.

```text
ui
-> 도메인과 무관한 재사용 UI

game
-> 게임 기능과 직접 관련된 UI
```

폴더 구조는 팀이나 프로젝트에 따라 달라질 수 있지만, 역할을 기준으로 나누는 기준은 유용하다.

## 정리

이번 강의에서는 시작 화면과 게임 화면의 UI를 더 깔끔하게 다듬고, 반복되는 UI를 재사용 컴포넌트로 분리했다. 시작 화면에는 `Title`을 재사용해 `Guess My Number` 제목을 추가했고, 입력 박스 위에는 `InstructionText`로 안내 문구를 추가했다.

또한 시작 화면의 입력 박스와 게임 화면의 피드백 박스가 같은 형태를 공유하므로 `Card` 컴포넌트를 만들었다. `Card`는 `children`을 받아 어떤 콘텐츠든 감쌀 수 있는 래퍼 컴포넌트이며, 카드의 배경색, padding, borderRadius, shadow 같은 스타일을 한곳에서 관리한다.

마지막으로 `InstructionText`를 분리해 `Enter a Number`, `Higher or lower?` 같은 안내 문구를 같은 스타일로 재사용할 수 있게 했다. 컴포넌트를 옮기거나 새로 만들 때는 `Colors`, `View`, `Text` 같은 import와 상대 경로를 반드시 함께 확인해야 한다.
