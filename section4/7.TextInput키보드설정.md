# 7. TextInput 키보드 설정

## 핵심 요약

- `TextInput`은 입력값의 길이뿐 아니라 모바일 기기에 표시되는 키보드 종류도 제어할 수 있다.
- `keyboardType` prop을 사용하면 숫자 키보드, 이메일 키보드, URL 키보드 등 입력 목적에 맞는 키보드를 열 수 있다.
- 숫자 입력 필드에는 `keyboardType="number-pad"`를 사용하면 숫자 중심 키보드가 표시된다.
- `autoCapitalize`와 `autoCorrect`는 텍스트 입력 UX를 조정할 때 중요하다.
- 이메일, URL, 숫자 입력처럼 자동 대문자화나 자동 수정이 방해가 되는 입력 필드에서는 관련 옵션을 꺼야 한다.

## 학습 목표

- `TextInput`의 `keyboardType` prop으로 모바일 키보드를 제어하는 방법을 익힌다.
- 숫자 입력에 적합한 `number-pad` 키보드 타입을 이해한다.
- `autoCapitalize`와 `autoCorrect`가 어떤 상황에서 중요한지 배운다.
- TextInput prop 중 일부는 플랫폼별 지원 여부가 다를 수 있음을 인지한다.
- 입력 필드 UX는 스타일뿐 아니라 키보드 설정까지 포함한다는 점을 이해한다.

## 주요 내용

### 입력 가능한 문자와 키보드 UX

이전 강의에서는 `maxLength={2}`로 입력 가능한 문자 수를 제한했다.

하지만 아직 문제는 남아 있다.

```text
일반 문자도 입력할 수 있다.
일반 텍스트 키보드가 열린다.
숫자 입력에 최적화된 UX가 아니다.
```

사용자가 1부터 99 사이의 숫자를 입력해야 하는 상황이라면 일반 키보드보다 숫자 키보드가 열리는 편이 훨씬 자연스럽다.

### keyboardType 사용하기

`TextInput`에서 기기에 표시되는 키보드 종류를 제어하려면 `keyboardType` prop을 사용한다.

숫자 입력에는 `number-pad`를 사용한다.

```jsx
<TextInput
  style={styles.numberInput}
  maxLength={2}
  keyboardType="number-pad"
/>
```

이렇게 설정하면 iOS와 Android에서 숫자 중심 키보드가 표시된다.

```text
일반 텍스트 키보드
-> 문자 입력 중심

number-pad
-> 숫자 입력 중심
```

### keyboardType 값은 플랫폼마다 다를 수 있다

`keyboardType`에는 여러 값을 사용할 수 있다.

예를 들면 다음과 같은 입력 목적에 맞는 키보드 타입이 있다.

```text
default
number-pad
decimal-pad
numeric
email-address
phone-pad
url
```

다만 모든 값이 모든 플랫폼에서 동일하게 지원되는 것은 아니다. 일부 값은 iOS에서만 동작하거나 Android에서만 다르게 동작할 수 있다.

따라서 특정 키보드 타입을 사용할 때는 공식 문서에서 플랫폼 지원 여부를 확인하는 것이 좋다.

참고 문서:

- [React Native TextInput](https://reactnative.dev/docs/textinput)

### autoCapitalize

`autoCapitalize`는 사용자가 입력하는 텍스트를 자동으로 대문자화할지 제어한다.

이번 앱은 숫자를 입력하므로 큰 의미는 없지만, 텍스트 입력 필드에서는 중요한 옵션이다.

```jsx
<TextInput
  autoCapitalize="none"
/>
```

예를 들어 이메일 주소 입력 필드에서 첫 글자가 자동으로 대문자가 되면 사용자 경험이 나빠질 수 있다.

```text
user@example.com
-> User@example.com
```

이런 상황을 막으려면 `autoCapitalize="none"`처럼 설정한다.

### autoCorrect

`autoCorrect`는 자동 수정 기능을 켜거나 끄는 prop이다.

```jsx
<TextInput
  autoCorrect={false}
/>
```

일반 문장 입력에서는 자동 수정이 도움이 될 수 있지만, 이메일, URL, 코드, 사용자명 같은 입력에서는 오히려 방해가 된다.

예를 들어 이메일 입력란에서 자동 수정이 켜져 있으면 사용자가 의도하지 않은 값으로 바뀔 수 있다.

따라서 입력 필드의 목적에 따라 자동 수정 여부를 명확히 정해야 한다.

### 숫자 입력 필드 예시

현재 숫자 맞히기 앱의 `TextInput`은 다음과 같이 구성할 수 있다.

```jsx
<TextInput
  style={styles.numberInput}
  maxLength={2}
  keyboardType="number-pad"
  autoCapitalize="none"
  autoCorrect={false}
/>
```

숫자 입력에서는 `autoCapitalize`와 `autoCorrect`가 큰 영향을 주지는 않지만, 입력 필드 설정을 어떻게 조정할 수 있는지 보여주는 예시로 사용할 수 있다.

### 입력 UX는 스타일만의 문제가 아니다

입력 필드의 사용성은 색상, 크기, 정렬 같은 스타일만으로 결정되지 않는다.

실제 모바일 앱에서는 어떤 키보드가 열리는지도 매우 중요하다.

```text
숫자 입력
-> 숫자 키보드

이메일 입력
-> 이메일 키보드
-> 자동 대문자화 끄기
-> 자동 수정 끄기

URL 입력
-> URL 키보드
-> 자동 수정 끄기
```

사용자가 입력해야 하는 값의 성격에 맞게 키보드와 자동 입력 옵션을 조정해야 좋은 사용자 경험을 만들 수 있다.

### 아직 남은 작업

이번 단계에서는 키보드를 숫자 입력에 맞게 바꾸었다.

하지만 아직 다음 작업이 남아 있다.

```text
입력값을 state로 관리하기
입력값이 1부터 99 사이인지 검증하기
Reset 버튼 기능 구현하기
Confirm 버튼 기능 구현하기
버튼과 입력 영역 레이아웃 정리하기
```

다음 강의에서는 버튼과 박스 레이아웃을 더 다듬어 화면을 정리한다.

## 정리

이번 강의에서는 `TextInput`의 `keyboardType`을 사용해 모바일 기기에 표시되는 키보드 종류를 제어했다. 숫자 입력에는 `keyboardType="number-pad"`를 사용해 숫자 중심 키보드가 뜨도록 만들었다.

또한 `autoCapitalize`와 `autoCorrect` 같은 입력 옵션을 살펴보며, 입력 필드의 목적에 따라 키보드와 자동 입력 동작을 조정해야 한다는 점을 배웠다. 입력 UX는 스타일뿐 아니라 사용자가 어떤 방식으로 값을 입력하는지까지 포함한다.
