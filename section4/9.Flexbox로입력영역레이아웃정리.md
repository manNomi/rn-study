# 9. Flexbox로 입력 영역 레이아웃 정리

## 핵심 요약

- 시작 화면의 입력 카드 안에서 `TextInput`과 버튼들의 위치를 Flexbox로 정리한다.
- React Native의 기본 `flexDirection`은 `column`이므로, 기본 주축은 위에서 아래 방향이다.
- `justifyContent`는 주축 방향 정렬을 제어하고, `alignItems`는 교차축 방향 정렬을 제어한다.
- `alignItems: 'center'`를 사용하면 입력 필드와 버튼이 가로축 기준으로 중앙에 정렬된다.
- 두 버튼을 한 줄에 배치하려면 버튼들을 별도의 `View`로 묶고 `flexDirection: 'row'`를 적용한다.
- 두 버튼을 같은 너비로 만들려면 각 버튼을 다시 `View`로 감싸고 `flex: 1`을 적용한다.

## 학습 목표

- React Native에서 Flexbox를 이용해 카드 내부 레이아웃을 정리하는 방법을 익힌다.
- `justifyContent`와 `alignItems`가 각각 어떤 축에 영향을 주는지 이해한다.
- `alignItems`의 기본값인 `stretch`가 버튼 너비에 어떤 영향을 주는지 배운다.
- 여러 단계의 `View` 중첩으로 원하는 레이아웃을 만드는 React Native 패턴을 익힌다.
- 버튼을 같은 줄, 같은 크기로 배치하는 기본 구조를 이해한다.

## 주요 내용

### 입력 카드 레이아웃 개선하기

현재 시작 화면은 점점 형태를 갖춰가고 있지만, 입력 카드 내부 레이아웃은 아직 어색하다.

개선하고 싶은 점은 다음과 같다.

```text
TextInput을 카드 중앙에 배치하기
Reset과 Confirm 버튼을 위아래가 아니라 한 줄에 배치하기
버튼이 너무 넓게 늘어나지 않게 하기
두 버튼의 크기를 비슷하게 맞추기
```

이 작업은 `StartGameScreen`의 입력 카드에 적용된 `inputContainer` 스타일을 중심으로 진행한다.

```jsx
<View style={styles.inputContainer}>
  <TextInput style={styles.numberInput} />
  <PrimaryButton>Reset</PrimaryButton>
  <PrimaryButton>Confirm</PrimaryButton>
</View>
```

### React Native Flexbox의 기본 방향

React Native의 모든 `View`는 기본적으로 Flexbox 컨테이너처럼 동작한다.

웹 CSS와 달리 React Native의 기본 `flexDirection`은 `column`이다.

```text
React Native 기본값
-> flexDirection: 'column'

주축
-> 위에서 아래

교차축
-> 왼쪽에서 오른쪽
```

따라서 기본 상태에서는 자식 요소들이 위에서 아래로 쌓인다.

```text
TextInput
Reset Button
Confirm Button
```

### justifyContent와 alignItems

Flexbox에서 자식 요소의 위치는 주로 `justifyContent`와 `alignItems`로 제어한다.

```text
justifyContent
-> 주축 방향 정렬

alignItems
-> 교차축 방향 정렬
```

현재 `inputContainer`의 기본 `flexDirection`은 `column`이므로 다음처럼 이해할 수 있다.

```text
justifyContent
-> 세로 방향 정렬

alignItems
-> 가로 방향 정렬
```

즉 입력 필드와 버튼을 가로로 중앙 정렬하려면 `justifyContent`보다 `alignItems`가 중요하다.

### justifyContent가 크게 티 나지 않는 이유

`inputContainer`에 `justifyContent: 'center'`를 추가할 수 있다.

```jsx
inputContainer: {
  justifyContent: 'center',
}
```

하지만 이 단계에서는 화면상 변화가 거의 없을 수 있다.

이유는 입력 카드 컨테이너가 내부 요소들이 차지하는 높이만큼만 공간을 가지기 때문이다.

```text
컨테이너 높이
-> 내부 요소 높이와 거의 같음

남는 세로 공간
-> 거의 없음

justifyContent
-> 배치할 여유 공간이 없으므로 변화가 잘 보이지 않음
```

`justifyContent`는 주축에 남는 공간이 있을 때 효과가 더 분명하게 보인다.

### alignItems로 가로 중앙 정렬하기

입력 필드를 가로축 기준으로 중앙에 두려면 `alignItems: 'center'`를 사용한다.

```jsx
const styles = StyleSheet.create({
  inputContainer: {
    justifyContent: 'center',
    alignItems: 'center',
    padding: 16,
    marginTop: 100,
    marginHorizontal: 24,
    backgroundColor: '#4e0329',
    borderRadius: 8,
  },
});
```

`alignItems`의 기본값은 `stretch`다.

`stretch` 상태에서는 자식 요소들이 가능한 가로 공간을 채우려고 한다.

```text
alignItems: 'stretch'
-> 자식 요소가 가로로 늘어남

alignItems: 'center'
-> 자식 요소가 필요한 만큼의 너비만 차지하고 중앙 정렬됨
```

`TextInput`은 이미 `width: 50`처럼 너비를 지정했기 때문에 이전에도 크게 늘어나지 않았다.

반면 버튼은 별도 너비 제한이 없었기 때문에 기본적으로 넓게 늘어나 보일 수 있었다.

### 버튼을 한 줄에 배치하기

두 버튼을 위아래가 아니라 한 줄에 배치하려면 두 버튼을 별도의 `View`로 감싼다.

```jsx
<View style={styles.inputContainer}>
  <TextInput style={styles.numberInput} />

  <View style={styles.buttonsContainer}>
    <PrimaryButton>Reset</PrimaryButton>
    <PrimaryButton>Confirm</PrimaryButton>
  </View>
</View>
```

새로 추가한 `View`도 Flexbox 컨테이너다.

이 컨테이너에 `flexDirection: 'row'`를 적용하면 버튼들이 가로 방향으로 배치된다.

```jsx
const styles = StyleSheet.create({
  buttonsContainer: {
    flexDirection: 'row',
  },
});
```

결과적으로 구조는 다음처럼 바뀐다.

```text
inputContainer
-> TextInput
-> buttonsContainer
   -> Reset Button
   -> Confirm Button
```

`inputContainer`는 전체 입력 카드의 레이아웃을 담당하고, `buttonsContainer`는 버튼 두 개의 가로 배치를 담당한다.

### 버튼 크기 차이 해결하기

버튼을 가로로 배치하면 `Reset`과 `Confirm` 텍스트 길이가 달라서 버튼 크기도 달라질 수 있다.

```text
Reset
-> 짧은 텍스트

Confirm
-> 더 긴 텍스트
```

두 버튼을 같은 너비로 만들려면 각 버튼을 다시 `View`로 감싼다.

```jsx
<View style={styles.buttonsContainer}>
  <View style={styles.buttonContainer}>
    <PrimaryButton>Reset</PrimaryButton>
  </View>
  <View style={styles.buttonContainer}>
    <PrimaryButton>Confirm</PrimaryButton>
  </View>
</View>
```

그리고 각 버튼을 감싼 `buttonContainer`에 `flex: 1`을 적용한다.

```jsx
const styles = StyleSheet.create({
  buttonContainer: {
    flex: 1,
  },
});
```

`flex: 1`을 가진 두 컨테이너는 부모 컨테이너 안에서 사용 가능한 공간을 같은 비율로 나눠 가진다.

```text
buttonsContainer
-> buttonContainer flex: 1
-> buttonContainer flex: 1

결과
-> 두 버튼 영역이 1:1로 공간을 나눔
```

이제 Reset과 Confirm 버튼은 같은 폭의 영역을 기반으로 렌더링된다.

### 전체 예시

입력 영역 JSX는 다음과 같은 형태가 된다.

```jsx
function StartGameScreen() {
  return (
    <View style={styles.inputContainer}>
      <TextInput style={styles.numberInput} />

      <View style={styles.buttonsContainer}>
        <View style={styles.buttonContainer}>
          <PrimaryButton>Reset</PrimaryButton>
        </View>
        <View style={styles.buttonContainer}>
          <PrimaryButton>Confirm</PrimaryButton>
        </View>
      </View>
    </View>
  );
}
```

스타일은 다음처럼 정리할 수 있다.

```jsx
const styles = StyleSheet.create({
  inputContainer: {
    justifyContent: 'center',
    alignItems: 'center',
    padding: 16,
    marginTop: 100,
    marginHorizontal: 24,
    backgroundColor: '#4e0329',
    borderRadius: 8,
  },
  buttonsContainer: {
    flexDirection: 'row',
  },
  buttonContainer: {
    flex: 1,
  },
  numberInput: {
    height: 50,
    width: 50,
    textAlign: 'center',
  },
});
```

실제 코드에는 이전 강의에서 추가한 그림자, 색상, 입력 필드 스타일 등이 함께 들어간다.

### View 중첩과 레이아웃 설계

React Native에서 원하는 레이아웃을 만들 때는 `View`를 여러 단계로 중첩하는 일이 흔하다.

이번 강의의 구조도 그 예시다.

```text
inputContainer
-> 전체 카드 정렬

buttonsContainer
-> 버튼들을 가로 방향으로 배치

buttonContainer
-> 각 버튼이 같은 공간을 차지하도록 설정

PrimaryButton
-> 실제 버튼 UI와 터치 피드백 담당
```

하나의 컨테이너에서 모든 배치를 해결하려고 하기보다, 역할별로 `View`를 나누고 각 레벨에 필요한 Flexbox 속성을 적용하는 것이 React Native 레이아웃의 기본 패턴이다.

### 프론트엔드 개발자 관점에서의 핵심 차이

웹에서도 Flexbox를 사용하지만, React Native에서는 몇 가지 기본값과 감각이 다르다.

```text
웹 CSS Flexbox
-> flexDirection 기본값: row

React Native Flexbox
-> flexDirection 기본값: column
```

또 웹에서는 `button`, `input`, `div` 같은 HTML 요소 자체가 어느 정도 기본 레이아웃과 동작을 가진다.

React Native에서는 `View`, `TextInput`, `Pressable`, `Text`를 조합하고, 각 `View` 레벨에 Flexbox 속성을 직접 부여해 레이아웃을 만든다.

```text
웹
-> CSS 클래스 하나로 부모/자식 레이아웃을 한 번에 조정하는 경우가 많음

React Native
-> 여러 View를 중첩하고 각 레벨의 축과 공간 배분을 조정하는 경우가 많음
```

따라서 React Native 레이아웃을 다룰 때는 “어느 View가 어떤 축을 담당하는가”를 먼저 생각하는 것이 좋다.

## 정리

이번 강의에서는 시작 화면의 입력 카드 내부 레이아웃을 Flexbox로 정리했다. `inputContainer`에 `alignItems: 'center'`를 적용해 입력 필드와 버튼 영역을 가로축 기준으로 중앙 정렬했고, 버튼들을 `buttonsContainer`로 묶어 `flexDirection: 'row'`를 적용했다.

또한 각 버튼을 별도의 `View`로 감싼 뒤 `flex: 1`을 적용해 두 버튼이 같은 너비의 공간을 나누어 갖도록 만들었다. React Native에서는 이런 식으로 `View`를 중첩하고 각 레벨에 필요한 Flexbox 속성을 적용해 원하는 레이아웃을 구성한다.
