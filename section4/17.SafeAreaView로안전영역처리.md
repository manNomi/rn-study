# 17. SafeAreaView로 안전 영역 처리

## 핵심 요약

- 최신 스마트폰에는 노치, 상태 표시줄, 둥근 모서리처럼 콘텐츠가 겹치면 안 되는 영역이 있다.
- 기기마다 안전하게 사용할 수 있는 화면 영역이 다르므로 고정 `paddingTop`만으로 처리하기 어렵다.
- React Native의 `SafeAreaView`를 사용하면 콘텐츠가 노치나 상태 표시줄에 가려지지 않도록 안전 영역 안에 배치할 수 있다.
- 배경 이미지와 그러데이션은 노치 뒤까지 깔리도록 유지하고, 실제 화면 콘텐츠만 `SafeAreaView`로 감싼다.
- `SafeAreaView`에도 `flex: 1` 스타일을 적용해야 내부 화면이 사용 가능한 공간을 채울 수 있다.
- Android처럼 노치가 없는 환경에서는 큰 변화가 없을 수 있으므로, 필요한 경우 개별 화면의 `padding`을 추가로 조정한다.

## 학습 목표

- 모바일 기기의 안전 영역이 왜 중요한지 이해한다.
- `SafeAreaView`를 앱 전체 화면 구조에 적용하는 방법을 익힌다.
- 배경 레이어와 콘텐츠 레이어를 분리해 생각하는 방법을 배운다.
- `LinearGradient`, `ImageBackground`, `SafeAreaView`, `screen`의 올바른 중첩 순서를 이해한다.
- `SafeAreaView`와 일반 padding의 역할 차이를 구분한다.
- Android와 iOS에서 안전 영역 처리가 다르게 보일 수 있음을 인지한다.

## 주요 내용

### 노치와 안전 영역 문제

최신 스마트폰에는 다양한 형태의 노치가 있다.

```text
iPhone
-> 큰 노치 또는 Dynamic Island 영역

Android
-> 작은 카메라 홀, 노치, 상태 표시줄

기기 공통
-> 둥근 모서리
-> 홈 인디케이터 주변 영역
```

콘텐츠가 이런 영역에 너무 가까이 붙거나 가려지면 사용성이 나빠진다.

특히 게임 화면처럼 상단에 중요한 제목이나 숫자가 표시되는 화면에서는 콘텐츠가 노치와 겹치지 않도록 해야 한다.

```text
문제 상황
-> 텍스트가 노치 아래에 붙음
-> 상태 표시줄과 콘텐츠가 겹침
-> 기기마다 간격이 다르게 보임
```

### 고정 padding만으로는 부족하다

`GameScreen`에서 `padding`을 늘리면 콘텐츠가 아래로 내려간다.

```jsx
screen: {
  flex: 1,
  padding: 40,
}
```

하지만 기기마다 노치의 크기, 상태 표시줄 높이, 안전 영역이 다르다.

따라서 모든 기기에 대해 같은 padding 값을 쓰는 것은 안정적인 해결책이 아니다.

```text
노치가 큰 iPhone
-> 더 큰 상단 간격 필요

노치가 없는 Android
-> 같은 간격이 과하게 보일 수 있음

태블릿 또는 작은 화면
-> 또 다른 조정 필요
```

이런 문제를 자동으로 처리하기 위해 `SafeAreaView`를 사용한다.

### SafeAreaView란?

`SafeAreaView`는 콘텐츠를 기기의 안전 영역 안에 배치하도록 도와주는 React Native 컴포넌트다.

```jsx
import { SafeAreaView } from 'react-native';
```

안전 영역은 콘텐츠가 상태 표시줄, 노치, 화면 가장자리 등에 가려지지 않고 표시될 수 있는 영역을 말한다.

```text
SafeAreaView
-> 기기별 안전 영역을 고려
-> 콘텐츠가 노치나 상태 표시줄에 가려지지 않도록 도움
```

화면마다 따로 적용할 수도 있지만, 이번 앱에서는 모든 화면에 적용하고 싶으므로 `GameScreen`이 아니라 `App.js`에서 사용한다.

### 왜 App.js에서 적용하는가

현재 앱은 여러 화면을 조건부 렌더링한다.

```text
StartGameScreen
GameScreen
GameOverScreen 예정
```

이 모든 화면의 콘텐츠가 안전 영역 안에 들어가야 한다.

따라서 개별 화면마다 `SafeAreaView`를 넣기보다, 공통 루트인 `App.js`에서 `screen` 전체를 감싸는 편이 좋다.

```text
각 화면에서 SafeAreaView 사용
-> 화면마다 반복 필요

App.js에서 screen 감싸기
-> 모든 화면에 공통 적용
```

공통 레이아웃 성격의 처리는 앱 루트 근처에서 처리하면 관리가 쉬워진다.

### 배경은 SafeAreaView 바깥에 둔다

중요한 점은 `SafeAreaView`를 어디에 넣느냐다.

배경은 노치 뒤까지 자연스럽게 깔리는 것이 좋다.

```text
LinearGradient
-> 화면 전체 배경
-> 노치 뒤까지 깔려도 괜찮음

ImageBackground
-> 화면 전체 배경 이미지
-> 노치 뒤까지 깔려도 괜찮음

실제 screen 콘텐츠
-> 노치 뒤로 가면 안 됨
```

따라서 `SafeAreaView`는 `LinearGradient`와 `ImageBackground` 바깥이 아니라, 그 안쪽에서 실제 화면 콘텐츠를 감싸야 한다.

올바른 레이어 순서는 다음과 같다.

```text
LinearGradient
-> ImageBackground
   -> SafeAreaView
      -> screen
```

이렇게 하면 배경은 전체 화면을 채우고, 실제 UI 콘텐츠만 안전 영역 안에 들어간다.

### App.js에 SafeAreaView 추가하기

먼저 `SafeAreaView`를 import한다.

```jsx
import { ImageBackground, SafeAreaView, StyleSheet } from 'react-native';
```

그리고 `ImageBackground` 안에서 `screen`을 감싼다.

```jsx
<LinearGradient
  colors={['#4e0329', '#ddb52f']}
  style={styles.rootScreen}
>
  <ImageBackground
    source={require('./assets/images/background.png')}
    resizeMode="cover"
    style={styles.rootScreen}
    imageStyle={styles.backgroundImage}
  >
    <SafeAreaView style={styles.rootScreen}>
      {screen}
    </SafeAreaView>
  </ImageBackground>
</LinearGradient>
```

`SafeAreaView`에도 `style={styles.rootScreen}`을 적용한다.

이유는 `SafeAreaView`가 사용 가능한 공간을 채워야 그 안의 화면 컴포넌트도 제대로 배치될 수 있기 때문이다.

### rootScreen 스타일 재사용

기존 `rootScreen` 스타일은 `flex: 1`을 가지고 있다.

```jsx
const styles = StyleSheet.create({
  rootScreen: {
    flex: 1,
  },
  backgroundImage: {
    opacity: 0.15,
  },
});
```

이 스타일은 여러 레이어에 재사용된다.

```text
LinearGradient
-> 화면 전체 그러데이션

ImageBackground
-> 화면 전체 이미지 배경

SafeAreaView
-> 안전 영역 안에서 화면 콘텐츠를 채움
```

각 레이어가 `flex: 1`을 가져야 전체 화면 구조가 무너지지 않는다.

### SafeAreaView 적용 후 변화

`SafeAreaView`를 적용하면 iOS처럼 노치가 있는 기기에서는 콘텐츠가 아래로 내려와 노치와 겹치지 않는다.

```text
적용 전
-> GameScreen 제목이 상단에 너무 가까움

적용 후
-> 콘텐츠가 안전 영역 안쪽으로 내려옴
```

시작 화면은 이미 여백이 충분히 있었기 때문에 큰 변화가 없어 보일 수 있다.

하지만 게임 화면처럼 상단에 붙어 있던 콘텐츠는 차이가 더 잘 보인다.

### Android에서는 변화가 작을 수 있다

Android 에뮬레이터에 노치가 없다면 `SafeAreaView`를 적용해도 큰 변화가 보이지 않을 수 있다.

```text
iOS 노치 기기
-> SafeAreaView 효과가 눈에 잘 보임

Android 노치 없는 에뮬레이터
-> 변화가 작거나 거의 없음
```

그렇다고 `SafeAreaView`가 필요 없는 것은 아니다.

앱은 다양한 기기에서 실행될 수 있으므로, 노치가 있는 환경까지 고려해야 한다.

### GameScreen padding 조정하기

`SafeAreaView`가 노치와 상태 표시줄을 피하게 해주더라도, 디자인상 콘텐츠가 너무 위에 붙어 보일 수 있다.

그럴 때는 개별 화면의 padding을 조정한다.

```jsx
const styles = StyleSheet.create({
  screen: {
    flex: 1,
    padding: 24,
  },
});
```

이전에는 `padding: 12`였지만, `24` 정도로 올리면 콘텐츠 주변에 여유가 생긴다.

```text
SafeAreaView
-> 기기 안전 영역 보장

screen padding
-> 화면 내부 디자인 여백 조정
```

두 역할은 다르다.

`SafeAreaView`는 노치와 상태 표시줄을 피하는 구조적 안전장치이고, `padding`은 화면 디자인을 위한 여백이다.

### 전체 App.js 예시

이번 단계의 `App.js` 구조는 다음처럼 정리할 수 있다.

```jsx
import { useState } from 'react';
import {
  ImageBackground,
  SafeAreaView,
  StyleSheet,
} from 'react-native';
import { LinearGradient } from 'expo-linear-gradient';

import GameScreen from './screens/GameScreen';
import StartGameScreen from './screens/StartGameScreen';

export default function App() {
  const [userNumber, setUserNumber] = useState(null);

  function pickedNumberHandler(pickedNumber) {
    setUserNumber(pickedNumber);
  }

  let screen = (
    <StartGameScreen onPickNumber={pickedNumberHandler} />
  );

  if (userNumber) {
    screen = <GameScreen />;
  }

  return (
    <LinearGradient
      colors={['#4e0329', '#ddb52f']}
      style={styles.rootScreen}
    >
      <ImageBackground
        source={require('./assets/images/background.png')}
        resizeMode="cover"
        style={styles.rootScreen}
        imageStyle={styles.backgroundImage}
      >
        <SafeAreaView style={styles.rootScreen}>
          {screen}
        </SafeAreaView>
      </ImageBackground>
    </LinearGradient>
  );
}

const styles = StyleSheet.create({
  rootScreen: {
    flex: 1,
  },
  backgroundImage: {
    opacity: 0.15,
  },
});
```

이 구조에서는 배경 레이어와 콘텐츠 안전 영역이 명확히 분리된다.

```text
배경
-> LinearGradient
-> ImageBackground

콘텐츠
-> SafeAreaView
-> screen
```

### 프론트엔드 개발자 관점에서의 핵심 차이

웹에서는 보통 브라우저 viewport를 기준으로 레이아웃을 잡는다.

```css
body {
  min-height: 100vh;
}
```

모바일 웹에서도 CSS 환경 변수로 안전 영역을 다룰 수 있다.

```css
padding-top: env(safe-area-inset-top);
```

React Native에서는 이런 처리를 `SafeAreaView` 같은 컴포넌트로 해결한다.

```text
웹
-> CSS env(safe-area-inset-*)

React Native
-> SafeAreaView
```

또 배경과 콘텐츠를 분리해서 생각해야 한다.

```text
배경은 노치 뒤까지 확장되어도 자연스러움
콘텐츠는 노치 뒤로 들어가면 안 됨
```

따라서 `SafeAreaView`를 앱 전체 바깥에 두는 것이 아니라, 배경 컴포넌트 안에서 실제 화면 콘텐츠만 감싸는 구조가 자연스럽다.

## 정리

이번 강의에서는 노치와 상태 표시줄 같은 기기별 안전 영역을 고려하기 위해 `SafeAreaView`를 적용했다. 모든 화면에 공통으로 적용하기 위해 `GameScreen`이 아니라 `App.js`에서 `screen`을 감싸도록 구성했다.

중요한 점은 배경 레이어인 `LinearGradient`와 `ImageBackground`는 화면 전체와 노치 뒤까지 확장되게 두고, 실제 콘텐츠만 `SafeAreaView` 안에 넣는 것이다. `SafeAreaView`에는 `flex: 1`을 적용하고, 개별 화면의 디자인 여백은 `GameScreen`의 `padding`으로 별도로 조정한다.
