# 14. 입력값 유효성 검사와 Alert

## 핵심 요약

- `TextInput`에서 관리 중인 `enteredNumber`는 문자열이므로, 검증 전에 숫자로 변환해야 한다.
- 문자열을 숫자로 바꾸기 위해 JavaScript 전역 함수인 `parseInt()`를 사용한다.
- 숫자로 변환할 수 없는 값은 `isNaN()`으로 확인한다.
- 게임 규칙상 허용되는 값은 1 이상 99 이하의 숫자다.
- 유효하지 않은 값이면 `Alert.alert()`로 네이티브 경고창을 띄우고 함수 실행을 중단한다.
- Alert 버튼의 `onPress`에 `resetInputHandler`를 연결하면 경고 확인 후 입력값을 초기화할 수 있다.
- Reset 버튼에도 같은 `resetInputHandler`를 연결해 사용자가 직접 입력값을 비울 수 있게 한다.

## 학습 목표

- 문자열 입력값을 숫자로 변환하고 검증하는 흐름을 익힌다.
- `parseInt()`와 `isNaN()`을 React Native 앱에서도 그대로 사용할 수 있음을 이해한다.
- 유효하지 않은 입력값을 처리할 때 `return`으로 핸들러 실행을 중단하는 패턴을 배운다.
- React Native의 `Alert` API를 사용해 네이티브 경고창을 띄우는 방법을 익힌다.
- Alert 버튼 배열의 구조와 `text`, `style`, `onPress` 설정을 이해한다.
- controlled input 구조에서 state를 초기화하면 `TextInput` 값도 함께 초기화된다는 점을 복습한다.

## 주요 내용

### 입력값 검증이 필요한 이유

이전 강의에서 `TextInput`의 값을 `enteredNumber` state로 관리하고, Confirm 버튼을 눌렀을 때 `confirmInputHandler`가 실행되도록 연결했다.

이제 이 함수 안에서 입력값을 검증해야 한다.

게임 규칙은 간단하다.

```text
사용자는 1부터 99 사이의 숫자를 입력해야 한다.
```

따라서 다음 값들은 유효하지 않다.

```text
빈 입력값
숫자가 아닌 값
0
음수
100 이상
```

`keyboardType="number-pad"`와 `maxLength={2}`를 사용하고 있더라도 검증은 필요하다.

키보드 설정은 사용자 경험을 좋게 만드는 장치이고, 유효성 검사는 앱의 규칙을 보장하는 코드다.

### 문자열을 숫자로 변환하기

`TextInput` 값은 숫자 키보드로 입력하더라도 문자열이다.

따라서 숫자 비교를 하려면 먼저 숫자로 변환해야 한다.

```jsx
function confirmInputHandler() {
  const chosenNumber = parseInt(enteredNumber);
}
```

`parseInt()`는 JavaScript 전역 함수다.

브라우저 JavaScript에서 사용할 수 있는 것처럼 React Native 앱에서도 사용할 수 있다.

```text
parseInt('5')
-> 5

parseInt('99')
-> 99

parseInt('')
-> NaN

parseInt('abc')
-> NaN
```

`NaN`은 `Not a Number`의 약자로, 숫자로 변환할 수 없다는 뜻이다.

### isNaN으로 변환 실패 확인하기

`parseInt()`가 숫자 변환에 실패하면 `NaN`이 나온다.

이를 확인하기 위해 `isNaN()`을 사용한다.

```jsx
if (isNaN(chosenNumber)) {
  // 유효하지 않은 값
}
```

`isNaN()`도 JavaScript 전역 함수다.

```text
isNaN(5)
-> false

isNaN(NaN)
-> true
```

입력값이 비어 있거나 숫자로 바꿀 수 없는 문자열이면 `isNaN(chosenNumber)`가 `true`가 된다.

### 범위 검사 추가하기

숫자로 변환되었더라도 게임 규칙의 범위를 벗어나면 유효하지 않다.

검사 조건은 다음과 같다.

```text
숫자가 아닌 경우
또는 0 이하인 경우
또는 99보다 큰 경우
```

코드로는 `||` 연산자를 사용해 조건을 묶는다.

```jsx
function confirmInputHandler() {
  const chosenNumber = parseInt(enteredNumber);

  if (
    isNaN(chosenNumber) ||
    chosenNumber <= 0 ||
    chosenNumber > 99
  ) {
    return;
  }

  console.log('Valid number!');
}
```

`chosenNumber <= 0`은 0과 음수를 모두 막는다.

`chosenNumber > 99`는 100 이상의 값을 막는다.

유효하지 않은 경우에는 `return`을 사용해 함수 실행을 중단한다.

```text
입력값 유효하지 않음
-> return
-> 이후 게임 시작 로직 실행 안 함
```

### Alert API 가져오기

유효하지 않은 값을 조용히 무시하면 사용자는 왜 아무 일도 일어나지 않는지 알 수 없다.

그래서 React Native의 `Alert` API를 사용해 경고창을 보여준다.

`Alert`는 컴포넌트가 아니라 객체다.

```jsx
import { Alert } from 'react-native';
```

정확히는 `Alert.alert()` 메서드를 호출해 네이티브 경고창을 띄운다.

```text
Alert
-> React Native가 제공하는 API 객체

Alert.alert()
-> iOS/Android 네이티브 경고창 표시
```

### Alert.alert 기본 구조

`Alert.alert()`에는 제목, 메시지, 버튼 설정을 전달할 수 있다.

```jsx
Alert.alert(
  '유효하지 않은 숫자!',
  '1과 99 사이의 숫자여야 함.',
  [
    {
      text: 'Okay',
      style: 'destructive',
      onPress: resetInputHandler,
    },
  ]
);
```

인자의 의미는 다음과 같다.

```text
첫 번째 인자
-> title
-> 경고창 제목

두 번째 인자
-> message
-> 경고창 본문 메시지

세 번째 인자
-> buttons 배열
-> 경고창에 표시할 버튼 목록
```

이번 강의에서는 버튼을 하나만 추가한다.

버튼 객체는 다음 속성을 가진다.

| 속성 | 역할 |
| --- | --- |
| `text` | 버튼에 표시될 문구 |
| `style` | 버튼 스타일 |
| `onPress` | 버튼을 눌렀을 때 실행할 함수 |

### Alert 버튼 스타일

Alert 버튼의 `style`에는 몇 가지 값을 사용할 수 있다.

```text
default
destructive
cancel
```

강의에서는 `destructive`를 사용한다.

```jsx
style: 'destructive'
```

iOS에서는 destructive 스타일이 빨간색 텍스트처럼 보일 수 있다.

Android에서는 Android 플랫폼의 기본 Alert Dialog 스타일에 맞게 표시된다.

중요한 점은 `Alert.alert()`가 각 플랫폼의 네이티브 대화상자를 사용한다는 것이다.

```text
iOS
-> iOS 기본 경고창 모양

Android
-> Android 기본 경고창 모양
```

같은 JavaScript 코드로 호출하지만 실제 UI는 플랫폼의 네이티브 스타일을 따른다.

### resetInputHandler 만들기

잘못된 값이 입력되었을 때 경고창을 닫으면 입력값을 비워주는 것이 자연스럽다.

이를 위해 입력값을 초기화하는 함수를 만든다.

```jsx
function resetInputHandler() {
  setEnteredNumber('');
}
```

`enteredNumber`는 `TextInput`의 `value` prop과 연결되어 있다.

따라서 state를 빈 문자열로 바꾸면 입력 필드도 함께 비워진다.

```text
setEnteredNumber('')
-> enteredNumber state 변경
-> value={enteredNumber}
-> TextInput 표시값도 빈 문자열
```

이게 controlled input 구조의 장점이다.

### Alert 확인 버튼과 reset 연결하기

Alert 버튼 객체의 `onPress`에 `resetInputHandler`를 전달한다.

```jsx
Alert.alert(
  '유효하지 않은 숫자!',
  '1과 99 사이의 숫자여야 함.',
  [
    {
      text: 'Okay',
      style: 'destructive',
      onPress: resetInputHandler,
    },
  ]
);
```

이제 유효하지 않은 값을 입력한 뒤 Alert의 `Okay` 버튼을 누르면 입력란이 초기화된다.

### confirmInputHandler 전체 흐름

검증과 Alert를 합치면 `confirmInputHandler`는 다음과 같은 구조가 된다.

```jsx
function confirmInputHandler() {
  const chosenNumber = parseInt(enteredNumber);

  if (
    isNaN(chosenNumber) ||
    chosenNumber <= 0 ||
    chosenNumber > 99
  ) {
    Alert.alert(
      '유효하지 않은 숫자!',
      '1과 99 사이의 숫자여야 함.',
      [
        {
          text: 'Okay',
          style: 'destructive',
          onPress: resetInputHandler,
        },
      ]
    );
    return;
  }

  console.log('Valid number!');
}
```

여기서 `return`은 중요하다.

Alert를 띄운 뒤에도 함수가 계속 실행되면, 유효하지 않은 값인데도 게임 시작 로직이 실행될 수 있다.

```text
유효하지 않은 값
-> Alert 표시
-> return으로 함수 종료
-> Valid number 또는 게임 시작 로직 실행 방지
```

### Reset 버튼 연결하기

Alert에서만 입력값을 초기화할 것이 아니라, 화면의 Reset 버튼도 같은 함수를 사용해야 한다.

```jsx
<PrimaryButton onPress={resetInputHandler}>
  Reset
</PrimaryButton>
```

Confirm 버튼은 검증 핸들러를 실행한다.

```jsx
<PrimaryButton onPress={confirmInputHandler}>
  Confirm
</PrimaryButton>
```

이렇게 하면 두 버튼의 역할이 분명해진다.

```text
Reset
-> enteredNumber 초기화

Confirm
-> enteredNumber 검증
-> 유효하지 않으면 Alert
-> 유효하면 다음 단계 진행
```

### 전체 StartGameScreen 흐름 예시

핵심 흐름만 보면 다음과 같다.

```jsx
import { useState } from 'react';
import { Alert, TextInput, View } from 'react-native';

function StartGameScreen() {
  const [enteredNumber, setEnteredNumber] = useState('');

  function numberInputHandler(enteredText) {
    setEnteredNumber(enteredText);
  }

  function resetInputHandler() {
    setEnteredNumber('');
  }

  function confirmInputHandler() {
    const chosenNumber = parseInt(enteredNumber);

    if (
      isNaN(chosenNumber) ||
      chosenNumber <= 0 ||
      chosenNumber > 99
    ) {
      Alert.alert(
        '유효하지 않은 숫자!',
        '1과 99 사이의 숫자여야 함.',
        [
          {
            text: 'Okay',
            style: 'destructive',
            onPress: resetInputHandler,
          },
        ]
      );
      return;
    }

    console.log('Valid number!');
  }

  return (
    <View>
      <TextInput
        value={enteredNumber}
        onChangeText={numberInputHandler}
        keyboardType="number-pad"
        maxLength={2}
      />

      <PrimaryButton onPress={resetInputHandler}>
        Reset
      </PrimaryButton>
      <PrimaryButton onPress={confirmInputHandler}>
        Confirm
      </PrimaryButton>
    </View>
  );
}
```

실제 파일에는 스타일과 레이아웃용 `View`가 더 포함된다.

### 동작 확인

이번 단계에서 확인할 수 있는 동작은 다음과 같다.

```text
빈 값으로 Confirm
-> Alert 표시

0 또는 음수 입력 후 Confirm
-> Alert 표시

99보다 큰 값 입력 후 Confirm
-> Alert 표시

유효한 숫자 입력 후 Confirm
-> 콘솔에 Valid number! 출력

Reset 버튼 클릭
-> 입력값 초기화

Alert의 Okay 버튼 클릭
-> 입력값 초기화
```

여기서 유효한 숫자를 입력했을 때는 아직 화면 전환이 일어나지 않는다.

현재는 임시로 `console.log('Valid number!')`만 출력한다.

다음 강의에서는 유효한 숫자가 입력되었을 때 실제 게임 화면으로 넘어가는 흐름을 만들게 된다.

### 프론트엔드 개발자 관점에서의 핵심 차이

웹 React에서 입력 검증을 할 때와 기본 흐름은 같다.

```text
입력값 state로 관리
버튼 클릭 시 handler 실행
handler 안에서 값 변환 및 검증
유효하지 않으면 사용자에게 피드백
유효하면 다음 로직 진행
```

차이는 사용자에게 피드백을 주는 방식이다.

웹에서는 보통 직접 만든 모달, 브라우저 `alert()`, 혹은 폼 에러 메시지를 사용한다.

React Native에서는 `Alert.alert()`를 사용하면 각 플랫폼의 네이티브 경고창을 바로 띄울 수 있다.

```text
웹
-> window.alert()
-> 커스텀 모달
-> 폼 에러 메시지

React Native
-> Alert.alert()
-> iOS/Android 네이티브 경고창
```

또 한 가지 중요한 차이는 키보드 설정이 검증을 대체하지 않는다는 점이다.

```text
keyboardType="number-pad"
-> 입력 UX 개선

parseInt + isNaN + 범위 검사
-> 실제 데이터 검증
```

사용자 경험과 데이터 검증은 서로 다른 책임이다.

## 정리

이번 강의에서는 `enteredNumber` 문자열 state를 `parseInt()`로 숫자로 변환하고, `isNaN()`, `chosenNumber <= 0`, `chosenNumber > 99` 조건으로 유효성을 검사했다. 유효하지 않은 값이면 `Alert.alert()`를 사용해 네이티브 경고창을 띄우고 `return`으로 함수 실행을 중단했다.

또한 `resetInputHandler`를 만들어 입력값을 빈 문자열로 초기화하고, 이 함수를 Reset 버튼과 Alert의 `Okay` 버튼에 모두 연결했다. 이제 사용자는 잘못된 입력에 대한 피드백을 받을 수 있고, 유효한 값이 들어왔을 때 다음 게임 화면으로 넘어갈 준비가 되었다.
