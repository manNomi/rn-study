# 4. 커스텀 버튼 만들기

## 핵심 요약

- React Native의 기본 `Button` 대신 직접 `PrimaryButton` 컴포넌트를 만든다.
- 커스텀 버튼은 `View`와 `Text` 같은 핵심 컴포넌트를 조합해 구성한다.
- 버튼에 표시할 텍스트는 `props.children`을 통해 외부에서 전달받는다.
- `StartGameScreen`에서 `Reset`, `Confirm` 버튼을 `PrimaryButton`으로 렌더링한다.
- `App.js`에서 `StartGameScreen`을 import해 루트 화면으로 표시한다.

## 학습 목표

- React Native에서 커스텀 버튼을 직접 만드는 이유를 이해한다.
- 핵심 컴포넌트를 조합해 재사용 컴포넌트를 만드는 흐름을 익힌다.
- React의 `children` prop을 사용해 버튼 내부 텍스트를 전달하는 방법을 복습한다.
- `StartGameScreen`에서 커스텀 버튼을 사용하는 방법을 배운다.
- `App.js`가 앱의 루트 컴포넌트를 렌더링한다는 점을 다시 확인한다.

## 주요 내용

### 왜 커스텀 버튼을 만드는가

React Native에는 기본 `Button` 컴포넌트가 있다. 하지만 기본 버튼은 스타일링 자유도가 낮다.

이번 섹션에서는 앱의 주요 버튼으로 사용할 `PrimaryButton`을 직접 만든다.

직접 버튼을 만들면 다음을 더 자유롭게 제어할 수 있다.

```text
버튼 배경색
텍스트 색상
눌림 효과
여백
테두리와 모서리
플랫폼별 느낌
재사용 가능한 버튼 구조
```

React Native 앱에서는 핵심 컴포넌트를 조합해 원하는 UI 컴포넌트를 직접 만드는 일이 자주 있다.

### PrimaryButton 컴포넌트 만들기

`components` 폴더에 `PrimaryButton.js` 파일을 만든다.

```text
components/
  PrimaryButton.js
```

컴포넌트는 일반적인 React 함수형 컴포넌트로 작성하고, 다른 파일에서 사용할 수 있도록 export한다.

```jsx
function PrimaryButton({ children }) {
  return (
    <View>
      <Text>{children}</Text>
    </View>
  );
}

export default PrimaryButton;
```

이 컴포넌트도 특별한 React Native 문법이 아니라 일반 React 컴포넌트다. 내부에서 React Native 핵심 컴포넌트인 `View`와 `Text`를 조합할 뿐이다.

### View와 Text를 조합하는 이유

버튼은 단순히 텍스트만 있으면 되는 것이 아니라, 박스 형태의 영역과 그 안의 텍스트가 필요하다.

그래서 `View`를 외부 컨테이너로 사용하고, `Text`를 버튼 라벨로 사용한다.

```jsx
import { Text, View } from 'react-native';

function PrimaryButton({ children }) {
  return (
    <View>
      <Text>{children}</Text>
    </View>
  );
}
```

`Text`에도 스타일을 줄 수 있지만 모든 스타일을 자유롭게 적용하기에는 한계가 있다. 박스 형태 버튼을 만들려면 `View`를 함께 사용하는 편이 자연스럽다.

### children prop 사용하기

버튼에 들어갈 텍스트는 버튼 컴포넌트를 사용하는 곳에서 전달한다.

React에서는 여는 태그와 닫는 태그 사이에 들어간 내용을 `props.children`으로 받을 수 있다.

```jsx
<PrimaryButton>Reset</PrimaryButton>
```

위 코드는 `PrimaryButton` 내부에서 다음처럼 사용할 수 있다.

```jsx
function PrimaryButton(props) {
  return (
    <View>
      <Text>{props.children}</Text>
    </View>
  );
}
```

또는 객체 구조 분해를 사용해 `children`만 바로 꺼낼 수 있다.

```jsx
function PrimaryButton({ children }) {
  return (
    <View>
      <Text>{children}</Text>
    </View>
  );
}
```

이 방식은 React Native만의 기능이 아니라 React의 일반적인 패턴이다.

### StartGameScreen에서 버튼 사용하기

`StartGameScreen.js`에서 `PrimaryButton`을 import한다.

```jsx
import PrimaryButton from '../components/PrimaryButton';
```

그리고 `TextInput` 아래에 두 개의 버튼을 렌더링한다.

```jsx
function StartGameScreen() {
  return (
    <View>
      <TextInput />
      <PrimaryButton>Reset</PrimaryButton>
      <PrimaryButton>Confirm</PrimaryButton>
    </View>
  );
}
```

아직 버튼을 눌렀을 때 실행할 기능은 연결하지 않는다. 이번 단계에서는 버튼 컴포넌트를 만들고 화면에 표시하는 데 집중한다.

### App.js에서 StartGameScreen 렌더링하기

React Native가 처음 로딩하는 주요 파일은 `App.js`다.

따라서 새로 만든 `StartGameScreen`을 실제 앱 화면에 표시하려면 `App.js`에서 import하고 렌더링해야 한다.

```jsx
import StartGameScreen from './screens/StartGameScreen';

export default function App() {
  return <StartGameScreen />;
}
```

기존 더미 UI와 불필요한 import는 제거한다.

```text
Text
View
StatusBar
사용하지 않는 styles
```

필요하다면 나중에 다시 `StyleSheet`를 사용할 수 있지만, 현재 단계에서는 `StartGameScreen`을 루트 화면으로 렌더링하는 것이 핵심이다.

### 현재 상태

이 단계까지 진행하면 앱은 실행 가능하다.

다만 아직 다음 작업은 남아 있다.

```text
버튼 스타일링
버튼 눌림 처리
입력값 상태 관리
시작 화면 레이아웃 구성
전체 앱 스타일링
```

시각적으로는 완성되지 않았지만, `PrimaryButton`이라는 첫 재사용 컴포넌트를 만들고 화면에 연결했다는 점이 중요하다.

## 정리

이번 강의에서는 React Native의 기본 `Button` 대신 직접 `PrimaryButton` 컴포넌트를 만들었다. `View`와 `Text`를 조합해 버튼의 기본 구조를 만들고, `children` prop을 사용해 버튼 텍스트를 외부에서 전달받았다.

또한 `StartGameScreen`에서 `Reset`, `Confirm` 버튼을 렌더링하고, `App.js`에서 `StartGameScreen`을 앱의 루트 화면으로 표시했다. 다음 단계에서는 이 버튼과 화면에 스타일을 추가해 실제 앱 UI에 가까워지도록 다듬는다.
