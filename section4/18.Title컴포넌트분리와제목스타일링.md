# 18. Title 컴포넌트 분리와 제목 스타일링

## 핵심 요약

- `GameScreen`의 `Opponent's Guess` 텍스트는 일반 텍스트가 아니라 화면 제목 역할을 한다.
- 제목처럼 반복될 가능성이 있는 UI는 별도의 재사용 컴포넌트로 분리하는 것이 좋다.
- React Native에서는 텍스트 스타일을 `Text` 컴포넌트에 직접 적용해야 한다.
- `props.children`을 사용하면 커스텀 컴포넌트의 여는 태그와 닫는 태그 사이에 들어온 내용을 렌더링할 수 있다.
- 스타일도 해당 컴포넌트와 가까운 파일로 옮기면 컴포넌트의 책임이 명확해진다.
- UI 결과가 같더라도 컴포넌트 구조를 정리하면 이후 화면 확장과 재사용이 쉬워진다.

## 학습 목표

- `GameScreen`의 제목 텍스트에 스타일을 적용하는 방법을 익힌다.
- 제목 UI를 재사용 가능한 컴포넌트로 분리하는 이유를 이해한다.
- `children` 프로퍼티를 이용해 커스텀 컴포넌트의 내용을 유연하게 구성한다.
- 스타일을 사용하는 위치에서 컴포넌트 내부로 이동시키는 흐름을 익힌다.
- React 웹 개발에서 하던 컴포넌트 분리 패턴이 React Native에서도 그대로 적용됨을 확인한다.

## 주요 내용

### GameScreen 제목 스타일링

`GameScreen`에는 상대방, 즉 앱이 추측한 숫자를 보여주는 영역이 필요하다.

그 위에 표시되는 `Opponent's Guess`는 단순한 문장이 아니라 화면 섹션의 제목이다.

```jsx
<Text>Opponent's Guess</Text>
```

기본 텍스트 상태로 두면 글자가 작고 강조가 약하다.

따라서 제목답게 보이도록 스타일을 추가한다.

```jsx
<Text style={styles.title}>Opponent's Guess</Text>
```

### title 스타일 정의

`GameScreen`의 `StyleSheet`에 `title` 스타일을 추가한다.

```jsx
const styles = StyleSheet.create({
  screen: {
    flex: 1,
    padding: 24,
  },
  title: {
    fontSize: 24,
    fontWeight: 'bold',
    color: '#ddb52f',
    textAlign: 'center',
    borderWidth: 2,
    borderColor: '#ddb52f',
    padding: 12,
  },
});
```

각 스타일의 역할은 다음과 같다.

```text
fontSize
-> 제목답게 큰 글자 크기 지정

fontWeight
-> 굵은 글씨로 강조

color
-> 앱의 기존 노란색 테마와 맞춤

textAlign
-> 텍스트 중앙 정렬

borderWidth, borderColor
-> 제목 영역을 시각적으로 구분

padding
-> 경계선과 텍스트 사이 여백 확보
```

이 스타일을 적용하면 제목이 훨씬 명확하게 보인다.

### 제목 스타일은 Text에 직접 적용한다

React Native에서는 스타일 상속이 CSS처럼 동작하지 않는다.

따라서 텍스트 색상, 정렬, 글자 크기 같은 스타일은 실제 `Text` 컴포넌트에 적용해야 한다.

```jsx
<Text style={styles.title}>Opponent's Guess</Text>
```

부모 `View`에 `color`를 지정해도 내부 `Text`의 글자 색이 자동으로 바뀌지 않는다.

```text
웹 CSS
-> 부모 color가 자식 텍스트에 상속될 수 있음

React Native
-> Text 스타일은 Text 컴포넌트에 직접 지정하는 방식이 안전함
```

### 반복될 UI는 컴포넌트로 분리한다

제목 UI는 `GameScreen`에서만 쓰고 끝날 수도 있지만, 보통 앱에서는 여러 화면에서 비슷한 제목이 반복된다.

```text
StartGameScreen
-> 게임 시작 제목

GameScreen
-> 상대방 추측 제목

GameOverScreen
-> 게임 종료 제목
```

이런 경우 매번 같은 `Text`와 스타일을 복사하는 것보다, 재사용 가능한 컴포넌트로 분리하는 편이 좋다.

React에서 자주 쓰는 일반적인 컴포넌트 분리 기준이다.

```text
같은 UI가 반복될 가능성이 있음
-> 컴포넌트로 분리

스타일과 구조가 함께 의미를 가짐
-> 컴포넌트로 분리

화면 컴포넌트가 너무 많은 UI 세부사항을 가짐
-> 하위 컴포넌트로 분리
```

### Title.js 파일 추가

`components` 폴더에 `Title.js` 파일을 만든다.

```text
components/
  Title.js
```

`Title`은 화면 제목을 렌더링하는 재사용 컴포넌트다.

```jsx
function Title() {
  return (
    <Text style={styles.title}>Opponent's Guess</Text>
  );
}

export default Title;
```

하지만 이렇게 작성하면 항상 같은 문구만 렌더링한다.

재사용 가능한 컴포넌트로 만들려면 제목 내용을 외부에서 주입받아야 한다.

### children으로 제목 내용 받기

React 컴포넌트는 여는 태그와 닫는 태그 사이의 내용을 `children` 프로퍼티로 받을 수 있다.

```jsx
<Title>Opponent's Guess</Title>
```

이때 `Title` 컴포넌트에서는 `children`을 받아서 `Text` 안에 출력한다.

```jsx
function Title({ children }) {
  return (
    <Text style={styles.title}>{children}</Text>
  );
}

export default Title;
```

여기서 사용한 문법은 객체 구조 분해다.

```jsx
function Title(props) {
  return (
    <Text>{props.children}</Text>
  );
}
```

위 코드를 더 간결하게 작성하면 다음과 같다.

```jsx
function Title({ children }) {
  return (
    <Text>{children}</Text>
  );
}
```

React Native 전용 문법이 아니라 React 컴포넌트의 일반적인 사용 방식이다.

### Title 컴포넌트 전체 예시

`Title.js`는 다음처럼 구성할 수 있다.

```jsx
import { StyleSheet, Text } from 'react-native';

function Title({ children }) {
  return (
    <Text style={styles.title}>{children}</Text>
  );
}

export default Title;

const styles = StyleSheet.create({
  title: {
    fontSize: 24,
    fontWeight: 'bold',
    color: '#ddb52f',
    textAlign: 'center',
    borderWidth: 2,
    borderColor: '#ddb52f',
    padding: 12,
  },
});
```

이제 제목과 관련된 JSX와 스타일이 모두 `Title` 컴포넌트 안에 모인다.

### GameScreen에서 Title 사용하기

`GameScreen.js`에서는 기존 `Text`를 제거하고 `Title` 컴포넌트를 사용한다.

```jsx
import Title from '../components/Title';
```

그리고 JSX에서 다음처럼 작성한다.

```jsx
<Title>Opponent's Guess</Title>
```

전체 구조는 대략 다음과 같다.

```jsx
import { StyleSheet, View } from 'react-native';

import Title from '../components/Title';

function GameScreen() {
  return (
    <View style={styles.screen}>
      <Title>Opponent's Guess</Title>

      {/* GUESS */}

      <View>
        {/* Higher or lower? */}
        {/* + / - buttons */}
      </View>

      <View>
        {/* Log rounds */}
      </View>
    </View>
  );
}

export default GameScreen;

const styles = StyleSheet.create({
  screen: {
    flex: 1,
    padding: 24,
  },
});
```

`title` 스타일은 이제 `Title.js`로 이동했으므로 `GameScreen.js`에는 남겨둘 필요가 없다.

### 스타일을 컴포넌트로 옮기는 이유

처음에는 `GameScreen`에서 직접 스타일링했다.

```text
GameScreen
-> title 스타일 보유
-> Text 직접 렌더링
```

하지만 `Title` 컴포넌트로 분리한 뒤에는 제목과 관련된 책임이 한 곳으로 이동한다.

```text
Title
-> 제목 Text 렌더링
-> 제목 스타일 관리
-> children으로 제목 문구 수신
```

이렇게 하면 `GameScreen`은 게임 화면의 큰 구조에만 집중할 수 있다.

```text
GameScreen의 관심사
-> 게임 화면 구성
-> 추측 숫자 표시
-> 힌트 버튼
-> 라운드 기록

Title의 관심사
-> 제목을 어떻게 보여줄지
```

컴포넌트 경계가 더 명확해진다.

### UI는 같지만 구조는 좋아졌다

분리 전후로 화면에 보이는 결과는 거의 같다.

```text
분리 전
-> Text가 GameScreen 안에 직접 있음

분리 후
-> Title 컴포넌트가 Text를 감싸고 있음
```

하지만 유지보수 관점에서는 차이가 크다.

```text
제목 스타일 변경 필요
-> Title.js 한 곳만 수정

다른 화면에서 같은 제목 스타일 필요
-> <Title>...</Title> 재사용

화면 컴포넌트가 복잡해짐
-> 세부 UI를 하위 컴포넌트로 분리 가능
```

React Native에서도 결국 React 앱을 만드는 것이므로, 이런 컴포넌트 설계 습관이 중요하다.

### 프론트엔드 개발자 관점에서의 핵심 차이

React 웹 개발자라면 이 패턴은 익숙하다.

```jsx
function PageTitle({ children }) {
  return <h1>{children}</h1>;
}
```

React Native에서는 `h1` 같은 HTML 태그를 사용할 수 없으므로 `Text`로 직접 만든다.

```jsx
function Title({ children }) {
  return (
    <Text style={styles.title}>{children}</Text>
  );
}
```

개념은 같다.

```text
웹 React
-> h1, div, button 같은 HTML 요소 조합

React Native
-> Text, View, Pressable 같은 네이티브 핵심 컴포넌트 조합
```

차이는 기반 요소뿐이다.

컴포넌트를 나누고, `children`으로 내용을 주입하고, 스타일을 컴포넌트와 가까운 곳에 두는 설계 방식은 동일하다.

## 정리

이번 강의에서는 `GameScreen`의 `Opponent's Guess` 텍스트를 제목답게 스타일링한 뒤, 이를 재사용 가능한 `Title` 컴포넌트로 분리했다.

핵심은 단순히 스타일을 예쁘게 만드는 것이 아니라, 반복될 수 있는 UI를 독립적인 컴포넌트로 추출하는 것이다. `children`을 활용하면 제목 문구를 외부에서 자유롭게 전달할 수 있고, 제목 스타일은 `Title.js` 내부에 모아 관리할 수 있다.

React Native에서도 컴포넌트 설계의 기본 원칙은 React 웹과 같다. 다만 HTML 요소 대신 `Text`, `View` 같은 React Native 핵심 컴포넌트를 기반으로 직접 UI를 구성한다.
