# 10. 루트 View로 앱 배경색 설정

## 핵심 요약

- 앱 전체 배경을 만들기 위해 `App.js`에서 화면 컴포넌트를 루트 `View`로 감싼다.
- `app.json`의 `backgroundColor`로 배경색을 설정할 수도 있지만, 이번 강의에서는 `View` 스타일로 배경을 제어한다.
- 여러 화면이 추가되더라도 `App` 컴포넌트는 공통 루트이므로, 루트 `View`에 적용한 배경은 앱 화면 전체의 공통 배경이 될 수 있다.
- `View`는 기본적으로 콘텐츠가 필요한 만큼의 공간만 차지한다.
- 화면 전체에 배경색을 채우려면 루트 `View`에 `flex: 1`을 적용해야 한다.

## 학습 목표

- `App.js`에서 루트 화면을 감싸는 `View`를 추가하는 방법을 익힌다.
- React Native에서 `View`의 기본 크기 동작을 이해한다.
- `flex: 1`이 화면 전체 배경을 만드는 데 왜 필요한지 배운다.
- `app.json` 배경 설정과 컴포넌트 기반 배경 설정의 차이를 이해한다.
- 이후 그러데이션과 배경 이미지를 적용하기 위한 기본 구조를 준비한다.

## 주요 내용

### 배경색을 설정하는 두 가지 방법

React Native와 Expo 프로젝트에서 배경색을 설정하는 방법은 여러 가지가 있다.

대표적으로는 `app.json`에 `backgroundColor`를 설정하는 방식이 있다.

```json
{
  "expo": {
    "backgroundColor": "#ddb52f"
  }
}
```

이 방식은 Expo 설정을 통해 앱의 기본 배경색을 지정할 때 유용하다.

하지만 이번 강의에서는 컴포넌트 트리 안에서 직접 배경을 제어한다.

이후 그러데이션과 배경 이미지를 함께 사용할 예정이기 때문에, `View`를 이용해 배경 영역을 만드는 방식으로 시작한다.

### App.js에서 루트 View 추가하기

현재 `App.js`는 `StartGameScreen`을 바로 렌더링하고 있다.

```jsx
export default function App() {
  return <StartGameScreen />;
}
```

앱 전체 배경을 제어하려면 `StartGameScreen`을 `View`로 감싼다.

```jsx
import { View } from 'react-native';

import StartGameScreen from './screens/StartGameScreen';

export default function App() {
  return (
    <View>
      <StartGameScreen />
    </View>
  );
}
```

`View`를 사용했으므로 `react-native`에서 import해야 한다.

React Native에서는 `View`, `Text`, `TextInput`, `Pressable` 같은 핵심 컴포넌트를 사용할 때마다 직접 import해야 한다.

### 루트 View에 스타일 적용하기

루트 `View`에 배경색을 적용하기 위해 `StyleSheet`를 사용한다.

```jsx
import { StyleSheet, View } from 'react-native';
```

그리고 `rootScreen` 스타일 객체를 만든다.

```jsx
const styles = StyleSheet.create({
  rootScreen: {
    backgroundColor: '#ddb52f',
  },
});
```

이 스타일을 루트 `View`에 연결한다.

```jsx
export default function App() {
  return (
    <View style={styles.rootScreen}>
      <StartGameScreen />
    </View>
  );
}
```

여기서 사용한 `#ddb52f`는 이전에 숫자 입력 필드의 강조색으로 사용했던 노란색 계열이다.

### 배경색이 화면 일부에만 적용되는 이유

위 코드만 적용하면 배경색이 화면 전체가 아니라 위쪽 일부에만 적용될 수 있다.

이유는 `View`의 기본 크기 동작 때문이다.

React Native의 `View`는 기본적으로 자식 콘텐츠가 들어갈 만큼의 공간만 차지한다.

```text
루트 View의 자식
-> StartGameScreen

StartGameScreen이 차지하는 영역
-> 입력 카드 + 바깥 여백 정도

루트 View의 높이
-> 자식 콘텐츠가 필요한 만큼만
```

그래서 루트 `View`에 배경색을 주더라도, 루트 `View` 자체가 화면 전체를 차지하지 않으면 배경색도 일부 영역에만 보인다.

이 부분은 React Native 레이아웃에서 매우 중요하다.

```text
배경색이 화면 전체에 적용되지 않는 문제
-> backgroundColor 문제가 아님
-> View가 화면 전체 높이를 차지하지 않는 문제
```

### flex: 1로 화면 전체 채우기

루트 `View`가 사용 가능한 전체 공간을 차지하게 하려면 `flex: 1`을 추가한다.

```jsx
const styles = StyleSheet.create({
  rootScreen: {
    flex: 1,
    backgroundColor: '#ddb52f',
  },
});
```

`flex: 1`은 해당 요소가 부모로부터 받을 수 있는 공간을 최대한 차지하도록 만든다.

루트 화면에서는 사실상 기기 화면 전체를 차지하게 된다.

```text
flex 없음
-> 콘텐츠 높이만큼만 View가 커짐

flex: 1
-> 사용 가능한 화면 전체를 View가 채움
```

이제 배경색이 입력 카드 주변 일부가 아니라 화면 전체에 적용된다.

### 전체 App.js 예시

이번 단계의 `App.js`는 다음과 같은 형태가 된다.

```jsx
import { StyleSheet, View } from 'react-native';

import StartGameScreen from './screens/StartGameScreen';

export default function App() {
  return (
    <View style={styles.rootScreen}>
      <StartGameScreen />
    </View>
  );
}

const styles = StyleSheet.create({
  rootScreen: {
    flex: 1,
    backgroundColor: '#ddb52f',
  },
});
```

이 구조는 이후 여러 화면이 추가될 때도 계속 의미가 있다.

예를 들어 게임 앱에는 다음 화면들이 추가될 예정이다.

```text
StartGameScreen
GameScreen
GameOverScreen
```

이 화면들을 어떤 조건에 따라 교체하더라도 모두 `App` 컴포넌트 아래에서 렌더링된다.

따라서 루트 `View`의 배경 설정은 화면 전체에 공통으로 적용될 수 있다.

### StartGameScreen에 배경을 두지 않는 이유

배경색을 `StartGameScreen` 안에 직접 설정할 수도 있다.

하지만 앞으로 여러 화면을 만들 예정이라면 각 화면마다 같은 배경 설정을 반복해야 할 수 있다.

```text
StartGameScreen에 배경 설정
-> 시작 화면에만 적용
-> GameScreen, GameOverScreen에는 별도 설정 필요

App 루트 View에 배경 설정
-> 모든 화면을 감싸는 공통 배경으로 사용 가능
```

공통 레이아웃이나 앱 전체에 적용될 배경은 루트 컴포넌트 근처에 두는 편이 관리하기 좋다.

### 다음 단계: 그러데이션 배경

이번 강의에서는 단일 배경색만 적용했다.

하지만 최종 목표는 단순한 색상이 아니라 더 풍부한 배경이다.

앞으로 진행할 작업은 다음과 같다.

```text
1. 단일 배경색 적용
2. 그러데이션 배경으로 변경
3. 배경 이미지와 그러데이션 결합
```

이번 단계의 루트 `View` 구조는 이후 그러데이션과 배경 이미지를 적용하기 위한 기반이 된다.

### 프론트엔드 개발자 관점에서의 핵심 차이

웹에서는 `body`나 최상위 `div`에 배경색을 주고 `min-height: 100vh` 같은 CSS를 사용해 전체 화면 배경을 만드는 일이 많다.

React Native에서는 `body`가 없고, 화면 전체를 차지하는 컨테이너를 직접 만들어야 한다.

```text
웹
-> body
-> min-height: 100vh
-> background-color

React Native
-> root View
-> flex: 1
-> backgroundColor
```

즉 React Native에서 전체 화면 배경을 만들 때는 “어떤 `View`가 화면 전체를 차지하는가”를 먼저 확인해야 한다.

배경색 자체보다 더 중요한 것은 배경색을 가진 컨테이너가 실제로 화면 전체를 덮고 있는지다.

## 정리

이번 강의에서는 `App.js`에서 `StartGameScreen`을 루트 `View`로 감싸고, 해당 `View`에 `backgroundColor`를 적용해 앱 배경색을 만들었다. 처음에는 배경색이 일부 영역에만 보였는데, 그 이유는 `View`가 기본적으로 콘텐츠가 필요한 만큼의 공간만 차지하기 때문이다.

이를 해결하기 위해 루트 `View`에 `flex: 1`을 추가했고, 그 결과 배경색이 화면 전체에 적용되었다. 이 구조는 이후 여러 화면을 추가하더라도 공통 배경을 제공할 수 있으며, 다음 단계에서 그러데이션과 배경 이미지를 적용하기 위한 기반이 된다.
