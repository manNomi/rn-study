# 28. 중첩 Text 강조와 새 게임 버튼

## 핵심 요약

- React Native에서는 `Text` 컴포넌트 안에 또 다른 `Text` 컴포넌트를 중첩할 수 있다.
- 이 방식으로 문장 일부만 다른 스타일로 강조할 수 있다.
- `View`는 `Text` 안에 넣을 수 없지만, `Text` 안의 `Text`는 허용된다.
- 게임 오버 화면의 결과 문장에서 라운드 수 `X`와 정답 숫자 `Y`를 별도 `Text`로 감싸 강조한다.
- 전체 결과 문장에는 `summaryText` 스타일을 적용하고, 강조 부분에는 `highlight` 스타일을 적용한다.
- 부모 `Text`에 적용한 `fontSize`는 중첩된 `Text`에도 영향을 줄 수 있다.
- 결과 텍스트 아래에 `PrimaryButton`을 추가해 새 게임 시작 버튼을 배치한다.

## 학습 목표

- React Native에서 `Text` 컴포넌트 중첩이 가능한 이유와 사용 방법을 이해한다.
- 문장 일부에만 다른 스타일을 적용하는 패턴을 익힌다.
- `Text` 스타일과 `View` 스타일의 동작 차이를 복습한다.
- 게임 오버 화면의 결과 요약 텍스트를 스타일링하는 방법을 배운다.
- `PrimaryButton`을 재사용해 “Start New Game” 버튼을 추가하는 흐름을 익힌다.
- 아직 동적 데이터와 버튼 동작은 다음 단계에서 연결된다는 점을 이해한다.

## 주요 내용

### 결과 문장에서 강조할 부분

게임 오버 화면에는 결과 요약 문장이 들어간다.

```text
Your phone needed X rounds to guess the number Y.
```

여기서 강조하고 싶은 값은 두 가지다.

```text
X
-> 장치가 정답을 맞히기까지 걸린 라운드 수

Y
-> 사용자가 선택한 정답 숫자
```

현재는 `X`, `Y`가 플레이스홀더지만, 나중에 실제 값으로 교체된다.

우선 스타일상 이 두 값이 문장 안에서 눈에 띄도록 강조한다.

### Text 안에 Text 중첩하기

React Native에서는 `Text` 컴포넌트 안에 다른 `Text` 컴포넌트를 넣을 수 있다.

```jsx
<Text>
  Your phone needed <Text>X</Text> rounds to guess the number <Text>Y</Text>.
</Text>
```

이 패턴을 사용하면 문장 일부에만 별도 스타일을 줄 수 있다.

```text
외부 Text
-> 전체 문장 스타일

내부 Text
-> 특정 단어 또는 값 강조 스타일
```

이 방식은 HTML에서 문장 일부를 `span`으로 감싸는 것과 비슷하게 이해할 수 있다.

### 모든 컴포넌트를 Text 안에 넣을 수 있는 것은 아니다

중요한 제약이 있다.

`Text` 안에는 또 다른 `Text`를 넣을 수 있지만, `View` 같은 레이아웃 컴포넌트를 넣을 수는 없다.

```jsx
// 가능
<Text>
  Hello <Text>World</Text>
</Text>
```

```jsx
// 부적절
<Text>
  Hello <View />
</Text>
```

React Native에서는 텍스트 콘텐츠와 레이아웃 컨테이너의 역할이 명확히 구분된다.

```text
Text
-> 텍스트 출력
-> 텍스트 안의 텍스트 중첩 가능

View
-> 레이아웃 컨테이너
-> Text 안에 넣는 용도가 아님
```

### 전체 요약 텍스트 스타일 추가하기

먼저 전체 결과 문장에 적용할 스타일을 만든다.

```jsx
summaryText: {
  fontFamily: 'open-sans',
  fontSize: 24,
  textAlign: 'center',
}
```

각 속성의 역할은 다음과 같다.

```text
fontFamily: 'open-sans'
-> 일반 OpenSans 글꼴 적용

fontSize: 24
-> 결과 문장을 크게 표시

textAlign: 'center'
-> 문장을 중앙 정렬
```

게임 오버 화면은 중앙 정렬된 결과 화면이므로 텍스트도 중앙 정렬하는 편이 자연스럽다.

### 강조 텍스트 스타일 추가하기

`X`와 `Y`는 강조해야 하므로 별도 스타일을 만든다.

```jsx
highlight: {
  fontFamily: 'open-sans-bold',
  color: Colors.primary500,
}
```

각 속성의 역할은 다음과 같다.

```text
fontFamily: 'open-sans-bold'
-> 강조 값을 굵게 표시

color: Colors.primary500
-> 앱의 primary 색상으로 강조
```

`Colors`를 사용하므로 `GameOverScreen`에서 색상 상수를 import해야 한다.

```jsx
import Colors from '../constants/colors';
```

### 스타일 적용하기

외부 `Text`에는 `summaryText`를 적용한다.

내부 `Text`에는 `highlight`를 적용한다.

```jsx
<Text style={styles.summaryText}>
  Your phone needed{' '}
  <Text style={styles.highlight}>X</Text>
  {' '}rounds to guess the number{' '}
  <Text style={styles.highlight}>Y</Text>.
</Text>
```

문장 안의 공백이 어색해지지 않도록 문자열 사이 간격을 신경 써야 한다.

코드 포맷에 따라 `{ ' ' }` 또는 `{' '}`를 활용할 수 있다.

```text
summaryText
-> 전체 문장 스타일

highlight
-> X, Y 강조 스타일
```

### 중첩 Text의 fontSize 동작

외부 `Text`에 `fontSize: 24`를 적용하면, 내부 중첩 `Text`도 같은 크기로 보인다.

이는 일반적인 `View` 스타일 상속과는 다르다.

```jsx
<Text style={{ fontSize: 24 }}>
  Normal <Text>Nested</Text>
</Text>
```

이 경우 내부 `Text`도 외부 텍스트 흐름 안에서 함께 렌더링되므로 같은 텍스트 스타일의 영향을 받을 수 있다.

하지만 `View`에 `fontSize`를 준다고 그 안의 `Text`가 영향을 받지는 않는다.

```jsx
<View style={{ fontSize: 24 }}>
  <Text>Hello</Text>
</View>
```

이 방식은 기대대로 동작하지 않는다.

정리하면 다음과 같다.

```text
View 안의 Text
-> View의 텍스트 스타일이 자동 상속되지 않음

Text 안의 Text
-> 같은 텍스트 흐름 안에서 일부 텍스트 스타일 영향을 받을 수 있음
```

따라서 React Native의 스타일링은 CSS 상속과 같다고 생각하면 안 된다.

다만 `Text` 중첩은 텍스트 렌더링 특성상 예외적으로 문장 일부 스타일링에 유용하다.

### 결과 텍스트 여백 조정하기

결과 텍스트 아래에는 새 게임 시작 버튼을 추가할 예정이므로, 텍스트와 버튼 사이에 간격이 필요하다.

처음에는 위아래 모두 여백을 줄 수 있다.

```jsx
summaryText: {
  marginVertical: 24,
}
```

하지만 이미 이미지와 텍스트 사이의 간격이 충분하다면 아래쪽 여백만 주는 편이 더 자연스러울 수 있다.

```jsx
summaryText: {
  marginBottom: 24,
}
```

이번 화면에서는 버튼과 결과 텍스트 사이 간격이 필요하므로 `marginBottom`이 적절하다.

```text
marginVertical
-> 위아래 모두 여백

marginBottom
-> 아래쪽에만 여백
```

화면을 보면서 더 자연스러운 값을 선택한다.

### Start New Game 버튼 추가하기

게임 오버 화면 하단에는 새 게임을 시작하는 버튼이 필요하다.

이미 만든 `PrimaryButton`을 재사용한다.

```jsx
import PrimaryButton from '../components/ui/PrimaryButton';
```

결과 텍스트 아래에 버튼을 추가한다.

```jsx
<PrimaryButton>Start New Game</PrimaryButton>
```

아직 버튼 동작은 연결하지 않는다.

나중에 `onPress` prop을 추가해 새 게임을 시작하는 함수를 연결한다.

```text
이번 단계
-> 버튼 UI 추가

다음 단계
-> 버튼 동작 연결
-> 게임 상태 초기화
```

### GameOverScreen 구조 예시

이번 단계의 `GameOverScreen`은 다음처럼 정리할 수 있다.

```jsx
import { Image, StyleSheet, Text, View } from 'react-native';

import PrimaryButton from '../components/ui/PrimaryButton';
import Title from '../components/ui/Title';
import Colors from '../constants/colors';

function GameOverScreen() {
  return (
    <View style={styles.rootContainer}>
      <Title>GAME OVER!</Title>

      <View style={styles.imageContainer}>
        <Image
          source={require('../assets/images/success.png')}
          style={styles.image}
        />
      </View>

      <Text style={styles.summaryText}>
        Your phone needed{' '}
        <Text style={styles.highlight}>X</Text>
        {' '}rounds to guess the number{' '}
        <Text style={styles.highlight}>Y</Text>.
      </Text>

      <PrimaryButton>Start New Game</PrimaryButton>
    </View>
  );
}
```

스타일은 다음처럼 추가된다.

```jsx
const styles = StyleSheet.create({
  summaryText: {
    fontFamily: 'open-sans',
    fontSize: 24,
    textAlign: 'center',
    marginBottom: 24,
  },
  highlight: {
    fontFamily: 'open-sans-bold',
    color: Colors.primary500,
  },
});
```

### 아직 남은 작업

게임 오버 화면의 레이아웃과 스타일은 거의 완성되었다.

하지만 아직 실제 데이터와 동작이 연결되지 않았다.

```text
아직 남은 것
-> X를 실제 라운드 수로 대체
-> Y를 실제 선택 숫자로 대체
-> Start New Game 버튼 동작 연결
-> 게임 상태 초기화
```

이번 강의에서는 시각적 구조와 텍스트 강조를 완성하고, 데이터 연결은 다음 단계로 남겨둔다.

### 프론트엔드 개발자 관점에서의 핵심 차이

웹에서는 문장 일부를 강조할 때 보통 `span`을 사용한다.

```jsx
<p>
  Your phone needed <span className="highlight">X</span> rounds.
</p>
```

React Native에는 `span`이 없다.

대신 `Text` 안에 또 다른 `Text`를 중첩한다.

```jsx
<Text>
  Your phone needed <Text style={styles.highlight}>X</Text> rounds.
</Text>
```

개념적으로는 매우 비슷하다.

```text
웹
-> p 안의 span으로 일부 텍스트 강조

React Native
-> Text 안의 Text로 일부 텍스트 강조
```

하지만 `View`를 `Text` 안에 넣을 수 없다는 점은 기억해야 한다.

```text
문장 일부 스타일링
-> Text 중첩

레이아웃 분리
-> View 사용
```

### 실무 관점에서의 포인트

결과 화면, 온보딩 문구, 약관 안내, 가격 표시처럼 문장 안에서 일부 값만 강조해야 하는 경우가 많다.

```text
예시
-> 총 3개의 항목을 선택했습니다.
-> 이번 달 지출은 120,000원입니다.
-> 7일 무료 체험이 시작됩니다.
```

이럴 때 React Native에서는 중첩 `Text` 패턴이 유용하다.

```jsx
<Text style={styles.baseText}>
  이번 달 지출은{' '}
  <Text style={styles.highlight}>120,000원</Text>
  입니다.
</Text>
```

문장 일부 강조가 필요할 때는 별도 레이아웃을 만들기보다 먼저 `Text` 중첩을 고려하면 된다.

## 정리

이번 강의에서는 게임 오버 화면의 결과 문장에서 `X`와 `Y`를 강조 표시했다. React Native에서는 `Text` 안에 또 다른 `Text`를 중첩할 수 있으므로, 문장 일부에만 별도 스타일을 적용할 수 있다.

전체 문장에는 `summaryText` 스타일을 적용해 OpenSans 글꼴, 큰 글자 크기, 중앙 정렬을 지정했고, 강조 값에는 `highlight` 스타일을 적용해 굵은 글꼴과 primary 색상을 사용했다.

마지막으로 결과 텍스트 아래에 `PrimaryButton`을 재사용해 `Start New Game` 버튼을 추가했다. 아직 실제 라운드 수, 정답 숫자, 버튼 동작은 연결되지 않았고, 다음 단계에서 게임 상태와 함께 처리하게 된다.
