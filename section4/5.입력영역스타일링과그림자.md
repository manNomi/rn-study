# 5. 입력 영역 스타일링과 그림자

## 핵심 요약

- `StartGameScreen`의 입력 영역을 화면 가장자리에서 떨어진 카드 형태로 스타일링한다.
- `StyleSheet.create()`로 `inputContainer` 스타일 객체를 만들고 `View`에 적용한다.
- `flex: 1`은 가능한 공간을 모두 차지하게 하므로, 작은 카드 형태를 원할 때는 제거해야 한다.
- Android에서는 `elevation`으로 그림자를 추가한다.
- iOS에서는 `shadowColor`, `shadowOffset`, `shadowRadius`, `shadowOpacity`를 조합해 그림자를 추가한다.

## 학습 목표

- 입력 필드와 버튼을 감싸는 컨테이너를 카드 형태로 만드는 방법을 익힌다.
- `padding`, `marginTop`, `marginHorizontal`, `borderRadius`, `backgroundColor`를 활용한 기본 박스 스타일링을 복습한다.
- `flex: 1`이 레이아웃에 미치는 영향을 이해한다.
- Android와 iOS에서 그림자를 추가하는 방식이 다르다는 점을 배운다.
- React Native 스타일이 결국 각 플랫폼의 네이티브 스타일로 변환된다는 점을 이해한다.

## 주요 내용

### 입력 영역 위치 조정하기

현재 `TextInput`과 버튼은 기기 가장자리에 가까워 누르기 불편한 상태다. 입력 영역을 화면 안쪽으로 옮기기 위해 `StartGameScreen.js`에서 `StyleSheet`를 사용한다.

```jsx
import { StyleSheet, TextInput, View } from 'react-native';
```

스타일 객체를 만들고 입력 영역을 감싸는 `View`에 적용한다.

```jsx
const styles = StyleSheet.create({
  inputContainer: {
    padding: 16,
    marginTop: 100,
  },
});
```

```jsx
<View style={styles.inputContainer}>
  <TextInput />
  <PrimaryButton>Reset</PrimaryButton>
  <PrimaryButton>Confirm</PrimaryButton>
</View>
```

`padding`은 컨테이너 내부 여백을 만들고, `marginTop`은 컨테이너를 화면 위쪽 가장자리에서 떨어뜨린다.

### flex: 1의 영향

처음에는 `inputContainer`에 `flex: 1`을 추가할 수 있다.

```jsx
inputContainer: {
  flex: 1,
  padding: 16,
  marginTop: 100,
},
```

`flex: 1`은 해당 요소가 가능한 공간을 최대한 차지하도록 만든다. 이 화면에서는 입력 컨테이너가 유일한 주요 컨테이너이기 때문에 거의 화면 전체를 차지하게 된다.

입력 필드와 버튼 주변의 작은 카드 형태를 만들고 싶다면 `flex: 1`은 제거해야 한다.

```jsx
inputContainer: {
  padding: 16,
  marginTop: 100,
},
```

이렇게 하면 컨테이너는 내부 콘텐츠와 여백에 필요한 만큼만 공간을 차지한다.

### 배경색으로 입력 영역 구분하기

입력 영역을 나머지 화면과 구분하기 위해 배경색을 추가한다.

```jsx
inputContainer: {
  padding: 16,
  marginTop: 100,
  backgroundColor: '#27063c',
},
```

`flex: 1`이 남아 있으면 배경색이 화면 대부분을 덮게 된다. 카드처럼 보이려면 `flex: 1` 없이 콘텐츠 크기만큼만 배경을 차지하게 해야 한다.

### 좌우 여백과 둥근 모서리

입력 카드가 화면 좌우 가장자리에도 붙지 않도록 `marginHorizontal`을 추가한다.

```jsx
inputContainer: {
  padding: 16,
  marginTop: 100,
  marginHorizontal: 24,
  backgroundColor: '#27063c',
  borderRadius: 8,
},
```

`marginHorizontal`은 좌우 바깥 여백을 한 번에 설정한다.

`borderRadius`는 카드의 모서리를 둥글게 만든다.

### Android 그림자: elevation

React Native에는 웹 CSS의 `box-shadow`가 없다.

Android에서는 `elevation` 프로퍼티를 사용해 그림자를 만든다.

```jsx
inputContainer: {
  padding: 16,
  marginTop: 100,
  marginHorizontal: 24,
  backgroundColor: '#27063c',
  borderRadius: 8,
  elevation: 4,
},
```

`elevation`은 Android 전용 스타일 프로퍼티다. 값이 클수록 요소가 더 위에 떠 있는 것처럼 보이고 그림자도 더 뚜렷해진다.

```text
elevation: 0
-> 그림자 없음

elevation: 4
-> 약한 그림자

elevation: 8
-> 더 강한 그림자
```

### iOS 그림자: shadow 계열 프로퍼티

iOS에서는 `elevation`이 동작하지 않는다. iOS에는 Android의 `elevation`과 같은 개념이 없기 때문이다.

iOS에서 그림자를 만들려면 `shadow`로 시작하는 여러 프로퍼티를 함께 사용한다.

```jsx
inputContainer: {
  padding: 16,
  marginTop: 100,
  marginHorizontal: 24,
  backgroundColor: '#27063c',
  borderRadius: 8,
  elevation: 4,
  shadowColor: 'black',
  shadowOffset: { width: 0, height: 2 },
  shadowRadius: 6,
  shadowOpacity: 0.25,
},
```

각 프로퍼티의 역할은 다음과 같다.

| 프로퍼티 | 역할 |
| --- | --- |
| `shadowColor` | 그림자 색상 |
| `shadowOffset` | 그림자가 요소로부터 얼마나 이동하는지 |
| `shadowRadius` | 그림자가 얼마나 번지는지 |
| `shadowOpacity` | 그림자 투명도 |

예를 들어 `shadowOpacity`를 `1`로 높이면 iOS에서 그림자가 훨씬 강하게 보인다. 강의에서는 자연스러운 약한 그림자를 위해 `0.25`를 사용한다.

### 플랫폼별 스타일 차이 이해하기

React Native 스타일은 JavaScript 객체로 작성하지만, 최종적으로는 각 플랫폼의 네이티브 스타일 명령으로 변환된다.

많은 스타일 프로퍼티는 iOS와 Android 양쪽에서 비슷하게 동작한다. 하지만 그림자처럼 플랫폼마다 개념이 다른 경우도 있다.

```text
Android 그림자
-> elevation

iOS 그림자
-> shadowColor
-> shadowOffset
-> shadowRadius
-> shadowOpacity
```

따라서 React Native에서 스타일링할 때는 한 플랫폼에서 잘 보인다고 끝내지 말고, 가능하면 iOS와 Android 양쪽에서 확인해야 한다.

### 현재 단계의 결과

이번 단계에서는 앱 기능을 구현하지는 않았다.

아직 남은 작업은 다음과 같다.

```text
버튼 클릭 기능 연결
TextInput 값 처리
입력 검증
게임 시작 처리
전체 화면 배경 스타일링
```

하지만 입력 영역은 이제 단순한 요소 묶음이 아니라, 배경색과 둥근 모서리, 그림자를 가진 카드 형태에 가까워졌다.

## 정리

이번 강의에서는 `StartGameScreen`의 입력 영역을 카드처럼 보이도록 스타일링했다. `padding`, `marginTop`, `marginHorizontal`, `backgroundColor`, `borderRadius`로 기본 박스 형태를 만들고, Android에서는 `elevation`, iOS에서는 `shadow*` 프로퍼티로 그림자를 추가했다.

특히 그림자처럼 플랫폼별로 다른 스타일 처리 방식이 있다는 점이 중요하다. React Native는 하나의 코드로 여러 플랫폼을 다루지만, 내부적으로는 각 플랫폼의 네이티브 스타일 규칙으로 변환되므로 iOS와 Android의 차이를 이해해야 한다.
