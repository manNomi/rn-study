# 20. 초기 난수 생성과 NumberContainer

## 핵심 요약

- 게임 화면에서는 앱이 사용자가 고른 숫자를 추측해야 한다.
- 최초 추측값은 `Math.random()`을 이용해 1부터 99 사이의 난수로 만든다.
- 사용자가 고른 숫자는 첫 추측에서 제외해 게임이 바로 끝나지 않도록 한다.
- 추측값은 화면에 표시되고 이후 계속 바뀔 데이터이므로 `useState`로 관리한다.
- `GameScreen`은 사용자가 선택한 숫자를 `props`로 받아야 한다.
- 추측 숫자를 보기 좋게 출력하기 위해 `NumberContainer`라는 재사용 컴포넌트를 만든다.
- 컴포넌트가 늘어나면 `components/ui`, `components/game`처럼 역할별 폴더로 나누면 관리하기 좋다.

## 학습 목표

- 최솟값과 최댓값 사이의 난수를 생성하는 유틸리티 함수의 원리를 이해한다.
- 특정 숫자를 제외하고 난수를 다시 생성하는 재귀적 흐름을 이해한다.
- `GameScreen`에서 사용자의 선택값을 props로 받아 초기 추측값을 계산하는 방법을 배운다.
- 변하는 UI 데이터는 상태로 관리해야 한다는 React의 기본 원칙을 다시 확인한다.
- 숫자 출력 UI를 `NumberContainer`로 분리하는 이유를 이해한다.
- 컴포넌트 폴더를 역할별로 정리할 때 import 경로도 함께 관리해야 함을 익힌다.

## 주요 내용

### 게임 화면의 다음 목표

이제 `GameScreen`에는 기본 구조와 제목이 준비되어 있다.

다음으로 구현할 핵심 기능은 휴대폰의 첫 번째 추측을 화면에 보여주는 것이다.

```text
사용자
-> 1부터 99 사이 숫자 선택

앱
-> 선택된 숫자를 맞히기 위해 첫 추측 생성

화면
-> 앱이 추측한 숫자 표시
```

이 추측값은 이후 사용자의 힌트에 따라 계속 바뀐다.

따라서 단순한 지역 변수로 두기보다 React 상태로 관리해야 한다.

### 난수 생성 유틸리티 함수

강의에서는 최솟값과 최댓값 사이의 난수를 생성하는 헬퍼 함수를 사용한다.

```jsx
function generateRandomBetween(min, max, exclude) {
  const rndNum = Math.floor(Math.random() * (max - min)) + min;

  if (rndNum === exclude) {
    return generateRandomBetween(min, max, exclude);
  } else {
    return rndNum;
  }
}
```

이 함수는 세 개의 값을 받는다.

```text
min
-> 생성할 숫자의 최솟값

max
-> 생성할 숫자의 최댓값 경계

exclude
-> 생성 결과에서 제외할 숫자
```

이번 게임에서는 사용자가 고른 숫자를 `exclude`로 전달한다.

그래야 앱이 첫 추측에서 바로 정답을 맞히는 상황을 피할 수 있다.

### Math.random 계산식 이해하기

`Math.random()`은 0 이상 1 미만의 난수를 반환한다.

```jsx
Math.random()
```

이 값에 `max - min`을 곱하면 생성 범위를 넓힐 수 있다.

```jsx
Math.random() * (max - min)
```

여기에 `Math.floor()`를 적용해 소수점을 버린다.

```jsx
Math.floor(Math.random() * (max - min))
```

마지막으로 `min`을 더해 실제 시작점을 맞춘다.

```jsx
Math.floor(Math.random() * (max - min)) + min
```

정리하면 다음과 같다.

```text
Math.random()
-> 0 이상 1 미만 난수

* (max - min)
-> 원하는 범위만큼 확장

Math.floor(...)
-> 정수로 변환

+ min
-> 최솟값 기준으로 이동
```

숫자 맞히기 게임에서는 소수점이 필요 없기 때문에 정수로 변환하는 과정이 중요하다.

### exclude 값 제외하기

생성된 난수가 `exclude`와 같으면 다시 난수를 생성한다.

```jsx
if (rndNum === exclude) {
  return generateRandomBetween(min, max, exclude);
}
```

이 방식은 원하는 값이 나올 때까지 같은 함수를 다시 호출한다.

```text
난수 생성
-> exclude와 같은지 확인
-> 같으면 다시 생성
-> 다르면 반환
```

이번 게임에서는 `exclude`가 사용자가 선택한 숫자다.

따라서 첫 추측은 사용자의 숫자와 다르게 시작한다.

### GameScreen에서 userNumber 받기

사용자가 고른 숫자는 `StartGameScreen`에서 결정된다.

하지만 난수 생성은 `GameScreen`에서 필요하다.

다른 컴포넌트에서 결정된 값을 사용하려면 props로 전달한다.

```jsx
function GameScreen({ userNumber }) {
  // userNumber 사용 가능
}
```

`App.js`에서는 `GameScreen`을 렌더링할 때 `userNumber`를 넘긴다.

```jsx
if (userNumber) {
  screen = <GameScreen userNumber={userNumber} />;
}
```

이 흐름은 일반적인 React 데이터 전달 방식과 같다.

```text
App
-> userNumber 상태 보유
-> GameScreen에 prop으로 전달

GameScreen
-> userNumber를 받아 난수 생성에 사용
```

### 최초 추측값 계산하기

`GameScreen` 내부에서 최초 추측값을 계산한다.

```jsx
const initialGuess = generateRandomBetween(1, 100, userNumber);
```

여기서 `100`을 전달하는 이유는 `max`가 실질적인 상한 경계처럼 쓰이기 때문이다.

```text
min = 1
max = 100

실제 생성 범위
-> 1부터 99
```

이렇게 하면 게임 규칙인 1부터 99 사이의 숫자 범위와 맞는다.

### 추측값은 상태로 관리한다

최초 추측값은 화면에 출력된다.

그리고 이후 사용자가 “더 높다”, “더 낮다”라고 알려줄 때마다 새 추측값으로 바뀐다.

이처럼 값이 바뀌고 UI에 반영되어야 하는 데이터는 상태로 관리한다.

```jsx
import { useState } from 'react';
```

```jsx
const [currentGuess, setCurrentGuess] = useState(initialGuess);
```

`currentGuess`는 현재 앱이 추측한 숫자다.

```text
currentGuess
-> 화면에 표시할 현재 추측값

setCurrentGuess
-> 다음 추측값으로 업데이트할 함수
```

이번 단계에서는 첫 추측값만 보여주지만, 다음 단계에서 버튼 피드백에 따라 이 값을 바꾸게 된다.

### 숫자 출력 UI 분리하기

추측값을 그냥 `Text`로 출력할 수도 있다.

```jsx
<Text>{currentGuess}</Text>
```

하지만 숫자는 게임 화면에서 중요한 UI 요소다.

큰 숫자로 강조하고, 경계선과 여백을 주어 별도의 박스처럼 보여주고 싶다.

따라서 `NumberContainer` 컴포넌트로 분리한다.

```text
NumberContainer
-> 숫자를 감싸는 View
-> 숫자를 출력하는 Text
-> 숫자 전용 스타일 보유
```

컴포넌트를 분리하면 `GameScreen`의 JSX가 더 읽기 쉬워진다.

### components 폴더 정리

컴포넌트가 늘어나면 폴더를 역할별로 나눌 수 있다.

강의에서는 다음처럼 정리한다.

```text
components/
  ui/
    PrimaryButton.js
    Title.js
  game/
    NumberContainer.js
```

`PrimaryButton`과 `Title`은 여러 화면에서 재사용 가능한 일반 UI 컴포넌트다.

```text
components/ui
-> 앱 전반에서 쓰이는 공통 UI
```

`NumberContainer`는 게임 화면과 더 밀접한 컴포넌트다.

```text
components/game
-> 게임 기능에 특화된 UI
```

이런 분리는 필수는 아니지만, 앱이 커질수록 파일을 찾고 책임을 이해하기 쉬워진다.

### import 경로 변경 주의

파일을 폴더로 옮기면 import 경로도 바뀐다.

예를 들어 `Title.js`가 `components/ui`로 이동하면 `GameScreen`의 import도 바뀌어야 한다.

```jsx
import Title from '../components/ui/Title';
```

`Title.js` 내부에서 색상 상수를 import하는 경로도 바뀔 수 있다.

```jsx
import Colors from '../../constants/colors';
```

폴더를 한 단계 더 깊게 옮기면 `../` 개수도 달라진다.

```text
components/Title.js
-> ../constants/colors

components/ui/Title.js
-> ../../constants/colors
```

파일 이동 후 앱이 깨졌다면 import 경로를 먼저 확인하는 습관이 좋다.

### NumberContainer 컴포넌트 만들기

`components/game/NumberContainer.js`를 만든다.

```jsx
import { StyleSheet, Text, View } from 'react-native';

import Colors from '../../constants/colors';

function NumberContainer({ children }) {
  return (
    <View style={styles.container}>
      <Text style={styles.numberText}>{children}</Text>
    </View>
  );
}

export default NumberContainer;
```

여기서도 `children`을 사용한다.

```jsx
<NumberContainer>{currentGuess}</NumberContainer>
```

`NumberContainer`는 내부 숫자가 무엇인지 몰라도 된다.

그저 전달받은 내용을 정해진 모양으로 표시한다.

### NumberContainer 스타일

숫자 박스는 경계선, 안쪽 여백, 둥근 모서리, 중앙 정렬을 가진다.

```jsx
const styles = StyleSheet.create({
  container: {
    borderWidth: 4,
    borderColor: Colors.accent500,
    padding: 24,
    borderRadius: 8,
    margin: 24,
    alignItems: 'center',
    justifyContent: 'center',
  },
  numberText: {
    color: Colors.accent500,
    fontSize: 36,
    fontWeight: 'bold',
  },
});
```

`View`와 `Text`를 나누어 스타일링하는 점이 중요하다.

```text
View
-> 경계선
-> padding
-> borderRadius
-> margin
-> 내부 정렬

Text
-> 글자 색상
-> 글자 크기
-> 글자 굵기
```

특히 `borderRadius`는 `Text`에 직접 적용하면 플랫폼에 따라 일관되지 않을 수 있다.

따라서 박스형 UI는 `View`로 감싸고, 텍스트 스타일은 `Text`에 적용하는 편이 안전하다.

### GameScreen에서 NumberContainer 사용하기

`GameScreen`에서 `NumberContainer`를 import한다.

```jsx
import NumberContainer from '../components/game/NumberContainer';
```

그리고 제목 아래에 현재 추측값을 출력한다.

```jsx
<Title>Opponent's Guess</Title>

<NumberContainer>{currentGuess}</NumberContainer>
```

`currentGuess`가 상태이기 때문에 나중에 값이 바뀌면 `NumberContainer` 안의 숫자도 자동으로 갱신된다.

### GameScreen 전체 흐름 예시

이번 단계의 `GameScreen`은 대략 다음 흐름을 가진다.

```jsx
import { useState } from 'react';
import { StyleSheet, View } from 'react-native';

import NumberContainer from '../components/game/NumberContainer';
import Title from '../components/ui/Title';

function generateRandomBetween(min, max, exclude) {
  const rndNum = Math.floor(Math.random() * (max - min)) + min;

  if (rndNum === exclude) {
    return generateRandomBetween(min, max, exclude);
  }

  return rndNum;
}

function GameScreen({ userNumber }) {
  const initialGuess = generateRandomBetween(1, 100, userNumber);
  const [currentGuess, setCurrentGuess] = useState(initialGuess);

  return (
    <View style={styles.screen}>
      <Title>Opponent's Guess</Title>
      <NumberContainer>{currentGuess}</NumberContainer>

      <View>
        {/* Higher or lower controls */}
      </View>

      <View>
        {/* Guess log */}
      </View>
    </View>
  );
}

export default GameScreen;

const styles = StyleSheet.create({
  screen: {
    flex: 1,
    padding: 24,
  },
});
```

`setCurrentGuess`는 아직 사용하지 않는다.

다음 단계에서 사용자의 피드백 버튼을 누를 때 새로운 추측값을 계산하고 상태를 업데이트하는 데 쓰인다.

### Title 색상 조정

실제 화면을 확인해 보면 제목의 노란색과 숫자 박스의 노란색이 동시에 강조되어 조금 과해 보일 수 있다.

그래서 `Title`의 글자 색상을 흰색으로 바꿀 수 있다.

```jsx
title: {
  color: 'white',
  borderColor: Colors.accent500,
}
```

흰색은 React Native에서 지원하는 기본 색상 이름이므로 그대로 사용할 수 있다.

```text
Title
-> 흰색 텍스트
-> 노란색 경계선

NumberContainer
-> 노란색 숫자
-> 노란색 경계선
```

이렇게 하면 화면에서 추측 숫자가 더 잘 강조된다.

### 최초 추측값은 매번 달라진다

난수로 추측값을 생성하기 때문에 게임을 시작할 때마다 첫 추측값은 달라진다.

```text
첫 게임
-> 42

다음 게임
-> 71

다른 기기
-> 또 다른 숫자
```

iOS와 Android에서 동시에 테스트하면 서로 다른 숫자가 나올 수 있다.

각 앱 인스턴스가 독립적으로 난수를 생성하기 때문이다.

### 프론트엔드 개발자 관점에서의 핵심 차이

웹 React 개발자 입장에서 보면 이번 강의의 핵심은 익숙한 패턴이다.

```text
부모 상태를 자식에게 props로 전달
-> App에서 GameScreen으로 userNumber 전달

변하는 값을 useState로 관리
-> currentGuess 상태

표현 UI를 작은 컴포넌트로 분리
-> NumberContainer

공통 색상 토큰 사용
-> Colors.accent500
```

React Native만의 차이는 기반 컴포넌트다.

```text
웹
-> div, h1, span

React Native
-> View, Text
```

그리고 스타일링도 CSS 파일이 아니라 JavaScript 객체로 작성한다.

하지만 데이터 흐름, 상태 관리, 컴포넌트 분리 감각은 React 웹과 동일하다.

## 정리

이번 강의에서는 게임 화면에 최초 추측값을 표시하는 기능을 추가했다. `generateRandomBetween` 함수로 사용자가 선택한 숫자를 제외한 난수를 만들고, 그 값을 `currentGuess` 상태의 초깃값으로 사용했다.

또한 추측 숫자를 강조해서 보여주기 위해 `NumberContainer` 컴포넌트를 만들었다. 숫자 박스는 `View`로 감싸고 실제 숫자는 `Text`로 출력해, 경계선과 둥근 모서리, 텍스트 스타일을 각각 적절한 컴포넌트에 적용했다.

컴포넌트가 늘어나면서 `components/ui`, `components/game`처럼 폴더를 나누는 흐름도 함께 배웠다. 이때 파일 위치가 바뀌면 import 경로도 함께 수정해야 한다.
