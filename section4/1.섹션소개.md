# 1. 섹션 소개

## 핵심 요약

- 지난 섹션들에서 React Native의 기본 컴포넌트, 스타일링, 디버깅을 배웠다.
- 이번 섹션에서는 React Native의 필수 개념을 더 깊이 있게 다룬다.
- 새로운 데모 앱으로 숫자 맞히기 게임 앱을 만든다.
- 게임 앱을 만들면서 복잡한 레이아웃, 여러 화면, 배경 이미지, 그라데이션, 재사용 컴포넌트를 학습한다.
- 단순히 동작하는 앱이 아니라, 정교한 React 코드 구조를 갖춘 앱을 만드는 것이 목표다.

## 학습 목표

- React Native 핵심 개념을 이전보다 더 깊게 이해한다.
- 새로운 게임 앱을 통해 화면 전환과 복잡한 UI 구성을 연습한다.
- 배경 이미지와 그라데이션 같은 시각적 요소를 적용하는 방법을 배운다.
- 재사용 가능한 커스텀 컴포넌트를 설계하고 활용한다.
- React Native 앱도 결국 React 앱이라는 점을 바탕으로 좋은 컴포넌트 구조를 만든다.

## 주요 내용

### 이전 섹션에서 배운 것

지금까지는 React Native 앱을 만들기 위해 반드시 필요한 기본기를 살펴봤다.

대표적으로 다음 내용을 배웠다.

```text
핵심 컴포넌트
스타일링
레이아웃
상태 관리
이벤트 처리
목록 렌더링
디버깅
```

하지만 이 단계는 아직 React Native의 기본을 맛본 수준이다. 이번 섹션에서는 같은 기본 개념들을 더 깊이 다루면서 실제 앱 구조를 더 정교하게 만들어 간다.

### 새 데모 앱: 숫자 맞히기 게임

이번 섹션에서는 완전히 새로운 앱을 만든다.

앱은 1부터 99 사이의 숫자를 사용하는 간단한 게임이다.

게임 흐름은 다음과 같다.

```text
사용자가 1부터 99 사이의 숫자를 입력한다.
휴대폰이 사용자의 숫자를 추측한다.
사용자는 추측값이 너무 높은지 낮은지 힌트를 준다.
휴대폰이 정답을 맞힐 때까지 반복한다.
몇 번 만에 맞혔는지 확인한다.
```

앱 자체는 단순한 게임이지만, 이 안에서 다양한 React Native 개념을 자연스럽게 다룰 수 있다.

### 새롭게 다룰 개념들

숫자 맞히기 게임 앱을 만들면서 이전 섹션보다 더 다양한 UI와 앱 구조를 다룬다.

이번 섹션에서 다룰 주요 개념은 다음과 같다.

- 여러 화면 구성
- 화면 간 이동 흐름
- 복잡한 레이아웃
- 배경 이미지
- 그라데이션
- 새로운 핵심 컴포넌트
- 최신 스타일링 기법
- 재사용 가능한 커스텀 컴포넌트
- 더 정교한 React 컴포넌트 구조

이전 목표 관리 앱보다 시각적으로도 더 풍부하고, 코드 구조도 더 많은 고민이 필요한 앱이 된다.

### React Native 앱은 결국 React 앱

이번 섹션에서도 중요한 전제는 변하지 않는다.

React Native 앱은 결국 React 앱이다. 따라서 좋은 앱을 만들기 위해서는 React의 기본 원칙을 계속 적용해야 한다.

```text
컴포넌트를 작게 나누기
재사용 가능한 UI 만들기
상태를 적절한 위치에 두기
props로 데이터와 함수 전달하기
화면 흐름을 명확히 구성하기
```

단순히 화면에 보이는 UI만 대충 만드는 것이 아니라, 유지보수하기 좋은 React 코드로 앱을 구성하는 것이 목표다.

### 이번 섹션의 기대 결과

이 모듈이 끝나면 React Native가 어떻게 동작하는지 더 깊이 이해할 수 있다.

특히 다음 감각을 얻게 된다.

```text
React Native UI를 더 정교하게 구성하는 방법
앱 화면을 여러 단계로 나누는 방법
스타일링을 더 풍부하게 적용하는 방법
커스텀 컴포넌트로 앱 구조를 정리하는 방법
React 사고방식을 모바일 앱에 적용하는 방법
```

## 정리

이번 섹션에서는 숫자 맞히기 게임 앱을 만들며 React Native의 핵심 개념을 더 깊게 다룬다. 배경 이미지, 그라데이션, 여러 화면, 복잡한 레이아웃, 재사용 컴포넌트 등 이전보다 더 실제 앱에 가까운 요소를 학습한다.

목표는 단순히 작동하는 앱을 만드는 것이 아니라, React Native의 원리를 더 잘 이해하고 정교한 React 코드 구조로 앱을 만드는 것이다.
