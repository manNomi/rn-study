# 32. GuessLogItem으로 추측 로그 항목 스타일링

## 핵심 요약

- `FlatList`에서 직접 `Text`를 렌더링하던 로그 항목을 전용 컴포넌트인 `GuessLogItem`으로 분리한다.
- 각 로그 항목은 몇 번째 라운드인지와 해당 라운드에서 상대방이 추측한 숫자를 함께 보여준다.
- `FlatList`의 `renderItem`은 `itemData.item`뿐 아니라 `itemData.index`도 제공하므로, 이를 이용해 라운드 번호를 계산할 수 있다.
- 최신 추측값을 배열의 앞에 추가하는 구조에서는 인덱스가 곧 라운드 번호가 아니므로 전체 길이에서 인덱스를 빼는 계산이 필요하다.
- 로그 항목은 `View`, `Text`, `StyleSheet`, 공통 색상 상수를 조합해 카드처럼 스타일링한다.
- `flexDirection: 'row'`와 `justifyContent: 'space-between'`을 사용해 라운드 번호와 추측 숫자를 양쪽 끝에 배치한다.
- 항목 스타일링 이후 긴 로그 목록에서 하단 항목이 화면 경계에 걸리는 문제가 발견되며, 다음 단계에서 `FlatList` 컨테이너로 해결한다.

## 학습 목표

- 목록 항목 UI를 별도 컴포넌트로 분리하는 기준을 이해한다.
- `FlatList`의 `renderItem`에서 커스텀 컴포넌트를 반환하는 방법을 익힌다.
- `itemData.index`와 배열 길이를 이용해 화면에 표시할 라운드 번호를 계산한다.
- React Native에서 로그 항목을 카드형 UI로 스타일링하는 방법을 배운다.
- Android와 iOS 그림자 스타일의 차이를 다시 복습한다.

## 주요 내용

### 왜 GuessLogItem을 분리하는가

이전 단계에서는 `FlatList`의 `renderItem` 안에서 바로 `Text`를 반환했다.

```jsx
<FlatList
  data={guessRounds}
  renderItem={(itemData) => <Text>{itemData.item}</Text>}
/>
```

간단한 출력에는 충분하지만, 각 항목에 라운드 번호와 추측값을 함께 보여주고 스타일도 적용하려면 `renderItem`이 금방 복잡해진다.

그래서 `components/game/GuessLogItem.js` 파일을 만들고, 로그 항목 하나를 담당하는 컴포넌트로 분리한다.

```text
GameScreen
-> FlatList
-> GuessLogItem
-> 라운드 번호 + 추측 숫자 출력
```

프론트엔드 관점에서는 웹 React에서 `ListItem`, `TodoItem`, `PostCard` 같은 컴포넌트를 만드는 것과 같은 패턴이다.

### GuessLogItem 기본 구조

`GuessLogItem`은 로그 항목 하나를 렌더링한다.

```jsx
function GuessLogItem({ roundNumber, guess }) {
  return (
    <View>
      <Text>#{roundNumber}</Text>
      <Text>Opponent's Guess: {guess}</Text>
    </View>
  );
}

export default GuessLogItem;
```

여기서 `roundNumber`와 `guess`는 `GameScreen`에서 prop으로 전달한다.

```text
roundNumber
-> 몇 번째 라운드인지 표시

guess
-> 해당 라운드에서 장치가 추측한 숫자
```

React Native에서는 `View`와 `Text`를 반드시 `react-native`에서 import해야 한다.

```jsx
import { View, Text } from 'react-native';
```

웹의 `div`, `span`처럼 그냥 사용할 수 없다는 점을 다시 확인할 수 있다.

### 로그 항목 스타일링

`GuessLogItem` 안에서 `StyleSheet`를 사용해 항목 스타일을 정의한다.

```jsx
import { StyleSheet, Text, View } from 'react-native';

import Colors from '../../constants/colors';

function GuessLogItem({ roundNumber, guess }) {
  return (
    <View style={styles.listItem}>
      <Text style={styles.itemText}>#{roundNumber}</Text>
      <Text style={styles.itemText}>Opponent's Guess: {guess}</Text>
    </View>
  );
}

export default GuessLogItem;

const styles = StyleSheet.create({
  listItem: {
    borderColor: Colors.primary800,
    borderWidth: 1,
    borderRadius: 40,
    padding: 12,
    marginVertical: 8,
    backgroundColor: Colors.accent500,
    flexDirection: 'row',
    justifyContent: 'space-between',
    width: '100%',
    elevation: 4,
    shadowColor: 'black',
    shadowOffset: { width: 0, height: 0 },
    shadowOpacity: 0.25,
    shadowRadius: 3,
  },
  itemText: {
    fontFamily: 'open-sans',
  },
});
```

스타일의 역할은 다음과 같다.

```text
borderColor / borderWidth
-> 항목 외곽선

borderRadius
-> 둥근 알약 형태

padding
-> 내부 여백

marginVertical
-> 항목 간 위아래 간격

backgroundColor
-> 항목 배경색

flexDirection: 'row'
-> 텍스트 두 개를 가로 배치

justifyContent: 'space-between'
-> 왼쪽에는 라운드 번호, 오른쪽에는 추측값 배치

width: '100%'
-> 부모 컨테이너의 사용 가능한 너비 전체 사용
```

### Android와 iOS 그림자

React Native에서 그림자는 플랫폼별로 설정 방식이 다르다.

Android에서는 `elevation`을 사용한다.

```jsx
elevation: 4
```

iOS에서는 `shadow*` 계열 속성을 사용한다.

```jsx
shadowColor: 'black',
shadowOffset: { width: 0, height: 0 },
shadowOpacity: 0.25,
shadowRadius: 3,
```

이전 강의에서 배운 것처럼 React Native 스타일은 CSS처럼 보이지만 결국 각 플랫폼의 네이티브 스타일로 변환된다. 따라서 일부 스타일은 플랫폼별 설정을 따로 작성해야 한다.

### GameScreen에서 GuessLogItem 사용하기

이제 `GameScreen`의 `FlatList`에서 `Text` 대신 `GuessLogItem`을 반환한다.

```jsx
<FlatList
  data={guessRounds}
  keyExtractor={(item) => item.toString()}
  renderItem={(itemData) => (
    <GuessLogItem
      roundNumber={guessRoundsListLength - itemData.index}
      guess={itemData.item}
    />
  )}
/>
```

이를 위해 `GuessLogItem`을 import해야 한다.

```jsx
import GuessLogItem from '../components/game/GuessLogItem';
```

### 라운드 번호 계산하기

`guessRounds` 배열은 새 추측값을 앞에 추가하는 구조다.

```jsx
setGuessRounds((prevGuessRounds) => [
  newRndNumber,
  ...prevGuessRounds,
]);
```

즉 가장 최근 라운드가 항상 인덱스 `0`이 된다.

```text
guessRounds = [5, 12, 30]

index 0 -> 가장 최근 라운드
index 1 -> 이전 라운드
index 2 -> 더 이전 라운드
```

따라서 `itemData.index + 1`을 라운드 번호로 쓰면 반대로 표시된다.

```text
원하는 표시
3라운드: 5
2라운드: 12
1라운드: 30

단순 index + 1
1라운드: 5
2라운드: 12
3라운드: 30
```

그래서 목록 전체 길이를 구한 뒤 인덱스를 빼서 라운드 번호를 만든다.

```jsx
const guessRoundsListLength = guessRounds.length;
```

```jsx
roundNumber={guessRoundsListLength - itemData.index}
```

예를 들어 `guessRounds.length`가 `3`이면 다음처럼 계산된다.

```text
index 0 -> 3 - 0 = 3라운드
index 1 -> 3 - 1 = 2라운드
index 2 -> 3 - 2 = 1라운드
```

최신 항목이 위에 보이면서도 실제 라운드 번호는 올바르게 표시된다.

### FlatList의 itemData 구조 복습

`FlatList`의 `renderItem`은 각 항목을 렌더링할 때 객체를 넘겨준다.

```jsx
renderItem={(itemData) => {
  // itemData.item
  // itemData.index
}}
```

이번 강의에서는 두 값 모두 사용한다.

```text
itemData.item
-> 실제 추측 숫자

itemData.index
-> 현재 항목이 배열에서 몇 번째 위치인지
```

이처럼 `FlatList`가 단순한 값뿐 아니라 메타데이터도 제공한다는 점은 실무에서도 자주 유용하다.

### 스타일링 후 발견한 문제

로그 항목을 카드처럼 꾸미면 목록은 훨씬 보기 좋아진다.

다만 라운드가 많아져서 목록이 화면을 넘어가면 하단 항목이 화면 경계에 걸리는 문제가 보인다.

```text
문제 상황
-> 스크롤은 되지만
-> 마지막 라운드 항목이 화면 하단에 완전히 들어오지 않음
-> 읽기 어렵고 터치 영역도 답답함
```

이 문제는 `FlatList` 자체의 항목 스타일 문제가 아니라, `FlatList`가 차지할 수 있는 높이를 부모 컨테이너가 명확히 제한하지 않아서 발생한다.

다음 단계에서는 `FlatList`를 감싸는 컨테이너에 `flex: 1`과 여백을 추가해서 스크롤 영역을 안정적으로 만든다.

## 프론트엔드 개발자 관점에서의 핵심 차이

웹에서는 목록이 길어지면 문서 전체가 자연스럽게 스크롤되는 경우가 많다.

React Native에서는 화면과 스크롤 영역을 더 명시적으로 설계해야 한다.

```text
웹
-> document/body가 자연스럽게 스크롤될 수 있음

React Native
-> 스크롤 가능한 영역을 FlatList/ScrollView로 명시
-> 부모 컨테이너의 높이 제약도 중요
```

또한 웹 CSS에서는 텍스트 스타일이 상속되는 일이 많지만, React Native에서는 대부분의 스타일을 해당 컴포넌트에 직접 지정해야 한다.

```jsx
<Text style={styles.itemText}>#{roundNumber}</Text>
<Text style={styles.itemText}>Opponent's Guess: {guess}</Text>
```

로그 항목의 텍스트 두 개 모두에 `itemText` 스타일을 직접 부여한 이유다.

## 정리

이번 단계에서는 `FlatList`의 항목 렌더링을 전용 컴포넌트인 `GuessLogItem`으로 분리했다.

이 과정에서 라운드 번호와 추측값을 함께 보여주는 구조를 만들었고, `FlatList`가 제공하는 `itemData.item`과 `itemData.index`를 활용해 최신순 목록에서도 올바른 라운드 번호를 계산했다.

또한 `View`, `Text`, `StyleSheet`, 공통 색상 상수, 플랫폼별 그림자 스타일을 조합해 로그 항목을 카드형 UI로 꾸몄다.

마지막으로 목록이 길어졌을 때 하단 항목이 화면에 온전히 보이지 않는 문제를 발견했고, 다음 단계에서 `FlatList` 주변 컨테이너를 통해 스크롤 영역을 정리하게 된다.
