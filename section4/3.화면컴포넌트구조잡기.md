# 3. 화면 컴포넌트 구조 잡기

## 핵심 요약

- 여러 화면을 가진 앱을 만들기 위해 `screens` 폴더와 `components` 폴더를 나눈다.
- `screens` 폴더에는 화면 전체를 담당하는 컴포넌트를 둔다.
- `components` 폴더에는 화면을 구성하는 재사용 UI 컴포넌트를 둔다.
- 이번 앱은 `StartGameScreen`, `GameScreen`, `GameOverScreen` 세 화면으로 구성된다.
- 화면도 특별한 객체가 아니라 일반 React 컴포넌트이며, 내부에서는 React Native 핵심 컴포넌트를 조합해 UI를 만든다.

## 학습 목표

- 여러 화면으로 구성된 React Native 앱의 기본 폴더 구조를 이해한다.
- 화면 컴포넌트와 일반 재사용 컴포넌트를 구분하는 이유를 배운다.
- `StartGameScreen`, `GameScreen`, `GameOverScreen`의 역할을 정리한다.
- 시작 화면의 초기 JSX 구조를 잡는 방법을 익힌다.
- React Native 화면도 결국 일반 React 컴포넌트라는 점을 이해한다.

## 주요 내용

### screens 폴더와 components 폴더

이번 앱은 여러 개의 화면을 가진다. 그래서 프로젝트에 `screens` 폴더를 추가한다.

`screens` 폴더에는 화면 전체를 차지하는 컴포넌트를 저장한다.

```text
screens/
  StartGameScreen.js
  GameScreen.js
  GameOverScreen.js
```

그리고 화면 안에서 재사용할 UI 조각들은 `components` 폴더에 둔다.

```text
components/
  ...
```

폴더 이름은 반드시 `screens`나 `components`여야 하는 것은 아니다. 하지만 화면 컴포넌트와 일반 UI 컴포넌트를 분리해 두면 프로젝트 구조를 이해하기 쉬워진다.

### 이전 앱과의 차이

지난 목표 관리 앱은 사실상 하나의 메인 화면으로 구성되어 있었다.

모달을 사용해 입력 화면처럼 보이는 UI를 만들기는 했지만, 실제로는 `App` 컴포넌트 안에서 하나의 화면 흐름을 구성했다.

이번 숫자 맞히기 게임 앱은 여러 개의 메인 화면이 필요하다.

```text
시작 화면
게임 진행 화면
게임 오버 화면
```

따라서 각 화면을 독립적인 컴포넌트로 나누고, 앱 상태에 따라 어떤 화면을 보여줄지 결정해야 한다.

화면 간 전환은 나중에 내비게이션 섹션에서 더 깊게 배우지만, 이번 섹션에서는 기본적인 조건부 렌더링과 상태를 통해 화면 전환 흐름을 익힌다.

### 세 개의 화면 컴포넌트

이번 앱에서 만들 화면은 세 개다.

| 화면 컴포넌트 | 역할 |
| --- | --- |
| `StartGameScreen` | 사용자가 맞힐 숫자를 입력하고 게임을 시작하는 화면 |
| `GameScreen` | 휴대폰의 추측값을 보고 더 높음 / 더 낮음 힌트를 주는 화면 |
| `GameOverScreen` | 게임 종료 후 결과와 재시작 버튼을 보여주는 화면 |

처음에는 세 파일이 비어 있지만, 강의를 진행하면서 하나씩 내용을 채워 나간다.

### 화면도 일반 React 컴포넌트

`StartGameScreen`은 함수형 컴포넌트로 만든다.

```jsx
function StartGameScreen() {
  return (
    // JSX
  );
}

export default StartGameScreen;
```

여기서 중요한 점은 화면 컴포넌트라고 해서 특별한 문법이 있는 것이 아니라는 점이다.

화면도 일반 React 컴포넌트다. 다만 앱에서 화면 전체를 담당하는 역할을 할 뿐이다.

이 컴포넌트 안에서도 `View`, `TextInput` 같은 React Native 핵심 컴포넌트를 조합해 UI를 만든다.

### StartGameScreen에서 필요한 UI

시작 화면에서는 사용자가 숫자를 입력하고 게임을 시작해야 한다.

필요한 기본 요소는 다음과 같다.

```text
안내 타이틀
숫자 입력 필드
입력값 초기화 버튼
입력값 확정 버튼
```

이번 강의에서는 먼저 숫자 입력 필드와 버튼 영역의 기본 구조를 잡는다.

### TextInput과 View로 기본 구조 만들기

사용자 입력을 받기 위해 `TextInput`을 사용한다.

버튼 영역과 입력 필드를 감싸기 위해 `View`도 필요하다.

```jsx
import { TextInput, View } from 'react-native';

function StartGameScreen() {
  return (
    <View>
      <TextInput />
      {/* 버튼 영역 */}
    </View>
  );
}

export default StartGameScreen;
```

강의에서는 아직 버튼을 바로 만들지 않고, 버튼이 들어갈 자리를 임시로 표시한다.

다만 `View` 안에 일반 문자열을 직접 넣으면 React Native에서는 오류가 발생한다.

```jsx
// 잘못된 방식
<View>
  BUTTONS
</View>
```

텍스트를 표시하려면 반드시 `Text` 컴포넌트로 감싸야 한다.

```jsx
// 텍스트를 실제로 표시해야 한다면
<View>
  <Text>BUTTONS</Text>
</View>
```

하지만 이번 단계에서는 최종 UI가 아니라 구조를 잡는 중이므로, 버튼 영역은 이후 커스텀 버튼 컴포넌트를 만들며 채워 나간다.

### 기본 Button 대신 커스텀 버튼을 만들 예정

React Native의 기본 `Button`을 사용할 수도 있지만, 이번 섹션에서는 직접 버튼을 만든다.

이유는 기본 `Button`은 스타일링 자유도가 낮기 때문이다.

직접 버튼을 만들면 다음을 원하는 대로 제어할 수 있다.

```text
배경색
텍스트 색상
눌림 효과
여백
모서리
플랫폼별 느낌
재사용 가능한 버튼 API
```

React Native 앱에서는 기본 컴포넌트를 조합해 원하는 UI 컴포넌트를 직접 만드는 일이 자주 있다.

## 정리

이번 강의에서는 숫자 맞히기 게임 앱의 화면 구조를 잡기 시작했다. `screens` 폴더에는 `StartGameScreen`, `GameScreen`, `GameOverScreen`을 두고, `components` 폴더에는 화면을 구성할 재사용 컴포넌트를 둘 계획이다.

화면 컴포넌트도 특별한 것이 아니라 일반 React 컴포넌트이며, React Native 핵심 컴포넌트를 조합해 만든다. 첫 화면인 `StartGameScreen`에서는 숫자 입력을 위한 `TextInput`과 이후 커스텀 버튼이 들어갈 기본 구조를 준비한다.
