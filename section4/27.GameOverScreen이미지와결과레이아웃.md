# 27. GameOverScreen 이미지와 결과 레이아웃

## 핵심 요약

- 게임 시작 화면과 게임 화면을 어느 정도 구성했으므로 이제 게임 오버 화면을 만든다.
- `GameOverScreen`에는 제목, 성공 이미지, 결과 요약 텍스트가 들어간다.
- 이미지 파일은 `assets/images/success.png`에 두고 `require()`로 불러온다.
- 이미지를 원형으로 만들려면 `Image` 자체보다 `View` 컨테이너에 `borderRadius`와 `overflow: 'hidden'`을 적용하는 방식이 안정적이다.
- 이미지 크기는 `width`, `height`를 같은 값으로 만들고, `borderRadius`를 그 절반으로 설정하면 원형이 된다.
- 전체 화면은 루트 `View`에 `flex: 1`, `justifyContent: 'center'`, `alignItems: 'center'`를 적용해 중앙 정렬한다.
- 결과 텍스트에는 라운드 수와 정답 숫자를 넣을 예정이며, 이후 실제 값으로 대체한다.

## 학습 목표

- `GameOverScreen`을 실제 화면 컴포넌트로 구성하는 방법을 익힌다.
- React Native에서 로컬 이미지를 `Image` 컴포넌트로 표시하는 방법을 복습한다.
- 원형 이미지 UI를 만들기 위한 `View` 래퍼와 `overflow: 'hidden'` 패턴을 이해한다.
- 퍼센트 기반 `width: '100%'`, `height: '100%'`가 부모 컨테이너 기준으로 동작한다는 점을 이해한다.
- 화면 전체 콘텐츠를 중앙에 배치하는 Flexbox 스타일을 복습한다.
- 게임 결과 요약 텍스트를 추가하고 이후 동적 데이터로 대체할 준비를 한다.

## 주요 내용

### GameOverScreen에 들어갈 요소

게임이 끝났을 때 보여줄 화면에는 다음 요소가 필요하다.

```text
1. 게임 종료 제목
2. 성공 이미지
3. 게임 결과 요약 텍스트
```

예상 화면 구조는 다음과 같다.

```text
GAME OVER!

[성공 이미지]

당신의 휴대폰은 X번의 라운드 만에
숫자 Y를 맞혔습니다.
```

`X`와 `Y`는 아직 플레이스홀더다.

이후 실제 라운드 수와 정답 숫자로 교체한다.

### 기본 Text를 View 컨테이너로 교체하기

기존 `GameOverScreen`은 임시 텍스트만 렌더링했다.

```jsx
function GameOverScreen() {
  return <Text>Game is over!</Text>;
}
```

이제 여러 요소를 배치해야 하므로 `View` 컨테이너를 사용한다.

```jsx
import { View } from 'react-native';
```

`View`는 화면이나 복잡한 컴포넌트를 구성할 때 여러 자식 요소를 담는 기본 컨테이너다.

```jsx
function GameOverScreen() {
  return (
    <View>
      {/* title */}
      {/* image */}
      {/* summary text */}
    </View>
  );
}
```

### Title 컴포넌트 재사용하기

게임 오버 화면에도 제목이 필요하다.

이미 만든 `Title` 컴포넌트를 재사용한다.

```jsx
import Title from '../components/ui/Title';
```

```jsx
<Title>GAME OVER!</Title>
```

새로운 제목 컴포넌트를 만들 수도 있지만, 기존 제목 스타일이 앱 전체에 어울리므로 재사용한다.

```text
StartGameScreen
-> Guess My Number

GameScreen
-> Opponent's Guess

GameOverScreen
-> GAME OVER!
```

하나의 `Title` 컴포넌트를 여러 화면에서 사용하는 흐름이 이어진다.

### Image 컴포넌트로 성공 이미지 표시하기

게임 오버 화면에는 성공 이미지를 표시한다.

React Native에서 이미지를 보여주려면 `Image` 컴포넌트를 사용한다.

```jsx
import { Image } from 'react-native';
```

강의에서는 `success.png` 이미지를 사용한다.

파일은 다음 위치에 둔다.

```text
assets/
  images/
    success.png
```

이미지는 `require()`로 불러온다.

```jsx
<Image source={require('../assets/images/success.png')} />
```

경로는 `GameOverScreen.js` 파일 기준 상대 경로다.

```text
screens/GameOverScreen.js
-> ..
-> assets/images/success.png
```

### 이미지 크기 문제

이미지를 처음 추가하면 원본 크기 때문에 너무 크게 보일 수 있다.

```text
문제
-> 이미지가 화면을 과하게 차지함
-> 레이아웃이 깨져 보임
```

따라서 이미지 크기와 모양을 직접 조정해야 한다.

이번 강의에서는 이미지를 원형으로 만들고, 화면 안에서 적절한 크기로 줄인다.

### 원형 이미지 만들기

원형 이미지를 만들려면 다음 조건이 필요하다.

```text
1. 컨테이너의 width와 height가 같아야 함
2. borderRadius가 width 또는 height의 절반이어야 함
3. 컨테이너 밖으로 삐져나온 이미지를 숨겨야 함
```

예를 들어 너비와 높이가 300이면 반지름은 150이다.

```jsx
imageContainer: {
  width: 300,
  height: 300,
  borderRadius: 150,
  overflow: 'hidden',
}
```

처음에는 400으로 설정할 수도 있지만, 화면에서 너무 크게 보이면 300으로 줄인다.

```text
width: 400 / height: 400
-> 너무 큼

width: 300 / height: 300
-> 더 적절함
```

다음 섹션에서는 기기 크기에 따라 동적으로 이미지 크기를 조정하는 방법을 다룰 예정이다.

### 왜 Image를 View로 감싸는가

이미지 자체에 스타일을 줄 수도 있지만, 원형 마스크를 안정적으로 만들려면 `View`로 감싸는 방식이 좋다.

```jsx
<View style={styles.imageContainer}>
  <Image
    source={require('../assets/images/success.png')}
    style={styles.image}
  />
</View>
```

컨테이너 역할을 하는 `View`에 원형 모양과 마스크 처리를 맡긴다.

```text
View
-> width
-> height
-> borderRadius
-> overflow: hidden
-> border

Image
-> width: 100%
-> height: 100%
```

이 방식은 커스텀 버튼에서 ripple 효과를 잘라내기 위해 `overflow: 'hidden'`을 사용했던 패턴과 비슷하다.

### overflow: 'hidden'의 역할

이미지는 원래 사각형이다.

컨테이너에 `borderRadius`를 적용해도 이미지가 컨테이너 밖으로 튀어나오면 원형처럼 보이지 않을 수 있다.

이를 막기 위해 컨테이너에 `overflow: 'hidden'`을 적용한다.

```jsx
overflow: 'hidden'
```

그러면 컨테이너의 원형 경계 밖에 있는 이미지 부분이 잘려 보이지 않는다.

```text
borderRadius
-> 컨테이너를 둥글게 만듦

overflow: hidden
-> 컨테이너 밖 이미지 영역을 숨김
```

결과적으로 이미지가 원형으로 잘려 보인다.

### 이미지 컨테이너 스타일

이미지 컨테이너는 다음처럼 구성할 수 있다.

```jsx
imageContainer: {
  width: 300,
  height: 300,
  borderRadius: 150,
  borderWidth: 3,
  borderColor: Colors.primary800,
  overflow: 'hidden',
  margin: 36,
}
```

각 속성의 역할은 다음과 같다.

```text
width, height
-> 정사각형 컨테이너 생성

borderRadius
-> 원형으로 만들기 위한 반지름

borderWidth, borderColor
-> 이미지 주변 테두리

overflow
-> 원형 밖의 이미지 숨김

margin
-> 제목, 결과 텍스트와의 간격
```

테두리 색상은 `Colors.primary800`처럼 이미 분리해둔 색상 상수를 사용한다.

### 이미지 자체 스타일

이미지 자체는 컨테이너를 가득 채우면 된다.

```jsx
image: {
  width: '100%',
  height: '100%',
}
```

여기서 퍼센트 값은 부모 컨테이너 기준이다.

```text
imageContainer
-> width: 300
-> height: 300

Image
-> width: 100%
-> height: 100%

결과
-> 이미지도 300 x 300
```

React Native에서 퍼센트 크기는 해당 요소를 감싸는 부모 컨테이너를 기준으로 계산된다.

### StyleSheet와 Colors import

이미지 스타일을 만들려면 `StyleSheet`를 import한다.

```jsx
import { Image, StyleSheet, View } from 'react-native';
```

테두리 색상에는 색상 상수를 사용하므로 `Colors`도 가져온다.

```jsx
import Colors from '../constants/colors';
```

`GameOverScreen`은 `screens` 폴더에 있으므로 `../constants/colors` 경로를 사용한다.

### 화면 전체 중앙 정렬

게임 오버 화면의 제목, 이미지, 결과 텍스트는 화면 중앙에 배치하면 보기 좋다.

루트 컨테이너 스타일을 만든다.

```jsx
rootContainer: {
  flex: 1,
  padding: 24,
  justifyContent: 'center',
  alignItems: 'center',
}
```

각 속성의 역할은 다음과 같다.

```text
flex: 1
-> 사용 가능한 화면 전체를 차지

padding: 24
-> 화면 가장자리와 콘텐츠 사이 여백

justifyContent: 'center'
-> 세로축 중앙 정렬

alignItems: 'center'
-> 가로축 중앙 정렬
```

React Native의 기본 `flexDirection`은 `column`이므로, `justifyContent`는 세로축 배치를 담당한다.

`alignItems`는 가로축 배치를 담당한다.

```text
기본 flexDirection: column

justifyContent
-> 위아래 방향 정렬

alignItems
-> 좌우 방향 정렬
```

### GameOverScreen 전체 구조 예시

이번 단계의 화면 구조는 다음처럼 정리할 수 있다.

```jsx
import { Image, StyleSheet, Text, View } from 'react-native';

import Title from '../components/ui/Title';
import Colors from '../constants/colors';

function GameOverScreen() {
  return (
    <View style={styles.rootContainer}>
      <Title>GAME OVER!</Title>

      <View style={styles.imageContainer}>
        <Image
          source={require('../assets/images/success.png')}
          style={styles.image}
        />
      </View>

      <Text>
        Your phone needed X rounds to guess the number Y.
      </Text>
    </View>
  );
}

export default GameOverScreen;
```

아직 `X`와 `Y`는 실제 데이터가 아니다.

이후 라운드 수와 사용자가 고른 숫자를 prop으로 받아 채우게 된다.

### 결과 텍스트 추가하기

이미지 아래에는 게임 결과를 간략히 보여준다.

```jsx
<Text>
  Your phone needed X rounds to guess the number Y.
</Text>
```

여기서 `X`는 라운드 수, `Y`는 정답 숫자다.

```text
X
-> 장치가 몇 번 만에 맞혔는지

Y
-> 사용자가 고른 정답 숫자
```

현재는 플레이스홀더로 두고, 다음 단계에서 실제 값으로 대체한다.

### 텍스트 일부 강조 예고

결과 문장 안에서 `X`와 `Y`는 강조하고 싶다.

React Native에서는 텍스트 안에 또 다른 `Text`를 중첩할 수 있다.

```jsx
<Text>
  Your phone needed <Text>X</Text> rounds to guess the number <Text>Y</Text>.
</Text>
```

이 방식으로 문장 일부에만 다른 스타일을 적용할 수 있다.

이번 강의는 결과 텍스트를 추가하는 데서 끝나고, 강조 스타일은 이어지는 흐름에서 더 다룰 수 있다.

### 프론트엔드 개발자 관점에서의 핵심 차이

웹에서는 원형 이미지를 보통 다음처럼 처리한다.

```css
.avatar {
  width: 300px;
  height: 300px;
  border-radius: 50%;
  overflow: hidden;
}
```

React Native에서도 개념은 거의 같다.

```jsx
imageContainer: {
  width: 300,
  height: 300,
  borderRadius: 150,
  overflow: 'hidden',
}
```

차이는 CSS의 `50%` 대신 숫자 기반 `borderRadius`를 사용한다는 점이다.

```text
웹
-> border-radius: 50%

React Native
-> borderRadius: width / 2
```

이미지를 부모에 꽉 채우는 방식도 비슷하다.

```css
img {
  width: 100%;
  height: 100%;
}
```

```jsx
image: {
  width: '100%',
  height: '100%',
}
```

React Native에서 퍼센트 값은 부모 컨테이너 기준이라는 점을 기억하면 된다.

### 실무 관점에서의 포인트

이미지 UI는 모바일 앱에서 자주 등장한다.

```text
프로필 이미지
성공/실패 화면 이미지
카테고리 썸네일
결과 화면 일러스트
```

원형 이미지를 만들 때는 다음 패턴을 기억하면 좋다.

```text
1. 이미지 컨테이너 View 생성
2. width와 height를 같은 값으로 설정
3. borderRadius를 절반으로 설정
4. overflow: hidden 적용
5. Image는 width/height 100%로 채움
```

이 패턴은 플랫폼별 차이를 줄이고 스타일을 예측하기 쉽게 만든다.

## 정리

이번 강의에서는 `GameOverScreen`을 실제 화면답게 구성했다. 기존 임시 텍스트를 `View` 기반 레이아웃으로 바꾸고, `Title` 컴포넌트를 재사용해 `GAME OVER!` 제목을 표시했다.

또한 `assets/images/success.png` 이미지를 `Image` 컴포넌트로 렌더링하고, `View` 컨테이너에 `borderRadius`와 `overflow: 'hidden'`을 적용해 원형 이미지처럼 보이게 만들었다. 이미지 자체는 `width: '100%'`, `height: '100%'`로 부모 컨테이너를 채우도록 했다.

마지막으로 화면 전체를 중앙 정렬하고, 이미지 아래에 게임 결과 텍스트를 추가했다. 현재는 `X`, `Y` 플레이스홀더를 사용하지만, 이후 실제 라운드 수와 정답 숫자로 대체하게 된다.
