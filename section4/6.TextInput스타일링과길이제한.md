# 6. TextInput 스타일링과 길이 제한

## 핵심 요약

- 입력 영역 카드 안의 `TextInput`을 숫자 입력에 어울리도록 스타일링한다.
- `height`, `fontSize`, `borderBottomColor`, `borderBottomWidth`, `color`, `fontWeight` 등으로 입력 필드 모양을 조정한다.
- `width`와 `textAlign`을 사용해 두 자리 숫자 입력에 맞는 짧고 가운데 정렬된 입력란을 만든다.
- `maxLength` prop으로 입력 가능한 최대 문자 수를 제한할 수 있다.
- 숫자 prop은 문자열이 아니라 중괄호를 사용해 숫자 값으로 전달해야 한다.

## 학습 목표

- `TextInput`에 스타일 객체를 연결하는 방법을 복습한다.
- 숫자 입력 필드에 맞는 시각적 스타일을 구성한다.
- 입력 필드의 크기와 텍스트 정렬을 조정하는 방법을 익힌다.
- `maxLength` prop을 사용해 최대 입력 길이를 제한한다.
- React Native prop에 문자열과 숫자를 전달하는 방식의 차이를 이해한다.

## 주요 내용

### TextInput 스타일 객체 만들기

이전 강의에서는 입력 필드와 버튼을 감싸는 카드형 컨테이너를 만들었다. 이번에는 카드 안에 있는 `TextInput` 자체를 다듬는다.

`StartGameScreen`의 `StyleSheet`에 입력 필드 전용 스타일을 추가한다.

```jsx
const styles = StyleSheet.create({
  inputContainer: {
    // 입력 영역 컨테이너 스타일
  },
  numberInput: {
    height: 50,
    fontSize: 32,
    borderBottomColor: '#ddb52f',
    borderBottomWidth: 2,
    color: '#ddb52f',
    marginVertical: 8,
    fontWeight: 'bold',
  },
});
```

이 스타일은 `TextInput`의 `style` prop에 연결한다.

```jsx
<TextInput style={styles.numberInput} />
```

### 기본 입력 필드 스타일

숫자 입력 필드에 적용한 스타일의 의미는 다음과 같다.

| 스타일 | 역할 |
| --- | --- |
| `height: 50` | 입력 필드 높이 지정 |
| `fontSize: 32` | 입력되는 숫자를 크게 표시 |
| `borderBottomColor: '#ddb52f'` | 아래쪽 경계선 색상 |
| `borderBottomWidth: 2` | 아래쪽 경계선 두께 |
| `color: '#ddb52f'` | 입력 텍스트 색상 |
| `marginVertical: 8` | 위아래 여백 |
| `fontWeight: 'bold'` | 입력 텍스트 굵게 표시 |

`#ddb52f`는 강의에서 준비한 노란색 계열 색상이다. 앱 분위기에 맞게 다른 색상으로 바꿔도 된다.

### 입력 필드 너비 줄이기

사용자가 입력할 숫자는 1부터 99까지다. 즉 한 자리 또는 두 자리 숫자만 필요하다.

따라서 입력 필드가 화면 너비 전체를 차지할 필요가 없다.

```jsx
numberInput: {
  height: 50,
  width: 50,
  fontSize: 32,
  borderBottomColor: '#ddb52f',
  borderBottomWidth: 2,
  color: '#ddb52f',
  marginVertical: 8,
  fontWeight: 'bold',
},
```

`width: 50`을 지정하면 입력란이 두 자리 숫자에 어울리는 짧은 크기가 된다.

### 텍스트 가운데 정렬

입력 필드 안의 숫자가 가운데에 오도록 `textAlign`을 추가한다.

```jsx
numberInput: {
  height: 50,
  width: 50,
  fontSize: 32,
  borderBottomColor: '#ddb52f',
  borderBottomWidth: 2,
  color: '#ddb52f',
  marginVertical: 8,
  fontWeight: 'bold',
  textAlign: 'center',
},
```

`textAlign: 'center'`는 입력된 텍스트를 `TextInput` 내부의 가운데로 정렬한다.

다만 이 설정은 입력 필드 안의 텍스트를 가운데 정렬할 뿐이다. 입력 필드 자체가 카드 안에서 가운데에 위치하는 문제는 별도의 레이아웃 스타일로 해결해야 한다.

### maxLength로 입력 길이 제한하기

입력 필드가 짧아졌더라도, 사용자는 여전히 더 긴 숫자를 입력할 수 있다. 이 경우 입력 필드 안에서 스크롤되듯 보일 수 있다.

이를 막기 위해 `TextInput`에 `maxLength` prop을 추가한다.

```jsx
<TextInput
  style={styles.numberInput}
  maxLength={2}
/>
```

`maxLength={2}`는 입력 가능한 최대 문자 수를 2개로 제한한다.

이 앱에서는 1부터 99 사이의 숫자만 사용할 예정이므로 최대 두 자리 입력만 허용하는 것이 자연스럽다.

### 숫자 prop은 중괄호로 전달하기

주의할 점은 `maxLength`에 숫자를 전달해야 한다는 것이다.

아래처럼 따옴표를 사용하면 문자열 `"2"`가 전달된다.

```jsx
// 잘못된 방식
<TextInput maxLength="2" />
```

React Native는 `maxLength`에 숫자 타입을 기대하므로, 문자열을 전달하면 타입 에러가 발생할 수 있다.

숫자를 전달하려면 중괄호를 사용한다.

```jsx
// 올바른 방식
<TextInput maxLength={2} />
```

이것은 React와 JSX의 기본 규칙이다.

```text
문자열 prop
-> "..."

JavaScript 값 / 숫자 / 변수
-> {...}
```

### 에러 발생 시 확인할 점

`maxLength="2"`처럼 잘못 전달하면 터미널과 기기에 에러가 표시될 수 있다. Android 앱은 경우에 따라 종료될 수도 있다.

이럴 때는 에러 메시지에서 다음을 확인한다.

```text
어떤 prop에서 문제가 났는가
어떤 타입을 기대했는가
실제로 어떤 타입을 전달했는가
```

이번 경우에는 `maxLength`에 문자열이 아니라 숫자를 전달해야 한다는 점을 확인하면 된다.

앱이 종료되었다면 Android 앱을 다시 실행하거나, 필요하면 `npm start` 프로세스를 재시작한다.

### 아직 남은 문제

이번 단계에서 입력 길이는 최대 두 글자로 제한했다.

하지만 아직 다음 문제는 남아 있다.

```text
일반 문자도 입력할 수 있다.
숫자 키보드가 아니라 일반 키보드가 열린다.
1부터 99 사이인지 검증하지 않는다.
입력값을 state로 관리하지 않는다.
```

다음 단계에서는 키보드를 숫자 입력에 맞게 바꾸고, 이후 입력값 검증도 추가한다.

## 정리

이번 강의에서는 `TextInput`을 숫자 입력 필드처럼 보이도록 스타일링했다. 높이, 글자 크기, 노란색 아래 경계선, 텍스트 색상, 굵기, 너비, 가운데 정렬을 적용해 두 자리 숫자 입력에 적합한 입력란을 만들었다.

또한 `maxLength={2}`를 사용해 최대 입력 길이를 제한했다. 이때 숫자 prop은 문자열이 아니라 중괄호를 사용해 숫자 값으로 전달해야 하며, 잘못 전달하면 타입 에러가 발생할 수 있다.
