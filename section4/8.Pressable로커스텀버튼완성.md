# 8. Pressable로 커스텀 버튼 완성

## 핵심 요약

- `View`와 `Text`만 조합한 컴포넌트는 아직 버튼이 아니므로, 터치 가능한 동작을 위해 `Pressable`을 사용한다.
- `Pressable`의 `onPress` prop에 함수를 전달하면 사용자가 눌렀을 때 해당 함수가 실행된다.
- Android에서는 `android_ripple` prop으로 네이티브 물결 효과를 줄 수 있다.
- 둥근 버튼 안쪽으로 ripple 효과를 제한하려면 외부 `View`와 내부 `Pressable` 컨테이너를 분리하고, 외부 컨테이너에 `overflow: 'hidden'`을 적용한다.
- iOS에서는 Android ripple이 동작하지 않으므로 `Pressable`의 style 함수와 `pressed` 상태를 활용해 눌림 피드백을 만든다.

## 학습 목표

- React Native에서 커스텀 버튼을 실제로 누를 수 있게 만드는 방법을 익힌다.
- `Pressable`과 `onPress`의 기본 사용법을 이해한다.
- Android와 iOS의 터치 피드백 차이를 배운다.
- `android_ripple`, `pressed`, style 배열을 활용한 버튼 피드백 구현 방식을 익힌다.
- React Native 스타일은 CSS처럼 상속되지 않는다는 점을 버튼 텍스트 스타일링에서 다시 확인한다.

## 주요 내용

### View와 Text만으로는 버튼이 아니다

이전 강의에서 만든 `PrimaryButton`은 `View` 안에 `Text`를 넣은 형태였다.

```jsx
function PrimaryButton({ children }) {
  return (
    <View>
      <Text>{children}</Text>
    </View>
  );
}
```

이 구조는 버튼처럼 보일 수는 있지만 아직 터치할 수 있는 컴포넌트는 아니다.

React Native에서 어떤 요소를 누를 수 있게 만들려면 `Pressable` 같은 터치 처리 컴포넌트가 필요하다.

```text
View + Text
-> 화면에 보이는 UI

Pressable
-> 누름 이벤트를 감지하는 UI
```

예전에는 `TouchableOpacity`, `TouchableHighlight`, `TouchableNativeFeedback` 같은 컴포넌트를 많이 사용했지만, 현재는 더 유연한 `Pressable`을 사용하는 흐름이 권장된다.

### Pressable로 누름 이벤트 처리하기

`Pressable`을 사용하려면 `react-native`에서 import한다.

```jsx
import { Pressable, Text, View } from 'react-native';
```

그리고 터치 가능하게 만들 영역을 `Pressable`로 감싼다.

```jsx
function PrimaryButton({ children }) {
  function pressHandler() {
    console.log('Pressed!');
  }

  return (
    <Pressable onPress={pressHandler}>
      <View>
        <Text>{children}</Text>
      </View>
    </Pressable>
  );
}
```

중요한 점은 `onPress={pressHandler}`처럼 함수 포인터를 전달해야 한다는 것이다.

```jsx
onPress={pressHandler}
```

다음처럼 괄호를 붙이면 렌더링 시점에 함수가 바로 실행되므로 원하는 동작이 아니다.

```jsx
onPress={pressHandler()}
```

이제 버튼을 누르면 터미널에 `Pressed!`가 출력된다. 아직 시각적 피드백은 없지만, 버튼이 눌림 이벤트를 정상적으로 받고 있다는 뜻이다.

### 버튼 컨테이너 스타일링

버튼을 시각적으로 버튼답게 만들기 위해 `StyleSheet`를 사용한다.

```jsx
import { Pressable, StyleSheet, Text, View } from 'react-native';
```

버튼의 박스 영역에는 배경색, 둥근 모서리, 안쪽 여백, 바깥 여백을 줄 수 있다.

```jsx
const styles = StyleSheet.create({
  buttonContainer: {
    backgroundColor: '#27063c',
    borderRadius: 28,
    paddingVertical: 8,
    paddingHorizontal: 16,
    margin: 4,
    elevation: 2,
  },
});
```

`borderRadius`를 크게 주면 양쪽 끝이 둥근 캡슐 형태의 버튼을 만들 수 있다.

`elevation`은 Android에서 그림자를 만드는 데 사용한다. iOS 그림자는 이 강의에서는 따로 추가하지 않는다.

버튼 배경색과 입력 카드 배경색이 같으면 버튼이 구분되지 않는다. 그래서 `StartGameScreen`의 입력 영역 배경색은 더 어두운 색으로 조정한다.

```jsx
inputContainer: {
  backgroundColor: '#4e0329',
}
```

이처럼 버튼 자체의 스타일뿐 아니라 버튼이 놓이는 부모 영역의 배경도 함께 조정해야 시각적 위계가 생긴다.

### 텍스트 스타일은 Text에 직접 적용하기

버튼 텍스트를 흰색으로 만들고 가운데 정렬하려면 `Text` 컴포넌트에 직접 스타일을 적용해야 한다.

```jsx
const styles = StyleSheet.create({
  buttonText: {
    color: 'white',
    textAlign: 'center',
  },
});
```

```jsx
<Text style={styles.buttonText}>{children}</Text>
```

React Native 스타일은 CSS처럼 부모 스타일을 자식이 자동으로 상속하지 않는다.

따라서 다음처럼 컨테이너에 `color: 'white'`를 적용해도 내부 `Text`의 글자색은 바뀌지 않는다.

```jsx
buttonContainer: {
  color: 'white',
}
```

텍스트 관련 스타일은 실제 텍스트를 렌더링하는 `Text`에 적용해야 한다.

### Android ripple 효과 추가하기

Android에서는 `Pressable`에 `android_ripple` prop을 전달해 네이티브 물결 효과를 만들 수 있다.

```jsx
<Pressable
  android_ripple={{ color: '#640233' }}
  onPress={pressHandler}
>
  <View style={styles.buttonContainer}>
    <Text style={styles.buttonText}>{children}</Text>
  </View>
</Pressable>
```

`android_ripple`은 이름 그대로 Android 전용 피드백이다. iOS에서는 이 효과가 나타나지 않는다.

하지만 이 구조에서는 ripple이 버튼의 둥근 영역 안쪽에 깔끔하게 갇히지 않고 바깥쪽으로 번져 보일 수 있다.

### 외부 컨테이너와 내부 컨테이너 분리하기

ripple이 버튼 모양 안쪽에서만 보이도록 하려면 JSX 구조를 조금 바꾼다.

핵심은 외부 `View`가 둥근 모서리와 overflow 처리를 담당하고, 내부 `Pressable`이 실제 터치 영역과 배경색을 담당하도록 나누는 것이다.

```jsx
return (
  <View style={styles.buttonOuterContainer}>
    <Pressable
      android_ripple={{ color: '#640233' }}
      style={styles.buttonInnerContainer}
      onPress={pressHandler}
    >
      <Text style={styles.buttonText}>{children}</Text>
    </Pressable>
  </View>
);
```

스타일도 역할에 맞게 분리한다.

```jsx
const styles = StyleSheet.create({
  buttonOuterContainer: {
    borderRadius: 28,
    margin: 4,
    overflow: 'hidden',
  },
  buttonInnerContainer: {
    backgroundColor: '#27063c',
    paddingVertical: 8,
    paddingHorizontal: 16,
    elevation: 2,
  },
  buttonText: {
    color: 'white',
    textAlign: 'center',
  },
});
```

`overflow: 'hidden'`은 내부에서 발생하는 효과가 컨테이너 바깥으로 넘어가는 부분을 잘라낸다.

그래서 ripple이 둥근 버튼 바깥으로 번져 보이는 문제를 줄일 수 있다.

```text
buttonOuterContainer
-> borderRadius
-> margin
-> overflow: hidden

buttonInnerContainer
-> backgroundColor
-> padding
-> elevation
```

커스텀 버튼을 만들 때 이런 식으로 외부 컨테이너와 내부 터치 영역을 나누는 패턴은 자주 사용된다.

### iOS 눌림 피드백 추가하기

Android ripple은 iOS에서 동작하지 않는다.

iOS에서도 사용자가 버튼을 눌렀다는 느낌을 주려면 `Pressable`의 style prop에 함수를 전달할 수 있다.

`Pressable`은 눌림 상태가 바뀔 때마다 style 함수를 호출하고, 그 함수에 `{ pressed }` 값을 전달한다.

```jsx
<Pressable
  android_ripple={{ color: '#640233' }}
  style={({ pressed }) =>
    pressed
      ? [styles.buttonInnerContainer, styles.pressed]
      : styles.buttonInnerContainer
  }
  onPress={pressHandler}
>
  <Text style={styles.buttonText}>{children}</Text>
</Pressable>
```

`pressed`가 `true`이면 버튼이 눌린 상태다.

이때 기본 버튼 스타일과 눌림 스타일을 함께 적용하기 위해 style 배열을 사용한다.

```jsx
[styles.buttonInnerContainer, styles.pressed]
```

`pressed` 스타일은 불투명도를 낮춰 버튼이 눌린 것처럼 보이게 만든다.

```jsx
const styles = StyleSheet.create({
  pressed: {
    opacity: 0.75,
  },
});
```

`opacity`는 `0`부터 `1` 사이의 값을 사용한다.

```text
1
-> 완전히 불투명

0.75
-> 75% 불투명, 25% 투명

0
-> 완전히 투명
```

이 방식은 iOS에서도 눌림 피드백을 제공한다. 현재 코드에서는 Android에도 opacity 피드백이 함께 적용된다.

플랫폼별로 다른 피드백을 적용하거나 특정 플랫폼에서만 스타일을 적용하는 방법은 이후 강의에서 다룬다.

### 최종 PrimaryButton 예시

이번 단계의 `PrimaryButton`은 다음과 같은 형태가 된다.

```jsx
import { Pressable, StyleSheet, Text, View } from 'react-native';

function PrimaryButton({ children }) {
  function pressHandler() {
    console.log('Pressed!');
  }

  return (
    <View style={styles.buttonOuterContainer}>
      <Pressable
        android_ripple={{ color: '#640233' }}
        style={({ pressed }) =>
          pressed
            ? [styles.buttonInnerContainer, styles.pressed]
            : styles.buttonInnerContainer
        }
        onPress={pressHandler}
      >
        <Text style={styles.buttonText}>{children}</Text>
      </Pressable>
    </View>
  );
}

export default PrimaryButton;

const styles = StyleSheet.create({
  buttonOuterContainer: {
    borderRadius: 28,
    margin: 4,
    overflow: 'hidden',
  },
  buttonInnerContainer: {
    backgroundColor: '#27063c',
    paddingVertical: 8,
    paddingHorizontal: 16,
    elevation: 2,
  },
  buttonText: {
    color: 'white',
    textAlign: 'center',
  },
  pressed: {
    opacity: 0.75,
  },
});
```

### 프론트엔드 개발자 관점에서의 핵심 차이

웹에서는 버튼을 만들 때 `<button>` 요소에 `onClick`과 CSS를 붙이면 된다.

React Native에서는 다음을 직접 조합한다.

```text
터치 처리
-> Pressable

박스 스타일
-> View 또는 Pressable

텍스트 렌더링
-> Text

Android 피드백
-> android_ripple

iOS 피드백
-> pressed 상태 기반 스타일
```

즉 RN의 커스텀 버튼은 HTML 기본 요소 하나가 아니라 여러 핵심 컴포넌트와 플랫폼별 피드백을 조합해 만드는 UI 컴포넌트다.

## 정리

이번 강의에서는 `PrimaryButton`을 실제로 누를 수 있는 커스텀 버튼으로 발전시켰다. `Pressable`을 사용해 `onPress` 이벤트를 처리하고, Android에서는 `android_ripple`로 물결 효과를 추가했다.

또한 ripple이 둥근 버튼 바깥으로 번지지 않도록 외부 `View`와 내부 `Pressable`을 분리하고, 외부 컨테이너에 `overflow: 'hidden'`을 적용했다. iOS에서는 `Pressable`의 style 함수가 제공하는 `pressed` 상태를 사용해 opacity 기반 눌림 피드백을 구현했다.
