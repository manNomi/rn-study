# 15. 상태 기반 화면 전환

## 핵심 요약

- 게임 화면도 결국 일반 React 컴포넌트이므로 `GameScreen` 함수형 컴포넌트로 만든다.
- 네비게이션 라이브러리 없이도 React state와 조건부 렌더링으로 화면을 전환할 수 있다.
- `App` 컴포넌트가 사용자가 선택한 숫자인 `userNumber` state를 소유한다.
- `userNumber`가 없으면 `StartGameScreen`을 렌더링하고, 값이 있으면 `GameScreen`을 렌더링한다.
- `StartGameScreen`은 유효한 숫자가 확인되면 부모에게 받은 `onPickNumber` 함수를 호출한다.
- 자식 화면에서 부모 state를 바꾸려면 콜백 함수를 prop으로 내려주는 React의 일반적인 패턴을 사용한다.

## 학습 목표

- `GameScreen`을 새 화면 컴포넌트로 만드는 방법을 익힌다.
- 네비게이션 라이브러리 없이 조건부 렌더링으로 화면을 바꾸는 흐름을 이해한다.
- 화면 전환 상태를 어느 컴포넌트가 소유해야 하는지 판단하는 방법을 배운다.
- 자식 컴포넌트에서 부모 컴포넌트로 데이터를 올리는 prop callback 패턴을 복습한다.
- `null`을 초기 상태로 사용해 아직 선택된 숫자가 없음을 표현하는 이유를 이해한다.
- 유효성 검사를 통과한 값만 부모에게 전달하는 흐름을 익힌다.

## 주요 내용

### GameScreen 만들기

이전까지는 시작 화면만 있었다.

이제 사용자가 유효한 숫자를 선택하면 게임 화면으로 이동해야 한다.

`screens/GameScreen.js` 파일에 새 컴포넌트를 만든다.

```jsx
import { Text } from 'react-native';

function GameScreen() {
  return <Text>게임 화면!</Text>;
}

export default GameScreen;
```

여기서도 `GameScreen`은 특별한 네이티브 화면 객체가 아니라 일반 React 컴포넌트다.

React Native에서 화면 역할을 하는 컴포넌트도 결국 `View`, `Text` 같은 핵심 컴포넌트를 렌더링하는 함수형 컴포넌트일 뿐이다.

### 네비게이션 라이브러리 없이 화면 바꾸기

나중에는 React Navigation 같은 서드 파티 라이브러리로 여러 화면을 탐색하는 방법을 배운다.

하지만 이 단계에서는 앱 구조가 단순하므로 외부 의존성 없이 화면을 전환한다.

핵심은 React state와 조건부 렌더링이다.

```text
state가 아직 없음
-> StartGameScreen 렌더링

state에 선택된 숫자가 있음
-> GameScreen 렌더링
```

즉 화면 전환도 결국 “현재 state에 따라 어떤 컴포넌트를 보여줄지 결정하는 일”이다.

### App 컴포넌트가 화면 전환 상태를 소유한다

화면을 바꾸려면 `App` 컴포넌트에서 사용자가 숫자를 선택했는지 알아야 한다.

따라서 `App.js`에 state를 추가한다.

```jsx
import { useState } from 'react';
```

```jsx
export default function App() {
  const [userNumber, setUserNumber] = useState(null);

  // ...
}
```

초기값은 `null`로 둔다.

```text
null
-> 아직 사용자가 숫자를 선택하지 않음

number
-> 사용자가 유효한 숫자를 선택함
```

0을 초기값으로 둘 수도 있지만, 이 게임에서 0은 유효하지 않은 값이다.

따라서 “값이 아직 없다”는 의미를 명확히 표현하려면 `null`이 더 적절하다.

### 선택된 숫자를 저장하는 핸들러

사용자가 유효한 숫자를 확인하면 `App` 컴포넌트의 `userNumber`를 업데이트해야 한다.

이를 위해 핸들러 함수를 만든다.

```jsx
function pickedNumberHandler(pickedNumber) {
  setUserNumber(pickedNumber);
}
```

이 함수는 선택된 숫자를 인자로 받아 `userNumber` state에 저장한다.

```text
pickedNumberHandler(42)
-> setUserNumber(42)
-> App 컴포넌트 재실행
-> 조건부 렌더링 다시 평가
-> GameScreen 표시
```

React state가 바뀌면 컴포넌트 함수가 다시 실행되므로, 화면에 표시할 컴포넌트도 다시 결정된다.

### 조건부로 screen 변수 만들기

`App` 컴포넌트에서 어떤 화면을 렌더링할지 결정하기 위해 헬퍼 변수 `screen`을 사용할 수 있다.

처음에는 `StartGameScreen`을 기본값으로 둔다.

```jsx
let screen = <StartGameScreen />;
```

그리고 `userNumber`가 있으면 `GameScreen`으로 바꾼다.

```jsx
if (userNumber) {
  screen = <GameScreen />;
}
```

이제 JSX에서는 하드코딩된 `StartGameScreen` 대신 `screen` 변수를 렌더링한다.

```jsx
return (
  <LinearGradient
    colors={['#4e0329', '#ddb52f']}
    style={styles.rootScreen}
  >
    <ImageBackground
      source={require('./assets/images/background.png')}
      resizeMode="cover"
      style={styles.rootScreen}
      imageStyle={styles.backgroundImage}
    >
      {screen}
    </ImageBackground>
  </LinearGradient>
);
```

이제 `screen`에는 상황에 따라 `StartGameScreen` 또는 `GameScreen`이 들어간다.

### userNumber가 truthy인지 확인하기

조건문에서는 `userNumber`가 truthy인지 확인한다.

```jsx
if (userNumber) {
  screen = <GameScreen />;
}
```

초기값인 `null`은 falsy 값이다.

```text
userNumber === null
-> falsy
-> StartGameScreen 유지
```

유효한 숫자가 들어오면 truthy 값이 된다.

```text
userNumber === 42
-> truthy
-> GameScreen 렌더링
```

0은 falsy 값이지만, 이전 강의에서 0은 유효성 검사로 걸러지므로 이 조건에서는 문제가 되지 않는다.

### GameScreen import하기

`App.js`에서 새로 만든 `GameScreen`을 import한다.

```jsx
import GameScreen from './screens/GameScreen';
import StartGameScreen from './screens/StartGameScreen';
```

이제 `screen = <GameScreen />`처럼 사용할 수 있다.

### StartGameScreen에 콜백 전달하기

아직 `pickedNumberHandler`는 실행되지 않는다.

이 함수는 사용자가 시작 화면에서 숫자를 확인했을 때 실행되어야 한다.

즉 `App`에서 `StartGameScreen`으로 함수를 prop으로 전달해야 한다.

```jsx
let screen = (
  <StartGameScreen onPickNumber={pickedNumberHandler} />
);
```

`onPickNumber`라는 이름은 직접 정한 커스텀 prop 이름이다.

```text
App
-> pickedNumberHandler 함수 생성
-> StartGameScreen에 onPickNumber로 전달

StartGameScreen
-> 유효한 숫자가 확인되면 onPickNumber(chosenNumber) 호출
```

이 패턴은 앞에서 `PrimaryButton`에 `onPress`를 전달했던 것과 같은 React 패턴이다.

### StartGameScreen에서 prop 받기

`StartGameScreen`은 `onPickNumber` prop을 받아야 한다.

객체 구조 분해를 사용하면 바로 꺼낼 수 있다.

```jsx
function StartGameScreen({ onPickNumber }) {
  // ...
}
```

또는 `props.onPickNumber`로 접근할 수도 있다.

```jsx
function StartGameScreen(props) {
  props.onPickNumber(chosenNumber);
}
```

강의에서는 객체 구조 분해 방식을 사용한다.

### 유효한 숫자를 부모에게 전달하기

`confirmInputHandler` 안에는 이미 입력값 검증이 있다.

검증에 실패하면 Alert를 띄우고 `return`으로 함수 실행을 중단한다.

검증을 통과한 경우에만 부모에게 숫자를 전달한다.

```jsx
function confirmInputHandler() {
  const chosenNumber = parseInt(enteredNumber);

  if (
    isNaN(chosenNumber) ||
    chosenNumber <= 0 ||
    chosenNumber > 99
  ) {
    Alert.alert(
      '유효하지 않은 숫자!',
      '1과 99 사이의 숫자여야 함.',
      [
        {
          text: 'Okay',
          style: 'destructive',
          onPress: resetInputHandler,
        },
      ]
    );
    return;
  }

  onPickNumber(chosenNumber);
}
```

여기서 `chosenNumber`는 `parseInt()`로 변환된 숫자이며, 유효성 검사를 통과한 값이다.

```text
입력값 검증 실패
-> Alert
-> return
-> onPickNumber 호출 안 함

입력값 검증 성공
-> onPickNumber(chosenNumber)
-> App의 pickedNumberHandler 실행
-> userNumber 업데이트
-> GameScreen 표시
```

### 전체 App.js 흐름 예시

`App.js`의 핵심 구조는 다음과 같다.

```jsx
import { useState } from 'react';
import { ImageBackground, StyleSheet } from 'react-native';
import { LinearGradient } from 'expo-linear-gradient';

import GameScreen from './screens/GameScreen';
import StartGameScreen from './screens/StartGameScreen';

export default function App() {
  const [userNumber, setUserNumber] = useState(null);

  function pickedNumberHandler(pickedNumber) {
    setUserNumber(pickedNumber);
  }

  let screen = (
    <StartGameScreen onPickNumber={pickedNumberHandler} />
  );

  if (userNumber) {
    screen = <GameScreen />;
  }

  return (
    <LinearGradient
      colors={['#4e0329', '#ddb52f']}
      style={styles.rootScreen}
    >
      <ImageBackground
        source={require('./assets/images/background.png')}
        resizeMode="cover"
        style={styles.rootScreen}
        imageStyle={styles.backgroundImage}
      >
        {screen}
      </ImageBackground>
    </LinearGradient>
  );
}
```

이제 `App`은 현재 게임 상태를 기준으로 시작 화면 또는 게임 화면을 보여준다.

### 전체 StartGameScreen 흐름 예시

`StartGameScreen`에서는 `onPickNumber` prop을 받아 유효한 숫자를 부모에게 전달한다.

```jsx
function StartGameScreen({ onPickNumber }) {
  const [enteredNumber, setEnteredNumber] = useState('');

  function resetInputHandler() {
    setEnteredNumber('');
  }

  function confirmInputHandler() {
    const chosenNumber = parseInt(enteredNumber);

    if (
      isNaN(chosenNumber) ||
      chosenNumber <= 0 ||
      chosenNumber > 99
    ) {
      Alert.alert(
        '유효하지 않은 숫자!',
        '1과 99 사이의 숫자여야 함.',
        [
          {
            text: 'Okay',
            style: 'destructive',
            onPress: resetInputHandler,
          },
        ]
      );
      return;
    }

    onPickNumber(chosenNumber);
  }

  // JSX 생략
}
```

이 구조에서 `StartGameScreen`은 선택된 숫자를 직접 저장하지 않는다.

숫자를 확인하고 부모에게 알려주는 역할만 한다.

선택된 숫자의 최종 소유자는 `App`이다.

### 왜 App이 userNumber를 소유해야 할까?

`StartGameScreen`이 입력값을 가지고 있으므로 선택된 숫자도 그 안에서 관리할 수 있을 것처럼 보인다.

하지만 화면 전환을 결정하는 곳은 `App`이다.

```text
App
-> StartGameScreen을 보여줄지
-> GameScreen을 보여줄지 결정
```

따라서 화면 전환에 필요한 상태는 `App`이 소유해야 한다.

```text
입력 중인 값
-> StartGameScreen 내부 state

확정된 사용자 숫자
-> App state
```

이렇게 상태의 소유 위치를 나누면 역할이 선명해진다.

### 동작 확인

이제 앱에서 다음 흐름을 확인할 수 있다.

```text
빈 값 또는 잘못된 값 입력 후 Confirm
-> 기존처럼 Alert 표시
-> 시작 화면 유지

유효한 숫자 입력 후 Confirm
-> App의 userNumber가 업데이트
-> StartGameScreen 대신 GameScreen 렌더링
-> "게임 화면!" 텍스트 표시
```

아직 `GameScreen`에는 레이아웃이 없기 때문에 화면은 왼쪽 상단에 간단한 텍스트만 보인다.

하지만 핵심 목표인 화면 전환은 성공한 상태다.

### 프론트엔드 개발자 관점에서의 핵심 차이

웹 React에서도 라우터 없이 state로 화면을 바꿀 수 있다.

```jsx
{isLoggedIn ? <Dashboard /> : <Login />}
```

React Native에서도 같은 원리다.

```jsx
{userNumber ? <GameScreen /> : <StartGameScreen />}
```

다만 React Native 앱에서는 이후 복잡한 화면 전환, 스택, 탭, 드로어 등을 위해 React Navigation 같은 라이브러리를 많이 사용한다.

이번 강의에서는 앱 흐름이 단순하므로 조건부 렌더링으로 충분하다.

중요한 패턴은 자식이 부모의 화면 상태를 직접 바꾸지 않는다는 점이다.

```text
자식
-> prop으로 받은 콜백 호출

부모
-> state 업데이트
-> 렌더링할 화면 결정
```

이건 React 웹과 React Native 모두에서 공통으로 중요한 데이터 흐름이다.

## 정리

이번 강의에서는 `GameScreen` 컴포넌트를 만들고, `App` 컴포넌트의 `userNumber` state를 기준으로 `StartGameScreen`과 `GameScreen`을 조건부 렌더링했다. 아직 선택된 숫자가 없으면 시작 화면을 보여주고, 유효한 숫자가 선택되면 게임 화면을 보여준다.

또한 `App`에서 `pickedNumberHandler`를 만들고 이를 `StartGameScreen`에 `onPickNumber` prop으로 전달했다. `StartGameScreen`은 입력값 검증을 통과한 `chosenNumber`를 `onPickNumber(chosenNumber)`로 부모에게 전달하고, 부모는 `userNumber`를 업데이트해 화면을 전환한다.
