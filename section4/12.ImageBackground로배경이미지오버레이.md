# 12. ImageBackground로 배경 이미지 오버레이

## 핵심 요약

- 그러데이션 배경 위에 이미지를 겹치기 위해 React Native의 `ImageBackground` 컴포넌트를 사용한다.
- `ImageBackground`는 배경 이미지를 렌더링하면서 자식 컴포넌트를 그 위에 배치할 수 있게 해준다.
- 배경 이미지는 `assets/images/background.png`처럼 정적 자산 폴더에 저장한다.
- 이미지 경로는 문자열이 아니라 `require()` 함수로 전달한다.
- `resizeMode="cover"`를 사용하면 이미지 비율을 유지하면서 화면을 최대한 덮도록 크기가 조정된다.
- `style`은 `ImageBackground`의 바깥 컨테이너에 적용되고, `imageStyle`은 내부 이미지 자체에 적용된다.
- 이미지 위로 그러데이션이 비쳐 보이게 하려면 `imageStyle`에 `opacity`를 적용한다.

## 학습 목표

- `ImageBackground`를 사용해 배경 이미지를 추가하는 방법을 익힌다.
- `LinearGradient`와 `ImageBackground`를 중첩해 배경 레이어를 구성하는 방법을 이해한다.
- 정적 이미지 파일을 `require()`로 불러오는 React Native 방식에 익숙해진다.
- `resizeMode="cover"`의 의미를 이해한다.
- `style`과 `imageStyle`의 차이를 구분한다.
- 배경 이미지의 투명도를 조정해 그러데이션과 이미지를 함께 보이게 만드는 방법을 배운다.

## 주요 내용

### 배경 레이어 완성하기

이전 강의에서는 `LinearGradient`를 사용해 앱 전체 배경에 그러데이션을 적용했다.

이번에는 그 위에 배경 이미지를 추가한다.

목표는 다음과 같은 레이어를 만드는 것이다.

```text
가장 아래
-> LinearGradient

중간
-> ImageBackground

가장 위
-> StartGameScreen의 UI 요소
```

즉 배경 이미지는 그러데이션 위에 놓이고, 입력 카드 같은 UI 요소는 배경 이미지 위에 놓인다.

### ImageBackground란?

React Native에는 일반 이미지를 렌더링하는 `Image` 컴포넌트가 있다.

`ImageBackground`는 이름 그대로 이미지를 배경처럼 사용할 수 있게 해주는 컴포넌트다.

```text
Image
-> 이미지 자체를 화면 요소로 렌더링

ImageBackground
-> 이미지를 배경으로 렌더링
-> 그 안에 자식 컴포넌트를 배치 가능
```

배경 이미지 위에 텍스트, 버튼, 화면 컴포넌트를 올리고 싶다면 `ImageBackground`를 사용한다.

### ImageBackground import하기

`ImageBackground`는 `react-native`에서 제공하는 내장 컴포넌트다.

```jsx
import { ImageBackground, StyleSheet } from 'react-native';
```

이전 단계에서 `LinearGradient`를 사용하고 있으므로 `App.js`에는 다음 import가 함께 있게 된다.

```jsx
import { ImageBackground, StyleSheet } from 'react-native';
import { LinearGradient } from 'expo-linear-gradient';
```

### 이미지 파일 준비하기

배경으로 사용할 이미지는 프로젝트의 정적 자산 폴더에 둔다.

강의에서는 다음 구조를 사용한다.

```text
assets/
  images/
    background.png
```

이 이미지는 주사위 사진이며, 강의에서는 Unsplash에서 가져온 무료 이미지를 사용한다.

실제 프로젝트에서도 이런 배경 이미지는 직접 제작하거나, 라이선스를 확인한 무료 이미지 리소스에서 가져올 수 있다.

중요한 점은 앱에서 사용할 정적 이미지를 프로젝트 내부의 `assets` 폴더에 두고 관리한다는 것이다.

### require로 이미지 경로 지정하기

React Native에서 정적 이미지를 사용할 때는 일반 문자열 경로를 직접 전달하지 않는다.

다음 방식은 웹에서 익숙한 형태지만 React Native 정적 이미지 로딩 방식으로는 적절하지 않다.

```jsx
source="./assets/images/background.png"
```

대신 `require()`를 사용한다.

```jsx
source={require('./assets/images/background.png')}
```

현재 코드는 `App.js`에서 작성하고 있고, `assets` 폴더가 `App.js`와 같은 프로젝트 루트에 있으므로 `./assets/...` 경로를 사용한다.

```text
App.js
assets/
  images/
    background.png

App.js 기준 경로
-> ./assets/images/background.png
```

만약 다른 폴더 안의 컴포넌트에서 이미지를 불러온다면 상대 경로가 달라질 수 있다.

### LinearGradient 안에 ImageBackground 넣기

이전 단계의 구조는 다음과 같았다.

```jsx
<LinearGradient
  colors={['#4e0329', '#ddb52f']}
  style={styles.rootScreen}
>
  <StartGameScreen />
</LinearGradient>
```

이제 `StartGameScreen`을 바로 렌더링하지 않고, `ImageBackground`로 한 번 감싼다.

```jsx
<LinearGradient
  colors={['#4e0329', '#ddb52f']}
  style={styles.rootScreen}
>
  <ImageBackground
    source={require('./assets/images/background.png')}
    style={styles.rootScreen}
  >
    <StartGameScreen />
  </ImageBackground>
</LinearGradient>
```

이 구조에서는 `LinearGradient`가 가장 바깥 배경이고, `ImageBackground`가 그 위에 이미지를 올린다.

그리고 `StartGameScreen`은 `ImageBackground`의 자식이므로 시각적으로 배경 이미지 위에 표시된다.

### ImageBackground에도 flex: 1이 필요하다

`LinearGradient`에 `flex: 1`을 적용했던 것처럼, `ImageBackground`에도 화면 전체를 차지하도록 스타일을 적용해야 한다.

```jsx
<ImageBackground
  source={require('./assets/images/background.png')}
  style={styles.rootScreen}
>
  <StartGameScreen />
</ImageBackground>
```

```jsx
const styles = StyleSheet.create({
  rootScreen: {
    flex: 1,
  },
});
```

`LinearGradient`가 화면 전체를 덮고 있더라도, 그 안의 `ImageBackground`가 자식 콘텐츠 크기만큼만 차지하면 이미지는 화면 전체에 깔리지 않는다.

```text
LinearGradient
-> flex: 1 필요

ImageBackground
-> flex: 1 필요
```

배경 레이어가 여러 개일 때는 각 레이어가 화면 전체를 차지하는지 확인해야 한다.

### resizeMode로 이미지 크기 조정하기

이미지 파일의 크기와 실제 기기 화면 크기는 대부분 다르다.

그래서 이미지가 화면에 어떻게 맞춰질지 정해야 한다.

`ImageBackground`에는 `resizeMode` prop이 있다.

```jsx
<ImageBackground
  source={require('./assets/images/background.png')}
  resizeMode="cover"
  style={styles.rootScreen}
>
  <StartGameScreen />
</ImageBackground>
```

`cover`는 이미지 비율을 유지하면서 가능한 공간을 최대한 덮도록 이미지를 확대 또는 축소한다.

```text
resizeMode="cover"
-> 이미지 비율 유지
-> 화면을 빈 공간 없이 덮음
-> 일부 영역은 잘릴 수 있음
```

웹 CSS에서 `background-size: cover`나 `object-fit: cover`를 사용해본 프론트엔드 개발자라면 비슷한 개념으로 이해할 수 있다.

### 이미지가 그러데이션을 가리는 문제

`ImageBackground`를 추가하면 이미지가 보이지만, 동시에 그러데이션이 사라진 것처럼 보일 수 있다.

이는 실제로 그러데이션이 제거된 것이 아니라, 이미지가 불투명하게 그 위를 덮고 있기 때문이다.

```text
LinearGradient
-> 여전히 아래에 있음

ImageBackground
-> 불투명 이미지로 위를 덮음

결과
-> 그러데이션이 보이지 않음
```

그러데이션과 이미지를 함께 보이게 하려면 이미지 자체를 투명하게 만들어야 한다.

### style과 imageStyle의 차이

`ImageBackground`에는 `style` prop뿐 아니라 `imageStyle` prop도 있다.

둘은 적용 대상이 다르다.

```text
style
-> ImageBackground의 바깥 View에 적용

imageStyle
-> 내부 Image 컴포넌트에 적용
```

`ImageBackground`는 내부적으로 `View`와 `Image`를 조합한 컴포넌트라고 볼 수 있다.

그래서 바깥 컨테이너의 크기나 레이아웃은 `style`로 제어하고, 실제 이미지의 스타일은 `imageStyle`로 제어한다.

이미지 투명도를 조정하려면 내부 이미지 자체에 적용해야 하므로 `imageStyle`을 사용한다.

### imageStyle로 이미지 투명도 조정하기

먼저 이미지 스타일 객체를 만든다.

```jsx
const styles = StyleSheet.create({
  rootScreen: {
    flex: 1,
  },
  backgroundImage: {
    opacity: 0.15,
  },
});
```

그 다음 `ImageBackground`에 `imageStyle` prop으로 전달한다.

```jsx
<ImageBackground
  source={require('./assets/images/background.png')}
  resizeMode="cover"
  style={styles.rootScreen}
  imageStyle={styles.backgroundImage}
>
  <StartGameScreen />
</ImageBackground>
```

`opacity: 0.15`는 이미지가 15% 정도만 불투명하게 보인다는 뜻이다.

```text
opacity: 1
-> 완전히 불투명

opacity: 0.15
-> 매우 투명
-> 아래의 그러데이션이 강하게 비쳐 보임

opacity: 0
-> 완전히 투명
```

이렇게 하면 주사위 이미지가 아주 은은하게 보이고, 아래의 그러데이션도 함께 보인다.

### 전체 App.js 예시

이번 단계의 `App.js`는 다음과 같은 형태가 된다.

```jsx
import { ImageBackground, StyleSheet } from 'react-native';
import { LinearGradient } from 'expo-linear-gradient';

import StartGameScreen from './screens/StartGameScreen';

export default function App() {
  return (
    <LinearGradient
      colors={['#4e0329', '#ddb52f']}
      style={styles.rootScreen}
    >
      <ImageBackground
        source={require('./assets/images/background.png')}
        resizeMode="cover"
        style={styles.rootScreen}
        imageStyle={styles.backgroundImage}
      >
        <StartGameScreen />
      </ImageBackground>
    </LinearGradient>
  );
}

const styles = StyleSheet.create({
  rootScreen: {
    flex: 1,
  },
  backgroundImage: {
    opacity: 0.15,
  },
});
```

이 구조로 다음 효과를 얻는다.

```text
LinearGradient
-> 화면 전체 그러데이션

ImageBackground
-> 화면 전체 배경 이미지
-> 이미지 투명도 0.15

StartGameScreen
-> 이미지 위에 표시되는 실제 UI
```

### 배경 이미지를 다룰 때 확인할 점

배경 이미지는 앱 분위기를 크게 바꾸지만, 잘못 쓰면 UI를 방해할 수 있다.

확인해야 할 점은 다음과 같다.

```text
이미지가 너무 선명해서 UI를 방해하지 않는가
텍스트와 버튼이 충분히 읽히는가
기기 화면 비율이 달라도 어색하지 않은가
resizeMode 설정이 적절한가
이미지 라이선스가 사용 목적에 맞는가
```

이번 강의에서는 `opacity: 0.15`로 이미지를 거의 투명하게 만들어 UI보다 배경의 질감 역할을 하도록 했다.

### 프론트엔드 개발자 관점에서의 핵심 차이

웹에서는 CSS로 다음처럼 배경 이미지와 그러데이션을 한 번에 조합할 수 있다.

```css
body {
  min-height: 100vh;
  background:
    linear-gradient(#4e0329, #ddb52f),
    url('/images/background.png');
  background-size: cover;
}
```

React Native에서는 CSS 배경 레이어 문법 대신 컴포넌트를 중첩해서 레이어를 만든다.

```text
웹
-> CSS background 레이어

React Native
-> LinearGradient 컴포넌트
-> ImageBackground 컴포넌트
-> 화면 컴포넌트
```

즉 React Native에서는 배경도 컴포넌트 트리의 일부로 생각해야 한다.

또 웹에서는 이미지 경로를 문자열 URL로 쓰는 경우가 많지만, React Native 정적 이미지는 `require()`를 사용하는 패턴에 익숙해져야 한다.

```text
웹
-> url('/images/background.png')

React Native
-> require('./assets/images/background.png')
```

## 정리

이번 강의에서는 `ImageBackground`를 사용해 그러데이션 위에 배경 이미지를 추가했다. 이미지는 `assets/images/background.png`에 두고, `source={require('./assets/images/background.png')}` 형태로 불러왔다.

또한 `resizeMode="cover"`로 이미지가 화면을 덮도록 설정하고, `style={styles.rootScreen}`으로 `ImageBackground` 자체도 화면 전체를 차지하게 만들었다. 처음에는 이미지가 불투명해서 그러데이션을 가렸지만, `imageStyle={styles.backgroundImage}`와 `opacity: 0.15`를 사용해 이미지가 은은하게 보이도록 조정했다.
