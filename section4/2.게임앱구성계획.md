# 2. 게임 앱 구성 계획

## 핵심 요약

- 이번 섹션은 새 Expo 프로젝트의 간단한 `App` 컴포넌트에서 시작한다.
- 목표는 1부터 99 사이의 숫자를 맞히는 게임 앱을 만드는 것이다.
- 앱은 시작 화면, 게임 화면, 게임 오버 화면으로 구성된다.
- 숫자 입력 검증, 게임 시작, 힌트 제공, 잘못된 힌트 경고, 추측 기록, 게임 재시작 기능이 필요하다.
- 여러 화면과 재사용 컴포넌트를 만들며 React Native 앱 구조를 더 정교하게 연습한다.

## 학습 목표

- 숫자 맞히기 게임 앱의 전체 화면 흐름을 이해한다.
- 각 화면에서 필요한 기능과 상태를 미리 정리한다.
- 여러 화면으로 구성된 앱을 어떻게 나눌지 감을 잡는다.
- 직접 만들 재사용 컴포넌트가 왜 필요한지 이해한다.
- 지금까지 배운 React와 React Native 지식으로 기본 프로토타입을 만들 수 있음을 확인한다.

## 주요 내용

### 시작 프로젝트

강의는 새로 생성한 Expo 프로젝트에서 출발한다.

첨부된 시작 프로젝트는 매우 단순한 형태다.

```text
App 컴포넌트 하나
몇 가지 더미 콘텐츠
기본 스타일
```

이번 섹션에서는 이 단순한 시작점에서 숫자 맞히기 게임 앱을 하나씩 구축한다.

### 전체 앱 아이디어

게임은 사용자가 선택한 숫자를 휴대폰이 맞히는 방식이다.

기본 흐름은 다음과 같다.

```text
사용자가 1부터 99 사이의 숫자를 입력한다.
게임을 시작한다.
휴대폰이 숫자를 추측한다.
사용자는 추측값이 너무 큰지 작은지 힌트를 준다.
휴대폰은 힌트를 기반으로 다시 추측한다.
정답을 맞히면 게임 오버 화면으로 이동한다.
게임 기록을 확인하고 다시 시작할 수 있다.
```

단순해 보이지만 이 흐름 안에 화면 전환, 입력 검증, 상태 관리, 목록 표시, 경고창, 재사용 컴포넌트 같은 핵심 요소가 들어 있다.

### 화면 1: 시작 화면

첫 번째 화면은 사용자가 숫자를 입력하는 시작 화면이다.

이 화면에서 필요한 기능은 다음과 같다.

```text
1부터 99 사이의 숫자 입력
잘못된 숫자 입력 방지
입력값 초기화
게임 시작
```

입력값이 1보다 작거나 99보다 크면 게임을 시작하면 안 된다. 숫자가 아닌 값이나 빈 값도 처리해야 한다.

게임을 시작하면 다음 화면인 게임 화면으로 이동한다.

### 화면 2: 게임 화면

두 번째 화면은 실제 게임이 진행되는 화면이다.

이 화면에서는 휴대폰이 사용자의 숫자를 추측한다.

사용자는 매번 다음 중 하나의 힌트를 준다.

```text
휴대폰의 추측값이 너무 높다.
휴대폰의 추측값이 너무 낮다.
```

휴대폰은 힌트를 기반으로 다음 추측 범위를 좁혀 간다.

또한 사용자가 잘못된 힌트를 주는 경우를 막아야 한다. 예를 들어 실제 정답이 더 작은 숫자인데 사용자가 "더 높다"고 누르면 앱이 경고를 표시해야 한다.

게임 화면에서 필요한 기능은 다음과 같다.

```text
휴대폰의 현재 추측값 표시
더 높음 / 더 낮음 힌트 입력
잘못된 힌트 경고
새 추측값 생성
추측 기록 저장
추측 기록 목록 표시
```

추측 기록은 계속 쌓이기 때문에 화면 높이를 넘어갈 수 있다. 따라서 스크롤 가능한 목록으로 표시해야 한다.

### 화면 3: 게임 오버 화면

휴대폰이 정답을 맞히면 게임 오버 화면으로 이동한다.

이 화면에서는 게임 결과를 요약해서 보여준다.

필요한 요소는 다음과 같다.

```text
게임 종료 메시지
정답 숫자
휴대폰이 몇 번 만에 맞혔는지
결과 이미지
새 게임 시작 버튼
```

사용자는 이 화면에서 다시 게임을 시작할 수 있어야 한다.

### 재사용 컴포넌트

이번 앱은 단순히 화면을 쭉 작성하는 방식으로 만들지 않는다.

여러 곳에서 사용할 수 있는 컴포넌트를 직접 만든다.

예상되는 재사용 컴포넌트는 다음과 같다.

- 커스텀 버튼
- 제목 또는 안내 텍스트
- 카드 형태의 컨테이너
- 추측 기록 아이템
- 숫자 표시 컴포넌트

React Native 앱도 결국 React 앱이므로, 반복되는 UI와 로직을 컴포넌트로 잘 나누는 것이 중요하다.

### 직접 프로토타입을 만들어 보는 것도 좋은 연습

아직 배워야 할 새로운 컴포넌트와 스타일링 기법이 많기 때문에 최종 앱과 똑같은 모양을 처음부터 만들기는 어렵다.

하지만 지금까지 배운 지식만으로도 기본 프로토타입은 만들 수 있다.

예를 들면 다음 정도는 시도해 볼 수 있다.

```text
숫자 입력 화면 만들기
입력값을 state로 관리하기
버튼으로 화면 상태 바꾸기
현재 추측값 표시하기
추측 기록을 배열로 저장하기
목록으로 렌더링하기
```

직접 먼저 만들어 본 뒤 강의를 따라가면, 이후에 배우는 개념이 왜 필요한지 더 잘 이해할 수 있다.

## 정리

이번 강의에서는 숫자 맞히기 게임 앱의 전체 구조를 미리 살펴봤다. 앱은 시작 화면, 게임 화면, 게임 오버 화면으로 구성되며, 입력 검증, 힌트 처리, 잘못된 힌트 경고, 추측 기록, 게임 재시작 기능을 포함한다.

이 앱을 만드는 과정에서 여러 화면 구성, 복잡한 레이아웃, 재사용 컴포넌트, 스크롤 가능한 기록 목록 등 React Native 앱 개발에 필요한 더 깊은 개념을 학습하게 된다.
