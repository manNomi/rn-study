# 13. 사용자 입력 상태와 버튼 이벤트 연결

## 핵심 요약

- 게임 로직의 첫 단계는 사용자가 `TextInput`에 입력한 숫자를 React state로 관리하는 것이다.
- `TextInput`의 값은 숫자 키보드를 사용하더라도 항상 문자열로 들어온다.
- 입력값을 제어하려면 `value` prop에 state를 연결하고, `onChangeText`에서 state를 업데이트한다.
- Confirm 버튼을 누르면 입력값을 확인하는 `confirmInputHandler`가 실행되도록 연결한다.
- 재사용 가능한 `PrimaryButton`은 외부에서 전달받은 `onPress` 함수를 내부 `Pressable`의 `onPress`에 넘겨야 한다.
- 입력값 검증과 경고창 처리는 이 이벤트 연결 구조를 만든 뒤 이어서 구현한다.

## 학습 목표

- `useState`로 사용자 입력값을 관리하는 방법을 복습한다.
- `TextInput`을 controlled input으로 만드는 방법을 이해한다.
- 숫자 입력값이 문자열로 관리되는 이유를 배운다.
- 커스텀 버튼 컴포넌트에 이벤트 핸들러를 prop으로 전달하는 패턴을 익힌다.
- `PrimaryButton`의 prop 이름과 내부 `Pressable`의 `onPress` prop을 구분한다.
- 입력 검증과 화면 전환 로직을 추가하기 위한 기본 구조를 만든다.

## 주요 내용

### 이제 UI에서 게임 로직으로

지금까지는 시작 화면의 UI와 배경을 만들었다.

이제부터는 실제 게임 로직을 추가한다.

필요한 동작은 다음과 같다.

```text
사용자가 입력한 숫자 가져오기
Reset 버튼을 누르면 입력값 초기화하기
Confirm 버튼을 누르면 입력값 확인하기
입력값이 유효하면 게임 화면으로 넘어가기
입력값이 유효하지 않으면 경고창 보여주기
```

이번 강의에서는 이 로직의 기반이 되는 입력 상태와 버튼 이벤트 연결을 먼저 만든다.

### 입력값은 state로 관리한다

React에서는 사용자 입력처럼 변하는 값은 state로 관리한다.

`StartGameScreen`에서 `useState`를 import한다.

```jsx
import { useState } from 'react';
```

그리고 입력된 숫자를 저장할 state를 만든다.

```jsx
function StartGameScreen() {
  const [enteredNumber, setEnteredNumber] = useState('');

  // ...
}
```

초기값은 숫자 `0`이 아니라 빈 문자열 `''`이다.

이유는 `TextInput`이 다루는 값이 기본적으로 문자열이기 때문이다.

```text
keyboardType="number-pad"
-> 숫자 키보드를 보여줌

TextInput value
-> 여전히 문자열
```

웹의 `<input>`에서도 숫자를 입력해도 기본 입력값은 문자열로 들어오는 것과 비슷하다.

### TextInput을 controlled input으로 만들기

입력값 state를 `TextInput`과 연결하려면 `value` prop을 사용한다.

```jsx
<TextInput
  style={styles.numberInput}
  maxLength={2}
  keyboardType="number-pad"
  value={enteredNumber}
/>
```

이제 `enteredNumber` state가 `TextInput`에 표시되는 값의 기준이 된다.

이런 방식의 입력 필드를 controlled input이라고 볼 수 있다.

```text
state
-> 입력 필드의 현재 값

setState
-> 입력 필드 값을 바꾸는 유일한 통로
```

이 구조를 만들면 나중에 Reset 버튼에서 `setEnteredNumber('')`를 호출해 입력란을 비울 수 있다.

### onChangeText로 입력값 업데이트하기

사용자가 키보드로 값을 입력할 때마다 state도 업데이트되어야 한다.

이를 위해 입력 핸들러 함수를 만든다.

```jsx
function numberInputHandler(enteredText) {
  setEnteredNumber(enteredText);
}
```

그리고 `TextInput`의 `onChangeText` prop에 연결한다.

```jsx
<TextInput
  style={styles.numberInput}
  maxLength={2}
  keyboardType="number-pad"
  value={enteredNumber}
  onChangeText={numberInputHandler}
/>
```

`onChangeText`는 React Native의 `TextInput`이 제공하는 prop이다.

텍스트가 바뀔 때마다 React Native가 이 함수를 호출하고, 새 입력값을 첫 번째 인자로 전달한다.

```text
사용자가 5 입력
-> onChangeText 실행
-> numberInputHandler('5')
-> setEnteredNumber('5')
-> TextInput value도 '5'로 갱신
```

여기서도 함수를 직접 실행하면 안 된다.

```jsx
onChangeText={numberInputHandler}
```

이렇게 함수 참조를 넘겨야 한다.

### Confirm 버튼 핸들러 만들기

이제 Confirm 버튼을 눌렀을 때 실행할 함수를 만든다.

```jsx
function confirmInputHandler() {
  // 입력값 검증 로직은 이어서 추가
}
```

이 함수는 앞으로 다음 역할을 하게 된다.

```text
enteredNumber 확인
숫자로 변환
1 이상 99 이하인지 검증
유효하지 않으면 경고 표시
유효하면 게임 화면으로 이동
```

이번 단계에서는 먼저 이 함수가 버튼을 눌렀을 때 실행되도록 연결하는 것이 중요하다.

### PrimaryButton을 재사용 가능하게 바꾸기

현재 `PrimaryButton`은 내부에서 `pressHandler`를 만들고 `console.log('Pressed!')`만 실행하고 있었다.

```jsx
function PrimaryButton({ children }) {
  function pressHandler() {
    console.log('Pressed!');
  }

  return (
    <View style={styles.buttonOuterContainer}>
      <Pressable onPress={pressHandler}>
        <Text>{children}</Text>
      </Pressable>
    </View>
  );
}
```

이 구조는 테스트에는 괜찮지만 재사용 가능한 버튼으로는 부족하다.

버튼마다 실행해야 하는 동작이 다르기 때문이다.

```text
Reset 버튼
-> 입력값 초기화

Confirm 버튼
-> 입력값 검증
-> 게임 시작
```

따라서 `PrimaryButton`은 외부에서 실행할 함수를 prop으로 받아야 한다.

### onPress prop 전달하기

`PrimaryButton`에서 `onPress` prop을 받는다.

```jsx
function PrimaryButton({ children, onPress }) {
  return (
    <View style={styles.buttonOuterContainer}>
      <Pressable
        onPress={onPress}
      >
        <Text style={styles.buttonText}>{children}</Text>
      </Pressable>
    </View>
  );
}
```

이제 내부의 임시 `pressHandler`는 제거해도 된다.

중요한 점은 이름이 같아 보여도 역할이 다르다는 것이다.

```text
PrimaryButton의 onPress
-> 우리가 만든 커스텀 prop
-> 부모 컴포넌트에서 전달받는 함수

Pressable의 onPress
-> React Native가 제공하는 정해진 prop
-> 실제 터치 이벤트가 발생했을 때 호출됨
```

왼쪽의 `onPress`는 `Pressable`이 이해하는 prop 이름이므로 정해져 있다.

오른쪽의 `onPress`는 `PrimaryButton`이 부모로부터 받은 값이다.

```jsx
<Pressable onPress={onPress}>
```

즉 부모가 전달한 함수를 그대로 내부 `Pressable`에 연결하는 구조다.

### StartGameScreen에서 Confirm 버튼 연결하기

이제 `StartGameScreen`에서 Confirm 버튼에 `onPress` prop을 전달한다.

```jsx
<PrimaryButton onPress={confirmInputHandler}>
  Confirm
</PrimaryButton>
```

이렇게 하면 흐름은 다음과 같다.

```text
사용자가 Confirm 버튼 누름
-> PrimaryButton 내부 Pressable이 press 이벤트 감지
-> Pressable의 onPress 실행
-> 부모가 전달한 confirmInputHandler 실행
```

전체 구조는 다음처럼 볼 수 있다.

```jsx
function StartGameScreen() {
  const [enteredNumber, setEnteredNumber] = useState('');

  function numberInputHandler(enteredText) {
    setEnteredNumber(enteredText);
  }

  function confirmInputHandler() {
    // 이후 입력값 검증 로직 추가
  }

  return (
    <View style={styles.inputContainer}>
      <TextInput
        style={styles.numberInput}
        maxLength={2}
        keyboardType="number-pad"
        value={enteredNumber}
        onChangeText={numberInputHandler}
      />

      <View style={styles.buttonsContainer}>
        <View style={styles.buttonContainer}>
          <PrimaryButton>Reset</PrimaryButton>
        </View>
        <View style={styles.buttonContainer}>
          <PrimaryButton onPress={confirmInputHandler}>
            Confirm
          </PrimaryButton>
        </View>
      </View>
    </View>
  );
}
```

### Reset 버튼은 왜 value 연결이 필요할까?

강의에서는 앞으로 Reset 버튼으로 입력값을 비울 예정이다.

이때 `TextInput`에 `value={enteredNumber}`가 연결되어 있어야 state 변화가 입력 필드에도 반영된다.

예를 들어 다음과 같은 핸들러를 만들 수 있다.

```jsx
function resetInputHandler() {
  setEnteredNumber('');
}
```

그리고 Reset 버튼에 연결하면 된다.

```jsx
<PrimaryButton onPress={resetInputHandler}>
  Reset
</PrimaryButton>
```

이 구조가 동작하려면 `TextInput`이 state를 value로 사용하고 있어야 한다.

```text
setEnteredNumber('')
-> enteredNumber가 빈 문자열로 변경
-> value={enteredNumber}
-> TextInput도 비워짐
```

### 입력값 검증이 필요한 이유

게임은 사용자가 1부터 99 사이의 숫자를 고르는 것을 전제로 한다.

따라서 다음 값은 유효하지 않다.

```text
빈 문자열
0
-1
100 이상
숫자로 변환할 수 없는 값
```

현재 입력 필드는 `keyboardType="number-pad"`와 `maxLength={2}`를 사용하고 있지만, 이것만으로 비즈니스 규칙이 완전히 보장되는 것은 아니다.

```text
keyboardType
-> 키보드 UI를 숫자 중심으로 바꿈

maxLength
-> 입력 가능한 문자 수 제한

유효성 검사
-> 실제 게임 규칙을 코드로 보장
```

따라서 Confirm 버튼을 눌렀을 때 `enteredNumber`를 검사하는 로직이 반드시 필요하다.

이번 강의에서는 그 검증 로직을 넣을 함수와 버튼 연결 구조를 만든다.

### 다음 단계에서 추가될 로직

이제 `confirmInputHandler` 안에서 입력값을 검사할 준비가 되었다.

다음 단계에서는 다음 로직이 들어갈 수 있다.

```text
enteredNumber를 숫자로 변환
빈 값인지 확인
1보다 작은지 확인
99보다 큰지 확인
유효하지 않으면 Alert 표시
유효하면 게임 화면으로 이동
```

React Native에는 경고창을 띄우기 위한 API도 제공되므로, 유효하지 않은 값이 입력되었을 때 사용자에게 안내 메시지를 보여줄 수 있다.

### 프론트엔드 개발자 관점에서의 핵심 차이

웹 React를 알고 있다면 이번 흐름은 아주 익숙하다.

```text
useState로 입력값 관리
value로 input 제어
onChange 계열 이벤트에서 setState
버튼 클릭 시 handler 실행
```

React Native에서도 원리는 같다.

다만 컴포넌트와 이벤트 이름이 다르다.

```text
웹 React
-> <input value={...} onChange={...} />
-> <button onClick={...} />

React Native
-> <TextInput value={...} onChangeText={...} />
-> <Pressable onPress={...} />
```

그리고 커스텀 버튼을 만든 경우에는 부모 컴포넌트가 넘긴 handler를 내부 `Pressable`까지 전달해야 한다.

```text
StartGameScreen
-> PrimaryButton onPress prop 전달

PrimaryButton
-> Pressable onPress에 전달

Pressable
-> 실제 터치 이벤트에서 함수 실행
```

즉 React Native에서도 데이터 흐름은 React답게 단방향이다.

## 정리

이번 강의에서는 게임 로직의 첫 단계로 사용자 입력값을 `useState`로 관리하고, `TextInput`에 `value`와 `onChangeText`를 연결했다. 숫자 키보드를 사용하더라도 `TextInput` 값은 문자열이므로, `enteredNumber` state도 빈 문자열로 시작한다.

또한 `PrimaryButton`이 재사용 가능한 버튼이 되도록 외부에서 `onPress` prop을 받아 내부 `Pressable`의 `onPress`에 전달했다. 이제 Confirm 버튼은 `confirmInputHandler`를 실행할 수 있으며, 다음 단계에서 이 함수 안에 입력값 검증과 경고창 로직을 추가할 수 있다.
