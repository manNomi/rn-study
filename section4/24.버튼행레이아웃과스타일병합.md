# 24. 버튼 행 레이아웃과 스타일 병합

## 핵심 요약

- 게임 화면의 `+`, `-` 버튼은 가로로 나란히 배치하는 것이 자연스럽다.
- 버튼 두 개를 감싸는 외부 `View`에는 `flexDirection: 'row'`를 적용한다.
- 각 버튼을 개별 `View`로 감싸고 `flex: 1`을 적용하면 두 버튼의 너비를 균등하게 맞출 수 있다.
- `InstructionText`처럼 재사용 컴포넌트에 화면별 스타일을 추가하고 싶을 때는 `style` prop을 받을 수 있다.
- React Native의 `style` prop은 스타일 객체뿐 아니라 스타일 객체 배열도 받을 수 있다.
- 스타일 배열은 왼쪽에서 오른쪽 순서로 병합되며, 뒤쪽 스타일이 앞쪽 스타일을 덮어쓸 수 있다.
- 이 패턴을 사용하면 CSS의 계단식 스타일처럼 기본 스타일과 추가 스타일을 조합할 수 있다.

## 학습 목표

- `View`와 Flexbox를 사용해 버튼 두 개를 가로로 배치하는 방법을 익힌다.
- `flex: 1`을 이용해 버튼 영역을 균등 분배하는 패턴을 복습한다.
- 재사용 컴포넌트에 외부 스타일을 전달하는 방법을 배운다.
- `style={[baseStyle, overrideStyle]}` 배열 패턴을 이해한다.
- React Native에서 CSS 상속이나 cascade가 없을 때 스타일 확장을 어떻게 구현하는지 이해한다.

## 주요 내용

### 게임 화면의 버튼 레이아웃 다듬기

현재 `GameScreen`에는 `Card` 안에 안내 문구와 `+`, `-` 버튼이 있다.

```text
Card
-> Higher or lower?
-> + 버튼
-> - 버튼
```

버튼 두 개는 위아래보다 가로로 나란히 놓는 편이 게임 컨트롤처럼 보인다.

이를 위해 버튼들을 감싸는 `View`에 `flexDirection: 'row'`를 적용한다.

```jsx
<View style={styles.buttonsContainer}>
  <View style={styles.buttonContainer}>
    <PrimaryButton>+</PrimaryButton>
  </View>
  <View style={styles.buttonContainer}>
    <PrimaryButton>-</PrimaryButton>
  </View>
</View>
```

### buttonsContainer와 buttonContainer

버튼 행을 만들기 위해 두 단계의 컨테이너를 사용한다.

```text
buttonsContainer
-> 버튼들을 가로 방향으로 배치

buttonContainer
-> 각 버튼이 동일한 공간을 차지하게 함
```

스타일은 다음처럼 구성할 수 있다.

```jsx
const styles = StyleSheet.create({
  buttonsContainer: {
    flexDirection: 'row',
  },
  buttonContainer: {
    flex: 1,
  },
});
```

`buttonsContainer`는 자식 요소를 가로로 배치한다.

`buttonContainer`는 각각의 버튼 영역이 사용 가능한 공간을 균등하게 나눠 갖도록 한다.

```text
flexDirection: 'row'
-> + 버튼과 - 버튼을 옆으로 배치

flex: 1
-> 두 버튼 영역이 같은 너비를 차지
```

### StartGameScreen의 패턴과 같다

이 패턴은 이전에 `StartGameScreen`의 Reset/Confirm 버튼을 배치할 때 사용한 방식과 같다.

```text
StartGameScreen
-> Reset / Confirm 버튼 행

GameScreen
-> + / - 버튼 행
```

유사한 코드이므로 별도의 `ButtonRow` 같은 컴포넌트로 분리할 수도 있다.

하지만 이번 경우에는 스타일 한두 줄 수준이라 강의에서는 단순히 같은 패턴을 다시 적용한다.

```text
분리해도 되는 경우
-> 같은 버튼 행 구조가 여러 화면에서 반복됨
-> 버튼 행 자체에 복잡한 스타일이나 동작이 있음

굳이 분리하지 않아도 되는 경우
-> 코드가 매우 짧음
-> 재사용 가능성이 낮음
```

재사용 컴포넌트는 무조건 많이 만드는 것이 아니라, 반복과 복잡도에 맞춰 결정한다.

### 버튼과 안내 문구 사이 간격 추가하기

버튼이 가로로 정렬되면 화면이 더 좋아지지만, 안내 문구와 버튼 사이가 너무 붙어 보일 수 있다.

간격을 추가하는 방법은 여러 가지다.

```text
방법 1
-> buttonsContainer에 marginTop 추가

방법 2
-> InstructionText에 marginBottom 추가
```

강의에서는 `InstructionText`에 아래쪽 여백을 추가하는 방식을 사용한다.

여기서 중요한 새 개념은 재사용 컴포넌트에 추가 스타일을 전달하는 방법이다.

### 재사용 컴포넌트에 스타일을 추가하고 싶은 상황

`InstructionText`는 기본 스타일을 내부에 가지고 있다.

```jsx
const styles = StyleSheet.create({
  instructionText: {
    color: Colors.accent500,
    fontSize: 24,
  },
});
```

하지만 어떤 화면에서는 기본 스타일에 추가로 여백을 주고 싶을 수 있다.

```text
StartGameScreen
-> 기본 안내 문구 스타일만 필요

GameScreen
-> 기본 안내 문구 스타일 + marginBottom 필요
```

이럴 때 `InstructionText` 내부 스타일을 직접 바꾸면 모든 사용처에 영향을 준다.

필요한 곳에만 추가 스타일을 적용하려면 외부에서 스타일을 전달받아 병합해야 한다.

### style prop 받기

`InstructionText` 컴포넌트가 `children`뿐 아니라 `style` prop도 받도록 수정한다.

```jsx
function InstructionText({ children, style }) {
  return (
    <Text style={[styles.instructionText, style]}>
      {children}
    </Text>
  );
}
```

`style`이라는 이름은 필수는 아니지만, React Native의 일반적인 스타일 prop 이름과 맞추면 사용하기 자연스럽다.

```text
children
-> 화면에 표시할 문구

style
-> 사용하는 쪽에서 추가로 전달하는 스타일
```

### 스타일 배열로 병합하기

React Native의 `style` prop은 객체 하나만 받는 것이 아니라 배열도 받을 수 있다.

```jsx
style={[styles.instructionText, style]}
```

배열 안의 스타일은 왼쪽에서 오른쪽 순서로 적용된다.

```text
1. styles.instructionText 적용
2. 외부에서 받은 style 적용
```

같은 속성이 있다면 뒤쪽 스타일이 앞쪽 스타일을 덮어쓴다.

```jsx
style={[
  { fontSize: 24, color: 'yellow' },
  { fontSize: 30 },
]}
```

최종 결과는 다음과 같다.

```text
fontSize: 30
color: yellow
```

즉 외부에서 전달한 스타일이 기본 스타일을 확장하거나 덮어쓸 수 있다.

### 외부 스타일을 마지막에 두는 이유

이번 강의에서는 외부에서 전달받은 `style`을 배열의 마지막에 둔다.

```jsx
style={[styles.instructionText, style]}
```

이렇게 하면 사용하는 쪽에서 기본 스타일을 덮어쓸 수 있다.

```text
기본 스타일
-> color: Colors.accent500
-> fontSize: 24

외부 스타일
-> marginBottom: 12
-> 필요하면 fontSize도 덮어쓸 수 있음
```

만약 반대로 작성하면 기본 스타일이 외부 스타일을 덮어쓴다.

```jsx
style={[style, styles.instructionText]}
```

어떤 순서가 맞는지는 사용 의도에 따라 다르다.

```text
외부에서 커스터마이징 허용
-> [기본 스타일, 외부 스타일]

기본 스타일을 강하게 유지
-> [외부 스타일, 기본 스타일]
```

이번 경우에는 화면별 추가 스타일을 허용하는 편이 자연스럽다.

### GameScreen에서 추가 스타일 전달하기

`GameScreen`에서는 `InstructionText`에 `style` prop을 넘긴다.

```jsx
<InstructionText style={styles.instructionText}>
  Higher or lower?
</InstructionText>
```

그리고 `GameScreen`의 `StyleSheet`에 화면별 스타일을 추가한다.

```jsx
const styles = StyleSheet.create({
  instructionText: {
    marginBottom: 12,
  },
});
```

이 스타일은 기본 안내 문구 스타일을 덮어쓰는 것이 아니라 아래쪽 여백만 추가한다.

```text
기본 InstructionText 스타일
-> color
-> fontSize

GameScreen 추가 스타일
-> marginBottom
```

최종적으로 안내 문구와 버튼 행 사이에 여유가 생긴다.

### CSS cascade와 비슷하지만 직접 구성해야 한다

React Native에는 CSS의 cascade나 상속이 없다.

하지만 스타일 배열을 사용하면 유사한 효과를 직접 만들 수 있다.

```text
CSS
-> 여러 규칙이 cascade로 합쳐짐
-> 우선순위와 선언 순서에 따라 결정

React Native
-> 스타일 배열을 직접 전달
-> 배열 순서에 따라 병합
```

이 방식은 재사용 컴포넌트를 만들 때 매우 자주 사용한다.

```jsx
function CustomText({ children, style }) {
  return (
    <Text style={[styles.defaultText, style]}>
      {children}
    </Text>
  );
}
```

컴포넌트는 기본 스타일을 제공하고, 사용하는 쪽은 필요한 만큼만 조정할 수 있다.

### 전체 GameScreen 구조 예시

이번 단계의 게임 화면 피드백 영역은 다음처럼 정리할 수 있다.

```jsx
<Card>
  <InstructionText style={styles.instructionText}>
    Higher or lower?
  </InstructionText>

  <View style={styles.buttonsContainer}>
    <View style={styles.buttonContainer}>
      <PrimaryButton
        onPress={nextGuessHandler.bind(this, 'greater')}
      >
        +
      </PrimaryButton>
    </View>

    <View style={styles.buttonContainer}>
      <PrimaryButton
        onPress={nextGuessHandler.bind(this, 'lower')}
      >
        -
      </PrimaryButton>
    </View>
  </View>
</Card>
```

스타일은 다음처럼 정리된다.

```jsx
const styles = StyleSheet.create({
  buttonsContainer: {
    flexDirection: 'row',
  },
  buttonContainer: {
    flex: 1,
  },
  instructionText: {
    marginBottom: 12,
  },
});
```

### 프론트엔드 개발자 관점에서의 핵심 차이

웹 React에서는 `className` 조합이나 CSS cascade로 비슷한 문제를 해결한다.

```jsx
<p className="instruction instruction--spaced">
  Higher or lower?
</p>
```

또는 styled-components라면 props로 변형을 줄 수도 있다.

```jsx
<InstructionText $spaced>
  Higher or lower?
</InstructionText>
```

React Native에서는 `style` prop과 배열 조합이 가장 기본적인 방식이다.

```jsx
<InstructionText style={styles.instructionText}>
  Higher or lower?
</InstructionText>
```

```jsx
<Text style={[styles.defaultText, style]}>
  {children}
</Text>
```

개념적으로는 “기본 스타일 + 사용처별 스타일”을 합치는 것이다.

```text
웹
-> class 조합
-> CSS cascade
-> styled component variant

React Native
-> style prop
-> style array
-> 배열 순서로 우선순위 결정
```

React Native에서 재사용 컴포넌트를 만들 때 이 패턴을 익혀두면 스타일 확장이 훨씬 쉬워진다.

### 언제 style prop을 열어둘까

모든 컴포넌트에 무조건 `style` prop을 열어둘 필요는 없다.

하지만 다음 상황에서는 유용하다.

```text
기본 스타일은 유지하고 사용처별 여백만 다르게 주고 싶을 때
같은 텍스트 컴포넌트를 여러 화면에서 조금씩 다르게 쓰고 싶을 때
공통 UI 컴포넌트가 화면 레이아웃 안에서 자연스럽게 적응해야 할 때
```

이번 `InstructionText`는 이런 조건에 잘 맞는다.

```text
공통 역할
-> 안내 문구 표시

화면별 차이
-> 어떤 화면에서는 아래 여백 필요
```

따라서 `style` prop을 받아 기본 스타일과 병합하는 구조가 적절하다.

## 정리

이번 강의에서는 게임 화면의 `+`, `-` 버튼을 가로로 나란히 배치했다. 버튼들을 감싸는 `buttonsContainer`에는 `flexDirection: 'row'`를 적용하고, 각 버튼을 감싸는 `buttonContainer`에는 `flex: 1`을 적용해 두 버튼 영역이 균등하게 나뉘도록 했다.

또한 `InstructionText` 컴포넌트에 외부 `style` prop을 받을 수 있게 만들어, 기본 안내 문구 스타일에 화면별 스타일을 추가하는 방법을 배웠다. React Native의 `style` prop은 배열을 받을 수 있고, 배열의 뒤쪽 스타일이 앞쪽 스타일을 덮어쓸 수 있다.

이 패턴은 React Native에서 CSS의 cascade나 상속이 없는 대신, 기본 스타일과 추가 스타일을 직접 병합하는 핵심적인 방식이다.
