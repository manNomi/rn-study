# 5. React DevTools 사용하기

## 핵심 요약

- React Native 앱에서도 독립 실행형 React DevTools를 사용해 컴포넌트 트리를 확인할 수 있다.
- 브라우저 확장 형태가 아니라 `react-devtools` 패키지를 설치해 별도 앱처럼 실행한다.
- React DevTools에서는 컴포넌트 구조, props, state를 확인할 수 있다.
- 실행 중인 앱의 state 값을 직접 바꿔 보며 UI 변화를 테스트할 수 있다.
- 원격 디버깅과 함께 사용하면 유용하지만, 성능에 영향을 줄 수 있으므로 필요할 때만 켜는 것이 좋다.

## 학습 목표

- React Native에서 React DevTools를 설치하고 실행하는 방법을 익힌다.
- React DevTools로 컴포넌트 트리를 탐색하는 흐름을 이해한다.
- 선택한 컴포넌트의 props와 state를 확인하는 방법을 배운다.
- DevTools에서 state를 직접 수정해 앱 반응을 테스트할 수 있음을 이해한다.
- 원격 디버깅과 React DevTools 사용 시 주의할 점을 정리한다.

## 주요 내용

### React DevTools를 사용하는 이유

이전 강의에서는 원격 JavaScript 디버깅을 통해 Chrome DevTools에서 `Console`과 `Network`를 확인했다.

하지만 React 앱의 구조를 이해하려면 React DevTools가 더 유용할 때가 많다. React DevTools는 앱을 React 컴포넌트 관점에서 보여주기 때문이다.

확인할 수 있는 대표적인 정보는 다음과 같다.

```text
컴포넌트 트리
선택한 컴포넌트의 props
선택한 컴포넌트의 state
컴포넌트 내부 값 변경
```

복잡한 앱일수록 어떤 컴포넌트가 어떤 props를 받고, 어떤 state를 가지고 있는지 확인하는 일이 중요하다.

### React DevTools 설치

웹 React에서는 브라우저 확장 프로그램으로 React DevTools를 설치하는 경우가 많다. React Native에서는 독립 실행형 도구를 설치해 사용한다.

전역 설치 명령어는 다음과 같다.

```bash
npm install -g react-devtools
```

macOS에서 권한 문제가 발생하면 `sudo`를 앞에 붙일 수 있다.

```bash
sudo npm install -g react-devtools
```

Windows에서는 `sudo`가 없고 필요하지도 않으므로 사용하지 않는다.

전역 설치 대신 프로젝트 의존성으로 설치할 수도 있지만, 강의에서는 전역 설치를 사용한다.

### React DevTools 실행

설치가 끝나면 별도의 터미널에서 다음 명령어를 실행한다.

```bash
react-devtools
```

그러면 독립 실행형 React DevTools 창이 열린다.

이때 기존의 `npm start` 프로세스는 계속 실행 중이어야 한다. Expo 개발 서버가 앱을 실행하고 있어야 DevTools가 연결될 수 있다.

### 앱과 DevTools 연결하기

React DevTools를 실행한 뒤, 시뮬레이터나 에뮬레이터에서 개발자 메뉴를 연다.

```text
iOS Simulator: Command + D
Android Emulator: Ctrl + M
터미널: m
```

개발자 메뉴에서 `Debug Remote JS`를 실행하면 React DevTools가 앱과 연결된다.

환경에 따라 연결 과정에서 앱의 배경색이 사라지는 등 일시적인 이상 동작이 보일 수 있다. 강의에서는 `app.json`에 설정한 배경색이 원격 디버깅을 켰을 때 사라지는 현상을 확인했다.

중요한 것은 앱의 시각적 표시가 잠시 달라질 수 있어도, React DevTools 자체는 컴포넌트 분석에 사용할 수 있다는 점이다.

### 컴포넌트 트리 확인

React DevTools가 연결되면 앱의 컴포넌트 트리를 볼 수 있다.

예를 들어 다음과 같은 구조를 확인할 수 있다.

```text
App
View
Button
GoalInput
Modal
TextInput
GoalItem
```

직접 만든 컴포넌트뿐 아니라 React Native 내장 컴포넌트와 그 내부 구성 요소까지 확인할 수 있다.

이 트리는 앱이 실제로 어떤 컴포넌트 구조로 렌더링되고 있는지 파악하는 데 도움이 된다.

### props와 state 확인

특정 컴포넌트를 클릭하면 해당 컴포넌트가 받은 props와 가지고 있는 state를 확인할 수 있다.

예를 들어 `GoalInput` 컴포넌트를 선택하면 다음과 같은 정보를 확인할 수 있다.

```text
visible prop
onAddGoal prop
onCancel prop
enteredGoalText state
```

사용자가 입력창에 텍스트를 입력하면 DevTools에 표시된 state 값도 함께 업데이트된다.

이 기능은 "값이 제대로 전달되고 있는지", "상태가 원하는 시점에 바뀌고 있는지"를 확인할 때 매우 유용하다.

### state 직접 변경하기

React DevTools에서는 실행 중인 앱의 state 값을 직접 수정해 볼 수도 있다.

예를 들어 입력 상태를 직접 다른 문자열로 바꾸면 앱 화면에도 즉시 반영된다.

```text
enteredGoalText: "Learn React Native"
-> DevTools에서 값 수정
-> 앱의 TextInput 값도 변경됨
```

이 기능을 사용하면 사용자 입력이나 특정 상황을 직접 재현하지 않고도, 상태 변화에 따른 UI 반응을 빠르게 테스트할 수 있다.

### 언제 유용한가

React DevTools는 특히 다음 상황에서 유용하다.

```text
컴포넌트 구조가 복잡해졌을 때
props가 제대로 전달되는지 확인하고 싶을 때
state 값이 예상대로 바뀌는지 보고 싶을 때
특정 컴포넌트가 렌더링 트리에 포함되어 있는지 확인하고 싶을 때
상태를 직접 바꿔 UI 반응을 테스트하고 싶을 때
```

단순한 로그 확인은 `console.log()`로 충분할 수 있지만, 앱 전체 구조와 데이터 흐름을 이해하려면 React DevTools가 더 강력하다.

### 사용 후 정리

React DevTools와 원격 디버깅은 유용하지만 항상 켜둘 필요는 없다.

원격 디버깅은 성능에 영향을 줄 수 있고, 경우에 따라 앱의 동작이나 표시가 평소와 다르게 보일 수 있다.

디버깅이 끝났다면 다음을 정리한다.

```text
React DevTools 종료
원격 디버깅 비활성화
불필요한 console.log 제거
앱 새로고침
```

필요한 순간에만 켜고, 작업이 끝나면 일반 실행 상태로 돌아가는 것이 좋다.

## 정리

이번 강의에서는 React Native 앱에서 독립 실행형 React DevTools를 사용하는 방법을 배웠다. `react-devtools`를 설치하고 실행한 뒤, 개발자 메뉴에서 원격 디버깅을 켜면 앱의 컴포넌트 트리를 볼 수 있다.

React DevTools는 props와 state를 확인하고, state 값을 직접 바꿔 UI 반응을 테스트할 수 있어 복잡한 앱을 디버깅할 때 매우 유용하다. 다만 성능이나 실행 상태에 영향을 줄 수 있으므로 필요할 때만 사용하는 것이 좋다.
