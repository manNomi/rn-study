# 4. 개발자 메뉴와 원격 디버깅

## 핵심 요약

- Expo 개발 서버에서는 터미널 단축키를 통해 앱 실행, 새로고침, 개발자 메뉴 열기 등을 수행할 수 있다.
- `?`를 누르면 현재 사용할 수 있는 단축키 목록을 확인할 수 있다.
- 개발자 메뉴는 터미널의 `m`, iOS의 `Command + D`, Android의 `Ctrl + M`으로 열 수 있다.
- 개발자 메뉴에서 앱 새로고침, 홈으로 이동, JavaScript 원격 디버깅 같은 기능을 사용할 수 있다.
- 원격 디버깅을 켜면 Chrome DevTools에서 `Console` 로그와 `Network` 요청을 확인할 수 있다.

## 학습 목표

- Expo 개발 서버의 주요 단축키를 이해한다.
- React Native 앱에서 개발자 메뉴를 여는 방법을 익힌다.
- 개발자 메뉴에서 사용할 수 있는 기본 디버깅 기능을 파악한다.
- Chrome DevTools를 통해 앱의 로그와 네트워크 요청을 확인하는 흐름을 배운다.
- 원격 디버깅은 선택적으로 사용하는 도구이며, 항상 켜둘 필요는 없다는 점을 이해한다.

## 주요 내용

### Expo 개발 서버 단축키 확인

`npm start`로 Expo 개발 서버를 실행한 터미널에서 `?`를 누르면 사용할 수 있는 단축키 목록을 확인할 수 있다.

자주 사용하는 단축키는 다음과 같다.

| 단축키 | 역할 |
| --- | --- |
| `a` | Android Emulator에서 앱 열기 |
| `i` | iOS Simulator에서 앱 열기 |
| `r` | 앱 새로고침 |
| `m` | 개발자 메뉴 토글 |
| `?` | 사용 가능한 단축키 목록 표시 |

이 단축키들은 터미널이 활성화된 상태에서 입력해야 한다.

### 개발자 메뉴 열기

React Native와 Expo는 앱을 개발할 때 사용할 수 있는 개발자 메뉴를 제공한다.

개발자 메뉴는 여러 방법으로 열 수 있다.

```text
터미널: m
iOS Simulator: Command + D
Android Emulator: Ctrl + M
```

터미널에서 `m`을 누르면 실행 중인 에뮬레이터나 시뮬레이터 중 하나에서 개발자 메뉴가 열린다.

### 개발자 메뉴에서 할 수 있는 일

개발자 메뉴에서는 앱을 디버깅하고 개발 흐름을 제어하기 위한 기능을 사용할 수 있다.

대표적인 기능은 다음과 같다.

- 앱 새로고침
- 앱의 홈 화면으로 이동
- JavaScript 원격 디버깅 시작
- 기타 React Native / Expo 개발 도구 기능 실행

이 메뉴는 앱 내부에서 직접 개발 도구 기능을 실행할 수 있게 해주는 진입점이라고 볼 수 있다.

### JavaScript 원격 디버깅

개발자 메뉴에서 JavaScript 원격 디버깅을 선택하면 Chrome에 새 탭이 열린다.

이 탭에서 Chrome DevTools를 열면 웹 프론트엔드 개발에서 익숙한 방식으로 앱을 살펴볼 수 있다.

특히 다음 패널이 유용하다.

```text
Console
-> 앱에서 실행한 console.log 출력 확인

Network
-> 앱에서 발생한 네트워크 요청 확인
```

현재 목표 관리 앱은 백엔드에 요청을 보내지 않기 때문에 의미 있는 네트워크 요청은 많지 않다. 하지만 나중에 HTTP 요청을 보내는 앱에서는 `Network` 패널에서 요청과 응답을 확인할 수 있다.

### Console 로그 확인

원격 디버깅이 켜진 상태에서 앱 코드에 로그를 추가할 수 있다.

```jsx
console.log('GoalInput');
```

저장 후 앱이 새로고침되면 Chrome DevTools의 `Console` 패널에서 출력값을 확인할 수 있다.

만약 로그가 바로 보이지 않는다면 다음을 시도할 수 있다.

```text
앱 새로고침
시뮬레이터 앱 재시작
npm start 프로세스 재시작
```

원격 디버깅 연결은 환경에 따라 약간 불안정할 수 있으므로, 필요하면 새로고침이나 재시작을 통해 다시 연결한다.

### Network 패널의 의미

Chrome DevTools의 `Network` 패널에서는 앱에서 발생하는 네트워크 요청을 볼 수 있다.

지금 앱처럼 네트워크 요청이 없는 경우에는 개발과 디버깅에 필요한 자산이나 내부 요청 정도만 보일 수 있다.

하지만 나중에 백엔드 API와 연결하는 앱에서는 다음 정보를 확인할 수 있다.

```text
어떤 URL로 요청을 보냈는지
요청이 성공했는지 실패했는지
응답 상태 코드가 무엇인지
응답 데이터가 어떤 형태인지
요청이 얼마나 오래 걸렸는지
```

웹 프론트엔드에서 API 요청을 디버깅할 때 Network 패널을 보는 것과 비슷한 감각으로 활용할 수 있다.

### 원격 디버깅은 선택 도구

원격 디버깅은 앱을 더 깊게 살펴보고 싶을 때 유용한 도구다.

하지만 항상 켜두어야 하는 것은 아니다. 단순한 로그 확인이나 에러 메시지 확인만으로 충분한 경우도 많다.

필요할 때 켜고, 사용이 끝나면 개발자 메뉴에서 원격 디버깅을 끄고 일반 실행 상태로 돌아갈 수 있다.

## 정리

이번 강의에서는 Expo 개발 서버의 단축키와 React Native 개발자 메뉴를 살펴봤다. `?`로 단축키 목록을 확인하고, `m`, `Command + D`, `Ctrl + M`으로 개발자 메뉴를 열 수 있다.

개발자 메뉴의 원격 JavaScript 디버깅을 사용하면 Chrome DevTools에서 `Console` 로그와 `Network` 요청을 확인할 수 있다. 다만 이 기능은 선택적으로 사용하는 추가 도구이며, 필요할 때만 켜서 앱을 더 깊게 분석하면 된다.
