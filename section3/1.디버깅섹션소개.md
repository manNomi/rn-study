# 1. 디버깅 섹션 소개

## 핵심 요약

- 이번 섹션에서는 React Native 앱을 개발하면서 발생하는 오류를 찾고 해결하는 방법을 배운다.
- 앱 개발 중 에러는 자연스럽게 발생하므로, 에러 메시지를 읽고 원인을 추적하는 능력이 필요하다.
- 에러 메시지에는 문제 해결을 위한 힌트가 포함되어 있는 경우가 많다.
- React Native와 Expo가 제공하는 개발자 도구와 디버깅 기능을 살펴본다.
- 실습은 지난 섹션에서 만든 목표 관리 앱을 기반으로 이어간다.

## 학습 목표

- React Native 앱에서 발생하는 에러 메시지를 읽는 법을 익힌다.
- 에러 메시지에서 파일, 위치, 원인, 해결 힌트를 찾는 방법을 배운다.
- 개발자 도구를 사용해 앱 상태와 동작을 점검하는 흐름을 이해한다.
- Expo와 React Native가 제공하는 디버깅 관련 기능을 파악한다.
- 이전 섹션에서 만든 앱을 디버깅 실습 대상으로 활용한다.

## 주요 내용

### 왜 디버깅을 배워야 하는가

앱을 만들다 보면 코드를 잘못 작성하거나, 컴포넌트 사용법을 착각하거나, 상태와 props 흐름이 꼬이면서 에러가 발생한다.

중요한 것은 에러가 발생하지 않게 만드는 것이 아니라, 에러가 발생했을 때 빠르게 원인을 찾고 수정하는 능력이다.

React Native 앱은 웹 React와 비슷한 부분도 많지만, 네이티브 컴포넌트와 모바일 실행 환경을 사용하기 때문에 웹과는 다른 방식의 오류도 만날 수 있다.

### 에러 메시지 읽기

이번 섹션에서는 에러 메시지 예시를 살펴보면서 다음 내용을 확인한다.

```text
어떤 파일에서 문제가 발생했는가
어느 줄 근처에서 문제가 발생했는가
무엇이 잘못되었다고 설명하는가
해결을 위한 힌트가 포함되어 있는가
```

에러 메시지는 처음 보면 복잡해 보일 수 있지만, 대부분은 문제를 좁혀 갈 수 있는 정보를 제공한다. 따라서 메시지를 무시하지 않고 차근차근 읽는 습관이 중요하다.

### 개발자 도구 살펴보기

React Native 앱을 디버깅할 때는 개발자 도구를 함께 사용한다.

이번 섹션에서는 다음과 같은 도구와 기능을 살펴본다.

- React Native 앱에서 확인할 수 있는 에러 화면
- 터미널과 Expo 개발 서버의 로그
- 개발자 도구(DevTools)
- React Native와 Expo가 제공하는 추가 디버깅 기능

웹 프론트엔드에서 브라우저 DevTools를 사용하듯, React Native에서도 앱 실행 환경에 맞는 도구를 사용해 문제를 추적한다.

### 이전 섹션 앱 재사용

이번 섹션에서도 지난 섹션에서 만든 목표 관리 앱을 사용한다.

지난 섹션을 건너뛰었다면 강의에 첨부된 프로젝트 파일을 받아 같은 코드 상태에서 시작해야 한다. 그래야 강의에서 보여주는 에러와 디버깅 흐름을 동일하게 따라갈 수 있다.

실습 대상 앱은 이미 다음 기능을 가지고 있다.

```text
목표 입력
목표 추가
목표 삭제
모달 입력 화면
기본 스타일링
```

이 앱을 기반으로 의도적으로 에러를 만들거나, 실제 개발 중 만날 수 있는 문제를 확인하면서 디버깅 방법을 익힌다.

## 정리

이번 섹션은 React Native 앱을 더 잘 만들기 위한 디버깅 섹션이다. 앱 개발 중 발생하는 에러 메시지를 읽고, 원인을 찾고, 도구를 사용해 문제를 해결하는 흐름을 배운다.

지난 섹션에서 만든 목표 관리 앱을 계속 활용하므로, 이미 만든 앱의 구조를 다시 떠올리면서 디버깅 도구와 오류 해결 과정을 익히는 것이 중요하다.
