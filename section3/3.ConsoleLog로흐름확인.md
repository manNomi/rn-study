# 3. Console Log로 흐름 확인

## 핵심 요약

- 에러가 없어도 앱의 실행 흐름을 이해하기 위해 로그를 남길 수 있다.
- React Native에서도 웹 JavaScript처럼 `console.log()`를 사용할 수 있다.
- `console.log()` 출력은 `npm start`를 실행한 터미널에서 확인할 수 있다.
- 컴포넌트 함수, 이벤트 핸들러, 상태 변경 지점에 로그를 넣으면 어떤 코드가 언제 실행되는지 파악할 수 있다.
- 변수나 상태 값을 함께 출력하면 특정 시점의 데이터 흐름을 확인할 수 있다.

## 학습 목표

- React Native 앱에서 `console.log()`를 사용해 로그를 확인하는 방법을 익힌다.
- 컴포넌트가 언제 렌더링되는지 로그로 확인한다.
- 이벤트 핸들러나 상태 값이 어떻게 변하는지 터미널에서 추적한다.
- 여러 에뮬레이터나 시뮬레이터를 동시에 실행할 때 로그가 여러 번 찍힐 수 있음을 이해한다.
- 에러 메시지 없이도 앱 동작을 분석하는 기본 디버깅 습관을 만든다.

## 주요 내용

### 에러가 없을 때도 디버깅이 필요하다

디버깅은 에러가 발생했을 때만 하는 작업이 아니다. 앱이 정상적으로 실행되더라도, 어떤 컴포넌트가 언제 렌더링되는지, 어떤 함수가 언제 호출되는지, 특정 상태 값이 어떤 시점에 어떤 값을 가지는지 확인하고 싶을 때가 있다.

이럴 때 가장 간단하게 사용할 수 있는 도구가 `console.log()`다.

### 컴포넌트 렌더링 확인하기

예를 들어 `GoalInput` 컴포넌트 안에 로그를 추가할 수 있다.

```jsx
function GoalInput(props) {
  console.log('GoalInput 컴포넌트 렌더링 됨');

  return (
    // JSX 코드
  );
}
```

파일을 저장하면 Expo 개발 서버가 앱을 다시 반영하고, `npm start`를 실행한 터미널에 로그가 출력된다.

```text
GoalInput 컴포넌트 렌더링 됨
```

이 로그를 통해 해당 컴포넌트 함수가 실행되었고, 컴포넌트가 렌더링 흐름에 참여했다는 것을 확인할 수 있다.

### 여러 기기를 실행하면 로그도 여러 번 찍힐 수 있다

Android Emulator와 iOS Simulator를 동시에 실행하고 있다면 같은 코드가 각 실행 환경에서 동작한다.

따라서 같은 로그가 두 번 출력될 수 있다.

```text
GoalInput 컴포넌트 렌더링 됨
GoalInput 컴포넌트 렌더링 됨
```

이것은 이상한 현상이 아니라, 두 개의 앱 인스턴스가 각각 실행 중이기 때문에 발생하는 자연스러운 결과다.

### 코드 실행 위치 확인하기

`console.log()`는 컴포넌트 최상단뿐 아니라 다양한 위치에 넣을 수 있다.

```jsx
function goalInputHandler(enteredText) {
  console.log('입력값 변경됨');
  setEnteredGoalText(enteredText);
}
```

```jsx
function addGoalHandler() {
  console.log('목표 추가 버튼 눌림');
  props.onAddGoal(enteredGoalText);
}
```

이렇게 하면 다음을 확인할 수 있다.

```text
컴포넌트가 렌더링되는 시점
사용자가 입력할 때 실행되는 함수
버튼을 눌렀을 때 실행되는 함수
상태 업데이트 직전에 실행되는 코드
```

즉 앱의 흐름을 머릿속으로 추측하지 않고, 실제 실행 순서를 로그로 확인할 수 있다.

### 변수와 상태 값 출력하기

문자열만 출력할 필요는 없다. 변수나 상태 값을 함께 출력할 수도 있다.

```jsx
console.log(enteredGoalText);
```

또는 메시지와 값을 함께 출력하면 로그를 읽기 더 쉽다.

```jsx
console.log('현재 입력값:', enteredGoalText);
```

사용자가 입력창에 텍스트를 입력하면 터미널에서 값이 어떻게 바뀌는지 확인할 수 있다.

```text
현재 입력값: L
현재 입력값: Le
현재 입력값: Lea
현재 입력값: Learn
```

이 방식은 상태가 원하는 대로 업데이트되는지 확인할 때 특히 유용하다.

### console.log로 확인할 수 있는 것

`console.log()`는 다음을 확인할 때 유용하다.

```text
특정 컴포넌트가 렌더링되는지
특정 함수가 호출되는지
이벤트가 실제로 발생하는지
상태 값이 어떤 시점에 어떤 값을 가지는지
props가 제대로 전달되는지
조건문이 의도한 분기로 들어가는지
```

복잡한 도구를 열기 전에도 `console.log()`만으로 많은 문제를 빠르게 좁힐 수 있다.

### 주의할 점

`console.log()`는 개발 중 흐름을 파악하기에는 좋지만, 너무 많이 남기면 터미널 출력이 지저분해지고 중요한 로그를 놓치기 쉽다.

또한 최종 앱에는 불필요한 로그를 남기지 않는 것이 좋다. 디버깅이 끝나면 사용하지 않는 로그는 정리하는 습관을 들인다.

## 정리

이번 강의에서는 React Native 앱에서도 웹 JavaScript처럼 `console.log()`를 사용해 앱 흐름을 확인할 수 있다는 점을 배웠다.

로그는 터미널에서 확인할 수 있으며, 컴포넌트 렌더링 시점, 이벤트 핸들러 실행 여부, 상태와 변수 값을 파악하는 데 유용하다. 에러 메시지가 없더라도 앱이 어떻게 동작하는지 이해하고 싶을 때 가장 간단하게 사용할 수 있는 디버깅 도구다.
