# 6. 디버깅 섹션 마무리

## 핵심 요약

- 이번 섹션에서는 React Native 앱을 이해하고 문제를 해결하기 위한 핵심 디버깅 도구를 살펴봤다.
- 에러 메시지와 스택트레이스를 읽으면 문제 위치와 원인을 좁힐 수 있다.
- `console.log()`는 앱의 실행 흐름과 상태 값을 빠르게 확인하는 가장 기본적인 방법이다.
- 개발자 메뉴, 원격 디버깅, React DevTools는 앱을 더 깊게 분석할 때 사용할 수 있다.
- React Native 공식 문서의 Debugging 가이드는 추가 학습 자료로 활용하기 좋다.

## 학습 목표

- 섹션에서 다룬 디버깅 도구들을 한 번에 정리한다.
- 상황별로 어떤 디버깅 접근법을 사용할지 판단한다.
- 공식 문서를 추가 참고 자료로 활용하는 흐름을 이해한다.
- React Native 앱 개발에서 디버깅이 반복적으로 필요한 기본 역량이라는 점을 기억한다.

## 주요 내용

### 이번 섹션에서 다룬 도구

이번 섹션에서는 React Native 앱을 디버깅하기 위한 여러 접근법을 살펴봤다.

| 도구 / 접근법 | 용도 |
| --- | --- |
| 에러 메시지 | 어떤 문제가 발생했는지 첫 힌트 확인 |
| 스택트레이스 | 문제가 발생한 컴포넌트 흐름 추적 |
| `console.log()` | 코드 실행 시점과 변수 값 확인 |
| 개발자 메뉴 | 새로고침, 원격 디버깅 등 개발 기능 실행 |
| Chrome DevTools | Console 로그와 Network 요청 확인 |
| React DevTools | 컴포넌트 트리, props, state 확인 |

이 도구들은 서로 대체 관계라기보다 상황에 따라 함께 사용하는 도구들이다.

### 상황별 디버깅 접근

에러가 화면에 표시된다면 먼저 에러 메시지를 읽는다.

```text
에러 메시지
-> 문제 컴포넌트나 prop 확인
-> 스택트레이스로 컴포넌트 흐름 확인
-> 코드 수정
```

에러는 없지만 앱 흐름이 궁금하다면 `console.log()`를 사용한다.

```text
console.log()
-> 함수 실행 여부 확인
-> 상태 값 확인
-> props 전달 여부 확인
```

앱 구조나 상태를 더 깊게 보고 싶다면 React DevTools를 사용한다.

```text
React DevTools
-> 컴포넌트 트리 확인
-> props와 state 확인
-> state 값을 직접 바꿔 UI 반응 테스트
```

네트워크 요청이 있는 앱에서는 Chrome DevTools의 `Network` 패널도 유용하다.

### 공식 문서 참고하기

React Native 공식 문서에는 디버깅 관련 가이드가 따로 정리되어 있다.

참고 문서:

- [React Native Debugging Basics](https://reactnative.dev/docs/debugging.html)

공식 문서에서는 개발자 메뉴, DevTools, LogBox, 성능 모니터 같은 추가 기능과 최신 디버깅 흐름을 확인할 수 있다. React Native와 Expo 생태계는 시간이 지나며 도구와 권장 방식이 바뀔 수 있으므로, 실제 프로젝트에서는 공식 문서를 함께 확인하는 것이 좋다.

### 강의에서 배운 도구만으로도 충분한 출발점

공식 문서에는 더 많은 세부 기능이 있지만, 이번 섹션에서 배운 도구만으로도 React Native 앱을 이해하고 대부분의 기본 문제를 해결하는 데 큰 도움이 된다.

특히 초반에는 다음 흐름에 익숙해지는 것이 중요하다.

```text
에러 메시지를 읽는다.
터미널 로그를 확인한다.
필요하면 console.log를 추가한다.
컴포넌트 구조가 헷갈리면 React DevTools를 연다.
공식 문서에서 컴포넌트나 디버깅 기능을 확인한다.
```

이 흐름이 익숙해지면 앱이 복잡해져도 문제를 더 차분하게 좁혀 갈 수 있다.

## 정리

이번 섹션에서는 React Native 앱을 디버깅하는 핵심 방법을 배웠다. 에러 메시지, 스택트레이스, `console.log()`, 개발자 메뉴, 원격 디버깅, React DevTools를 상황에 맞게 사용하면 앱의 동작을 더 잘 이해하고 문제를 해결할 수 있다.

추가 기능이나 최신 도구 흐름이 필요할 때는 React Native 공식 Debugging 문서를 참고하면 된다. 중요한 것은 도구 자체를 많이 아는 것보다, 문제가 생겼을 때 메시지를 읽고 원인을 좁혀 가는 습관을 갖는 것이다.
