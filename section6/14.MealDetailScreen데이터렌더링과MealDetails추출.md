# 14. MealDetailScreen 데이터 렌더링과 MealDetails 추출

## 핵심 요약

- 이번 강의에서는 `MealDetailScreen`에 실제 음식 상세 데이터를 출력하기 시작한다.
- `route.params.mealId`로 받은 id를 사용해 `MEALS` 배열에서 선택된 음식을 찾는다.
- 상세 화면에는 이미지, 제목, 조리 시간, 난이도, 가격 수준, 재료 목록, 조리 단계 목록을 출력할 계획이다.
- `MealItem`에서 이미 사용하던 duration, complexity, affordability 표시 UI를 `MealDetails`라는 재사용 컴포넌트로 추출한다.
- `MealDetails`는 `MealItem`과 `MealDetailScreen` 양쪽에서 재사용된다.
- 재료와 조리 단계는 길이가 제한된 정적 배열이므로 `FlatList` 대신 `map()`으로 직접 JSX 리스트를 만든다.
- 원격 이미지는 `Image`에 `source={{ uri }}`를 설정해야 하며, 다음 단계에서 `width`와 `height` 스타일을 추가해야 보인다.

## 학습 목표

- route params로 받은 `mealId`를 실제 데이터 조회에 사용하는 방법을 익힌다.
- `Array.prototype.find()`로 특정 id에 해당하는 객체를 찾는 흐름을 복습한다.
- 화면 간 전달된 id를 기준으로 상세 데이터를 렌더링하는 구조를 이해한다.
- 중복 UI를 재사용 컴포넌트로 추출하는 기준을 익힌다.
- `MealDetails` 컴포넌트에 props를 전달해 여러 화면에서 재사용하는 방법을 익힌다.
- 짧고 고정적인 리스트를 `map()`으로 출력하는 방식과 `FlatList`를 쓰는 기준을 비교한다.

## 주요 내용

### 상세 화면에 필요한 데이터

이전 강의에서 `MealItem`을 눌렀을 때 `MealDetailScreen`으로 이동하면서 `mealId`를 전달했다.

```jsx
navigation.navigate('MealDetail', {
  mealId: id,
});
```

`MealDetailScreen`에서는 이 id를 읽을 수 있다.

```jsx
function MealDetailScreen({ route }) {
  const mealId = route.params.mealId;
}
```

이번 강의의 목표는 이 `mealId`를 사용해 실제 음식 데이터를 찾아 상세 화면에 출력하는 것이다.

### MEALS에서 선택된 음식 찾기

먼저 더미 데이터에서 `MEALS`를 import한다.

```jsx
import { MEALS } from '../data/dummy-data';
```

그 다음 `find()`를 사용해 `mealId`와 일치하는 음식을 찾는다.

```jsx
const selectedMeal = MEALS.find((meal) => {
  return meal.id === mealId;
});
```

`find()`는 배열을 순회하며 콜백이 `true`를 반환하는 첫 번째 항목을 반환한다.

```text
meal.id === mealId
-> 선택한 음식과 일치
-> selectedMeal로 반환
```

이제 `selectedMeal`을 통해 상세 화면에 필요한 모든 속성에 접근할 수 있다.

```text
selectedMeal.imageUrl
selectedMeal.title
selectedMeal.duration
selectedMeal.complexity
selectedMeal.affordability
selectedMeal.ingredients
selectedMeal.steps
```

### 상세 화면의 기본 구조

상세 화면에는 다음 정보를 배치할 계획이다.

```text
MealDetailScreen
-> 음식 이미지
-> 음식 제목
-> 조리 시간 / 난이도 / 가격 수준
-> Ingredients 제목
-> 재료 목록
-> Steps 제목
-> 조리 단계 목록
```

처음에는 전체를 감싸는 `View`를 만들고, 그 안에 필요한 컴포넌트를 순서대로 배치한다.

```jsx
<View>
  <Image />
  <Text>{selectedMeal.title}</Text>
  <MealDetails />
  <Text>Ingredients</Text>
  {/* ingredients list */}
  <Text>Steps</Text>
  {/* steps list */}
</View>
```

스타일링은 다음 강의에서 본격적으로 다룬다.

### 원격 이미지 출력

음식 이미지는 더미 데이터에 URL로 저장되어 있다.

React Native에서 원격 이미지를 표시하려면 `source`에 `{ uri }` 객체를 넣는다.

```jsx
<Image
  source={{ uri: selectedMeal.imageUrl }}
/>
```

`Image`를 사용하려면 `react-native`에서 import해야 한다.

```jsx
import { Image, Text, View } from 'react-native';
```

주의할 점은 원격 이미지는 `width`와 `height`를 명시해야 보인다는 것이다.

이번 강의에서는 이미지 source만 연결하고, 실제 이미지 크기 스타일은 다음 강의에서 추가한다.

```text
원격 이미지
-> source={{ uri: ... }}
-> width/height 스타일 필요
```

### MealDetails 컴포넌트 추출

`MealItem`에는 이미 조리 시간, 난이도, 가격 수준을 출력하는 UI가 있었다.

```jsx
<View style={styles.details}>
  <Text style={styles.detailItem}>{duration}m</Text>
  <Text style={styles.detailItem}>{complexity.toUpperCase()}</Text>
  <Text style={styles.detailItem}>{affordability.toUpperCase()}</Text>
</View>
```

이 UI는 상세 화면에서도 필요하다.

따라서 중복해서 복사하지 않고 `MealDetails`라는 재사용 컴포넌트로 추출한다.

```text
components/
  MealDetails.js
```

`MealDetails`는 `duration`, `complexity`, `affordability`를 props로 받는다.

```jsx
function MealDetails({ duration, complexity, affordability }) {
  return (
    <View style={styles.details}>
      <Text style={styles.detailItem}>{duration}m</Text>
      <Text style={styles.detailItem}>{complexity.toUpperCase()}</Text>
      <Text style={styles.detailItem}>{affordability.toUpperCase()}</Text>
    </View>
  );
}
```

### MealDetails 스타일 이동

기존 `MealItem`에 있던 `details`, `detailItem` 스타일도 `MealDetails`로 옮긴다.

```jsx
const styles = StyleSheet.create({
  details: {
    flexDirection: 'row',
    alignItems: 'center',
    justifyContent: 'center',
    padding: 8,
  },
  detailItem: {
    marginHorizontal: 4,
    fontSize: 12,
  },
});
```

이제 `MealItem`은 직접 `details` UI를 가지고 있지 않고, `MealDetails`를 사용한다.

```jsx
<MealDetails
  duration={duration}
  complexity={complexity}
  affordability={affordability}
/>
```

이 방식의 장점은 같은 표시 UI를 여러 화면에서 일관되게 사용할 수 있다는 점이다.

```text
재사용 전
-> MealItem과 MealDetailScreen에 같은 View/Text 복사

재사용 후
-> MealDetails 한 곳에서 관리
-> 스타일과 출력 형식 일관성 유지
```

### MealItem에서 MealDetails 사용

`MealItem.js`에서 기존 상세 정보 `View`를 제거하고 `MealDetails`를 사용한다.

```jsx
import MealDetails from './MealDetails';
```

```jsx
<MealDetails
  duration={duration}
  complexity={complexity}
  affordability={affordability}
/>
```

이제 `MealItem`은 이미지, 제목, `MealDetails` 조합으로 구성된다.

### MealDetailScreen에서 MealDetails 사용

`MealDetailScreen`에서도 같은 컴포넌트를 사용한다.

```jsx
import MealDetails from '../components/MealDetails';
```

```jsx
<MealDetails
  duration={selectedMeal.duration}
  complexity={selectedMeal.complexity}
  affordability={selectedMeal.affordability}
/>
```

이렇게 하면 목록 카드와 상세 화면에서 같은 메타 정보 표시 형식을 공유한다.

### 재료 목록 출력

음식 데이터의 `ingredients`는 문자열 배열이다.

```text
selectedMeal.ingredients
-> ['Tomatoes', 'Olive Oil', ...]
```

이 목록은 보통 아주 길지 않다.

따라서 `FlatList` 대신 `map()`으로 직접 출력한다.

```jsx
{selectedMeal.ingredients.map((ingredient) => (
  <Text key={ingredient}>{ingredient}</Text>
))}
```

수동으로 리스트를 만들 때는 React의 기본 규칙대로 `key` prop을 넣어야 한다.

강의에서는 각 재료 문자열이 한 음식 안에서 고유하다고 보고 `ingredient` 자체를 key로 사용한다.

### 조리 단계 목록 출력

`steps`도 문자열 배열이다.

```text
selectedMeal.steps
-> ['Cut the tomatoes...', 'Boil water...', ...]
```

재료 목록과 같은 방식으로 `map()`을 사용한다.

```jsx
{selectedMeal.steps.map((step) => (
  <Text key={step}>{step}</Text>
))}
```

`FlatList`를 쓰지 않는 이유는 목록 길이가 제한적이기 때문이다.

```text
FlatList가 필요한 경우
-> 길거나 동적인 목록
-> 성능 최적화가 중요한 목록
-> 화면에 보이지 않는 아이템이 많을 수 있는 목록

map으로 충분한 경우
-> 짧고 제한된 목록
-> 재료, 조리 단계처럼 보통 길지 않은 데이터
```

물론 아주 복잡한 요리라면 길어질 수도 있지만, 일반적인 음식 상세 화면에서는 `map()`으로도 충분하다.

### 현재 화면 상태

이 단계까지 적용하면 텍스트 데이터는 대부분 화면에 나타난다.

하지만 이미지는 보이지 않을 수 있다.

이유는 원격 이미지에 아직 크기 스타일을 지정하지 않았기 때문이다.

```text
이미지가 안 보이는 이유
-> source는 설정됨
-> 하지만 width/height 없음
-> React Native가 원격 이미지 크기를 추론하지 못함
```

다음 강의에서는 상세 화면의 스타일을 다듬고, 이미지 크기도 설정한다.

## 예시 코드

### MealDetails

```jsx
import { StyleSheet, Text, View } from 'react-native';

function MealDetails({ duration, complexity, affordability }) {
  return (
    <View style={styles.details}>
      <Text style={styles.detailItem}>{duration}m</Text>
      <Text style={styles.detailItem}>{complexity.toUpperCase()}</Text>
      <Text style={styles.detailItem}>{affordability.toUpperCase()}</Text>
    </View>
  );
}

export default MealDetails;

const styles = StyleSheet.create({
  details: {
    flexDirection: 'row',
    alignItems: 'center',
    justifyContent: 'center',
    padding: 8,
  },
  detailItem: {
    marginHorizontal: 4,
    fontSize: 12,
  },
});
```

### MealDetailScreen

```jsx
import { Image, Text, View } from 'react-native';

import MealDetails from '../components/MealDetails';
import { MEALS } from '../data/dummy-data';

function MealDetailScreen({ route }) {
  const mealId = route.params.mealId;
  const selectedMeal = MEALS.find((meal) => meal.id === mealId);

  return (
    <View>
      <Image source={{ uri: selectedMeal.imageUrl }} />
      <Text>{selectedMeal.title}</Text>

      <MealDetails
        duration={selectedMeal.duration}
        complexity={selectedMeal.complexity}
        affordability={selectedMeal.affordability}
      />

      <Text>Ingredients</Text>
      {selectedMeal.ingredients.map((ingredient) => (
        <Text key={ingredient}>{ingredient}</Text>
      ))}

      <Text>Steps</Text>
      {selectedMeal.steps.map((step) => (
        <Text key={step}>{step}</Text>
      ))}
    </View>
  );
}

export default MealDetailScreen;
```

## 프론트엔드 개발자 관점에서의 핵심 차이

### 상세 화면은 id 기반 조회의 전형적인 패턴

웹에서도 상세 페이지는 보통 id를 기준으로 데이터를 찾는다.

```text
/meals/m1
-> mealId = m1
-> MEALS에서 id가 m1인 데이터 조회
```

React Native에서는 URL 대신 route params를 사용한다.

```jsx
const mealId = route.params.mealId;
const selectedMeal = MEALS.find((meal) => meal.id === mealId);
```

패턴 자체는 웹과 매우 비슷하다.

### 재사용 컴포넌트 추출 기준

`duration`, `complexity`, `affordability` 표시 UI는 목록 카드와 상세 화면 양쪽에서 필요하다.

이런 경우 컴포넌트로 추출하기 좋은 신호다.

```text
추출하기 좋은 경우
-> 동일한 UI가 두 곳 이상에서 반복됨
-> 스타일과 출력 형식이 같아야 함
-> 나중에 변경 시 한 곳만 수정하고 싶음
```

`MealDetails`는 작지만 좋은 재사용 컴포넌트 예시다.

### FlatList를 항상 써야 하는 것은 아니다

React Native에서 긴 목록은 `FlatList`가 좋다.

하지만 재료와 조리 단계처럼 짧고 제한적인 배열은 `map()`으로도 충분하다.

```text
FlatList
-> 긴 목록
-> 동적 대량 데이터
-> 성능 최적화 필요

map
-> 짧은 목록
-> 제한된 데이터
-> 단순한 상세 페이지 섹션
```

도구를 상황에 맞게 고르는 감각이 중요하다.

### 원격 이미지의 크기 문제는 계속 중요하다

React Native의 `Image`는 웹의 `img`처럼 URL만 넣는다고 항상 자연스럽게 크기가 잡히지 않는다.

특히 원격 이미지는 반드시 스타일로 크기를 지정해야 한다.

이번 강의에서는 이미지가 아직 안 보일 수 있고, 다음 강의에서 스타일을 추가하며 해결한다.

## 정리

이번 강의에서는 `MealDetailScreen`에서 `mealId`를 사용해 선택한 음식 데이터를 찾고, 이미지, 제목, 메타 정보, 재료, 조리 단계를 출력하는 기본 구조를 만들었다.

또한 `MealItem`과 상세 화면에서 공통으로 쓰이는 조리 시간, 난이도, 가격 수준 표시 UI를 `MealDetails` 컴포넌트로 추출했다.

아직 스타일링은 완성되지 않았지만, 상세 화면에 필요한 데이터 렌더링 구조는 준비되었다.
