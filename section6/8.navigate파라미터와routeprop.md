# 8. navigate 파라미터와 route prop

## 핵심 요약

- 현재 앱은 카테고리 화면에서 음식 목록 화면으로 이동할 수 있지만, 아직 선택한 카테고리 정보는 전달하지 않는다.
- `navigation.navigate()`는 첫 번째 인자로 이동할 화면 이름을 받고, 두 번째 인자로 대상 화면에 전달할 params 객체를 받을 수 있다.
- 카테고리 타일을 누를 때 `categoryId`를 params로 전달하면, `MealsOverviewScreen`에서 어떤 카테고리가 선택되었는지 알 수 있다.
- React Navigation은 `Stack.Screen`에 등록된 화면 컴포넌트에 `navigation` prop뿐 아니라 `route` prop도 자동으로 전달한다.
- 전달된 params는 대상 화면의 `route.params`에서 읽을 수 있다.
- screen 컴포넌트가 아닌 중첩 컴포넌트에서 route 정보가 필요하면 `useRoute` 훅을 사용할 수 있다.

## 학습 목표

- `navigation.navigate()`로 화면 이동과 데이터 전달을 동시에 처리하는 방법을 익힌다.
- 두 번째 인자인 params 객체의 역할을 이해한다.
- `FlatList`의 `itemData.item.id`를 이용해 선택한 카테고리 id를 가져오는 흐름을 파악한다.
- 대상 화면에서 `route.params`를 통해 전달된 값을 읽는 방법을 익힌다.
- `navigation` prop과 `route` prop이 screen 컴포넌트에 자동 제공된다는 점을 이해한다.
- `useRoute` 훅이 언제 필요한지 파악한다.

## 주요 내용

### 현재 문제

이전 강의까지 앱은 카테고리 화면에서 음식 목록 화면으로 이동할 수 있다.

```text
CategoriesScreen
-> MealsOverviewScreen
```

하지만 아직 어떤 카테고리를 눌렀는지 `MealsOverviewScreen`은 알지 못한다.

모든 카테고리가 같은 `MealsOverviewScreen`으로 이동하기 때문이다.

```text
Italian 클릭
-> MealsOverviewScreen

German 클릭
-> MealsOverviewScreen

Asian 클릭
-> MealsOverviewScreen
```

이 상태에서는 대상 화면에서 어떤 음식 목록을 보여줘야 할지 알 수 없다.

따라서 선택한 카테고리 정보를 화면 이동과 함께 전달해야 한다.

### 왜 데이터를 전달해야 하는가?

`MealsOverviewScreen`에서는 선택한 카테고리에 맞는 음식만 보여줘야 한다.

더미 데이터의 `MEALS` 배열에는 각 음식이 어떤 카테고리에 속하는지 나타내는 정보가 있다.

```text
Meal
-> categoryIds: ['c1', 'c2']
```

즉 `MealsOverviewScreen`이 선택한 카테고리 id를 알면 해당 카테고리에 속한 음식만 필터링할 수 있다.

```jsx
const displayedMeals = MEALS.filter((meal) => {
  return meal.categoryIds.includes(categoryId);
});
```

이번 강의에서는 우선 `categoryId`를 전달하고 읽는 흐름을 만든다.

실제 필터링과 목록 출력은 다음 단계에서 확장할 수 있다.

### navigate의 두 번째 인자

`navigation.navigate()`는 첫 번째 인자로 이동할 화면 이름을 받는다.

```jsx
navigation.navigate('MealsOverview');
```

여기에 두 번째 인자를 추가할 수 있다.

두 번째 인자는 대상 화면에 전달할 params 객체다.

```jsx
navigation.navigate('MealsOverview', {
  categoryId: 'c1',
});
```

즉 `navigate`는 다음 구조를 가진다.

```text
navigation.navigate(screenName, params?)
```

```text
screenName
-> 이동할 화면의 name
-> Stack.Screen에 등록된 이름

params
-> 대상 화면으로 전달할 데이터 객체
-> 선택 사항
```

### 선택한 카테고리 id 전달

`CategoriesScreen`의 `renderCategoryItem` 함수는 `FlatList`가 제공하는 `itemData`를 받는다.

```jsx
function renderCategoryItem(itemData) {
  ...
}
```

실제 카테고리 객체는 `itemData.item`에 들어 있다.

```text
itemData.item
-> 현재 렌더링 중인 카테고리 객체

itemData.item.id
-> 현재 카테고리의 고유 id
```

따라서 카테고리 타일을 눌렀을 때 해당 id를 params로 전달할 수 있다.

```jsx
function pressHandler() {
  navigation.navigate('MealsOverview', {
    categoryId: itemData.item.id,
  });
}
```

여기서 `categoryId`라는 이름은 개발자가 정하는 식별자다.

대상 화면에서도 같은 이름으로 값을 읽어야 한다.

```text
보내는 쪽
-> categoryId: itemData.item.id

받는 쪽
-> route.params.categoryId
```

### route prop

React Navigation은 screen으로 등록된 컴포넌트에 `navigation` prop을 자동으로 전달한다.

그리고 `route` prop도 함께 전달한다.

```jsx
function MealsOverviewScreen({ route }) {
  ...
}
```

`route`에는 현재 화면과 관련된 정보가 들어 있다.

```text
route
-> 현재 로드된 화면에 대한 정보
-> 화면 이름
-> key
-> params
```

이번 강의에서 필요한 것은 `route.params`다.

### route.params에서 값 읽기

`CategoriesScreen`에서 다음처럼 params를 전달했다.

```jsx
navigation.navigate('MealsOverview', {
  categoryId: itemData.item.id,
});
```

그럼 `MealsOverviewScreen`에서는 `route.params`로 이 객체를 받을 수 있다.

```jsx
function MealsOverviewScreen({ route }) {
  const catId = route.params.categoryId;

  return (
    <View>
      <Text>{catId}</Text>
    </View>
  );
}
```

`route.params`는 `navigate`의 두 번째 인자로 넘긴 객체와 대응된다.

```text
navigate 두 번째 인자
-> { categoryId: 'c1' }

route.params
-> { categoryId: 'c1' }

route.params.categoryId
-> 'c1'
```

이제 카테고리마다 다른 값이 표시된다.

```text
Italian
-> c1

German
-> c4

Asian
-> c8
```

### navigation prop과 route prop

screen 컴포넌트는 두 가지 중요한 prop을 받을 수 있다.

```text
navigation
-> 화면 이동을 실행하는 객체
-> navigate, goBack 등 제공

route
-> 현재 화면에 대한 정보
-> params 등 제공
```

예를 들어 `CategoriesScreen`에서는 이동을 실행해야 하므로 `navigation`이 중요하다.

```jsx
function CategoriesScreen({ navigation }) {
  navigation.navigate('MealsOverview');
}
```

반면 `MealsOverviewScreen`에서는 전달된 값을 읽어야 하므로 `route`가 중요하다.

```jsx
function MealsOverviewScreen({ route }) {
  const catId = route.params.categoryId;
}
```

물론 하나의 화면에서 둘 다 사용할 수도 있다.

```jsx
function SomeScreen({ navigation, route }) {
  ...
}
```

### useRoute 훅

`navigation` 객체를 얻기 위해 `useNavigation` 훅을 사용할 수 있었던 것처럼, 현재 route 정보를 얻기 위한 훅도 있다.

```jsx
import { useRoute } from '@react-navigation/native';
```

`useRoute`를 호출하면 현재 route 객체를 얻을 수 있다.

```jsx
function SomeNestedComponent() {
  const route = useRoute();
  const catId = route.params.categoryId;
}
```

이 훅은 특히 screen 컴포넌트가 아닌 중첩 컴포넌트에서 현재 route 정보가 필요할 때 유용하다.

```text
screen 컴포넌트
-> route prop 사용 가능

중첩 컴포넌트
-> route prop 자동 제공 안 됨
-> useRoute 사용 가능
```

이번 예제에서는 `MealsOverviewScreen`이 screen 컴포넌트이므로 `route` prop을 그대로 사용해도 충분하다.

### route prop과 useRoute 중 무엇을 쓸까?

둘 다 같은 route 객체에 접근할 수 있다.

```text
route prop
-> screen 컴포넌트에서 간단하게 사용
-> 명시적으로 props에서 받음

useRoute
-> 중첩 컴포넌트에서 유용
-> prop drilling 없이 route 접근 가능
```

screen 컴포넌트에서는 보통 `route` prop을 사용하는 것이 간단하다.

하지만 깊은 자식 컴포넌트에서 현재 route params가 필요하다면 `useRoute`를 사용할 수 있다.

## 예시 코드

### CategoriesScreen에서 params 전달

```jsx
function CategoriesScreen({ navigation }) {
  function renderCategoryItem(itemData) {
    function pressHandler() {
      navigation.navigate('MealsOverview', {
        categoryId: itemData.item.id,
      });
    }

    return (
      <CategoryGridTile
        title={itemData.item.title}
        color={itemData.item.color}
        onPress={pressHandler}
      />
    );
  }

  return (
    <FlatList
      data={CATEGORIES}
      keyExtractor={(item) => item.id}
      renderItem={renderCategoryItem}
      numColumns={2}
    />
  );
}
```

### MealsOverviewScreen에서 params 읽기

```jsx
import { StyleSheet, Text, View } from 'react-native';

function MealsOverviewScreen({ route }) {
  const catId = route.params.categoryId;

  return (
    <View style={styles.container}>
      <Text>Meals Overview Screen - {catId}</Text>
    </View>
  );
}

export default MealsOverviewScreen;

const styles = StyleSheet.create({
  container: {
    flex: 1,
    padding: 16,
  },
});
```

### useRoute 대안

```jsx
import { useRoute } from '@react-navigation/native';

function MealsOverviewScreen() {
  const route = useRoute();
  const catId = route.params.categoryId;

  return (
    <View>
      <Text>{catId}</Text>
    </View>
  );
}
```

## 프론트엔드 개발자 관점에서의 핵심 차이

### React Router params와 비슷하지만 전달 방식이 다르다

웹에서는 URL path에 params를 담는 경우가 많다.

```text
/categories/c1
```

React Router에서는 `useParams`로 값을 읽는다.

```jsx
const { categoryId } = useParams();
```

React Navigation에서는 URL path가 아니라 `navigate`의 두 번째 인자로 params를 전달한다.

```jsx
navigation.navigate('MealsOverview', {
  categoryId: 'c1',
});
```

그리고 대상 화면에서 `route.params`로 읽는다.

```jsx
const catId = route.params.categoryId;
```

### 화면 이름과 params는 분리된다

웹에서는 경로 자체가 데이터 식별자를 포함할 수 있다.

```text
/meals/c1
```

React Navigation에서는 화면 이름과 params를 분리해서 생각한다.

```text
screen name
-> MealsOverview

params
-> { categoryId: 'c1' }
```

이 구조는 모바일 앱의 화면 스택 모델과 잘 맞는다.

### route.params는 현재 화면의 입력값이다

프론트엔드 관점에서 `route.params`는 페이지 컴포넌트의 입력값처럼 볼 수 있다.

```text
화면 컴포넌트
-> route.params를 통해 필요한 식별자 수신
-> 해당 식별자로 데이터 필터링 또는 조회
-> 결과 UI 렌더링
```

이번 예제에서는 `categoryId`를 받아 `MEALS` 배열을 필터링할 준비를 한다.

실제 앱이라면 이 id를 기반으로 API 요청을 보내거나 캐시된 데이터를 조회할 수도 있다.

### params 이름은 계약이다

`categoryId`라는 이름은 보내는 쪽과 받는 쪽이 맞춰야 하는 계약이다.

```jsx
// 보내는 쪽
navigation.navigate('MealsOverview', {
  categoryId: itemData.item.id,
});

// 받는 쪽
route.params.categoryId;
```

오타가 나면 값이 `undefined`가 된다.

TypeScript 프로젝트라면 navigation params 타입을 정의해 이런 실수를 줄일 수 있다.

## 정리

이번 강의에서는 React Navigation에서 화면 간 데이터를 전달하는 방법을 배웠다.

`navigation.navigate()`의 두 번째 인자로 params 객체를 전달하고, 대상 화면에서는 `route.params`를 통해 값을 읽는다.

이를 통해 `CategoriesScreen`에서 선택한 카테고리 id를 `MealsOverviewScreen`으로 전달할 수 있게 되었고, 다음 단계에서 해당 id를 사용해 알맞은 음식 목록을 필터링할 수 있다.
