# 4. FlatList 그리드와 CategoryGridTile 스타일링

## 핵심 요약

- 이번 강의에서는 카테고리 목록을 단순 리스트가 아니라 2열 그리드로 표시한다.
- `FlatList`의 `numColumns` prop을 사용하면 여러 열의 그리드 형태로 아이템을 렌더링할 수 있다.
- `CategoryGridTile` 컴포넌트에 외부 컨테이너, `Pressable`, 내부 컨테이너를 나누어 스타일을 적용한다.
- Android에서는 `android_ripple`로 터치 피드백을 추가하고, iOS에서는 `Pressable`의 `pressed` 상태를 이용해 opacity 효과를 추가한다.
- `overflow: hidden`은 Android ripple을 둥근 모서리 안에 가두는 데 필요하지만, iOS 그림자를 잘라낼 수 있으므로 `Platform` API로 분기한다.
- 카테고리 데이터의 `color` 값을 이용해 각 타일의 배경색을 동적으로 설정한다.
- 앱 배경색과 상태바 스타일도 조정해 화면의 전체적인 완성도를 높인다.

## 학습 목표

- `FlatList`에서 여러 열을 렌더링하는 방법을 익힌다.
- 그리드 타일을 구성할 때 `View`, `Pressable`, 내부 `View`를 나누는 이유를 이해한다.
- React Native에서 `flex: 1`이 부모 컨테이너 기준으로 동작한다는 점을 복습한다.
- Android와 iOS의 터치 피드백 처리 차이를 이해한다.
- `Platform.OS`를 이용해 플랫폼별 스타일을 분기하는 방법을 복습한다.
- 데이터 기반 동적 스타일링을 적용하는 방법을 익힌다.

## 주요 내용

### FlatList를 그리드로 만들기

기본적으로 `FlatList`는 한 줄에 하나의 아이템을 렌더링한다.

즉 기본값은 1열 리스트다.

```jsx
<FlatList
  data={CATEGORIES}
  renderItem={renderCategoryItem}
/>
```

여러 열로 표시하려면 `numColumns` prop을 사용한다.

```jsx
<FlatList
  data={CATEGORIES}
  keyExtractor={(item) => item.id}
  renderItem={renderCategoryItem}
  numColumns={2}
/>
```

`numColumns={2}`를 설정하면 한 행에 두 개의 아이템이 나란히 배치된다.

```text
기본 FlatList
-> 1열 리스트

numColumns={2}
-> 2열 그리드

numColumns={3}
-> 3열 그리드
```

이번 앱에서는 음식 카테고리를 카드형 타일로 보여주기 위해 2열 그리드를 사용한다.

### CategoryGridTile의 기본 구조

각 카테고리 타일은 다음 구조로 만든다.

```jsx
<View>
  <Pressable>
    <View>
      <Text>{title}</Text>
    </View>
  </Pressable>
</View>
```

겉으로 보면 `View`가 여러 번 중첩되어 불필요해 보일 수 있다.

하지만 React Native에서는 스타일링과 터치 피드백을 제대로 처리하기 위해 컨테이너를 나누는 경우가 많다.

```text
외부 View
-> FlatList 안에서 타일의 크기, margin, 그림자, 둥근 모서리 관리

Pressable
-> 사용자가 누를 수 있는 영역
-> Android ripple, iOS pressed 상태 처리

내부 View
-> 배경색, padding, 중앙 정렬 등 타일 내부 스타일 관리

Text
-> 카테고리 제목 출력
```

이 구조는 이후 내비게이션을 연결할 때도 그대로 활용된다.

### 외부 컨테이너 스타일

외부 `View`에는 타일이 그리드 안에서 차지할 공간과 외곽 스타일을 설정한다.

```jsx
gridItem: {
  flex: 1,
  margin: 16,
  height: 150,
  borderRadius: 8,
  elevation: 4,
}
```

핵심은 `flex: 1`이다.

`FlatList`가 2열 레이아웃을 만들면, 각 아이템은 해당 열 안에서 가능한 공간을 나누어 가져야 한다.

```text
flex: 1
-> 현재 부모 컨테이너 안에서 가능한 공간을 차지
-> 여기서는 FlatList가 만든 열 너비 안에서 확장
```

또한 `margin: 16`을 주어 각 타일 사이에 간격을 만들고, `height: 150`으로 타일 높이를 고정한다.

강의에서는 내비게이션 학습에 집중하기 위해 이 값을 고정했지만, 필요하면 이전 섹션에서 배운 `Dimensions` API나 `useWindowDimensions`를 이용해 기기 크기에 따라 다르게 만들 수도 있다.

### 내부 컨테이너 스타일

내부 `View`는 타일 안의 콘텐츠 배치를 담당한다.

```jsx
innerContainer: {
  flex: 1,
  padding: 16,
  justifyContent: 'center',
  alignItems: 'center',
}
```

여기서도 `flex: 1`이 필요하다.

타일 내부 전체를 채워야 텍스트를 중앙에 배치할 수 있기 때문이다.

```text
justifyContent: 'center'
-> 주축 방향 중앙 정렬

alignItems: 'center'
-> 교차축 방향 중앙 정렬
```

React Native의 기본 `flexDirection`은 `column`이므로, `justifyContent`는 세로 중앙 정렬, `alignItems`는 가로 중앙 정렬 역할을 한다.

### Pressable에도 flex가 필요한 이유

처음 내부 `View`에만 `flex: 1`을 주면 텍스트가 사라지거나 원하는 영역을 차지하지 못할 수 있다.

이유는 내부 `View`의 부모가 외부 `View`가 아니라 `Pressable`이기 때문이다.

```text
외부 View
  -> Pressable
    -> 내부 View
      -> Text
```

내부 `View`가 `flex: 1`로 확장되려면 부모인 `Pressable`도 공간을 차지하고 있어야 한다.

따라서 `Pressable`에도 스타일을 추가한다.

```jsx
button: {
  flex: 1,
}
```

```jsx
<Pressable style={styles.button}>
  <View style={styles.innerContainer}>
    <Text>{title}</Text>
  </View>
</Pressable>
```

부모가 공간을 가지지 않으면 자식의 `flex: 1`도 기대한 대로 동작하지 않는다.

### iOS 그림자와 backgroundColor

Android에서는 `elevation`으로 그림자를 줄 수 있다.

```jsx
elevation: 4
```

iOS에서는 `shadowColor`, `shadowOpacity`, `shadowOffset`, `shadowRadius`를 사용한다.

```jsx
shadowColor: 'black',
shadowOpacity: 0.25,
shadowOffset: { width: 0, height: 2 },
shadowRadius: 8,
```

하지만 iOS에서는 그림자가 보이려면 배경색이 필요하다.

```jsx
backgroundColor: 'white'
```

배경색이 없으면 iOS에서 shadow 스타일을 설정해도 그림자가 보이지 않을 수 있다.

이 부분은 웹 CSS와 다른 React Native 스타일링의 작은 함정이다.

### 제목 텍스트 스타일

카테고리 제목에는 글자 크기와 굵기를 설정한다.

```jsx
title: {
  fontWeight: 'bold',
  fontSize: 18,
}
```

이전 섹션에서 배운 커스텀 폰트를 적용할 수도 있지만, 이번 강의에서는 기본 시스템 폰트를 사용한다.

```jsx
<Text style={styles.title}>{title}</Text>
```

텍스트 관련 스타일은 반드시 `Text` 컴포넌트에 직접 적용해야 한다.

부모 `View`에 `fontSize`나 `fontWeight`를 설정해도 일반적으로 자식 `Text`에 상속되지 않는다.

### Android 터치 피드백: android_ripple

Android에서는 `Pressable`에 `android_ripple` prop을 추가해 물결 효과를 줄 수 있다.

```jsx
<Pressable
  android_ripple={{ color: '#ccc' }}
>
  ...
</Pressable>
```

이렇게 하면 사용자가 타일을 눌렀을 때 Android 네이티브에 가까운 ripple 효과가 나타난다.

다만 ripple이 둥근 모서리 밖으로 퍼져 보일 수 있다.

이를 막기 위해 외부 컨테이너에 `overflow: 'hidden'`을 설정한다.

```jsx
overflow: 'hidden'
```

### overflow와 iOS 그림자 문제

`overflow: 'hidden'`은 Android ripple을 둥근 모서리 안에 가두는 데 유용하다.

하지만 iOS에서는 문제가 생긴다.

그림자는 컨테이너 바깥에 그려지는 효과인데, `overflow: 'hidden'`을 설정하면 바깥으로 나가는 그림자도 잘려 버린다.

```text
overflow: 'hidden'
-> Android ripple을 둥근 모서리 안에 가둠
-> iOS shadow도 잘라버림
```

따라서 플랫폼별로 다르게 설정해야 한다.

```jsx
import { Platform } from 'react-native';

gridItem: {
  overflow: Platform.OS === 'android' ? 'hidden' : 'visible',
}
```

Android에서는 `hidden`, iOS에서는 `visible`을 사용하면 양쪽의 목적을 모두 만족할 수 있다.

### iOS 터치 피드백: pressed 상태

`android_ripple`은 Android 전용이다.

iOS에서는 `Pressable`의 `style` prop에 함수를 전달해 pressed 상태에 따라 스타일을 바꾼다.

```jsx
<Pressable
  style={({ pressed }) => [
    styles.button,
    pressed ? styles.buttonPressed : null,
  ]}
>
  ...
</Pressable>
```

`pressed`는 React Native가 자동으로 제공하는 boolean 값이다.

```text
pressed === true
-> 사용자가 누르고 있는 중

pressed === false
-> 누르고 있지 않음
```

눌린 상태에서는 opacity를 낮춘다.

```jsx
buttonPressed: {
  opacity: 0.5,
}
```

이렇게 하면 iOS에서도 사용자가 누르는 동작에 대한 시각적 피드백을 줄 수 있다.

### 데이터 기반 배경색 적용

각 카테고리 데이터에는 `color` 값이 있다.

```text
Category
-> id
-> title
-> color
```

`CategoryGridTile`은 props로 `color`를 받는다.

```jsx
function CategoryGridTile({ title, color }) {
  ...
}
```

이 값을 배경색에 적용하면 카테고리마다 다른 색의 타일을 만들 수 있다.

처음에는 외부 `View`에 배경색을 적용할 수 있다.

```jsx
<View style={[styles.gridItem, { backgroundColor: color }]}>
```

하지만 iOS의 opacity 피드백을 더 자연스럽게 보이게 하려면 내부 컨테이너에 배경색을 적용하는 편이 좋다.

```jsx
<View style={[styles.innerContainer, { backgroundColor: color }]}>
```

이 경우 둥근 모서리를 유지하려면 외부 컨테이너와 내부 컨테이너 모두에 `borderRadius`를 설정해야 한다.

```jsx
gridItem: {
  borderRadius: 8,
}

innerContainer: {
  borderRadius: 8,
}
```

### app.json으로 배경색 설정

앱 전체 배경색은 `app.json`에서 설정할 수 있다.

```json
{
  "expo": {
    "backgroundColor": "#24180f"
  }
}
```

Expo 프로젝트에서는 `backgroundColor`를 설정하면 앱의 기본 배경색으로 사용된다.

변경 후에는 터미널에서 `r`을 눌러 앱을 강제 새로고침해야 반영될 수 있다.

### StatusBar 스타일 설정

배경색을 어둡게 만들면 상태바도 밝게 설정해야 잘 보인다.

`App.js`에서 `StatusBar`를 추가한다.

```jsx
import { StatusBar } from 'expo-status-bar';

export default function App() {
  return (
    <>
      <StatusBar style="light" />
      <CategoriesScreen />
    </>
  );
}
```

`StatusBar`와 `CategoriesScreen`이 형제 요소이므로 `Fragment`로 감싼다.

```jsx
<>
  <StatusBar style="light" />
  <CategoriesScreen />
</>
```

## 예시 코드

### CategoriesScreen

```jsx
<FlatList
  data={CATEGORIES}
  keyExtractor={(item) => item.id}
  renderItem={renderCategoryItem}
  numColumns={2}
/>
```

### CategoryGridTile

```jsx
import { Platform, Pressable, StyleSheet, Text, View } from 'react-native';

function CategoryGridTile({ title, color }) {
  return (
    <View style={styles.gridItem}>
      <Pressable
        android_ripple={{ color: '#ccc' }}
        style={({ pressed }) => [
          styles.button,
          pressed ? styles.buttonPressed : null,
        ]}
      >
        <View style={[styles.innerContainer, { backgroundColor: color }]}>
          <Text style={styles.title}>{title}</Text>
        </View>
      </Pressable>
    </View>
  );
}

export default CategoryGridTile;

const styles = StyleSheet.create({
  gridItem: {
    flex: 1,
    margin: 16,
    height: 150,
    borderRadius: 8,
    elevation: 4,
    backgroundColor: 'white',
    shadowColor: 'black',
    shadowOpacity: 0.25,
    shadowOffset: { width: 0, height: 2 },
    shadowRadius: 8,
    overflow: Platform.OS === 'android' ? 'hidden' : 'visible',
  },
  button: {
    flex: 1,
  },
  buttonPressed: {
    opacity: 0.5,
  },
  innerContainer: {
    flex: 1,
    padding: 16,
    borderRadius: 8,
    justifyContent: 'center',
    alignItems: 'center',
  },
  title: {
    fontWeight: 'bold',
    fontSize: 18,
  },
});
```

## 프론트엔드 개발자 관점에서의 핵심 차이

### CSS Grid가 아니라 FlatList numColumns

웹에서는 그리드 레이아웃을 만들 때 CSS Grid를 많이 사용한다.

```css
.grid {
  display: grid;
  grid-template-columns: repeat(2, 1fr);
}
```

React Native에는 CSS Grid가 없다.

이 강의에서는 `FlatList`의 `numColumns`를 사용해 2열 그리드를 만든다.

```jsx
<FlatList numColumns={2} />
```

즉 모바일 RN에서는 목록 컴포넌트 자체가 그리드 배치를 지원하는 방식으로 접근할 수 있다.

### 터치 피드백은 플랫폼별 감각이 다르다

웹에서는 `:active`, `:hover`, `cursor` 같은 CSS 상태를 생각하기 쉽다.

React Native에서는 플랫폼별 터치 피드백을 고려해야 한다.

```text
Android
-> ripple 효과가 자연스러움
-> android_ripple 사용

iOS
-> opacity 변화가 흔함
-> Pressable의 pressed 상태로 처리
```

같은 `Pressable`을 사용하더라도, 플랫폼별로 자연스러운 피드백 방식이 다를 수 있다.

### overflow는 그림자와 충돌할 수 있다

웹에서도 `overflow: hidden`은 그림자를 잘라낼 수 있다.

React Native에서도 같은 문제가 발생한다.

특히 iOS shadow는 컨테이너 바깥에 그려지는 효과이므로 `overflow: hidden`과 충돌한다.

```text
ripple을 자르려면 overflow hidden 필요
shadow를 보이려면 overflow visible 필요
```

이런 경우 `Platform` API로 분기한다.

### 데이터와 스타일의 연결

카테고리의 `color` 값을 UI 스타일에 직접 연결하는 방식은 웹에서도 익숙한 패턴이다.

```jsx
style={{ backgroundColor: category.color }}
```

React Native에서도 스타일 객체를 배열로 병합해 동적 스타일을 추가한다.

```jsx
style={[styles.innerContainer, { backgroundColor: color }]}
```

기본 스타일과 데이터 기반 스타일을 자연스럽게 합성하는 방식이다.

## 정리

이번 강의에서는 카테고리 목록을 2열 그리드로 만들고, 각 카테고리 타일에 스타일과 터치 피드백을 추가했다.

`FlatList`의 `numColumns`로 그리드 형태를 만들고, `CategoryGridTile` 내부에서는 외부 컨테이너, `Pressable`, 내부 컨테이너를 나누어 스타일을 세밀하게 제어했다.

Android ripple, iOS pressed opacity, 플랫폼별 `overflow` 처리, 데이터 기반 배경색 적용까지 더해지면서 이제 카테고리 타일은 실제로 누를 수 있는 UI처럼 보인다.

다음 단계에서는 이 타일을 눌렀을 때 새 화면으로 이동하도록 내비게이션을 본격적으로 연결한다.
