# 15. MealDetailScreen 스타일링과 재사용 컴포넌트

## 핵심 요약

- 이번 강의에서는 `MealDetailScreen`의 스타일을 다듬어 실제 상세 화면처럼 만든다.
- 원격 이미지는 `width`와 `height`를 지정해야 보이므로 이미지 스타일을 추가한다.
- 제목과 상세 정보 텍스트 색상을 어두운 배경에 맞게 조정한다.
- `MealDetails`에 `style`, `textStyle` prop을 추가해 외부에서 스타일을 덮어쓸 수 있게 만든다.
- `Text` 컴포넌트는 border 스타일을 직접 적용하기 어렵기 때문에 subtitle에는 `View` 래퍼를 사용한다.
- `Subtitle`과 `List` 컴포넌트를 새로 만들어 재료와 조리 단계 섹션을 재사용 가능한 구조로 정리한다.
- 상세 화면 전체를 `ScrollView`로 감싸 긴 콘텐츠를 스크롤할 수 있게 만든다.
- 리스트 영역의 너비를 제한하고 중앙 정렬해 모바일 화면에서 더 보기 좋게 만든다.

## 학습 목표

- `MealDetailScreen`에 `StyleSheet`를 추가해 이미지와 텍스트 스타일을 적용하는 방법을 익힌다.
- 재사용 컴포넌트에 외부 스타일을 주입하는 패턴을 복습한다.
- `Text`와 `View`가 지원하는 스타일 차이를 이해한다.
- 반복되는 subtitle/list UI를 컴포넌트로 추출하는 기준을 익힌다.
- 짧은 문자열 배열을 재사용 가능한 `List` 컴포넌트로 출력하는 방법을 이해한다.
- 상세 화면에서 `ScrollView`가 필요한 상황을 파악한다.

## 주요 내용

### 상세 화면 스타일링 시작

이전 강의에서 `MealDetailScreen`은 실제 데이터를 출력하기 시작했다.

하지만 아직 스타일이 거의 없어서 이미지가 보이지 않거나, 텍스트가 배경과 잘 어울리지 않았다.

이번 강의에서는 `MealDetailScreen`에 `StyleSheet`를 추가한다.

```jsx
import { StyleSheet } from 'react-native';

const styles = StyleSheet.create({
  // styles
});
```

강의에서는 컴포넌트를 더 많이 쪼개지 않고, 우선 `MealDetailScreen` 내부에서 스타일을 작성한다.

실제 프로젝트에서는 화면이 커지면 더 작은 컴포넌트로 나눌 수 있다.

### 이미지 스타일

상세 화면 상단에는 음식 이미지를 크게 보여준다.

원격 이미지는 크기를 자동으로 추론할 수 없기 때문에 `width`와 `height`를 지정해야 한다.

```jsx
image: {
  width: '100%',
  height: 350,
}
```

```jsx
<Image
  source={{ uri: selectedMeal.imageUrl }}
  style={styles.image}
/>
```

`width: '100%'`는 부모 컨테이너의 사용 가능한 너비 전체를 의미한다.

`height: 350`은 고정 높이다.

```text
이미지 스타일
-> 부모 너비 전체 사용
-> 350px 높이 지정
-> 원격 이미지 표시 가능
```

### 제목 스타일

이미지 아래에는 음식 제목을 크게 표시한다.

```jsx
title: {
  fontWeight: 'bold',
  fontSize: 24,
  margin: 8,
  textAlign: 'center',
  color: 'white',
}
```

```jsx
<Text style={styles.title}>{selectedMeal.title}</Text>
```

배경이 어두운 갈색 계열이므로 텍스트 색상은 흰색으로 설정한다.

```text
제목 스타일
-> 굵게
-> 크게
-> 가운데 정렬
-> 주변 여백
-> 흰색 텍스트
```

### MealDetails의 색상 문제

`MealDetails`는 목록 카드와 상세 화면에서 함께 사용된다.

목록 카드에서는 검은 텍스트가 적절하지만, 상세 화면에서는 어두운 배경 때문에 흰 텍스트가 필요하다.

```text
MealItem 안의 MealDetails
-> 검은 텍스트가 적절

MealDetailScreen 안의 MealDetails
-> 흰 텍스트가 필요
```

공통 컴포넌트 내부에서 색상을 흰색으로 바꾸면 목록 카드에서도 흰색이 되어 문제가 생긴다.

따라서 외부에서 추가 스타일을 주입할 수 있게 만든다.

### style, textStyle prop으로 스타일 확장하기

`MealDetails`에 `style`과 `textStyle` prop을 추가한다.

```jsx
function MealDetails({ duration, complexity, affordability, style, textStyle }) {
  return (
    <View style={[styles.details, style]}>
      <Text style={[styles.detailItem, textStyle]}>{duration}m</Text>
      <Text style={[styles.detailItem, textStyle]}>
        {complexity.toUpperCase()}
      </Text>
      <Text style={[styles.detailItem, textStyle]}>
        {affordability.toUpperCase()}
      </Text>
    </View>
  );
}
```

`style`은 바깥 `View`에 병합되고, `textStyle`은 각각의 `Text`에 병합된다.

여기서 중요한 점은 텍스트 색상은 부모 `View`에 설정해도 `Text`에 상속되지 않는다는 것이다.

따라서 텍스트 색상을 바꾸려면 `Text`에 직접 적용되는 `textStyle`이 필요하다.

```text
style
-> MealDetails 컨테이너 View 스타일 확장

textStyle
-> 내부 Text 스타일 확장
-> 색상 변경에 필요
```

상세 화면에서는 다음처럼 사용한다.

```jsx
<MealDetails
  duration={selectedMeal.duration}
  complexity={selectedMeal.complexity}
  affordability={selectedMeal.affordability}
  textStyle={styles.detailText}
/>
```

```jsx
detailText: {
  color: 'white',
}
```

### subtitle 스타일

`Ingredients`와 `Steps`는 섹션 제목처럼 보이게 만든다.

처음에는 `Text`에 직접 border와 padding을 주려고 하지만, React Native의 `Text`는 border 스타일을 기대한 대로 지원하지 않는다.

그래서 `Text`를 `View`로 감싼다.

```jsx
<View style={styles.subtitleContainer}>
  <Text style={styles.subtitle}>Ingredients</Text>
</View>
```

`subtitle`은 텍스트 자체 스타일이다.

```jsx
subtitle: {
  color: '#e2b497',
  fontSize: 18,
  fontWeight: 'bold',
  textAlign: 'center',
}
```

`subtitleContainer`는 border와 여백을 담당한다.

```jsx
subtitleContainer: {
  padding: 6,
  marginVertical: 4,
  marginHorizontal: 24,
  borderBottomColor: '#e2b497',
  borderBottomWidth: 2,
}
```

```text
Text
-> 글자 색상, 크기, 굵기, 정렬

View 래퍼
-> border, padding, margin
```

### Subtitle 컴포넌트 추출

`Ingredients`와 `Steps`는 같은 형태의 subtitle이다.

따라서 별도 컴포넌트로 추출한다.

```text
components/
  MealDetail/
    Subtitle.js
```

`Subtitle`은 `children`을 받아 텍스트로 출력한다.

```jsx
function Subtitle({ children }) {
  return (
    <View style={styles.subtitleContainer}>
      <Text style={styles.subtitle}>{children}</Text>
    </View>
  );
}

export default Subtitle;
```

이제 상세 화면에서는 다음처럼 사용할 수 있다.

```jsx
<Subtitle>Ingredients</Subtitle>
<Subtitle>Steps</Subtitle>
```

시각적으로는 동일하지만, JSX가 훨씬 읽기 쉬워진다.

### List 컴포넌트 추출

재료 목록과 조리 단계 목록도 같은 방식으로 문자열 배열을 출력한다.

이 역시 재사용 가능한 컴포넌트로 만든다.

```text
components/
  MealDetail/
    List.js
```

`List`는 `data` prop으로 문자열 배열을 받는다.

```jsx
function List({ data }) {
  return data.map((dataPoint) => (
    <View key={dataPoint} style={styles.listItem}>
      <Text style={styles.itemText}>{dataPoint}</Text>
    </View>
  ));
}

export default List;
```

각 항목은 `View`로 감싼다.

이유는 list item에 배경색, borderRadius, padding 같은 View 스타일을 주기 위해서다.

```jsx
listItem: {
  borderRadius: 6,
  paddingHorizontal: 8,
  paddingVertical: 4,
  marginVertical: 4,
  marginHorizontal: 12,
  backgroundColor: '#e2b497',
}
```

텍스트는 어두운 갈색으로 설정해 밝은 배경과 대비를 만든다.

```jsx
itemText: {
  color: '#351401',
  textAlign: 'center',
}
```

상세 화면에서는 다음처럼 사용한다.

```jsx
<List data={selectedMeal.ingredients} />
<List data={selectedMeal.steps} />
```

### 리스트 영역 너비 제한

재료와 단계 항목이 화면 전체 너비를 다 차지하면 다소 넓어 보일 수 있다.

그래서 리스트 영역을 별도 컨테이너로 감싸고 너비를 제한한다.

```jsx
listContainer: {
  width: '80%',
}
```

```jsx
<View style={styles.listOuterContainer}>
  <View style={styles.listContainer}>
    <Subtitle>Ingredients</Subtitle>
    <List data={selectedMeal.ingredients} />
    <Subtitle>Steps</Subtitle>
    <List data={selectedMeal.steps} />
  </View>
</View>
```

`listContainer`의 너비가 80%이므로, 이를 가운데 정렬하기 위해 바깥 컨테이너에 `alignItems: 'center'`를 준다.

```jsx
listOuterContainer: {
  alignItems: 'center',
}
```

```text
listOuterContainer
-> 전체 너비 사용
-> 자식 listContainer를 가운데 정렬

listContainer
-> 80% 너비만 사용
```

### ScrollView 적용

상세 화면에는 이미지, 제목, 재료, 단계가 모두 들어간다.

조리 단계가 길면 화면 아래로 콘텐츠가 넘어간다.

현재는 스크롤되지 않으면 아래 항목을 볼 수 없다.

따라서 전체 화면을 `ScrollView`로 감싼다.

```jsx
import { ScrollView } from 'react-native';
```

```jsx
<ScrollView style={styles.rootContainer}>
  {/* detail content */}
</ScrollView>
```

여기서는 재료 목록이나 조리 단계 목록만 따로 스크롤하지 않고, 상세 화면 전체가 스크롤되도록 한다.

이 편이 사용 경험상 자연스럽다.

```text
부분 스크롤
-> ingredients/steps만 따로 스크롤

전체 스크롤
-> 이미지부터 단계까지 한 화면 흐름으로 스크롤
-> 상세 페이지에 더 자연스러움
```

### 하단 여백 추가

스크롤 화면의 마지막 항목이 너무 화면 끝에 붙지 않도록 하단 여백을 준다.

```jsx
rootContainer: {
  marginBottom: 32,
}
```

이제 사용자가 아래로 더 여유 있게 스크롤할 수 있다.

## 프론트엔드 개발자 관점에서의 핵심 차이

### CSS의 cascade를 직접 흉내 내기

웹에서는 부모나 class 조합으로 스타일을 덮어쓰는 일이 자연스럽다.

React Native에는 CSS cascade가 없으므로, 컴포넌트가 외부 스타일 prop을 받아 내부 스타일 배열에 병합해야 한다.

```jsx
style={[styles.defaultStyle, incomingStyle]}
```

이 패턴을 직접 설계해야 재사용 컴포넌트를 유연하게 만들 수 있다.

### Text와 View의 스타일 지원 범위 차이

웹에서는 텍스트 요소에도 border를 줄 수 있는 경우가 많다.

React Native에서는 `Text`와 `View`가 지원하는 스타일이 다르다.

이번 강의에서는 subtitle border를 위해 `Text`를 `View`로 감쌌다.

```text
Text
-> 텍스트 색상, 크기, 굵기, 정렬

View
-> border, padding, margin, background, layout
```

스타일이 적용되지 않을 때는 해당 컴포넌트가 그 스타일을 지원하는지 의심해봐야 한다.

### 상세 페이지는 전체 ScrollView가 자연스럽다

웹에서는 페이지 전체가 기본적으로 스크롤된다.

React Native에서는 명시적으로 `ScrollView`를 사용해야 한다.

상세 페이지처럼 콘텐츠가 화면보다 길어질 수 있는 경우, 전체를 `ScrollView`로 감싸는 것이 자연스럽다.

```jsx
<ScrollView>
  {/* detail content */}
</ScrollView>
```

### 작은 컴포넌트 추출의 기준

`Subtitle`, `List`는 작지만 반복되는 UI 의도를 표현한다.

```text
Subtitle
-> 섹션 제목 표현

List
-> 문자열 배열을 보기 좋은 항목으로 출력
```

컴포넌트가 작아도 반복되는 패턴과 스타일 의도를 담고 있다면 추출할 가치가 있다.

## 정리

이번 강의에서는 `MealDetailScreen`을 실제 상세 페이지처럼 스타일링했다.

이미지 크기와 제목 스타일을 잡고, `MealDetails`가 외부 텍스트 스타일을 받을 수 있도록 개선했으며, `Subtitle`과 `List` 컴포넌트를 새로 만들어 재료와 조리 단계 섹션을 정리했다.

마지막으로 상세 화면 전체를 `ScrollView`로 감싸 긴 콘텐츠를 스크롤할 수 있게 만들었다.
