# 13. MealDetailScreen 추가와 MealItem에서 이동

## 핵심 요약

- 이번 강의에서는 음식 아이템을 눌렀을 때 음식 상세 화면으로 이동하는 흐름을 추가한다.
- 새로운 화면 컴포넌트인 `MealDetailScreen`을 만들고 Stack Navigator에 등록한다.
- `MealItem`은 screen 컴포넌트가 아니므로 `navigation` prop을 자동으로 받지 않는다.
- 대신 `useNavigation` 훅을 사용해 `MealItem` 내부에서 navigation 객체를 얻는다.
- `navigation.navigate('MealDetail', { mealId })`를 호출해 상세 화면으로 이동하면서 선택한 음식 id를 params로 전달한다.
- `MealsOverviewScreen`에서 `MealItem`에 `id` prop을 전달해야 한다.
- `MealDetailScreen`은 screen 컴포넌트이므로 `route` prop을 받고, `route.params.mealId`로 전달된 음식 id를 읽을 수 있다.

## 학습 목표

- 새로운 screen 컴포넌트를 만들고 Stack Navigator에 등록하는 과정을 복습한다.
- 일반 자식 컴포넌트에서 `useNavigation` 훅으로 화면 이동을 실행하는 방법을 익힌다.
- `navigate`를 호출할 때 params로 `mealId`를 전달하는 패턴을 이해한다.
- 이벤트 핸들러 안에서 `navigate`를 호출해야 하는 이유를 이해한다.
- `MealDetailScreen`에서 `route.params`로 전달된 데이터를 읽는 방법을 복습한다.
- 카테고리 목록, 음식 목록, 음식 상세로 이어지는 3단계 Stack 흐름을 파악한다.

## 주요 내용

### 다음 목표: 음식 상세 화면

현재 앱은 다음 흐름까지 완성되어 있다.

```text
CategoriesScreen
-> MealsOverviewScreen
```

이제 음식 목록에서 특정 음식을 누르면 상세 화면으로 이동해야 한다.

```text
CategoriesScreen
-> MealsOverviewScreen
-> MealDetailScreen
```

상세 화면에는 나중에 조리 방법, 재료, 단계 등을 보여줄 예정이다.

이번 강의에서는 우선 상세 화면으로 이동하고, 선택한 음식 id를 전달하는 흐름만 만든다.

### MealDetailScreen 추가

`screens` 폴더에 `MealDetailScreen.js` 파일을 추가한다.

```text
screens/
  CategoriesScreen.js
  MealsOverviewScreen.js
  MealDetailScreen.js
```

처음에는 더미 텍스트만 출력한다.

```jsx
import { Text } from 'react-native';

function MealDetailScreen() {
  return <Text>This is the Meal Detail screen.</Text>;
}

export default MealDetailScreen;
```

나중에 선택한 음식 데이터를 기반으로 실제 상세 정보를 렌더링할 예정이다.

### Stack.Navigator에 화면 등록

새 screen 컴포넌트를 만들었으면 `App.js`에서 Stack에 등록해야 한다.

```jsx
import MealDetailScreen from './screens/MealDetailScreen';
```

```jsx
<Stack.Navigator>
  <Stack.Screen
    name="MealsCategories"
    component={CategoriesScreen}
  />
  <Stack.Screen
    name="MealsOverview"
    component={MealsOverviewScreen}
  />
  <Stack.Screen
    name="MealDetail"
    component={MealDetailScreen}
  />
</Stack.Navigator>
```

`Stack.Screen`의 위치는 이 경우 크게 중요하지 않다.

중요한 것은 screen name과 component가 등록되는 것이다.

```text
name="MealDetail"
-> navigate 대상 식별자

component={MealDetailScreen}
-> 실제 렌더링할 화면 컴포넌트
```

### MealItem을 눌렀을 때 이동하기

상세 화면으로 이동하는 트리거는 음식 카드 클릭이다.

`MealItem` 컴포넌트 안에는 이미 `Pressable`이 있다.

```jsx
<Pressable>
  ...
</Pressable>
```

이제 `Pressable`의 `onPress`에 상세 화면 이동 함수를 연결하면 된다.

### 왜 useNavigation을 사용하는가?

`MealItem`은 `Stack.Screen`에 직접 등록된 화면 컴포넌트가 아니다.

따라서 `navigation` prop을 자동으로 받지 않는다.

```text
MealDetailScreen
-> Stack.Screen에 등록됨
-> navigation, route prop 자동 제공

MealItem
-> 일반 자식 컴포넌트
-> navigation prop 자동 제공 안 됨
```

이 경우 선택지는 두 가지다.

```text
방법 1
-> MealsOverviewScreen에서 onPress 함수를 만들어 MealItem에 전달

방법 2
-> MealItem 내부에서 useNavigation 훅 사용
```

카테고리 타일에서는 방법 1을 사용했다.

이번에는 방법 2를 연습하기 위해 `MealItem` 안에서 `useNavigation`을 사용한다.

### useNavigation으로 navigation 객체 얻기

`useNavigation`은 `@react-navigation/native`에서 가져온다.

```jsx
import { useNavigation } from '@react-navigation/native';
```

컴포넌트 함수 안에서 호출하면 navigation 객체를 얻을 수 있다.

```jsx
function MealItem({ id, title, imageUrl, duration, complexity, affordability }) {
  const navigation = useNavigation();
}
```

이 객체는 screen 컴포넌트에서 `navigation` prop으로 받는 객체와 같은 형태를 가진다.

따라서 `navigate`를 호출할 수 있다.

```jsx
navigation.navigate('MealDetail');
```

### mealId params 전달

상세 화면에서는 어떤 음식이 선택되었는지 알아야 한다.

따라서 `MealDetail` 화면으로 이동할 때 `mealId`를 params로 전달한다.

```jsx
navigation.navigate('MealDetail', {
  mealId: id,
});
```

여기서 `id`는 `MealItem`이 prop으로 받은 음식 id다.

```text
보내는 쪽
-> mealId: id

받는 쪽
-> route.params.mealId
```

### navigate는 이벤트 핸들러 안에서 호출해야 한다

처음에 실수하기 쉬운 부분은 `navigate`를 컴포넌트 함수 본문에서 바로 호출하는 것이다.

```jsx
function MealItem() {
  navigation.navigate('MealDetail'); // 잘못된 위치
}
```

이렇게 하면 컴포넌트가 렌더링되는 즉시 이동이 실행된다.

사용자가 아이템을 누르기도 전에 상세 화면으로 이동해버린다.

따라서 `navigate`는 이벤트 핸들러 안에 넣어야 한다.

```jsx
function selectMealItemHandler() {
  navigation.navigate('MealDetail', {
    mealId: id,
  });
}
```

그리고 이 함수를 `Pressable`의 `onPress`에 연결한다.

```jsx
<Pressable onPress={selectMealItemHandler}>
  ...
</Pressable>
```

이제 음식 아이템을 실제로 눌렀을 때만 상세 화면으로 이동한다.

### MealItem에 id prop 전달하기

`MealItem`에서 `id`를 사용하려면 `MealsOverviewScreen`에서 id를 넘겨줘야 한다.

기존에는 title, imageUrl, duration, complexity, affordability 등을 넘겼다.

여기에 `id`를 추가한다.

```jsx
function renderMealItem(itemData) {
  const item = itemData.item;

  const mealItemProps = {
    id: item.id,
    title: item.title,
    imageUrl: item.imageUrl,
    duration: item.duration,
    complexity: item.complexity,
    affordability: item.affordability,
  };

  return <MealItem {...mealItemProps} />;
}
```

이제 `MealItem`은 자신이 어떤 음식 카드를 나타내는지 알 수 있다.

### MealDetailScreen에서 mealId 읽기

`MealDetailScreen`은 `Stack.Screen`에 등록된 화면 컴포넌트다.

따라서 React Navigation이 `route` prop을 자동으로 제공한다.

```jsx
function MealDetailScreen({ route }) {
  const mealId = route.params.mealId;
}
```

`mealId`는 `MealItem`에서 navigate를 호출할 때 전달한 params와 연결된다.

```jsx
navigation.navigate('MealDetail', {
  mealId: id,
});
```

테스트를 위해 화면에 출력할 수 있다.

```jsx
function MealDetailScreen({ route }) {
  const mealId = route.params.mealId;

  return <Text>This is the Meal Detail screen: {mealId}</Text>;
}
```

다른 음식을 누르면 다른 meal id가 표시된다.

이제 상세 화면이 선택된 음식에 따라 달라질 준비가 된 것이다.

### 현재까지의 네비게이션 흐름

이제 앱은 3단계 Stack Navigation 흐름을 가진다.

```text
1. CategoriesScreen
   -> categoryId 전달

2. MealsOverviewScreen
   -> categoryId로 음식 목록 필터링
   -> MealItem에 meal id 전달

3. MealDetailScreen
   -> mealId를 params로 수신
   -> 나중에 해당 meal 상세 정보 출력
```

이 흐름은 실제 앱에서 매우 흔하다.

```text
목록
-> 필터링된 목록
-> 상세
```

## 예시 코드

### App.js

```jsx
<Stack.Navigator>
  <Stack.Screen
    name="MealsCategories"
    component={CategoriesScreen}
  />
  <Stack.Screen
    name="MealsOverview"
    component={MealsOverviewScreen}
  />
  <Stack.Screen
    name="MealDetail"
    component={MealDetailScreen}
  />
</Stack.Navigator>
```

### MealItem

```jsx
import { useNavigation } from '@react-navigation/native';
import { Pressable, Text, View } from 'react-native';

function MealItem({
  id,
  title,
  imageUrl,
  duration,
  complexity,
  affordability,
}) {
  const navigation = useNavigation();

  function selectMealItemHandler() {
    navigation.navigate('MealDetail', {
      mealId: id,
    });
  }

  return (
    <View>
      <Pressable onPress={selectMealItemHandler}>
        <Text>{title}</Text>
      </Pressable>
    </View>
  );
}

export default MealItem;
```

### MealsOverviewScreen

```jsx
function renderMealItem(itemData) {
  const item = itemData.item;

  const mealItemProps = {
    id: item.id,
    title: item.title,
    imageUrl: item.imageUrl,
    duration: item.duration,
    complexity: item.complexity,
    affordability: item.affordability,
  };

  return <MealItem {...mealItemProps} />;
}
```

### MealDetailScreen

```jsx
import { Text } from 'react-native';

function MealDetailScreen({ route }) {
  const mealId = route.params.mealId;

  return <Text>This is the Meal Detail screen: {mealId}</Text>;
}

export default MealDetailScreen;
```

## 프론트엔드 개발자 관점에서의 핵심 차이

### 목록 아이템에서 상세 페이지로 이동하는 패턴

웹에서는 보통 카드나 리스트 아이템을 클릭하면 URL이 바뀐다.

```jsx
<Link to={`/meals/${meal.id}`}>
  <MealCard meal={meal} />
</Link>
```

React Native에서는 `Pressable`과 `navigation.navigate()`를 사용한다.

```jsx
<Pressable onPress={selectMealItemHandler}>
  ...
</Pressable>
```

```jsx
navigation.navigate('MealDetail', {
  mealId: id,
});
```

역할은 비슷하지만 URL path 대신 screen name과 params를 사용한다.

### useNavigation은 하위 컴포넌트용 escape hatch

이번에는 `MealItem` 내부에서 `useNavigation`을 사용했다.

이는 하위 컴포넌트가 직접 화면 이동을 실행해야 할 때 유용하다.

```text
장점
-> 부모에서 onPress를 계속 내려주지 않아도 됨
-> 컴포넌트 내부에서 이동 로직 처리 가능

주의점
-> 컴포넌트가 navigation 라이브러리에 의존
-> 재사용 가능한 순수 UI 컴포넌트 성격은 약해질 수 있음
```

카테고리 타일에서는 부모가 이동 함수를 넘겼고, 음식 아이템에서는 `useNavigation`을 썼다.

둘 다 가능한 방식이며, 상황에 맞게 선택하면 된다.

### 렌더링 중 navigate 호출은 피해야 한다

React 컴포넌트 함수는 렌더링할 때 실행된다.

따라서 함수 본문에서 바로 `navigate()`를 호출하면 렌더링 중에 화면 이동이 발생한다.

```jsx
function Component() {
  navigation.navigate('Somewhere'); // 렌더링 즉시 실행
}
```

이벤트가 발생했을 때 이동해야 하므로 반드시 핸들러 안에서 호출해야 한다.

```jsx
function pressHandler() {
  navigation.navigate('Somewhere');
}
```

이 원칙은 웹 React에서도 같다.

### params 이름은 다시 계약이다

이번에는 `mealId`라는 params 이름을 사용한다.

```jsx
navigation.navigate('MealDetail', {
  mealId: id,
});
```

대상 화면은 같은 이름으로 값을 읽어야 한다.

```jsx
route.params.mealId;
```

`categoryId` 때와 마찬가지로 params 이름은 보내는 쪽과 받는 쪽 사이의 계약이다.

## 정리

이번 강의에서는 `MealDetailScreen`을 추가하고, `MealItem`을 눌렀을 때 상세 화면으로 이동하도록 만들었다.

`MealItem`은 screen 컴포넌트가 아니므로 `useNavigation` 훅으로 navigation 객체를 얻었고, `navigation.navigate('MealDetail', { mealId: id })`를 통해 선택한 음식 id를 상세 화면에 전달했다.

이제 앱은 카테고리 목록, 음식 목록, 음식 상세 화면으로 이어지는 기본적인 3단계 내비게이션 구조를 갖추었다.
