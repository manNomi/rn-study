# 5. React Navigation 설치와 Native Stack 기본 설정

## 핵심 요약

- 이번 강의부터 실제 화면 이동을 위해 React Navigation을 사용한다.
- 이전 게임 앱처럼 상태와 `if` 문으로 화면을 교체할 수도 있지만, 애니메이션, 뒤로가기, 화면 히스토리까지 직접 구현하려면 번거롭다.
- React Navigation은 Expo와 React Native 앱에서 라우팅과 내비게이션을 구현할 수 있게 해주는 대표적인 서드파티 패키지다.
- 기본 패키지인 `@react-navigation/native`를 설치하고, Expo 프로젝트에서는 추가 의존성인 `react-native-screens`, `react-native-safe-area-context`도 설치한다.
- Stack Navigation을 사용하기 위해 `@react-navigation/native-stack`을 추가로 설치한다.
- `NavigationContainer`로 앱의 내비게이션 영역을 감싸고, `createNativeStackNavigator`로 Stack Navigator를 생성한다.
- `Stack.Navigator` 안에 `Stack.Screen`을 등록하면 React Navigation이 해당 화면을 관리한다.

## 학습 목표

- 상태 기반 화면 전환 대신 내비게이션 라이브러리를 사용하는 이유를 이해한다.
- React Navigation의 역할과 설치 흐름을 파악한다.
- `NavigationContainer`가 어떤 역할을 하는지 이해한다.
- Native Stack Navigator를 생성하고 화면을 등록하는 기본 패턴을 익힌다.
- `Stack.Navigator`와 `Stack.Screen`의 관계를 이해한다.
- React Navigation이 기본으로 제공하는 헤더, safe area, 화면 래퍼를 확인한다.

## 주요 내용

### 왜 React Navigation을 사용하는가?

카테고리 타일을 누르면 새 화면으로 이동해야 한다.

이 작업은 이전 게임 앱처럼 상태로 직접 구현할 수도 있다.

```text
직접 구현하는 방식
-> 선택된 카테고리 상태 저장
-> if 조건으로 다른 화면 렌더링
-> 뒤로가기 상태 직접 처리
```

이 방식이 완전히 잘못된 것은 아니다.

단순한 앱에서는 충분히 쓸 수 있다.

하지만 이번 섹션의 목표는 더 자연스러운 모바일 앱 내비게이션이다.

```text
React Navigation을 쓰는 이유
-> 화면 전환 애니메이션 제공
-> 뒤로가기 흐름 제공
-> 화면 스택 관리
-> 헤더 제공
-> 탭, 드로어 같은 패턴 지원
-> 네이티브 앱다운 사용자 경험 제공
```

이 모든 것을 직접 만들 수는 있지만 매우 번거롭다.

그래서 강의에서는 React Navigation이라는 서드파티 패키지를 사용한다.

### React Navigation이란?

React Navigation은 Expo와 React Native 앱에서 화면 이동을 구현하기 위한 라이브러리다.

이름은 React Navigation이지만, 웹 React 라우팅용이라기보다는 React Native 앱의 내비게이션을 위해 만들어진 패키지로 이해하면 된다.

```text
React Navigation
-> React Native 앱의 화면 이동 관리
-> Expo 앱에서도 쉽게 통합 가능
-> Stack, Drawer, Tabs 등 여러 내비게이션 패턴 제공
```

Expo 앱도 결국 React Native 앱이므로 React Navigation을 사용할 수 있다.

특히 Expo 프로젝트에서는 설치와 설정이 비교적 간단하다.

### 공식 문서를 보는 습관

강의에서는 React Navigation의 핵심 기능만 다룬다.

하지만 React Navigation은 매우 강력하고 옵션도 많다.

따라서 막히거나 더 깊은 사용 사례가 필요하면 공식 문서를 확인하는 것이 좋다.

```text
공식 문서에서 확인할 것
-> 설치 방법
-> Navigator 종류
-> screen options
-> params 전달
-> nested navigation
-> drawer, tab 설정
-> 플랫폼별 동작
```

강의에서 배우는 내용은 기본기를 잡기 위한 출발점이고, 실제 프로젝트에서는 문서를 함께 보는 습관이 필요하다.

### 기본 패키지 설치

먼저 React Navigation의 핵심 패키지를 설치한다.

```bash
npm install @react-navigation/native
```

이 패키지는 React Navigation의 코어 기능을 제공한다.

설치 중 경고가 나올 수 있지만, 에러 없이 완료되면 계속 진행할 수 있다.

### Expo 프로젝트의 추가 의존성

Expo 프로젝트에서는 추가 패키지를 설치해야 한다.

```bash
expo install react-native-screens react-native-safe-area-context
```

이 패키지들은 React Navigation이 네이티브 화면과 safe area를 더 잘 처리할 수 있게 도와준다.

```text
react-native-screens
-> 네이티브 screen 최적화에 사용

react-native-safe-area-context
-> 노치, 상태바, 안전 영역 처리를 위해 사용
```

Expo에서는 `expo install`을 사용하는 것이 좋다.

프로젝트의 Expo 버전에 맞는 호환 버전을 설치해주기 때문이다.

```text
npm install
-> 일반 npm 패키지 설치

expo install
-> 현재 Expo SDK와 호환되는 버전 설치
```

### NavigationContainer

React Navigation은 컴포넌트 기반 라이브러리다.

즉 설정 파일 하나로만 내비게이션을 선언하는 것이 아니라, JSX 안에서 내비게이션 관련 컴포넌트를 사용한다.

가장 먼저 `NavigationContainer`를 가져온다.

```jsx
import { NavigationContainer } from '@react-navigation/native';
```

`NavigationContainer`는 내비게이션을 사용할 앱 영역을 감싸는 루트 컨테이너다.

보통 앱 전체를 감싼다.

```jsx
<NavigationContainer>
  <CategoriesScreen />
</NavigationContainer>
```

`StatusBar`는 내비게이션 대상 화면이 아니므로 굳이 `NavigationContainer` 안에 넣을 필요가 없다.

```jsx
<>
  <StatusBar style="dark" />
  <NavigationContainer>
    {/* navigators and screens */}
  </NavigationContainer>
</>
```

이 단계만으로는 화면 이동이 동작하지 않는다.

`NavigationContainer`는 내비게이션 상태를 관리하는 기반일 뿐이고, 실제 화면 이동 방식은 Navigator가 담당한다.

### Navigator가 필요한 이유

React Navigation에는 여러 종류의 Navigator가 있다.

Navigator는 화면 전환 방식과 UI 패턴을 구현한다.

```text
Navigator 예시
-> Stack Navigator
-> Native Stack Navigator
-> Drawer Navigator
-> Bottom Tabs Navigator
-> Material Top Tabs Navigator
```

각 Navigator는 서로 다른 내비게이션 경험을 제공한다.

```text
Stack
-> 화면을 위로 쌓고 뒤로가기 가능

Drawer
-> 옆에서 열리는 메뉴

Tabs
-> 탭으로 주요 화면 전환
```

이번 강의에서는 가장 기본적인 Stack Navigation부터 시작한다.

### Native Stack 설치

Stack Navigation을 사용하기 위해 native stack 패키지를 설치한다.

```bash
expo install @react-navigation/native-stack
```

강의에서는 `expo install`을 사용하지만, 내부적으로는 npm 설치를 수행한다.

설치 후 `createNativeStackNavigator`를 가져온다.

```jsx
import { createNativeStackNavigator } from '@react-navigation/native-stack';
```

### Stack 생성

`createNativeStackNavigator` 함수를 호출해 Stack 객체를 만든다.

```jsx
const Stack = createNativeStackNavigator();
```

이 코드는 보통 컴포넌트 바깥에 둔다.

Stack 객체는 Navigator와 Screen 컴포넌트를 제공한다.

```text
Stack.Navigator
-> stack navigation 영역을 정의

Stack.Screen
-> stack 안에서 관리할 화면을 등록
```

이름은 `Stack`이 아니어도 되지만, 스택 화면 묶음을 나타내므로 `Stack`이라는 이름이 자연스럽다.

### Stack.Navigator와 Stack.Screen

`NavigationContainer` 안에 `Stack.Navigator`를 넣고, 그 안에 `Stack.Screen`을 등록한다.

```jsx
<NavigationContainer>
  <Stack.Navigator>
    <Stack.Screen
      name="Categories"
      component={CategoriesScreen}
    />
  </Stack.Navigator>
</NavigationContainer>
```

`Stack.Screen`은 Navigator가 관리할 하나의 화면을 등록한다.

여기서는 아직 화면이 하나뿐이므로 `CategoriesScreen`만 등록한다.

### name prop

`Stack.Screen`의 `name`은 화면의 고유 식별자다.

```jsx
<Stack.Screen name="Categories" component={CategoriesScreen} />
```

이 이름은 나중에 다른 화면으로 이동할 때 사용된다.

예를 들어 카테고리 화면에서 음식 목록 화면으로 이동할 때는 등록된 screen name을 기준으로 이동한다.

```text
name의 역할
-> 화면의 고유 식별자
-> navigate 호출 시 대상 이름으로 사용
-> 헤더 제목의 기본값으로도 사용될 수 있음
```

### component prop

`component` prop에는 렌더링할 화면 컴포넌트를 전달한다.

중요한 점은 JSX 요소가 아니라 컴포넌트 함수 자체를 전달한다는 것이다.

```jsx
component={CategoriesScreen}
```

다음처럼 쓰지 않는다.

```jsx
component={<CategoriesScreen />}
```

React Navigation이 직접 이 컴포넌트를 화면으로 렌더링하고, 필요한 navigation 관련 props를 주입해야 하기 때문이다.

### App.js 기본 형태

설정이 끝난 `App.js`는 대략 다음 구조가 된다.

```jsx
import { StatusBar } from 'expo-status-bar';
import { NavigationContainer } from '@react-navigation/native';
import { createNativeStackNavigator } from '@react-navigation/native-stack';

import CategoriesScreen from './screens/CategoriesScreen';

const Stack = createNativeStackNavigator();

export default function App() {
  return (
    <>
      <StatusBar style="dark" />
      <NavigationContainer>
        <Stack.Navigator>
          <Stack.Screen
            name="Categories"
            component={CategoriesScreen}
          />
        </Stack.Navigator>
      </NavigationContainer>
    </>
  );
}
```

`StatusBar`와 `NavigationContainer`가 형제 요소이므로 `Fragment`로 감싼다.

### 기본으로 생기는 변화

이 설정을 적용하면 앱 화면이 조금 달라진다.

```text
변화
-> 기존 배경색이 사라짐
-> 상단에 기본 헤더가 생김
-> 기본 화면 배경이 적용됨
-> safe area가 자동으로 고려됨
```

React Navigation이 기본 레이아웃을 제공하기 때문이다.

즉 우리가 직접 `SafeAreaView`를 감싸지 않아도, Navigator가 화면이 노치나 상태바와 충돌하지 않도록 기본 처리를 해준다.

이전 `StatusBar` 스타일이 밝은 색이었다면, 기본 헤더와 배경에서는 잘 안 보일 수 있다.

이 경우 상태바를 어두운 스타일로 바꿀 수 있다.

```jsx
<StatusBar style="dark" />
```

### 아직 화면 이동은 없다

현재는 `CategoriesScreen` 하나만 등록되어 있다.

따라서 내비게이션 구조는 준비되었지만, 실제로 이동할 두 번째 화면은 아직 없다.

```text
현재 상태
-> NavigationContainer 설정 완료
-> Native Stack 설정 완료
-> CategoriesScreen 등록 완료
-> 아직 이동할 대상 화면 없음
```

다음 단계에서는 두 번째 화면을 추가하고, 카테고리 타일을 눌렀을 때 새 화면으로 이동하도록 만든다.

## 프론트엔드 개발자 관점에서의 핵심 차이

### React Router와 비슷하지만 JSX 구조가 다르다

웹 React Router에서도 라우터 루트와 라우트 선언이 있다.

```jsx
<BrowserRouter>
  <Routes>
    <Route path="/" element={<HomePage />} />
  </Routes>
</BrowserRouter>
```

React Navigation도 비슷하게 루트 컨테이너와 화면 선언이 있다.

```jsx
<NavigationContainer>
  <Stack.Navigator>
    <Stack.Screen name="Categories" component={CategoriesScreen} />
  </Stack.Navigator>
</NavigationContainer>
```

다만 React Native에서는 URL path보다 screen name이 중심이다.

```text
React Router
-> path 기반
-> URL이 중요

React Navigation
-> screen name 기반
-> 화면 스택이 중요
```

### component prop에 JSX를 넣지 않는다

React Router의 `element`는 JSX 요소를 받는다.

```jsx
<Route element={<HomePage />} />
```

React Navigation의 `component`는 컴포넌트 함수 자체를 받는다.

```jsx
<Stack.Screen component={CategoriesScreen} />
```

이 차이를 헷갈리기 쉽다.

React Navigation이 화면 컴포넌트를 직접 생성하면서 navigation props를 주입해야 하므로 함수 참조를 넘긴다.

### Navigator는 UI까지 제공한다

웹 라우터는 보통 URL과 컴포넌트 매핑만 담당한다.

헤더나 safe area, 전환 애니메이션은 개발자가 별도로 구성하는 경우가 많다.

React Navigation의 Stack Navigator는 기본 헤더와 네이티브스러운 화면 전환 경험까지 제공한다.

```text
Stack Navigator가 기본 제공하는 것
-> 화면 관리
-> 기본 헤더
-> safe area 처리
-> 전환 애니메이션
-> 뒤로가기 기반 구조
```

그래서 React Native에서는 내비게이션 라이브러리가 앱의 화면 레이아웃에도 영향을 준다.

### Expo install과 호환성

웹 프로젝트에서는 보통 `npm install`이나 `pnpm add`로 패키지를 설치한다.

Expo 프로젝트에서는 네이티브 의존성과 Expo SDK 호환성이 중요하다.

```text
Expo 프로젝트
-> expo install 권장
-> 현재 SDK와 맞는 버전 설치
-> 네이티브 의존성 충돌 가능성 감소
```

React Navigation 자체는 `npm install @react-navigation/native`로 설치하지만, Expo 관련 네이티브 의존성은 `expo install`로 맞추는 흐름을 기억해두면 좋다.

## 정리

이번 강의에서는 React Navigation을 설치하고 Native Stack Navigator의 기본 구조를 만들었다.

`NavigationContainer`는 앱의 내비게이션 상태를 관리하는 루트이고, `Stack.Navigator`와 `Stack.Screen`은 어떤 화면들이 스택 내비게이션에 포함될지 정의한다.

아직 실제 화면 이동은 없지만, 이제 카테고리 타일을 눌렀을 때 새 화면으로 이동할 수 있는 기반이 마련되었다.
