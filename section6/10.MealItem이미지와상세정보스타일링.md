# 10. MealItem 이미지와 상세 정보 스타일링

## 핵심 요약

- 이번 강의에서는 `MealItem`을 단순 텍스트가 아니라 이미지와 메타 정보를 포함한 카드형 UI로 만든다.
- 음식 아이템은 나중에 상세 화면으로 이동해야 하므로 `Pressable`로 감싸 터치 가능한 요소로 만든다.
- 더미 데이터의 `imageUrl`을 사용해 웹 이미지를 로드한다.
- 웹 이미지는 로컬 이미지와 달리 크기를 추론할 수 없으므로 `width`와 `height` 스타일을 명시해야 한다.
- `duration`, `complexity`, `affordability` 같은 추가 정보를 props로 전달해 음식 카드에 출력한다.
- 여러 props를 한 번에 전달하기 위해 객체를 만들고 spread syntax로 컴포넌트에 전달한다.
- iOS shadow와 `overflow: hidden` 충돌 문제를 피하기 위해 외부 컨테이너와 내부 컨테이너를 나누어 스타일링한다.
- Android는 `android_ripple`, iOS는 `pressed` 상태의 opacity로 터치 피드백을 준다.

## 학습 목표

- React Native에서 원격 이미지를 로드하는 방법을 익힌다.
- 원격 이미지에 크기 스타일이 필요한 이유를 이해한다.
- 목록 아이템 컴포넌트에 여러 데이터 prop을 전달하는 패턴을 익힌다.
- JSX spread syntax로 props를 전달하는 방법을 복습한다.
- 카드형 UI에서 외부 컨테이너와 내부 컨테이너를 분리하는 이유를 이해한다.
- 그림자, 둥근 모서리, overflow, Pressable 피드백이 서로 어떻게 영향을 주는지 이해한다.

## 주요 내용

### MealItem을 카드형 UI로 확장하기

이전 강의의 `MealItem`은 음식 제목만 보여주는 단순한 컴포넌트였다.

```jsx
function MealItem({ title }) {
  return (
    <View>
      <Text>{title}</Text>
    </View>
  );
}
```

이번 강의에서는 이 컴포넌트를 더 실제 앱다운 카드 UI로 확장한다.

```text
MealItem에 추가할 것
-> 음식 이미지
-> 음식 제목
-> 조리 시간
-> 난이도
-> 가격 수준
-> 터치 피드백
-> 카드 스타일
```

또한 음식 아이템은 나중에 상세 화면으로 이동해야 하므로 눌릴 수 있는 요소가 되어야 한다.

그래서 `Pressable`을 추가한다.

### 기본 구조

`MealItem`은 대략 다음 구조를 가진다.

```jsx
<View>
  <Pressable>
    <View>
      <Image />
      <Text>{title}</Text>
    </View>
    <View>
      <Text>{duration}m</Text>
      <Text>{complexity}</Text>
      <Text>{affordability}</Text>
    </View>
  </Pressable>
</View>
```

이후 shadow와 overflow 문제를 해결하기 위해 내부 컨테이너를 하나 더 감싸게 된다.

```text
외부 View
-> 카드의 margin, shadow, 배경 등 외곽 스타일

Pressable
-> 터치 가능한 영역

내부 View
-> 둥근 모서리와 overflow hidden 처리

Image/Text/Details
-> 실제 콘텐츠
```

React Native에서는 이런 중첩이 꽤 흔하다.

CSS 하나로 처리하던 웹과 달리, shadow와 clipping, ripple, pressed 효과를 동시에 만족시키려면 컨테이너를 역할별로 나누는 편이 안정적이다.

### 웹 이미지 로드하기

더미 데이터의 음식 객체에는 이미지 URL이 들어 있다.

```text
Meal
-> imageUrl
```

React Native에서 로컬 이미지를 사용할 때는 `require()`를 사용했다.

```jsx
<Image source={require('../assets/images/success.png')} />
```

웹 이미지를 사용할 때는 `source`에 객체를 전달하고, 그 안에 `uri` 키를 넣는다.

```jsx
<Image source={{ uri: imageUrl }} />
```

즉 원격 이미지의 기본 형태는 다음과 같다.

```jsx
<Image
  source={{ uri: imageUrl }}
  style={styles.image}
/>
```

### 원격 이미지에는 크기가 필요하다

로컬 이미지는 React Native가 파일을 확인해서 기본 크기를 어느 정도 추론할 수 있다.

물론 실제 앱에서는 로컬 이미지도 스타일로 크기를 조정하는 경우가 많다.

하지만 웹 이미지는 React Native가 미리 파일 크기를 알 수 없다.

따라서 원격 이미지를 표시하려면 `width`와 `height`를 명시해야 한다.

```jsx
image: {
  width: '100%',
  height: 200,
}
```

```text
로컬 이미지
-> 파일을 기준으로 기본 크기를 추론할 수 있음

원격 이미지
-> URL만 있으므로 기본 크기 추론 불가
-> width, height 스타일 필요
```

이번 강의에서는 이미지가 카드 너비를 꽉 채우고, 높이는 200픽셀을 차지하도록 설정한다.

### 제목 스타일링

음식 제목에는 별도 스타일을 적용한다.

```jsx
title: {
  fontWeight: 'bold',
  textAlign: 'center',
  fontSize: 18,
  margin: 8,
}
```

```jsx
<Text style={styles.title}>{title}</Text>
```

`margin: 8`을 추가해 이미지와 상세 정보 사이에서 제목 주변에 여백을 만든다.

텍스트 스타일은 `Text` 컴포넌트에 직접 적용해야 한다.

### 추가 정보 출력

더미 데이터의 `Meal` 모델에는 다음 정보가 있다.

```text
duration
-> 조리 시간, 분 단위 숫자

complexity
-> 난이도 문자열

affordability
-> 가격 수준 문자열
```

`MealItem`은 이 값을 props로 받는다.

```jsx
function MealItem({
  title,
  imageUrl,
  duration,
  complexity,
  affordability,
}) {
  ...
}
```

출력할 때는 다음처럼 표시한다.

```jsx
<Text>{duration}m</Text>
<Text>{complexity.toUpperCase()}</Text>
<Text>{affordability.toUpperCase()}</Text>
```

`duration`은 숫자이므로 `m`을 붙여 분 단위임을 나타낸다.

`complexity`와 `affordability`는 문자열이므로 JavaScript의 `toUpperCase()`를 사용해 대문자로 변환한다.

```text
simple -> SIMPLE
affordable -> AFFORDABLE
```

### MealsOverviewScreen에서 여러 props 전달

`MealItem`에 전달할 props가 많아지면 JSX가 길어진다.

```jsx
<MealItem
  title={itemData.item.title}
  imageUrl={itemData.item.imageUrl}
  duration={itemData.item.duration}
  complexity={itemData.item.complexity}
  affordability={itemData.item.affordability}
/>
```

강의에서는 먼저 현재 아이템을 변수로 빼고, props 객체를 만든다.

```jsx
function renderMealItem(itemData) {
  const item = itemData.item;

  const mealItemProps = {
    title: item.title,
    imageUrl: item.imageUrl,
    duration: item.duration,
    complexity: item.complexity,
    affordability: item.affordability,
  };

  return <MealItem {...mealItemProps} />;
}
```

`{...mealItemProps}`는 객체의 프로퍼티들을 컴포넌트 props로 펼쳐서 전달한다.

```jsx
<MealItem {...mealItemProps} />
```

위 코드는 다음과 같은 의미다.

```jsx
<MealItem
  title={mealItemProps.title}
  imageUrl={mealItemProps.imageUrl}
  duration={mealItemProps.duration}
  complexity={mealItemProps.complexity}
  affordability={mealItemProps.affordability}
/>
```

React에서 자주 사용하는 표준 JavaScript/JSX 패턴이다.

### 카드 외부 스타일

외부 컨테이너에는 카드처럼 보이도록 margin, radius, shadow, 배경색을 추가한다.

```jsx
mealItem: {
  margin: 16,
  borderRadius: 8,
  backgroundColor: 'white',
  elevation: 4,
  shadowColor: 'black',
  shadowOpacity: 0.35,
  shadowOffset: { width: 0, height: 2 },
  shadowRadius: 16,
  overflow: Platform.OS === 'android' ? 'hidden' : 'visible',
}
```

`elevation`은 Android 그림자다.

iOS에서는 `shadowColor`, `shadowOpacity`, `shadowOffset`, `shadowRadius`가 필요하다.

이전 `CategoryGridTile`에서 사용했던 shadow 스타일과 비슷하다.

이런 스타일이 여러 곳에서 반복된다면, 나중에는 재사용 가능한 카드 컴포넌트나 공통 스타일로 추출할 수도 있다.

### 상세 정보 영역 스타일

조리 시간, 난이도, 가격 수준은 한 줄에 나란히 배치한다.

```jsx
details: {
  flexDirection: 'row',
  alignItems: 'center',
  justifyContent: 'center',
  padding: 8,
}
```

`flexDirection: 'row'`를 설정하면 자식 `Text`들이 가로로 배치된다.

```text
duration | complexity | affordability
```

각 항목에는 작은 폰트와 좌우 여백을 준다.

```jsx
detailItem: {
  marginHorizontal: 4,
  fontSize: 12,
}
```

```jsx
<Text style={styles.detailItem}>{duration}m</Text>
<Text style={styles.detailItem}>{complexity.toUpperCase()}</Text>
<Text style={styles.detailItem}>{affordability.toUpperCase()}</Text>
```

### overflow와 shadow 문제

둥근 모서리 안에서 이미지가 삐져나가지 않게 하려면 `overflow: 'hidden'`이 필요하다.

하지만 iOS에서는 외부 컨테이너에 `overflow: 'hidden'`을 주면 shadow가 잘린다.

이 문제는 이전 카테고리 타일에서도 경험했다.

```text
외부 컨테이너에 overflow hidden
-> 둥근 모서리 clipping은 됨
-> iOS shadow가 잘림
```

해결책은 shadow가 있는 외부 컨테이너와 clipping을 담당하는 내부 컨테이너를 분리하는 것이다.

```jsx
<View style={styles.mealItem}>
  <Pressable>
    <View style={styles.innerContainer}>
      {/* image, title, details */}
    </View>
  </Pressable>
</View>
```

외부 컨테이너는 shadow를 담당한다.

내부 컨테이너는 둥근 모서리와 overflow clipping을 담당한다.

```jsx
innerContainer: {
  borderRadius: 8,
  overflow: 'hidden',
}
```

이렇게 하면 iOS에서도 shadow는 유지되고, 이미지도 둥근 모서리 안에 잘린다.

### Pressable 피드백

`MealItem`은 나중에 상세 화면으로 이동할 항목이므로 `Pressable`로 감싼다.

Android에서는 `android_ripple`을 사용한다.

```jsx
<Pressable android_ripple={{ color: '#ccc' }}>
  ...
</Pressable>
```

iOS에서는 `Pressable`의 `pressed` 상태를 이용해 opacity를 줄인다.

```jsx
<Pressable
  android_ripple={{ color: '#ccc' }}
  style={({ pressed }) =>
    pressed ? styles.buttonPressed : null
  }
>
  ...
</Pressable>
```

```jsx
buttonPressed: {
  opacity: 0.5,
}
```

Android에서는 ripple과 opacity가 함께 적용될 수도 있다.

원한다면 `Platform` API로 iOS에서만 opacity를 적용하도록 분기할 수 있지만, 강의에서는 두 효과가 함께 있는 것도 괜찮다고 보고 그대로 둔다.

### 현재까지의 MealItem 구조

완성된 구조는 대략 다음과 같다.

```jsx
<View style={styles.mealItem}>
  <Pressable
    android_ripple={{ color: '#ccc' }}
    style={({ pressed }) =>
      pressed ? styles.buttonPressed : null
    }
  >
    <View style={styles.innerContainer}>
      <View>
        <Image source={{ uri: imageUrl }} style={styles.image} />
        <Text style={styles.title}>{title}</Text>
      </View>

      <View style={styles.details}>
        <Text style={styles.detailItem}>{duration}m</Text>
        <Text style={styles.detailItem}>{complexity.toUpperCase()}</Text>
        <Text style={styles.detailItem}>{affordability.toUpperCase()}</Text>
      </View>
    </View>
  </Pressable>
</View>
```

이제 음식 목록은 단순 텍스트 리스트가 아니라 실제 카드형 목록처럼 보인다.

## 예시 코드

### MealItem

```jsx
import {
  Image,
  Platform,
  Pressable,
  StyleSheet,
  Text,
  View,
} from 'react-native';

function MealItem({
  title,
  imageUrl,
  duration,
  complexity,
  affordability,
}) {
  return (
    <View style={styles.mealItem}>
      <Pressable
        android_ripple={{ color: '#ccc' }}
        style={({ pressed }) =>
          pressed ? styles.buttonPressed : null
        }
      >
        <View style={styles.innerContainer}>
          <View>
            <Image source={{ uri: imageUrl }} style={styles.image} />
            <Text style={styles.title}>{title}</Text>
          </View>

          <View style={styles.details}>
            <Text style={styles.detailItem}>{duration}m</Text>
            <Text style={styles.detailItem}>
              {complexity.toUpperCase()}
            </Text>
            <Text style={styles.detailItem}>
              {affordability.toUpperCase()}
            </Text>
          </View>
        </View>
      </Pressable>
    </View>
  );
}

export default MealItem;

const styles = StyleSheet.create({
  mealItem: {
    margin: 16,
    borderRadius: 8,
    backgroundColor: 'white',
    elevation: 4,
    shadowColor: 'black',
    shadowOpacity: 0.35,
    shadowOffset: { width: 0, height: 2 },
    shadowRadius: 16,
    overflow: Platform.OS === 'android' ? 'hidden' : 'visible',
  },
  buttonPressed: {
    opacity: 0.5,
  },
  innerContainer: {
    borderRadius: 8,
    overflow: 'hidden',
  },
  image: {
    width: '100%',
    height: 200,
  },
  title: {
    fontWeight: 'bold',
    textAlign: 'center',
    fontSize: 18,
    margin: 8,
  },
  details: {
    flexDirection: 'row',
    alignItems: 'center',
    justifyContent: 'center',

  },
  detailItem: {
    marginHorizontal: 4,
    fontSize: 12,
  },
});
```

## 프론트엔드 개발자 관점에서의 핵심 차이

### 웹 이미지와 React Native Image의 차이

웹에서는 이미지 URL을 `src`에 바로 넣는다.

```html
<img src="https://example.com/image.jpg" />
```

React Native에서는 `Image`의 `source` prop에 `{ uri }` 객체를 넣는다.

```jsx
<Image source={{ uri: imageUrl }} />
```

그리고 원격 이미지는 크기를 명시해야 한다.

```jsx
style={{ width: '100%', height: 200 }}
```

### 카드 UI는 컨테이너 역할 분리가 중요하다

웹에서는 하나의 카드 div에 `border-radius`, `overflow`, `box-shadow`를 같이 주는 경우가 많다.

React Native, 특히 iOS에서는 shadow와 overflow가 충돌할 수 있다.

```text
외부 컨테이너
-> shadow
-> margin
-> background

내부 컨테이너
-> borderRadius
-> overflow hidden
-> 이미지 clipping
```

이 역할 분리는 React Native 카드 UI를 만들 때 자주 쓰이는 패턴이다.

### props가 많아질 때 spread 사용

React에서 props가 많아질 때 객체로 묶고 spread하는 패턴은 흔하다.

```jsx
<MealItem {...mealItemProps} />
```

다만 너무 많은 값을 무분별하게 spread하면 어떤 props가 전달되는지 한눈에 보기 어려울 수 있다.

이번처럼 한 아이템 컴포넌트에 필요한 필드를 명확히 모아 전달하는 정도는 읽기 좋은 편이다.

### 다음 내비게이션 단계의 준비

`MealItem`은 이미 `Pressable`로 만들어졌다.

아직 `onPress`로 상세 화면 이동을 연결하지는 않았지만, 구조상 다음 단계에서 자연스럽게 붙일 수 있다.

```text
현재
-> MealItem 카드 UI 완성
-> Pressable 준비

다음 단계
-> MealItem을 누르면 MealDetailScreen으로 이동
-> mealId 전달
```

즉 이번 강의는 다시 내비게이션으로 돌아가기 위한 UI 기반 작업이다.

## 정리

이번 강의에서는 `MealItem`을 단순 텍스트에서 이미지와 상세 정보를 포함한 카드형 리스트 아이템으로 발전시켰다.

원격 이미지 로딩, 명시적 이미지 크기, props 확장, spread syntax, 그림자와 overflow 분리, Android/iOS 터치 피드백까지 다루면서 실제 모바일 목록 아이템에 가까운 UI를 만들었다.

이제 음식 목록은 시각적으로 충분히 갖춰졌고, 다음 단계에서는 이 아이템을 눌렀을 때 음식 상세 화면으로 이동하는 내비게이션을 이어서 구현할 수 있다.
