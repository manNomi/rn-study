# 22. Drawer 스타일 튜닝과 메뉴 아이콘

## 핵심 요약

- 중첩된 Drawer Navigator도 이전에 배운 Drawer 옵션으로 자유롭게 스타일링할 수 있다.
- Drawer 전체 배경색은 `drawerContentStyle`로 설정한다.
- 선택되지 않은 메뉴 항목의 텍스트와 아이콘 색상은 `drawerInactiveTintColor`로 설정한다.
- 선택된 메뉴 항목의 텍스트와 아이콘 색상은 `drawerActiveTintColor`로 설정한다.
- 선택된 메뉴 항목의 배경색은 `drawerActiveBackgroundColor`로 설정한다.
- 모든 Drawer 화면에 공통 적용할 색상 옵션은 `Drawer.Navigator`의 `screenOptions`에 둔다.
- 화면마다 달라야 하는 메뉴 아이콘은 각 `Drawer.Screen`의 `options.drawerIcon`에 둔다.
- `drawerIcon` 함수가 제공하는 `color`, `size` 값을 `Ionicons`에 그대로 넘기면 활성 상태에 따른 색상과 크기를 React Navigation에 맡길 수 있다.

## 학습 목표

- Meals 앱의 Drawer UI를 앱의 갈색 테마에 맞게 조정한다.
- `drawerContentStyle`, `drawerInactiveTintColor`, `drawerActiveTintColor`, `drawerActiveBackgroundColor`의 역할을 이해한다.
- 공통 Drawer 스타일과 화면별 Drawer 옵션을 구분한다.
- `Ionicons`를 사용해 Drawer 메뉴 항목에 아이콘을 추가하는 방법을 익힌다.
- 메뉴 아이콘이 활성 상태 색상을 자동으로 따르게 만드는 패턴을 이해한다.

## 주요 내용

### Drawer를 앱 테마에 맞게 다듬기

이전 강의에서는 Meals 앱에 Drawer Navigator를 중첩했다.

이제 기능적으로는 `Categories`와 `Favorites` 사이를 이동할 수 있지만, Drawer의 기본 스타일은 앱의 기존 갈색 테마와 어울리지 않는다.

따라서 Drawer 전체 배경과 메뉴 항목의 활성/비활성 색상을 조정한다.

```text
목표
-> Drawer 전체 배경을 어두운 갈색으로 변경
-> 선택되지 않은 항목은 흰색으로 표시
-> 선택된 항목은 밝은 배경과 어두운 텍스트로 강조
-> 각 메뉴에 의미 있는 아이콘 추가
```

### 공통 Drawer 스타일은 screenOptions에 둔다

이번 스타일은 `Categories`와 `Favorites` 화면 모두에 적용되어야 한다.

따라서 각 `Drawer.Screen`의 `options`에 반복해서 넣지 않고, `Drawer.Navigator`의 `screenOptions`에 둔다.

```jsx
<Drawer.Navigator
  screenOptions={{
    // common drawer options
  }}
>
  ...
</Drawer.Navigator>
```

`screenOptions`는 해당 Drawer Navigator에 속한 모든 screen의 기본 옵션으로 사용된다.

### drawerContentStyle

Drawer 패널 전체의 배경색은 `drawerContentStyle`로 설정한다.

```jsx
drawerContentStyle: {
  backgroundColor: '#351401',
}
```

이 옵션을 적용하면 Drawer가 열렸을 때 보이는 메뉴 영역 전체가 어두운 갈색으로 바뀐다.

```text
drawerContentStyle
-> Drawer 내부 콘텐츠 영역 스타일
-> 메뉴 목록이 놓이는 패널 배경 조정
```

이전 강의에서 `sceneContainerStyle`은 화면 콘텐츠 배경을 조정하는 데 사용했다.

이번 `drawerContentStyle`은 화면이 아니라 Drawer 메뉴 패널 자체를 꾸미는 옵션이다.

### 비활성 메뉴 색상

Drawer 배경을 어둡게 만들면 기본 텍스트 색상이 잘 보이지 않을 수 있다.

선택되지 않은 항목의 텍스트와 아이콘 색상은 `drawerInactiveTintColor`로 설정한다.

```jsx
drawerInactiveTintColor: 'white'
```

이제 현재 선택되지 않은 메뉴 항목도 어두운 Drawer 배경 위에서 읽기 쉬워진다.

```text
drawerInactiveTintColor
-> 선택되지 않은 항목의 텍스트 색상
-> 선택되지 않은 항목의 아이콘 색상
```

### 활성 메뉴 색상

현재 선택된 항목은 배경색과 텍스트 색상을 함께 조정한다.

```jsx
drawerActiveBackgroundColor: '#e4baa1',
drawerActiveTintColor: '#351401',
```

`drawerActiveBackgroundColor`는 선택된 메뉴 항목의 배경색이다.

`drawerActiveTintColor`는 선택된 메뉴 항목의 텍스트와 아이콘 색상이다.

```text
drawerActiveBackgroundColor
-> 선택된 항목의 배경색

drawerActiveTintColor
-> 선택된 항목의 텍스트/아이콘 색상
```

강의에서는 처음에 다소 진한 색을 시도한 뒤, 더 밝은 색상으로 바꿔 선택 상태가 더 잘 보이도록 조정한다.

색상 값 자체는 정답이 아니다.

앱의 전체 톤, 대비, 가독성에 맞게 조정하면 된다.

### Drawer 공통 스타일 예시

최종적으로 Drawer의 공통 스타일은 다음처럼 구성할 수 있다.

```jsx
<Drawer.Navigator
  screenOptions={{
    headerStyle: { backgroundColor: '#351401' },
    headerTintColor: 'white',
    sceneContainerStyle: { backgroundColor: '#3f2f25' },
    drawerContentStyle: { backgroundColor: '#351401' },
    drawerInactiveTintColor: 'white',
    drawerActiveTintColor: '#351401',
    drawerActiveBackgroundColor: '#e4baa1',
  }}
>
  ...
</Drawer.Navigator>
```

이 설정은 Drawer에 속한 모든 화면에 공통으로 적용된다.

### 화면별 아이콘은 options에 둔다

Drawer 전체 색상은 공통 옵션으로 둘 수 있지만, 메뉴 아이콘은 화면마다 달라야 한다.

```text
Categories
-> list 아이콘

Favorites
-> star 아이콘
```

따라서 아이콘 설정은 각 `Drawer.Screen`의 `options`에 둔다.

```jsx
<Drawer.Screen
  name="Categories"
  component={CategoriesScreen}
  options={{
    drawerIcon: ...
  }}
/>
```

### Ionicons 사용

아이콘은 이전 강의에서 사용했던 `Ionicons`를 활용한다.

```jsx
import { Ionicons } from '@expo/vector-icons';
```

`Ionicons`는 컴포넌트처럼 사용할 수 있다.

```jsx
<Ionicons
  name="list"
  color={color}
  size={size}
/>
```

아이콘 이름은 Expo vector icons 문서에서 찾아 원하는 것으로 바꿀 수 있다.

강의에서는 카테고리 화면에 `list`, 즐겨찾기 화면에 `star`를 사용한다.

### drawerIcon 함수

`drawerIcon`은 JSX를 반환하는 함수다.

React Navigation은 이 함수에 몇 가지 값을 자동으로 전달한다.

```jsx
drawerIcon: ({ color, size }) => (
  <Ionicons
    name="list"
    color={color}
    size={size}
  />
)
```

여기서 `color`와 `size`는 Drawer Navigator가 현재 상태에 맞게 계산한 값이다.

```text
color
-> 현재 항목이 선택되었는지에 따라 달라짐
-> active/inactive tint color가 반영됨

size
-> Drawer 메뉴에 맞는 아이콘 크기
```

따라서 아이콘에 색상을 하드코딩하지 않고 `color={color}`로 넘기는 것이 좋다.

이렇게 하면 메뉴 항목이 선택될 때 아이콘 색상도 텍스트 색상과 함께 자연스럽게 바뀐다.

### Categories 아이콘

카테고리 목록 화면에는 `list` 아이콘을 사용한다.

```jsx
<Drawer.Screen
  name="Categories"
  component={CategoriesScreen}
  options={{
    title: 'All Categories',
    drawerIcon: ({ color, size }) => (
      <Ionicons
        name="list"
        color={color}
        size={size}
      />
    ),
  }}
/>
```

`title`은 헤더와 Drawer 항목에 표시될 이름으로 사용될 수 있다.

필요하다면 `drawerLabel`을 따로 설정해 Drawer에 보이는 이름만 별도로 바꿀 수도 있다.

### Favorites 아이콘

즐겨찾기 화면에는 의미상 `star` 아이콘이 잘 어울린다.

```jsx
<Drawer.Screen
  name="Favorites"
  component={FavoritesScreen}
  options={{
    drawerIcon: ({ color, size }) => (
      <Ionicons
        name="star"
        color={color}
        size={size}
      />
    ),
  }}
/>
```

이제 Drawer 메뉴에서 `Categories`와 `Favorites`가 각각 다른 아이콘과 함께 표시된다.

## 예시 코드

### App.js의 DrawerNavigator

```jsx
import { Ionicons } from '@expo/vector-icons';

function DrawerNavigator() {
  return (
    <Drawer.Navigator
      screenOptions={{
        headerStyle: { backgroundColor: '#351401' },
        headerTintColor: 'white',
        sceneContainerStyle: { backgroundColor: '#3f2f25' },
        drawerContentStyle: { backgroundColor: '#351401' },
        drawerInactiveTintColor: 'white',
        drawerActiveTintColor: '#351401',
        drawerActiveBackgroundColor: '#e4baa1',
      }}
    >
      <Drawer.Screen
        name="Categories"
        component={CategoriesScreen}
        options={{
          title: 'All Categories',
          drawerIcon: ({ color, size }) => (
            <Ionicons
              name="list"
              color={color}
              size={size}
            />
          ),
        }}
      />
      <Drawer.Screen
        name="Favorites"
        component={FavoritesScreen}
        options={{
          drawerIcon: ({ color, size }) => (
            <Ionicons
              name="star"
              color={color}
              size={size}
            />
          ),
        }}
      />
    </Drawer.Navigator>
  );
}
```

## 프론트엔드 개발자 관점에서의 핵심 차이

### Drawer 패널과 화면 콘텐츠는 다른 영역이다

웹에서도 사이드바 배경과 메인 콘텐츠 배경은 서로 다른 영역이다.

React Navigation Drawer에서도 마찬가지다.

```text
sceneContainerStyle
-> 화면 콘텐츠 영역 배경

drawerContentStyle
-> Drawer 메뉴 패널 배경
```

둘을 구분해야 원하는 영역에 정확히 스타일이 적용된다.

### 활성 상태 스타일은 직접 관리하지 않는다

Drawer 메뉴에서 어떤 항목이 활성 상태인지 직접 상태로 관리하지 않아도 된다.

React Navigation이 현재 선택된 screen을 알고 있고, active/inactive 색상을 자동으로 적용한다.

```text
drawerActiveTintColor
drawerActiveBackgroundColor
drawerInactiveTintColor
```

개발자는 색상 규칙만 지정하면 된다.

### 아이콘은 화면별 의미를 전달한다

Drawer 메뉴는 앱의 주요 섹션을 보여준다.

따라서 아이콘은 단순 장식이 아니라 메뉴 의미를 빠르게 전달하는 역할을 한다.

```text
list
-> 카테고리 목록

star
-> 즐겨찾기
```

화면별 의미가 다르므로 아이콘은 각 `Drawer.Screen`의 `options`에 두는 것이 자연스럽다.

### 기능은 아직 다음 단계다

이번 강의에서는 Favorites 화면과 별 아이콘을 추가했지만, 실제 즐겨찾기 상태는 아직 구현하지 않는다.

```text
현재
-> Favorites 화면 존재
-> Drawer에서 이동 가능
-> 메뉴 아이콘 표시

아직 없음
-> 즐겨찾기 추가/삭제
-> 앱 전체 상태 관리
-> Favorites 화면에 실제 데이터 표시
```

이 기능은 다음 섹션의 앱 전역 상태 관리에서 다룰 예정이다.

## 정리

이번 강의에서는 Meals 앱의 Drawer Navigator를 앱 테마에 맞게 튜닝했다.

`drawerContentStyle`로 Drawer 전체 배경을 바꾸고, `drawerInactiveTintColor`, `drawerActiveTintColor`, `drawerActiveBackgroundColor`로 메뉴 항목의 활성/비활성 색상을 조정했다.

또한 `Ionicons`와 `drawerIcon`을 사용해 `Categories`에는 목록 아이콘, `Favorites`에는 별 아이콘을 추가했다.

이제 Drawer는 기능적으로만 존재하는 메뉴가 아니라, 앱의 색상 체계와 정보 구조를 더 잘 드러내는 네비게이션 UI가 되었다.
