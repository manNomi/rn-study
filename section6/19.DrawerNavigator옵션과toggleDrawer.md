# 19. Drawer Navigator 옵션과 toggleDrawer

## 핵심 요약

- Drawer Navigator도 Stack Navigator처럼 `options`와 `screenOptions`로 헤더와 드로어 UI를 설정할 수 있다.
- `headerStyle`, `headerTintColor` 같은 헤더 옵션은 Drawer에서도 사용할 수 있다.
- `drawerLabel`, `drawerIcon`처럼 Drawer 전용 옵션도 제공된다.
- 특정 화면에만 적용되는 설정은 `Drawer.Screen`의 `options`에 둔다.
- 여러 화면에 공통으로 적용할 설정은 `Drawer.Navigator`의 `screenOptions`로 올리는 편이 좋다.
- Drawer 메뉴 항목의 활성 배경색과 텍스트 색상은 `drawerActiveBackgroundColor`, `drawerActiveTintColor`로 조정할 수 있다.
- 화면 컴포넌트는 Drawer Navigator에 등록되면 `navigation.toggleDrawer()` 같은 Drawer 전용 메서드를 사용할 수 있다.

## 학습 목표

- Drawer Navigator의 기본 UI를 커스터마이징하는 방법을 익힌다.
- 화면별 옵션과 네비게이터 공통 옵션의 차이를 이해한다.
- `drawerLabel`로 드로어 메뉴 표시 이름을 바꾸는 방법을 익힌다.
- `drawerIcon`으로 드로어 메뉴에 아이콘을 추가하는 방법을 이해한다.
- `drawerIcon` 콜백이 제공하는 `color`, `size`, `focused` 값을 이해한다.
- 화면 내부 버튼에서 `navigation.toggleDrawer()`로 드로어를 제어하는 방법을 익힌다.

## 주요 내용

### Drawer도 options로 설정할 수 있다

Drawer Navigator는 Stack Navigator와 마찬가지로 각 화면에 `options`를 줄 수 있다.

```jsx
<Drawer.Screen
  name="Welcome"
  component={WelcomeScreen}
  options={{
    headerStyle: {
      backgroundColor: '#3c0a6b',
    },
    headerTintColor: 'white',
  }}
/>
```

`headerStyle`은 헤더 배경을 바꾼다.

`headerTintColor`는 헤더 제목 색상뿐만 아니라 기본 드로어 아이콘 색상도 함께 바꾼다.

```text
headerTintColor
-> 헤더 타이틀 색상
-> 헤더 버튼/아이콘 색상
```

React Navigation이 기본 헤더와 기본 드로어 아이콘을 만들어 주기 때문에, 색상 옵션만으로도 기본 UI를 꽤 자연스럽게 바꿀 수 있다.

### Drawer 전용 옵션

Drawer Navigator에는 Stack Navigator에는 없는 Drawer 전용 옵션도 있다.

대표적으로 다음 옵션들이 있다.

```text
drawerLabel
-> 드로어 메뉴에 표시될 이름

drawerIcon
-> 드로어 메뉴 항목 옆에 표시할 아이콘

drawerActiveBackgroundColor
-> 현재 선택된 항목의 배경색

drawerActiveTintColor
-> 현재 선택된 항목의 텍스트/아이콘 색상

drawerStyle
-> 드로어 전체 컨테이너 스타일
```

강의에서는 전체 옵션을 다 외우기보다, 공식 문서를 보면서 필요한 설정을 찾는 방식을 강조한다.

React Navigation은 설정 가능한 항목이 많기 때문에, 실제 프로젝트에서는 원하는 동작을 먼저 정하고 해당 옵션을 찾아 적용하는 식으로 접근하는 편이 좋다.

### drawerLabel로 메뉴 이름 바꾸기

`Drawer.Screen`의 `name`은 화면 식별자다.

기본적으로 이 값이 드로어 메뉴에도 표시된다.

```jsx
<Drawer.Screen
  name="Welcome"
  component={WelcomeScreen}
/>
```

이 경우 드로어에는 `Welcome`이 표시된다.

메뉴에 더 자연스러운 이름을 보여주고 싶다면 `drawerLabel`을 사용한다.

```jsx
<Drawer.Screen
  name="Welcome"
  component={WelcomeScreen}
  options={{
    drawerLabel: 'Welcome Screen',
  }}
/>
```

이제 내부 screen name은 그대로 `Welcome`이지만, 드로어 메뉴에는 `Welcome Screen`이 표시된다.

```text
name
-> 네비게이션에서 사용하는 식별자

drawerLabel
-> 사용자에게 보이는 드로어 메뉴 이름
```

### 활성 항목 색상 변경

현재 선택된 드로어 항목의 배경색은 `drawerActiveBackgroundColor`로 바꿀 수 있다.

```jsx
drawerActiveBackgroundColor: '#f0e1ff'
```

현재 선택된 항목의 텍스트 색상은 `drawerActiveTintColor`로 바꾼다.

```jsx
drawerActiveTintColor: '#3c0a6b'
```

이 두 옵션을 조합하면 선택된 메뉴 항목을 브랜드 색상이나 앱 테마에 맞게 강조할 수 있다.

### 드로어 전체 스타일 변경

드로어 전체 컨테이너는 `drawerStyle`로 스타일링할 수 있다.

```jsx
drawerStyle: {
  backgroundColor: '#ccc',
}
```

이렇게 하면 드로어 패널 전체의 배경색이 바뀐다.

다만 강의에서는 회색 배경이 마음에 들지 않아 다시 제거한다.

중요한 점은 이런 식으로 드로어 패널 자체도 스타일링할 수 있다는 것이다.

### 화면별 options와 공통 screenOptions

처음에는 `WelcomeScreen`의 `options`에 여러 설정을 넣었다.

```jsx
<Drawer.Screen
  name="Welcome"
  component={WelcomeScreen}
  options={{
    headerStyle: { backgroundColor: '#3c0a6b' },
    headerTintColor: 'white',
    drawerLabel: 'Welcome Screen',
    drawerActiveBackgroundColor: '#f0e1ff',
    drawerActiveTintColor: '#3c0a6b',
  }}
/>
```

하지만 이렇게 하면 이 설정은 `WelcomeScreen`에만 적용된다.

`UserScreen`으로 이동하면 드로어 활성 색상이나 헤더 색상이 달라질 수 있다.

여러 화면에 공통으로 적용되어야 하는 설정은 `Drawer.Navigator`의 `screenOptions`로 옮기는 편이 좋다.

```jsx
<Drawer.Navigator
  screenOptions={{
    headerStyle: { backgroundColor: '#3c0a6b' },
    headerTintColor: 'white',
    drawerActiveBackgroundColor: '#f0e1ff',
    drawerActiveTintColor: '#3c0a6b',
  }}
>
  <Drawer.Screen
    name="Welcome"
    component={WelcomeScreen}
    options={{
      drawerLabel: 'Welcome Screen',
    }}
  />
  <Drawer.Screen
    name="User"
    component={UserScreen}
  />
</Drawer.Navigator>
```

이제 헤더와 드로어의 기본 색상은 모든 화면에 동일하게 적용된다.

화면마다 달라야 하는 `drawerLabel` 같은 설정만 개별 화면의 `options`에 남긴다.

```text
공통 설정
-> screenOptions

화면별 설정
-> options
```

### drawerIcon 추가

드로어 메뉴 항목에는 아이콘도 추가할 수 있다.

강의에서는 이전에 사용했던 `Ionicons`를 다시 사용한다.

```jsx
import { Ionicons } from '@expo/vector-icons';
```

`drawerIcon`은 JSX를 직접 받는 것이 아니라 JSX를 반환하는 함수를 받는다.

```jsx
options={{
  drawerIcon: ({ color, size }) => {
    return (
      <Ionicons
        name="home"
        color={color}
        size={size}
      />
    );
  },
}}
```

여기서 중요한 점은 `color`와 `size`를 React Navigation이 넘겨준다는 것이다.

```text
color
-> 현재 항목이 활성인지 아닌지에 따라 정해지는 색상

size
-> Drawer Navigator가 기대하는 아이콘 크기

focused
-> 현재 항목이 선택되었는지 나타내는 boolean
```

직접 색상을 하드코딩하지 않고 전달받은 `color`를 쓰면, 활성 상태에 따라 아이콘 색상도 자동으로 바뀐다.

### 화면별 아이콘 설정

`WelcomeScreen`에는 홈 아이콘을 넣을 수 있다.

```jsx
<Drawer.Screen
  name="Welcome"
  component={WelcomeScreen}
  options={{
    drawerLabel: 'Welcome Screen',
    drawerIcon: ({ color, size }) => (
      <Ionicons
        name="home"
        color={color}
        size={size}
      />
    ),
  }}
/>
```

`UserScreen`에는 사용자 아이콘을 넣을 수 있다.

```jsx
<Drawer.Screen
  name="User"
  component={UserScreen}
  options={{
    drawerIcon: ({ color, size }) => (
      <Ionicons
        name="person"
        color={color}
        size={size}
      />
    ),
  }}
/>
```

아이콘은 보통 화면마다 달라야 하므로 `screenOptions`보다는 각 `Drawer.Screen`의 `options`에 두는 편이 자연스럽다.

### 드로어를 코드로 열고 닫기

Drawer Navigator는 기본 헤더 아이콘을 제공한다.

사용자는 그 아이콘을 눌러 드로어를 열 수 있다.

하지만 화면 내부 버튼으로 드로어를 열어야 하는 경우도 있다.

예를 들어 `UserScreen` 안에 버튼을 하나 추가한다.

```jsx
import { Button, View } from 'react-native';
```

```jsx
function UserScreen({ navigation }) {
  function openDrawerHandler() {
    navigation.toggleDrawer();
  }

  return (
    <View>
      <Button
        title="Open Drawer"
        onPress={openDrawerHandler}
      />
    </View>
  );
}
```

Drawer Navigator에 등록된 화면 컴포넌트는 `navigation` prop을 받는다.

이 `navigation` 객체에는 Drawer 전용 메서드가 추가된다.

```text
navigation.toggleDrawer()
-> 드로어가 닫혀 있으면 열기
-> 드로어가 열려 있으면 닫기
```

Stack Navigator만 사용할 때는 없던 메서드다.

즉 어떤 네비게이터에 속한 화면인지에 따라 `navigation` 객체가 제공하는 메서드가 달라질 수 있다.

## 예시 코드

### App.js

```jsx
import { NavigationContainer } from '@react-navigation/native';
import { createDrawerNavigator } from '@react-navigation/drawer';
import { Ionicons } from '@expo/vector-icons';

import WelcomeScreen from './screens/WelcomeScreen';
import UserScreen from './screens/UserScreen';

const Drawer = createDrawerNavigator();

export default function App() {
  return (
    <NavigationContainer>
      <Drawer.Navigator
        screenOptions={{
          headerStyle: { backgroundColor: '#3c0a6b' },
          headerTintColor: 'white',
          drawerActiveBackgroundColor: '#f0e1ff',
          drawerActiveTintColor: '#3c0a6b',
        }}
      >
        <Drawer.Screen
          name="Welcome"
          component={WelcomeScreen}
          options={{
            drawerLabel: 'Welcome Screen',
            drawerIcon: ({ color, size }) => (
              <Ionicons
                name="home"
                color={color}
                size={size}
              />
            ),
          }}
        />
        <Drawer.Screen
          name="User"
          component={UserScreen}
          options={{
            drawerIcon: ({ color, size }) => (
              <Ionicons
                name="person"
                color={color}
                size={size}
              />
            ),
          }}
        />
      </Drawer.Navigator>
    </NavigationContainer>
  );
}
```

### UserScreen에서 드로어 제어

```jsx
import { Button, View } from 'react-native';

function UserScreen({ navigation }) {
  function openDrawerHandler() {
    navigation.toggleDrawer();
  }

  return (
    <View>
      <Button
        title="Open Drawer"
        onPress={openDrawerHandler}
      />
    </View>
  );
}

export default UserScreen;
```

## 프론트엔드 개발자 관점에서의 핵심 차이

### 공통 스타일은 screenOptions로 올린다

웹 프론트엔드에서도 같은 스타일을 여러 컴포넌트에 복사하면 유지보수가 어려워진다.

React Navigation도 마찬가지다.

```text
여러 화면에서 공유되는 헤더 색상
여러 화면에서 공유되는 드로어 활성 색상
여러 화면에서 공유되는 기본 드로어 스타일
```

이런 설정은 `screenOptions`로 올려 두는 편이 좋다.

개별 화면의 이름이나 아이콘처럼 화면마다 달라지는 것만 `options`에 둔다.

### drawerIcon은 상태를 직접 계산하지 않아도 된다

직접 “현재 선택된 항목인가?”를 계산해서 아이콘 색상을 바꾸지 않아도 된다.

React Navigation이 `drawerIcon` 함수에 `color`, `size`, `focused`를 넘겨준다.

따라서 아이콘 컴포넌트는 전달받은 값을 그대로 사용하면 된다.

```jsx
drawerIcon: ({ color, size }) => (
  <Ionicons name="home" color={color} size={size} />
)
```

이 패턴은 컴포넌트를 네비게이터의 상태와 자연스럽게 연결해 준다.

### navigation 객체는 네비게이터 종류에 따라 확장된다

Stack Navigator에서는 `navigate`, `goBack` 같은 동작이 중요했다.

Drawer Navigator에서는 여기에 더해 `toggleDrawer()` 같은 Drawer 전용 동작이 제공된다.

```text
Stack 화면
-> navigate
-> goBack

Drawer 화면
-> navigate
-> toggleDrawer
-> openDrawer / closeDrawer 계열 메서드
```

화면이 어떤 네비게이터 아래에 등록되어 있는지에 따라 사용할 수 있는 네비게이션 액션이 달라진다.

### 기본 UI를 커스터마이징하는 방식

Drawer Navigator는 기본 헤더, 기본 드로어 메뉴, 기본 아이콘 위치를 제공한다.

개발자는 모든 UI를 직접 만들 필요 없이 옵션으로 원하는 부분을 조정할 수 있다.

```text
React Navigation이 제공
-> 헤더
-> 메뉴 버튼
-> 드로어 패널
-> 활성 상태 처리

개발자가 설정
-> 색상
-> 라벨
-> 아이콘
-> 필요할 경우 커스텀 드로어 컴포넌트
```

이렇게 기본 동작을 활용하면서 필요한 부분만 커스터마이징하는 것이 React Navigation을 효율적으로 쓰는 방식이다.

## 정리

이번 강의에서는 Drawer Navigator의 세부 옵션을 설정하는 방법을 배웠다.

헤더 색상, 드로어 활성 항목 색상, 드로어 메뉴 라벨, 아이콘을 설정했고, 공통 옵션은 `screenOptions`로 올리고 화면별 옵션은 각 `Drawer.Screen`의 `options`에 남기는 기준을 확인했다.

또한 Drawer Navigator에 등록된 화면에서는 `navigation.toggleDrawer()`를 사용해 코드로 드로어를 열고 닫을 수 있다는 점도 살펴봤다.

이제 Drawer Navigator의 기본 사용법과 커스터마이징을 알았으므로, 다음 단계에서는 탭 기반 네비게이터를 살펴본다.
