# 11. Stack Navigator 화면 옵션과 기본 스타일 설정

## 핵심 요약

- React Navigation은 기본 헤더와 화면 배경을 제공하지만, 실제 앱에서는 이를 커스터마이징해야 하는 경우가 많다.
- `Stack.Screen`의 `options` prop을 사용하면 개별 화면의 제목, 헤더 색상, 콘텐츠 배경 등을 설정할 수 있다.
- `name`은 내비게이션 식별자이자 기본 헤더 제목으로도 사용되지만, 보통 화면 제목은 `options.title`로 따로 설정한다.
- `headerStyle`로 헤더 배경색을, `headerTintColor`로 헤더 제목과 뒤로가기 버튼 색상을 바꿀 수 있다.
- `contentStyle`을 사용하면 Navigator가 제공하는 화면 콘텐츠 영역의 배경색을 바꿀 수 있다.
- 여러 화면에 같은 스타일을 적용하려면 개별 `options`를 복사하기보다 `Stack.Navigator`의 `screenOptions`를 사용한다.
- 화면별 `options`와 Navigator의 `screenOptions`가 충돌하면 화면별 `options`가 우선한다.

## 학습 목표

- React Navigation의 기본 헤더 제목이 어디서 오는지 이해한다.
- `Stack.Screen`의 `options` prop으로 개별 화면을 설정하는 방법을 익힌다.
- `title`, `headerStyle`, `headerTintColor`, `contentStyle`의 역할을 이해한다.
- `screenOptions`로 Stack 전체의 기본 스타일을 설정하는 방법을 익힌다.
- 공통 설정과 화면별 설정의 우선순위를 이해한다.
- 정적 옵션과 동적 옵션의 차이를 다음 단계와 연결해 이해한다.

## 주요 내용

### 기본 헤더와 기본 제목

React Navigation의 Stack Navigator를 사용하면 화면 상단에 기본 헤더가 생긴다.

처음에는 헤더 제목이 `Stack.Screen`의 `name` 값으로 표시된다.

```jsx
<Stack.Screen
  name="MealsCategories"
  component={CategoriesScreen}
/>
```

이 경우 헤더에는 기본적으로 `MealsCategories`가 표시될 수 있다.

하지만 `name`의 주된 목적은 화면 이동을 위한 식별자다.

```text
name의 주된 역할
-> navigate 대상 식별
-> Stack 내부에서 화면 구분
-> 기본 헤더 제목으로도 사용됨
```

즉 `name`은 사용자에게 보여줄 문구라기보다 내부 라우트 이름에 가깝다.

따라서 실제 헤더 제목은 별도로 설정하는 편이 좋다.

### Stack.Screen의 options prop

개별 화면의 설정은 `Stack.Screen`의 `options` prop으로 지정한다.

```jsx
<Stack.Screen
  name="MealsCategories"
  component={CategoriesScreen}
  options={{
    title: 'All Categories',
  }}
/>
```

`options`는 JavaScript 객체를 받는다.

이 객체 안에 React Navigation이 지원하는 여러 옵션을 넣을 수 있다.

```text
options
-> 화면별 설정 객체
-> 헤더 제목
-> 헤더 스타일
-> 콘텐츠 스타일
-> 기타 내비게이션 옵션
```

React Navigation은 옵션이 매우 많으므로, 원하는 설정이 있다면 공식 문서의 Native Stack 옵션을 확인하는 습관이 좋다.

### title 옵션

`title`은 헤더에 표시될 제목을 설정한다.

```jsx
options={{
  title: 'All Categories',
}}
```

이렇게 하면 `name`이 `MealsCategories`여도 사용자에게는 `All Categories` 같은 더 자연스러운 제목을 보여줄 수 있다.

```text
name
-> 내부 식별자

title
-> 사용자에게 보이는 헤더 제목
```

이 둘을 분리해두면 화면 이동 이름은 안정적으로 유지하면서, UI 문구는 자유롭게 바꿀 수 있다.

### headerStyle

헤더의 배경색은 `headerStyle`로 설정한다.

```jsx
options={{
  headerStyle: {
    backgroundColor: '#351401',
  },
}}
```

강의 시점 기준으로 Native Stack의 `headerStyle`에서 주로 설정하는 값은 `backgroundColor`다.

```text
headerStyle
-> 헤더 영역 스타일
-> 대표적으로 backgroundColor 설정
```

헤더 배경색을 어둡게 바꾸면 제목 색상이 잘 보이지 않을 수 있다.

이때 `headerTintColor`를 함께 설정한다.

### headerTintColor

`headerTintColor`는 헤더 제목과 뒤로가기 버튼 색상을 설정한다.

```jsx
options={{
  headerTintColor: 'white',
}}
```

헤더 배경이 어두운 색이라면 보통 흰색을 사용한다.

```text
headerTintColor가 영향을 주는 것
-> 헤더 제목 색상
-> 뒤로가기 버튼 색상
-> 뒤로가기 텍스트 색상
```

따라서 헤더 배경색과 함께 대비를 고려해야 한다.

### contentStyle

React Navigation을 적용한 뒤에는 이전에 `app.json`에서 설정한 배경색이 기대처럼 보이지 않을 수 있다.

이유는 Stack Navigator가 화면 콘텐츠 영역을 감싸는 기본 배경을 제공하기 때문이다.

이 콘텐츠 영역의 배경색은 `contentStyle`로 바꿀 수 있다.

```jsx
options={{
  contentStyle: {
    backgroundColor: '#3f2f25',
  },
}}
```

```text
headerStyle
-> 헤더 영역 배경

contentStyle
-> 헤더 아래 실제 화면 콘텐츠 영역 배경
```

카테고리 그리드 뒤의 배경색을 바꾸고 싶다면 `contentStyle.backgroundColor`를 사용한다.

### 개별 화면 options의 한계

개별 화면에 `options`를 설정하면 해당 화면에는 잘 적용된다.

하지만 다른 화면으로 이동하면 설정이 사라질 수 있다.

```text
MealsCategories
-> 헤더 색상 적용됨

MealsOverview
-> 기본 헤더 스타일로 돌아감
```

이것은 버그가 아니다.

React Navigation은 화면마다 다른 스타일을 가질 수 있도록 설계되어 있다.

따라서 특정 화면에만 `options`를 설정하면 그 화면에만 적용된다.

### 같은 설정을 여러 화면에 복사하는 문제

다른 화면에도 같은 스타일을 적용하려면 각 `Stack.Screen`에 같은 `options`를 복사할 수 있다.

```jsx
<Stack.Screen
  name="MealsCategories"
  component={CategoriesScreen}
  options={{
    headerStyle: { backgroundColor: '#351401' },
    headerTintColor: 'white',
    contentStyle: { backgroundColor: '#3f2f25' },
  }}
/>

<Stack.Screen
  name="MealsOverview"
  component={MealsOverviewScreen}
  options={{
    headerStyle: { backgroundColor: '#351401' },
    headerTintColor: 'white',
    contentStyle: { backgroundColor: '#3f2f25' },
  }}
/>
```

하지만 이 방식은 중복이 많다.

```text
문제점
-> 화면이 많아질수록 복사 코드 증가
-> 색상을 바꿀 때 여러 곳 수정 필요
-> 설정 불일치가 생기기 쉬움
```

공통 스타일은 Navigator 단위에서 기본값으로 설정하는 편이 좋다.

### screenOptions

`Stack.Navigator`에는 `screenOptions` prop을 설정할 수 있다.

```jsx
<Stack.Navigator
  screenOptions={{
    headerStyle: { backgroundColor: '#351401' },
    headerTintColor: 'white',
    contentStyle: { backgroundColor: '#3f2f25' },
  }}
>
  <Stack.Screen
    name="MealsCategories"
    component={CategoriesScreen}
    options={{
      title: 'All Categories',
    }}
  />
  <Stack.Screen
    name="MealsOverview"
    component={MealsOverviewScreen}
  />
</Stack.Navigator>
```

`screenOptions`는 이 Navigator에 등록된 모든 화면에 적용되는 기본 옵션이다.

```text
screenOptions
-> Navigator에 등록된 모든 화면의 기본 옵션
-> 공통 헤더 스타일
-> 공통 콘텐츠 배경
-> 공통 전환 옵션 등 설정 가능
```

이제 `MealsCategories`와 `MealsOverview` 모두 같은 헤더 색상과 콘텐츠 배경을 가진다.

### options와 screenOptions의 우선순위

`screenOptions`는 기본값이고, 개별 `Stack.Screen`의 `options`는 화면별 설정이다.

둘이 같은 옵션을 설정하면 개별 화면의 `options`가 우선한다.

```jsx
<Stack.Navigator
  screenOptions={{
    headerTintColor: 'white',
  }}
>
  <Stack.Screen
    name="MealsCategories"
    component={CategoriesScreen}
    options={{
      headerTintColor: 'yellow',
    }}
  />
</Stack.Navigator>
```

이 경우 `MealsCategories` 화면에서는 `yellow`가 적용된다.

```text
우선순위
-> Stack.Screen options
-> Stack.Navigator screenOptions
```

공통 설정은 `screenOptions`에 두고, 특정 화면만 다르게 하고 싶을 때 `options`로 덮어쓴다.

### 정적 옵션과 동적 옵션

이번 강의에서 다룬 `options`는 정적인 객체다.

```jsx
options={{
  title: 'All Categories',
}}
```

하지만 어떤 경우에는 화면에 전달된 params에 따라 제목이 달라져야 한다.

예를 들어 카테고리를 눌러 `MealsOverviewScreen`으로 이동했을 때, 헤더에 `Meals Overview` 대신 실제 카테고리 이름을 보여주고 싶을 수 있다.

```text
Italian 클릭
-> 헤더 제목: Italian

German 클릭
-> 헤더 제목: German
```

이런 경우에는 정적 객체만으로는 부족하고, 동적으로 options를 설정해야 한다.

이 내용은 다음 강의에서 다룬다.

## 예시 코드

### 개별 화면 options

```jsx
<Stack.Screen
  name="MealsCategories"
  component={CategoriesScreen}
  options={{
    title: 'All Categories',
    headerStyle: {
      backgroundColor: '#351401',
    },
    headerTintColor: 'white',
    contentStyle: {
      backgroundColor: '#3f2f25',
    },
  }}
/>
```

### 공통 screenOptions

```jsx
<Stack.Navigator
  screenOptions={{
    headerStyle: {
      backgroundColor: '#351401',
    },
    headerTintColor: 'white',
    contentStyle: {
      backgroundColor: '#3f2f25',
    },
  }}
>
  <Stack.Screen
    name="MealsCategories"
    component={CategoriesScreen}
    options={{
      title: 'All Categories',
    }}
  />
  <Stack.Screen
    name="MealsOverview"
    component={MealsOverviewScreen}
  />
</Stack.Navigator>
```

## 프론트엔드 개발자 관점에서의 핵심 차이

### 라우트 설정이 UI 셸 설정도 포함한다

웹 라우터에서는 라우트 설정이 주로 URL과 컴포넌트 매핑에 집중한다.

React Navigation의 화면 설정은 그보다 더 UI 셸에 가깝다.

```text
React Navigation screen options
-> 헤더 제목
-> 헤더 색상
-> 뒤로가기 버튼 색상
-> 콘텐츠 배경
-> 화면 전환 옵션
```

즉 내비게이션 설정은 단순 라우팅이 아니라 모바일 앱의 화면 껍데기까지 제어한다.

### name과 title을 분리해서 생각하기

웹에서도 route path와 페이지 타이틀은 보통 다르다.

```text
path
-> /categories

page title
-> All Categories
```

React Navigation에서도 `name`과 `title`을 분리해서 생각하면 좋다.

```text
name
-> navigate 대상

title
-> 사용자에게 보이는 헤더 문구
```

내부 식별자를 사용자에게 그대로 보여주지 않는 것이 일반적으로 더 좋다.

### 공통 설정은 한 곳에

웹에서 테마나 레이아웃 공통 스타일을 한 곳에 두는 것처럼, React Navigation에서도 공통 헤더 스타일은 `screenOptions`에 둔다.

```text
공통 설정
-> screenOptions

화면별 예외
-> options
```

이 구조를 지키면 화면이 늘어나도 내비게이션 스타일을 관리하기 쉽다.

### app.json 배경색과 Navigation 배경색

Expo의 `app.json`에 배경색을 설정해도 React Navigation이 제공하는 화면 컨테이너 배경이 그 위를 덮을 수 있다.

이 경우 실제 화면 콘텐츠 배경은 `contentStyle`로 설정해야 한다.

```text
app.json backgroundColor
-> 앱 기본 배경

Stack contentStyle.backgroundColor
-> Navigator 화면 콘텐츠 배경
```

React Navigation을 적용한 뒤 배경색이 사라진 것처럼 보이면 `contentStyle`을 확인해야 한다.

## 정리

이번 강의에서는 React Navigation이 제공하는 기본 헤더와 화면 스타일을 커스터마이징하는 방법을 배웠다.

개별 화면은 `Stack.Screen`의 `options`로 설정하고, 여러 화면에 공통으로 적용할 기본값은 `Stack.Navigator`의 `screenOptions`로 설정한다.

이제 앱의 헤더 제목, 헤더 색상, 콘텐츠 배경을 원하는 스타일로 조정할 수 있으며, 다음 단계에서는 route params에 따라 헤더 제목을 동적으로 바꾸는 방법을 배우게 된다.
