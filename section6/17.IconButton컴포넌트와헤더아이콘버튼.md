# 17. IconButton 컴포넌트와 헤더 아이콘 버튼

## 핵심 요약

- 이번 강의에서는 헤더 오른쪽에 넣을 커스텀 아이콘 버튼을 만든다.
- 기본 `Button` 대신 `Pressable`과 `@expo/vector-icons`의 `Ionicons`를 조합해 `IconButton` 컴포넌트를 만든다.
- `IconButton`은 `icon`, `color`, `onPress` prop을 받아 재사용 가능하게 구성한다.
- `Pressable`의 `style` prop에 함수를 전달해 눌렸을 때 opacity가 낮아지는 피드백을 추가한다.
- `MealDetailScreen`의 `navigation.setOptions()`에서 `headerRight`로 `IconButton`을 렌더링한다.
- 실제 기능은 아직 `console.log` 수준이지만, 즐겨찾기 같은 헤더 액션을 구현할 기반이 마련된다.

## 학습 목표

- `Pressable`과 아이콘 컴포넌트를 조합해 커스텀 버튼을 만드는 방법을 익힌다.
- Expo 프로젝트에서 `@expo/vector-icons`의 `Ionicons`를 사용하는 방법을 복습한다.
- 커스텀 컴포넌트에 `icon`, `color`, `onPress` 같은 prop을 설계하는 방법을 이해한다.
- `Pressable`의 `pressed` 상태를 이용한 시각적 피드백을 복습한다.
- `headerRight`에 직접 만든 컴포넌트를 넣는 방법을 익힌다.

## 주요 내용

### 왜 커스텀 IconButton이 필요한가?

이전 강의에서는 헤더 오른쪽에 기본 `Button`을 넣었다.

```jsx
<Button title="Tap me" onPress={headerButtonPressHandler} />
```

기본 `Button`은 빠르게 기능을 확인하기에는 좋지만, 헤더에 넣기에는 스타일 제약이 크다.

특히 즐겨찾기 버튼처럼 별 아이콘만 보여주고 싶은 경우에는 커스텀 버튼이 더 적합하다.

```text
기본 Button
-> 빠르게 테스트 가능
-> 스타일 제약 있음
-> 헤더 디자인과 어색할 수 있음

커스텀 IconButton
-> 아이콘만 표시 가능
-> 원하는 색상과 피드백 설정 가능
-> 헤더 액션 버튼에 적합
```

### IconButton 파일 추가

`components` 폴더에 `IconButton.js` 파일을 만든다.

```text
components/
  IconButton.js
```

기본 컴포넌트 구조는 다음과 같다.

```jsx
function IconButton() {
  return (
    // pressable icon
  );
}

export default IconButton;
```

이 컴포넌트는 헤더뿐만 아니라 앱의 다른 곳에서도 재사용할 수 있다.

### Pressable 사용

아이콘 버튼은 눌릴 수 있어야 하므로 `Pressable`을 사용한다.

```jsx
import { Pressable } from 'react-native';
```

```jsx
<Pressable>
  {/* icon */}
</Pressable>
```

`Pressable`은 터치 가능한 영역을 만들고, 눌린 상태에 따라 스타일을 다르게 적용할 수 있다.

### Ionicons 사용

아이콘은 Expo 프로젝트에서 사용할 수 있는 `@expo/vector-icons` 패키지를 활용한다.

```jsx
import { Ionicons } from '@expo/vector-icons';
```

`Ionicons`는 컴포넌트처럼 사용할 수 있다.

```jsx
<Ionicons
  name="star"
  size={24}
  color="white"
/>
```

```text
name
-> 사용할 아이콘 이름

size
-> 아이콘 크기

color
-> 아이콘 색상
```

강의에서는 즐겨찾기 예시로 `star` 아이콘을 사용한다.

### onPress prop 전달

`IconButton`이 실제 버튼처럼 동작하려면 외부에서 누를 때 실행할 함수를 받아야 한다.

```jsx
function IconButton({ onPress }) {
  return (
    <Pressable onPress={onPress}>
      <Ionicons name="star" size={24} color="white" />
    </Pressable>
  );
}
```

`onPress`라는 prop 이름은 직접 정한 것이다.

React Native의 `Pressable`이 제공하는 `onPress` prop에 그대로 전달한다.

```text
IconButton onPress
-> 외부에서 전달받는 함수

Pressable onPress
-> 실제 터치 이벤트에 연결
```

### icon과 color를 prop으로 만들기

처음에는 아이콘 이름과 색상을 하드코딩할 수 있다.

```jsx
<Ionicons name="star" size={24} color="white" />
```

하지만 재사용성을 높이려면 `icon`과 `color`도 props로 받는 편이 좋다.

```jsx
function IconButton({ icon, color, onPress }) {
  return (
    <Pressable onPress={onPress}>
      <Ionicons name={icon} size={24} color={color} />
    </Pressable>
  );
}
```

이제 사용하는 쪽에서 원하는 아이콘과 색상을 지정할 수 있다.

```jsx
<IconButton
  icon="star"
  color="white"
  onPress={headerButtonPressHandler}
/>
```

이런 prop 설계 덕분에 나중에 다른 화면에서 다른 아이콘 버튼으로 재사용하기 쉽다.

### pressed 상태 피드백

버튼을 눌렀을 때 시각적 피드백이 있어야 사용자가 터치가 인식되었음을 알 수 있다.

`Pressable`의 `style` prop에는 함수를 전달할 수 있다.

```jsx
style={({ pressed }) => {
  return pressed ? styles.pressed : null;
}}
```

`pressed`는 사용자가 버튼을 누르고 있는 동안 `true`가 된다.

```text
pressed === true
-> 눌림 상태

pressed === false
-> 기본 상태
```

눌림 상태에서는 opacity를 낮춘다.

```jsx
pressed: {
  opacity: 0.7,
}
```

또는 더 강한 효과를 원하면 `0.5`로 설정할 수도 있다.

### IconButton 전체 구조

최종적으로 `IconButton`은 다음과 같은 구조가 된다.

```jsx
import { Ionicons } from '@expo/vector-icons';
import { Pressable, StyleSheet } from 'react-native';

function IconButton({ icon, color, onPress }) {
  return (
    <Pressable
      onPress={onPress}
      style={({ pressed }) =>
        pressed ? styles.pressed : null
      }
    >
      <Ionicons name={icon} size={24} color={color} />
    </Pressable>
  );
}

export default IconButton;

const styles = StyleSheet.create({
  pressed: {
    opacity: 0.7,
  },
});
```

단순하지만 헤더 액션 버튼으로 충분히 사용할 수 있다.

### MealDetailScreen에서 사용하기

이제 `MealDetailScreen`의 `headerRight`에서 기본 `Button` 대신 `IconButton`을 사용한다.

```jsx
import IconButton from '../components/IconButton';
```

```jsx
useLayoutEffect(() => {
  navigation.setOptions({
    headerRight: () => {
      return (
        <IconButton
          icon="star"
          color="white"
          onPress={headerButtonPressHandler}
        />
      );
    },
  });
}, [navigation, headerButtonPressHandler]);
```

이제 헤더 오른쪽에 별 아이콘이 표시되고, 누르면 `headerButtonPressHandler`가 실행된다.

현재 핸들러는 `console.log('pressed')`만 실행하지만, 실제 앱에서는 즐겨찾기 토글 같은 동작을 넣을 수 있다.

### Android와 iOS 피드백

이번 `IconButton`은 `Pressable`의 opacity 효과만 사용한다.

Android에서는 필요하다면 `android_ripple`을 추가할 수도 있다.

```jsx
<Pressable
  android_ripple={{ color: '#ccc' }}
>
  ...
</Pressable>
```

강의에서는 헤더 아이콘 버튼의 단순 opacity 효과만으로도 충분하다고 보고 진행한다.

```jsx

function IconButton({ icon, color, onPress }) {
  return (
    <Pressable
      onPress={onPress}
      style={({ pressed }) =>
        pressed ? styles.pressed : null

    >

    </Pressable>
  );

export default IconButton;

const styles = StyleSheet.create({

    opacity: 0.7,
  },
```

## 프론트엔드 개발자 관점에서의 핵심 차이

### 아이콘 버튼은 결국 일반 컴포넌트다

React Navigation의 헤더에 들어간다고 해서 특별한 API만 써야 하는 것은 아니다.

`headerRight`가 요구하는 것은 JSX를 반환하는 함수다.

따라서 우리가 직접 만든 React Native 컴포넌트를 그대로 넣을 수 있다.

```text
IconButton
-> Pressable + Ionicons
-> 일반 React Native 컴포넌트
-> headerRight에서 사용 가능
```

### 웹의 icon button 패턴과 비슷하다

웹에서도 아이콘 버튼은 보통 버튼과 아이콘을 조합해 만든다.

```jsx
<button onClick={onClick}>
  <StarIcon />
</button>
```

React Native에서는 `button` 대신 `Pressable`, SVG 아이콘 대신 `Ionicons` 같은 아이콘 컴포넌트를 사용한다.

```jsx
<Pressable onPress={onPress}>
  <Ionicons name="star" />
</Pressable>
```

### prop으로 재사용성을 만든다

처음부터 `star`, `white`를 하드코딩하면 이 버튼은 즐겨찾기 버튼으로만 쓸 수 있다.

하지만 `icon`, `color`, `onPress`를 prop으로 받으면 훨씬 재사용 가능하다.

```text
재사용 가능한 IconButton
-> icon 변경 가능
-> color 변경 가능
-> onPress 동작 변경 가능
```

작은 컴포넌트라도 prop 설계를 잘하면 앱 전체에서 반복적으로 사용할 수 있다.

### 다음 단계: 실제 즐겨찾기 상태

현재 버튼은 눌렀을 때 로그만 출력한다.

하지만 이 구조를 바탕으로 나중에 실제 즐겨찾기 상태를 연결할 수 있다.

```text
현재
-> 버튼 렌더링
-> onPress 실행
-> console.log

다음 확장
-> 즐겨찾기 상태 저장
-> 별 아이콘 채움/비움 변경
-> 전역 상태나 Context와 연결
```

즉 이번 강의는 실제 기능 구현 전, 헤더 액션 버튼의 UI와 이벤트 연결 기반을 만든 것이다.

## 정리

이번 강의에서는 기본 `Button` 대신 헤더에 어울리는 커스텀 `IconButton` 컴포넌트를 만들었다.

`Pressable`과 `Ionicons`를 조합하고, `icon`, `color`, `onPress` prop을 받아 재사용성을 확보했다.

이제 `MealDetailScreen`의 헤더 오른쪽에는 별 아이콘 버튼이 표시되고, 이 버튼은 화면 내부 핸들러와 연결되어 실제 기능을 붙일 준비가 되었다.
