# 21. Nested Navigators와 Stack 안의 Drawer

## 핵심 요약

- React Navigation에서는 여러 네비게이터를 하나의 앱 안에서 중첩해서 사용할 수 있다.
- 네비게이터 자체도 React 컴포넌트처럼 만들 수 있고, 다른 네비게이터의 `Screen`으로 등록할 수 있다.
- Meals 앱에서는 기존 Stack Navigator의 첫 화면인 `CategoriesScreen` 자리에 Drawer Navigator를 넣는다.
- Drawer Navigator 안에는 `CategoriesScreen`과 새로 만든 `FavoritesScreen`을 등록한다.
- 이렇게 하면 앱의 시작 영역에서는 Drawer로 `Categories`와 `Favorites`를 오가고, 카테고리 선택 이후에는 기존 Stack 흐름으로 상세 화면까지 이동할 수 있다.
- 네비게이터를 중첩하면 각 네비게이터가 자기 헤더를 렌더링하므로 헤더가 두 개 생길 수 있다.
- 원하지 않는 헤더는 `headerShown: false`로 숨긴다.
- Drawer Navigator의 콘텐츠 배경은 Stack의 `contentStyle`이 아니라 `sceneContainerStyle`로 설정해야 할 수 있다.

## 학습 목표

- Nested Navigator의 개념을 이해한다.
- 네비게이터를 컴포넌트 함수로 만들고 다른 네비게이터의 screen component로 등록하는 패턴을 익힌다.
- Stack Navigator와 Drawer Navigator를 조합하는 구조를 이해한다.
- `FavoritesScreen` 같은 새로운 루트 섹션 화면을 Drawer에 추가하는 방법을 익힌다.
- 중첩 네비게이터에서 헤더가 중복되는 이유와 해결 방법을 이해한다.
- 네비게이터별 옵션 이름이 다를 수 있음을 인지한다.

## 주요 내용

### 왜 네비게이터를 중첩하는가?

지금까지 Meals 앱은 Stack Navigator 중심으로 구성되어 있었다.

```text
Categories
-> MealsOverview
-> MealDetail
```

이 구조는 카테고리에서 식사 목록으로, 다시 상세 화면으로 들어가는 흐름에 적합하다.

하지만 앱에 즐겨찾기 화면이 추가된다면 이야기가 달라진다.

```text
All Categories
Favorites
```

이 두 화면은 서로 “상세 흐름의 앞뒤 관계”라기보다 앱의 주요 섹션에 가깝다.

따라서 시작 화면 레벨에서는 Drawer로 섹션을 전환하고, 각 섹션 내부에서는 필요할 때 Stack으로 더 깊은 화면에 들어가는 구조가 자연스럽다.

```text
Drawer
-> Categories
-> Favorites

Stack
-> Drawer 영역
-> MealsOverview
-> MealDetail
```

### 네비게이터도 Screen으로 등록할 수 있다

React Navigation의 중요한 특징은 네비게이터를 중첩할 수 있다는 점이다.

즉 어떤 네비게이터 설정을 하나의 React 컴포넌트로 만들고, 그 컴포넌트를 다른 네비게이터의 `Screen`에 등록할 수 있다.

```text
Stack.Screen
-> component={DrawerNavigator}

DrawerNavigator
-> 내부에서 Drawer.Navigator 렌더링
```

이 구조를 통해 하나의 앱 안에서 여러 네비게이션 패턴을 조합할 수 있다.

### Drawer Navigator 설치

Meals 앱에서도 Drawer Navigator를 사용해야 하므로 패키지를 설치한다.

```bash
npm install @react-navigation/drawer
```

Drawer Navigator가 내부적으로 사용하는 의존성도 설치한다.

강의에서는 Expo 환경에서 필요한 패키지를 설치하고, 강의 당시 발생한 `react-native-reanimated` 문제를 피하기 위해 특정 버전을 고정하는 우회 방법도 보여준다.

```bash
expo install react-native-gesture-handler react-native-reanimated
npm install react-native-reanimated@1 --save --save-exact
```

다만 이 버전 고정은 강의 당시 환경의 이슈에 대한 해결책이다.

실제 프로젝트에서는 현재 Expo SDK와 React Navigation 버전에 맞는 공식 설치 안내를 따르는 것이 더 안전하다.

### FavoritesScreen 추가

Drawer에 넣을 새 화면으로 `FavoritesScreen`을 만든다.

처음에는 실제 즐겨찾기 로직 없이 더미 텍스트만 출력한다.

```jsx
import { Text } from 'react-native';

function FavoritesScreen() {
  return <Text>The favorites screen!</Text>;
}

export default FavoritesScreen;
```

이 화면은 나중에 사용자가 즐겨찾기로 저장한 meal 목록을 보여주는 역할을 맡게 된다.

### createDrawerNavigator 추가

기존 `App.js`에는 Native Stack Navigator가 이미 있었다.

여기에 Drawer Navigator도 추가한다.

```jsx
import { createNativeStackNavigator } from '@react-navigation/native-stack';
import { createDrawerNavigator } from '@react-navigation/drawer';
```

그리고 각각의 네비게이터 객체를 만든다.

```jsx
const Stack = createNativeStackNavigator();
const Drawer = createDrawerNavigator();
```

Stack은 기존 상세 흐름을 담당하고, Drawer는 앱 시작 영역의 주요 섹션 전환을 담당한다.

### DrawerNavigator 컴포넌트 만들기

`App.js` 안에 `DrawerNavigator`라는 컴포넌트 함수를 만든다.

이름은 자유지만, 이 함수가 Drawer 설정을 담는다는 점이 드러나면 좋다.

```jsx
function DrawerNavigator() {
  return (
    <Drawer.Navigator>
      <Drawer.Screen
        name="Categories"
        component={CategoriesScreen}
      />
      <Drawer.Screen
        name="Favorites"
        component={FavoritesScreen}
      />
    </Drawer.Navigator>
  );
}
```

이 컴포넌트는 화면 UI를 직접 그린다기보다, Drawer 기반 네비게이션 설정을 반환한다.

별도 파일로 분리해도 되지만, 강의에서는 네비게이션 설정을 한곳에서 보기 위해 `App.js`에 둔다.

### Stack의 첫 화면을 DrawerNavigator로 교체

기존에는 Stack의 첫 화면이 `CategoriesScreen`이었다.

```jsx
<Stack.Screen
  name="MealsCategories"
  component={CategoriesScreen}
/>
```

이제 이 자리에 `DrawerNavigator`를 넣는다.

```jsx
<Stack.Screen
  name="Drawer"
  component={DrawerNavigator}
/>
```

중요한 점은 `CategoriesScreen`이 사라진 것이 아니라 위치가 바뀐 것이다.

```text
기존
Stack.Screen -> CategoriesScreen

변경 후
Stack.Screen -> DrawerNavigator
Drawer.Screen -> CategoriesScreen
```

즉 `CategoriesScreen`은 이제 Stack에 직접 등록되는 것이 아니라 Drawer 안에 등록된다.

### screen name 중복 피하기

React Navigation에서 `name`은 화면 식별자다.

중첩 네비게이터를 사용할 때 같은 이름을 여러 곳에 반복해서 쓰면 경고가 나올 수 있다.

```text
Stack.Screen name="Categories"
Drawer.Screen name="Categories"
```

기술적으로 가능할 때도 있지만, 네비게이션 대상이 헷갈릴 수 있다.

강의에서는 Stack 쪽 이름을 `Drawer` 같은 별도 이름으로 두고, Drawer 내부 화면 이름을 `Categories`, `Favorites`로 둔다.

```jsx
<Stack.Screen
  name="Drawer"
  component={DrawerNavigator}
/>
```

이렇게 하면 screen name이 역할에 맞게 구분된다.

### 중첩 후 생기는 두 개의 헤더

Drawer Navigator를 Stack 안에 넣으면 처음에는 헤더가 두 개 보일 수 있다.

```text
Stack Navigator의 헤더
Drawer Navigator의 헤더
```

각 네비게이터가 자기 자신만의 헤더를 렌더링하기 때문이다.

이것이 항상 문제는 아니다.

어떤 앱에서는 중첩된 헤더가 필요한 경우도 있다.

하지만 이번 Meals 앱에서는 시작 화면에서 Drawer 아이콘이 있는 헤더 하나만 있으면 충분하다.

따라서 Stack 쪽 헤더를 숨긴다.

### headerShown: false

Stack에서 DrawerNavigator를 렌더링하는 screen에 `headerShown: false`를 설정한다.

```jsx
<Stack.Screen
  name="Drawer"
  component={DrawerNavigator}
  options={{
    headerShown: false,
  }}
/>
```

이 설정은 해당 Stack Screen의 헤더를 제거한다.

```text
Stack header
-> 숨김

Drawer header
-> 유지
-> 드로어 아이콘 사용 가능
```

이제 시작 화면에는 Drawer 헤더만 보이고, 카테고리에서 meal 목록이나 상세 화면으로 들어가면 기존 Stack 흐름의 헤더가 다시 나타난다.

### Drawer 스타일 설정

기존 Stack Navigator에는 헤더 스타일과 콘텐츠 배경색이 설정되어 있었다.

Drawer에도 같은 분위기를 적용하려면 `Drawer.Navigator`의 `screenOptions`에 옵션을 추가한다.

```jsx
<Drawer.Navigator
  screenOptions={{
    headerStyle: { backgroundColor: '#351401' },
    headerTintColor: 'white',
    sceneContainerStyle: { backgroundColor: '#3f2f25' },
  }}
>
  ...
</Drawer.Navigator>
```

여기서 주의할 점이 있다.

Stack Navigator에서는 화면 콘텐츠 배경을 `contentStyle`로 설정했다.

하지만 Drawer Navigator에서는 강의 시점 기준으로 `sceneContainerStyle`을 사용한다.

```text
Stack Navigator
-> contentStyle

Drawer Navigator
-> sceneContainerStyle
```

네비게이터마다 지원하는 옵션 이름이 다를 수 있으므로 공식 문서를 확인해야 한다.

### Categories 화면 제목 설정

기존에는 Stack의 `MealsCategories` screen에서 제목을 `All Categories`로 설정했다.

하지만 이제 `CategoriesScreen`은 Drawer 안에 있다.

따라서 제목도 Drawer의 해당 screen에서 설정해야 한다.

```jsx
<Drawer.Screen
  name="Categories"
  component={CategoriesScreen}
  options={{
    title: 'All Categories',
  }}
/>
```

화면이 어느 네비게이터에 등록되어 있는지에 따라 옵션을 설정해야 하는 위치도 달라진다.

### MealDetail 제목 설정

강의에서는 부가적으로 `MealDetailScreen`의 제목도 고정값으로 바꾼다.

```jsx
<Stack.Screen
  name="MealDetail"
  component={MealDetailScreen}
  options={{
    title: 'About the Meal',
  }}
/>
```

동적으로 meal title을 보여줄 수도 있지만, 여기서는 간단히 `About the Meal`이라는 고정 제목을 사용한다.

## 예시 코드

### App.js 구조

```jsx
import { NavigationContainer } from '@react-navigation/native';
import { createNativeStackNavigator } from '@react-navigation/native-stack';
import { createDrawerNavigator } from '@react-navigation/drawer';

import CategoriesScreen from './screens/CategoriesScreen';
import MealsOverviewScreen from './screens/MealsOverviewScreen';
import MealDetailScreen from './screens/MealDetailScreen';
import FavoritesScreen from './screens/FavoritesScreen';

const Stack = createNativeStackNavigator();
const Drawer = createDrawerNavigator();

function DrawerNavigator() {
  return (
    <Drawer.Navigator
      screenOptions={{
        headerStyle: { backgroundColor: '#351401' },
        headerTintColor: 'white',
        sceneContainerStyle: { backgroundColor: '#3f2f25' },
      }}
    >
      <Drawer.Screen
        name="Categories"
        component={CategoriesScreen}
        options={{
          title: 'All Categories',
        }}
      />
      <Drawer.Screen
        name="Favorites"
        component={FavoritesScreen}
      />
    </Drawer.Navigator>
  );
}

export default function App() {
  return (
    <NavigationContainer>
      <Stack.Navigator
        screenOptions={{
          headerStyle: { backgroundColor: '#351401' },
          headerTintColor: 'white',
          contentStyle: { backgroundColor: '#3f2f25' },
        }}
      >
        <Stack.Screen
          name="Drawer"
          component={DrawerNavigator}
          options={{
            headerShown: false,
          }}
        />
        <Stack.Screen
          name="MealsOverview"
          component={MealsOverviewScreen}
        />
        <Stack.Screen
          name="MealDetail"
          component={MealDetailScreen}
          options={{
            title: 'About the Meal',
          }}
        />
      </Stack.Navigator>
    </NavigationContainer>
  );
}
```

## 프론트엔드 개발자 관점에서의 핵심 차이

### 네비게이터는 라우트 그룹처럼 생각할 수 있다

웹 프론트엔드에서 라우트 그룹이나 layout route를 사용하는 것처럼, React Navigation에서도 특정 화면 묶음을 하나의 네비게이터로 구성할 수 있다.

```text
DrawerNavigator
-> Categories
-> Favorites

StackNavigator
-> DrawerNavigator
-> MealsOverview
-> MealDetail
```

네비게이터 자체가 컴포넌트이므로, 다른 네비게이터 안에 넣을 수 있다.

### UX 패턴을 조합한다

Nested Navigator는 단순히 기술적인 조합이 아니다.

각 네비게이터는 서로 다른 UX 패턴을 담당한다.

```text
Drawer
-> 앱의 큰 섹션 전환

Stack
-> 특정 섹션 안에서 깊이 들어가는 흐름
```

Meals 앱에서는 Drawer가 `Categories`와 `Favorites` 사이의 전환을 맡고, Stack은 카테고리 선택 이후 meal 목록과 상세 화면 이동을 맡는다.

### 헤더 중복은 자연스러운 결과다

네비게이터를 중첩하면 헤더가 두 개 생길 수 있다.

이것은 버그가 아니라 각 네비게이터가 자기 UI를 렌더링하기 때문에 생기는 자연스러운 결과다.

따라서 어떤 헤더를 남길지 결정해야 한다.

```text
Drawer 아이콘이 필요하다
-> Drawer 헤더 유지
-> Stack 헤더 숨김
```

이번 앱에서는 Drawer를 열 수 있어야 하므로 Drawer 헤더를 남긴다.

### 옵션은 화면이 등록된 곳에 설정한다

`CategoriesScreen`의 제목을 바꾸려면 이제 Stack이 아니라 Drawer screen options를 수정해야 한다.

왜냐하면 `CategoriesScreen`이 더 이상 Stack에 직접 등록되어 있지 않기 때문이다.

```text
화면 컴포넌트가 어디에 등록되어 있는가?
-> 그 네비게이터의 Screen options에서 설정
```

Nested Navigator를 다룰 때 이 감각이 중요하다.

## 정리

이번 강의에서는 Meals 앱에 Drawer Navigator를 추가하고, 기존 Stack Navigator와 중첩했다.

기존 Stack의 첫 화면을 `CategoriesScreen`에서 `DrawerNavigator`로 바꾸고, Drawer 내부에 `CategoriesScreen`과 새 `FavoritesScreen`을 등록했다.

중첩 후 생기는 헤더 중복 문제는 Stack Screen에 `headerShown: false`를 설정해 해결했고, Drawer 쪽에는 `screenOptions`로 헤더 색상과 콘텐츠 배경을 맞췄다.

이제 앱은 시작 영역에서는 Drawer로 주요 섹션을 전환하고, 그 이후에는 Stack으로 상세 화면에 들어가는 복합 네비게이션 구조를 갖게 되었다.
