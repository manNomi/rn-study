# 3. CategoriesScreen과 카테고리 목록 출력

## 핵심 요약

- 이번 강의에서는 Meals 앱의 첫 화면인 카테고리 목록 화면을 만든다.
- 아직 내비게이션 관련 패키지나 특별한 트릭은 사용하지 않는다.
- `screens` 폴더를 만들고, 전체 화면 역할을 하는 `CategoriesScreen` 컴포넌트를 추가한다.
- 제공된 `dummy-data.js`에서 `CATEGORIES` 데이터를 가져와 화면에 출력한다.
- 카테고리 목록은 정적인 제한된 데이터이므로 꼭 `FlatList`가 필요하지는 않지만, 연습을 위해 `FlatList`를 사용한다.
- 각 카테고리 아이템은 `CategoryGridTile`이라는 별도 컴포넌트로 분리한다.

## 학습 목표

- 화면 단위 컴포넌트를 `screens` 폴더에 분리하는 구조를 이해한다.
- 더미 데이터에서 배열 데이터를 가져와 화면에 출력하는 흐름을 익힌다.
- `FlatList`의 `data`, `renderItem`, `keyExtractor` 사용법을 복습한다.
- `renderItem` 함수를 컴포넌트 바깥에 둘 수 있는 이유를 이해한다.
- 카테고리 타일 UI를 별도 컴포넌트로 분리하는 기준을 익힌다.

## 주요 내용

### screens 폴더 추가

Meals 앱에는 여러 화면이 필요하다.

따라서 화면 역할을 하는 컴포넌트를 모아두기 위해 `screens` 폴더를 만든다.

```text
screens/
  CategoriesScreen.js
```

`CategoriesScreen.js`에는 음식 카테고리 목록을 보여주는 화면 컴포넌트를 작성한다.

```jsx
function CategoriesScreen() {
  return (
    // categories output
  );
}

export default CategoriesScreen;
```

여기서 `CategoriesScreen`도 특별한 네이티브 화면 타입이 아니라 일반 React 컴포넌트다.

다만 앱에서 전체 화면처럼 사용되기 때문에 `screens` 폴더에 둔다.

### 더미 데이터 가져오기

카테고리 목록을 출력하려면 제공된 더미 데이터를 가져와야 한다.

```jsx
import { CATEGORIES } from '../data/dummy-data';
```

`CATEGORIES`는 카테고리 객체들이 들어 있는 배열이다.

각 카테고리는 모델 클래스를 통해 생성되어 있고, 다음과 같은 정보를 가진다.

```text
Category
-> id
-> title
-> color
```

이 데이터 중 `title`은 화면에 표시할 텍스트이고, `color`는 나중에 타일 배경색 스타일에 사용할 수 있다.

### FlatList 사용

카테고리 데이터는 정적이고 개수도 많지 않다.

따라서 반드시 `FlatList`를 써야 하는 상황은 아니다.

```text
이 경우 가능한 선택지
-> categories.map(...)으로 직접 JSX 배열 생성
-> ScrollView로 감싸기
-> FlatList 사용
```

강의에서는 `FlatList` 연습을 위해 `FlatList`를 사용한다.

```jsx
import { FlatList } from 'react-native';
```

`FlatList`에는 출력할 배열을 `data` prop으로 전달한다.

```jsx
<FlatList data={CATEGORIES} />
```

### keyExtractor

`FlatList`는 각 아이템을 구분할 key가 필요하다.

React에서 목록을 렌더링할 때 key가 필요한 것과 같은 이유다.

`FlatList`는 기본적으로 데이터 객체의 `key` 프로퍼티를 찾는다.

하지만 이번 카테고리 데이터에는 `key`가 아니라 `id`가 있다.

따라서 `keyExtractor`를 설정한다.

```jsx
<FlatList
  data={CATEGORIES}
  keyExtractor={(item) => item.id}
/>
```

`keyExtractor`는 `FlatList`가 각 아이템을 렌더링할 때 호출하는 함수다.

```text
item
-> 현재 렌더링 중인 데이터 항목

return item.id
-> 해당 항목의 고유 key로 사용
```

카테고리의 `id`는 고유하므로 key로 적합하다.

### renderItem

`FlatList`는 각 항목을 어떻게 렌더링할지 알아야 한다.

이를 위해 `renderItem` prop을 사용한다.

```jsx
<FlatList
  data={CATEGORIES}
  keyExtractor={(item) => item.id}
  renderItem={renderCategoryItem}
/>
```

`renderCategoryItem`은 각 카테고리 항목을 받아 JSX를 반환하는 함수다.

```jsx
function renderCategoryItem(itemData) {
  return (
    <CategoryGridTile
      title={itemData.item.title}
      color={itemData.item.color}
    />
  );
}
```

여기서 `renderItem` 함수가 받는 값은 실제 데이터 객체 자체가 아니라 `itemData` 객체다.

```text
itemData
-> FlatList가 넘겨주는 메타 객체

itemData.item
-> 실제 데이터 항목

itemData.index
-> 현재 항목의 인덱스
```

따라서 카테고리의 제목은 `itemData.item.title`로 접근한다.

### renderCategoryItem을 컴포넌트 밖에 두는 이유

강의에서는 `renderCategoryItem` 함수를 `CategoriesScreen` 컴포넌트 바깥에 정의한다.

```jsx
function renderCategoryItem(itemData) {
  return <CategoryGridTile title={itemData.item.title} />;
}

function CategoriesScreen() {
  return <FlatList renderItem={renderCategoryItem} />;
}
```

이 함수는 컴포넌트의 상태나 props에 의존하지 않는다.

따라서 컴포넌트가 다시 렌더링될 때마다 함수를 새로 만들 필요가 없다.

```text
컴포넌트 밖에 둘 수 있는 함수
-> 컴포넌트 내부 상태에 의존하지 않음
-> props에 의존하지 않음
-> 매번 새로 생성될 필요가 없음
```

현재 `CategoriesScreen`은 리렌더링될 일이 거의 없지만, 그래도 이런 분리는 좋은 습관이 될 수 있다.

### CategoryGridTile 컴포넌트 추가

각 카테고리 아이템을 별도 컴포넌트로 분리한다.

```text
components/
  CategoryGridTile.js
```

이 컴포넌트는 카테고리 하나를 그리드 타일 형태로 보여줄 예정이다.

```jsx
function CategoryGridTile({ title, color }) {
  return (
    <View>
      <Pressable>
        <View>
          <Text>{title}</Text>
        </View>
      </Pressable>
    </View>
  );
}

export default CategoryGridTile;
```

아직 스타일링은 완성되지 않았다.

하지만 구조는 미리 잡아둔다.

```text
View
-> 타일의 외부 컨테이너

Pressable
-> 나중에 카테고리를 눌러 다음 화면으로 이동하기 위한 터치 영역

내부 View
-> 스타일링을 위한 추가 래퍼

Text
-> 카테고리 제목 출력
```

### 왜 Pressable을 미리 넣는가?

카테고리 타일은 나중에 사용자가 누를 수 있어야 한다.

카테고리를 누르면 해당 카테고리에 속한 음식 목록 화면으로 이동할 예정이다.

아직 이동할 화면과 내비게이션 로직은 없지만, UI 컴포넌트 구조상 타일은 누를 수 있는 요소가 되어야 한다.

```text
카테고리 타일의 역할
-> 카테고리 제목 표시
-> 카테고리 색상 반영
-> 사용자가 누를 수 있음
-> 누르면 다음 화면으로 이동
```

따라서 지금부터 `Pressable`을 포함한 구조로 만든다.

### App.js에서 CategoriesScreen 렌더링

마지막으로 `App.js`에서 기존 더미 UI를 제거하고 `CategoriesScreen`을 렌더링한다.

```jsx
import CategoriesScreen from './screens/CategoriesScreen';

export default function App() {
  return <CategoriesScreen />;
}
```

더 이상 기존 `Text`, `View`, `StyleSheet`가 필요 없다면 import와 styles를 정리한다.

```text
정리할 것
-> 사용하지 않는 Text import
-> 사용하지 않는 View import
-> 사용하지 않는 StyleSheet import
-> 기존 더미 스타일 객체
```

이 상태로 저장하면 양쪽 디바이스에서 카테고리 목록이 보인다.

아직 스타일과 레이아웃은 완성되지 않았지만, 데이터가 화면에 출력되는 첫 단계는 성공이다.

## 예시 코드

### CategoriesScreen

```jsx
import { FlatList } from 'react-native';

import CategoryGridTile from '../components/CategoryGridTile';
import { CATEGORIES } from '../data/dummy-data';

function renderCategoryItem(itemData) {
  return (
    <CategoryGridTile
      title={itemData.item.title}
      color={itemData.item.color}
    />
  );
}

function CategoriesScreen() {
  return (
    <FlatList
      data={CATEGORIES}
      keyExtractor={(item) => item.id}
      renderItem={renderCategoryItem}
    />
  );
}

export default CategoriesScreen;
```

### CategoryGridTile

```jsx
import { Pressable, Text, View } from 'react-native';

function CategoryGridTile({ title, color }) {
  return (
    <View>
      <Pressable>
        <View>
          <Text>{title}</Text>
        </View>
      </Pressable>
    </View>
  );
}

export default CategoryGridTile;
```

## 프론트엔드 개발자 관점에서의 핵심 차이

### screens와 components의 역할 구분

웹 React에서도 페이지 컴포넌트와 UI 컴포넌트를 나누는 경우가 많다.

```text
웹 React
-> pages/
-> components/

React Native
-> screens/
-> components/
```

`screens`는 라우트나 내비게이션 대상이 되는 큰 단위의 화면이다.

`components`는 화면 안에서 재사용되는 작은 UI 단위다.

이번 강의에서는 `CategoriesScreen`이 화면 컴포넌트이고, `CategoryGridTile`이 UI 컴포넌트다.

### FlatList는 map의 대체재이자 모바일 최적화 도구

웹에서는 보통 배열을 `map`으로 렌더링한다.

```jsx
categories.map((category) => (
  <CategoryCard key={category.id} category={category} />
));
```

React Native에서도 이것이 가능하다.

하지만 긴 목록이나 동적 목록에서는 `FlatList`가 더 적합하다.

```text
FlatList의 장점
-> 필요한 항목만 지연 렌더링
-> 스크롤 가능한 목록 처리
-> key 추출, 아이템 렌더링 구조 제공
-> 모바일 성능에 유리
```

이번 카테고리 목록은 작고 정적이므로 꼭 필요하지는 않지만, 실제 앱에서는 자주 쓰이므로 반복 연습할 가치가 있다.

### itemData.item 패턴에 익숙해지기

`FlatList`의 `renderItem`은 배열의 원소를 바로 넘겨주지 않는다.

웹의 `map`과 비교하면 차이가 있다.

```jsx
// 웹 map
categories.map((category) => category.title);

// React Native FlatList
renderItem={(itemData) => itemData.item.title}
```

처음에는 `itemData.item`이 조금 장황하게 느껴질 수 있다.

하지만 `itemData` 안에는 `item`뿐 아니라 `index` 같은 추가 정보도 들어 있기 때문에 이런 구조를 가진다.

### 컴포넌트 분리는 스타일링 복잡도에 대비하는 선택

현재 `CategoryGridTile`은 단순해 보인다.

하지만 곧 배경색, 그림자, ripple 효과, pressed 상태, 그리드 레이아웃 같은 스타일이 들어갈 예정이다.

이런 UI를 `CategoriesScreen` 안에 직접 작성하면 화면 컴포넌트가 빠르게 복잡해진다.

```text
분리 전
-> 화면 컴포넌트가 데이터 처리와 UI 스타일을 모두 가짐

분리 후
-> 화면 컴포넌트는 목록 렌더링에 집중
-> 타일 컴포넌트는 개별 아이템 UI에 집중
```

프론트엔드 실무에서도 리스트 화면을 만들 때 아이템 컴포넌트를 분리하는 패턴은 매우 흔하다.

## 정리

이번 강의에서는 Meals 앱의 첫 화면인 카테고리 목록 화면을 만들었다.

`screens` 폴더에 `CategoriesScreen`을 만들고, 제공된 `CATEGORIES` 데이터를 `FlatList`로 출력했다.

각 카테고리 항목은 `CategoryGridTile` 컴포넌트로 분리해 나중에 스타일링과 터치 동작을 추가할 수 있는 기반을 마련했다.

아직 화면은 예쁘지 않고 내비게이션도 없지만, 이제 데이터를 화면에 출력하는 기본 구조가 준비되었다.
