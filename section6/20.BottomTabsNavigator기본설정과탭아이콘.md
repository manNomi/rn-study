# 20. Bottom Tabs Navigator 기본 설정과 탭 아이콘

## 핵심 요약

- React Navigation은 Drawer뿐만 아니라 여러 종류의 Tab Navigator도 제공한다.
- Bottom Tabs Navigator는 화면 하단에 탭 바를 두고 주요 화면을 전환하는 네비게이션 패턴이다.
- Bottom Tabs Navigator는 iOS와 Android 모두에서 자연스럽게 사용할 수 있는 범용적인 탭 UI를 제공한다.
- 사용하려면 `@react-navigation/bottom-tabs` 패키지를 설치하고 `createBottomTabNavigator`를 사용한다.
- 설정 구조는 Stack, Drawer와 비슷하다.
  - `NavigationContainer`로 감싼다.
  - `BottomTabs.Navigator`를 둔다.
  - 내부에 `BottomTabs.Screen`들을 등록한다.
- `tabBarIcon` 옵션으로 각 탭에 아이콘을 추가할 수 있다.
- `tabBarIcon` 함수는 React Navigation이 제공하는 `color`, `size` 값을 받아 현재 활성 상태에 맞는 아이콘을 렌더링할 수 있다.
- 탭의 공통 색상은 `screenOptions`에서 `tabBarActiveTintColor` 같은 옵션으로 설정한다.

## 학습 목표

- React Navigation의 Tab Navigator 계열을 이해한다.
- Bottom Tabs Navigator가 어떤 UI 패턴을 구현하는지 파악한다.
- Drawer Navigator 설정을 Bottom Tabs Navigator 설정으로 바꾸는 흐름을 익힌다.
- `createBottomTabNavigator`와 `BottomTabs.Navigator`, `BottomTabs.Screen`의 관계를 이해한다.
- `tabBarIcon`으로 탭 아이콘을 추가하는 방법을 익힌다.
- 탭별 옵션과 공통 `screenOptions`의 역할을 구분한다.

## 주요 내용

### 여러 종류의 Tab Navigator

React Navigation은 여러 탭 네비게이터를 제공한다.

```text
Bottom Tabs
-> 화면 하단 탭
-> iOS와 Android 모두에 적합

Material Top Tabs
-> 화면 상단 탭
-> Android Material Design 느낌

Material Bottom Tabs
-> Material 스타일 하단 탭
-> Android에 더 가까운 외형
```

강의에서는 iOS와 Android 양쪽에서 무난하게 사용할 수 있는 Bottom Tabs Navigator를 사용한다.

공식 문서에는 각 탭 네비게이터의 옵션과 동작 차이가 자세히 정리되어 있으므로, 실제 앱에서는 원하는 UX에 맞게 선택하면 된다.

### Bottom Tabs Navigator 설치

Bottom Tabs Navigator는 별도 패키지로 설치한다.

```bash
npm install @react-navigation/bottom-tabs
```

강의 시점에서는 Drawer와 달리 추가 Expo 패키지를 설치하지 않아도 된다고 설명한다.

다만 실제 프로젝트에서는 사용하는 Expo SDK와 React Navigation 버전에 따라 요구 패키지가 달라질 수 있으므로, 공식 문서의 설치 안내를 확인하는 습관이 필요하다.

### Drawer 설정에서 Bottom Tabs 설정으로 교체

이전 강의에서는 Drawer Navigator를 사용했다.

```jsx
import { createDrawerNavigator } from '@react-navigation/drawer';

const Drawer = createDrawerNavigator();
```

Bottom Tabs를 사용하려면 이 import와 navigator 생성을 바꾼다.

```jsx
import { createBottomTabNavigator } from '@react-navigation/bottom-tabs';

const BottomTabs = createBottomTabNavigator();
```

그리고 JSX에서도 `Drawer.Navigator`, `Drawer.Screen` 대신 `BottomTabs.Navigator`, `BottomTabs.Screen`을 사용한다.

```jsx
<NavigationContainer>
  <BottomTabs.Navigator>
    <BottomTabs.Screen
      name="Welcome"
      component={WelcomeScreen}
    />
    <BottomTabs.Screen
      name="User"
      component={UserScreen}
    />
  </BottomTabs.Navigator>
</NavigationContainer>
```

구조는 매우 비슷하다.

```text
Drawer.Navigator
Drawer.Screen

BottomTabs.Navigator
BottomTabs.Screen
```

차이는 이 설정이 만들어 내는 사용자 경험이다.

Drawer는 사이드 메뉴를 만들고, Bottom Tabs는 하단 탭 바를 만든다.

### Drawer 전용 옵션 제거

Drawer에서 사용하던 옵션은 Bottom Tabs에서 그대로 사용할 수 없다.

예를 들어 다음 옵션들은 Drawer 전용이다.

```text
drawerLabel
drawerIcon
drawerActiveBackgroundColor
drawerActiveTintColor
drawerStyle
```

Bottom Tabs로 전환할 때는 이런 옵션을 제거하고, 탭 전용 옵션으로 바꿔야 한다.

```text
drawerIcon
-> tabBarIcon

drawerActiveTintColor
-> tabBarActiveTintColor
```

React Navigation의 네비게이터들은 공통 패턴을 공유하지만, 각 네비게이터마다 전용 옵션이 있다는 점을 기억해야 한다.

### 기본 탭 바

Bottom Tabs Navigator를 설정하면 화면 하단에 탭 바가 자동으로 렌더링된다.

```text
BottomTabs.Screen 등록
-> 하단 탭 항목 생성
-> 탭을 누르면 해당 화면으로 전환
```

기본 스타일도 제공된다.

따라서 별도의 탭 바 컴포넌트를 직접 만들지 않아도, 등록한 화면들이 자동으로 탭 항목으로 나타난다.

강의에서는 처음에 아이콘이 없지만 탭 자체는 동작한다고 설명한다.

이 상태에서도 `Welcome` 탭과 `User` 탭을 눌러 화면을 전환할 수 있다.

### tabBarIcon으로 아이콘 추가

탭 항목에는 보통 아이콘이 함께 표시된다.

Bottom Tabs Navigator에서는 `tabBarIcon` 옵션을 사용한다.

아이콘은 이전 강의들에서 사용했던 `Ionicons`를 다시 사용할 수 있다.

```jsx
import { Ionicons } from '@expo/vector-icons';
```

`tabBarIcon`은 JSX를 반환하는 함수를 받는다.

```jsx
options={{
  tabBarIcon: ({ color, size }) => {
    return (
      <Ionicons
        name="home"
        color={color}
        size={size}
      />
    );
  },
}}
```

React Navigation이 `color`와 `size`를 자동으로 넘겨준다.

```text
color
-> 현재 탭이 활성인지 아닌지에 따라 정해진 색상

size
-> 탭 바가 기대하는 아이콘 크기
```

이 값을 그대로 사용하면 탭이 선택되었을 때 아이콘 색상이 자동으로 바뀐다.

직접 활성 상태를 계산하거나 스타일을 분기할 필요가 없다.

### 화면별 탭 아이콘 설정

`Welcome` 화면에는 홈 아이콘을 넣을 수 있다.

```jsx
<BottomTabs.Screen
  name="Welcome"
  component={WelcomeScreen}
  options={{
    tabBarIcon: ({ color, size }) => (
      <Ionicons
        name="home"
        color={color}
        size={size}
      />
    ),
  }}
/>
```

`User` 화면에는 사람 아이콘을 넣을 수 있다.

```jsx
<BottomTabs.Screen
  name="User"
  component={UserScreen}
  options={{
    tabBarIcon: ({ color, size }) => (
      <Ionicons
        name="person"
        color={color}
        size={size}
      />
    ),
  }}
/>
```

아이콘은 화면마다 다르므로 각 `BottomTabs.Screen`의 `options`에 두는 편이 자연스럽다.

### 탭 활성 색상 설정

탭의 활성 색상은 공통 설정으로 두는 경우가 많다.

따라서 `BottomTabs.Navigator`의 `screenOptions`에 설정한다.

```jsx
<BottomTabs.Navigator
  screenOptions={{
    tabBarActiveTintColor: '#3c0a6b',
  }}
>
  ...
</BottomTabs.Navigator>
```

이제 선택된 탭의 텍스트와 아이콘 색상이 기본 파란색이 아니라 지정한 보라색으로 표시된다.

React Navigation 버전에 따라 문서나 강의에서 `activeTintColor`라는 표현을 볼 수 있다.

현재 Bottom Tabs 설정에서는 보통 `tabBarActiveTintColor`, `tabBarInactiveTintColor`처럼 `tabBar` 접두사가 붙은 옵션을 사용한다.

### 더 많은 탭 옵션

Bottom Tabs Navigator는 다양한 옵션을 제공한다.

예를 들어 다음과 같은 설정을 조정할 수 있다.

```text
tabBarLabel
-> 탭에 표시될 텍스트

tabBarShowLabel
-> 라벨 표시 여부

tabBarIcon
-> 탭 아이콘

tabBarActiveTintColor
-> 활성 탭 색상

tabBarInactiveTintColor
-> 비활성 탭 색상

tabBarStyle
-> 탭 바 전체 스타일
```

탭 바를 완전히 커스텀 컴포넌트로 대체할 수도 있다.

하지만 대부분의 경우에는 기본 탭 바를 사용하고, 옵션으로 색상과 아이콘만 조정해도 충분히 자연스러운 결과를 얻을 수 있다.

## 예시 코드

### App.js

```jsx
import { NavigationContainer } from '@react-navigation/native';
import { createBottomTabNavigator } from '@react-navigation/bottom-tabs';
import { Ionicons } from '@expo/vector-icons';

import WelcomeScreen from './screens/WelcomeScreen';
import UserScreen from './screens/UserScreen';

const BottomTabs = createBottomTabNavigator();

export default function App() {
  return (
    <NavigationContainer>
      <BottomTabs.Navigator
        screenOptions={{
          tabBarActiveTintColor: '#3c0a6b',
        }}
      >
        <BottomTabs.Screen
          name="Welcome"
          component={WelcomeScreen}
          options={{
            tabBarIcon: ({ color, size }) => (
              <Ionicons
                name="home"
                color={color}
                size={size}
              />
            ),
          }}
        />
        <BottomTabs.Screen
          name="User"
          component={UserScreen}
          options={{
            tabBarIcon: ({ color, size }) => (
              <Ionicons
                name="person"
                color={color}
                size={size}
              />
            ),
          }}
        />
      </BottomTabs.Navigator>
    </NavigationContainer>
  );
}
```

## 프론트엔드 개발자 관점에서의 핵심 차이

### Bottom Tabs는 앱의 주요 섹션 전환에 적합하다

Stack Navigator는 깊이 있는 흐름에 적합하다.

```text
목록
-> 상세
-> 더 상세한 화면
```

Bottom Tabs는 앱의 주요 영역을 빠르게 오갈 때 적합하다.

```text
홈
검색
즐겨찾기
프로필
```

웹에서 상단 내비게이션이나 주요 메뉴 탭으로 섹션을 전환하는 느낌과 비슷하지만, 모바일에서는 하단 탭이 엄지 손가락 접근성 측면에서 자주 사용된다.

### 네비게이터마다 옵션 이름이 다르다

Drawer에서 쓰던 옵션을 Tabs에 그대로 가져오면 동작하지 않는다.

```text
Drawer
-> drawerLabel
-> drawerIcon
-> drawerActiveTintColor

Bottom Tabs
-> tabBarLabel
-> tabBarIcon
-> tabBarActiveTintColor
```

공통 구조는 비슷하지만, 옵션은 네비게이터별로 문서를 확인해야 한다.

이 점은 React Navigation을 사용할 때 꽤 중요하다.

### 아이콘 콜백 패턴은 Drawer와 비슷하다

Drawer의 `drawerIcon`과 Bottom Tabs의 `tabBarIcon`은 비슷한 형태다.

```jsx
({ color, size }) => (
  <Ionicons name="home" color={color} size={size} />
)
```

React Navigation이 활성 상태에 맞는 `color`와 적절한 `size`를 넘겨주고, 개발자는 그 값을 아이콘 컴포넌트에 연결한다.

이 패턴을 이해하면 Drawer와 Tabs를 모두 더 쉽게 다룰 수 있다.

### 기본 UI를 먼저 쓰고 필요한 부분만 조정한다

Bottom Tabs Navigator는 탭 바의 배치, 탭 전환, 활성 상태 스타일, 안전 영역 처리 등을 기본으로 제공한다.

따라서 처음부터 탭 바를 직접 만들 필요가 없다.

```text
먼저 기본 탭 바 사용
-> 아이콘 추가
-> 색상 조정
-> 필요할 때 라벨/스타일 커스터마이징
-> 정말 필요할 때 커스텀 탭 바 구현
```

이 순서가 유지보수와 개발 속도 면에서 유리하다.

## 정리

이번 강의에서는 Bottom Tabs Navigator를 사용해 화면 하단 탭 기반 네비게이션을 구성했다.

`@react-navigation/bottom-tabs`를 설치하고, `createBottomTabNavigator`로 네비게이터를 만든 뒤 `BottomTabs.Navigator`와 `BottomTabs.Screen`으로 화면을 등록했다.

또한 `tabBarIcon`으로 각 탭에 `Ionicons` 아이콘을 추가하고, `screenOptions`의 `tabBarActiveTintColor`로 활성 탭 색상을 공통 설정했다.

이제 Drawer와 Bottom Tabs의 기본 사용법을 모두 살펴봤으므로, 이후에는 이러한 네비게이터를 기존 Meals 앱의 Stack Navigator와 조합하는 흐름으로 이어질 수 있다.
