# 6. Stack Screen 등록과 navigate 화면 이동

## 핵심 요약

- 이번 강의에서는 카테고리 타일을 눌렀을 때 새 화면으로 이동하는 기능을 추가한다.
- 두 번째 화면으로 `MealsOverviewScreen`을 만들고, 선택한 카테고리에 속한 음식 목록을 보여줄 준비를 한다.
- 실제 음식 필터링은 다음 단계에서 처리하고, 이번에는 화면 이동이 되는지 확인하기 위한 더미 텍스트를 먼저 출력한다.
- 새 화면을 사용하려면 `Stack.Navigator` 안에 `Stack.Screen`으로 등록해야 한다.
- 화면 등록만으로는 이동이 발생하지 않는다. 사용자가 타일을 눌렀을 때 명시적으로 `navigation.navigate()`를 호출해야 한다.
- React Navigation은 `Stack.Screen`에 등록된 화면 컴포넌트에 `navigation` prop을 자동으로 주입한다.
- `CategoryGridTile`은 재사용 가능한 dumb component로 두고, 실제 이동 로직은 부모인 `CategoriesScreen`에서 결정한다.

## 학습 목표

- Stack Navigator에 두 번째 화면을 등록하는 방법을 익힌다.
- `Stack.Screen`의 `name`과 `component`가 실제 이동에서 어떤 역할을 하는지 이해한다.
- React Navigation이 화면 컴포넌트에 주입하는 `navigation` prop을 이해한다.
- `navigation.navigate('ScreenName')`으로 다른 화면으로 이동하는 방법을 익힌다.
- 자식 UI 컴포넌트에 `onPress`를 전달해 부모가 동작을 결정하게 만드는 패턴을 복습한다.
- Stack Navigation이 기본으로 제공하는 애니메이션과 뒤로가기 버튼을 확인한다.

## 주요 내용

### 두 번째 화면이 필요한 이유

현재 앱에는 카테고리 목록 화면만 있다.

다음 단계에서는 사용자가 카테고리를 탭했을 때, 해당 카테고리에 속한 음식 목록을 보여줘야 한다.

```text
현재 화면
-> CategoriesScreen

추가할 화면
-> MealsOverviewScreen
```

Meals 앱의 기본 흐름은 다음과 같다.

```text
CategoriesScreen
-> 카테고리 선택
-> MealsOverviewScreen
-> 선택한 카테고리에 속한 음식 목록 표시
```

이번 강의에서는 먼저 `MealsOverviewScreen`을 추가하고, 화면 이동이 되는지 확인한다.

실제 음식 데이터를 필터링해서 출력하는 작업은 이후 단계에서 다룬다.

### MealsOverviewScreen 추가

`screens` 폴더에 `MealsOverviewScreen.js` 파일을 만든다.

```text
screens/
  CategoriesScreen.js
  MealsOverviewScreen.js
```

컴포넌트는 일반 함수형 컴포넌트로 작성한다.

```jsx
import { StyleSheet, Text, View } from 'react-native';

function MealsOverviewScreen() {
  return (
    <View style={styles.container}>
      <Text>Meals Overview Screen</Text>
    </View>
  );
}

export default MealsOverviewScreen;

const styles = StyleSheet.create({
  container: {
    flex: 1,
    padding: 16,
  },
});
```

아직 음식 목록을 출력하지 않고, 화면 이동을 확인하기 위한 텍스트만 넣는다.

```text
이번 단계의 목적
-> 새 화면 컴포넌트 생성
-> Stack에 등록
-> 카테고리 타일 클릭 시 이동 확인
```

### 음식 데이터는 나중에 필터링

더미 데이터에는 `MEALS` 배열이 있다.

각 음식 객체는 자신이 속한 카테고리 id 목록을 가지고 있다.

```text
Meal
-> id
-> title
-> categoryIds
-> imageUrl
-> duration
-> complexity
-> affordability
-> ingredients
-> steps
```

나중에는 선택한 카테고리 id를 기준으로 `MEALS` 배열을 필터링한다.

```jsx
const displayedMeals = MEALS.filter((meal) => {
  return meal.categoryIds.includes(selectedCategoryId);
});
```

하지만 이번 강의에서는 아직 선택한 카테고리 id를 전달하지 않는다.

먼저 화면 이동 자체부터 성공시키는 것이 목표다.

### Stack.Screen에 두 번째 화면 등록

`App.js`의 `Stack.Navigator` 안에 두 번째 화면을 등록한다.

```jsx
import MealsOverviewScreen from './screens/MealsOverviewScreen';
```

```jsx
<Stack.Navigator>
  <Stack.Screen
    name="MealsCategories"
    component={CategoriesScreen}
  />
  <Stack.Screen
    name="MealsOverview"
    component={MealsOverviewScreen}
  />
</Stack.Navigator>
```

`name`은 해당 화면의 고유 식별자다.

나중에 `navigate`를 호출할 때 이 이름을 사용한다.

```text
MealsCategories
-> 카테고리 목록 화면

MealsOverview
-> 선택한 카테고리의 음식 목록 화면
```

`component`에는 JSX가 아니라 컴포넌트 함수 자체를 전달한다.

```jsx
component={MealsOverviewScreen}
```

### 화면 등록만으로는 이동하지 않는다

`Stack.Screen`에 화면을 등록했다고 해서 React Navigation이 자동으로 이동해주지는 않는다.

사용자가 어떤 행동을 했을 때 어느 화면으로 이동해야 하는지 명확히 알려줘야 한다.

```text
화면 등록
-> Navigator가 해당 화면을 알게 됨

화면 이동
-> 특정 이벤트에서 navigation.navigate 호출 필요
```

React Navigation은 앱이 어떤 UI를 눌렀을 때 어디로 이동해야 하는지 추측하지 않는다.

개발자가 명시적으로 이동 조건을 작성해야 한다.

### CategoryGridTile에 onPress prop 추가

카테고리 타일을 누르면 이동해야 한다.

하지만 `CategoryGridTile` 컴포넌트 안에 직접 내비게이션 로직을 넣지는 않는다.

타일 컴포넌트는 가능한 단순하고 재사용 가능한 UI 컴포넌트로 두는 편이 좋다.

그래서 `CategoryGridTile`은 `onPress` prop을 받아 `Pressable`에 전달한다.

```jsx
function CategoryGridTile({ title, color, onPress }) {
  return (
    <View style={styles.gridItem}>
      <Pressable
        android_ripple={{ color: '#ccc' }}
        style={({ pressed }) => [
          styles.button,
          pressed ? styles.buttonPressed : null,
        ]}
        onPress={onPress}
      >
        <View style={[styles.innerContainer, { backgroundColor: color }]}>
          <Text style={styles.title}>{title}</Text>
        </View>
      </Pressable>
    </View>
  );
}
```

이렇게 하면 타일은 “눌렸을 때 실행할 함수”만 외부에서 받는다.

무엇을 할지는 부모 컴포넌트가 결정한다.

```text
CategoryGridTile
-> UI와 터치 영역 담당
-> onPress 실행만 담당

CategoriesScreen
-> 실제로 어떤 화면으로 이동할지 결정
```

### navigation prop

React Navigation은 `Stack.Screen`에 등록된 화면 컴포넌트에 특별한 prop을 주입한다.

그중 하나가 `navigation` prop이다.

```jsx
function CategoriesScreen({ navigation }) {
  ...
}
```

이 prop은 `CategoriesScreen`이 `Stack.Screen`의 `component`로 등록되어 있기 때문에 받을 수 있다.

```jsx
<Stack.Screen
  name="MealsCategories"
  component={CategoriesScreen}
/>
```

중요한 점은 `navigation` prop이 모든 하위 컴포넌트에 자동으로 들어가는 것은 아니라는 점이다.

```text
navigation prop을 받는 컴포넌트
-> Stack.Screen에 직접 등록된 화면 컴포넌트

자동으로 받지 않는 컴포넌트
-> CategoryGridTile 같은 일반 자식 컴포넌트
```

따라서 자식 컴포넌트에서 이동이 필요하면 부모 화면 컴포넌트가 함수를 만들어 prop으로 내려주는 방식이 자연스럽다.

### renderCategoryItem을 컴포넌트 안으로 옮기기

이전에는 `renderCategoryItem`을 `CategoriesScreen` 바깥에 둘 수 있었다.

하지만 이제 `navigation` 객체를 사용해야 한다.

`navigation`은 `CategoriesScreen` 함수의 매개변수로 들어오는 prop이므로, 이를 사용하려면 `renderCategoryItem`을 컴포넌트 안에 정의하는 것이 간단하다.

```jsx
function CategoriesScreen({ navigation }) {
  function renderCategoryItem(itemData) {
    function pressHandler() {
      navigation.navigate('MealsOverview');
    }

    return (
      <CategoryGridTile
        title={itemData.item.title}
        color={itemData.item.color}
        onPress={pressHandler}
      />
    );
  }

  return (
    <FlatList
      data={CATEGORIES}
      keyExtractor={(item) => item.id}
      renderItem={renderCategoryItem}
      numColumns={2}
    />
  );
}
```

이제 `pressHandler`는 `navigation`에 접근할 수 있다.

이 함수가 카테고리 타일의 `onPress`로 전달되고, 타일이 눌리면 실행된다.

### navigation.navigate

다른 화면으로 이동하려면 `navigation.navigate()`를 호출한다.

```jsx
navigation.navigate('MealsOverview');
```

인자로 전달하는 문자열은 `Stack.Screen`에 등록한 `name`과 같아야 한다.

```jsx
<Stack.Screen
  name="MealsOverview"
  component={MealsOverviewScreen}
/>
```

즉 `navigate`의 대상은 파일명이나 컴포넌트명이 아니라 screen name이다.

```text
navigation.navigate('MealsOverview')
-> name이 MealsOverview인 Stack.Screen으로 이동
```

이 이름이 틀리면 React Navigation은 이동할 화면을 찾지 못한다.

### 자동으로 생기는 애니메이션과 뒤로가기

이제 카테고리 타일을 누르면 `MealsOverviewScreen`으로 이동한다.

React Navigation의 Stack Navigator는 기본적으로 다음을 제공한다.

```text
자동 제공 기능
-> 화면 전환 애니메이션
-> 상단 헤더
-> 뒤로가기 버튼
-> 플랫폼별 기본 전환 느낌
```

iOS와 Android의 화면 전환 애니메이션은 서로 다를 수 있다.

React Navigation이 각 플랫폼에 자연스러운 방식으로 처리해준다.

이전 화면으로 돌아가는 버튼도 헤더에 자동으로 추가된다.

```text
CategoriesScreen
-> MealsOverviewScreen
-> Back 버튼
-> CategoriesScreen으로 복귀
```

이것이 상태 기반 화면 교체와 Stack Navigation의 큰 차이다.

## 예시 코드

### App.js

```jsx
import { StatusBar } from 'expo-status-bar';
import { NavigationContainer } from '@react-navigation/native';
import { createNativeStackNavigator } from '@react-navigation/native-stack';

import CategoriesScreen from './screens/CategoriesScreen';
import MealsOverviewScreen from './screens/MealsOverviewScreen';

const Stack = createNativeStackNavigator();

export default function App() {
  return (
    <>
      <StatusBar style="dark" />
      <NavigationContainer>
        <Stack.Navigator>
          <Stack.Screen
            name="MealsCategories"
            component={CategoriesScreen}
          />
          <Stack.Screen
            name="MealsOverview"
            component={MealsOverviewScreen}
          />
        </Stack.Navigator>
      </NavigationContainer>
    </>
  );
}
```

### CategoriesScreen

```jsx
function CategoriesScreen({ navigation }) {
  function renderCategoryItem(itemData) {
    function pressHandler() {
      navigation.navigate('MealsOverview');
    }

    return (
      <CategoryGridTile
        title={itemData.item.title}
        color={itemData.item.color}
        onPress={pressHandler}
      />
    );
  }

  return (
    <FlatList
      data={CATEGORIES}
      keyExtractor={(item) => item.id}
      renderItem={renderCategoryItem}
      numColumns={2}
    />
  );
}
```

### CategoryGridTile

```jsx
function CategoryGridTile({ title, color, onPress }) {
```

## 프론트엔드 개발자 관점에서의 핵심 차이

### Link 대신 navigate

웹 React에서는 보통 링크나 라우터 함수를 사용한다.

```jsx
<Link to="/categories/c1">Italian</Link>
```

또는

```jsx
navigate('/categories/c1');
```

React Native에서는 URL 대신 screen name을 사용한다.

```jsx
navigation.navigate('MealsOverview');
```

화면 이동 대상이 path가 아니라 Stack에 등록된 screen name이라는 점이 중요하다.

### navigation prop은 화면 컴포넌트의 특권

React Navigation은 screen으로 등록된 컴포넌트에만 `navigation` prop을 자동으로 넣어준다.

따라서 일반 UI 컴포넌트인 `CategoryGridTile`에서는 `navigation`을 바로 쓸 수 없다.

```text
좋은 구조
-> 화면 컴포넌트에서 navigation 사용
-> UI 컴포넌트에는 onPress prop 전달

피하고 싶은 구조
-> 모든 작은 UI 컴포넌트가 navigation에 직접 의존
```

이렇게 하면 `CategoryGridTile`은 내비게이션과 무관한 재사용 가능한 컴포넌트로 남는다.

### screen name은 라우트 이름과 비슷하다

React Router의 path와 React Navigation의 screen name은 비슷한 역할을 한다.

하지만 사용자에게 보이는 URL이 아니라 내부 식별자다.

```text
React Router
-> /meals/:categoryId

React Navigation
-> MealsOverview
```

나중에 데이터를 전달할 때는 screen name과 함께 params를 넘긴다.

이번 강의에서는 아직 params를 넘기지 않고 화면 이동만 한다.

### 자동 헤더와 뒤로가기의 가치

이전 게임 앱에서는 화면이 바뀌어도 뒤로가기나 애니메이션이 없었다.

React Navigation을 쓰면 이 기능들이 기본으로 제공된다.

```text
직접 구현하면 필요한 것
-> 화면 상태 관리
-> 이전 화면 저장
-> 뒤로가기 버튼 UI
-> 플랫폼별 애니메이션

React Navigation 사용 시
-> Stack에 등록
-> navigate 호출
-> 나머지는 기본 제공
```

이것이 실제 앱에서 내비게이션 라이브러리를 사용하는 가장 큰 이유다.

## 정리

이번 강의에서는 `MealsOverviewScreen`을 추가하고 Stack Navigator에 두 번째 화면으로 등록했다.

이후 `CategoriesScreen`에서 `navigation.navigate('MealsOverview')`를 호출해 카테고리 타일을 누르면 새 화면으로 이동하도록 만들었다.

이제 앱은 단순히 컴포넌트를 교체하는 수준을 넘어, 애니메이션과 뒤로가기 버튼이 있는 실제 모바일 내비게이션 흐름을 갖게 되었다.

다음 단계에서는 선택한 카테고리 정보를 새 화면에 전달하고, 그 카테고리에 속한 음식 목록을 출력하게 된다.
