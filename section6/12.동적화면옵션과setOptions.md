# 12. 동적 화면 옵션과 setOptions

## 핵심 요약

- 이전 강의에서는 `options`와 `screenOptions`로 정적인 화면 옵션을 설정했다.
- 이번 강의에서는 선택한 카테고리에 따라 헤더 제목을 동적으로 바꾸는 방법을 배운다.
- `Stack.Screen`의 `options`에는 객체뿐 아니라 함수도 전달할 수 있다.
- `options` 함수는 React Navigation이 실행하며, `route`와 `navigation`을 인자로 받아 동적 옵션 객체를 반환할 수 있다.
- 또 다른 방법은 화면 컴포넌트 내부에서 `navigation.setOptions()`를 호출하는 것이다.
- `setOptions()`는 화면 옵션을 나중에 변경할 수 있게 해준다.
- 컴포넌트 함수 본문에서 직접 `setOptions()`를 호출하면 경고가 발생할 수 있으므로 `useEffect` 또는 `useLayoutEffect` 안에서 처리해야 한다.
- 헤더 제목처럼 전환 애니메이션 중 바로 반영되어야 하는 옵션은 `useLayoutEffect`가 더 자연스럽다.

## 학습 목표

- route params를 기반으로 화면 제목을 동적으로 설정하는 방법을 이해한다.
- `Stack.Screen options`에 함수를 전달하는 패턴을 익힌다.
- `navigation.setOptions()`의 역할을 이해한다.
- `useEffect`와 `useLayoutEffect`의 차이를 내비게이션 옵션 설정 맥락에서 이해한다.
- `CATEGORIES` 데이터에서 `categoryId`로 카테고리 제목을 찾는 흐름을 복습한다.
- StatusBar 스타일을 헤더 배경에 맞게 조정하는 이유를 이해한다.

## 주요 내용

### 왜 동적 옵션이 필요한가?

현재 `MealsOverviewScreen`의 헤더 제목은 정적인 이름이다.

예를 들어 모든 카테고리에서 같은 제목이 보일 수 있다.

```text
MealsOverview
```

하지만 사용자가 `Italian` 카테고리를 눌렀다면 헤더 제목도 `Italian`으로 보이는 것이 더 자연스럽다.

```text
Italian 클릭
-> 헤더 제목: Italian

German 클릭
-> 헤더 제목: German

Quick & Easy 클릭
-> 헤더 제목: Quick & Easy
```

즉 화면 옵션이 route params나 현재 데이터에 따라 달라져야 한다.

이런 경우 정적인 객체 형태의 `options`만으로는 부족하다.

### 방법 1: options에 함수 전달하기

`Stack.Screen`의 `options` prop에는 객체를 전달할 수 있었다.

```jsx
<Stack.Screen
  name="MealsOverview"
  component={MealsOverviewScreen}
  options={{
    title: 'Meals Overview',
  }}
/>
```

하지만 객체 대신 함수를 전달할 수도 있다.

```jsx
<Stack.Screen
  name="MealsOverview"
  component={MealsOverviewScreen}
  options={({ route, navigation }) => {
    return {
      title: route.params.categoryId,
    };
  }}
/>
```

이 함수는 React Navigation이 실행한다.

함수는 `route`와 `navigation`을 포함한 객체를 인자로 받는다.

```text
options 함수가 받는 값
-> route
-> navigation
```

그리고 반환값은 일반 `options` 객체와 같은 형태여야 한다.

```jsx
return {
  title: 'Some Title',
};
```

### route params로 title 설정하기

이전 강의에서 `MealsOverviewScreen`으로 이동할 때 `categoryId`를 전달했다.

```jsx
navigation.navigate('MealsOverview', {
  categoryId: itemData.item.id,
});
```

따라서 `options` 함수 안에서도 `route.params.categoryId`를 읽을 수 있다.

```jsx
options={({ route }) => {
  const catId = route.params.categoryId;

  return {
    title: catId,
  };
}}
```

이렇게 하면 헤더 제목으로 `c1`, `c4`, `c8` 같은 카테고리 id가 표시된다.

실제로 필요한 것은 id가 아니라 카테고리 이름이므로, `CATEGORIES` 데이터에서 해당 id를 찾아 title을 꺼내면 된다.

```jsx
options={({ route }) => {
  const catId = route.params.categoryId;
  const categoryTitle = CATEGORIES.find(
    (category) => category.id === catId
  ).title;

  return {
    title: categoryTitle,
  };
}}
```

이 방식은 화면 설정 위치인 `App.js`에서 모든 옵션을 관리할 수 있다는 장점이 있다.

### 방법 2: 화면 컴포넌트 내부에서 setOptions 사용

대안으로 화면 컴포넌트 안에서 `navigation.setOptions()`를 호출할 수 있다.

`MealsOverviewScreen`은 `Stack.Screen`으로 등록된 화면이므로 `route`와 `navigation` prop을 모두 받을 수 있다.

```jsx
function MealsOverviewScreen({ route, navigation }) {
  const catId = route.params.categoryId;
}
```

`navigation` 객체에는 `navigate`뿐 아니라 `setOptions` 메서드도 있다.

```jsx
navigation.setOptions({
  title: 'Some Title',
});
```

이 메서드를 사용하면 현재 화면의 옵션을 컴포넌트 내부에서 변경할 수 있다.

### CATEGORIES에서 제목 찾기

먼저 `CATEGORIES`를 import한다.

```jsx
import { CATEGORIES, MEALS } from '../data/dummy-data';
```

그 다음 `catId`와 일치하는 카테고리를 찾는다.

```jsx
const categoryTitle = CATEGORIES.find((category) => {
  return category.id === catId;
}).title;
```

`find()`는 JavaScript 배열 메서드다.

조건에 맞는 첫 번째 항목을 반환한다.

```text
find 콜백이 true 반환
-> 해당 항목 반환

끝까지 못 찾음
-> undefined 반환
```

이번 앱에서는 `MealsOverviewScreen`에 도달하는 유일한 경로가 유효한 카테고리 id를 전달하는 경로이므로, 결과가 없을 가능성은 거의 없다.

실무에서는 그래도 undefined 방어를 하는 편이 안전하다.

### 컴포넌트 본문에서 setOptions를 바로 호출하면 안 되는 이유

처음에는 다음처럼 컴포넌트 함수 본문에서 바로 `setOptions()`를 호출할 수 있어 보인다.

```jsx
function MealsOverviewScreen({ route, navigation }) {
  const catId = route.params.categoryId;
  const categoryTitle = CATEGORIES.find(
    (category) => category.id === catId
  ).title;

  navigation.setOptions({
    title: categoryTitle,
  });

  ...
}
```

하지만 이 방식은 경고를 만들 수 있다.

화면 옵션을 설정하는 것은 렌더링 중에 바로 수행할 작업이 아니라 side effect에 가깝기 때문이다.

```text
setOptions
-> 현재 화면의 외부 설정을 변경
-> 렌더 결과 계산 자체가 아님
-> side effect로 다루는 것이 적절
```

따라서 `useEffect` 또는 `useLayoutEffect` 안에서 처리해야 한다.

### useEffect로 setOptions 처리하기

먼저 React에서 `useEffect`를 가져온다.

```jsx
import { useEffect } from 'react';
```

그리고 `setOptions`를 effect 안에서 호출한다.

```jsx
useEffect(() => {
  const categoryTitle = CATEGORIES.find((category) => {
    return category.id === catId;
  }).title;

  navigation.setOptions({
    title: categoryTitle,
  });
}, [catId, navigation]);
```

의존성 배열에는 effect 안에서 사용하는 외부 값을 넣는다.

```text
dependencies
-> catId
-> navigation
```

`CATEGORIES`는 컴포넌트 내부 값이 아니라 import된 모듈 상수이므로 의존성에 넣을 필요는 없다.

이제 경고는 사라진다.

하지만 화면이 먼저 렌더링되고 나서 제목이 뒤늦게 바뀌는 느낌이 보일 수 있다.

### useEffect의 타이밍 문제

`useEffect`는 컴포넌트가 렌더링된 뒤 실행된다.

따라서 화면 전환 애니메이션 중에는 기본 제목이 잠깐 보이고, 이후에 카테고리 제목으로 바뀔 수 있다.

```text
useEffect 동작 흐름
-> 컴포넌트 렌더링
-> 화면 표시
-> effect 실행
-> setOptions로 제목 변경
```

헤더 제목처럼 화면 전환 중 자연스럽게 보여야 하는 값에는 이 타이밍이 어색할 수 있다.

### useLayoutEffect 사용

이럴 때는 `useLayoutEffect`를 사용할 수 있다.

```jsx
import { useLayoutEffect } from 'react';
```

사용 방식은 `useEffect`와 거의 같다.

```jsx
useLayoutEffect(() => {
  const categoryTitle = CATEGORIES.find((category) => {
    return category.id === catId;
  }).title;

  navigation.setOptions({
    title: categoryTitle,
  });
}, [catId, navigation]);
```

`useLayoutEffect`는 레이아웃 반영 타이밍에 더 가깝게 실행되어, 화면이 사용자에게 그려지기 전에 동기적으로 처리해야 하는 UI 관련 side effect에 적합하다.

강의에서는 이를 사용하면 화면 전환 애니메이션 중에도 제목이 자연스럽게 설정된다고 설명한다.

```text
useEffect
-> 렌더링 이후 실행
-> 제목이 나중에 바뀌는 느낌이 보일 수 있음

useLayoutEffect
-> 화면이 그려지기 전/레이아웃 단계에 가깝게 실행
-> 헤더 제목 변경이 더 매끄러움
```

평소에는 `useEffect`를 더 많이 쓰지만, 내비게이션 헤더나 레이아웃 관련 즉시 반영이 필요한 경우 `useLayoutEffect`가 유용할 수 있다.

### React Navigation이 제공하는 뒤로가기 텍스트 처리

React Navigation은 헤더에 뒤로가기 버튼을 자동으로 제공한다.

또 이전 화면의 제목을 뒤로가기 텍스트로 보여주거나, 현재 화면 제목이 너무 길면 단순히 `Back`으로 줄이는 등 여러 기본 동작을 알아서 처리한다.

```text
자동 처리 예시
-> 이전 화면 제목 표시
-> 제목이 길면 Back으로 축약
-> 플랫폼별 뒤로가기 스타일 적용
```

이런 부분이 React Navigation을 사용할 때 얻는 편리함이다.

### StatusBar 스타일 조정

헤더 배경을 어둡게 바꾸면 상태바도 잘 보이도록 조정해야 한다.

`App.js`에서 `StatusBar`의 스타일을 `light`로 바꾼다.

```jsx
<StatusBar style="light" />
```

상태바는 화면 상단 시스템 UI이므로, 헤더 배경과 대비를 맞추는 것이 중요하다.

## 예시 코드

### options 함수 방식

```jsx
<Stack.Screen
  name="MealsOverview"
  component={MealsOverviewScreen}
  options={({ route }) => {
    const catId = route.params.categoryId;
    const categoryTitle = CATEGORIES.find((category) => {
      return category.id === catId;
    }).title;

    return {
      title: categoryTitle,
    };
  }}
/>
```

### setOptions + useLayoutEffect 방식

```jsx
import { useLayoutEffect } from 'react';
import { CATEGORIES, MEALS } from '../data/dummy-data';

function MealsOverviewScreen({ route, navigation }) {
  const catId = route.params.categoryId;

  const displayedMeals = MEALS.filter((mealItem) => {
    return mealItem.categoryIds.indexOf(catId) >= 0;
  });

  useLayoutEffect(() => {
    const categoryTitle = CATEGORIES.find((category) => {
      return category.id === catId;
    }).title;

    navigation.setOptions({
      title: categoryTitle,
    });
  }, [catId, navigation]);

  // FlatList 렌더링
}
```

## 프론트엔드 개발자 관점에서의 핵심 차이

### document.title 설정과 비슷한 감각

웹에서 route params에 따라 `document.title`을 바꾸거나 페이지 헤더를 바꾸는 경우가 있다.

```jsx
useEffect(() => {
  document.title = categoryTitle;
}, [categoryTitle]);
```

React Navigation의 `setOptions`도 비슷한 감각으로 이해할 수 있다.

```jsx
navigation.setOptions({
  title: categoryTitle,
});
```

다만 여기서는 브라우저 문서 제목이 아니라 네이티브 내비게이션 헤더 옵션을 바꾼다.

### 동적 옵션의 위치 선택

동적 옵션을 설정하는 위치는 두 가지가 있다.

```text
App.js의 Stack.Screen options 함수
-> 내비게이션 설정을 한곳에 모음
-> route 정보를 기반으로 동적 옵션 반환

Screen 컴포넌트 내부 setOptions
-> 화면 내부 데이터 계산과 가까움
-> 화면 컴포넌트가 직접 헤더를 설정
```

둘 다 유효하다.

앱 구조와 팀 컨벤션에 맞춰 선택하면 된다.

### useLayoutEffect는 자주 쓰진 않지만 알아두면 좋다

웹 프론트엔드에서도 `useLayoutEffect`는 일반적인 데이터 fetching에는 잘 쓰지 않는다.

하지만 레이아웃이나 화면 표시 직전 동기적 반영이 필요한 경우에는 의미가 있다.

React Native 내비게이션에서는 헤더 제목이 전환 애니메이션 중 깜빡이지 않게 설정할 때 유용할 수 있다.

```text
일반 side effect
-> useEffect

레이아웃/헤더 등 즉시 반영이 중요한 UI effect
-> useLayoutEffect 고려
```

### params 이름과 데이터 조회는 연결되어 있다

`categoryId`는 단순히 화면 간 전달되는 값이 아니라, 화면의 데이터 조회 기준이다.

```text
route.params.categoryId
-> CATEGORIES에서 제목 찾기
-> MEALS에서 목록 필터링
-> 헤더 title 설정
```

하나의 route param이 화면의 데이터와 UI 설정을 동시에 결정할 수 있다.

## 정리

이번 강의에서는 route params를 기반으로 Stack 화면 옵션을 동적으로 설정하는 방법을 배웠다.

`Stack.Screen`의 `options`에 함수를 전달해 동적 객체를 반환할 수도 있고, 화면 컴포넌트 내부에서 `navigation.setOptions()`를 호출할 수도 있다.

헤더 제목처럼 화면 전환 중 자연스럽게 반영되어야 하는 값은 `useLayoutEffect` 안에서 `setOptions()`를 호출하면 더 매끄럽게 처리할 수 있다.
