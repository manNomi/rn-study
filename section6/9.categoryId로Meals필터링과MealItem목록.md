# 9. categoryId로 Meals 필터링과 MealItem 목록

## 핵심 요약

- 이전 강의에서 `CategoriesScreen`에서 `MealsOverviewScreen`으로 `categoryId`를 전달했다.
- 이번 강의에서는 전달받은 `categoryId`를 사용해 해당 카테고리에 속한 음식만 필터링한다.
- `MEALS` 데이터의 각 음식은 여러 카테고리에 속할 수 있으므로 `categoryIds` 배열을 가진다.
- `Array.prototype.filter()`와 `indexOf()`를 이용해 선택한 카테고리 id를 포함하는 음식만 추린다.
- 필터링된 음식 목록은 `FlatList`로 출력한다.
- 각 음식 아이템은 앞으로 복잡한 UI가 될 예정이므로 `MealItem` 컴포넌트로 분리한다.

## 학습 목표

- 전달받은 route params를 실제 데이터 필터링에 사용하는 방법을 익힌다.
- 배열 데이터에서 특정 조건에 맞는 항목만 필터링하는 흐름을 복습한다.
- 음식이 여러 카테고리에 속할 수 있는 데이터 구조를 이해한다.
- `FlatList`로 필터링된 목록을 렌더링하는 방법을 복습한다.
- 목록 아이템 UI를 `MealItem` 컴포넌트로 분리하는 기준을 이해한다.
- 이후 상세 화면 이동을 고려해 `renderMealItem`을 컴포넌트 내부에 둘 수 있음을 이해한다.

## 주요 내용

### 전달받은 categoryId 사용하기

이전 단계에서 `CategoriesScreen`은 선택한 카테고리 id를 params로 전달했다.

```jsx
navigation.navigate('MealsOverview', {
  categoryId: itemData.item.id,
});
```

`MealsOverviewScreen`에서는 이를 `route.params`에서 읽었다.

```jsx
const catId = route.params.categoryId;
```

이제 이 `catId`를 사용해 `MEALS` 배열에서 해당 카테고리에 속한 음식만 가져온다.

### Meal 데이터 구조

더미 데이터에서 음식 객체는 하나의 카테고리에만 속하지 않을 수 있다.

하나의 음식이 여러 카테고리에 들어갈 수 있으므로, 음식 객체는 `categoryIds` 배열을 가진다.

```text
Meal
-> id
-> title
-> categoryIds
-> imageUrl
-> duration
-> complexity
-> affordability
-> ingredients
-> steps
```

예를 들어 어떤 음식은 다음처럼 여러 카테고리에 속할 수 있다.

```js
categoryIds: ['c1', 'c2']
```

따라서 카테고리 id 하나와 정확히 같은지 비교하는 것이 아니라, 배열 안에 해당 id가 포함되어 있는지를 확인해야 한다.

### MEALS 필터링

`MEALS` 배열에서 선택한 카테고리에 속한 음식만 남기려면 `filter()`를 사용한다.

```jsx
const displayedMeals = MEALS.filter((mealItem) => {
  return mealItem.categoryIds.indexOf(catId) >= 0;
});
```

`filter()`는 콜백 함수가 `true`를 반환하는 항목만 새 배열에 남긴다.

```text
true 반환
-> 해당 meal 유지

false 반환
-> 해당 meal 제거
```

`mealItem.categoryIds.indexOf(catId)`는 배열 안에서 `catId`의 위치를 찾는다.

```text
indexOf 결과
-> 찾으면 0 이상의 index 반환
-> 못 찾으면 -1 반환
```

따라서 `>= 0`이면 선택한 카테고리에 속한 음식이라는 뜻이다.

```jsx
mealItem.categoryIds.indexOf(catId) >= 0
```

요즘 JavaScript에서는 `includes()`를 써도 같은 의미를 더 읽기 쉽게 표현할 수 있다.

```jsx
const displayedMeals = MEALS.filter((mealItem) => {
  return mealItem.categoryIds.includes(catId);
});
```

강의에서는 `indexOf()`를 사용하지만, 둘 다 같은 목적을 가진다.

### FlatList로 음식 목록 출력

필터링된 `displayedMeals`를 화면에 출력한다.

음식 목록도 현재 더미 데이터에서는 많지 않지만, 실제 앱이라면 훨씬 많아질 수 있다.

따라서 목록 렌더링에는 `FlatList`를 사용한다.

```jsx
import { FlatList } from 'react-native';
```

```jsx
<FlatList
  data={displayedMeals}
  keyExtractor={(item) => item.id}
  renderItem={renderMealItem}
/>
```

여기서 `data`는 필터링된 음식 배열이다.

```text
data
-> displayedMeals
-> 선택한 카테고리에 속한 음식들
```

`keyExtractor`는 각 음식의 고유 id를 key로 사용한다.

```jsx
keyExtractor={(item) => item.id}
```

`Meal` 모델을 보면 각 음식은 `id` 프로퍼티를 가지고 있으므로 key로 적합하다.

### renderMealItem 함수

`FlatList`의 `renderItem`에는 각 음식 항목을 어떻게 렌더링할지 정하는 함수를 전달한다.

```jsx
function renderMealItem(itemData) {
  return (
    <MealItem title={itemData.item.title} />
  );
}
```

`itemData.item`은 현재 렌더링 중인 음식 객체다.

```text
itemData.item
-> 현재 meal 객체

itemData.item.title
-> 음식 제목
```

강의에서는 이 함수를 `MealsOverviewScreen` 컴포넌트 내부에 정의한다.

이유는 나중에 각 음식 아이템을 눌렀을 때 상세 화면으로 이동해야 하며, 그때 `navigation` 같은 화면 props에 접근할 필요가 생길 수 있기 때문이다.

```text
renderMealItem을 내부에 두는 이유
-> 나중에 navigation 사용 가능
-> 상세 화면 이동 로직 추가 예정
```

현재는 단순히 제목만 출력한다.

### MealItem 컴포넌트 추가

음식 아이템은 앞으로 이미지, 제목, 조리 시간, 난이도, 가격 수준 등 여러 정보를 보여줄 예정이다.

따라서 `MealsOverviewScreen` 안에 직접 JSX를 작성하지 않고 `MealItem` 컴포넌트로 분리한다.

```text
components/
  MealItem.js
```

기본 구조는 간단하다.

```jsx
import { Text, View } from 'react-native';

function MealItem({ title }) {
  return (
    <View>
      <Text>{title}</Text>
    </View>
  );
}

export default MealItem;
```

처음에는 제목만 출력하지만, 이후 강의에서 더 복잡한 UI로 확장한다.

```text
MealItem이 나중에 표시할 수 있는 정보
-> 이미지
-> 제목
-> 조리 시간
-> 난이도
-> 가격 수준
-> 기타 메타 정보
```

### MealsOverviewScreen에서 MealItem 사용

`MealsOverviewScreen`은 `MealItem`을 import하고, `renderMealItem`에서 사용한다.

```jsx
import MealItem from '../components/MealItem';
```

```jsx
function renderMealItem(itemData) {
  return (
    <MealItem title={itemData.item.title} />
  );
}
```

이제 카테고리를 누르면 해당 카테고리에 속한 음식 제목들이 출력된다.

```text
Italian
-> Italian category에 속한 음식 출력

German
-> German category에 속한 음식 출력

Quick & Easy
-> 여러 음식 출력
```

`FlatList`이므로 음식이 많아져도 스크롤 가능한 목록이 된다.

### 현재까지 완성된 흐름

이제 앱의 화면 흐름은 다음과 같다.

```text
CategoriesScreen
-> 카테고리 선택
-> categoryId를 params로 전달
-> MealsOverviewScreen
-> categoryId로 MEALS 필터링
-> MealItem 목록 출력
```

이제 화면 이동뿐만 아니라 화면 간 데이터 전달과 데이터 기반 렌더링까지 연결되었다.

## 예시 코드

### MealsOverviewScreen

```jsx
import { FlatList, StyleSheet, View } from 'react-native';

import MealItem from '../components/MealItem';
import { MEALS } from '../data/dummy-data';

function MealsOverviewScreen({ route }) {
  const catId = route.params.categoryId;

  const displayedMeals = MEALS.filter((mealItem) => {
    return mealItem.categoryIds.indexOf(catId) >= 0;
  });

  function renderMealItem(itemData) {
    return (
      <MealItem title={itemData.item.title} />
    );
  }

  return (
    <View style={styles.container}>
      <FlatList
        data={displayedMeals}
        keyExtractor={(item) => item.id}
        renderItem={renderMealItem}
      />
    </View>
  );
}

export default MealsOverviewScreen;

const styles = StyleSheet.create({
  container: {
    flex: 1,
    padding: 16,
  },
});
```

### MealItem

```jsx
import { Text, View } from 'react-native';

function MealItem({ title }) {
  return (
    <View>
      <Text>{title}</Text>
    </View>
  );

export default MealItem;
```

## 프론트엔드 개발자 관점에서의 핵심 차이

### 라우트 파라미터로 필터링 조건을 만든다

웹에서도 상세 페이지나 목록 페이지는 route params를 기반으로 데이터를 가져오는 경우가 많다.

```text
/categories/c1
-> categoryId = c1
-> 해당 카테고리 데이터 조회
```

React Native에서는 URL 대신 `route.params.categoryId`를 사용한다.

```jsx
const catId = route.params.categoryId;
```

이 값이 화면의 데이터 필터링 조건이 된다.

### 클라이언트 더미 데이터와 서버 데이터의 차이

이번 강의에서는 `MEALS` 배열을 클라이언트에서 바로 필터링한다.

```jsx
MEALS.filter(...)
```

실제 앱에서는 이 자리에 API 요청이 들어갈 수 있다.

```text
더미 데이터 앱
-> route.params.categoryId
-> 로컬 배열 필터링

실제 앱
-> route.params.categoryId
-> API 요청 또는 캐시 조회
-> 서버에서 카테고리별 음식 목록 받기
```

패턴은 비슷하다.

화면은 route params를 입력값으로 받고, 그 값으로 필요한 데이터를 결정한다.

### 목록 아이템 컴포넌트 분리

웹에서도 리스트 아이템은 보통 별도 컴포넌트로 분리한다.

```jsx
meals.map((meal) => (
  <MealCard key={meal.id} meal={meal} />
));
```

React Native에서도 같은 패턴을 사용한다.

```jsx
<FlatList
  data={displayedMeals}
  renderItem={({ item }) => <MealItem title={item.title} />}
/>
```

다만 React Native에서는 긴 목록 성능을 위해 `FlatList`를 기본 선택지로 고려하는 경우가 많다.

### indexOf와 includes

강의에서는 `indexOf()`를 사용했다.

```jsx
mealItem.categoryIds.indexOf(catId) >= 0
```

현대 JavaScript에서는 `includes()`가 더 의도를 잘 드러낸다.

```jsx
mealItem.categoryIds.includes(catId)
```

둘 다 동작하지만, 실무 코드에서는 `includes()`가 더 읽기 쉬운 경우가 많다.

## 정리

이번 강의에서는 `route.params.categoryId`를 사용해 선택한 카테고리에 속한 음식만 필터링했다.

필터링된 결과는 `FlatList`로 렌더링했고, 각 음식 항목은 앞으로 확장할 수 있도록 `MealItem` 컴포넌트로 분리했다.

이제 앱은 카테고리 선택, 화면 이동, 파라미터 전달, 데이터 필터링, 목록 출력까지 이어지는 기본 내비게이션 기반 흐름을 갖추게 되었다.
