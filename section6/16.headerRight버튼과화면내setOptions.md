# 16. headerRight 버튼과 화면 내 setOptions

## 핵심 요약

- React Navigation에서는 screen options를 통해 헤더 왼쪽이나 오른쪽에 커스텀 요소를 추가할 수 있다.
- `headerRight`와 `headerLeft` 옵션을 사용하면 제목 옆에 버튼, 텍스트, 아이콘 같은 React 컴포넌트를 렌더링할 수 있다.
- `headerRight`에는 JSX 자체가 아니라 JSX를 반환하는 컴포넌트 함수가 들어간다.
- `App.js`의 screen 설정에서 헤더 버튼을 추가할 수 있지만, 이 방식은 화면 컴포넌트와 직접 상호작용할 필요가 없을 때 적합하다.
- 헤더 버튼이 현재 화면의 상태나 함수와 상호작용해야 한다면 화면 컴포넌트 내부에서 `navigation.setOptions()`를 사용해 설정하는 편이 좋다.
- `setOptions()`는 side effect이므로 `useLayoutEffect` 안에서 호출한다.
- 헤더 버튼의 `onPress`에서 화면 컴포넌트 내부 함수를 호출하면 현재 화면 로직과 연결할 수 있다.

## 학습 목표

- `headerRight`와 `headerLeft` 옵션의 역할을 이해한다.
- 헤더에 텍스트나 버튼 같은 React 컴포넌트를 넣는 방법을 익힌다.
- `App.js`에서 정적으로 헤더 버튼을 설정하는 방식의 한계를 이해한다.
- 화면 컴포넌트 내부에서 `navigation.setOptions()`로 헤더 요소를 설정하는 방법을 익힌다.
- 헤더 버튼이 화면 내부 함수와 상호작용하는 패턴을 이해한다.
- `useLayoutEffect` 의존성 배열에 포함해야 할 값을 파악한다.

## 주요 내용

### 헤더에 버튼을 추가해야 하는 상황

`MealDetailScreen`에는 음식 상세 정보가 표시된다.

여기에 즐겨찾기 버튼을 추가하고 싶을 수 있다.

예를 들어 헤더 오른쪽에 별 아이콘이나 버튼을 두고, 사용자가 누르면 해당 음식을 즐겨찾기에 저장하는 식이다.

```text
MealDetailScreen
-> 헤더 오른쪽 버튼
-> 누르면 즐겨찾기 저장
```

React Navigation은 이런 헤더 커스터마이징을 screen options로 지원한다.

### headerRight와 headerLeft

헤더의 오른쪽이나 왼쪽에 요소를 추가하려면 다음 옵션을 사용한다.

```text
headerRight
-> 헤더 오른쪽에 요소 렌더링

headerLeft
-> 헤더 왼쪽에 요소 렌더링
```

예를 들어 `MealDetailScreen`에 오른쪽 헤더 요소를 추가할 수 있다.

```jsx
<Stack.Screen
  name="MealDetail"
  component={MealDetailScreen}
  options={{
    headerRight: () => {
      return <Text>In the header</Text>;
    },
  }}
/>
```

`headerRight`의 값은 JSX 자체가 아니라 JSX를 반환하는 함수다.

```text
headerRight
-> component function
-> JSX 반환
```

이 함수가 반환하는 JSX가 헤더 오른쪽에 표시된다.

### 헤더에 텍스트 넣기

처음에는 간단히 `Text`를 넣어 확인할 수 있다.

```jsx
import { Text } from 'react-native';
```

```jsx
options={{
  headerRight: () => {
    return <Text>In the header</Text>;
  },
}}
```

텍스트도 일반 React Native 컴포넌트이므로 스타일을 적용할 수 있다.

다만 어두운 헤더 배경에서는 검은 텍스트가 잘 보이지 않을 수 있으므로 색상 조정이 필요하다.

### 헤더에 기본 Button 넣기

텍스트 대신 React Native의 기본 `Button`을 넣을 수도 있다.

```jsx
import { Button } from 'react-native';
```

```jsx
options={{
  headerRight: () => {
    return (
      <Button
        title="Tap me"
        onPress={() => {
          console.log('pressed');
        }}
      />
    );
  },
}}
```

이렇게 하면 iOS와 Android 헤더에 버튼이 나타난다.

다만 기본 `Button`은 스타일링 자유도가 낮고, 헤더에 들어가기에는 시각적으로 아주 예쁘지 않을 수 있다.

그래서 다음 강의에서는 직접 커스텀 아이콘 버튼을 만든다.

### App.js에서 설정하는 방식의 한계

`App.js`의 `Stack.Screen` 설정에서 `headerRight`를 지정하면 헤더 요소를 쉽게 추가할 수 있다.

하지만 이 방식에는 한계가 있다.

헤더 버튼이 현재 화면 컴포넌트의 상태나 함수와 직접 상호작용해야 한다면 `App.js`에서 정의한 버튼은 화면 내부 로직에 접근하기 어렵다.

```text
App.js에서 headerRight 설정
-> 간단한 정적 버튼에는 적합
-> 화면 내부 상태/함수와 상호작용하기 어려움
```

예를 들어 즐겨찾기 버튼이 현재 meal id, 즐겨찾기 상태, 화면 내부 함수와 연결되어야 한다면 화면 컴포넌트 내부에서 헤더 버튼을 설정하는 편이 더 자연스럽다.

### 화면 컴포넌트 내부에서 setOptions 사용

화면 컴포넌트는 `navigation` prop을 받을 수 있다.

`navigation` 객체에는 `setOptions()` 메서드가 있다.

```jsx
function MealDetailScreen({ route, navigation }) {
  ...
}
```

`setOptions()`를 사용하면 화면 컴포넌트 내부에서 헤더 옵션을 설정할 수 있다.

```jsx
navigation.setOptions({
  headerRight: () => {
    return <Button title="Tap me" onPress={...} />;
  },
});
```

이렇게 하면 헤더 버튼이 화면 내부 함수나 상태에 접근할 수 있다.

### useLayoutEffect 안에서 setOptions 호출

`setOptions()`는 렌더링 결과를 계산하는 순수 작업이 아니라 화면 설정을 바꾸는 side effect다.

따라서 컴포넌트 함수 본문에서 바로 호출하지 않고 `useLayoutEffect` 안에서 호출한다.

```jsx
import { useLayoutEffect } from 'react';
```

```jsx
useLayoutEffect(() => {
  navigation.setOptions({
    headerRight: () => {
      return (
        <Button
          title="Tap me"
          onPress={headerButtonPressHandler}
        />
      );
    },
  });
}, [navigation, headerButtonPressHandler]);
```

`useLayoutEffect`를 사용하는 이유는 헤더 옵션이 화면이 보이는 타이밍에 최대한 자연스럽게 반영되도록 하기 위해서다.

이전 강의에서 동적 title을 설정할 때도 같은 이유로 `useLayoutEffect`를 사용했다.

### 헤더 버튼과 화면 내부 함수 연결

`MealDetailScreen` 내부에 버튼 클릭 핸들러를 만든다.

```jsx
function headerButtonPressHandler() {
  console.log('pressed');
}
```

그리고 `setOptions()`에서 만든 헤더 버튼의 `onPress`에 연결한다.

```jsx
useLayoutEffect(() => {
  navigation.setOptions({
    headerRight: () => {
      return (
        <Button
          title="Tap me"
          onPress={headerButtonPressHandler}
        />
      );
    },
  });
}, [navigation, headerButtonPressHandler]);
```

이제 헤더 버튼을 누르면 화면 내부 함수가 실행된다.

현재 예제에서는 `console.log('pressed')`만 실행하지만, 실제로는 다양한 작업을 할 수 있다.

```text
헤더 버튼에서 할 수 있는 일
-> 즐겨찾기 토글
-> 저장 요청
-> 공유 기능 실행
-> 다른 화면으로 이동
-> 화면 내부 상태 업데이트
```

### 의존성 배열

`useLayoutEffect` 안에서 사용하는 외부 값은 의존성 배열에 넣어야 한다.

```jsx
}, [navigation, headerButtonPressHandler]);
```

여기서는 `navigation` 객체와 `headerButtonPressHandler` 함수를 사용한다.

따라서 둘 다 의존성으로 넣는다.

```text
의존성
-> navigation
-> headerButtonPressHandler
```

다만 일반 함수는 렌더링마다 새로 생성될 수 있다.

실제 앱에서는 필요에 따라 `useCallback`으로 핸들러를 안정화하는 경우도 있다.

강의에서는 아직 그 단계까지는 다루지 않는다.

### App.js 방식과 화면 내부 방식 비교

두 방식 모두 유효하다.

```text
App.js에서 options 설정
-> 정적인 헤더 요소에 적합
-> 화면 내부 상태와 무관한 경우 좋음

화면 내부 setOptions
-> 현재 화면의 데이터/상태/함수와 연결하기 좋음
-> 동적 헤더 버튼에 적합
```

이번 예제에서는 헤더 버튼이 `MealDetailScreen`과 상호작용할 수 있어야 한다고 가정하므로 화면 내부에서 `setOptions()`를 사용한다.

## 예시 코드

### App.js에서 정적으로 headerRight 추가

```jsx
import { Button } from 'react-native';

<Stack.Screen
  name="MealDetail"
  component={MealDetailScreen}
  options={{
    headerRight: () => {
      return (
        <Button
          title="Tap me"
          onPress={() => console.log('pressed')}
        />
      );
    },
  }}
/>
```

### MealDetailScreen 내부에서 headerRight 설정

```jsx
import { useLayoutEffect } from 'react';
import { Button } from 'react-native';

function MealDetailScreen({ route, navigation }) {
  function headerButtonPressHandler() {
    console.log('pressed');
  }

  useLayoutEffect(() => {
    navigation.setOptions({
      headerRight: () => {
        return (
          <Button
            title="Tap me"
            onPress={headerButtonPressHandler}
          />
        );
      },
    });
  }, [navigation, headerButtonPressHandler]);

  // detail content
}
```

## 프론트엔드 개발자 관점에서의 핵심 차이

### 웹의 Header slot과 비슷하다

웹 앱에서도 페이지별로 헤더 오른쪽에 버튼을 넣는 구조가 있다.

```text
예시
-> 상세 페이지의 즐겨찾기 버튼
-> 설정 페이지의 저장 버튼
-> 편집 페이지의 완료 버튼
```

React Navigation의 `headerRight`는 이런 header slot처럼 생각할 수 있다.

다만 React Navigation에서는 이 설정을 screen options로 관리한다.

### 헤더도 화면 상태와 연결될 수 있다

헤더는 화면 바깥에 있는 것처럼 보이지만, 실제로는 현재 화면의 상태와 연결되어야 하는 경우가 많다.

```text
예시
-> 현재 항목이 즐겨찾기인지 여부
-> 저장 가능 여부
-> 편집 모드 여부
-> 선택된 데이터 id
```

이럴 때는 화면 컴포넌트 내부에서 `setOptions()`로 헤더를 설정하는 편이 유리하다.

### 이벤트 핸들러 연결은 일반 React와 같다

헤더에 렌더링되는 버튼도 결국 React 컴포넌트다.

```jsx
<Button onPress={handler} />
```

따라서 이벤트 핸들러를 연결하고, 그 안에서 상태 업데이트나 API 요청을 하는 방식은 일반 React와 크게 다르지 않다.

차이는 그 컴포넌트가 화면 본문이 아니라 네이티브 내비게이션 헤더에 렌더링된다는 점이다.

### 커스텀 버튼이 필요해지는 이유

React Native 기본 `Button`은 빠르게 동작 확인하기에는 좋다.

하지만 헤더에 넣기에는 스타일 제약이 있다.

```text
기본 Button의 한계
-> 스타일 커스터마이징 제한
-> 헤더 디자인과 어울리지 않을 수 있음
-> 아이콘 버튼 구현에는 부족
```

그래서 다음 강의에서는 직접 Pressable 기반 아이콘 버튼을 만든다.

## 정리

이번 강의에서는 React Navigation 헤더에 버튼을 추가하는 방법을 배웠다.

`headerRight` 또는 `headerLeft` 옵션으로 헤더에 React 컴포넌트를 넣을 수 있고, 단순한 정적 버튼은 `App.js`의 screen options에서 설정할 수 있다.

하지만 헤더 버튼이 현재 화면의 상태나 함수와 연결되어야 한다면, 화면 컴포넌트 내부에서 `navigation.setOptions()`를 `useLayoutEffect`와 함께 사용하는 것이 더 적절하다.
