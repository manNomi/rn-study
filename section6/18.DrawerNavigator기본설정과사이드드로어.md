# 18. Drawer Navigator 기본 설정과 사이드 드로어

## 핵심 요약

- React Navigation은 Stack 계열 네비게이터뿐만 아니라 Drawer, Tabs 같은 다른 네비게이션 패턴도 제공한다.
- Drawer Navigator는 화면 왼쪽 또는 오른쪽에서 열리는 사이드 메뉴 기반 네비게이션을 구현할 때 사용한다.
- 어떤 네비게이터를 사용하든 기본 구조는 비슷하다.
  - `NavigationContainer`로 감싼다.
  - 네비게이터 생성 함수를 호출한다.
  - `Navigator` 안에 `Screen`들을 등록한다.
- Drawer Navigator를 사용하려면 `@react-navigation/drawer`와 내부 의존성 패키지를 설치해야 한다.
- `Drawer.Navigator`에 등록된 `Drawer.Screen`들은 기본적으로 드로어 메뉴 항목으로 표시된다.
- 첫 번째로 등록된 화면이 기본 시작 화면이지만, `initialRouteName`으로 시작 화면을 바꿀 수 있다.
- Drawer Navigator도 Stack Navigator처럼 `options`, `screenOptions`로 헤더와 화면 설정을 조정할 수 있다.

## 학습 목표

- React Navigation이 여러 종류의 네비게이터를 제공한다는 점을 이해한다.
- Drawer Navigator가 어떤 UI 패턴을 구현하는지 파악한다.
- Drawer Navigator 설치에 필요한 패키지 구성을 이해한다.
- `NavigationContainer`, `createDrawerNavigator`, `Drawer.Navigator`, `Drawer.Screen`의 관계를 익힌다.
- Stack Navigator와 Drawer Navigator의 공통점과 차이를 비교한다.
- `initialRouteName`으로 기본 시작 화면을 설정하는 방법을 이해한다.

## 주요 내용

### Stack 외의 네비게이터

지금까지 Meals 앱에서는 주로 Native Stack Navigator를 사용했다.

Stack Navigator는 화면을 앞으로 쌓고, 뒤로 돌아갈 수 있는 구조를 만든다.

```text
Categories
-> MealsOverview
-> MealDetail
```

하지만 모든 앱이 Stack 구조만으로 만들어지는 것은 아니다.

React Navigation은 여러 네비게이션 패턴을 제공한다.

```text
Stack / Native Stack
-> 앞으로 이동하고 뒤로 돌아가는 화면 스택

Drawer
-> 옆에서 열리는 사이드 메뉴

Tabs
-> 하단 또는 상단 탭으로 주요 화면 전환
```

이번 강의에서는 Meals 앱을 잠시 떠나, 간단한 새 앱에서 Drawer Navigator를 먼저 실험한다.

### Drawer Navigator란?

Drawer Navigator는 사이드 드로어를 제공한다.

사용자가 헤더의 메뉴 아이콘을 누르거나 제스처를 사용하면 화면 옆에서 메뉴가 열린다.

```text
헤더 메뉴 버튼 탭
-> 사이드 드로어 열림
-> 등록된 화면 목록 표시
-> 메뉴 항목 선택
-> 해당 화면으로 전환
```

웹 프론트엔드 관점에서는 모바일 앱의 사이드바 메뉴 또는 햄버거 메뉴와 비슷하다.

다만 React Native에서는 이 메뉴가 네이티브 모바일 제스처와 화면 전환 경험에 맞게 동작한다.

### Drawer Navigator 설치

Drawer Navigator는 React Navigation의 core 패키지와 별도로 설치한다.

```bash
npm install @react-navigation/drawer
```

Expo 프로젝트에서는 Drawer Navigator가 내부적으로 사용하는 패키지도 함께 설치해야 한다.

강의에서는 다음과 같은 Expo 설치 명령을 사용한다.

```bash
expo install react-native-gesture-handler react-native-reanimated
```

프로젝트나 React Navigation 버전에 따라 필요한 패키지는 달라질 수 있으므로 실제 프로젝트에서는 공식 문서의 설치 안내를 함께 확인해야 한다.

강의 중에는 `react-native-reanimated` 관련 오류가 발생할 수 있다고 언급한다.

이 경우 강의에서는 특정 버전으로 낮추는 방식의 임시 해결책을 소개한다.

```bash
npm install react-native-reanimated@1 --save --save-exact
```

다만 이것은 강의 당시 환경에서의 우회 방법이다.

현대 프로젝트에서는 무조건 이 버전을 설치하기보다, 현재 Expo SDK와 React Navigation 버전에 맞는 설치 가이드를 따르는 편이 안전하다.

### NavigationContainer는 여전히 필요하다

어떤 네비게이터를 사용하든 React Navigation의 최상위 래퍼는 `NavigationContainer`다.

```jsx
import { NavigationContainer } from '@react-navigation/native';
```

`NavigationContainer`는 네비게이션 상태를 관리하고, 등록된 네비게이터가 동작할 수 있는 컨텍스트를 제공한다.

```jsx
function App() {
  return (
    <NavigationContainer>
      {/* navigator configuration */}
    </NavigationContainer>
  );
}
```

Stack Navigator를 사용할 때와 동일한 첫 단계다.

### createDrawerNavigator

Drawer Navigator를 만들려면 `@react-navigation/drawer`에서 `createDrawerNavigator`를 가져온다.

```jsx
import { createDrawerNavigator } from '@react-navigation/drawer';
```

그리고 컴포넌트 바깥에서 호출해 Drawer 객체를 만든다.

```jsx
const Drawer = createDrawerNavigator();
```

이 객체는 Stack Navigator에서 만들었던 `Stack` 객체와 비슷하다.

```text
Stack.Navigator
Stack.Screen

Drawer.Navigator
Drawer.Screen
```

즉 네비게이터 종류는 다르지만 화면을 등록하는 구조는 거의 같다.

### Drawer.Navigator와 Drawer.Screen

`NavigationContainer` 안에 `Drawer.Navigator`를 넣는다.

그리고 그 안에 `Drawer.Screen`으로 화면을 등록한다.

```jsx
<NavigationContainer>
  <Drawer.Navigator>
    <Drawer.Screen
      name="Welcome"
      component={WelcomeScreen}
    />
    <Drawer.Screen
      name="User"
      component={UserScreen}
    />
  </Drawer.Navigator>
</NavigationContainer>
```

각 `Drawer.Screen`에는 Stack Navigator와 마찬가지로 두 가지 핵심 값이 들어간다.

```text
name
-> 화면을 식별하는 고유 이름

component
-> 해당 화면에 렌더링할 컴포넌트
```

`WelcomeScreen`과 `UserScreen`은 미리 준비된 더미 화면이다.

```jsx
import WelcomeScreen from './screens/WelcomeScreen';
import UserScreen from './screens/UserScreen';
```

### 기본 시작 화면

기본적으로 가장 위에 등록된 화면이 시작 화면이 된다.

```jsx
<Drawer.Navigator>
  <Drawer.Screen name="Welcome" component={WelcomeScreen} />
  <Drawer.Screen name="User" component={UserScreen} />
</Drawer.Navigator>
```

이 경우 `WelcomeScreen`이 처음 표시된다.

시작 화면을 바꾸고 싶다면 `initialRouteName`을 사용한다.

```jsx
<Drawer.Navigator initialRouteName="User">
  <Drawer.Screen name="Welcome" component={WelcomeScreen} />
  <Drawer.Screen name="User" component={UserScreen} />
</Drawer.Navigator>
```

이렇게 하면 앱이 처음 열릴 때 `UserScreen`이 표시된다.

### Drawer Navigator의 기본 UI

Drawer Navigator를 설정하면 React Navigation이 기본 UI를 제공한다.

```text
기본 헤더
-> 상단에 표시됨

드로어 아이콘
-> 헤더에 자동 표시됨

사이드 드로어
-> 아이콘을 누르면 열림

화면 목록
-> 등록된 Drawer.Screen들이 메뉴 항목으로 표시됨
```

따라서 별도의 버튼을 만들지 않아도, 기본적으로 드로어를 열 수 있는 메뉴 아이콘이 헤더에 나타난다.

사용자는 드로어 안에서 `Welcome`, `User` 같은 화면 항목을 선택해 화면을 전환할 수 있다.

### Drawer도 options를 지원한다

Drawer Navigator도 Stack Navigator처럼 화면 옵션을 설정할 수 있다.

개별 화면에는 `options`를 줄 수 있다.

```jsx
<Drawer.Screen
  name="Welcome"
  component={WelcomeScreen}
  options={{
    title: 'Welcome',
  }}
/>
```

모든 화면에 공통 옵션을 적용하려면 `Drawer.Navigator`에 `screenOptions`를 줄 수 있다.

```jsx
<Drawer.Navigator
  screenOptions={{
    headerStyle: { backgroundColor: '#351401' },
    headerTintColor: 'white',
  }}
>
  ...
</Drawer.Navigator>
```

이번 강의에서는 아직 세부 옵션을 많이 다루지는 않고, Drawer Navigator의 기본 동작을 확인하는 데 집중한다.

## 예시 코드

### App.js 기본 구조

```jsx
import { NavigationContainer } from '@react-navigation/native';
import { createDrawerNavigator } from '@react-navigation/drawer';

import WelcomeScreen from './screens/WelcomeScreen';
import UserScreen from './screens/UserScreen';

const Drawer = createDrawerNavigator();

export default function App() {
  return (
    <NavigationContainer>
      <Drawer.Navigator>
        <Drawer.Screen
          name="Welcome"
          component={WelcomeScreen}
        />
        <Drawer.Screen
          name="User"
          component={UserScreen}
        />
      </Drawer.Navigator>
    </NavigationContainer>
  );
}
```

### 시작 화면 변경

```jsx
<Drawer.Navigator initialRouteName="User">
  <Drawer.Screen
    name="Welcome"
    component={WelcomeScreen}
  />
  <Drawer.Screen
    name="User"
    component={UserScreen}
  />
</Drawer.Navigator>
```

## 프론트엔드 개발자 관점에서의 핵심 차이

### 라우터가 아니라 네이티브 네비게이터에 가깝다

웹에서는 URL을 기준으로 라우팅을 생각하는 경우가 많다.

```text
/welcome
/user
```

React Native 앱에서는 사용자가 URL을 직접 입력하지 않는다.

대신 앱 내부의 네비게이터가 화면 전환 상태를 관리한다.

Drawer Navigator는 URL 기반 라우팅보다는 모바일 앱의 내비게이션 셸에 가깝다.

### 같은 Screen 등록 문법, 다른 UX

Stack과 Drawer는 화면 등록 문법이 비슷하다.

```jsx
<Stack.Screen name="A" component={AScreen} />
<Drawer.Screen name="A" component={AScreen} />
```

하지만 사용자 경험은 다르다.

```text
Stack
-> 앞으로 이동
-> 뒤로 가기
-> 화면이 쌓이는 느낌

Drawer
-> 메뉴에서 주요 화면 선택
-> 앱의 큰 섹션 전환
-> 사이드바/햄버거 메뉴 느낌
```

즉 네비게이터 선택은 문법보다 앱의 정보 구조와 사용 흐름에 따라 결정해야 한다.

### Drawer.Screen은 메뉴 항목이 된다

Stack Navigator에서는 등록된 화면이 자동으로 목록에 보이지 않는다.

하지만 Drawer Navigator에서는 등록된 화면들이 기본적으로 드로어 메뉴 항목으로 나타난다.

이 점이 Stack과 Drawer의 중요한 차이다.

```text
Drawer.Screen 등록
-> 화면 등록
-> 드로어 메뉴 항목으로도 표시
```

### initialRouteName은 기본 진입점을 바꾼다

`initialRouteName`은 앱이 처음 열릴 때 어느 화면을 보여줄지 결정한다.

웹으로 비유하면 앱의 기본 진입 경로를 정하는 것과 비슷하다.

다만 여기서도 URL이 아니라 네비게이터 내부의 screen name을 사용한다.

```jsx
initialRouteName="User"
```

이 값은 `Drawer.Screen`의 `name`과 정확히 일치해야 한다.

## 정리

이번 강의에서는 React Navigation의 또 다른 네비게이터인 Drawer Navigator를 살펴봤다.

Drawer Navigator는 사이드 메뉴 기반의 화면 전환을 제공하며, `NavigationContainer`, `createDrawerNavigator`, `Drawer.Navigator`, `Drawer.Screen` 조합으로 설정한다.

Stack Navigator와 문법은 매우 비슷하지만, 등록된 화면들이 드로어 메뉴 항목으로 표시되고 사용자는 사이드 메뉴를 통해 주요 화면을 전환한다.

이후에는 Drawer Navigator의 세부 옵션을 설정하고, 다시 Meals 앱으로 돌아가 Stack Navigator와 다른 네비게이터를 함께 조합하는 흐름으로 이어진다.
