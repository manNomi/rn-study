# 4. React Native 원리

## 핵심 요약

- React Native 앱도 React 컴포넌트와 JSX를 사용해 UI를 구성한다.
- `View`, `Text`, `TextInput` 같은 React Native 컴포넌트는 iOS와 Android의 네이티브 UI 요소로 매핑된다.
- React Native는 웹의 `react-dom`처럼 React의 결과물을 특정 플랫폼에 맞게 렌더링하는 역할을 한다.
- UI 컴포넌트는 네이티브 요소로 변환되지만, JavaScript 로직 자체가 네이티브 코드로 컴파일되는 것은 아니다.
- 상태 관리, 조건문, 함수 같은 JavaScript 로직은 앱 안에서 JavaScript 스레드로 실행되며 네이티브 플랫폼과 상호작용한다.

## 학습 목표

- React Native 컴포넌트가 실제 모바일 UI로 연결되는 방식을 이해한다.
- 웹에서 사용하는 HTML 요소와 React Native 컴포넌트의 차이를 구분한다.
- UI 요소와 JavaScript 로직이 React Native에서 서로 다르게 처리된다는 점을 이해한다.

## 주요 내용

### React 컴포넌트 기반의 앱 작성

React Native 앱을 만들 때도 기본적으로 React 컴포넌트를 작성한다. 함수형 컴포넌트, 클래스 컴포넌트 등 React에서 익숙한 방식으로 UI 구조를 만들 수 있다.

다만 JSX 안에서 사용하는 요소는 웹에서 쓰는 HTML 태그가 아니다. React Native에서는 `View`, `Text` 같은 전용 컴포넌트를 사용한다.

### React Native 컴포넌트의 변환

React Native의 핵심 역할 중 하나는 JSX에 작성한 UI 컴포넌트를 각 모바일 플랫폼의 네이티브 요소로 연결하는 것이다.

예를 들어 텍스트 입력 UI를 만들 때 플랫폼별로 사용하는 요소는 다르다.

```text
웹: <input>
Android: EditText
iOS: UITextField
React Native: <TextInput>
```

웹에서는 `react-dom`이 `<input>` 같은 JSX 요소를 브라우저 DOM으로 렌더링한다. 반면 React Native에서는 `<TextInput>` 같은 컴포넌트를 Android의 `EditText`, iOS의 `UITextField`에 대응되는 네이티브 UI로 처리한다.

이처럼 React Native는 공통 컴포넌트를 제공하고, 실제 실행 환경에 따라 해당 플랫폼의 네이티브 요소로 매핑한다.

### View와 Text

`View`와 `Text`는 React Native에서 가장 기본적으로 사용하는 컴포넌트다. 웹 개발에서 익숙한 `div`, `p`, `span` 같은 HTML 요소 대신 React Native 전용 컴포넌트로 화면을 구성한다.

이 컴포넌트들은 단순한 HTML 태그가 아니라 iOS와 Android에서 실제로 표시될 네이티브 UI 요소와 연결된다. 앞으로의 강의에서는 이런 컴포넌트들이 어떤 역할을 하고, 어떤 네이티브 요소와 대응되는지 더 자세히 배우게 된다.

### JavaScript 로직의 실행 방식

React Native에서 UI 요소는 네이티브 요소로 변환되지만, JavaScript 로직은 네이티브 코드로 컴파일되지 않는다.

조건문, 함수, 상태 관리 같은 JavaScript 코드는 앱 안에서 JavaScript 그대로 실행된다. React Native는 네이티브 앱 내부에 JavaScript를 실행할 수 있는 환경을 만들고, 이 JavaScript 프로세스를 관리한다.

이 JavaScript 스레드는 React Native를 통해 iOS와 Android 플랫폼과 상호작용한다. 즉 화면에 표시되는 UI는 네이티브 요소를 사용하지만, 앱의 로직은 JavaScript로 작성하고 실행할 수 있다.

### React Native의 기본 동작 흐름

React Native 앱의 기본 흐름은 다음처럼 이해할 수 있다.

```text
React 컴포넌트 작성
-> React Native 컴포넌트 사용
-> 플랫폼별 네이티브 UI 요소로 연결
-> JavaScript 로직은 앱 내부의 JS 스레드에서 실행
-> React Native를 통해 네이티브 플랫폼과 상호작용
```

이 구조 덕분에 개발자는 React와 JavaScript를 사용하면서도 iOS와 Android에서 동작하는 실제 네이티브 앱을 만들 수 있다.

## 정리

React Native는 JSX에 작성한 UI 컴포넌트를 iOS와 Android의 네이티브 UI 요소로 연결한다. 하지만 JavaScript 로직은 네이티브 코드로 컴파일되는 것이 아니라 앱 내부의 JavaScript 스레드에서 실행된다.

따라서 React Native의 핵심은 React 방식으로 UI와 로직을 작성하되, 그 결과를 모바일 플랫폼의 네이티브 요소와 API에 연결해주는 데 있다.
