# 9. 시뮬레이터로 앱 미리보기

## 핵심 요약

- Expo Go를 사용하면 실제 기기에서 쉽게 앱을 미리볼 수 있다.
- 다른 플랫폼에서 테스트하려면 로컬 머신에 Android 에뮬레이터나 iOS 시뮬레이터를 설치할 수 있다.
- Android 에뮬레이터는 Android Studio를 통해 만들고 실행한다.
- iOS 시뮬레이터는 macOS에서 Xcode를 설치해야 사용할 수 있다.
- Expo 개발 서버가 실행 중이면 Android와 iOS 시뮬레이터를 동시에 연결해 미리볼 수 있다.

## 학습 목표

- 실제 기기 외에 시뮬레이터로 React Native 앱을 미리보는 방법을 이해한다.
- Android Studio와 Xcode가 각각 어떤 역할을 하는지 구분한다.
- Expo 개발 서버에서 제공하는 주요 단축키를 이해한다.

## 주요 내용

### 실제 기기와 시뮬레이터

Expo Go 앱을 사용하면 iPhone이나 Android 기기에서 개발 중인 앱을 바로 미리볼 수 있다. 하지만 현재 가지고 있지 않은 플랫폼에서도 앱을 확인하고 싶을 수 있다.

이때는 로컬 컴퓨터에 시뮬레이터를 설치해 사용할 수 있다. 시뮬레이터는 실제 iOS 기기나 Android 기기를 소프트웨어로 모방하는 도구다.

### Android 에뮬레이터 준비

Android 앱을 로컬에서 미리보기하려면 Android Studio를 설치한다.

```text
Android Studio: developer.android.com/studio
```

Android Studio를 실행하면 `More Actions` 메뉴에서 `Virtual Device Manager`를 열 수 있다. 여기서 새로운 가상 기기, 즉 Android 에뮬레이터를 생성한다.

기기 프리셋을 선택할 때는 `Play Store` 아이콘이 있는 템플릿을 고르는 것이 중요하다. Expo Go 앱을 에뮬레이터에 설치하고 실행하기 위해 Play Store가 포함된 이미지가 필요하기 때문이다.

이후 Android 버전을 선택하고 기본값을 유지한 채 생성하면 에뮬레이터를 실행할 수 있다.

### iOS 시뮬레이터 준비

iOS 시뮬레이터는 Xcode를 통해 사용할 수 있다. Xcode는 macOS에서만 설치할 수 있으므로 Windows나 Linux에서는 iOS 시뮬레이터를 실행할 수 없다.

macOS에서는 App Store에서 Xcode를 설치한 뒤, Xcode 설정에서 Command Line Tools 버전이 선택되어 있는지 확인해야 한다.

시뮬레이터 앱은 Xcode 패키지 내부에서 직접 실행할 수 있다.

```text
Xcode
-> Show Package Contents
-> Contents
-> Developer
-> Applications
-> Simulator.app
```

시뮬레이터를 실행한 뒤 원하는 iPhone 기기를 선택해 앱을 미리볼 수 있다.

### 플랫폼별 제한

Android 에뮬레이터는 macOS, Windows, Linux에서 사용할 수 있다. 반면 iOS 시뮬레이터는 macOS에서만 사용할 수 있다.

Windows나 Linux를 사용하는 경우 로컬에서 iOS 시뮬레이터를 실행할 수는 없다. 나중에 클라우드 빌드 서비스를 통해 iOS 앱을 빌드할 수는 있지만, 개발 중 미리보기는 iPhone의 Expo Go 앱이나 macOS의 Xcode 시뮬레이터가 필요하다.

### Expo 개발 서버 유지

앱을 미리보기하려면 프로젝트에서 실행한 `npm start` 프로세스가 계속 켜져 있어야 한다.

```bash
npm start
```

이 프로세스가 중단되면 Expo가 코드 변경을 감지하지 못하고, 실제 기기나 시뮬레이터에 변경 사항을 반영할 수 없다.

프로세스를 종료하려면 터미널에서 `Ctrl + C`를 누르면 된다. 다시 시작하려면 같은 위치에서 `npm start`를 다시 실행한다.

### Expo 개발 서버 단축키

Expo 개발 서버가 실행 중이면 터미널에서 단축키를 눌러 앱을 특정 플랫폼에서 실행할 수 있다.

```text
a: Android 에뮬레이터에서 실행
i: iOS 시뮬레이터에서 실행
r: 앱 수동 새로고침
```

Android 에뮬레이터가 실행 중인 상태에서 `a`를 누르면 Expo가 에뮬레이터를 찾아 Expo Go 앱을 설치하고 앱을 실행한다.

iOS 시뮬레이터가 실행 중인 상태에서 `i`를 누르면 Expo가 시뮬레이터를 감지하고 Expo Go 앱에서 프로젝트를 실행한다.

### 변경 사항 확인

Android와 iOS 시뮬레이터를 동시에 켜두고 같은 Expo 프로젝트를 미리볼 수 있다.

`App.js`에서 텍스트를 수정하고 저장하면 실제 기기와 시뮬레이터에 변경 사항이 자동으로 반영된다. 필요하다면 터미널에서 `r`을 눌러 수동으로 앱을 다시 로딩할 수도 있다.

## 정리

Expo를 사용하면 실제 기기뿐 아니라 Android 에뮬레이터와 iOS 시뮬레이터에서도 앱을 미리볼 수 있다. Android는 Android Studio, iOS는 macOS의 Xcode가 필요하다.

개발 중에는 `npm start` 프로세스를 유지한 채 코드를 수정하고, `a`, `i`, `r` 단축키를 활용해 각 플랫폼에서 앱을 실행하거나 새로고침할 수 있다.
