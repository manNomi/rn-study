# 2. React Native란

## 핵심 요약

- React Native는 React를 기반으로 iOS와 Android용 네이티브 모바일 앱을 만들 수 있게 해주는 도구다.
- React 자체는 특정 플랫폼에 종속되지 않고, 상태 관리와 컴포넌트 트리 구성을 담당한다.
- 웹에서는 `react-dom`이 React의 결과물을 브라우저에 렌더링한다.
- 모바일에서는 React Native가 `react-dom`과 비슷한 역할을 하며, React 컴포넌트를 iOS와 Android의 네이티브 UI로 연결한다.
- React Native는 네이티브 UI 요소뿐 아니라 카메라 같은 기기 API를 JavaScript 코드에서 사용할 수 있도록 제공한다.

## 학습 목표

- React, `react-dom`, React Native의 역할 차이를 이해한다.
- React Native가 웹 앱이 아니라 실제 모바일 앱을 만드는 도구라는 점을 이해한다.
- React 지식이 React Native 학습의 기반이 되는 이유를 안다.

## 주요 내용

### React Native의 역할

React Native는 React를 사용해 iOS와 Android 앱을 만들 수 있게 해준다. 이렇게 만든 앱은 앱스토어에 배포할 수 있는 실제 모바일 앱이며, iOS와 Android 기기 사용자가 설치해서 사용할 수 있다.

이름에서 알 수 있듯이 React Native는 React와 밀접하게 연결되어 있다. React 방식으로 코드를 작성하되, 웹 브라우저가 아니라 모바일 운영체제 위에서 동작하는 네이티브 앱을 만드는 것이 핵심이다.

### React와 react-dom의 관계

React는 사용자 인터페이스를 만들기 위한 JavaScript 라이브러리다. 보통 웹 개발에서 많이 사용되지만, React 자체가 브라우저에 직접 화면을 그리는 것은 아니다.

웹 앱을 만들 때는 React와 함께 `react-dom`을 사용한다. React는 상태를 관리하고 가상의 컴포넌트 트리를 구성하며, `react-dom`은 그 결과를 브라우저의 DOM에 렌더링한다.

즉 React 자체는 플랫폼에 구애받지 않는 도구다. 브라우저, 모바일, 다른 환경 중 어디에 표시할지는 React와 함께 사용되는 별도의 렌더링 라이브러리가 담당한다.

### React Native와 react-dom의 차이

React Native는 웹 환경에서의 `react-dom`과 비슷한 위치에 있다. 차이점은 대상 플랫폼이 웹 브라우저가 아니라 iOS와 Android라는 점이다.

React Native는 JSX 안에서 사용할 수 있는 특별한 컴포넌트를 제공한다. 이 컴포넌트들은 브라우저의 HTML 요소가 아니라, iOS와 Android의 네이티브 UI 요소로 연결된다.

예를 들어 웹에서는 React 코드가 DOM 요소로 렌더링되지만, React Native에서는 React Native 컴포넌트가 모바일 플랫폼의 실제 네이티브 UI로 변환된다.

### 네이티브 기능 접근

React Native는 화면을 구성하는 컴포넌트만 제공하는 것이 아니다. 카메라 같은 기기 기능이나 특정 네이티브 플랫폼 API도 JavaScript 코드에서 사용할 수 있도록 연결해준다.

이 덕분에 React 문법과 JavaScript를 사용하면서도 모바일 기기의 네이티브 기능을 활용하는 앱을 만들 수 있다.

### React 지식이 필요한 이유

React Native 앱은 React 코드 위에서 만들어진다. 따라서 컴포넌트, 상태 관리, JSX 같은 React의 핵심 개념을 알고 있어야 React Native 컴포넌트와 API를 자연스럽게 사용할 수 있다.

React Native는 React를 대체하는 것이 아니라, React를 모바일 앱 개발에 사용할 수 있도록 확장해주는 역할에 가깝다.

## 정리

React Native는 React를 이용해 iOS와 Android용 네이티브 앱을 만들 수 있게 해주는 도구다. React가 컴포넌트와 상태를 관리하고, React Native가 그 결과를 모바일 플랫폼의 네이티브 UI와 API로 연결한다.
