# 7. 프로젝트 열기와 구조 살펴보기

## 핵심 요약

- Expo 프로젝트를 생성한 뒤에는 코드 에디터에서 프로젝트 폴더를 열어 작업한다.
- 강의에서는 무료 코드 에디터인 Visual Studio Code 사용을 추천한다.
- 생성된 프로젝트에는 설정 파일, 의존성 정보, 에셋 폴더, 기본 앱 코드가 포함된다.
- `package.json`은 프로젝트 의존성과 실행 스크립트를 확인하는 중요한 파일이다.
- `app.json`은 Expo 앱의 설정을 관리하는 파일이며, 이후 앱 이름이나 배경색 같은 설정을 다룰 때 자주 사용된다.

## 학습 목표

- 생성된 Expo 프로젝트를 코드 에디터에서 여는 방법을 이해한다.
- Expo 프로젝트의 기본 폴더와 파일이 어떤 역할을 하는지 파악한다.
- 앞으로 주로 수정하게 될 파일과 건드리지 않아도 되는 파일을 구분한다.

## 주요 내용

### VS Code에서 프로젝트 열기

Expo 프로젝트를 생성했다면 먼저 코드 에디터에서 프로젝트 폴더를 연다. 강의에서는 Visual Studio Code를 추천한다. VS Code는 무료이고, React Native 프로젝트를 편집하기에 충분히 편리한 기능을 제공한다.

VS Code를 실행한 뒤 `File` 메뉴에서 프로젝트 폴더를 열 수 있다. macOS에서는 `Open Folder`, Windows에서는 `Open` 메뉴를 사용해 조금 전 생성한 프로젝트 폴더를 선택한다.

강의에서는 `RNCourse` 폴더를 VS Code에서 열어 Expo React Native 프로젝트를 확인한다.

### 에디터 환경 설정

VS Code의 화면 구성은 필요에 따라 바꿀 수 있다. `View` 메뉴의 `Appearance`에서 사이드바나 상태 바를 표시하거나 숨길 수 있고, `Preferences`에서 색상 테마도 변경할 수 있다.

강의에서는 `Dark+` 테마를 사용한다. 자동 서식을 위해 `Prettier` 같은 확장 기능을 설치해 사용할 수도 있다.

### 생성된 폴더

Expo 프로젝트를 생성하면 여러 폴더가 함께 만들어진다.

```text
.expo-shared
.vscode
assets
node_modules
```

`.vscode` 폴더에는 VS Code 작업 공간 설정이 들어갈 수 있다. 개인 환경 설정에 가까우므로 프로젝트 핵심 코드로 볼 필요는 없다.

`.expo-shared` 폴더에는 Expo가 내부적으로 사용하는 정보가 들어 있다.

`assets` 폴더는 이미지 같은 정적 파일을 저장하는 곳이다. 앱 아이콘이나 앱 내부에서 사용하는 이미지를 이곳에 둘 수 있으며, 이후 강의에서 중요하게 다루게 된다.

`node_modules` 폴더에는 프로젝트가 사용하는 서드 파티 패키지들이 설치된다. 직접 수정하는 폴더가 아니며, `package.json`의 의존성 정보를 바탕으로 설치된다.

### package.json

`package.json`은 프로젝트의 의존성과 실행 스크립트를 관리하는 파일이다.

여기에는 `expo`, `react`, `react-native` 같은 핵심 패키지가 포함된다. Expo는 단순한 명령줄 도구가 아니라, React Native 앱 개발을 돕는 JavaScript 패키지이기도 하다.

프로젝트에 따라 `react-native-web`과 `react-dom`도 포함될 수 있다. React Native로 웹 앱을 만들 수도 있지만, 이 강의는 Android와 iOS 앱 개발에 집중하므로 웹 부분은 다루지 않는다.

`scripts` 항목에는 NPM으로 실행할 수 있는 명령어들이 정의되어 있다. 앱 실행이나 개발 서버 시작 같은 작업에 사용된다.

### package-lock.json과 babel.config.js

`package-lock.json`은 설치된 패키지 버전을 고정해주는 파일이다. 같은 프로젝트를 다른 환경에서 설치하더라도 일관된 의존성 버전을 유지할 수 있게 돕는다.

`babel.config.js`는 JavaScript 코드가 내부적으로 어떻게 변환되는지 설정하는 파일이다. 익숙하지 않다면 직접 수정하지 않는 것이 좋다.

### app.json

`app.json`은 Expo 앱 설정을 관리하는 파일이다. 앱이 Expo에서 실행되거나, 나중에 실제 앱스토어 배포용으로 빌드될 때 이 설정이 사용된다.

앱 이름, 배경색, 아이콘, 표시 방식 같은 설정을 여기서 다룰 수 있다. Expo 프로젝트에서는 자주 확인하게 될 중요한 설정 파일이다.

### App.js

`App.js`는 현재 프로젝트에서 가장 중요한 코드 파일이다. 처음 생성된 프로젝트에서는 기본 앱 코드가 이 파일에 들어 있다.

이 파일은 일반적인 React 컴포넌트 구조를 가지고 있지만, JSX에서는 웹의 HTML 요소가 아니라 React Native가 제공하는 특별한 컴포넌트를 사용한다.

파일 아래쪽에는 스타일링 코드도 포함되어 있다. React Native의 스타일링 방식은 웹 CSS와 다르기 때문에 이후 강의에서 별도로 다룬다.

### 다음 단계

프로젝트 구조를 확인했다면 이제 앱이 실제로 실행되는 모습을 볼 차례다. 이를 위해서는 앱 미리보기를 표시할 기기나 시뮬레이터가 필요하다.

다음 강의에서는 생성된 앱을 실행하고 확인할 수 있는 환경을 준비하는 흐름으로 이어진다.

## 정리

Expo 프로젝트를 생성한 뒤에는 VS Code 같은 코드 에디터에서 프로젝트 폴더를 열고 기본 구조를 확인한다. 처음에는 파일이 많아 보여도 핵심은 `package.json`, `app.json`, `App.js`, `assets` 폴더의 역할을 이해하는 것이다.

이후 강의에서는 `App.js`를 중심으로 코드를 수정하면서, Expo와 React Native가 제공하는 설정과 기능을 하나씩 활용하게 된다.
