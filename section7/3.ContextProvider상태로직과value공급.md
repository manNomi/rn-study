# 3. Context Provider 상태 로직과 value 공급

## 핵심 요약

- 이전 단계에서는 `FavoritesContext`와 `FavoritesContextProvider`의 껍데기만 만들었다.
- 이번 단계에서는 Provider 컴포넌트 안에 실제 즐겨찾기 상태 관리 로직을 채운다.
- 즐겨찾기 meal id 목록은 `useState`로 관리한다.
- `favoriteMealIds`는 현재 즐겨찾기 meal id 배열이다.
- `addFavorite(id)`는 기존 배열에 새 id를 추가한다.
- `removeFavorite(id)`는 기존 배열에서 특정 id를 제거한다.
- 이전 상태를 기반으로 새 상태를 만들기 때문에 `setFavoriteMealIds((currentFavIds) => ...)` 형태의 함수형 업데이트를 사용한다.
- Context Provider에는 반드시 `value` prop으로 제공할 값을 넘겨야 한다.
- Provider의 `value` 객체에 `ids`, `addFavorite`, `removeFavorite`를 담으면 하위 컴포넌트들이 Context를 통해 상태를 읽고 함수를 호출할 수 있다.

## 학습 목표

- Context Provider 내부에서 `useState`로 앱 전역 상태를 관리하는 방법을 익힌다.
- 배열 상태를 불변성 있게 추가/삭제하는 패턴을 복습한다.
- 이전 상태 기반 업데이트에서 함수형 `setState`를 사용해야 하는 이유를 이해한다.
- Context Provider의 `value` prop이 실제로 어떤 역할을 하는지 이해한다.
- Context 기본값과 Provider가 실제로 공급하는 값의 관계를 정리한다.

## 주요 내용

### Provider에 실제 로직 채우기

이전 강의의 `FavoritesContextProvider`는 아직 실제 상태를 제공하지 않았다.

```jsx
function FavoritesContextProvider({ children }) {
  return (
    <FavoritesContext.Provider>
      {children}
    </FavoritesContext.Provider>
  );
}
```

이 상태에서는 Provider가 감싸고 있더라도 하위 컴포넌트에 실제 즐겨찾기 상태를 공급하지 못한다.

이번 단계에서는 이 Provider 안에 상태와 상태 변경 함수를 넣는다.

### useState로 즐겨찾기 id 배열 관리

즐겨찾기 상태는 meal id 배열로 관리한다.

```jsx
const [favoriteMealIds, setFavoriteMealIds] = useState([]);
```

초기값은 빈 배열이다.

```text
favoriteMealIds
-> 현재 즐겨찾기로 저장된 meal id 목록

setFavoriteMealIds
-> favoriteMealIds 상태를 업데이트하는 함수
```

예를 들어 사용자가 `m1`, `m3`을 즐겨찾기로 저장했다면 상태는 다음과 비슷한 형태가 된다.

```jsx
['m1', 'm3']
```

`useState`를 사용하려면 React에서 가져와야 한다.

```jsx
import { createContext, useState } from 'react';
```

### addFavorite 함수

즐겨찾기에 새 meal id를 추가하는 함수가 필요하다.

```jsx
function addFavorite(id) {
  setFavoriteMealIds((currentFavIds) => [
    ...currentFavIds,
    id,
  ]);
}
```

이 함수는 추가할 meal id를 인자로 받는다.

```text
addFavorite('m1')
-> favoriteMealIds 배열에 'm1' 추가
```

기존 배열을 직접 수정하지 않고 새 배열을 만든다.

```jsx
[
  ...currentFavIds,
  id,
]
```

이 방식은 React 상태 업데이트에서 중요한 불변성 원칙을 지킨다.

### 왜 함수형 업데이트를 사용하는가?

`addFavorite`은 기존 즐겨찾기 배열을 바탕으로 새 배열을 만든다.

즉 이전 상태가 필요하다.

이런 경우에는 상태 업데이트 함수에 값을 바로 넘기기보다 함수를 넘기는 것이 안전하다.

```jsx
setFavoriteMealIds((currentFavIds) => {
  return [...currentFavIds, id];
});
```

React는 이 함수에 최신 상태 스냅샷을 전달한다.

```text
currentFavIds
-> React가 넘겨주는 최신 favoriteMealIds 상태
```

이 패턴은 여러 상태 업데이트가 연속으로 발생하거나, 상태 업데이트가 비동기적으로 처리될 수 있는 상황에서 특히 중요하다.

### removeFavorite 함수

즐겨찾기에서 특정 meal id를 제거하는 함수도 필요하다.

```jsx
function removeFavorite(id) {
  setFavoriteMealIds((currentFavIds) =>
    currentFavIds.filter((mealId) => mealId !== id)
  );
}
```

`filter`는 새 배열을 반환한다.

조건이 `true`인 항목만 남긴다.

```text
mealId !== id
-> 삭제하려는 id가 아닌 항목만 유지

mealId === id
-> false가 되어 새 배열에서 제외
```

예를 들어 현재 상태가 다음과 같다고 하자.

```jsx
['m1', 'm2', 'm3']
```

`removeFavorite('m2')`를 호출하면 결과는 다음과 같다.

```jsx
['m1', 'm3']
```

기존 배열을 직접 변경하지 않고, 삭제 대상이 빠진 새 배열을 만들어 상태로 설정한다.

### Context value 객체 만들기

이제 관리 중인 상태와 함수를 Provider에 전달해야 한다.

이를 위해 value 객체를 만든다.

```jsx
const value = {
  ids: favoriteMealIds,
  addFavorite: addFavorite,
  removeFavorite: removeFavorite,
};
```

여기서 앞의 key 이름은 Context를 사용하는 컴포넌트가 접근할 이름이다.

```text
ids
-> context.ids로 접근

addFavorite
-> context.addFavorite(id)로 호출

removeFavorite
-> context.removeFavorite(id)로 호출
```

뒤의 값은 Provider 내부에서 실제로 관리하는 상태와 함수다.

```text
favoriteMealIds
-> useState로 관리 중인 실제 배열

addFavorite
-> 실제 추가 로직 함수

removeFavorite
-> 실제 삭제 로직 함수
```

객체의 key와 함수 이름이 같기 때문에 축약 문법을 사용할 수도 있다.

```jsx
const value = {
  ids: favoriteMealIds,
  addFavorite,
  removeFavorite,
};
```

강의에서는 key와 value가 어떤 의미인지 분명히 보기 위해 명시적으로 연결한다.

### Provider의 value prop

Context Provider는 `value` prop을 통해 하위 컴포넌트에 값을 제공한다.

```jsx
return (
  <FavoritesContext.Provider value={value}>
    {children}
  </FavoritesContext.Provider>
);
```

이제 `FavoritesContextProvider`로 감싼 모든 컴포넌트는 이 value 객체에 접근할 수 있다.

```text
Provider value
-> ids
-> addFavorite
-> removeFavorite
```

이 값은 나중에 `useContext(FavoritesContext)`로 읽는다.

### 전체 Provider 구조

최종적으로 `favorites-context.js`는 다음과 같은 흐름을 갖는다.

```text
FavoritesContext 생성
-> 기본값으로 ids/addFavorite/removeFavorite 형태 정의

FavoritesContextProvider
-> useState로 favoriteMealIds 관리
-> addFavorite 함수 정의
-> removeFavorite 함수 정의
-> value 객체 생성
-> FavoritesContext.Provider에 value 전달
```

즉 Context 객체는 사용 가능한 API의 형태를 만들고, Provider 컴포넌트는 실제 상태와 로직을 공급한다.

## 예시 코드

### store/context/favorites-context.js

```jsx
import { createContext, useState } from 'react';

export const FavoritesContext = createContext({
  ids: [],
  addFavorite: (id) => {},
  removeFavorite: (id) => {},
});

function FavoritesContextProvider({ children }) {
  const [favoriteMealIds, setFavoriteMealIds] = useState([]);

  function addFavorite(id) {
    setFavoriteMealIds((currentFavIds) => [
      ...currentFavIds,
      id,
    ]);
  }

  function removeFavorite(id) {
    setFavoriteMealIds((currentFavIds) =>
      currentFavIds.filter((mealId) => mealId !== id)
    );
  }

  const value = {
    ids: favoriteMealIds,
    addFavorite: addFavorite,
    removeFavorite: removeFavorite,
  };

  return (
    <FavoritesContext.Provider value={value}>
      {children}
    </FavoritesContext.Provider>
  );
}

export default FavoritesContextProvider;
```

## 프론트엔드 개발자 관점에서의 핵심 차이

### Context는 상태 저장소가 아니라 전달 통로다

Context 자체가 상태를 자동으로 관리해 주는 것은 아니다.

상태는 여전히 `useState`가 관리한다.

Context는 그 상태와 함수를 여러 컴포넌트가 접근할 수 있도록 전달해 주는 통로 역할을 한다.

```text
useState
-> 실제 상태 관리

Context Provider
-> 상태와 함수를 하위 컴포넌트에 공급

useContext
-> 하위 컴포넌트에서 공급받은 값 사용
```

이 구분을 이해하면 Context API가 훨씬 명확해진다.

### 상태 변경 함수도 Context value에 넣는다

전역 상태를 읽기만 하면 기능이 완성되지 않는다.

`MealDetailScreen`에서는 즐겨찾기를 추가하거나 제거해야 한다.

따라서 상태 값뿐만 아니라 상태를 변경하는 함수도 Context에 넣는다.

```text
ids
-> 읽기

addFavorite
removeFavorite
-> 쓰기
```

이 구조는 Context를 작은 전역 store처럼 사용할 수 있게 해 준다.

### 배열 상태는 불변성 있게 다룬다

즐겨찾기 id 배열을 직접 수정하면 안 된다.

```jsx
favoriteMealIds.push(id); // 피해야 함
```

대신 새 배열을 반환한다.

```jsx
[...currentFavIds, id]
currentFavIds.filter((mealId) => mealId !== id)
```

React는 상태 객체나 배열의 참조가 바뀌었을 때 변경을 감지하고 리렌더링한다.

따라서 배열을 다룰 때는 항상 새 배열을 만드는 습관이 중요하다.

### Context 기본값과 value의 이름을 맞춘다

`createContext()`의 기본값에서 `ids`, `addFavorite`, `removeFavorite`를 정의했다면 Provider의 실제 value도 같은 이름을 제공해야 한다.

```jsx
createContext({
  ids: [],
  addFavorite: (id) => {},
  removeFavorite: (id) => {},
});
```

```jsx
const value = {
  ids: favoriteMealIds,
  addFavorite,
  removeFavorite,
};
```

이 이름이 일관되어야 사용하는 컴포넌트에서 혼란이 없다.

## 정리

이번 강의에서는 `FavoritesContextProvider` 안에 실제 즐겨찾기 상태 관리 로직을 추가했다.

`useState`로 `favoriteMealIds` 배열을 관리하고, `addFavorite`와 `removeFavorite` 함수로 배열에 id를 추가하거나 제거했다.

마지막으로 `ids`, `addFavorite`, `removeFavorite`를 담은 `value` 객체를 `FavoritesContext.Provider`에 전달해, Provider 아래의 모든 화면이 이 상태와 함수에 접근할 수 있는 준비를 마쳤다.
