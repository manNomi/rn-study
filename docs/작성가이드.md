# 강의 스크립트 정리 가이드

앞으로 전달되는 한국어 강의 스크립트는 이 기준에 맞춰 마크다운 파일로 정리합니다.

## 입력 해석

스크립트 첫 줄에 있는 강의 번호와 제목을 우선 사용합니다.

```text
1-1 안녕하세요 강의에 오신 걸 환영합니다
```

위 입력은 `section1/1.강의소개.md`로 저장합니다.

- `1-1`의 앞 숫자 `1`은 섹션 번호입니다.
- `1-1`의 뒤 숫자 `1`은 해당 섹션 안의 강의 번호입니다.
- 제목은 원문 제목을 그대로 쓰기보다 내용을 보고 복습하기 쉬운 이름으로 다듬습니다.

## 제목 다듬기 기준

원문 제목이 인사말이나 문장형이면 학습 주제가 드러나도록 바꿉니다.

```text
안녕하세요 강의에 오신 걸 환영합니다
-> 강의소개
```

파일명은 공백 없이 짧게 작성합니다.

```text
section1/1.강의소개.md
section1/2.ReactNative란.md
section2/1.개발환경설정.md
```

## 요약 기준

좋은 강의 노트는 원문을 압축한 문서가 아니라, 다시 학습하기 쉽게 재구성한 문서입니다.

- 강사의 인사, 반복, 전환 멘트는 제거합니다.
- 핵심 개념과 강의 흐름을 중심으로 정리합니다.
- 강의에서 강조한 전제 지식이나 준비 사항은 별도로 적습니다.
- 실습으로 이어지는 내용은 "앞으로 할 일"이나 "다음 학습 흐름"에 반영합니다.
- 원문에 없는 내용을 과하게 추측해서 추가하지 않습니다.

## 권장 문서 구조

```markdown
# M. 강의 제목

## 핵심 요약

- 핵심 내용
- 핵심 내용
- 핵심 내용

## 학습 목표

- 이 강의를 통해 이해해야 하는 내용
- 다음 강의로 이어지는 포인트

## 주요 내용

### 주제 1

정리된 설명을 작성합니다.

### 주제 2

정리된 설명을 작성합니다.

## 정리

이번 강의에서 기억할 내용을 2-3문장으로 마무리합니다.
```

## 작성 톤

- 한국어 학습 노트처럼 자연스럽게 작성합니다.
- 문장은 짧고 명확하게 씁니다.
- 과도한 존댓말, 대본식 표현, 감탄사는 줄입니다.
- 복습할 때 바로 읽히도록 개념 중심으로 정리합니다.
