# 9. GameOverScreen 동적 이미지와 스크롤

## 핵심 요약

- 게임 오버 화면도 가로 모드와 작은 화면에서 이미지가 너무 크거나 버튼에 접근하기 어려운 문제가 있다.
- 이전에는 이미지 크기를 `Dimensions.get()`으로 화면 너비 기준만 확인해 조정했다.
- 화면 방향이 바뀌는 상황까지 고려하려면 `useWindowDimensions` 훅을 사용해 컴포넌트 안에서 `width`, `height`를 동적으로 받아야 한다.
- 이미지 크기는 기본값을 `300`으로 두고, 작은 너비에서는 `150`, 낮은 높이에서는 `80`처럼 더 작게 조정할 수 있다.
- 동적으로 계산한 `width`, `height`, `borderRadius`는 기존 `imageContainer` 스타일과 배열로 병합한다.
- 게임 오버 화면 전체를 `ScrollView`로 감싸면 일부 기기에서 버튼이 화면 밖으로 밀려도 사용자가 스크롤해서 접근할 수 있다.
- `ScrollView`에도 `flex: 1` 스타일을 적용해 화면 전체를 차지하도록 해야 한다.

## 학습 목표

- `GameOverScreen`에서 `useWindowDimensions`를 적용하는 방법을 익힌다.
- 이미지 크기를 너비뿐 아니라 높이까지 고려해 조정하는 이유를 이해한다.
- 동적으로 계산한 스타일 객체를 기존 `StyleSheet` 스타일과 병합하는 방법을 복습한다.
- 가로 모드와 작은 화면에서 버튼 접근성을 위해 `ScrollView`를 사용하는 패턴을 배운다.
- 정적인 `Dimensions.get()` 기반 스타일을 동적인 훅 기반 스타일로 바꾸는 흐름을 이해한다.

## 주요 내용

### 문제 상황: 게임 오버 화면이 가로 모드에서 어색함

가로 모드 대응을 위해 게임 화면의 레이아웃 구조는 개선했다.

하지만 게임 오버 화면은 여전히 문제가 있다.

```text
GameOverScreen 문제
-> 이미지가 너무 큼
-> 가로 모드에서 화면 높이를 많이 차지함
-> 새 게임 버튼이 화면 밖으로 밀릴 수 있음
-> 스크롤이 없어 버튼에 접근하기 어려울 수 있음
```

특히 이미지가 화면에서 가장 큰 요소이므로 먼저 이미지 크기를 줄이는 것이 효과적이다.

### 기존 방식의 한계

앞선 강의에서는 `Dimensions.get('window').width`를 사용해 이미지 크기를 조정했다.

```jsx
const deviceWidth = Dimensions.get('window').width;
```

그리고 `deviceWidth`가 작을 때 이미지 크기를 줄였다.

```jsx
width: deviceWidth < 380 ? 150 : 300,
height: deviceWidth < 380 ? 150 : 300,
borderRadius: deviceWidth < 380 ? 75 : 150,
```

이 방식은 작은 기기 너비 대응에는 도움이 된다.

하지만 두 가지 한계가 있다.

```text
1. 높이를 고려하지 않음
2. 화면 방향 변경에 동적으로 반응하지 않음
```

가로 모드에서는 너비가 넓어질 수 있지만 높이는 매우 낮아진다.

따라서 너비만 기준으로 보면 큰 이미지가 유지되어 화면 높이를 과하게 차지할 수 있다.

### useWindowDimensions로 width와 height 가져오기

이제 `GameOverScreen`에서도 `useWindowDimensions`를 사용한다.

```jsx
import { useWindowDimensions } from 'react-native';
```

컴포넌트 함수 안에서 호출한다.

```jsx
function GameOverScreen({ roundsNumber, userNumber, onStartNewGame }) {
  const { width, height } = useWindowDimensions();

  // ...
}
```

`useWindowDimensions`는 기기 방향이 바뀌면 최신 `width`, `height`를 제공하고 컴포넌트를 다시 렌더링한다.

```text
portrait
-> height 큼
-> imageSize 크게 가능

landscape
-> height 작음
-> imageSize 더 작게 조정
```

### imageSize 동적 계산

먼저 이미지 크기의 기본값을 정한다.

```jsx
let imageSize = 300;
```

너비가 작은 기기에서는 이미지 크기를 줄인다.

```jsx
if (width < 380) {
  imageSize = 150;
}
```

그리고 높이가 낮은 경우에는 더 작게 줄인다.

```jsx
if (height < 400) {
  imageSize = 80;
}
```

이 흐름은 다음과 같다.

```text
기본
-> imageSize 300

좁은 화면
-> imageSize 150

낮은 화면, 예: 가로 모드
-> imageSize 80
```

가로 모드에서는 높이가 부족하므로 이미지가 더 작아져야 한다.

### 동적 imageStyle 만들기

계산한 `imageSize`를 기반으로 동적 스타일 객체를 만든다.

```jsx
const imageStyle = {
  width: imageSize,
  height: imageSize,
  borderRadius: imageSize / 2,
};
```

원형 이미지를 유지하려면 다음 관계가 중요하다.

```text
width === height
borderRadius === imageSize / 2
```

이미지 크기가 `80`이면 `borderRadius`는 `40`이 된다.

이미지 크기가 `300`이면 `borderRadius`는 `150`이 된다.

### 기존 imageContainer 스타일과 병합하기

기존 `imageContainer`에는 이미지 크기 외에도 여러 스타일이 들어 있다.

```jsx
imageContainer: {
  borderWidth: 3,
  borderColor: Colors.primary800,
  overflow: 'hidden',
  margin: 36,
}
```

따라서 기존 스타일을 완전히 대체하면 안 된다.

동적 크기 스타일만 추가로 병합해야 한다.

```jsx
<View style={[styles.imageContainer, imageStyle]}>
  <Image
    style={styles.image}
    source={require('../assets/images/success.png')}
  />
</View>
```

스타일 배열은 왼쪽에서 오른쪽으로 적용된다.

```text
styles.imageContainer
-> 기본 테두리, overflow, margin 등

imageStyle
-> 동적 width, height, borderRadius
```

이렇게 하면 기존 스타일은 유지하면서 크기만 현재 화면에 맞게 바뀐다.

### ScrollView로 전체 화면 감싸기

이미지를 줄여도 어떤 기기에서는 게임 오버 화면의 버튼이 화면 밖으로 밀릴 수 있다.

따라서 전체 화면을 `ScrollView`로 감싸 접근성을 보장한다.

```jsx
import { ScrollView } from 'react-native';
```

구조는 다음과 같다.

```jsx
<ScrollView style={styles.screen}>
  <View style={styles.rootContainer}>
    <Title>GAME OVER!</Title>
    <View style={[styles.imageContainer, imageStyle]}>
      <Image style={styles.image} source={...} />
    </View>
    <Text style={styles.summaryText}>
      {/* result */}
    </Text>
    <PrimaryButton onPress={onStartNewGame}>
      Start New Game
    </PrimaryButton>
  </View>
</ScrollView>
```

`ScrollView`에는 `flex: 1` 스타일을 적용한다.

```jsx
screen: {
  flex: 1,
}
```

이렇게 하면 화면이 부족할 때 스크롤할 수 있다.

```text
콘텐츠가 화면 안에 들어감
-> 스크롤이 거의 필요 없음

콘텐츠가 화면보다 큼
-> 사용자가 스크롤해서 버튼 접근 가능
```

### rootContainer와 screen의 역할

두 컨테이너의 역할은 다르다.

```text
ScrollView style={styles.screen}
-> 화면 전체 공간 차지
-> 스크롤 가능 영역 제공

View style={styles.rootContainer}
-> 내부 콘텐츠 배치
-> 제목, 이미지, 텍스트, 버튼 정렬
```

새 래퍼를 추가할 때 기존 레이아웃이 깨진다면, 보통 최상위 래퍼가 `flex: 1`을 가지는지 확인해야 한다.

## 예시 코드

```jsx
import {
  Image,
  ScrollView,
  StyleSheet,
  Text,
  useWindowDimensions,
  View,
} from 'react-native';

function GameOverScreen({ roundsNumber, userNumber, onStartNewGame }) {
  const { width, height } = useWindowDimensions();

  let imageSize = 300;

  if (width < 380) {
    imageSize = 150;
  }

  if (height < 400) {
    imageSize = 80;
  }

  const imageStyle = {
    width: imageSize,
    height: imageSize,
    borderRadius: imageSize / 2,
  };

  return (
    <ScrollView style={styles.screen}>
      <View style={styles.rootContainer}>
        <Title>GAME OVER!</Title>
        <View style={[styles.imageContainer, imageStyle]}>
          <Image
            style={styles.image}
            source={require('../assets/images/success.png')}
          />
        </View>
        <Text style={styles.summaryText}>
          Your phone needed <Text style={styles.highlight}>{roundsNumber}</Text>{' '}
          rounds to guess the number{' '}
          <Text style={styles.highlight}>{userNumber}</Text>.
        </Text>
        <PrimaryButton onPress={onStartNewGame}>Start New Game</PrimaryButton>
      </View>
    </ScrollView>
  );
}

const styles = StyleSheet.create({
  screen: {
    flex: 1,
  },
  imageContainer: {
    borderWidth: 3,
    borderColor: Colors.primary800,
    overflow: 'hidden',
    margin: 36,
  },
  image: {
    width: '100%',
    height: '100%',
  },
});
```

## 프론트엔드 개발자 관점에서의 핵심 차이

### width만 보면 부족하다

웹 반응형에서도 가로 모드는 너비가 넓지만 높이가 낮아서 문제가 생기는 경우가 많다.

React Native에서도 마찬가지다.

```text
작은 기기 대응
-> width 기준이 유용

가로 모드 대응
-> height 기준도 중요
```

이번 게임 오버 화면의 이미지는 너비뿐 아니라 높이에 따라 더 작게 줄여야 했다.

### 동적 스타일은 컴포넌트 내부에서 계산한다

화면 방향 변화에 반응하려면 `useWindowDimensions`를 사용해 컴포넌트 내부에서 계산해야 한다.

```jsx
const { width, height } = useWindowDimensions();
```

파일 상단에서 `Dimensions.get()`으로 계산한 값은 방향 변화에 자동 반응하지 않는다.

```text
초기값만 필요
-> Dimensions.get()

회전/크기 변화에 반응 필요
-> useWindowDimensions()
```

### 스크롤은 안전장치다

이미지를 줄였더라도 모든 기기에서 항상 버튼이 보인다고 확신하기 어렵다.

`ScrollView`는 이런 경우 사용성의 안전장치가 된다.

```text
스크롤이 없어도 보이는 기기
-> 그대로 사용

스크롤이 필요한 기기
-> 버튼과 텍스트에 접근 가능
```

모바일 앱에서는 “현재 테스트 기기에서 보인다”보다 “다른 기기에서도 접근 가능하다”가 중요하다.

## 정리

이번 강의에서는 게임 오버 화면을 가로 모드와 작은 화면에서도 사용할 수 있도록 개선했다.

`useWindowDimensions`로 현재 `width`와 `height`를 가져오고, 이미지 크기를 `300`, `150`, `80`처럼 조건부로 조정했다.

이미지의 원형 모양을 유지하기 위해 `width`, `height`, `borderRadius`를 같은 `imageSize`에서 계산했고, 이 동적 스타일을 기존 `imageContainer` 스타일과 배열로 병합했다.

마지막으로 전체 화면을 `ScrollView`로 감싸 어떤 기기에서도 새 게임 버튼에 접근할 수 있게 만들었다.

이로써 게임 오버 화면도 다양한 화면 크기와 방향에서 더 안정적으로 동작하게 되었다.
