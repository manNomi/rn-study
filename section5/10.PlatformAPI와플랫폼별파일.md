# 10. Platform API와 플랫폼별 파일

## 핵심 요약

- 화면 크기와 방향 대응 다음에는 iOS와 Android 플랫폼 차이에 대응하는 방법을 배운다.
- React Native는 하나의 코드베이스로 여러 플랫폼을 지원하지만, 필요하면 플랫폼별 스타일이나 코드를 작성할 수 있다.
- `Platform` API를 사용하면 현재 실행 중인 플랫폼을 확인할 수 있다.
- `Platform.OS`는 `'ios'`, `'android'`, `'web'` 등의 문자열을 반환한다.
- 간단한 분기는 삼항 연산자와 `Platform.OS`로 처리할 수 있다.
- 여러 플랫폼별 값을 더 읽기 좋게 작성하려면 `Platform.select()`를 사용할 수 있다.
- 차이가 큰 경우에는 `Title.ios.js`, `Title.android.js`처럼 플랫폼별 파일을 만들고, import할 때는 확장자 없이 공통 이름으로 가져온다.
- 플랫폼별 파일 분리는 컴포넌트뿐 아니라 색상 상수, 스타일, 일반 JavaScript 코드에도 사용할 수 있다.

## 학습 목표

- React Native에서 플랫폼별 스타일이 필요한 상황을 이해한다.
- `Platform.OS`를 사용해 iOS와 Android를 구분하는 방법을 익힌다.
- `Platform.select()`의 사용법과 장점을 이해한다.
- `.ios.js`, `.android.js` 파일 네이밍 규칙을 배운다.
- 플랫폼별 파일을 사용할 때 import 경로에서 플랫폼 확장자를 제거해야 하는 이유를 이해한다.

## 주요 내용

### 왜 플랫폼별 코드가 필요한가

React Native의 장점은 하나의 코드베이스로 iOS와 Android 앱을 만들 수 있다는 점이다.

가능하면 두 플랫폼에서 같은 UI와 로직을 공유하는 것이 좋다.

하지만 때로는 플랫폼별 차이를 주고 싶을 수 있다.

```text
플랫폼별 차이가 필요한 경우
-> iOS와 Android의 디자인 관례가 다를 때
-> 특정 플랫폼에서만 지원되는 스타일을 사용할 때
-> 버튼 피드백이나 그림자 구현이 다를 때
-> 플랫폼별 색상이나 레이아웃을 적용하고 싶을 때
```

이번 예제에서는 `Title` 컴포넌트를 플랫폼별로 다르게 보이게 만든다.

```text
Android
-> 기존처럼 border 유지

iOS
-> border 제거
```

### Platform API

React Native는 현재 실행 중인 플랫폼을 확인할 수 있는 `Platform` API를 제공한다.

```jsx
import { Platform } from 'react-native';
```

`Platform.OS`를 사용하면 현재 플랫폼을 문자열로 얻을 수 있다.

```jsx
Platform.OS
```

주요 값은 다음과 같다.

```text
ios
android
web
macos
windows
```

강의에서는 iOS와 Android에 집중한다.

중요한 점은 플랫폼은 앱 실행 중에 바뀌지 않는다는 것이다.

```text
Dimensions
-> 화면 회전이나 크기 변화에 따라 바뀔 수 있음

Platform
-> 앱 실행 중 바뀌지 않음
```

따라서 `Platform` API는 동적 listener나 hook이 필요하지 않다.

### Platform.OS로 간단히 분기하기

`Title` 컴포넌트에서 Android에만 border를 주고 싶다면 다음처럼 작성할 수 있다.

```jsx
import { Platform, StyleSheet, Text } from 'react-native';

const styles = StyleSheet.create({
  title: {
    borderWidth: Platform.OS === 'android' ? 2 : 0,
  },
});
```

의미는 다음과 같다.

```text
Platform.OS === 'android'
-> borderWidth 2

그 외 플랫폼
-> borderWidth 0
```

이렇게 하면 Android에서는 기존 border가 유지되고, iOS에서는 border가 사라진다.

간단한 값 하나를 바꾸는 경우에는 이 방식이 짧고 읽기 쉽다.

### Platform.select()

React Native는 플랫폼별 값을 선택하기 위한 `Platform.select()`도 제공한다.

```jsx
borderWidth: Platform.select({
  ios: 0,
  android: 2,
})
```

`Platform.select()`는 현재 플랫폼에 맞는 값을 반환한다.

```text
iOS에서 실행
-> ios 값 반환
-> 0

Android에서 실행
-> android 값 반환
-> 2
```

삼항 연산자와 결과는 같지만, 플랫폼별 값이 여러 개일 때 더 읽기 좋을 수 있다.

```jsx
const styles = StyleSheet.create({
  title: {
    borderWidth: Platform.select({
      ios: 0,
      android: 2,
    }),
  },
});
```

### Platform.OS와 Platform.select 비교

두 방식은 모두 유용하다.

```text
Platform.OS
-> 간단한 조건 하나에 적합
-> 삼항 연산자로 빠르게 작성 가능

Platform.select()
-> 플랫폼별 값을 명시적으로 나열
-> 여러 플랫폼 값이 있을 때 읽기 좋음
```

예를 들어 하나의 값만 살짝 바꾸는 경우:

```jsx
borderWidth: Platform.OS === 'android' ? 2 : 0
```

플랫폼별 값을 더 명확히 드러내고 싶은 경우:

```jsx
borderWidth: Platform.select({
  ios: 0,
  android: 2,
})
```

실무에서는 팀 컨벤션이나 코드 복잡도에 따라 선택하면 된다.

### 플랫폼별 파일

플랫폼별 차이가 커지면 조건문이 많아질 수 있다.

이럴 때는 파일 자체를 플랫폼별로 나눌 수 있다.

예를 들어 `Title` 컴포넌트를 iOS와 Android로 분리할 수 있다.

```text
Title.android.js
Title.ios.js
```

Android 파일에서는 border를 유지한다.

```jsx
// Title.android.js
const styles = StyleSheet.create({
  title: {
    borderWidth: 2,
    borderColor: 'white',
  },
});
```

iOS 파일에서는 border를 제거한다.

```jsx
// Title.ios.js
const styles = StyleSheet.create({
  title: {
    borderWidth: 0,
  },
});
```

또는 iOS 파일에서는 border 관련 스타일 자체를 제거할 수도 있다.

### import할 때 플랫폼 확장자를 쓰지 않는다

플랫폼별 파일을 사용할 때 중요한 규칙이 있다.

import할 때는 `.ios`나 `.android`를 쓰지 않는다.

```jsx
import Title from '../components/ui/Title';
```

다음처럼 쓰면 안 된다.

```jsx
import Title from '../components/ui/Title.ios';
```

React Native가 현재 플랫폼에 맞는 파일을 자동으로 선택하기 때문이다.

```text
iOS에서 실행
-> Title.ios.js 선택

Android에서 실행
-> Title.android.js 선택
```

IDE가 자동 import를 하면서 `.ios`나 `.android`를 붙일 수 있으므로 반드시 확인해야 한다.

### 컴포넌트 외 파일에도 적용 가능

플랫폼별 파일 기능은 컴포넌트에만 제한되지 않는다.

색상 파일이나 일반 유틸 파일에도 사용할 수 있다.

예를 들어 색상 팔레트를 플랫폼별로 다르게 만들 수 있다.

```text
colors.android.js
colors.ios.js
```

import는 여전히 공통 이름으로 한다.

```jsx
import Colors from '../constants/colors';
```

그러면 React Native가 플랫폼에 맞는 파일을 선택한다.

```text
Android
-> colors.android.js

iOS
-> colors.ios.js
```

이 방식으로 다음을 플랫폼별로 다르게 만들 수 있다.

```text
컴포넌트
스타일
색상 팔레트
유틸 함수
상수
플랫폼별 네이티브 기능 wrapper
```

### 언제 플랫폼별 파일을 사용할까

단순한 차이는 `Platform.OS`나 `Platform.select()`로 충분하다.

```jsx
borderWidth: Platform.OS === 'android' ? 2 : 0
```

하지만 플랫폼별 코드가 많아지면 파일 분리가 더 깔끔하다.

```text
Platform API가 적합한 경우
-> 스타일 값 하나 또는 몇 개만 다름
-> 차이가 작고 한눈에 읽힘

플랫폼별 파일이 적합한 경우
-> JSX 구조가 다름
-> 스타일이 많이 다름
-> 로직 자체가 다름
-> 플랫폼별 색상/설정 파일이 필요함
```

이번 `Title` 예제는 사실 파일을 나누기에는 작은 차이다.

하지만 기능을 보여주기 위해 플랫폼별 파일로도 구현해본다.

## 예시 코드

### Platform.OS 사용

```jsx
import { Platform, StyleSheet, Text } from 'react-native';

function Title({ children }) {
  return <Text style={styles.title}>{children}</Text>;
}

const styles = StyleSheet.create({
  title: {
    fontFamily: 'open-sans-bold',
    fontSize: 24,
    color: 'white',
    textAlign: 'center',
    borderWidth: Platform.OS === 'android' ? 2 : 0,
    borderColor: 'white',
    padding: 12,
    maxWidth: '80%',
    width: 300,
  },
});
```

### Platform.select 사용

```jsx
borderWidth: Platform.select({
  ios: 0,
  android: 2,
})
```

### 플랫폼별 파일 사용

```text
components/ui/Title.android.js
components/ui/Title.ios.js
```

사용하는 쪽:

```jsx
import Title from '../components/ui/Title';
```

React Native가 플랫폼별 파일을 자동 선택한다.

## 프론트엔드 개발자 관점에서의 핵심 차이

### 웹의 feature detection과 비슷하지만 플랫폼 단위다

웹에서는 브라우저 기능이나 화면 크기에 따라 분기하는 일이 있다.

React Native에서는 플랫폼 자체가 중요한 분기 기준이 될 수 있다.

```text
웹
-> 브라우저 기능
-> CSS 지원 여부
-> viewport 크기

React Native
-> iOS
-> Android
-> 화면 크기
-> 방향
```

`Platform.OS`는 브라우저의 user agent를 직접 파싱하는 것보다 훨씬 명확한 플랫폼 정보다.

### 조건문이 많아지면 파일 분리를 고려한다

웹에서도 컴포넌트 안에 조건문이 너무 많아지면 컴포넌트를 나누거나 설정 파일을 분리한다.

React Native도 마찬가지다.

```text
작은 차이
-> Platform.select()

큰 차이
-> .ios.js / .android.js 파일 분리
```

플랫폼별 파일은 코드가 더 명확해질 수 있지만, 파일 수가 늘고 중복이 생길 수 있으므로 필요한 경우에만 사용한다.

### import 경로가 중요하다

플랫폼별 파일을 만들었더라도 import 경로는 플랫폼 없는 이름이어야 한다.

```jsx
import Colors from '../constants/colors';
```

이 규칙을 어기면 특정 플랫폼 파일만 강제로 가져오게 되어 자동 선택의 장점이 사라진다.

## 정리

이번 강의에서는 React Native에서 플랫폼별 스타일과 코드를 작성하는 세 가지 방법을 배웠다.

첫째, `Platform.OS`를 사용해 현재 플랫폼을 직접 비교할 수 있다.

둘째, `Platform.select()`를 사용해 플랫폼별 값을 더 명시적으로 선택할 수 있다.

셋째, `.ios.js`, `.android.js` 파일을 만들어 플랫폼별 구현을 완전히 분리할 수 있다.

작은 차이는 `Platform` API로 처리하고, 컴포넌트 구조나 로직 차이가 커지면 플랫폼별 파일을 고려하면 된다.

이 기능은 React Native가 하나의 코드베이스를 유지하면서도 iOS와 Android의 차이를 세밀하게 다룰 수 있게 해주는 중요한 도구다.
