# 11. StatusBar 스타일 제어

## 핵심 요약

- `StatusBar`는 모바일 화면 상단의 시간, 배터리, 신호 상태 등이 표시되는 영역이다.
- Expo는 `expo-status-bar` 패키지를 통해 상태바 스타일을 제어할 수 있는 `StatusBar` 컴포넌트를 제공한다.
- 보통 `App.js`에 전역으로 추가하지만, 전체 화면을 차지하는 특정 화면 컴포넌트에 넣어 화면별로 다르게 설정할 수도 있다.
- `StatusBar`를 앱의 루트 컴포넌트 옆에 렌더링하려면 `Fragment`로 감싸야 한다.
- `style="light"`를 설정하면 밝은 상태바 아이콘을, `style="dark"`를 설정하면 어두운 상태바 아이콘을 사용할 수 있다.
- 배경이 어두운 앱에서는 `light` 스타일이 더 잘 보인다.

## 학습 목표

- `StatusBar` 컴포넌트의 역할을 이해한다.
- `expo-status-bar`에서 `StatusBar`를 가져와 사용하는 방법을 익힌다.
- 루트 컴포넌트 옆에 추가할 때 `Fragment`가 필요한 이유를 복습한다.
- 상태바 스타일을 앱 배경에 맞춰 조정하는 기준을 이해한다.
- 전역 상태바 설정과 화면별 상태바 설정의 차이를 파악한다.

## 주요 내용

### StatusBar란?

상태바는 모바일 기기 상단에 표시되는 시스템 UI다.

```text
StatusBar에 표시되는 정보
-> 시간
-> 배터리
-> 네트워크 신호
-> Wi-Fi
-> 기타 시스템 상태
```

앱의 배경색이나 레이아웃에 따라 상태바가 잘 보이지 않을 수 있다.

예를 들어 어두운 배경 위에 어두운 상태바 아이콘이 표시되면 사용자는 시간이나 배터리 정보를 읽기 어렵다.

따라서 앱 UI에 맞춰 상태바 스타일을 조정해야 한다.

### expo-status-bar

Expo 프로젝트에서는 `expo-status-bar` 패키지를 사용할 수 있다.

이 패키지는 기본적으로 설치되어 있는 경우가 많다.

```jsx
import { StatusBar } from 'expo-status-bar';
```

`StatusBar`는 JSX에서 렌더링하는 컴포넌트다.

```jsx
<StatusBar style="light" />
```

React Native 자체에도 상태바 관련 API가 있지만, Expo 프로젝트에서는 `expo-status-bar`를 사용하면 간단하게 처리할 수 있다.

### App.js에 추가하기

보통 상태바 설정은 앱 전체에 적용되므로 `App.js`에 추가한다.

예를 들어 기존 루트 UI가 `LinearGradient`라면 다음처럼 옆에 `StatusBar`를 렌더링한다.

```jsx
return (
  <>
    <StatusBar style="light" />
    <LinearGradient style={styles.rootScreen} colors={[...]}>
      {/* app screens */}
    </LinearGradient>
  </>
);
```

여기서 `StatusBar`와 `LinearGradient`는 형제 컴포넌트다.

JSX는 루트에 하나의 요소만 반환할 수 있으므로 `Fragment`로 감싸야 한다.

```jsx
<>
  <StatusBar style="light" />
  <LinearGradient />
</>
```

`Fragment`는 실제 네이티브 UI를 추가하지 않고 JSX 요소를 그룹화하는 역할을 한다.

### style prop

`StatusBar`에는 `style` prop을 설정할 수 있다.

```jsx
<StatusBar style="light" />
```

주요 값은 다음과 같다.

```text
light
-> 밝은 상태바 아이콘과 텍스트
-> 어두운 배경에 적합

dark
-> 어두운 상태바 아이콘과 텍스트
-> 밝은 배경에 적합

auto
-> 시스템 또는 환경에 맞춰 자동 처리
```

이번 앱은 보라색/어두운 계열 배경을 사용하므로 `light`가 더 잘 어울린다.

```jsx
<StatusBar style="light" />
```

### 화면별로 다르게 설정할 수도 있다

보통은 `App.js`에 한 번만 넣어 전체 앱에 같은 상태바 스타일을 적용한다.

하지만 화면마다 배경색이 크게 다르다면 특정 화면 컴포넌트 안에 `StatusBar`를 넣어 다르게 설정할 수도 있다.

```jsx
function SomeScreen() {
  return (
    <>
      <StatusBar style="dark" />
      <View>{/* screen content */}</View>
    </>
  );
}
```

이 방식은 특정 화면이 전체 화면을 차지하고, 상태바 스타일도 화면마다 달라져야 할 때 유용하다.

```text
전역 설정
-> 대부분의 화면에서 같은 상태바 스타일 사용

화면별 설정
-> 화면마다 배경색과 상태바 대비가 다를 때 사용
```

### 상태바가 사라지거나 안 보이는 문제

강의에서는 iOS에서 상태바가 잘 보이지 않는 상황이 있었다.

이때 `StatusBar`를 명시적으로 추가하고 `style="light"`를 설정하면 상태바가 다시 잘 보인다.

```text
문제
-> 배경과 상태바 아이콘 대비가 낮음
-> 상태바가 없는 것처럼 보임

해결
-> StatusBar 추가
-> style="light" 설정
```

상태바는 앱 사용 중 항상 보이는 시스템 정보이므로, 앱 배경에 맞게 신경 써야 한다.

## 예시 코드

```jsx
import { StatusBar } from 'expo-status-bar';
import { LinearGradient } from 'expo-linear-gradient';

export default function App() {
  return (
    <>
      <StatusBar style="light" />
      <LinearGradient
        colors={['#4e0329', '#ddb52f']}
        style={styles.rootScreen}
      >
        {/* screens */}
      </LinearGradient>
    </>
  );
}
```

루트에 `StatusBar`와 앱 UI가 나란히 있으므로 `Fragment`가 필요하다.

```jsx
<>
  <StatusBar style="light" />
  {app}
</>
```

## 프론트엔드 개발자 관점에서의 핵심 차이

### 웹에는 없는 모바일 시스템 UI

웹 프론트엔드에서는 브라우저 상단 UI를 직접 제어하지 않는다.

하지만 모바일 앱에서는 상태바가 앱 화면의 일부처럼 느껴진다.

```text
웹
-> 브라우저 UI는 앱 밖에 있음

React Native 앱
-> 상태바가 앱 화면과 시각적으로 이어짐
-> 배경색과 대비를 고려해야 함
```

따라서 모바일 앱에서는 상태바도 디자인의 일부로 취급해야 한다.

### SafeAreaView와 함께 생각하기

상태바 스타일을 제어하는 것과 콘텐츠가 상태바에 가려지지 않게 하는 것은 별개의 문제다.

```text
StatusBar
-> 상태바 아이콘/텍스트 스타일 제어

SafeAreaView
-> 콘텐츠가 노치와 상태바에 가려지지 않도록 배치
```

둘은 함께 사용될 수 있다.

상태바를 잘 보이게 만들고, 콘텐츠는 안전 영역 안에 배치해야 한다.

### 상태바 스타일도 플랫폼 감각의 일부

iOS와 Android는 상태바의 기본 동작과 시각적 느낌이 조금씩 다를 수 있다.

그래서 실제 기기나 시뮬레이터에서 상태바가 잘 보이는지 확인하는 것이 좋다.

```text
확인할 것
-> 배경색과 상태바 아이콘 대비
-> 세로/가로 모드에서 상태바 위치
-> Safe Area와 콘텐츠 간격
```

## 정리

이번 강의에서는 Expo의 `StatusBar` 컴포넌트를 사용해 앱의 상태바 스타일을 제어하는 방법을 배웠다.

`expo-status-bar`에서 `StatusBar`를 import하고, `App.js`의 루트 UI 옆에 렌더링한 뒤 `style="light"` 또는 `style="dark"`를 설정하면 된다.

상태바는 작은 요소처럼 보이지만 사용자가 시간과 배터리 같은 시스템 정보를 확인하는 중요한 영역이다.

따라서 모바일 앱에서는 상태바가 앱 배경과 잘 어울리고 충분히 읽기 쉬운지 항상 확인해야 한다.
