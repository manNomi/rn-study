# 1. 섹션 소개

## 핵심 요약

- 이번 섹션은 이전 섹션에서 완성한 숫자 맞히기 게임 앱을 기반으로 진행된다.
- 목표는 React Native 앱의 UI를 더 적응형(adaptive)이고 반응형(responsive)으로 만드는 방법을 배우는 것이다.
- 하나의 코드베이스로 iOS와 Android를 지원하면서도, 필요할 때는 플랫폼별로 다른 코드나 스타일을 적용할 수 있다.
- 기기 크기, 사용 가능한 너비와 높이, 화면 방향에 따라 레이아웃과 스타일을 조정하는 방법을 배운다.
- 최신 기기에서는 괜찮아 보이는 UI도 작은 기기나 오래된 기기에서는 답답하거나 깨져 보일 수 있다.
- 이번 섹션의 출발점은 “모든 사용자 기기에서 앱이 보기 좋고 사용 가능해야 한다”는 문제의식이다.

## 학습 목표

- React Native에서 적응형 UI가 왜 필요한지 이해한다.
- iOS와 Android를 하나의 코드베이스로 다루되, 플랫폼별 차이를 처리하는 관점을 잡는다.
- 다양한 기기 크기에 따라 스타일과 레이아웃을 바꾸는 필요성을 이해한다.
- 화면 방향, 즉 세로 모드와 가로 모드에 대응해야 하는 이유를 파악한다.
- 이전 섹션의 게임 앱을 다음 단계의 실습 기반으로 사용한다.

## 주요 내용

### 이전 섹션에서 이어지는 내용

이전 섹션에서는 숫자 맞히기 게임 앱을 만들며 React Native 앱 개발의 핵심을 연습했다.

```text
이전 섹션에서 다룬 내용
-> 핵심 컴포넌트
-> 스타일링
-> 커스텀 컴포넌트
-> 상태 관리
-> 화면 전환
-> 이미지, 폰트, 아이콘
-> FlatList 목록 렌더링
```

이번 섹션은 그 결과물을 더 현실적인 모바일 앱처럼 다듬는 단계다.

즉 새로운 앱을 처음부터 만드는 것이 아니라, 이미 만든 앱을 다양한 실행 환경에 맞게 조정한다.

### 적응형 UI와 반응형 UI

이번 섹션의 핵심 키워드는 adaptive와 responsive다.

```text
responsive
-> 화면 크기나 방향 변화에 따라 UI가 자연스럽게 반응하는 것

adaptive
-> 플랫폼, 기기 특성, 사용 환경에 맞춰 UI나 코드가 적절히 달라지는 것
```

웹 프론트엔드에서 반응형 웹을 만들 때는 미디어 쿼리, 뷰포트 단위, CSS Grid, Flexbox 등을 사용한다.

React Native에서도 같은 문제를 해결해야 하지만 도구와 접근 방식은 다르다.

```text
웹
-> CSS media query
-> viewport width / height
-> responsive CSS

React Native
-> 사용 가능한 화면 크기 측정
-> 기기 방향 감지
-> 플랫폼별 분기
-> 조건부 스타일 적용
```

### 하나의 코드베이스와 플랫폼별 코드

React Native의 장점은 하나의 코드베이스로 iOS와 Android 앱을 만들 수 있다는 점이다.

하지만 모든 코드가 항상 두 플랫폼에서 완전히 똑같이 동작해야 하는 것은 아니다.

때로는 플랫폼별로 약간 다른 코드를 작성해야 한다.

```text
공통 코드
-> 대부분의 UI와 로직은 iOS/Android가 공유

플랫폼별 코드
-> 특정 스타일, 특정 컴포넌트 동작, 플랫폼 관례가 다를 때 분기
```

예를 들어 이전 섹션에서도 플랫폼 차이를 이미 경험했다.

```text
Android
-> elevation으로 그림자 처리
-> android_ripple 사용 가능

iOS
-> shadow* 스타일로 그림자 처리
-> Pressable pressed 상태로 피드백 처리
```

이번 섹션에서는 이런 플랫폼 차이를 더 명시적으로 다루는 방법을 배운다.

### 기기 크기 차이

강의에서 보여준 게임 앱은 최신 iPhone이나 최신 Android 기기에서는 꽤 괜찮게 보인다.

하지만 모든 사용자가 최신 대형 기기를 쓰는 것은 아니다.

작고 오래된 Android 기기, 예를 들어 Nexus 5 같은 화면에서는 다음 문제가 생길 수 있다.

```text
작은 기기에서 발생하는 문제
-> 제목이 화면 상단에 너무 붙음
-> 요소들이 상대적으로 커 보임
-> 게임 오버 화면의 이미지와 텍스트가 답답해짐
-> 한 화면에 들어가긴 하지만 여백과 비율이 어색함
```

여기서 중요한 점은 단순히 에뮬레이터 창을 크게 키우는 것이 실제 기기 크기를 바꾸는 것은 아니라는 것이다.

```text
에뮬레이터 창 확대
-> 개발자 화면에서 크게 보일 뿐

실제 사용 가능한 픽셀 / 레이아웃 공간
-> 기기 이미지와 해상도에 따라 제한됨
```

따라서 앱은 실제 사용 가능한 화면 크기를 기준으로 적절히 조정되어야 한다.

### 왜 작은 기기를 고려해야 하는가

최신 기기에서만 확인하면 UI가 충분히 좋아 보일 수 있다.

하지만 실제 사용자 환경은 다양하다.

```text
사용자 기기 다양성
-> 최신 대형 iPhone
-> 작은 Android 기기
-> 오래된 기기
-> 태블릿
-> 세로 모드
-> 가로 모드
```

모바일 앱은 이런 환경에서 최소한 깨지지 않고, 가능하면 보기 좋고 사용하기 쉬워야 한다.

프론트엔드 개발자에게 익숙한 말로 바꾸면, React Native에서도 반응형 레이아웃 QA가 필요하다는 뜻이다.

### 화면 방향 대응

이번 섹션에서는 기기 방향에도 반응하는 방법을 배운다.

```text
portrait
-> 세로 모드

landscape
-> 가로 모드
```

웹에서도 가로폭이 바뀌면 레이아웃을 바꾸듯이, 모바일 앱에서도 기기를 회전했을 때 UI가 잘 보여야 한다.

특히 게임 앱의 경우 가로 모드에서는 세로 모드와 완전히 같은 배치가 어색할 수 있다.

```text
세로 모드
-> 위에서 아래로 쌓는 레이아웃이 자연스러움

가로 모드
-> 좌우 공간이 넓으므로 일부 UI를 나란히 배치할 수 있음
```

이번 섹션은 이런 방향 변화에 반응하는 도구들을 소개한다.

## 프론트엔드 개발자 관점에서의 핵심 차이

### 웹의 반응형과 React Native의 반응형

웹에서는 CSS가 반응형의 중심이다.

```css
@media (max-width: 600px) {
  .card {
    padding: 12px;
  }
}
```

React Native에는 CSS 파일과 미디어 쿼리가 없다.

대신 JavaScript에서 화면 크기와 플랫폼 정보를 읽고 조건부로 스타일을 적용한다.

```text
React Native 반응형 사고
-> 현재 화면 크기 확인
-> 현재 플랫폼 확인
-> 현재 방향 확인
-> 조건에 따라 스타일/레이아웃 변경
```

즉 반응형 로직이 CSS가 아니라 컴포넌트 코드와 스타일 객체 안으로 들어오는 경우가 많다.

### “보인다”와 “잘 맞는다”는 다르다

작은 기기에서 앱이 한 화면에 들어간다고 해서 UI가 좋은 것은 아니다.

강의에서 작은 Nexus 5 화면을 확인했을 때, 게임 오버 화면은 일단 들어가긴 하지만 요소들이 너무 크고 여백이 답답하다.

```text
단순히 보이는 상태
-> UI가 화면 안에 들어감

좋은 모바일 UI
-> 요소 크기, 여백, 위치가 기기 크기에 맞음
-> 터치 영역과 가독성이 유지됨
-> 상태바와 화면 경계에 너무 붙지 않음
```

이번 섹션은 “깨지지 않는 UI”를 넘어서 “기기 환경에 맞게 조정되는 UI”를 만드는 방향으로 진행된다.

### 하나의 앱, 여러 환경

React Native 개발자는 하나의 앱이 여러 환경에서 실행된다는 사실을 항상 염두에 둬야 한다.

```text
환경 차이
-> iOS / Android
-> 작은 화면 / 큰 화면
-> 최신 기기 / 오래된 기기
-> 세로 / 가로 방향
```

따라서 처음부터 절대값에만 의존한 레이아웃을 만들면 이후에 문제가 생기기 쉽다.

이 섹션에서는 고정 스타일을 조건부 스타일과 동적 레이아웃으로 바꾸는 방법을 배우게 된다.

## 이번 섹션에서 배우게 될 도구들

강의 도입부에서 예고한 주제는 다음과 같다.

```text
1. 플랫폼별 코드 작성
2. 기기 크기에 따라 스타일 조정
3. 사용 가능한 너비와 높이에 따른 레이아웃 변경
4. 화면 방향 변화에 반응
5. 하나의 코드베이스 안에서 환경별로 적응하는 컴포넌트 작성
```

이 도구들은 각각 따로 떨어진 기능이 아니라, 실제 앱에서 함께 사용된다.

예를 들어 작은 Android 기기에서는 제목 여백을 줄이고, iOS에서는 다른 그림자 스타일을 쓰고, 가로 모드에서는 버튼 배치를 바꿀 수 있다.

## 정리

이번 섹션은 이전에 만든 숫자 맞히기 게임 앱을 더 현실적인 모바일 환경에 맞게 개선하는 출발점이다.

핵심은 다음과 같다.

```text
React Native 앱은 하나의 코드베이스로 iOS와 Android를 지원한다.
하지만 플랫폼과 기기 환경은 다양하다.
따라서 UI는 화면 크기, 플랫폼, 방향에 따라 적응할 수 있어야 한다.
```

이제부터는 단순히 “앱이 동작한다”를 넘어서, “다양한 기기에서 자연스럽게 보이고 사용할 수 있는 앱”을 만드는 방법을 배우게 된다.
