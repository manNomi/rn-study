# 5. 화면 방향 설정과 가로 모드 문제

## 핵심 요약

- `Dimensions` API는 화면 크기에 맞춰 스타일을 조정하는 데 유용하지만, 화면 방향 전환 문제를 자동으로 해결해주지는 않는다.
- Expo 프로젝트는 `app.json`의 `orientation` 설정으로 앱의 화면 방향을 제어할 수 있다.
- 기본 설정이 `portrait`이면 앱은 세로 모드로 고정되어 기기를 회전해도 화면이 가로로 전환되지 않는다.
- `orientation`을 `default`로 바꾸면 iOS와 Android에서 세로/가로 방향 전환이 가능해진다.
- 설정 변경 후에는 앱을 새로고침하거나 재시작해야 반영된다.
- Android에서 회전이 안 된다면 기기 또는 에뮬레이터의 auto-rotate 설정도 확인해야 한다.
- 가로 모드를 허용하면 기존 레이아웃이 깨지거나 키보드가 입력창을 가리는 문제가 드러난다.

## 학습 목표

- Expo의 `app.json`에서 화면 방향을 설정하는 방법을 이해한다.
- `portrait`, `landscape`, `default` 설정의 차이를 구분한다.
- 화면 회전을 허용했을 때 기존 UI가 왜 깨지는지 이해한다.
- 가로 모드에서 키보드가 입력창을 가리는 문제를 인식한다.
- 다음 단계에서 다룰 방향별 레이아웃 대응의 필요성을 파악한다.

## 주요 내용

### Dimensions API가 해결하지 못하는 문제

앞선 강의에서는 `Dimensions` API를 사용해 작은 화면에서 패딩, 마진, 폰트 크기, 이미지 크기를 조정했다.

하지만 `Dimensions` API만으로 모든 반응형 문제가 해결되는 것은 아니다.

대표적인 예가 화면 방향 전환이다.

```text
세로 모드
-> portrait

가로 모드
-> landscape
```

사용자가 기기를 회전하면 사용 가능한 너비와 높이가 크게 바뀐다.

그런데 앱이 세로 방향으로 고정되어 있으면 아무리 기기를 돌려도 화면이 가로 모드로 전환되지 않는다.

### 기본 설정: portrait 고정

Expo 프로젝트에서는 `app.json` 파일이 앱 설정을 담당한다.

이 파일 안에 `orientation` 설정이 있다.

```json
{
  "expo": {
    "orientation": "portrait"
  }
}
```

`portrait`로 설정되어 있으면 앱은 세로 방향으로 고정된다.

```text
orientation: "portrait"
-> 세로 모드 고정
-> 기기를 회전해도 앱 화면은 가로로 바뀌지 않음
```

이런 설정은 앱에 따라 적절할 수 있다.

예를 들어 이번 숫자 맞히기 게임은 사실 세로 모드로만 사용해도 충분하다.

하지만 다양한 방향을 지원해야 하는 앱이라면 이 설정을 바꿔야 한다.

### orientation 설정 값

`orientation`에는 여러 값을 사용할 수 있다.

```json
{
  "expo": {
    "orientation": "default"
  }
}
```

주요 설정은 다음과 같다.

```text
portrait
-> 세로 모드로 고정

landscape
-> 가로 모드로 고정

default
-> 기기 방향에 따라 세로/가로 전환 허용
```

이번 강의에서는 가로 모드 대응을 연습하기 위해 `default`로 변경한다.

### 설정 변경 후 앱 재시작 필요

`app.json`의 `orientation`을 바꾼 뒤 바로 화면이 회전하지 않을 수 있다.

이런 설정은 앱 시작 시 읽히는 경우가 많기 때문에 앱을 다시 로드하거나 재시작해야 한다.

Expo 개발 서버에서는 터미널에서 `r`을 눌러 강제 새로고침할 수 있다.

```text
app.json 변경
-> 앱 새로고침 또는 재시작
-> orientation 변경 반영
```

iOS 시뮬레이터에서는 이후 기기를 회전하면 화면이 가로 모드로 전환된다.

### Android에서 auto-rotate 확인하기

Android에서는 `orientation: "default"`로 바꿔도 화면이 회전하지 않을 수 있다.

그럴 때는 기기나 에뮬레이터의 자동 회전 설정을 확인해야 한다.

```text
Android에서 확인할 것
-> auto-rotate 켜져 있는지
-> 에뮬레이터 회전 버튼을 눌렀는지
-> 앱이 재시작 또는 새로고침되었는지
```

실제 Android 기기에서도 사용자가 자동 회전을 꺼둔 경우 앱이 기대대로 회전하지 않을 수 있다.

### 가로 모드에서 드러나는 문제

가로 모드를 허용하면 앱이 회전은 되지만, UI가 바로 좋은 상태가 되는 것은 아니다.

이번 게임 앱에서는 다음 문제가 드러난다.

```text
Android 가로 모드
-> 화면을 겨우 사용할 수는 있지만 레이아웃이 답답함
-> 요소들이 가로 높이에 비해 너무 큼

iOS 가로 모드
-> Android보다 낫지만 완벽하지 않음
-> 상단 여백이 너무 큼
```

즉 세로 모드 기준으로 만든 스타일이 가로 모드에서는 맞지 않는다.

```text
세로 모드에서 자연스러운 레이아웃
-> 위에서 아래로 쌓는 구조

가로 모드에서의 문제
-> 높이가 부족함
-> 위아래 여백이 과함
-> 키보드가 입력 영역을 가림
```

### 키보드가 입력창을 가리는 문제

iOS 시뮬레이터에서는 기본적으로 하드웨어 키보드 입력을 사용할 수 있다.

하지만 소프트웨어 키보드를 강제로 표시하면 문제가 드러난다.

```text
가로 모드 + 소프트웨어 키보드
-> 키보드가 입력창을 덮음
-> 사용자가 입력 내용을 보기 어려움
-> 화면 아무 곳이나 눌러도 키보드가 닫히지 않음
```

이 상태에서는 실제 사용자가 앱을 정상적으로 사용하기 어렵다.

시뮬레이터에서는 단축키로 키보드를 숨길 수 있지만, 실제 사용자는 그런 도구가 없다.

따라서 앱 안에서 키보드와 레이아웃을 고려한 처리가 필요하다.

### 지금 단계의 핵심 문제 정의

이번 강의는 아직 해결책을 적용하는 단계가 아니라 문제를 드러내는 단계다.

```text
1. 앱 방향 고정을 해제한다.
2. 기기를 회전해 본다.
3. 가로 모드에서 UI가 어떻게 깨지는지 관찰한다.
4. 키보드가 입력창을 가리는 문제를 확인한다.
5. 다음 단계에서 이를 개선한다.
```

즉 반응형 UI를 만들려면 먼저 여러 환경에서 앱을 실제로 실행해보고, 어떤 부분이 깨지는지 확인해야 한다.

## 프론트엔드 개발자 관점에서의 핵심 차이

### 웹의 viewport 회전과 비슷하지만 설정이 필요하다

웹에서는 브라우저 창을 줄이거나 모바일 기기를 회전하면 viewport가 바뀐다.

앱에서는 먼저 앱이 그 방향 전환을 허용해야 한다.

```text
웹
-> 브라우저 viewport가 자연스럽게 바뀜

React Native + Expo
-> app.json orientation 설정이 방향 전환 허용 여부를 결정
```

따라서 React Native에서는 레이아웃 코드뿐 아니라 앱 설정 파일도 반응형 동작에 영향을 준다.

### 가로 모드는 단순히 너비가 넓어진 상태가 아니다

가로 모드는 너비가 넓어지는 대신 높이가 크게 줄어든다.

```text
portrait
-> 높이가 넉넉함
-> 세로 스택 레이아웃에 유리

landscape
-> 너비가 넓음
-> 높이가 부족함
-> 키보드가 뜨면 더 좁아짐
```

웹에서도 모바일 가로 모드는 종종 까다롭지만, React Native 앱에서는 키보드와 Safe Area까지 함께 고려해야 해서 더 명시적인 대응이 필요하다.

### 화면 방향 지원은 UX 결정이다

모든 앱이 반드시 가로 모드를 지원해야 하는 것은 아니다.

이번 게임 앱은 세로 고정이 더 자연스러울 수도 있다.

```text
세로 고정이 적합한 앱
-> 단순 폼
-> 짧은 흐름의 게임
-> SNS 피드형 앱

가로 모드 지원이 유용한 앱
-> 동영상 앱
-> 지도 앱
-> 게임
-> 차트/데이터 시각화 앱
-> 태블릿 대응 앱
```

중요한 것은 지원 여부를 의식적으로 결정하는 것이다.

## 정리

이번 강의에서는 `app.json`의 `orientation` 설정을 통해 앱의 화면 방향을 제어하는 방법을 확인했다.

`portrait`는 세로 고정, `landscape`는 가로 고정, `default`는 기기 방향에 따른 전환을 허용한다.

화면 회전을 허용하면 기존 세로 기준 레이아웃이 가로 모드에서 깨질 수 있고, 특히 소프트웨어 키보드가 입력창을 가리는 문제가 드러난다.

이제 다음 단계에서는 이러한 방향 전환과 키보드 문제에 대응해 앱이 가로 모드에서도 사용할 수 있도록 개선하게 된다.
