# 3. Dimensions API로 화면 크기 기반 스타일 조정

## 핵심 요약

- `minWidth`, `maxWidth`, `minHeight`, `maxHeight`만으로 해결하기 어려운 반응형 문제도 있다.
- 작은 기기에서는 특정 요소 하나의 너비뿐 아니라 전체적으로 폰트, 패딩, 마진을 줄이고 싶을 수 있다.
- React Native의 내장 `Dimensions` API를 사용하면 현재 사용 가능한 화면 너비와 높이를 가져올 수 있다.
- `Dimensions.get('window')`는 실제 UI가 그려지는 사용 가능 영역을 기준으로 크기 정보를 제공한다.
- 가져온 너비나 높이를 조건문, 삼항 연산자, 계산식에 사용해 스타일 값을 동적으로 결정할 수 있다.
- 예제에서는 작은 화면에서 `NumberContainer`의 `padding`, `margin`, `fontSize`를 줄이고, `Card`의 `marginTop`도 줄인다.

## 학습 목표

- `Dimensions` API의 역할을 이해한다.
- `screen`과 `window`의 차이를 구분한다.
- 현재 화면 너비를 기준으로 조건부 스타일 값을 만드는 방법을 익힌다.
- 작은 기기에서 패딩, 마진, 폰트 크기를 줄이는 패턴을 배운다.
- 정적인 스타일 객체 안에서도 화면 크기 기반 값을 사용할 수 있음을 이해한다.

## 주요 내용

### maxWidth만으로는 부족한 경우

이전 강의에서는 `width`와 `maxWidth`를 조합해 요소의 너비를 유연하게 만들었다.

하지만 작은 화면 대응이 항상 너비 문제만은 아니다.

작은 기기에서는 전체 UI가 상대적으로 너무 커 보일 수 있다.

```text
작은 화면에서 조정하고 싶은 것
-> 폰트 크기
-> 패딩
-> 마진
-> 요소 사이 간격
-> 이미지 크기
```

이런 값들은 단순히 `maxWidth`만으로 해결하기 어렵다.

예를 들어 숫자 컨테이너는 너비가 문제가 아니라 내부 여백과 숫자 크기가 너무 커 보일 수 있다.

```text
NumberContainer
-> padding이 큼
-> margin이 큼
-> fontSize가 큼
```

이럴 때 화면 크기를 직접 확인한 뒤 스타일 값을 다르게 정할 수 있다.

### Dimensions API란?

`Dimensions`는 React Native에 내장된 API다.

JSX에 렌더링하는 컴포넌트가 아니라, JavaScript 코드에서 호출하는 객체다.

```jsx
import { Dimensions } from 'react-native';
```

이 API를 통해 현재 기기의 화면 크기 정보를 가져올 수 있다.

```jsx
const deviceWidth = Dimensions.get('window').width;
```

`Dimensions.get()`은 문자열 인수를 받는다.

```text
Dimensions.get('screen')
Dimensions.get('window')
```

### screen과 window의 차이

`Dimensions.get()`에는 `'screen'` 또는 `'window'`를 전달할 수 있다.

```jsx
Dimensions.get('screen')
Dimensions.get('window')
```

의미는 다음과 같다.

```text
screen
-> 전체 화면 크기
-> Android에서는 상태바 영역까지 포함할 수 있음

window
-> 실제 앱 UI가 그려지고 사용되는 영역
-> Android에서는 상태바를 제외한 영역
```

iOS에서는 둘 사이의 차이가 거의 없을 수 있다.

강의에서는 실제 UI가 사용 가능한 영역을 기준으로 삼기 위해 `'window'`를 사용한다.

```jsx
const deviceWidth = Dimensions.get('window').width;
```

### Dimensions에서 얻을 수 있는 값

`Dimensions.get('window')`는 객체를 반환한다.

여기에는 여러 정보가 들어 있다.

```text
width
-> 사용 가능한 너비

height
-> 사용 가능한 높이

scale
-> 화면 픽셀 밀도 관련 값

fontScale
-> 사용자의 글꼴 확대 설정 관련 값
```

이번 강의에서는 주로 `width`를 사용한다.

```jsx
const deviceWidth = Dimensions.get('window').width;
```

다만 `fontScale`도 실무에서는 중요하다.

사용자가 접근성 설정에서 글자 크기를 키웠다면, 이 값을 고려해 폰트 크기를 조정할 수 있다.

### 화면 너비로 padding 조정하기

예제로 `NumberContainer`의 패딩을 작은 화면에서 줄인다.

```jsx
const deviceWidth = Dimensions.get('window').width;
```

이 값을 스타일 객체 안에서 사용할 수 있다.

```jsx
const styles = StyleSheet.create({
  container: {
    padding: deviceWidth < 380 ? 12 : 24,
  },
});
```

의미는 다음과 같다.

```text
deviceWidth < 380
-> 작은 화면으로 판단
-> padding 12

deviceWidth >= 380
-> 상대적으로 큰 화면
-> padding 24
```

이처럼 삼항 연산자를 사용하면 화면 크기에 따라 스타일 값을 바꿀 수 있다.

### 적절한 break point 찾기

처음에는 `450` 같은 기준을 사용할 수도 있다.

하지만 실제로 확인해보니 iPhone 화면에서도 작은 화면용 스타일이 적용될 수 있다.

그렇다면 기준값을 조정해야 한다.

강의에서는 예시로 `380`을 더 적절한 기준으로 사용한다.

```jsx
padding: deviceWidth < 380 ? 12 : 24
```

여기서 중요한 점은 break point가 정답처럼 고정되어 있지 않다는 것이다.

```text
break point 선택 기준
-> 앱 UI가 실제로 답답해지는 지점
-> 타깃 기기 크기
-> 디자인 요구사항
-> 테스트 결과
```

웹에서 `@media (max-width: 768px)` 같은 값을 프로젝트마다 조정하는 것과 비슷하다.

### margin도 같은 방식으로 조정하기

패딩뿐 아니라 마진도 같은 방식으로 조정할 수 있다.

```jsx
container: {
  padding: deviceWidth < 380 ? 12 : 24,
  margin: deviceWidth < 380 ? 12 : 24,
}
```

작은 기기에서는 요소 사이의 빈 공간도 줄여야 더 많은 콘텐츠가 한 화면에 들어간다.

다만 화면에서 보이는 간격이 하나의 마진만으로 만들어지는 것은 아닐 수 있다.

```text
요소 사이 간격
-> 이전 요소의 margin
-> 다음 요소의 margin
-> 부모의 padding
-> 중간 컨테이너의 marginTop
```

따라서 특정 간격이 여전히 크다면 관련된 다른 컴포넌트의 스타일도 함께 확인해야 한다.

### fontSize도 화면 크기에 따라 조정하기

숫자 컨테이너의 숫자도 작은 화면에서는 조금 작게 만들 수 있다.

```jsx
numberText: {
  fontSize: deviceWidth < 380 ? 28 : 36,
}
```

의미는 다음과 같다.

```text
작은 화면
-> fontSize 28

큰 화면
-> fontSize 36
```

이렇게 하면 작은 기기에서 숫자가 화면에 비해 과하게 커 보이는 문제를 줄일 수 있다.

### Card의 marginTop 줄이기

숫자 컨테이너와 카드 사이의 간격도 작은 기기에서는 줄이는 편이 좋다.

해당 간격은 `GameScreen`의 직접 스타일이 아니라 `Card` 컴포넌트의 `marginTop`에서 만들어질 수 있다.

따라서 `Card` 컴포넌트에서도 `Dimensions`를 사용할 수 있다.

```jsx
import { Dimensions, StyleSheet, View } from 'react-native';

const deviceWidth = Dimensions.get('window').width;
```

그리고 `marginTop`을 조건부로 지정한다.

```jsx
const styles = StyleSheet.create({
  card: {
    marginTop: deviceWidth < 380 ? 18 : 36,
  },
});
```

작은 기기에서는 카드 위쪽 간격이 줄고, 큰 기기에서는 기존의 넉넉한 간격을 유지한다.

```text
작은 화면
-> marginTop 18

큰 화면
-> marginTop 36
```

### 조건문 방식과 계산식 방식

`Dimensions`에서 얻은 값은 두 가지 방식으로 사용할 수 있다.

첫째, 조건부 스타일 값으로 사용할 수 있다.

```jsx
padding: deviceWidth < 380 ? 12 : 24
```

둘째, 계산식에 사용할 수 있다.

```jsx
padding: deviceWidth / 20
```

두 방식은 목적이 다르다.

```text
조건부 스타일
-> 특정 break point 기준으로 명확히 스타일 전환

계산식
-> 화면 크기에 따라 연속적으로 값 변화
```

이번 강의에서는 작은 화면과 큰 화면을 나누는 조건부 스타일 방식을 사용한다.

## 예시 코드 흐름

### NumberContainer

```jsx
import { Dimensions, StyleSheet, Text, View } from 'react-native';

const deviceWidth = Dimensions.get('window').width;

const styles = StyleSheet.create({
  container: {
    borderWidth: 4,
    borderColor: Colors.accent500,
    padding: deviceWidth < 380 ? 12 : 24,
    margin: deviceWidth < 380 ? 12 : 24,
    borderRadius: 8,
    alignItems: 'center',
    justifyContent: 'center',
  },
  numberText: {
    color: Colors.accent500,
    fontSize: deviceWidth < 380 ? 28 : 36,
    fontFamily: 'open-sans-bold',
  },
});
```

### Card

```jsx
import { Dimensions, StyleSheet, View } from 'react-native';

const deviceWidth = Dimensions.get('window').width;

const styles = StyleSheet.create({
  card: {
    marginTop: deviceWidth < 380 ? 18 : 36,
    marginHorizontal: 24,
    padding: 16,
    borderRadius: 8,
    backgroundColor: Colors.primary800,
  },
});
```

실제 스타일 전체는 프로젝트 코드에 따라 더 많은 속성을 포함할 수 있다.

핵심은 `Dimensions.get('window').width`로 얻은 값을 스타일 결정에 사용한다는 점이다.

## 프론트엔드 개발자 관점에서의 핵심 차이

### 미디어 쿼리 대신 JS 조건부 스타일

웹에서는 보통 CSS에서 break point를 처리한다.

```css
@media (max-width: 380px) {
  .number {
    font-size: 28px;
    padding: 12px;
  }
}
```

React Native에서는 JS 값으로 처리한다.

```jsx
fontSize: deviceWidth < 380 ? 28 : 36
```

즉 반응형 로직이 CSS가 아니라 JavaScript 스타일 객체에 들어온다.

### break point는 직접 테스트하며 잡는다

웹에서도 디자인 시안과 실제 브라우저 크기를 보며 break point를 조정하듯이, React Native에서도 에뮬레이터와 실제 기기로 확인해야 한다.

```text
처음 기준값
-> 450

테스트 후 조정
-> 380
```

이 과정은 정교한 수학보다는 실제 UI 관찰이 중요하다.

### 작은 화면 대응은 여러 컴포넌트에 걸쳐 있다

화면 하나가 답답해 보이는 이유는 한 컴포넌트 때문이 아닐 수 있다.

```text
NumberContainer
-> padding, margin, fontSize

Card
-> marginTop

GameScreen
-> 전체 padding, alignItems
```

따라서 React Native에서 반응형 UI를 만들 때는 화면 단위로 전체 간격 구조를 같이 봐야 한다.

## 주의할 점

### Dimensions.get은 정적인 값처럼 사용된다

이번 방식은 파일이 평가될 때 `deviceWidth`를 계산하고 스타일에 넣는다.

```jsx
const deviceWidth = Dimensions.get('window').width;
```

이 방식은 처음 렌더링 시점의 화면 크기를 기준으로 스타일을 만들기에는 간단하다.

하지만 사용자가 기기를 회전해서 화면 크기가 바뀌는 경우에는 별도 대응이 필요하다.

이번 섹션의 뒤쪽에서 화면 방향 변화와 동적 크기 변경을 더 다룰 예정이다.

### 너무 많은 magic number는 관리가 어려워진다

`380`, `12`, `24`, `28`, `36` 같은 숫자가 여러 컴포넌트에 흩어지면 유지보수가 어려울 수 있다.

실무에서는 다음처럼 상수화할 수도 있다.

```jsx
const isSmallDevice = deviceWidth < 380;
```

또는 break point 값을 별도 파일로 관리할 수 있다.

```jsx
const Breakpoints = {
  smallPhone: 380,
};
```

강의에서는 개념을 보여주기 위해 각 파일에서 바로 사용한다.

## 정리

이번 강의에서는 `Dimensions` API를 사용해 현재 사용 가능한 화면 너비를 가져오고, 그 값을 기준으로 스타일을 조건부로 바꾸는 방법을 배웠다.

`NumberContainer`에서는 작은 화면에서 `padding`, `margin`, `fontSize`를 줄였고, `Card`에서는 작은 화면에서 `marginTop`을 줄였다.

핵심은 다음과 같다.

```text
Dimensions.get('window').width
-> 현재 UI가 사용할 수 있는 너비 확인

deviceWidth < 380 ? 작은 값 : 큰 값
-> 화면 크기에 따른 스타일 분기
```

이 방식은 React Native에서 미디어 쿼리 없이도 기기 크기에 맞춰 UI를 조정하는 기본 도구다.
