# 4. Dimensions API로 이미지 크기 조정

## 핵심 요약

- 작은 기기에서는 게임 오버 화면의 이미지가 너무 크게 보인다.
- 이미지 자체는 `width: '100%'`, `height: '100%'`를 사용하지만, 실제 크기는 이미지를 감싸는 컨테이너의 `width`와 `height`가 결정한다.
- 기존 `imageContainer`는 `width: 300`, `height: 300`처럼 고정값을 사용하고 있었다.
- 고정값은 다양한 기기 크기에 대응하기 어렵기 때문에 `Dimensions` API로 화면 너비를 확인해 조건부 크기를 적용한다.
- 원형 이미지를 만들려면 컨테이너의 `width`와 `height`가 반드시 같아야 한다.
- 퍼센트 값은 너비와 높이의 기준이 서로 다를 수 있으므로 원형 이미지 크기 조정에는 적합하지 않다.
- 작은 화면에서는 `150`, 큰 화면에서는 `300`을 사용하고, `borderRadius`는 각각 절반인 `75`, `150`으로 조정한다.

## 학습 목표

- 이미지 크기가 실제로 어떤 컨테이너 스타일에 의해 결정되는지 이해한다.
- `width: '100%'`, `height: '100%'`가 부모 컨테이너 기준이라는 점을 복습한다.
- 원형 이미지를 만들 때 정사각형 컨테이너와 `borderRadius`의 관계를 이해한다.
- 퍼센트 단위가 이미지 원형 처리에 적합하지 않은 이유를 설명할 수 있다.
- `Dimensions` API로 작은 기기와 큰 기기에서 서로 다른 이미지 크기를 적용하는 방법을 익힌다.

## 주요 내용

### 문제 상황: 게임 오버 화면 이미지가 너무 큼

이전 강의에서는 게임 화면의 숫자 컨테이너와 카드 간격을 작은 기기에 맞게 줄였다.

이제 게임 오버 화면을 보면 또 다른 문제가 보인다.

```text
게임 오버 화면의 문제
-> 이미지가 작은 화면에서 너무 큼
-> 제목, 이미지, 결과 문장, 버튼이 전체적으로 답답해짐
-> 특히 이미지가 화면의 많은 영역을 차지함
```

가장 먼저 조정할 대상은 이미지다.

나머지 텍스트와 버튼도 조정할 수 있지만, 이미지가 가장 명확하게 큰 요소이기 때문이다.

### 이미지 크기를 결정하는 실제 위치

게임 오버 화면에는 `Image` 컴포넌트가 있다.

이미지 자체에는 보통 다음 스타일이 적용되어 있다.

```jsx
image: {
  width: '100%',
  height: '100%',
}
```

여기서 `100%`는 화면 전체가 아니라 이미지의 부모 컨테이너를 기준으로 한다.

이미지를 감싸는 `View`에는 `imageContainer` 스타일이 적용되어 있다.

```jsx
imageContainer: {
  width: 300,
  height: 300,
  borderRadius: 150,
  overflow: 'hidden',
}
```

따라서 실제 이미지 크기를 결정하는 것은 `Image` 자체보다 이 컨테이너다.

```text
Image
-> width: '100%'
-> height: '100%'

부모 View(imageContainer)
-> width: 300
-> height: 300

결과
-> Image도 300 x 300
```

이미지 크기를 조정하려면 `imageContainer`의 크기를 바꿔야 한다.

### 고정값의 한계

기존 코드는 다음처럼 고정 크기를 사용했다.

```jsx
width: 300,
height: 300,
borderRadius: 150,
```

큰 화면에서는 괜찮아 보일 수 있다.

하지만 작은 화면에서는 이 값이 너무 크다.

```text
큰 기기
-> 300 x 300 이미지가 적당함

작은 기기
-> 300 x 300 이미지가 화면을 너무 많이 차지함
```

모바일 앱은 다양한 화면 크기에서 실행되므로, 이런 고정값은 자주 문제가 된다.

### 왜 퍼센트 값이 적합하지 않은가

이미지 크기를 상대적으로 만들기 위해 다음처럼 생각할 수도 있다.

```jsx
width: '50%',
height: '50%',
```

하지만 원형 이미지를 만들 때는 이 방식이 적합하지 않다.

이유는 `width: '50%'`와 `height: '50%'`가 서로 다른 기준을 참조할 수 있기 때문이다.

```text
width: '50%'
-> 부모 컨테이너의 너비 50%

height: '50%'
-> 부모 컨테이너의 높이 50%
```

부모의 너비와 높이가 같지 않다면 두 값은 서로 다른 숫자가 된다.

```text
부모 너비: 360
부모 높이: 700

width 50%
-> 180

height 50%
-> 350
```

이렇게 되면 정사각형이 아니라 직사각형이 된다.

원형 이미지를 만들려면 컨테이너가 반드시 정사각형이어야 한다.

```text
원형 이미지 조건
-> width === height
-> borderRadius === width / 2
-> overflow: 'hidden'
```

따라서 이번 상황에서는 퍼센트보다 `Dimensions`를 이용한 조건부 숫자 값이 더 적합하다.

### Dimensions API로 화면 너비 가져오기

`GameOverScreen`에서도 `Dimensions`를 import한다.

```jsx
import { Dimensions, Image, StyleSheet, Text, View } from 'react-native';
```

그리고 사용 가능한 화면 너비를 가져온다.

```jsx
const deviceWidth = Dimensions.get('window').width;
```

`width` 대신 `height`를 기준으로 삼을 수도 있다.

```jsx
const deviceHeight = Dimensions.get('window').height;
```

이번 예제에서는 이미지 크기를 화면 너비 기준으로 조정한다.

### 작은 화면과 큰 화면의 이미지 크기 분기

작은 화면 기준을 `380`으로 잡고, 이미지 크기를 조건부로 바꾼다.

```jsx
const imageSize = deviceWidth < 380 ? 150 : 300;
```

강의에서는 스타일 객체 안에서 바로 삼항 연산자를 사용할 수 있다.

```jsx
imageContainer: {
  width: deviceWidth < 380 ? 150 : 300,
  height: deviceWidth < 380 ? 150 : 300,
  borderRadius: deviceWidth < 380 ? 75 : 150,
}
```

의미는 다음과 같다.

```text
작은 화면
-> width: 150
-> height: 150
-> borderRadius: 75

큰 화면
-> width: 300
-> height: 300
-> borderRadius: 150
```

작은 기기에서는 이미지가 훨씬 작아지고, 큰 기기에서는 기존처럼 큰 이미지를 유지한다.

### borderRadius도 함께 조정해야 한다

이미지 컨테이너 크기를 줄이면 `borderRadius`도 함께 줄여야 한다.

원형을 유지하려면 `borderRadius`가 너비와 높이의 절반이어야 한다.

```text
300 x 300
-> borderRadius 150

150 x 150
-> borderRadius 75
```

만약 이미지 크기만 줄이고 `borderRadius`를 그대로 두면 의도와 다르게 보일 수 있다.

따라서 크기와 반지름은 같은 기준으로 함께 관리해야 한다.

### 더 깔끔한 작성 방식

강의 흐름에서는 스타일 안에 삼항 연산자를 직접 넣어도 된다.

하지만 같은 조건이 여러 번 반복되면 상수로 빼는 편이 더 읽기 좋다.

```jsx
const deviceWidth = Dimensions.get('window').width;
const imageSize = deviceWidth < 380 ? 150 : 300;

const styles = StyleSheet.create({
  imageContainer: {
    width: imageSize,
    height: imageSize,
    borderRadius: imageSize / 2,
    overflow: 'hidden',
  },
});
```

이 방식은 다음 장점이 있다.

```text
1. width와 height가 같은 값임이 명확함
2. borderRadius 계산이 자연스러움
3. 같은 조건식을 세 번 반복하지 않아도 됨
4. 나중에 크기 기준을 바꾸기 쉬움
```

강의에서는 개념 설명을 위해 직접 삼항 연산자를 반복하지만, 실무에서는 이런 식의 상수화가 더 깔끔할 수 있다.

## 예시 코드

```jsx
import { Dimensions, Image, StyleSheet, Text, View } from 'react-native';

const deviceWidth = Dimensions.get('window').width;

function GameOverScreen({ roundsNumber, userNumber, onStartNewGame }) {
  return (
    <View style={styles.rootContainer}>
      {/* ... */}
      <View style={styles.imageContainer}>
        <Image
          style={styles.image}
          source={require('../assets/images/success.png')}
        />
      </View>
      {/* ... */}
    </View>
  );
}

const styles = StyleSheet.create({
  imageContainer: {
    width: deviceWidth < 380 ? 150 : 300,
    height: deviceWidth < 380 ? 150 : 300,
    borderRadius: deviceWidth < 380 ? 75 : 150,
    overflow: 'hidden',
    borderWidth: 3,
  },
  image: {
    width: '100%',
    height: '100%',
  },
});
```

정리하면 `Image`는 부모 크기를 꽉 채우고, 부모인 `imageContainer`가 실제 크기와 원형 모양을 결정한다.

## 프론트엔드 개발자 관점에서의 핵심 차이

### 퍼센트가 항상 좋은 반응형 해법은 아니다

웹에서도 원형 아바타를 만들 때 `width`와 `height`를 동일한 값으로 맞추는 일이 많다.

React Native에서도 마찬가지다.

다만 다음처럼 너비와 높이에 퍼센트를 넣으면 정사각형이 보장되지 않는다.

```jsx
width: '50%',
height: '50%',
```

퍼센트는 각각 부모의 너비와 높이를 기준으로 계산되기 때문이다.

```text
정사각형이 중요한 UI
-> 숫자 기반 크기 또는 계산된 동일 값 사용

단순히 부모 너비에 맞추는 UI
-> 퍼센트 사용 가능
```

### 반응형은 “상대값”만이 아니다

반응형 UI라고 해서 모든 값을 퍼센트로 만들어야 하는 것은 아니다.

이번 예시는 오히려 조건부 고정값이 더 적절하다.

```text
작은 화면
-> 150

큰 화면
-> 300
```

즉 반응형 설계는 다음 중 상황에 맞는 방식을 고르는 것이다.

```text
퍼센트 값
-> 부모 크기에 비례해야 할 때

min/max 제약
-> 너무 작거나 커지는 것을 막을 때

Dimensions 조건부 값
-> 화면 크기별로 명확히 다른 스타일을 줄 때

계산식
-> 화면 크기에 따라 연속적으로 변화해야 할 때
```

### 이미지와 마스크는 컨테이너가 중요하다

React Native에서 이미지를 원형으로 자를 때는 이미지 자체보다 컨테이너 설계가 중요하다.

```text
View
-> width / height / borderRadius / overflow

Image
-> width: '100%'
-> height: '100%'
```

웹에서 `overflow: hidden`을 가진 원형 wrapper 안에 `img`를 넣는 패턴과 거의 같다.

## 정리

이번 강의에서는 게임 오버 화면의 이미지가 작은 기기에서 너무 크게 보이는 문제를 `Dimensions` API로 해결했다.

핵심은 이미지 자체가 아니라 이미지를 감싸는 컨테이너의 크기를 화면 너비에 따라 조정하는 것이다.

원형 이미지를 유지하려면 `width`와 `height`가 같아야 하며, `borderRadius`는 그 절반이어야 한다.

따라서 퍼센트 값보다는 `Dimensions.get('window').width`를 기준으로 작은 화면에서는 `150`, 큰 화면에서는 `300` 같은 조건부 숫자 값을 사용하는 것이 더 적합하다.

이 예시는 `Dimensions` API가 폰트나 패딩뿐 아니라 이미지와 시각적 구조에도 유용하게 쓰일 수 있음을 보여준다.
