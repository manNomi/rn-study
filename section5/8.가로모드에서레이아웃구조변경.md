# 8. 가로 모드에서 레이아웃 구조 변경

## 핵심 요약

- 가로 모드에서는 세로 모드용 레이아웃을 그대로 쓰면 높이가 부족해 UI가 답답하거나 사용하기 어려워진다.
- 단순히 `margin`, `padding`, `fontSize`를 줄이는 것만으로 해결되지 않을 때는 JSX 구조 자체를 바꿀 수 있다.
- `useWindowDimensions`로 현재 `width`와 `height`를 가져와, 특정 너비 이상이면 넓은 화면용 레이아웃을 렌더링한다.
- 기본 레이아웃은 `NumberContainer` 아래에 `Card`를 두는 구조다.
- 넓은 화면에서는 `NumberContainer` 좌우에 `+`, `-` 버튼을 배치하고, 불필요한 안내 텍스트나 카드 래퍼를 제거해 높이를 절약한다.
- 조건부 렌더링은 스타일뿐 아니라 컴포넌트 구조와 콘텐츠 구성까지 바꿀 수 있다.
- 가로 모드에서도 게임을 플레이할 수 있는 기본 구조가 마련되지만, 게임 오버 화면은 아직 추가 대응이 필요하다.

## 학습 목표

- 화면 크기에 따라 JSX 레이아웃 구조를 다르게 렌더링하는 방법을 이해한다.
- `useWindowDimensions`를 스타일 값뿐 아니라 컴포넌트 구조 분기에 사용하는 방법을 익힌다.
- 세로 모드와 가로 모드에서 필요한 UI 요소가 다를 수 있음을 이해한다.
- `Fragment`를 사용해 여러 형제 컴포넌트를 하나의 변수에 담는 방법을 복습한다.
- 넓은 화면에서 `flexDirection: 'row'`를 이용해 버튼과 숫자 컨테이너를 가로 배치하는 패턴을 배운다.

## 주요 내용

### 문제 상황: GameScreen이 가로 모드에서 사용하기 어려움

이전 단계에서 시작 화면의 키보드 문제를 해결했다.

이제 게임 화면으로 넘어가면 가로 모드에서 또 다른 문제가 보인다.

```text
가로 모드의 GameScreen 문제
-> 화면 높이가 부족함
-> 숫자 컨테이너, 카드, 로그 영역이 세로로 쌓여 답답함
-> Android 작은 기기에서는 거의 사용하기 어려움
-> iOS에서도 사용할 수는 있지만 보기 좋지 않음
```

세로 모드에서는 위에서 아래로 쌓는 구조가 자연스럽다.

하지만 가로 모드에서는 높이가 부족하고 너비가 넓기 때문에 다른 배치를 사용하는 편이 좋다.

### 스타일 값 조정만으로는 부족할 때

이전 강의들에서는 다음처럼 스타일 값을 조정했다.

```text
작은 화면
-> padding 줄이기
-> margin 줄이기
-> fontSize 줄이기
-> image size 줄이기
```

하지만 이번 문제는 단순히 값 몇 개를 줄이는 것으로 충분하지 않다.

기존 구조 자체가 가로 모드에 맞지 않기 때문이다.

```text
기존 portrait 구조
-> Title
-> NumberContainer
-> Card
   -> Higher or lower?
   -> - button / + button
-> Log list
```

가로 모드에서는 `NumberContainer`와 버튼을 나란히 배치하면 세로 공간을 절약할 수 있다.

### useWindowDimensions로 현재 화면 크기 확인

`GameScreen`에서 `useWindowDimensions`를 사용한다.

```jsx
import { useWindowDimensions } from 'react-native';
```

컴포넌트 함수 안에서 호출한다.

```jsx
function GameScreen({ userNumber, onGameOver }) {
  const { width, height } = useWindowDimensions();

  // ...
}
```

이번에는 `width`를 기준으로 넓은 화면인지 판단한다.

```text
width가 충분히 큼
-> 가로 모드이거나 넓은 화면일 가능성이 큼
-> wide layout 사용

width가 작음
-> 기존 portrait layout 사용
```

강의에서는 예시로 `500` 또는 `600` 같은 값을 기준으로 삼을 수 있다고 설명한다.

```jsx
if (width > 500) {
  // wide layout
}
```

실제 기준값은 UI를 보며 조정하면 된다.

### 기본 content 변수 만들기

먼저 기본적으로 렌더링할 레이아웃을 변수에 담는다.

```jsx
let content = (
  <>
    <NumberContainer>{currentGuess}</NumberContainer>
    <Card>
      <InstructionText style={styles.instructionText}>
        Higher or lower?
      </InstructionText>
      <View style={styles.buttonsContainer}>
        <View style={styles.buttonContainer}>
          <PrimaryButton onPress={nextGuessHandler.bind(this, 'lower')}>
            -
          </PrimaryButton>
        </View>
        <View style={styles.buttonContainer}>
          <PrimaryButton onPress={nextGuessHandler.bind(this, 'greater')}>
            +
          </PrimaryButton>
        </View>
      </View>
    </Card>
  </>
);
```

`NumberContainer`와 `Card`는 형제 컴포넌트다.

JSX에서 형제 컴포넌트 여러 개를 하나의 값으로 담으려면 `Fragment`가 필요하다.

```jsx
<>
  <NumberContainer />
  <Card />
</>
```

이 기본 `content`는 세로 모드용 레이아웃이다.

### content를 JSX에 삽입하기

이제 `return` 안에서 기존 `NumberContainer`와 `Card` 코드를 직접 쓰는 대신 `content` 변수를 렌더링한다.

```jsx
return (
  <View style={styles.screen}>
    <Title>Opponent's Guess</Title>
    {content}
    <View style={styles.listContainer}>
      <FlatList
        data={guessRounds}
        renderItem={...}
      />
    </View>
  </View>
);
```

이 단계만 적용하면 화면은 기존과 동일하다.

차이는 이제 조건에 따라 `content`를 다른 JSX로 교체할 수 있다는 점이다.

### 넓은 화면용 content로 교체하기

현재 화면 너비가 충분히 크면 `content`를 가로 모드용 레이아웃으로 바꾼다.

```jsx
if (width > 500) {
  content = (
    <>
      <View style={styles.buttonsContainerWide}>
        <View style={styles.buttonContainer}>
          <PrimaryButton onPress={nextGuessHandler.bind(this, 'lower')}>
            -
          </PrimaryButton>
        </View>
        <NumberContainer>{currentGuess}</NumberContainer>
        <View style={styles.buttonContainer}>
          <PrimaryButton onPress={nextGuessHandler.bind(this, 'greater')}>
            +
          </PrimaryButton>
        </View>
      </View>
    </>
  );
}
```

이 구조는 기존과 다르다.

```text
기존 세로 구조
-> NumberContainer
-> Card
   -> 안내 텍스트
   -> 버튼들

넓은 화면 구조
-> - 버튼
-> NumberContainer
-> + 버튼
```

가로 모드에서는 높이를 아끼기 위해 카드 래퍼와 안내 텍스트를 제거하고, 버튼을 숫자 좌우에 배치한다.

### 레이아웃 전환은 콘텐츠 전환이기도 하다

중요한 점은 두 레이아웃이 같은 컴포넌트를 단순히 다른 스타일로 보여주는 수준이 아니라는 것이다.

가로 모드에서는 일부 콘텐츠를 제거한다.

```text
portrait
-> Higher or lower? 안내 문구 있음
-> Card로 버튼 영역 감쌈

landscape / wide
-> 안내 문구 제거 가능
-> Card 제거 가능
-> 버튼을 숫자 좌우에 배치
```

React Native에서 반응형 UI를 만들 때는 스타일뿐 아니라 렌더링하는 컴포넌트 자체를 다르게 구성할 수 있다.

웹 React에서도 화면 크기에 따라 다른 컴포넌트를 렌더링하는 것과 같은 사고방식이다.

### buttonsContainerWide 스타일

넓은 화면용 레이아웃에서는 버튼과 숫자 컨테이너를 가로로 배치해야 한다.

이를 위해 새 스타일을 추가한다.

```jsx
buttonsContainerWide: {
  flexDirection: 'row',
  alignItems: 'center',
}
```

`flexDirection: 'row'`는 자식 요소를 좌우로 배치한다.

`alignItems: 'center'`는 교차축인 세로 방향 중앙 정렬을 담당한다.

```text
flexDirection: 'row'
-> - 버튼, NumberContainer, + 버튼을 가로 배치

alignItems: 'center'
-> 세 요소를 세로 가운데 정렬
```

이렇게 하면 가로 모드에서 훨씬 플레이하기 쉬운 구조가 된다.

### 로그 영역은 아직 추가 조정 가능

가로 레이아웃으로 바꾼 뒤에도 로그 영역이 다소 잘려 보이거나 접근성이 떨어질 수 있다.

그래서 강의에서는 안내 문구를 제거해 로그 영역을 조금 더 확보한다.

추가로 다음 값들을 조정할 수도 있다.

```text
listContainer
-> paddingTop / paddingBottom 줄이기

NumberContainer
-> margin 줄이기

Card
-> 가로 모드에서는 아예 제거

InstructionText
-> 가로 모드에서는 제거
```

이번 강의의 목적은 완벽한 디자인이 아니라, 화면 크기에 따라 구조 자체를 바꿀 수 있다는 점을 보여주는 것이다.

## 예시 코드 흐름

```jsx
function GameScreen({ userNumber, onGameOver }) {
  const { width, height } = useWindowDimensions();

  let content = (
    <>
      <NumberContainer>{currentGuess}</NumberContainer>
      <Card>
        <InstructionText style={styles.instructionText}>
          Higher or lower?
        </InstructionText>
        <View style={styles.buttonsContainer}>
          <View style={styles.buttonContainer}>
            <PrimaryButton onPress={nextGuessHandler.bind(this, 'lower')}>
              -
            </PrimaryButton>
          </View>
          <View style={styles.buttonContainer}>
            <PrimaryButton onPress={nextGuessHandler.bind(this, 'greater')}>
              +
            </PrimaryButton>
          </View>
        </View>
      </Card>
    </>
  );

  if (width > 500) {
    content = (
      <View style={styles.buttonsContainerWide}>
        <View style={styles.buttonContainer}>
          <PrimaryButton onPress={nextGuessHandler.bind(this, 'lower')}>
            -
          </PrimaryButton>
        </View>
        <NumberContainer>{currentGuess}</NumberContainer>
        <View style={styles.buttonContainer}>
          <PrimaryButton onPress={nextGuessHandler.bind(this, 'greater')}>
            +
          </PrimaryButton>
        </View>
      </View>
    );
  }

  return (
    <View style={styles.screen}>
      <Title>Opponent's Guess</Title>
      {content}
      {/* log list */}
    </View>
  );
}
```

핵심은 `content` 변수를 먼저 기본 레이아웃으로 만들고, 특정 조건에서 다른 JSX로 덮어쓴다는 점이다.

## 프론트엔드 개발자 관점에서의 핵심 차이

### 반응형은 스타일만 바꾸는 것이 아니다

웹에서 반응형 UI를 만들 때도 미디어 쿼리로 단순히 크기만 바꾸는 경우가 있고, 아예 레이아웃 구조를 바꾸는 경우가 있다.

React Native도 마찬가지다.

```text
단순 반응형
-> padding 줄이기
-> fontSize 줄이기
-> imageSize 줄이기

구조적 반응형
-> 컴포넌트 배치 변경
-> 일부 UI 제거
-> 다른 JSX 렌더링
```

이번 강의는 구조적 반응형의 예시다.

### CSS media query 대신 조건부 JSX

웹에서는 다음처럼 미디어 쿼리로 레이아웃을 바꿀 수 있다.

```css
@media (orientation: landscape) {
  .controls {
    flex-direction: row;
  }
}
```

React Native에서는 `useWindowDimensions`와 조건부 렌더링을 사용할 수 있다.

```jsx
if (width > 500) {
  content = <WideLayout />;
}
```

즉 반응형 분기가 JavaScript와 JSX 안으로 들어온다.

### 같은 화면에도 두 개의 UI 전략이 있을 수 있다

세로 모드와 가로 모드는 단순히 크기만 다른 것이 아니라 사용 방식도 다를 수 있다.

```text
portrait
-> 위에서 아래로 읽고 조작
-> 설명 텍스트와 카드 구조가 자연스러움

landscape
-> 좌우 공간 활용
-> 버튼을 숫자 좌우에 배치하는 것이 효율적
```

따라서 두 모드에서 꼭 같은 UI 요소를 모두 유지할 필요는 없다.

필요하다면 가로 모드에서는 안내 문구를 제거하거나, 컨테이너를 단순화해 사용성을 높일 수 있다.

## 주의할 점

### width 기준이 항상 orientation을 의미하지는 않는다

강의에서는 `width > 500`을 넓은 화면 기준으로 사용한다.

하지만 이것이 항상 가로 모드만 의미하지는 않는다.

```text
width > 500
-> 가로 모드일 가능성이 큼
-> 하지만 태블릿 세로 모드도 해당될 수 있음
```

따라서 실제 앱에서는 “가로 모드인지”보다 “이 정도 너비가 있으면 이 레이아웃이 적합한지”를 기준으로 생각하는 편이 좋다.

### 중복 JSX가 늘어날 수 있다

조건부로 완전히 다른 JSX를 작성하면 코드 중복이 생길 수 있다.

예를 들어 `PrimaryButton` 두 개가 두 레이아웃에 반복된다.

실무에서는 다음처럼 더 분리할 수 있다.

```text
GuessControlsPortrait
GuessControlsWide
GuessButtons
```

강의에서는 개념을 보여주기 위해 한 파일 안에서 작성한다.

## 정리

이번 강의에서는 `useWindowDimensions`를 사용해 화면 너비가 충분히 넓을 때 게임 화면의 JSX 구조 자체를 바꾸는 방법을 배웠다.

세로 모드에서는 `NumberContainer` 아래에 `Card`와 안내 문구, 버튼을 배치하고, 넓은 화면에서는 `NumberContainer` 좌우에 `+`, `-` 버튼을 배치해 세로 공간을 절약한다.

이는 React Native에서 반응형 UI가 단순히 스타일 값만 바꾸는 것이 아니라, 상황에 따라 렌더링하는 컴포넌트 구조와 콘텐츠 자체를 바꿀 수 있음을 보여준다.

이제 게임 화면은 가로 모드에서도 기본적으로 플레이 가능해졌고, 다음 단계에서는 아직 가로 모드에서 제대로 보이지 않는 게임 오버 화면을 개선한다.
