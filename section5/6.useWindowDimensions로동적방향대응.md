# 6. useWindowDimensions로 동적 방향 대응

## 핵심 요약

- `Dimensions.get()`을 파일 상단에서 호출하면 초기 화면 크기만 고정해서 사용하게 된다.
- 사용자가 앱 실행 중에 기기를 회전하면 파일 상단의 코드는 다시 실행되지 않으므로, 이전 방향 기준 스타일이 남을 수 있다.
- 화면 방향이나 크기 변화에 동적으로 반응해야 한다면 `useWindowDimensions` 훅을 사용하는 것이 더 적합하다.
- `useWindowDimensions`는 컴포넌트 함수 안에서 호출하며, 화면 크기가 바뀌면 컴포넌트를 다시 실행해 최신 `width`, `height`를 제공한다.
- 동적으로 계산한 값은 스타일 배열을 이용해 기존 `StyleSheet` 스타일과 병합할 수 있다.
- 예제에서는 `StartGameScreen`의 `marginTop`을 현재 `height`에 따라 `30` 또는 `100`으로 바꾼다.

## 학습 목표

- `Dimensions.get()`을 파일 상단에서 사용할 때의 한계를 이해한다.
- `useWindowDimensions` 훅의 역할과 사용 위치를 익힌다.
- 화면 회전 시 컴포넌트가 최신 화면 크기를 기준으로 다시 렌더링되는 흐름을 이해한다.
- 동적 스타일 값을 기존 `StyleSheet` 스타일과 병합하는 방법을 복습한다.
- 화면 방향 대응에서 정적 크기 계산과 동적 크기 계산을 구분한다.

## 주요 내용

### 가로 모드에서 marginTop 줄이기

가로 모드에서는 화면 높이가 줄어든다.

그런데 시작 화면의 루트 컨테이너에는 기존에 큰 `marginTop`이 고정되어 있었다.

```jsx
rootContainer: {
  flex: 1,
  marginTop: 100,
  alignItems: 'center',
}
```

세로 모드에서는 괜찮아 보일 수 있지만, 가로 모드에서는 위쪽 여백이 너무 커진다.

```text
portrait
-> 높이가 넉넉함
-> marginTop 100도 괜찮음

landscape
-> 높이가 부족함
-> marginTop 100은 과함
```

그래서 화면 높이가 작을 때는 `marginTop`을 줄이고 싶다.

### Dimensions.get으로 시도하기

먼저 이전 강의에서 배운 `Dimensions` API를 사용할 수 있다.

```jsx
import { Dimensions } from 'react-native';

const deviceHeight = Dimensions.get('window').height;
```

그리고 스타일 안에서 조건부 값을 적용한다.

```jsx
rootContainer: {
  flex: 1,
  marginTop: deviceHeight < 380 ? 30 : 100,
  alignItems: 'center',
}
```

의미는 다음과 같다.

```text
deviceHeight < 380
-> 가로 모드처럼 높이가 낮은 환경으로 판단
-> marginTop 30

deviceHeight >= 380
-> 세로 모드처럼 높이가 충분한 환경
-> marginTop 100
```

이 방식은 앱이 처음 로드될 때의 방향에 맞춰 스타일을 정하는 데는 동작한다.

하지만 중요한 문제가 있다.

### 파일 상단 Dimensions 값의 한계

`Dimensions.get()`을 컴포넌트 함수 밖, 파일 상단에서 호출하면 그 코드는 파일이 처음 평가될 때 한 번 실행된다.

```jsx
const deviceHeight = Dimensions.get('window').height;
```

이 값은 사용자가 앱 실행 중에 기기를 회전해도 자동으로 새로 계산되지 않는다.

```text
앱을 가로 모드에서 시작
-> deviceHeight는 가로 모드 높이
-> marginTop 30

사용자가 세로 모드로 회전
-> 파일 상단 코드는 다시 실행되지 않음
-> marginTop 30이 계속 남을 수 있음
```

컴포넌트 함수는 상태 변화나 props 변화로 다시 실행될 수 있다.

하지만 컴포넌트 함수 바깥의 파일 스코프 코드는 일반적으로 다시 실행되지 않는다.

```text
파일 스코프 코드
-> 처음 import/평가될 때 실행
-> 화면 회전 때 자동 재실행되지 않음

컴포넌트 함수 내부 코드
-> 렌더링마다 실행
-> 상태, props, 훅 변화에 반응 가능
```

따라서 화면 크기 변화에 반응해야 하는 값은 컴포넌트 함수 안에서 계산해야 한다.

### useWindowDimensions 훅

React Native는 `Dimensions` API를 훅 형태로 사용할 수 있는 방법을 제공한다.

바로 `useWindowDimensions`다.

```jsx
import { useWindowDimensions } from 'react-native';
```

React 훅이므로 컴포넌트 함수 안에서 호출해야 한다.

```jsx
function StartGameScreen({ onPickNumber }) {
  const { width, height } = useWindowDimensions();

  // ...
}
```

`useWindowDimensions()`는 현재 window의 크기 정보를 반환한다.

객체 구조 분해를 사용해 `width`와 `height`를 꺼낼 수 있다.

```jsx
const { width, height } = useWindowDimensions();
```

핵심은 이 훅이 화면 크기 변화를 감지한다는 점이다.

```text
화면 방향 변경
-> window width/height 변경
-> useWindowDimensions가 최신 값 제공
-> 컴포넌트 함수 다시 실행
-> 동적 스타일 값 다시 계산
```

### 동적 marginTop 계산하기

이제 컴포넌트 함수 내부에서 현재 `height`를 기준으로 여백을 계산한다.

```jsx
function StartGameScreen({ onPickNumber }) {
  const { height } = useWindowDimensions();

  const marginTopDistance = height < 380 ? 30 : 100;

  return (
    <View style={[styles.rootContainer, { marginTop: marginTopDistance }]}>
      {/* ... */}
    </View>
  );
}
```

이렇게 하면 방향이 바뀔 때마다 `height`가 갱신되고, `marginTopDistance`도 다시 계산된다.

```text
세로 모드
-> height 큼
-> marginTopDistance = 100

가로 모드
-> height 작음
-> marginTopDistance = 30

다시 세로 모드
-> height 다시 커짐
-> marginTopDistance = 100
```

### 기존 StyleSheet 스타일과 동적 스타일 병합

기존 `rootContainer` 스타일에는 `flex`, `alignItems` 같은 정적 스타일을 남겨둔다.

```jsx
const styles = StyleSheet.create({
  rootContainer: {
    flex: 1,
    alignItems: 'center',
  },
});
```

동적으로 바뀌는 `marginTop`은 JSX에서 스타일 배열로 병합한다.

```jsx
<View style={[styles.rootContainer, { marginTop: marginTopDistance }]}>
```

React Native의 `style` prop은 객체뿐 아니라 배열도 받을 수 있다.

```text
style 배열
-> 왼쪽에서 오른쪽으로 적용
-> 뒤쪽 스타일이 앞쪽 스타일을 덮어쓸 수 있음
```

따라서 정적인 스타일은 `StyleSheet`에 두고, 화면 크기에 따라 바뀌는 값은 컴포넌트 함수 안에서 계산해 추가할 수 있다.

### 왜 이 방식이 동적으로 반응하는가

`useWindowDimensions`는 화면 크기 변화에 맞춰 컴포넌트를 다시 실행시킨다.

그래서 다음 흐름이 가능하다.

```text
1. 앱이 세로 모드로 시작
2. height가 큰 값으로 들어옴
3. marginTopDistance = 100
4. 사용자가 기기를 가로로 회전
5. height가 작은 값으로 변경됨
6. 컴포넌트 함수가 다시 실행됨
7. marginTopDistance = 30
8. UI가 가로 모드에 맞게 조정됨
9. 다시 세로로 회전하면 marginTopDistance = 100
```

이전 `Dimensions.get()` 방식은 처음 계산된 값을 계속 사용했기 때문에 이런 동적 반응이 어려웠다.

### 언제 Dimensions.get을 쓰고 언제 useWindowDimensions를 쓸까

두 방식은 모두 유용하지만 목적이 다르다.

```text
Dimensions.get()
-> 초기 화면 크기 기준으로 값을 한 번 계산해도 충분할 때
-> 파일 스코프나 스타일 정의에서 간단히 사용 가능

useWindowDimensions()
-> 화면 크기와 방향 변화에 실시간으로 반응해야 할 때
-> 컴포넌트 함수 내부에서 사용
```

이번처럼 사용자가 앱을 사용하는 중에 기기를 회전할 수 있다면 `useWindowDimensions`가 더 안전하다.

## 예시 코드

```jsx
import { StyleSheet, TextInput, View, useWindowDimensions } from 'react-native';

function StartGameScreen({ onPickNumber }) {
  const { height } = useWindowDimensions();

  const marginTopDistance = height < 380 ? 30 : 100;

  return (
    <View style={[styles.rootContainer, { marginTop: marginTopDistance }]}>
      {/* title, card, input, buttons */}
    </View>
  );
}

const styles = StyleSheet.create({
  rootContainer: {
    flex: 1,
    alignItems: 'center',
  },
});
```

실제 코드에서는 `TextInput`, `PrimaryButton`, `Card`, `Title` 등 다른 컴포넌트도 함께 들어간다.

핵심은 `marginTop`처럼 화면 크기에 따라 달라져야 하는 값을 `useWindowDimensions` 기반으로 컴포넌트 내부에서 계산한다는 것이다.

## 프론트엔드 개발자 관점에서의 핵심 차이

### 브라우저 resize 이벤트와 비슷한 역할

웹에서는 화면 크기가 바뀔 때 CSS 미디어 쿼리가 자동으로 반응하거나, 필요하면 resize 이벤트를 사용할 수 있다.

React Native에서는 `useWindowDimensions`가 이와 비슷한 역할을 한다.

```text
웹
-> CSS media query
-> window resize event

React Native
-> useWindowDimensions
-> width / height 변경 감지
```

컴포넌트가 최신 크기 값을 받아 다시 렌더링되므로, React다운 방식으로 반응형 값을 계산할 수 있다.

### 파일 스코프와 렌더 스코프를 구분해야 한다

웹 React에서도 모듈 상단의 코드는 컴포넌트 렌더링마다 다시 실행되지 않는다.

React Native에서도 마찬가지다.

```jsx
// 파일 스코프
const deviceHeight = Dimensions.get('window').height;

function Component() {
  // 렌더 스코프
}
```

화면 회전처럼 런타임에 변할 수 있는 값은 컴포넌트 안에서 훅으로 다루는 편이 안전하다.

```jsx
function Component() {
  const { height } = useWindowDimensions();
}
```

### 동적 스타일은 배열 병합으로 처리하기 좋다

기본 스타일과 동적 스타일을 분리하면 코드가 읽기 쉬워진다.

```jsx
style={[styles.rootContainer, { marginTop: marginTopDistance }]}
```

웹에서 기본 class와 조건부 inline style을 함께 쓰는 감각과 비슷하다.

```jsx
<div className="root" style={{ marginTop }} />
```

## 주의할 점

### 모든 값을 동적으로 만들 필요는 없다

모든 스타일 값을 `useWindowDimensions`로 계산하면 코드가 복잡해진다.

정말 화면 변화에 반응해야 하는 값만 동적으로 만들고, 나머지는 `StyleSheet`에 두는 편이 좋다.

```text
정적 스타일
-> 색상
-> 폰트 패밀리
-> 기본 borderRadius

동적 스타일
-> 방향에 따라 달라지는 margin
-> 화면 크기에 따라 달라지는 이미지 크기
-> 가로/세로 모드에 따라 달라지는 레이아웃
```

### 다른 컴포넌트도 실험해볼 수 있다

강의에서는 시간상 `StartGameScreen`의 `marginTop`만 조정한다.

하지만 같은 훅을 다른 컴포넌트에도 적용할 수 있다.

```text
적용 후보
-> GameOverScreen 이미지 크기
-> NumberContainer 패딩과 폰트 크기
-> Card 간격
-> 버튼 배치
-> FlatList 영역 크기
```

앱 전체를 더 자연스럽게 만들려면 여러 화면에서 이 패턴을 조합할 수 있다.

## 정리

이번 강의에서는 `Dimensions.get()`을 파일 상단에서 사용하는 방식의 한계를 확인하고, 화면 크기 변화에 동적으로 반응하기 위해 `useWindowDimensions` 훅을 사용하는 방법을 배웠다.

`useWindowDimensions`는 컴포넌트 함수 안에서 현재 `width`와 `height`를 제공하며, 화면 방향이 바뀌면 컴포넌트를 다시 실행해 최신 값을 반영한다.

이 값을 이용해 `marginTopDistance` 같은 동적 스타일 값을 계산하고, 스타일 배열로 기존 `StyleSheet` 스타일과 병합하면 세로/가로 모드 전환에 맞춰 UI를 자연스럽게 조정할 수 있다.
