# 2. maxWidth와 퍼센트 단위로 크기 제어

## 핵심 요약

- 작은 기기에서는 기존 UI 요소가 상대적으로 너무 크게 보일 수 있다.
- 게임 화면의 `Title`처럼 화면 전체 너비를 차지하는 요소는 `width`, `maxWidth`, 퍼센트 단위를 조합해 더 유연하게 만들 수 있다.
- React Native에서도 `width`, `maxWidth`, `minWidth`, `height`, `maxHeight`, `minHeight` 같은 크기 제약 스타일을 사용할 수 있다.
- 퍼센트 값은 항상 해당 요소의 부모 컨테이너를 기준으로 계산된다.
- `width: 300`과 `maxWidth: '80%'`를 함께 쓰면 큰 화면에서는 300px을 쓰고, 작은 화면에서는 부모 너비의 80%까지만 차지하게 만들 수 있다.
- 크기를 줄인 요소가 부모 안에서 가운데 정렬되지 않는다면, 부모 컨테이너의 Flexbox 정렬을 함께 조정해야 한다.

## 학습 목표

- React Native에서 고정 크기와 상대 크기를 함께 사용하는 방법을 이해한다.
- `width`와 `maxWidth`의 차이를 구분한다.
- 퍼센트 단위가 어떤 기준으로 계산되는지 이해한다.
- 부모 컨테이너의 `alignItems`가 자식 요소 정렬에 어떤 영향을 주는지 복습한다.
- 작은 화면 대응의 첫 단계로 요소의 최대 너비를 제한하는 패턴을 익힌다.

## 주요 내용

### 문제 상황: 작은 화면에서 요소가 너무 커 보임

이전 섹션의 게임 화면은 최신 iPhone이나 큰 Android 기기에서는 괜찮아 보였다.

하지만 작은 Android 기기에서는 일부 요소가 답답하게 보인다.

특히 게임 화면의 제목 영역은 다음 문제가 있다.

```text
작은 화면에서의 문제
-> 제목이 화면 상단에 너무 가까움
-> 제목 박스가 화면 너비를 거의 다 차지함
-> 전체 UI가 상대적으로 크게 느껴짐
```

이번 강의에서는 우선 제목의 너비를 줄이면서 반응형 크기 제어를 배우는 데 집중한다.

### Title 컴포넌트 확인하기

게임 화면의 제목은 `GameScreen.js` 안에서 직접 스타일링하지 않는다.

별도 공통 컴포넌트인 `Title`을 사용한다.

```text
GameScreen
-> Title 컴포넌트 사용

components/ui/Title.js
-> 실제 제목 스타일 정의
```

따라서 제목 너비를 조정하려면 `components/ui/Title.js`의 스타일을 수정해야 한다.

이 흐름은 React 컴포넌트 분리의 장점이기도 하다.

```text
공통 제목 스타일 변경
-> Title 컴포넌트를 사용하는 모든 화면에 반영
```

### maxWidth로 최대 너비 제한하기

처음 시도할 수 있는 방법은 `Title` 스타일에 `maxWidth`를 추가하는 것이다.

```jsx
const styles = StyleSheet.create({
  title: {
    // 기존 스타일
    maxWidth: '80%',
  },
});
```

React Native에서 퍼센트 값을 사용할 때는 문자열로 작성한다.

```jsx
maxWidth: '80%'
```

숫자 `80`을 넣으면 80픽셀처럼 처리되지만, `'80%'`는 부모 컨테이너 너비의 80%를 의미한다.

### 요소가 줄어든 뒤 가운데 정렬이 깨지는 이유

`maxWidth: '80%'`를 적용하면 제목 박스는 더 작아진다.

하지만 줄어든 제목이 부모 안에서 가운데 정렬되지 않을 수 있다.

이유는 부모 컨테이너가 자식 요소를 가운데 정렬하고 있지 않기 때문이다.

React Native의 모든 `View`는 Flexbox를 사용하지만, 기본 정렬이 항상 가운데는 아니다.

```text
React Native View 기본값
-> flexDirection: 'column'
-> alignItems: 'stretch'
```

`alignItems: 'stretch'` 상태에서는 자식이 가능한 너비를 채우는 방향으로 배치된다.

제목 자체에 너비 제한을 걸면 더 이상 전체 너비를 채우지 않기 때문에, 부모에서 가로 정렬을 명시해야 한다.

### 부모 컨테이너에서 alignItems 설정하기

게임 화면의 최상위 `View` 스타일에 `alignItems: 'center'`를 추가한다.

```jsx
const styles = StyleSheet.create({
  screen: {
    flex: 1,
    padding: 24,
    alignItems: 'center',
  },
});
```

이렇게 하면 해당 `View` 안의 자식 요소들이 가로축 기준 중앙에 배치된다.

```text
flexDirection 기본값: column
-> 주축: 세로
-> 교차축: 가로

alignItems: 'center'
-> 교차축인 가로 방향 중앙 정렬
```

이전 섹션에서 배운 Flexbox 개념이 그대로 이어진다.

### width와 maxWidth의 차이

`width`와 `maxWidth`는 비슷해 보이지만 역할이 다르다.

```jsx
width: '80%'
```

위 코드는 항상 부모 너비의 80%를 차지한다.

```text
큰 화면
-> 부모 너비의 80%

작은 화면
-> 부모 너비의 80%
```

반면 `maxWidth`는 “최대 이만큼까지만”이라는 제한이다.

```jsx
maxWidth: '80%'
```

이 경우 요소는 필요한 만큼만 차지하되, 부모 너비의 80%를 넘지 않는다.

```text
width
-> 실제 너비를 지정

maxWidth
-> 너비의 상한선을 지정
```

웹 CSS의 `max-width`와 같은 감각으로 이해하면 된다.

### 고정 width와 maxWidth 조합하기

더 실용적인 패턴은 고정 너비와 최대 너비를 함께 사용하는 것이다.

```jsx
title: {
  width: 300,
  maxWidth: '80%',
}
```

이 설정의 의미는 다음과 같다.

```text
기본 너비
-> 300

단, 부모 너비의 80%를 넘을 수 없음
```

큰 화면에서는 300px 너비를 사용할 수 있다.

하지만 작은 화면에서 300px이 부모 너비의 80%보다 크다면, `maxWidth: '80%'`가 상한선으로 작동한다.

```text
큰 화면
-> 300px 사용 가능
-> 좌우 여백 넉넉함

작은 화면
-> 300px이 너무 큼
-> 부모 너비의 80%로 제한
```

웹으로 치면 다음과 비슷한 느낌이다.

```css
.title {
  width: 300px;
  max-width: 80%;
}
```

React Native에서도 이런 조합이 작은 화면 대응에 매우 유용하다.

### 퍼센트 단위의 기준은 부모 컨테이너

React Native에서 퍼센트 값은 항상 부모 컨테이너를 기준으로 계산된다.

예를 들어 `Title`에 다음 스타일을 적용했다고 하자.

```jsx
maxWidth: '80%'
```

이 `80%`는 화면 전체가 아니라 `Title`을 감싸는 부모 `View`의 너비를 기준으로 한다.

```text
GameScreen의 View
-> Title의 부모

Title maxWidth: '80%'
-> GameScreen View 너비의 80%
```

따라서 퍼센트 단위가 예상과 다르게 보인다면 먼저 부모 컨테이너의 크기와 스타일을 확인해야 한다.

```text
퍼센트 값이 이상할 때 확인할 것
-> 부모 View의 width
-> 부모 View의 padding
-> 부모 View의 alignItems
-> 부모 View가 flex: 1인지
```

### minWidth, minHeight, maxHeight도 사용할 수 있다

이번 강의에서는 `width`와 `maxWidth`를 중심으로 다뤘지만, 같은 방식으로 다른 제약 스타일도 사용할 수 있다.

```jsx
minWidth: 200
maxWidth: '80%'
minHeight: 100
maxHeight: 300
```

사용 예시는 다음과 같다.

```text
minWidth
-> 너무 작아지면 터치/가독성이 나빠지는 버튼에 유용

maxWidth
-> 큰 화면에서 카드나 제목이 과하게 넓어지는 것을 방지

minHeight
-> 최소 터치 영역 보장

maxHeight
-> 이미지나 카드가 화면을 너무 많이 차지하는 것 방지
```

모바일 UI에서는 특히 터치 가능한 요소의 최소 크기와 콘텐츠 박스의 최대 크기를 함께 고려해야 한다.

## 프론트엔드 개발자 관점에서의 핵심 차이

### CSS max-width와 거의 같은 사고방식

웹 프론트엔드 경험이 있다면 `maxWidth`는 어렵지 않다.

```css
.card {
  width: 300px;
  max-width: 80%;
}
```

React Native에서는 같은 개념을 JavaScript 스타일 객체로 작성한다.

```jsx
card: {
  width: 300,
  maxWidth: '80%',
}
```

차이는 단위 작성 방식이다.

```text
고정 크기
-> 숫자
-> width: 300

퍼센트 크기
-> 문자열
-> maxWidth: '80%'
```

### 미디어 쿼리 없이도 시작할 수 있다

웹에서는 작은 화면 대응을 바로 미디어 쿼리로 떠올리기 쉽다.

하지만 모든 반응형 처리가 미디어 쿼리 수준의 분기를 필요로 하지는 않는다.

`width`와 `maxWidth`를 잘 조합하는 것만으로도 작은 화면에서 꽤 자연스럽게 줄어드는 UI를 만들 수 있다.

```text
1차 대응
-> width, maxWidth, minWidth, percentage

더 복잡한 대응
-> 화면 크기 측정
-> 조건부 스타일
-> 방향 감지
```

이번 강의는 그중 가장 단순하고 효과적인 첫 단계다.

### 부모 레이아웃과 함께 생각해야 한다

자식의 크기를 줄이면 정렬 문제가 생길 수 있다.

웹에서도 `max-width`를 준 뒤 `margin: 0 auto`로 가운데 정렬하는 것처럼, React Native에서도 부모의 정렬을 함께 봐야 한다.

```text
웹
-> max-width + margin: auto

React Native
-> maxWidth + 부모 alignItems: 'center'
```

크기 제약과 배치 정렬은 항상 같이 다뤄야 한다.

## 정리

이번 단계에서는 작은 화면에서 UI가 너무 넓거나 크게 보이는 문제를 해결하기 위한 첫 도구로 `maxWidth`와 퍼센트 단위를 사용했다.

`width: 300`과 `maxWidth: '80%'`를 조합하면 큰 화면에서는 적절한 고정 너비를 유지하고, 작은 화면에서는 부모 컨테이너 기준 80%까지만 차지하도록 제한할 수 있다.

또한 퍼센트 단위는 항상 부모 컨테이너 기준으로 계산되며, 자식 요소의 크기를 제한한 뒤에는 부모의 `alignItems` 같은 Flexbox 정렬 속성도 함께 조정해야 한다.

이 패턴은 React Native에서 반응형 UI를 만들 때 가장 먼저 사용할 수 있는 단순하지만 강력한 방법이다.
